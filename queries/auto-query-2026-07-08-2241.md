# 적응적 추론의 주체 재정의: 외부 예산 할당에서 내부 판독으로

질문하신 재정의는 Wiki의 지식 곳곳에 산재된 단서들을 하나의 일관된 논증으로 연결하는 매우 정교한 통찰입니다. 단계적으로 검토하겠습니다.

## 1. Wiki가 기록한 적응의 '내부화' 궤적

[[concepts/adaptive-inference.md|adaptive inference]] 엔티티에 기록된 SpecKV의 기여를 보면, 적응의 주체가 이미 이동 중이었습니다:

> 기존 컨텍스트 기반 적응(CADENCE 등)이 **외부 환경**에 반응했다면, SpecKV는 **압축이라는 내부 시스템 상태**에 반응하는 적응의 새로운 유형을 제시한다.

즉, 모델 라우팅(외부 스케줄러가 결정) → 추측 길이 γ 선택(SpecKV가 KV 캐시 압축 상태라는 내부 지표에 반응)으로, 적응의 근거가 점점 모델 내부로 이동해 왔습니다. 그러나 SpecKV 단계에서조차 여전히 **"외부 런타임이 내부 상태를 읽고 γ를 결정한다"**는 구조입니다. 적응의 주체는 여전히 외부 시스템입니다.

## 2. 결정적 단서: "How Much is Left?" 논문

최근 논문 목록에 수록된 *"How Much is Left? LLMs Linearly Encode Their Remaining Output Length"*가 이 논의를 근본적으로 뒤집습니다. 해당 논문의 핵심 발견은:

> LLM은 토큰을 하나씩 생성하지만, 응답은 놀랍도록 일관된 길이 구조를 보인다. 단계별 해답은 예측 가능한 토큰 수에서 수렴하고, 검색은 특정 지점에서 멈춘다.

잔류 스트림에 남은 출력 길이가 **선형적으로 인코딩**되어 있다는 것은, 모델이 매 토큰 생성 시점마다 "나는 앞으로 몇 토큰을 더 생성할 것인가"를 이미 내부 표현으로 보유하고 있다는 뜻입니다. 이는 단순한 부산물이 아닙니다 — 자기 완결성(self-termination)을 가능하게 하는 **기능적 표현**입니다.

## 3. 재정의의 논리적 필연성

이 발견을 SpecKV의 궤적과 결합하면, 질문하신 재정의가 논리적으로 강제됩니다.

**기존 패러다임의 모순:** 외부 시스템이 "이 프롬프트는 복잡하니 더 많은 추론 단계를 할당하라"고 결정하는 것은, 모델이 이미 자신의 잔여 출력 길이를 알고 있음에도 그 정보를 무시하고 외부에서 예산을 강제하는 셈입니다. 이는 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]이 기록한 Crab의 사례와 구조적으로 동일합니다:

> 에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다.

마찬가지로, 모델의 내부 길이 표현(의미론적 상태)을 외부 예산 할당 메커니즘(시스템 제어)으로 번역하는 것이 적응적 추론의 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]입니다.

**재정의된 본질:** 적응적 추론은 "외부 시스템이 계산 예산을 어떻게 할당할 것인가"가 아니라, **"모델이 잔류 스트림에 이미 형성한 출력 길이 예측을 어떻게 판독하고, 이를 추론 아키텍처(추측 디코딩, 사고 단계 분기, 조기 종료 등)에 어떻게 연동할 것인가"**로 재정의되어야 합니다.

## 4. 새로운 개념적 연결

이 재정의를 통해 Wiki의 기존 엔티티들 사이에 새로운 관계가 드러납니다.

**① [[concepts/adaptive-control-hierarchy.md|adaptive control hierarchy]]와의 관계 재설정**
적응의 제어 계층이 "외부 스케줄러 → 모델"에서 "모델 내부 표현 → 판독 계층 → 추론 아키텍처"로 역전됩니다. 제어의 근원(source of control)이 모델 내부로 이동합니다.

**② [[concepts/adaptive-bottleneck-migration.md|adaptive bottleneck migration]]의 새로운 해석**
병목이 "어떤 모델을 선택할 것인가"에서 "γ를 몇으로 할 것인가"로 이동했던 것은, 실제로는 "외부 결정의 병목"이 "내부 표현의 판독 병목"으로 이동하는 과정이었습니다. 다음 단계의 병목은 **잔류 스트림에서 길이 정보를 얼마나 충실하게 추출할 것인가**가 될 것입니다.

**③ [[concepts/agentic-microphysics.md|agentic microphysics]]와의 직접적 연결**
모델이 잔여 출력 길이를 선형적으로 인코딩한다는 것은 에이전트의 미시적 연산 역학(agentic microphysics)에서 자기 모니터링(self-monitoring)이 이미 구현되어 있음을 의미합니다. 이 미시적 표현을 거시적 추론 제어로 어떻게 스케일업할 것인가가 핵심 문제가 됩니다.

## 5. 한 가지 남은 역설

다만, 완전한 재정의를 위해 하나의 역설을 해결해야 합니다. 모델이 "남은 출력 길이"를 인코딩한다는 것은 **현재 추론 경로가 주어졌을 때**의 예측입니다. 그런데 적응적 추론의 목적은 **더 나은 추론 경로를 선택**하는 것입니다. 즉, 모델이 예측하는 길이가 "최적 길이"인지, 아니면 "현재 패턴이 관성적으로 수렴할 길이"인지의 구분이 필요합니다. 이는 단순한 판독의 문제를 넘어, 내부 표현의 **신뢰성 평가** 문제로 확장됩니다 — 이 지점에서 [[concepts/adaptive-validity.md|adaptive validity]]가 새로운 의미를 획득합니다.

## 결론

질문하신 재정의는 정당합니다. Wiki의 [[concepts/adaptive-inference.md|adaptive inference]]가 추적해 온 궤적(외부 환경 반응 → 내부 시스템 상태 반응)과 "How Much is Left?" 논문의 발견(잔류 스트림 내 선형적 길이 인코딩)을 결합하면, 적응의 주체가 외부에서 모델 내부로 이동하는 것은 단순한 경향이 아니라 구조적 필연입니다. 적응적 추론의 본질은 예산 할당이 아니라 **내부 형성된 예측의 판독과 활용**이며, 이는 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]을 에이전트 추론의 핵심 간극으로 격상시킵니다.
