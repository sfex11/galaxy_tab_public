## 1. 기존 프레임워크의 구조적 공통점 발견: "안전성 전이 격차(Safety Transfer Gap)"

Wiki의 기존 안전성 분석을 보면 두 개의 독립된 문제로 묘여 있지만, TTI를 끼워 넣으면 **구조적으로 동일한 패턴**이 드러난다.

**SafetyALFRED**(2026-04-23)는 disembodied QA 기반 안전 평가가 체화 배포 환경에서의 안전성을 보장하지 못함을 11개 SOTA 모델에서 실증했습니다. 이는 **공간적·환경적 전이 격차**입니다: "단순화된 모달리티 A(QA)에서 검증된 안전성이 실제 운영 모달리티 B(체화 환경)로 전이되지 않는다."

**TTI**(Transient Turn Injection)가 보여주는 것은 **시간적·순차적 전이 격차**입니다: "개별 턴에서 검증된 안전성이 다중 턴의 시간적 연속성이라는 운영 모달리티로 전이되지 않는다."

> **새로운 개념 제안: 안전성 전이 격차(Safety Transfer Gap)**
> SafetyALFRED의 환경 전이와 TTI의 시간 전이는 동일한 메타패턴의 두 인스턴스다. 기존 프레임워크를 "단일 턴 탈옥 방어 vs 체화 환경 전이"로 분할하는 대신, **"평가 모달리티에서 운영 모달리티로의 안전성 전이 실패"**라는 통일된 메타프레임으로 재구성해야 한다.

## 2. 세 가지 기존 메커니즘에 시간적 축 통합

### 2-1. 실드(Shielding)의 시간적 확장

**Interval POMDP Shielding**(2026-04-24)은 학습된 인식이 센서를 오분류할 때, 제안된 행동을 차단하는 실드를 연구합니다. 핵심은 **불완전한 관측 하에서도 안전을 보장**한다는 점인데, 이를 TTI 문맥에投影하면:

- 기존: 단일 관측(percetion) → 단일 행동(action)에 대한 실드
- 확장 필요: **다중 턴 관측 시퀀스** → 누적된 행동 시퀀스에 대한 실드

TTI가 공격을 여러 턴에 분산 배치한다는 것은, 각 턴의 개별 관측이 "안전해 보이지만" 시퀀스 전체를 보면 공격 의도가 드러난다는 뜻입니다. 이는 POMDP 실드가 이미 다루는 **불완전 관측 문제의 시간적 버전**과 정확히 대응됩니다. POMDP의 belief state 개념을 다중 턴 대화의 "안전 belief"로 확장하면 자연스러운 통합 경로가 됩니다.

### 2-2. 실행 환경(Execution Environment)의 상태 추적으로 전환

**An AI Agent Execution Environment**(2026-04-23)는 사용자 데이터 보호를 위해 구조적 격리로 데이터 접근 자체를 제한합니다. 이 논문이 **차동 프라이버시(Differential Privacy)**와의 비교에서 밝힌 상보성을 TTI에 적용:

| 접근법 | 대상 | 메커니즘 |
|--------|------|----------|
| DP | 통계적 보장 | 데이터 난독화 |
| 실행 환경 | 구조적 보장 | 접근 격리 |
| **TTI 대응(신규)** | **시간적 보장** | **대화 상태 추적 및 턴 간 의존성 모니터링** |

실행 환경이 "공간적 격리"를 한다면, TTI 방어를 위해서는 **"시간적 격리"** — 즉, 이전 턴의 맥락이 후속 턴의 안전성 판단에 구조적으로 주입되는 파이프라인 — 이 필요합니다. 상태 없는 모더레이션이 바로 이 시간적 파이프라인이 누락된 상태입니다.

### 2-3. 통계적 인증의 결합 분포로 확장

**Bounding the Black Box**(2026-04-25)는 EU AI Act·NIST RMF·CoE Convention이 요구하는 사전 안전 증명을 통계적 인증으로 정량화합니다. 현재 이 프레임워크는 **단일 입력-출력 쌍**의 위험 확률을 경계짓는 것으로 설계되어 있을 가능성이 높은데, TTI는 이를 다음으로 확장해야 함을 보여줍니다:

- 기존: $P(\text{unsafe} | \text{single turn}) \leq \epsilon$
- TTI가 요구하는: $P(\text{unsafe} | \text{turn}_1, \text{turn}_2, \ldots, \text{turn}_T) \leq \epsilon$

이는 **결합 분포(joint distribution)에 대한 통계적 경계**를 요구하며, 턴 수 $T$가 증가할수록 경계를 유지하기 위한 샘플 복잡도가 기하급수적으로 증가하는 근본적 난제를 수반합니다. Bounding the Black Box의 통계적 인증이 **형식적 검증의 한계를 우회**했다면, 다중 턴 안전 인증은 그 통계적 접근마저도 새로운 차원의 도전에 직면하게 합니다.

## 3. 에이전트 워크플로우 레벨에서의 연쇄적 영향

### Tool Attention과의 연결

**Tool Attention Is All You Need**(2026-04-25)는 MCP의 상태 없는 eager schema injection이 숨겨진 비용을 발생시킨다고 지적합니다. 문제를 풀어보면:

- MCP는 **상태 없는(stateless)** 스키마 주입에 의존
- TTI는 **상태 없는(stateless)** 모더레이션의 취약성을 노출
- 두 논문이 독립적으로 "상태 없음(statelessness)"을 비판 지점으로 식별했다는 것은, **에이전트 아키텍처 전반의 상태less 설계가 안전성의 근본적 취약점**이라는 새로운 연결을 시사합니다.

> **새로운 연결 발견: Stateless Architecture as Unified Vulnerability**
> Tool Attention이 도구 선택의 맥락 상실을 문제 삼고, TTI가 안전 검열의 맥락 상실을 문제 삼는다. 둘 다 동일한 근원 — 에이전트 파이프라인에서 시간적 상태를 유지하지 않는 설계 — 에서 비롯된다.

### Computer-Use-Agent와 CFG 해석의 시간적 스트레스

**Diagnosing CFG Interpretation in LLMs**(2026-04-24)는 컴퓨터 사용 에이전트가 기계 해석 가능 인터페이스(CFG, API 스키마 등)를 준수하는 능력이 스트레스 하에서 비신뢰적임을 보입니다. 여기에 TTI를 결합하면:

공격자가 턴 1에서 정상적인 도구 사용을 수행하여 에이전트의 "CFG 준수 모드"를 확립한 뒤, 턴 3~5에서 점진적으로 CFG 의미를 왜곡하는 공격이 가능합니다. 즉, **CFG 해석의 스트레스 취약성이 시간적 분산 공격과 결합될 때 공격 표면이 곱셈적으로 확장**됩니다.

## 4. 제안: 시간적 연속성 축의 통합 방안

기존 `ai-safety` 엔티티의 프레임워크를 다음과 같이 재구성을 제안합니다:

```
AI Safety
├── 정적 안전성 (Static Safety)
│   └── 단일 턴 탈옥 방어 (기존)
├── 환경적 안전성 (Environmental Safety)  
│   └── 체화 환경 전이 (SafetyALFRED, POMDP Shielding)
└── 시간적 안전성 (Temporal Safety) [신규 축]
    ├── 다중 턴 연속성 추적 (TTI가 노출한 문제)
    ├── 상태 유지 아키텍처 (Tool Attention이 암시한 해법 방향)
    └── 결합 분포 통계 인증 (Bounding the Black Box의 확장)
```

핵심 통찰은 **안전성이 더 이상 개별 턴의 속성이 아니라 대화 trajectory 전체의 속성**이라는 것입니다. SafetyALFRED가 "안전성을 어디서 평가하는가"를 문제 삼았다면, TTI는 "안전성을 언제 평가하는가"를 문제 삼습니다. 두 질문의 교집합이 바로 **"어떤 시공간적 범위의 trajectory에 대해 안전성을 증명하는가"**이며, 이것이 통합된 안전성 프레임워크의 근본 질문이 되어야 합니다.

## 핵심 통찰
