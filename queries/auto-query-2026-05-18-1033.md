## 재정의가 타당한 이유: 세 가지 근거

### 1. 인프라 패러다임의 내재적 한계가 이미 노출되어 있다

[[entities/agora-opt.md|agora opt]]의 사례가 이를 명확히 보여준다. 런타임 메모리·토론 구조로 정렬을 달성한다는 것은, 메모리가 "저장-검색"의 역할을 넘어 "계산적 매개체"로 기능함을 의미한다. 그런데 MASPO가 이를 **설계 시점(프롬프트)으로 사전 분산**시킬 수 있다는 발견은, 런타임 메모리 인프라가 수행하던 기능이 사실상 모델 내부로 "컴파일" 가능함을 시사한다. 즉, 인프라로서의 메모리가 해결하려던 문제가 모델 파라미터 공간으로 흡수될 수 있는 구조적 여지가 이미 존재한다.

MeMo는 이 방향성의 극단적 완성이다. 메모리를 별도 인프라가 아니라 **모델 자체의 연장선**으로 배치함으로써, 저장·검색 효율이라는 인프라 최적화 문제를 파라미터 갱신이라는 모델 최적화 문제로 치환한다.

### 2. "동결된 모델 + 외부 메모리" 구조의 근본적 모순

MeMo의 도입부가 정확히 이 지점을 찌른다: *"LLMs are frozen after pretraining until subsequent updates. Many real-world applications require timely, domain-"* — 동결된 본체에 외부 메모리를 부착하는 구조([[entities/adema.md|adema]]의 지식 상태 조직, [[entities/agora-opt.md|agora opt]]의 런타임 메모리 모두 이 패턴)는 본질적으로 **모델의 정적성과 환경의 동적성 사이의 간극을 인프라로 우회하는 것**이다.

이 우회 구조에서 핵심 지표가 "검색 효율"과 "기억 충실도"가 되는 것은 자연스럽다 — 메모리가 모델 밖에 있으니 모델이 그것을 얼마나 잘 "꺼내 쓰는가"가 성능을 결정한다. 그러나 메모리가 모델 자체라면, 문제는 "꺼내 쓰기"가 아니라 **"내재화하고 일반화하기"**가 된다.

### 3. [[concepts/continual-learning.md|continual learning]]과의 구조적 동일성

메모리를 학습 가능한 모델로 정의하는 순간, 메모리 아키텍처의 핵심 문제는 [[concepts/continual-learning.md|continual learning]]의 핵심 문제와 구조적으로 동일해진다:

- **일반화**: 새로운 도메인 지식이 메모리 모델에 통합된 후, 이전 도메인에 대한 추론 능력이 유지되는가?
- **지속적 갱신**: 스트리밍 환경에서 메모리 모델을 점진적 갱신할 수 있는가? 아니면 전체 재학습이 필요한가?

이것은 검색 인덱스의 충실도 문제가 아니라, **모델의 파라미터 공간에서 지식이 어떻게 축적·간섭·소실되는가**의 문제다.

## 그러나 "재정의"가 아니라 "문제 공간의 확장"이 더 정확한 표현

여기서 한 가지 주의할 점이 있다. 질문의 논리를 따라가면 마치 기존 문제(충실도·검색 효율)가 새로운 문제(일반화·갱신)로 **대체**되어야 하는 것처럼 읽히지만, Wiki의 전체 지식 맥락을 보면 이는 **대체가 아니라 확장**이다.

이유는 다음과 같다:

**[[concepts/adaptive-inference.md|adaptive inference]]**의 SpecKV 사례를 보면, 적응적 추론의 결정 차원이 "모델/라우트 선택"에서 "추측 길이(γ) 선택"으로 **확장**되었지 대체되지 않았다. 마찬가지로 메모리 아키텍처도:

| 차원 | 인프라 패러다임 ([[entities/adema.md|adema]], [[entities/agora-opt.md|agora opt]]) | 모델 패러다임 (MeMo) |
|------|------------------------------------------|---------------------|
| 저장 | 외부 저장소에 지식 상태 보존 | 파라미터에 지식 내재화 |
| 검색 | 검색 효율·충실도가 핵심 | 암묵적 접근 (检索 자체가 문제 아님) |
| 갱신 | 메모리 조직 구조 재설계 | [[concepts/continual-learning.md|continual learning]] 문제 |
| 일반화 | 검색된 정보의 모델 통합에 의존 | 메모리 모델 자체의 일반화 능력 |

두 패러다임은 공존할 수 있으며, 실제로 **[[entities/agora-opt.md|agora opt]]/MASPO의 관계가 이미 이 공존의 prototypes**를 보여준다 — 런타임 메모리 인프라와 설계 시점 모델 최적화가 상보적으로 분산되는 구조.

## 새로운 연결 및 발견

이 분석 과정에서 다음 연결이 새롭게 식별됩니다:

### 1. [[concepts/adaptive-forgetting.md|adaptive forgetting]] ↔ 메모리 모델 패러다임의 핵심 메커니즘
인프라 메모리에서 "망각"은 버그(데이터 손실)이지만, MeMo 패러다임에서 [[concepts/adaptive-forgetting.md|adaptive forgetting]]은 **기능**이 된다. 파라미터 공간에서 새 지식을 수용하기 위해 이전 지식을 의도적으로 압축·폐기하는 적응적 메커니즘이 메모리 모델의 일반화와 직결된다.

### 2. [[concepts/agent-knowledge-separation.md|agent knowledge separation]]의 근본적 재검토 필요
인프라 패러다임에서는 에이전트와 지식(메모리)의 분리가 자연스러운 전제다. 그러나 MeMo가 메모리를 모델로 정의하면, [[concepts/agent-knowledge-separation.md|agent knowledge separation]]은 **아키텍처적 분리**가 아니라 **동일 파라미터 공간 내의 기능적 분리**로 재개념화되어야 한다. 이는 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]과도 연결되는데 — 알고리즘적 설계(에이전트-지식 분리)를 시스템 구현(단일 모델 내 메모리)으로 번역할 때의 간극이 여기서 다시 현현한다.

### 3. [[concepts/compensatory-memory-dynamics.md|compensatory memory dynamics]]의 새로운 해석 가능성
이 슬러그가 의미하는 바를 MeMo 렌즈로 재해석하면: 인프라 메모리의 한계(검색 비효율, 충실도 저하)를 보상하기 위해 모델 파라미터가 암묵적으로 기억 역할을 분담하는 **보상적 메모리 역학**이 존재할 수 있다. MeMo는 이를 숨겨진 현상에서 명시적 아키텍처로 끌어올리는 것이다.

### 4. [[concepts/computational-exploration-feasibility.md|computational exploration feasibility]]이 메모리 모델의 실질적 제약
메모리를 모델로 정의하면 이상적으로는 일반화·갱신이 핵심이 되지만, 현실적으로는 **메모리 모델을 얼마나 자주, 얼마나 크게 갱신할 수 있는가**라는 계산적 탐색 가능성의 문제가 병목으로 등장한다. 이는 [[concepts/carbon-efficiency-accuracy-tradeoff.md|carbon efficiency accuracy tradeoff]]와도 연결된다.

## 결론

질문의 직감은 정확하다. MeMo가 메모리를 학습 가능한 모델로 정의하는 것은, 메모리 아키텍처의 핵심 축을 **인프라 최적화(충실도·검색 효율)에서 모델 최적화(일반화·지속적 갱신)로 이동**시킨다. 다만, 이를 [[entities/agora-opt.md|agora opt]]/MASPO의 상보성이 보여주듯 **대체가 아닌 확장**으로 이해하는 것이 Wiki 지식 전체와 더 정합하며, 이 확장을 통해 [[concepts/adaptive-forgetting.md|adaptive forgetting]], [[concepts/agent-knowledge-separation.md|agent knowledge separation]], [[concepts/compensatory-memory-dynamics.md|compensatory memory dynamics]], [[concepts/computational-exploration-feasibility.md|computational exploration feasibility]] 등 기존에 독립적으로 취급되던 개념들이 메모리 모델 패러다임 아래에서 새로운 연결망을 형성하게 된다.

## 핵심 통찰

메모리 아키텍처의 핵심 문제는 충실도·검색 효율에서 일반화·갱신 가능성으로 확장되며, 인프라 패러다임과 모델 패러다임은 대체가 아니라 상보적 공존 구조를 형성한다.
