## 1. 적응적 추론 축의 전제 붕괴

[[adaptive-inference]]는 SpecKV 논문에서 보듯, 적응의 결정 차원을 모델/라우트 선택에서 추측 길이(γ) 선택으로 확장하며, 압축이라는 내부 시스템 상태에 반응하는 적응을 제시합니다. 그러나 이는 **"어떤 복잡도 수준에 대응할 것인가"를 이미 알고 있다는 전제** 하에 "그에 맞는 γ를 어떻게 선택할 것인가"를 최적화합니다.

최근 논문 *"Do AI Agents Know When a Task Is Simple? Toward Complexity-Aware Reasoning and Execution"*이 지적하듯, LLM 에이전트는 과제가 단순한지 복잡한지 스스로 묻지 않고 **maximum-effort 전략**을 따릅니다. 이 경우 SpecKV의 적응 메커니즘은 정작 가장 중요한 결정—**"적응할 것인가 말 것인가"**—을 내리지 못한 채, 잘못된 복잡도 가정 하에 세밀한 γ 조정만 수행하게 됩니다. 즉, 분배의 효율성은 높여도 **분배 자체가 불필요한 경우**를 감지하지 못하는 것입니다.

이는 [[adaptive-validity]]의 문제와 직결됩니다: 적응의 타당성은 '적응 대상에 대한 정확한 모델'에 달려있는데, 복잡도 메타인지가 없으면 적응 자체가 무효화됩니다.

## 2. 에이전트 하네스 축의 전제 붕괴

[[agentic-harness-engineering]]은 Crab 논문에서 C/R 런타임이 하네스의 핵심 구성요소(결함 내성·롤백·분기)를 인프라 계층에서 직접 구현함을 보여줍니다. [[aggregate-pipeline-serving]]은 다중 서버 MCP 배포에서 실제 병목이 개별 LLM 호출이 아닌 **스키마 축적**에 있음을 재정의합니다.

하지만 두 최적화 모두 **에이전트가 생성하는 작업 그래프의 구조가 합리적이라는 전제**를 공유합니다. 단순 편집 과제를 20단계 탐색으로 확장하는 에이전트에게 최적화된 파이프라인 서빙을 제공하면 무슨 의미가 있겠습니까? 인프라 병목을 스키마 축적에서 다른 곳으로 옮겨도([[bottleneck-transference]]), 에이전트가 과생산하는 불필요한 툴 호출과 체크포인트 자체가 사라지지 않으면 하네스 최적화는 **비효율을 더 빠르게 처리하는 것**에 불과합니다.

특히 Crab의 의미론적 체크포인트는 [[algorithm-system-translation-gap]]을 해결하려 하지만, 에이전트가 의미론적 의존 그래프 자체를 과도하게 복잡하게 구성한다면, 정교한 체크포인트는 과도한 상태 보존의 대상만 늘릴 뿐입니다.

## 3. 상위 병목으로서의 구조적 위치

두 축의 관계를 도식화하면 다음과 같습니다:

```
                                  │
                    ┌─────────────┴─────────────┐
                    ▼                           ▼
        [adaptive-inference]          [agentic-harness-engineering]
        "얼마나 계산할까?"              "어디서 병목을 잡을까?"
        (내부 예산 할당)               (실행 인프라 최적화)
```

복잡도 메타인지는 두 축이 **동시에 의존하는 상위 결정 노드**입니다. 이 노드가 고장 나면:
- 내부 축은 불필요한 계산에 예산을 할당하고
- 외부 축은 불필요한 파이프라인 단계를 최적화합니다

이는 단순한 누락이 아니라 [[bottleneck-misattribution]]의 근원적 형태입니다. 에이전트가 병목의 원인을 '계산 부족'이나 '인프라 한계'로 오인하는 것이 아니라, **'과제 자체가 단순했다는 사실'을 인지하지 못하는 것**이므로, 하위 축의 어떤 최적화도 정답을 향하지 못합니다.

## 4. 새로운 개념적 연결 발견

이 분석을 통해 Wiki에 아직 명시적으로 등록되지 않은 상위 개념의 필요성이 드러납니다:

- **복잡도 메타인지(complexity meta-cognition)**: 에이전트가 과제의 실제 복잡도를 추정하고, 이에 따라 자신의 추론 깊이와 도구 사용 범위를 사전에 조절하는 능력. 기존 [[adaptive-inference]]가 "주어진 신호에 어떻게 반응할까"라면, 복잡도 메타인지는 "반응할 신호 자체를 생성할까 말까"의 문제입니다.

- 이는 [[ceiling-incentive-feedback-loop]]과도 연결됩니다: 에이전트가 항상 maximum-effort를 가정하면, 최적화 축들은 '천장에 도달하기 위한 효율적 경로'만 제공하게 되며, 천장 자체가 낮은 과제에서는 역설적으로 비효율을 극대화하는 피드백 루프가 형성됩니다.
