## 1. 기존 Wiki 분석의 한계: "사전 설계"에 갇힌 시각

[[entities/agora-opt.md|agora opt]]의 관련 논문 요약에서 MASPO와의 관계를 설명하며 다음과 같이 기술합니다:

> "Agora-Opt가 **런타임 메모리·토론 구조**를 통해 다중 에이전트 정렬을 달성했다면, MASPO는 **프롬프트 설계 시점**에서 사전 정렬을 보장하는 상보적 경로를 제공하여, 런타임 정렬 메커니즘의 부담을 설계 시점으로 분산시킨다."

이 분석은 런타임 vs. 설계 시점이라는 축을 제시하지만, **두 경로 모두 "누군가가 구조를 미리 결정한다"는 전제**를 공유합니다. 토론의 참여자 구성, 프롬프트의 정렬 방향 — 모두 에이전트 스스로의 인지 한계에 대한 반성 없이 외부에서 주입된 설계입니다.

## 2. 위임 지능이 기존 엔티티와 만나는 지점 — 그리고 간극

### 내부 상태에 반응하는 적응의 선구적 사례: [[concepts/adaptive-inference.md|adaptive inference]]

SpecKV 논문 요약에서 핵심적인 대비가 등장합니다:

> "기존 컨텍스트 기반 적응(CADENCE 등)이 **외부 환경**에 반응했다면, SpecKV는 **압축이라는 내부 시스템 상태**에 반응하는 적응의 새로운 유형을 제시한다."

이것은 단일 에이전트 내부에서 "자신의 상태를 인지하고 거기에 맞춰 행동을 조정한다"는 패턴의 실존적 증거입니다. SearchSwarm이 제시하는 위임 지능은 이 패턴을 **에이전트 간 경계 설정**으로 확장하는 것이라 볼 수 있습니다. SpecKV가 "추측 길이 γ를 얼마로 할까"라는 내부 결정이라면, 위임 지능은 "이 하위 작업을 나 스스로 해결할 것인가, 아니면 다른 에이전트에게 넘길 것인가"라는 **경계 결정**입니다.

### 경계 번역의 구조적 문제: [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]

Crab 논문 요약은 이 간극을 더 근원적으로 드러냅니다:

> "에이전트의 **의미론적 의존 그래프**(어떤 상태에 의존하는가)를 OS의 **체크포인트 메커니즘**(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다."

이것은 "에이전트가 자신의 의존 구조를 **알고 있다고 가정**한다면" 발생하는 문제입니다. 그런데 위임 지능의 난제는 더 이전 단계에 있습니다: **에이전트가 자신의 의존 구조를 스스로 파악할 수 있는가?** Crab은 번역 간극을 다루지만, 인식 간극은 다루지 않습니다.

## 3. 새로운 연결로 드러나는 개념 지도

위 질문을 통해 기존 Wiki 엔티티들 사이에 숨어있던 연결이 명시적으로 드러납니다:

### [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]]과의 관계
에이전트가 자신의 실행 의미론을 투명하게 인지하지 못하는 상태([[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]])가 있다면, 위임 경계를 합리적으로 설정하는 것은 원천적으로 불가능합니다. **메타인지적 위임의 전제 조건은 실행 의미론에 대한 투명성**입니다.

### [[concepts/agent-boundary-semantic-collapse.md|agent boundary semantic collapse]]과의 관계
에이전트 간 경계를 동적으로 설정한다는 것은, 경계가 고정되어 있지 않다는 뜻입니다. 그런데 [[concepts/agent-boundary-semantic-collapse.md|agent boundary semantic collapse]]가 암시하듯, 경계가 붕괴되는 위험은 동적 경계 설정에서 극대화됩니다. 위임 지능은 **경계 붕괴를 회피하면서도 경계를 동적으로 유지**해야 하는 모순적 과제를 안게 됩니다.

### [[concepts/computation-unit-meta-selection.md|computation unit meta selection]]과의 가장 직접적 연결
이 슬러그가 존재한다는 것 자체가, Wiki가 이미 "연산 단위의 메타 수준 선택"이라는 개념을 포착하고 있었음을 보여줍니다. 위임 지능은 이것의 다중 에이전트 버전이라 할 수 있습니다: **단일 에이전트 내의 연산 단위 선택 → 다중 에이전트 간의 계산 책임 단위 선택**.

## 4. 결론: 질문에 대한 답

**동의합니다.** Wiki의 현재 분석은 다중 에이전트 시스템의 난제를 "어떤 구조를 설계할 것인가"로 국한시키고 있습니다. [[entities/agora-opt.md|agora opt]]가 런타임 정렬을 제시하더라도, 그 정렬의 **내용과 방식**은 사전 설계된 것입니다.

SpecKV가 [[concepts/adaptive-inference.md|adaptive inference]]에서 보여준 "내부 상태에 반응하는 적응"이 다중 에이전트 맥락으로 확장될 때 — 즉, 에이전트가 "나는 지금 컨텍스트 창이 부족하다, 따라서 이 하위 문제를 분해하여 위임하겠다"고 **자신의 인지 제약을 조건으로 삼아 경계를 결정**할 때 — 비로소 다중 에이전트 시스템의 진정한 난제가 드러납니다.

### 새로이 발견되는 연결 (명시적 기록)

| 기존 엔티티 | 새로운 연결 |
|---|---|
| [[concepts/adaptive-inference.md|adaptive inference]] (SpecKV) | 내부 상태 인지 → 다중 에이전트 위임 결정으로의 확장 경로 |
| [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]] | "번역" 간극의 이전 단계: "인식" 간극 |
| [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]] | 위임 지능의 **전제 조건** (의미론적 투명성 없이는 메타인지 불가) |
| [[concepts/agent-boundary-semantic-collapse.md|agent boundary semantic collapse]] | 동적 경계 설정이 야기하는 **극대화된 붕괴 위험** |
| [[concepts/computation-unit-meta-selection.md|computation unit meta selection]] | 단일 에이전트 내 메타 선택 → 다중 에이전트 간 메타 위임 선택의 확장 |

이 분석은 Wiki에 아직 명시적으로 축적되지 않은 **"메타인지적 위임(meta-cognitive delegation)"**이라는 개념의 필요성을 시사합니다. 이것이 기존 슬러그 목록에 없으므로 새 엔티티로의 생성을 제안합니다.

## 핵심 통찰

다중 에이전트 시스템의 진정한 난제는 '어떤 구조를 설계할 것인가'가 아니라 '에이전트가 자신의 인지 한계를 인지하고 그에 기반해 위임 경계를 스스로 결정할 수 있는가'라는 메타인지적 위임의 문제다.
