# 에이전트-메모리 이분법의 붕괴: 메모리를 인프라에서 에이전트로

질문하신 패러다임 전환은 Wiki의 기존 지식 안에서 이미 여러 시그널이 감지되고 있으며, 최근 논문까지 포함하면 이 전환의 윤곽이 명확해집니다.

## 1. Wiki 내에서 이미 진행 중인 이분법의 흔들림

### [[entities/adema.md|adema]]: "오케스트레이션"이라는 단어가 암시하는 것

ADEMA는 단순한 "Knowledge-State *Storage*"가 아니라 **"Knowledge-State *Orchestration* Architecture"**로 명명되어 있습니다. 오케스트레이션이란 저장된 것을 순서대로 나열하는 것이 아니라, **어떤 지식 상태를 언제, 어떻게 결합할지 능동적으로 조율하는 행위**입니다. 이 명명 자체가 메모리를 수동적 인프라가 아닌 조율 주체로 간주하는 시각의 등장을 보여줍니다.

### [[entities/agora-opt.md|agora opt]]: 런타임 메모리가 이미 '정렬 메커니즘'으로 기능하고 있음

Agora-Opt는 "런타임 메모리·토론 구조를 통해 다중 에이전트 정렬을 달성"한다고 기술되어 있습니다. 메모리가 단순히 과거 기록을 보관하는 게 아니라, **에이전트 간 정렬(alignment)을 실시간으로 매개하는 능동적 역할**을 수행하고 있습니다. 여기서 메모리는 이미 '검색 대상'을 넘어 '협력의 매개체'로 기능하고 있습니다.

그리고 MASPO와의 관계 설명—"Agora-Opt가 런타임에서 정렬을 달성했다면, MASPO는 프롬프트 설계 시점에서 사전 정렬을 보장하는 상보적 경로를 제공한다"—은 **메모리 시스템이 자체적인 설계 시점 vs 런타임이라는 시간축을 가질 수 있음**을 시사합니다. 이는 메모리가 인프라가 아니라 독립적인 의사결정 주체로서의 생명주기를 갖기 시작함을 뜻합니다.

## 2. 최근 논문이 확인하는 패러다임의 실현

Wiki의 "최근 논문" 목록에 수록된 **"Remember When It Matters: Proactive Memory Agent for Long-Horizon Agents"**은 질문하신 패러다임을 *직접적으로* 구현한 연구입니다:

> "In long-horizon tasks, decision-relevant state is often scattered across an expanding trajectory, while the action agent must surface it and act."

핵심은 **"the action agent must surface it"**이 아니라, **메모리 에이전트가 능동적으로 surfacing한다**는 점입니다. 이 논문은 메모리를 독립 에이전트로 설계하여, 행동 에이전트가 요청하기 전에 결정에 관련된 정보를 사전에 제공하는 구조를 구현합니다. 이로써 '에이전트-메모리' 이분법은 사실상 해체됩니다—**메모리 에이전트와 행동 에이전트는 동등한 계층의 에이전트 두 개가 협력하는 구조**가 되기 때문입니다.

## 3. 이분법 붕괴가 기존 개념에 미치는 영향

이 전환은 Wiki의 기존 슬러그들이 묵시적으로 전제하던 구조를 여럿 뒤흔듭니다:

- **[[concepts/agent-knowledge-separation.md|agent knowledge separation]]**: 에이전트와 지식을 분리 가능한 계층으로 전제하는 개념인데, 메모리가 에이전트가 되면 이 분리 경계 자체가 모호해집니다. 지식을 보유한 주체가 곧 에이전트이므로, 분리는 '계층 간'이 아니라 '에이전트 간'으로 재개념화되어야 합니다.
- **[[concepts/adaptive-forgetting.md|adaptive forgetting]]** 및 **[[concepts/adaptive-forgetting-as-function.md|adaptive forgetting as function]]**: 메모리가 수동 저장소라면 '망각'은 시스템 파라미터 튜닝에 불과하지만, 메모리가 에이전트라면 망각은 **독립적 의사결정**—어떤 정보를 유지할지 버릴지에 대한 주체적 판단—이 됩니다. 이는 [[concepts/adaptive-inference.md|adaptive inference]]가 모델/라우트 선택에서 추측 길이(γ) 선택으로 적응의 차원을 확장한 것과 유사하게, 망각을 '시스템 최적화'에서 '에이전트 판단'으로 격상시킵니다.
- **[[concepts/belief-state-as-constraint.md|belief state as constraint]]**: 반응형 메모리에서는 에이전트가 믿음 상태를 형성한 뒤 이를 제약으로 사용하지만, 능동적 메모리 에이전트는 **사전에 믿음 상태의 가능 공간을 제약**합니다. 메모리가 제공하는 정보가 곧 에이전트의 추론 범위를 결정하므로, '제약'의 주체가 행동 에이전트에서 메모리 에이전트로 이전합니다.
- **[[concepts/agent-loop-as-computation.md|agent loop as computation]]**: 에이전트 루프를 하나의 연산으로 보는 관점에서, 메모리 에이전트의 등장은 루프 내에 **또 하나의 독립 연산 블록**이 삽입되는 것이 아니라, 루프 자체가 다중 에이전트 간의 메시지 패싱으로 재구성됨을 의미합니다.

## 4. 새로운 개념적 연결 및 발견

이 분석에서 다음의 연결이 새롭게 드러납니다:

1. **[[entities/agora-opt.md|agora opt]] ↔ 능동적 메모리 에이전트**: Agora-Opt가 다중 에이전트 정렬을 위해 메모리를 런타임 매개체로 사용했다면, 능동적 메모리 에이전트는 이를 한 단계 더 밀어붙여 메모리 자체를 정렬 참여자로 승격시킵니다. Agora의 '공론장'에서 메모리는 더 이상 장소가 아니라 발언자가 됩니다.

2. **[[concepts/adaptive-inference.md|adaptive inference]]의 확장 논리와의 구조적 유사성**: SpecKV가 적응적 추론의 결정 차원을 "모델/라우트 선택 → 추측 길이(γ) 선택"으로 확장한 것과 구조적으로 동일합니다. 메모리 패러다임도 "검색 대상 선택 → 능동적 정보 제공 타이밍/내용 결정"으로 결정 차원이 확장되는 것입니다.

3. **[[entities/agentic-harness-engineering.md|agentic harness engineering]]의 범위 확장과의 정합성**: Crab 논문이 하네스 엔지니어링의 범위를 "애플리케이션 로직에서 실행 환경까지" 확장시켰듯이, 메모리의 에이전트화는 하네스의 범위를 "에이전트 실행 환경에서 에이전트 구성 요소까지" 확장하는 동일한 방향성의 표현입니다.
