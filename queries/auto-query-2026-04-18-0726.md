### 2. 메모리 전이를 뒷받침하는 Wiki 내 논문 증거들

Wiki의 여러 논문이 이미 도메인 횡단적 메모리 구조의 필요성을 시사하고 있었으나, 현재 프레임에서는 분산되어 연결되지 못했다:

**(a) 구조화된 스킬의 재사용 — D2Skill**
- 이중 세분화 스킬 뱅크(task-level + step-level)를 **정책과 공동 학습**하여 동적 갱신
- 핵심 함의: 스킬이 특정 도메인이 아닌 **세분화 수준(granularity)**으로 조직되면, 동일 step-level 스킬이 다른 도메인의 태스크에 재활용될 수 있다
- transfer-learning, meta-learning 개념과 명시적으로 연결됨

**(b) 에피소딕 메모리의 도메인 독립적 구조 — Chameleon**
- 로봇 조작에서 **원시 지각 단서(raw perceptual cues)**를 보존하는 에피소딕 메모리가 의미 압축 기반보다 우수
- 생물학적 대응: 해마(에피소딕) → 내후각피질(인지 지도) → 전전두피질(작업 기억)
- 함의: 메모리 구조가 도메인 특화가 아닌 **인지 기능별(perceptual, spatial, working)**로 설계되면, 도메인 전이의 기반이 된다

**(c) 적응적 망각의 도메인 비의존성 — Novel Memory Forgetting**
- Recency × Frequency × Semantic Alignment의 3차원 스코링
- 이 망각 메커니즘은 도메인에 무관한 **범용적 메모리 관리 원칙**
- 코딩 에이전트든 대화 에이전트든 동일한 수학적 구조(I(m_i, t) = α·R + β·F + γ·S)로 메모리를 관리

**(d) 제어·메모리·검증의 결합 설계 — SCRAT**
- 메모리를 "미분류 아카이브가 아닌 **제어 자원**"으로 정의
- 5개 상태 요소(체화 상태, 잠재 동역학, 에피소딕 메모리, 관찰자 신념, 태스크 상태) 중 메모리(m_t)는 도메인 독립적으로 구조화
- 역할 분화(제안자/실행자/감사자/적대자)도 도메인이 아닌 **기능(function)**으로 구분

**(e) 이중 트레이스 인코딩 — Drawing on Memory**
- 사실 기록에 **장면 트레이스(scene trace)**를 쌍으로 저장하여 cross-session recall 향상
- 이 인코딩 방식은 도메인 특화가 아닌 **인지 과학적 원리(drawing effect)**에 기반 → 어떤 도메인에서든 적용 가능

**(f) 인식론적 인프라 — Retrieval Is Not Enough**
- 검색 충실도가 아닌 **인식론적 충실도(epistemic fidelity)**가 조직 AI의 천장
- 약속 강도, 모순 상태, 조직적 무지를 구조화 → 이것이 도메인 횡단 메모리 전이의 **의미론적 기반** 역할 가능

### 3. 새로운 연결: "도메인 축" → "인프라 층" 재구성

| 층 | 내용 | 관련 논문 |
|---|---|---|
| **L3: 도메인 특화 능력** | GUI grounding, 물리 내비게이션, 코드 실행 등 | ClawBench, AgentVLN, UI-Voyager |
