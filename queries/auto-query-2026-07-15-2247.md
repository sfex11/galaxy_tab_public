## 1. 모델 단위 안전성의 전이 실패: 평가 환경과 배포 환경의 구조적 단절

[[concepts/ai-safety.md|ai safety]]에 연결된 **SafetyALFRED** 논문은 이 문제의 가장 직접적인 실증입니다. 11개 SOTA 모델이 *disembodied QA* 기반 안전 평가에서는 통과했지만, 체화된 배포 환경에서는 안전성을 보장하지 못했습니다. 이는 안전성 평가 방법론 자체가 "모델이 단독으로 응답하는 상황"에 고정되어 있으며, 에이전트가 환경과 상호작용하는 상황의 안전성을 전혀 포착하지 못함을 구조적으로 노출합니다.

[[concepts/agentic-vlm.md|agentic vlm]]에 기록된 **Three-Step Nav**의 발견은 이를 더욱 날카롭게 합니다. MLLM 기반 에이전트가 '단계별 판단' 능력은 갖추고 있으나 '다단계 경로 기획'에서 구조적 실패(경로 이탈·조기 정지)를 보인다는 실증은, **모델의 단위 능력이 에이전트의 다단계 실행에서 어떻게 붕괴하는지**를 보여줍니다. 범용 프레임워크가 "이 모델은 시각-언어 능력이 높으므로 안전하다"고 분류할 때, 실제 위험은 능력 간 간극에서 발생합니다.

## 2. 구성적 해악(Compositional Harm)의 국소 감시 한계

최근 논문 **"When Local Monitors Miss Compositional Harm: Diagnosing Distributed Backdoors in Multi-Agent Systems"**은 이 질문에 대한 가장 결정적인 증거를 제공합니다. 이 논문은 다중 에이전트·도구 사용 시스템에서 각 메시지, 도구 호출, 단계를 개별적으로 검사하는 런타임 모니터에 **근본적 구멍(fundamental hole)**이 존재함을 증명합니다. 개별 단계는 각각 안전해 보이지만, 그 **조합(composition)**으로서 창발하는 해악을 국소 감시는 포착할 수 없습니다.

이것이 [[entities/agentic-security-investigation.md|agentic security investigation]]의 한계이자 동시에 필요성의 근거입니다. 에이전트 시스템의 위험은 모델 출력의 독립적 분포에서가 아니라, 도구 호출 체인·에이전트 간 메시지 교환·상태 의존성 그래프의 **조합적 공간**에서 발생하는데, 범용 프레임워크는 이 공간을 분류 체계에 포함하지 않습니다.

## 3. 통계적 인증의 모델 고정주의와 창발적 위험의 충돌

[[entities/ai-risk-regulation.md|ai risk regulation]]에 연결된 **"Bounding the Black Box"**는 EU AI Act·NIST RMF·CoE Convention의 규제 요구를 통계적 인증으로 번역하는 최초의 프레임워크를 제공합니다. 이는 규제를 "요구사항 나열"에서 "준수 가능한 기술 표준"으로 전환하는 진전이지만, **그 인증의 대상이 여전히 모델 출력의 통계적 특성에 고정**되어 있습니다.

[[concepts/acceptable-risk-quantification.md|acceptable risk quantification]]의 **"When No Benchmark Exists"**가 정확히 이 지점을 짚습니다. 에이전트 시스템의 위험 — 예컨대 "에이전트 A가 도구 X를 호출한 후 에이전트 B가 그 결과를 해석하여 행동 Y를 취할 때 발생하는 위험" — 은 기존 벤치마크가 존재하지 않는 영역입니다. 벤치마크가 없는 영역에서의 비교 안전성 평가는, [[concepts/benchmark-specification-gap.md|benchmark specification gap]]이 지적하듯, 명세 자체가 에이전트 시스템의 실제 위험을 반영하지 못한 상태에서 이루어집니다.

## 4. 알고리즘-시스템 변환 간극이 드러내는 위험의 소재 이동

[[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]에 기록된 **Crab**의 발견은 위험의 실제 소재가 어디에 있는지를 보여줍니다. 에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율은, **에이전트 시스템의 위험이 알고리즘(모델) 레벨이 아니라 시스템(상태 관리·체크포인트·롤백) 레벨에서 현현**함을 의미합니다. 범용 프레임워크가 분류하는 "모델의 위험 등급"과 실제 "에이전트 시스템의 위험 등급" 사이에는 이 변환 간극만큼의 체계적 괴리가 존재합니다.

### (2) 구성적 해악 논문 → [[concepts/agent-native-immune-system.md|agent native immune system]]의 필요성
"When Local Monitors Miss Compositional Harm"은 [[entities/agentic-security-investigation.md|agentic security investigation]]의 한계를 넘어, **에이전트 네이티브 수준의 면역 체계**([[concepts/agent-native-immune-system.md|agent native immune system]])가 필요함을 시사합니다. 모델 단위의 안전 필터링이 아니라, 에이전트 간 상호작용의 조합적 공간 전체를 모니터링할 수 있는 시스템 수준의 방어가 필요한데, 이는 범용 프레임워크의 분류 체계가 전혀 포착하지 못하는 차원입니다.

### (3) [[concepts/benchmark-specification-gap.md|benchmark specification gap]] → [[concepts/acceptable-risk-quantification.md|acceptable risk quantification]]의 근본적 한계
벤치마크가 명세하지 않은 위험을 양화할 수 없다는 점에서, [[concepts/benchmark-specification-gap.md|benchmark specification gap]]은 [[concepts/acceptable-risk-quantification.md|acceptable risk quantification]]의 방법론적 전제를 뒤흔듭니다. "허용 가능한 위험"을 정량화하려면 먼저 그 위험을 명세하는 벤치마크가 있어야 하는데, 에이전트 시스템의 창발적 위험은 벤치마크 명세 자체가 불가능한 영역(조합적 폭발)에 존재합니다.

## 요약

범용 AI 위험 프레임워크 기반의 분류 체계는 에이전트 시스템의 위험을 **단순히 '부족하게 포착'하는 것이 아니라 '체계적으로 오분류'**합니다. 그 이유는:

1. **평가 대상의 오지정**: 모델 출력 → 에이전트-환경-도구 상호작용 (SafetyALFRED, Three-Step Nav)
2. **위험의 조합적 성질 무시**: 개별 단계 안전 → 구성적 해악 창발 (When Local Monitors Miss Compositional Harm)
3. **인증의 고정점 오류**: 모델 분포의 통계적 특성 → 시스템 수준 의미론적 의존성 (Crab, Bounding the Black Box)
4. **능력-안전 분리 가정의 붕괴**: 분리 가능한 안전 속성 → 불가분한 능력-안전 역학 (capability-safety-inseparability)

에이전트 시스템에 적용 가능한 위험 분류는 모델의 정적 속성이 아니라, **에이전트가 어떤 도구를 사용하며, 어떤 상태 의존성을 가지며, 다른 에이전트와 어떻게 상호작용하는가**에 대한 동적·관계적 분류 체계를 요구합니다. 현재의 프레임워크는 이 전환이 이루어지지 않은 상태에서 에이전트 시스템에 적용되고 있으므로, 오분류는 우연이 아니라 구조적 필연입니다.
