### 1. Crab의 접근: [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]의 극복과 복원 충실도
[[entities/agentic-harness-engineering.md|agentic harness engineering]]의 관점에서 볼 때, Crab 논문("Crab: A Semantics-Aware Checkpoint/Restore Runtime for Agent Sandboxes")이 '복원 충실도'를 최우선 가치로 삼은 것은 에이전트 실행 환경의 근본적인 구조적 한계 때문입니다. 

Wiki의 [[concepts/agent-os-semantic-gap.md|agent os semantic gap]] 엔티티에서 확인할 수 있듯, 기존 OS의 체크포인트 메커니즘은 '전체 스냅샷'을 기반으로 합니다. 이는 에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS 레벨로 번역할 때 발생하는 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]을 초래합니다. Crab은 이 간극을 메우기 위해 C/R 런타임 자체가 에이전트의 의미론을 인지(Semantics-Aware)하도록 설계되었으며, 하네스의 범위를 단순 애플리케이션 로직에서 실행 환경 인프라까지 확장했습니다. 즉, Crab의 목표는 **'불필요한 상태를 배제한 정확한 의미론적 복원'**입니다.

### 2. DeltaBox의 접근: 밀리초급 델타 C/R과 탐색 처리량
반면, 최근 논문 목록에 등장한 "DeltaBox: Scaling Stateful AI Agents with Millisecond-Level Sandbox Checkpoint/Rollback"는 완전히 다른 병목을 공략합니다. DeltaBox는 에이전트가 테스트타임 트리 탐색(Test-time tree search)이나 강화학습을 수행할 때 요구되는 **'고빈도 상태 탐색'**에 초점을 맞춥니다. 

DeltaBox는 Crab처럼 스냅샷의 '의미론적 정교함'을 추구하기보다는, 전체 샌드박스 상태를 밀리초 단위로 델타(Delta) 방식으로 저장하고 롤백하는 '시스템 레벨의 브루트 포스(무차별 대입)적 가속'을 선택했습니다. 이는 상태 분기가 극도로 빈번하게 발생하는 탐색 중심 에이전트에서는 복원의 '정확성'보다 분기의 '처리량(Throughput)'이 절대적 병목이 되기 때문입니다.

### 3. 설계 목표의 재정의가 아닌 스펙트럼의 발견
따라서 설계 목표가 재정의되기보다는, [[concepts/checkpoint-restore-efficiency-spectrum.md|checkpoint restore efficiency spectrum]]라는 새로운 설계 축이 드러났다고 보는 것이 타당합니다.

*   **지능적 최소화(Semantic Minimization) 축**: Crab처럼 상태의 의미를 이해하고 불필요한 것을 덜어내어 정확성과 메모리 효율을 높이는 방식. 장기적이고 복잡한 의존성을 가진 에이전트 세션에 적합.
*   **물리적 가속(Physical Acceleration) 축**: DeltaBox처럼 전체 상태를 델타 인코딩으로 밀리초급으로 통째로 복사하고 롤백하여 탐색의 주기를 극단적으로 단축하는 방식. 넓은 탐색 공간을 단시간에 훑어야 하는 MCTS 등에 적합.

Crab의 의미론적 인식 C/R이 없는 DeltaBox는 불필요하게 큰 상태를 매 밀리초마다 복원하는 부하를 안을 수 있고, DeltaBox의 밀리초급 보장이 없는 Crab은 트리 탐색의 깊이가 깊어질수록 분기 지연으로 인해 탐색 자체가 불가능해질 수 있습니다.

### 💡 새롭게 발견된 개념 및 연결

이 두 논문의 대비를 통해 기존 Wiki에 명시적으로 정의되어 있지 않지만, 다음과 같은 새로운 개념적 연결이 도출됩니다.

1.  **C/R 패러다임의 분기 (Intelligent C/R vs. Brute-force C/R)**: [[entities/agentic-harness-engineering.md|agentic harness engineering]]은 C/R이 단순한 인프라가 아니라 에이전트의 인지 구조(의존성 그래프)와 직결됨을 보여줍니다. Crab은 이를 '이해'하여 최적화하고, DeltaBox는 이를 '외면'하고 속도로 압도하는 두 가지 극단적 패러다임을 제시합니다.
2.  **테스트타임 컴퓨테이션(Test-time Compute) 유형에 따른 하네스 선택**: 탐색 처리량을 극대화하는 얕고 넓은 탐색(Shallow & Broad Search)에는 DeltaBox식 처리량 최적화가, 의미론적 일관성이 중요한 깊은 추론(Deep Reasoning)에는 Crab식 충실도 최적화가 각각 최적의 하네스가 됨을 알 수 있습니다.
3.  **[[concepts/agent-sandbox-state-management.md|agent sandbox state management]]의 이원적 목표 함수**: 상태 관리의 평가 기준이 '상태의 무결성(Integrity)'과 '상태 전환의 지연(Latency)'이라는 서로 trade-off 관계에 있는 두 가지 지표로 정식화될 필요가 있습니다.
