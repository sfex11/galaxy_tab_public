## 1. 병목 지점의 계층적 오귀속 (Meta-level [[concepts/bottleneck-misattribution.md|bottleneck misattribution]])

Wiki는 이미 미시적 수준에서 병목 오귀속의 패턴을 기록하고 있다. [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]에 수록된 Tool Attention 논문은 "다중 서버 MCP 배포에서 파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 **스키마 축적**에 의해 결정됨"을 보여주며, 당연하게 여겨지던 병목 가설을 뒤엎는다.

이와 정확히 동형(isomorphic)인 왜곡이 메타 수준에서 발생한다. Wiki는 프레임워크 생존의 병목을 [[entities/agentic-harness-engineering.md|agentic harness engineering]]의 정교함, [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]의 해소 정도, [[concepts/adaptive-inference.md|adaptive inference]]의 효율성 등 **기술 아키텍처 품질**에 귀속한다. 그러나 "Adoption and Ecosystem Health" 논문이 보여주듯, 실제 병목은 커뮤니티 규모, 마이그레이션 비용, 네트워크 효과 등 **생태계 역학**에 있다. Wiki의 분류 체계는 [[concepts/bottleneck-misattribution.md|bottleneck misattribution]]을 기술 최적화 대상으로만 기록하지만, 자기 자신이 동일한 오귀속을 메타 수준에서 재생산하고 있다.

## 2. 번역 간극의 확장: 기술→생태계로의 비성공적 매핑

[[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]은 Crab 논문의 핵심 발견을 담는다: "에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율." 이는 한 계층의 의미 구조가 하위 계층의 메커니즘으로 손실 없이 번역되지 않음을 보여준다.

동일한 구조가 기술→생태계 방향으로 확장된다. [[entities/agora-opt.md|agora opt]]가 "런타임 메모리·토론 구조를 통해 다중 에이전트 정렬을 달성"하는 기계적 정교함을 기록하더라도, 이것이 프레임워크 **채택**이라는 생태계적 정렬로 번역되지 않는다. [[entities/ai-risk-regulation.md|ai risk regulation]] 항목이 EU AI Act·NIST RMF의 규제 요구를 "준수 가능한 기술 표준"으로 번역하는 경로를 최초로 제시했다고 평가하지만, 규제 준수가 프레임워크 채택의 실제 동력인지, 아니면 채택 이후의 필수 조건인지는 이 번역 간극 내에 묻혀있다.

핵심적으로, [[entities/agentic-harness-engineering.md|agentic harness engineering]]이 "하네스 엔지니어링의 범위를 애플리케이션 로직에서 실행 환경까지 확장"한다고 기록하지만, 생태계 생존의 실제 "하네스"는 문서화 품질, 온보딩 경험, 생태계 파트너십 등 Wiki가 분류하지 않는 차원에 존재한다.

## 3. 적응의 주체 전이 누락 ([[concepts/adaptation-subject-transfer.md|adaptation subject transfer]]의 메타적 적용)

Wiki는 [[concepts/adaptive-inference.md|adaptive inference]]에서 SpecKV가 "적응적 추론의 결정 차원을 모델/라우트 선택에서 추측 길이(γ) 선택으로 확장"하고, "압축이라는 내부 시스템 상태에 반응하는 적응의 새로운 유형을 제시"한다고 기록한다. [[concepts/adaptive-forgetting.md|adaptive forgetting]] 역시 모델 내부의 적응 메커니즘으로 분류된다.

그러나 프레임워크 생존에서 진정으로 결정적인 적응은 **생태계 압력에 대한 프레임워크 위치 조정**이다—다른 프레임워크와의 API 호환성 확보, 커뮤니티 규범 수용, 의존성 생태계 변화에 대한 대응. 이 적응의 주체는 모델이나 런타임이 아니라 프레임워크 자체인데, Wiki의 분류 체계는 [[concepts/adaptation-subject-transfer.md|adaptation subject transfer]]를 기술-시스템 경계 내에서만 인식한다.

[[concepts/agent-environment-generation.md|agent environment generation]]이 Nemobot Games를 통해 "게임 도메인 특화 전략 학습 환경 생성이라는 특화 경로를 제공하여 에이전트 환경 생성의 스펙트럼을 확장"한다고 평가하지만, 프레임워크 자체가 생존하기 위해 자신의 "환경"(생태계)을 어떻게 생성·수정하는지에 대해서는 무음이다.

## 새로이 발견되는 개념적 연결

### ① [[concepts/survivorship-biased-technical-documentation.md|survivorship biased technical documentation]] (신규 개념 제안)
Wiki는 생존한 프레임워크의 기술 아키텍처만 축적한다. 기술적으로 우월했으나 생태계 역학에 의해 소멸한 프레임워크의 아키텍처는 기록되지 않으므로, 기술 우월성과 생존 간의 인과관계가 가공된다. 이는 [[concepts/benchmark-specification-gap.md|benchmark specification gap]]이 벤치마크가 측정하려는 것과 실제로 측정하는 것 사이에 간극이 있듯, Wiki가 "기록하려는 것"(에이전트 기술 진화)과 "실제로 기록하는 것"(생존자의 기술 아키텍처) 사이에 구조적 간극이 있음을 시사한다.

### ② [[concepts/ecosystem-native.md|ecosystem native]] vs [[concepts/agent-native-serving.md|agent native serving]]의 대칭성
[[concepts/agent-native-serving.md|agent native serving]](Pythia 논문)이 에이전트 워크로드에 최적화된 서빙을 기록한다면, 생태계 관점에서의 "native"는 기술 최적화가 아니라 생태계 규범·도구 체인·커뮤니티 프로토콜에 내재적(alignment)인 상태를 의미할 수 있다. 두 "native"의 축이 일치하지 않을 때, 기술적으로 agent-native한 프레임워크가 ecosystem-native하지 않아 도태되는 경로가 Wiki의 분류 체계에서는 보이지 않는다.

### ③ [[concepts/ai-literacy.md|ai literacy]]의 생태계적 재해석
[[concepts/ai-literacy.md|ai literacy]]가 "에이전트 구축 경험을 통해 체득된다"는 Nemobot Games의 실증을 담지만, 이 리터러시의 내용이 **어떤 프레임워크를 사용하는가**에 의해 편향됨이 간과된다. 특정 프레임워크 생태계에서 체득된 리터러시가 해당 프레임워크의 생존을 자가강화하는 락인 효과를 만들며, 이 동적 순환은 기술 분류 체계로 포착 불가능하다.

## 결론: 분류 체계 자체가 생태계 왜곡의 일부

Wiki의 기술 아키텍처 중심 분류는 단순히 생태계 역학을 "놓치는" 것이 아니다. 더 근본적으로, [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]이 "파이프라인 서빙 최적화의 실제 병목 지점을 재정의"하듯, Wiki 자체가 **에이전트 진화의 병목을 어디에 둘 것인가를 재정의**하는 행위자로 기능한다. 기술 아키텍처를 진화의 주축으로 배치하는 분류 결정은, 연구자와 엔지니어의 주의력을 생태계 역학에서 기술 최적화로 유도함으로써, 자기충족적 예언의 형태로 왜곡을 생산한다. "Adoption and Ecosystem Health" 논문이 GitHub 스타가 기술적 판단을 가린다고 경고하는 것과 정확히 동일한 메커니즘이, Wiki의 정교한 기술 분류가 생태계적 판단을 가리는 방식으로 재현된다.
