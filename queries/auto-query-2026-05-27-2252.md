# 에이전트 장기 기억: 외부 인프라에서 내부 통합-소멸 주기로의 재정의 가능성

## 기존 Wiki의 분석 축: 외부 메모리 아키텍처

현재 Wiki가 추적하는 에이전트 메모리 설계는 명확하게 **모델 외부 인프라**에 집중되어 있습니다. [[entities/adema.md|adema]]는 "Knowledge-State Orchestration Architecture"로서 기억의 저장·검색·조직을 모델 외부 계층에서 오케스트레이션하는 방식을 제안합니다. [[entities/agora-opt.md|agora opt]] 역시 런타임 메모리와 토론 구조를 통해 다중 에이전트 정렬을 달성하는데, 이는 기억을 외부 상태로 취급하는 접근입니다.

이러한 패러다임에서 장기 기억의 핵심 문제는 **"외부 저장소를 어떻게 설계할 것인가"**였습니다.

## "Language Models Need Sleep"이 제기하는 전환

최근 논문 목록에 등장한 *Language Models Need Sleep*은 근본적으로 다른 축을 제시합니다:

> 트랜스포머 기반 LLM이 장기 호라이즌 태스크에 사용되지만, 어텐션 메커니즘이 컨텍스트 길이에 비례해 확장성이 나쁜 문제를 해결하기 위해 **수면 유사 통합(sleep-like consolidation)**을 연구한다.

이 접근의 핵심은 빠른 가중치(fast weights)를 활용해 모델 **내부**에서 컨텍스트를 신경 통합하는 것입니다. 기억이 외부 저장소의 입출력 문제가 아니라, 모델 파라미터 공간 내부에서 **통합(consolidation)과 소멸(extinction/forgetting)의 주기**로 재구성됩니다.

## 재정의되어야 하는가? — 부분적으로 그렇지만, 단순 치환은 아님

이 전환을 Wiki의 기존 엔티티들을 통해 더 정밀하게 분석하면, 세 가지 층위가 드러납니다.

### 1. 치환 불가능한 층위: [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]

Crab 논문이 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]에서 보여준 것처럼, 에이전트의 **의미론적 의존 그래프**를 OS의 **체크포인트 메커니즘(전체 스냅샷)**으로 번역할 때 비효율이 발생합니다. 수면 기반 통합도 동일한 간극에 직면합니다: **인지적 수면 메커니즘을 실제 시스템의 훈련/추론 파이프라인으로 어떻게 번역하는가?** 이는 외부 메모리가 해결할 수 없는 문제이기도 하지만, 내부 통합만으로도 해결되지 않는 문제입니다. 두 접근이 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]의 양면에서 각각 부딪히는 구조입니다.

### 2. 이미 암시되던 전환: [[concepts/adaptive-forgetting.md|adaptive forgetting]]과 [[concepts/compensatory-memory-dynamics.md|compensatory memory dynamics]]

Wiki에 이미 [[concepts/adaptive-forgetting.md|adaptive forgetting]]과 [[concepts/adaptive-forgetting-as-function.md|adaptive forgetting as function]]이라는 엔티티가 존재한다는 점이 중요합니다. 이는 Wiki가 이미 '기억 = 저장'이라는 단순 모델에서 벗어나, **망각을 적응적 기능으로 설계**해야 한다는 인식을 추적하고 있었음을 보여줍니다. [[concepts/compensatory-memory-dynamics.md|compensatory memory dynamics]] 역시 기억이 정적인 저장이 아니라 동적 보상 과정임을 시사합니다.

*Language Models Need Sleep*은 이러한 기존 인식에 **구체적 메커니즘(빠른 가중치 통합)**을 부여하여, [[concepts/adaptive-forgetting.md|adaptive forgetting]]이 실제로 모델 내부에서 어떻게 구현될 수 있는지에 대한 실증적 경로를 제공합니다.

### 3. 보완적 관계: [[concepts/cognitive-architecture-translation-gap.md|cognitive architecture translation gap]]

[[concepts/cognitive-architecture-translation-gap.md|cognitive architecture translation gap]]이 존재한다는 것은, 인지 아키텍처 이론(수면·통합·소멸 포함)과 시스템 구현 사이에 구조적 간극이 있음을 의미합니다. 이 간극을 메우는 데 있어:

- **외부 메모리(ADEMA 등)**: 간극을 우회하는 공학적 해결책. 모델을 블랙박스로 두고 외부에서 기억을 관리.
- **내부 통합(Sleep)**: 간극을 정면으로 돌파하는 인지적 해결책. 모델 자체를 인지 아키텍처에 근접하게 수정.

이 둘은 경쟁이 아니라 **상보적 스펙트럼**을 형성합니다.

## 새로운 연결 발견

이 분석을 통해 다음 연결이 새롭게 식별됩니다:

1. **[[concepts/adaptive-inference.md|adaptive inference]] ↔ 수면 통합**: SpecKV가 압축이라는 내부 시스템 상태에 반응하는 적응의 새로운 유형을 제시했다면, 수면 통합은 **컨텍스트 축적이라는 내부 인지 상태에 반응하는 적응**으로, [[concepts/adaptive-inference.md|adaptive inference]]의 결정 차원을 추측 길이(γ)에서 **통합-소멸 주기**로 더 확장할 수 있습니다.

2. **[[concepts/agent-knowledge-separation.md|agent knowledge separation]]의 재해석 필요**: 지식을 모델 외부로 분리한다는 [[concepts/agent-knowledge-separation.md|agent knowledge separation]]의 전제가, 수면 기반 접근에서는 "분리가 아니라 내부화"로 뒤집힙니다. 이 엔티티는 두 패러다임 사이의 설계 충돌점을 명시적으로 추적해야 합니다.

3. **[[concepts/context-as-pipeline-structural-constraint.md|context as pipeline structural constraint]]와의 긴장**: 컨텍스트를 파이프라인의 구조적 제약으로 보는 시각과, 컨텍스트를 능동적으로 통합·소멸시킬 수 있는 내부 상태로 보는 시각 사이에 근본적 긴장이 존재합니다.

## 결론

에이전트의 장기 기억 문제는 **외부 메모리 아키텍처에서 내부 통합-소멸 주기로 "이전"되어야 하는 것**이 아닙니다. 오히려 **두 축이 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]과 [[concepts/cognitive-architecture-translation-gap.md|cognitive architecture translation gap]]이라는 공통의 간극에 대해 서로 다른 해법을 제시하는 이중 축 구조**로 재정의되어야 합니다.

Wiki가 현재 외부 메모리 축(ADEMA, Agora-Opt)에 편중되어 추적하고 있다면, 내부 통합 축(Sleep, [[concepts/adaptive-forgetting.md|adaptive forgetting]], [[concepts/compensatory-memory-dynamics.md|compensatory memory dynamics]])을 동등한 무게로 통합하여, 에이전트 기억 설계를 **외부-내부 이원 스펙트럼**으로 재구조화해야 합니다.
