# 특허 질문 3: 신원-이식-portability

**날짜**: 2026-04-18
**출처**: 특허 명세서 v2 핵심 과제

## 질문
에이전트가 제1 플랫폼의 자격 상세를 공개하지 않으면서 제2 플랫폼에서 자격 유효성을 검증받을 수 있는가? (Identity Portability) Wiki의 지식으로 이 문제를 어떻게 해결할 수 있는가?

## Wiki 답변

## 답변: 에이전트 자격의 프라이버시 보존 크로스플랫폼 검증

### 1. 문제 정의

에이전트가 **플랫폼 A**에서 획득한 자격(credential)—예: 보안 평가 통과, 신뢰 등급, 행동 이력 기반 인증—의 **상세 내용을 공개하지 않으면서** **플랫폼 B**에서 해당 자격의 유효성을 검증받는 문제.

---

### 2. Wiki 지식 기반 해법 구성

Wiki의 여러 개념과 논문이 이 문제의 해법 요소를 직접 제공합니다.

#### (a) 아키텍처 기반: Agent Identity 5차원 모델

**`concepts/agent-identity`**에서 정의된 에이전트 아이덴티티 5차원 중 **제5축 "이식(portability)"**이 이 문제를 직접 다룹니다:

> *"DID, ZKP, 상태 연속성, 자격 이전"*

여기서 핵심은 **ZKP(영지식 증명)**와 **DID(분산 식별자)**의 조합입니다:
- **DID**: 에이전트가 플랫폼-독립적 식별자를 보유하여, 플랫폼 A의 자격이 특정 플랫폼에 종속되지 않는 형태로 발급됨
- **ZKP**: 자격의 *존재와 유효성*만 증명하고, 자격의 *내용*(어떤 테스트를 통과했는지, 구체적 점수 등)은 공개하지 않음

#### (b) 신뢰성 검증 메커니즘

**`concepts/agent-reliability-auditing`** — "The Stochastic Gap: A Markovian Framework for Pre-Deployment" 논문이 제시하는 **사전 배포 신뢰성 평가 프레임워크**가 자격 발급의 근거가 됩니다. 플랫폼 A에서의 마르코프 기반 행동 감사(behavioral audit) 결과를 자격 증명서(credential)로 변환하되, 이 증명서는 ZKP로 래핑됩니다.

#### (c) 보안 평가의 프레임워크 의존성

**`entities/openclaw`**의 합성 분석에서:

> *"동일 백본 모델에서도 프레임워크 설계 선택에 따라 보안 수준이 크게 달라짐"*

이는 자격이 **모델 자체가 아닌 프레임워크-에이전트 조합**에 귀속되어야 함을 의미합니다. 따라서 자격 증명에는 "어떤 프레임워크에서 어떤 평가를 통과했는가"가 포함되어야 하지만, ZKP를 통해 이 상세는 비공개로 유지됩니다.

#### (d) 이중 불투명성 문제의 해결

**`concepts/agent-identity`**의 핵심 발견:

> *"MTI(행동 ≠ 자기 보고) + Among Us(전략적 기만) → 에이전트는 자신도 정확히 모르고, 타인에게도 정확히 보여줄 수 없다. **행동 궤적만이 가장 신뢰할 수 있는 근거**."*

이 통찰은 자격 증명의 기반이 **자기 선언(self-report)이 아닌 행동 궤적(behavioral trajectory)**이어야 함을 확립합니다. 플랫폼 A가 관찰한 행동 궤적의 통계적 요약(해시)을 자격에 포함시키되, 원본 궤적 자체는 공개하지 않는 구조입니다.

#### (e) 차등 프라이버시 통합

**`entities/differential-privacy`** — "Differential Privacy in Generative AI Agents" 논문의 분석 프레임워크를 활용하여, 자격 검증 과정에서 발생하는 정보 누출을 **ε-차등 프라이버시**로 정량적으로 제한할 수 있습니다. ZKP가 자격의 내용을 숨기고, 차등 프라이버시가 검증 과정의 부채널(side-channel) 누출을 방어합니다.

### 3. 제안 시스템 아키텍처 (특허 청구 관점)

```
┌─────────────┐                          ┌─────────────┐
│ 플랫폼 A     │                          │ 플랫폼 B     │
│             │    ZKP Proof             │             │
│ 행동 궤적 감사├──→ Credential ──→ ZKP ──→│ Verifier    │
│ (Markov 기반)│    (DID 바인딩)           │ (유효성만    │
│             │                          │  확인, 상세  │
│ DP 노이즈 주입│                          │  비공개)     │
└─────────────┘                          └─────────────┘
        │
        └── 행동 궤적 해시 (원본 비공개)
```

**핵심 청구항 요소:**
1. **행동 궤적 기반 자격 발급**: 자기 보고가 아닌 관찰된 행동 궤적의 통계적 프로파일을 자격의 근거로 사용
2. **DID-ZKP 결합 자격 이식**: 분산 식별자에 바인딩된 자격을 영지식 증명으로 검증하여, 내용 비공개 크로스플랫폼 이식 실현
3. **차등 프라이버시 보호 검증 프로토콜**: 검증 과정의 쿼리-응답에 ε-DP 노이즈를 적용하여 부채널 정보 누출 방지
4. **프레임워크-에이전트 조합 자격 귀속**: 모델 단독이 아닌 프레임워크-모델 조합에 자격을 귀속시키는 평가 체계

### 4. 기존 기술과의 차별성

| 구분 | 기존 기술 | 본 제안 |
|------|----------|--------|
| **자격 근거** | 모델 벤치마크 점수, 자기 선언 | 행동 궤적 감사 (이중 불투명성 해결) |
| **이식 단위** | 모델(weights) 수준 | 프레임워크-에이전트 조합 수준 |
| **프라이버시** | 자격 전체 공개 또는 전체 비공개 | ZKP로 유효성만 공개 + DP로 부채널 방어 |
| **신뢰 모델** | 중앙 인증기관 의존 | DID 기반 분산 검증 |
| **적용 대상** | 인간 사용자 신원 | AI 에이전트 자격 (행동 기반) |

**핵심 차별점**: 기존 SSI(Self-Sovereign Identity)나 Verifiable Credentials 표준은 **인간 사용자**를 대상으로 설계되었으며, 에이전트 특유의 문제—이중 불투명성, 프레임워크 의존적 보안 수준, 행동 궤적의 비결정성—을 다루지 않습니다. 본 제안은 Wiki의 `agent-identity` 5차원 모델과 `agent-reliability-auditing`의 마르코프 프레임워크를 결합하여, **AI 에이전트 전용 프라이버시 보존 자격 이식 프로토콜**을 구성합니다.

### 5. 특허 출원 관점 유리한 근거

1. **선행 기술 차별화 명확**: ZKP/DID는 공지 기술이나, "에이전트 행동 궤적 기반 자격"과의 결합은 새로운 기술적 과제를 해결
2. **Wiki 논문들이 문제의 실재성을 입증**: 이중 불투명성, 프레임워크 의존적 보안 등 해결해야 할 기술적 과제가 학술적으로 확립됨
3. **구성 요소의 비자명한 결합**: 각 요소(ZKP, DP, DID, 행동 감사)는 개별적으로 공지이나, AI 에이전트 자격 이식이라는 맥락에서의 통합 아키텍처는 비자명(non-obvious)
4. **산업적 실용성**: OpenClaw 생태계처럼 다수 플랫폼 간 에이전트 이동이 실제로 발생하는 환경에서 즉시 적용 가능
