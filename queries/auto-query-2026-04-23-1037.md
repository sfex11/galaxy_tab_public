# LPSR가 안전성/무결성 이분법을 무너뜨리는 방식

## 1. 기존 이분법의 구조

Wiki의 `llm` 항목에 따르면, 현재 LLM 안전성 연구는 다음 두 축으로 정리됩니다:

- **Box Maze (사전 제어)**: "RLHF와 출력 필터링을 넘어 **추론 프로세스 자체의 무결성을 아키텍처 수준에서 강제**하는 프로세스 제어 패러다임"
- **TraceSafe (사후 평가)**: "**가드레일 기반 사후 평가 프레임워크**"로 아키텍처적 제어를 보완

핵심 구조는 **"생성 전에 아키텍처로 막느냐(Box Maze), 생성 후에 평가하느냐(TraceSafe)"**입니다. 생성 과정 자체는 블랙박스로 남아있습니다.

## 2. LPSR이 제기하는 제3의 축: 추론 중간 개입(Intra-generative Intervention)

최근 논문 **"Latent Phase-Shift Rollback"**은 이 이분법이 성립하는 전제 자체를 무너뜨립니다.

> "Large language models frequently commit **unrecoverable reasoning errors mid-generation**: once a wrong step is taken, subsequent tokens compound the mistake rather than correct it."

이 논문이 밝히는 문제는 정확히 기존 이분법의 사각지대입니다:
- **Box Maze**는 오류가 발생하지 않도록 아키텍처를 **사전에** 설계하지만, 복잡한 다단계 추론에서 모든 오류 경로를 사전에 차단할 수 없다는 한계가 있습니다.
- **TraceSafe**는 오류가 **이미 완료된 후** 평가하지만, "회복 불가능한 추론 오류(unrecoverable reasoning errors)"는 한 번 잘못된 단계를 밟으면 이후 토큰들이 오류를 **복합화**하므로, 사후 평가 시점에서는 이미 복구 불가능합니다.

LPSR은 **잔류 스트림(residual stream)을 실시간 모니터링**하여 오류를 생성 **중간에** 감지하고, **KV-Cache를 조작해 되감기(rollback)**를 수행합니다. 이는 사전 제어도 사후 평가도 아닌 **생성 중간의 개입**이라는 제3의 범주를 창출합니다.

## 3. 이분법 붕괴의 구체적 메커니즘

### 3-1. "감지"의 시점 이동: 잔류 스트림 모니터링

Box Maze는 **구조적 제약**으로 오류 발생 가능성을 원천 차단하고, TraceSafe는 **출력 텍스트**를 사후 분석합니다. 반면 LPSR의 잔류 스트림 모니터링은 **토큰이 생성되는 순간의 내부 표현(activation)**을 실시간으로 읽습니다. 이는 감지 대상이 "아키텍처 설계"도 "최종 출력"도 아닌 **진행 중인 추론의 잠재적 상태**임을 의미합니다.

### 3-2. "수정"의 방식 이동: KV-Cache 조작에 의한 되감기

TraceSafe의 사후 평가는 오류를 **발견**할 뿐 **수정**하지 않습니다(재생성은 별도 프로세스). Box Maze는 오류가 나올 구조 자체를 **막습니다**. LPSR의 KV-Cache 조작은 이미 잘못된 단계가 생성된 이후, 캐시를 조작해 **해당 단계로 되돌아가 다시 생성**합니다. 이는 수정이 사후 재시도도 아닌 **동일 생성 세션 내의 내부 상태 조작**이라는 점에서 두 축 모두에 속하지 않습니다.

### 3-3. "회복 불가능성"이라는 새로운 실패 모드의 발견

LPSR 논문이 도입한 **"unrecoverable reasoning errors"**라는 개념 자체가 이분법의 균열입니다. 기존 프레임워크에서는 오류를 "막아야 할 것(사전)" 또는 "평가해야 할 것(사후)"으로만 보았는데, LPSR은 오류를 **"생성 도중 복합화되어 스스로 복구 불가능해지는 동적 과정"**으로 재정의합니다. 이 동적 과정에 개입하려면 사전/사후 이분법은 구조적으로 부적합합니다.

## 4. 새로운 개념 연결: reasoning-integrity와의 접합

Wiki의 `reasoning-integrity` 항목에 연결된 논문 **"Discovering a Shared Logical Subspace: Steering LLM Logical Reasoning via Alignment of Natural-Language and Symbolic Views"**과 LPSR 사이에 중요한 연결점이 발견됩니다.

> 해당 논문은 "자연어 형태의 추론 연쇄를 순수하게 다듬거나 기호적 솔버를 외부 도구로 부착하는 기존 접근"을 넘어, **자연어 관점과 기호적 관점의 정렬(alignment)을 통해 공유된 논리적 부분공간(shared logical subspace)을 발견**하고 이를 통해 추론을 조향(steering)한다고 합니다.

이 "논리적 부분공간"이라는 개념이 LPSR의 **잔류 스트림 모니터링**과 직접 연결됩니다:
- 공유된 논리적 부분공간이 존재한다면, 잔류 스트림에서 **이 부분공간으로부터의 이탈**을 실시간으로 감지할 수 있어야 합니다.
- LPSR의 잔류 스트림 모니터링은 사실상 이 부분공간의 **추론 중간 실시간 진단 도구**로 기능할 수 있습니다.
- 즉, "논리적 부분공간 발견(사전/이론적)" → "잔류 스트림 모니터링(실시간 감지)" → "KV-Cache 되감기(실시간 수정)"로 이어지는 **완전한 파이프라인**이 구성됩니다.

**🔍 새로운 연결 발견**: `reasoning-integrity`의 "shared logical subspace" 개념은 LPSR의 residual stream monitoring에 대한 **이론적 기반**을 제공할 수 있으며, 반대로 LPSR의 실시간 모니터링 데이터는 논리적 부분공간의 존재를 **경험적으로 검증**하는 수단이 될 수 있습니다. Wiki에 아직 이 두 논문 간의 연결이 명시되어 있지 않으므로 새로운 개념 연결입니다.

이분법의 붕괴를 수용하면, Wiki의 `llm` 항목에 제시된 안전성 분석 축은 다음과 같이 재구성되어야 합니다:

| 축 | 대표 사례 | 개입 시점 | 개입 대상 |
|---|---|---|---|
| **구조적 사전 제어** | Box Maze | 생성 전 (아키텍처 설계) | 프로세스 구조 |

LPSR은 이 표에서 중간에 위치하지만, 단순히 "시점이 중간"인 것이 아니라 **감지(잔류 스트림)와 수정(KV-Cache 조작)이 모두 내부 표현 공간에서 발생**한다는 점에서 차원이 다른 접근입니다. Box Maze가 "바깥에서 구조를 강제"하고, TraceSafe가 "바깥에서 출력을 검사"한다면, LPSR은 **"안에서 흐르는 추론의 내부 강물을 직접 다루는"** 방식입니다.

## 요약

LPSR은 (1) 회복 불가능한 추론 오류라는 기존 프레임워크가 간과한 실패 모드를 식별하고, (2) 잔류 스트림 모니터링과 KV-Cache 조작이라는 **생성 중간 내부 개입** 메커니즘을 도입함으로써, Box Maze-TraceSafe 이분법이 암묵적으로 전제한 "생성 과정은 블랙박스"라는 가정을 해체합니다. 또한 `reasoning-integrity`의 "shared logical subspace" 개념과 결합할 때, **"논리적 부분공간 이탈을 실시간 감지→내부 상태 되감기→재생성"**이라는 사전-중간-사후를 아우르는 통합 무결성 보장 파이프라인의 가능성을 열어줍니다.

## 핵심 통찰

사전 제어(Box Maze)와 사후 평가(TraceSafe)의 이분법은 '회복 불가능한 추론 오류'라는 실패 모드를 설명할 수 없으며, 잔류 스트림 모니터링과 KV-Cache 되감기를 결합한 추론 중간 개입이 shared logical subspace의 이론적 기반과 결합하여 완전한 실시간 추론 무결성 파이프라인을 구성한다.
