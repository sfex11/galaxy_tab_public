# Crab의 "의미론적 인식"의 실제 범위: 인프라 의미론에 갇혀 있는가?

## 결론부터: 현재 Wiki 지식에 따르면 인프라 의미론 수준에 갇혀 있음

Crab의 논문 초록 자체가 상태의 스코프를 명시적으로 제한하고 있습니다:

> "Autonomous agents act through sandboxed containers and microVMs whose state spans **filesystems, processes, and runtime artifacts**."

이 세 가지(filesystems, processes, runtime artifacts)는 모두 OS/인프라 계층의 의미론입니다. 권한 부여(permission assignment)나 제어 흐름의 상태 전이(control flow state transition)는 에이전트의 **논리적 실행 의미론**에 속하는데, 이가 포착 범위에 명시적으로 포함되어 있지 않습니다.

## [[entities/algorithm-system-translation-gap]]이 이 문제를 정확히 진단함

Wiki의 이 엔티티는 Crab이 직면한 근본적 한계를 다음과 같이 서술합니다:

> "에이전트의 **의미론적 의존 그래프**(어떤 상태에 의존하는가)를 OS의 **체크포인트 메커니즘**(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다."

이 진단에서 핵심은 **"번역"**이라는 단어입니다. Crab이 인식하는 의미론은 에이전트 관점의 의미론이 아니라, OS가 스냅샷으로 직렬화할 수 있는 인프라 의미론입니다. 진정한 "semantics-aware"라면 의존 그래프를 **그 자체로** 포착해야 하지만, 실제로는 그것을 OS 체크포인트 메커니즘으로 **번역(축소)**하는 과정에서 간극이 발생한다는 것입니다.

## 새로운 개념 연결 발견

### 1. [[entities/agentic-security-investigation]]와의 간극

[[entities/agentic-security-investigation]] 엔티티는 "Towards Agentic Investigation of Security Alerts" 논문과 연결되어 있습니다. 보안 조사에서 핵심적으로 추적해야 하는 것은 **"어떤 권한이 언제 어떤 맥락에서 부여·승계되었는가"**입니다. Crab의 스냅샷이 이를 포착하지 못한다면, 보안 조사 에이전트는 체크포인트를 복원하더라도 **권한 전이의 causal chain을 재구성할 수 없게 됩니다**. 이는 [[entities/agentic-security-investigation]]의 실질적 한계로 직결됩니다.

### 2. [[entities/agentic-harness-engineering]]의 근본적 과제로서의 재해석

[[entities/agentic-harness-engineering]]은 Crab이 직접적으로 연결된 엔티티입니다. 하지만 위 분석에 따르면, 이 엔티티가 실제로 해결해야 할 과제는 "인프라 상태의 체크포인트/복원"이 아니라 **"에이전트의 논리적 실행 상태를 인프라 상태로 충실하게 매핑하는 번역 계층의 설계"**입니다. 즉, 현재 [[entities/agentic-harness-engineering]]의 정의가 "Wiki 축적 중"으로 비어 있는데, 이 간극을 메우는 것이 이 엔티티의 핵심 내용이 되어야 합니다.

### 3. 제안: 인프라 의미론과 논리적 실행 의미론의 분리라는 새로운 개념

Wiki에 현재 존재하지 않지만, 이 분석에서 도출되는 새로운 개념적 구분이 필요합니다:

- **Infrastructure-level semantics**: 파일시스템, 프로세스, 메모리 스냅샷 (Crab이 포착)
- **Agent-level execution semantics**: 권한 승계 그래프, 제어 흐름 의존성, 도구 호출의 causal chain (Crab이 포착 불가)

이 분리는 [[entities/algorithm-system-translation-gap]]이 이미 암시하고 있지만 명시적으로 분류되지 않은 차원입니다. Crab의 "semantics-aware"라는 명칭이 이 두 계층 중 어느 것을 지칭하는지 모호성을 내포하고 있으며, 논문 초록의 "filesystems, processes, runtime artifacts"라는 열거는 이것이 **전자(인프라 계층)**에 갇혀 있음을 시사합니다.

## 요약

Crab의 semantics-awareness는 **에이전트 관점의 의미론이 아니라 OS 관점의 의미론**입니다. [[entities/algorithm-system-translation-gap]]이 정확히 지적하듯, 이는 "번역" 과정에서 발생하는 비효율의 문제이며, 권한 부여와 제어 흐름의 상태 전이와 같은 논리적 실행 의미론은 현재 스냅샷 범위 밖에 있습니다. 이는 [[entities/agentic-security-investigation]]의 실현 가능성에 구조적 제약을 부과합니다.

## 핵심 통찰

Crab의 'semantics-aware'는 에이전트 관점의 의미론이 아니라 OS 관점의 인프라 의미론에 갇혀 있어, 권한 승계나 제어 흐름과 같은 논리적 실행 상태를 스냅샷으로 포착할 수 없다.
