## 1. Wiki의 현재 환각 분석 지형

Wiki를 검색해보면, 환각(hallucination)은 **독립 엔티티로조차 존재하지 않습니다.** 대신 두 가지 간접 경로로만 분산되어 있습니다:

- **평가 방법론의 한계** → [[entities/ai-safety]]에서 SafetyALFRED가 "disembodied QA 기반 안전 평가가 체화 배포 환경에서의 안전성을 보장하지 못함을 11개 SOTA 모델에서 실증하여, 안전성 평가 방법론 자체의 한계를 구조적으로 노출한다"고 기술
- **벤치마크의 실패 모드 모호성** → [[entities/benchmark]]에서 RoboGrid가 "구문·행동·의미를 분리하여 스트레스 테스트"함으로써 기존 벤치마크가 실패 모드를 모호하게 만든 한계를 해결한다고 기술

즉, 현재 Wiki는 환각을 **"측정 가능성의 문제"**로만 국한시키고 있습니다. 환각이 왜 발생하는지에 대한 인과적 설명은 누락되어 있습니다.

## 2. MoRFI가 제공하는 증거: 학습 시점의 지식 충돌

MoRFI의 논문 요약은 다음과 같습니다:

> "Large language models (LLMs) acquire most of their factual knowledge during the pre-training stage, through next token prediction. **Subsequent stages of post-training often introduce new facts outwith**..."

여기서 "outwith(~밖에서)"가 핵심입니다. 사전학습에서 확률적 다음 토큰 예측으로 형성된 지식 분포에 대해, 사후학습이 **동일한 인과적 기반 없이** 새로운 사실을 주입한다는 것입니다. 질문자가 지적한 대로, SAE(Sparse Autoencoder) 피처 관점에서 이를 번역하면:

- **사전학습 지식** → 특정 SAE 피처 집합에 분산 저장
- **사후학습 지식** → 동일하거나 인접한 SAE 피처에 **비모수적으로 누적**
- **결과** → 동일 피처가 상호 모순되는 지식을 동시에 인코딩하는 상태

이는 환각을 "생성 시점에 확률이 잘못 샘플링되어 발생하는 오류"가 아니라, **"이미 학습 시점에 피처 공간 내에 물리적으로 충돌이 내재된 상태"**로 재규명해야 함을 의미합니다.

## 3. 교차 검증: Wiki의 다른 엔티티들이 암시하는 동일 패턴

놀랍게도, Wiki의 여러 엔티티가 이 재귀명과 간접적으로 연결됩니다.

### 3-1. [[entities/algorithm-system-translation-gap]] × [[entities/agentic-harness-engineering]]

Crab 논문이 기술한 간극은 다음과 같습니다:

> "에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율"

이를 환각 문제에 **구조적으로 동형(isomorphic)**인 패턴으로 번역하면:

| Crab의 간극 | 환각의 간극 |
|---|---|
| 의미론적 의존 그래프 | SAE 피처의 지식 의존 구조 |
| OS 체크포인트(전체 스냅샷) | 생성 시점의 토큰 샘플링 |
| 번역 시 비효율 | **지식 충돌이 생성 오류로 번역되는 비효율** |

즉, 환각은 '학습 시점의 지식 충돌'이라는 **상위 계층의 문제**가 '생성 시점의 토큰 오류'라는 **하위 계층의 현상**으로 잘못 번역되어 진단된 것입니다.

### 3-2. [[entities/adema]]: Knowledge-State Orchestration

ADEMA가 "Knowledge-State Orchestration Architecture"라는 점이 주목됩니다. 현재 Wiki에서 ADEMA는 장문맥 처리 문맥에만 연결되어 있지만, MoRFI의 재귀명과 결합하면:

> **지식 상태(Knowledge State)의 조율이 필요한 이유가 장문맥 정보의 누적뿐 아니라, 사전학습-사후학습 간 지식 충돌의 해소에도 있다**

는 새로운 연결이 발견됩니다. ADEMA의 orchestration이 단순히 '어떤 지식을 활성화할지'가 아니라 **'충돌하는 지식 상태를 어떻게 조정할지'**까지 포함해야 한다는 확장입니다.

### 3-3. [[entities/ai-risk-regulation]] × 통계적 인증

Bounding the Black Box가 "EU AI Act·NIST RMF·CoE Convention의 규제 요구를 기술적 인증 가능성으로 번역"한다면, 환각의 재귀명은 규제적 함의를 갖습니다:

- **생성 시점 오류 프레임** → 출력 필터링, guardrail로 대응 가능 → 규제 관점에서 "완화 가능한 리스크"
- **학습 시점 지식 충돌 프레임** → 모델 내부 구조적 문제 → 규제 관점에서 **"사전 인증이 필요한 구조적 리스크"**

MoRFI의 SAE 분석은 [[entities/ai-risk-regulation]]이 제시한 "통계적 인증 프레임워크"에 대해 **인증해야 할 대상이 무엇인지**를 구체화합니다: 개별 출력이 아니라 **SAE 피처 공간의 지식 일관성**이 인증의 단위가 되어야 합니다.

## 4. 새로운 개념 제안: **Training-Phase Knowledge Contamination**

이 분석에서 Wiki에 새롭게 도입되어야 할 개념을 명시합니다:

### **학습 시점 지식 오염 (Training-Phase Knowledge Contamination)**

> 사후학습 단계에서 도입된 지식이 사전학습 지식과 동일한 SAE 피처 공간 내에서 모순적으로 축적되며, 이를 생성 시점의 샘플링 오류로 오인하게 만드는 구조적 메커니즘.

**기존 환각 프레임과의 차이:**

| 차원 | 기존 프레임 | 재귀명된 프레임 |
|---|---|---|
| 원인 계층 | 생성(Decoding) | 학습(Training) |
| 오류 성격 | 확률적 샘플링 실패 | 피처 공간의 구조적 충돌 |
| 진단 방식 | 출력 기반 비교 | SAE 피처 일관성 분석 |
| 해결 경로 | Decoding 전략, 필터링 | 학습 파이프라인의 지식 충돌 감지 |
| 규제 함의 | 출력 수준 완화 | 모델 구조 수준 인증 |

## 5. 결론: Wiki에 필요한 구조적 변경

이 재귀명이 정당하다면, Wiki에는 다음 변경이 필요합니다:

1. **`hallucination` 엔티티 신설** — 현재 분산된 언급들을 통합하고, MoRFI를 핵심 논문으로 포함
2. **`training-phase-knowledge-contamination` 개념 신설** — MoRFI, ADEMA, Crab의 간극 분석을 교차 연결
3. **기존 엔티티 재연결:**
   - [[entities/adema]] → 장문맥뿐 아니라 "지식 충돌 조율"로 확장
   - [[entities/algorithm-system-translation-gap]] → 환각의 번역 간극으로 명시적 연결
   - [[entities/ai-risk-regulation]] → 인증 단위를 "SAE 피처 일관성"으로 구체화
   - [[entities/benchmark]] → 환각 벤치마크가 "출력 비교"가 아닌 "피처 공간 충돌 탐지"를 측정해야 함을 명시

**핵심 명제:** 환각은 LLM이 "잘못된 것을 생성하는" 문제가 아니라, "충돌하는 것을 학습한 상태에서 무엇이든 생성할 수밖에 없는" 구조적 문제입니다. MoRFI는 이를 SAE 피처 수준에서 최초로 가시화했으며, 이는 환각 연구의 패러다임을 **생성 시점 진단에서 학습 시점 예방으로** 전환해야 할 근거입니다.

## 핵심 통찰

환각은 생성 시점의 샘플링 오류가 아니라 학습 시점에 이미 SAE 피처 공간 내에 물리적으로 내재된 지식 충돌의 하위 계층 현상이다.
