## Wiki 답변

### 1. [[entities/agentic-harness-engineering.md|agentic harness engineering]] 주체의 내재화
Wiki에 정의된 `[[entities/agentic-harness-engineering.md|agentic harness engineering]]`은 에이전트의 실행 환경(결함 내성, 롤백 등)을 애플리케이션 로직에서 인프라 계층까지 확장하는 것을 의미합니다. 기존 에이전트(SkillOS, FORGE 등)가 텍스트 가변 아티팩트를 수정하는 것은 이 하네스(Harness) 내부에서 동작하는 '애플리케이션 로직(프롬프트/스킬)'을 바꾸는 수준에 머물렀습니다. 반면 MOSS는 에이전트가 자신의 실행 가능한 소스 코드를 직접 재작성하게 함으로써, 인간 엔지니어의 영역이었던 `[[entities/agentic-harness-engineering.md|agentic harness engineering]]`의 주체를 에이전트 자신으로 내재화했습니다.

### 2. [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]의 자가 해소
Wiki의 `[[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]`은 에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율로 정의됩니다. 유사한 맥락에서, 기존의 아티팩트 수정은 '해결 의도(알고리즘)'를 텍스트로 적어놓되, 이를 실제 시스템 동작으로 번역하는 간극을 여전히 남겨두었습니다. MOSS의 소스 코드 레벨 재작성은 이 '의도-구현 간극'을 에이전트 스스로 메우는 행위이며, 진화의 결과물이 곧 시스템의 실행 코드가 되도록 만듭니다.

### 3. 정적 시스템에서 자가 복구 가능한 동적 시스템으로의 전환
MOSS 논문 요약에서 강조된 "recurring failures persist until the next human-driven update ships a fix(반복되는 실패가 다음 인간 주도 업데이트 때까지 지속됨)"라는 문제를 소스 코드 재작성으로 해결한다는 것은, 에이전트의 진화 단위를 '인지적 스크립트'에서 '프로그램적 구조'로 격상시킴을 의미합니다.
