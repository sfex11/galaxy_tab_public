# 차원별 최적화가 구조적 부채를 은폐하고 확률적 세금을 증가시키는 역설 진단

## 1. 역설의 발생 메커니즘: 세 가지 상호보완적 진단 경로

### 경로 ① 병목점 오귀속(Bottleneck Misattribution)에 의한 부채의 이동

[[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]에 연결된 "Tool Attention Is All You Need" 논문은 이 역설의 핵심 증거를 제공합니다. 다중 서버 MCP 배포에서 **파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 스키마 축적에 의해 결정된다**는 실증은, LLM 추론 자체를 최적화하는 것이 실제 병목을 해결하지 않고 **다른 계층으로 부채를 밀어넣는 것**임을 보여줍니다.

이때 [[concepts/bottleneck-misattribution.md|bottleneck misattribution]]이 발생합니다: 파이프라인 서빙 차원에서 스키마 로딩을 최적화하면 병목이 C/R 계층으로 이동하고, C/R을 최적화하면 다시 상태 관리 계층으로 이동합니다. 각 차원의 최적화는 지역적 개선을 달성하지만, **부채가 사라지는 것이 아니라 순환 이동할 뿐**입니다. 최종적으로 어느 한 계층에 누적된 부채가 임계점을 넘으면, 개별 최적화의 효과가 상쇄되는 구조적 한계에 직면합니다.

### 경로 ② 알고리즘-시스템 변환 간극의 가려움

[[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]에서 Crab 논문이 구체화한 문제가 핵심입니다. 에이전트의 의미론적 의존 그래프(어떤 상태가 무엇에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질인데, C/R 속도를 높이는 최적화는 이 **번역 자체의 부정확성을 은폐**합니다.

[[entities/agentic-harness-engineering.md|agentic harness engineering]]에서 Crab이 하네스의 핵심 구성요소(결함 내성·롤백·분기)를 C/R 런타임이 직접 구현하게 하여 인프라 계층까지 범위를 확장한 것은 진전이지만, 동시에 [[concepts/agent-os-semantic-preservation-failure.md|agent os semantic preservation failure]]의 리스크를 증가시킵니다. 스냅샷이 빨라질수록 "의미론이 보존되었는가"를 검증할 동기가 약화되고, 복원된 상태가 논리적으로 일관성 있는지에 대한 [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]]이 증가합니다.

즉, C/R 최적화는 **"빠르게 잘못된 상태로 복원"하는 능력**을 향상시키며, 이는 확률적 세금(잘못된 상태에서 재추론해야 하는 비용)을 줄이는 대신 **잘못되었음을 인지하지 못하게 만드는** 더 위험한 형태의 세금으로 전환됩니다.

### 경로 ③ 적응적 추론 레이어의 비선형적 상호작용

[[concepts/adaptive-inference.md|adaptive inference]]에서 SpecKV 논문이 제시한 패러다임 확장이 역설의 세 번지 경로입니다. 기존 컨텍스트 기반 적응(CADENCE 등)이 외부 환경에 반응했다면, SpecKV는 압축이라는 **내부 시스템 상태에 반응하는 적응**을 추가합니다.

문제는 이 두 적응 축이 독립적으로 최적화될 때 발생합니다:
- **외부 환경 적응**은 툴 호출 빈도를 조절하고 → [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]의 스키마 축적 패턴을 변경합니다
- **내부 상태 적응**은 추측 길이(γ)를 조절하고 → C/R의 체크포인트 크기와 빈도를 변경합니다

각각은 지역적으로 최적이지만, **두 적응 축이 동시에 작동할 때 파이프라인 전체의 상태 전이가 예측 불가능해집니다**. [[concepts/adaptive-bottleneck-migration.md|adaptive bottleneck migration]]이 의도치 않게 발생하여, 어느 적응 레이어도 자신이 유발한 전이 효과를 추적할 수 없는 상태가 됩니다.

## 2. 역설의 존재론: 왜 각 최적화가 "역설적"인가

세 경로가 공유하는 구조적 패턴을 도출할 수 있습니다.

**단축의 국소성-전역성 불일치**: 각 차원(C/R, 파이프라인, 적응 추론)의 최적화는 자신의 계층 내에서는 결정론적 개선을 달성하지만, 에이전트 루프 전체에서는 [[concepts/agent-loop-as-computation.md|agent loop as computation]]의 확률적 특성을 변경할 뿐입니다. 최근 논문 "Modeling Agentic Technical Debt and Stochastic Tax"가 제시한 프레임워크대로, 각 단축이 감소시키는 결정론적 비용보다 증가하는 확률적 비용이 클 때 역설이 성립합니다.

**가시성의 역전**: 최적화가 성공할수록 [[concepts/auditable-gap.md|auditable gap]]이 widening됩니다. C/R이 빨라지면 체크포인트 간의 상태 차이를 감사하기 어려워지고, 파이프라인이 최적화되면 각 스테이지의 입출력을 개별적으로 검증하기 어려워집니다. [[concepts/agent-reliability-auditing.md|agent reliability auditing]]이 가능하려면 오히려 일정 수준의 '느림'이 필요한 역설적 상황이 됩니다.

## 4. 진단 프레임워크 제안

이 역설을 체계적으로 진단하려면 세 차원의 최적화가 공유하는 **상태 전이 그래프**를 통합적으로 모니터링해야 합니다:

1. **부채 이동 추적**: [[concepts/bottleneck-misattribution.md|bottleneck misattribution]]을 실시간으로 탐지하기 위해, 파이프라인 각 스테이지의 대기 시간 분포(단순 평균이 아닌 분산과 왜도)를 추적하여 병목이 어느 계층으로 이동했는지 가시화
2. **의미론 보존 검증 계층 분리**: C/R 최적화와 의미론 보존 검증을 동일한 런타임이 아닌 독립 계층에서 수행하여, [[concepts/agent-os-semantic-preservation-failure.md|agent os semantic preservation failure]]의 은폐를 방지
3. **적응 레이어 간 간섭 측정**: 외부 환경 적응과 내부 상태 적응이 동시에 활성화될 때의 [[concepts/computational-complexity.md|computational complexity]] 변화를 측정하여, 비선형적 상호작용이 임계점을 넘는지 모니터링

결론적으로, 차원별 최적화가 부채를 은폐하거나 세금을 증가시키는 역설은 각 최적화가 **자신이 변경하는 것(국소 비용)과 자신이 변경하지 않지만 영향받는 것(전역 확률적 구조)을 구분하지 않기 때문**에 발생합니다. 최근 "Governed Evolution of Agent Runtimes through Executable Operational Cognition"이 코드를 실행 가능한 인지 기질로 처리해야 한다고 주장하는 것도, 이 국소-전역 불일치를 인지 기질 수준에서 통합하려는 시도로 읽을 수 있습니다.

## 핵심 통찰

각 차원의 최적화는 부채를 소멸시키는 것이 아니라 형태를 변환하며 순환 이동시킬 뿐이며, 최적화가 성공할수록 오히려 감사 가능성이 저하되는 가시성 역전이 발생한다.
