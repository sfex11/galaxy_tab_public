## 1. 하네스가 '중립적 인프라'가 아닌 경우의 근거

특히 연구 수명주기 전체를 벤치마킹하는 논문("Act As a Real Researcher")의 관점에서 보면, 하네스가 내장한 탐색 전략·검증 루프·롤백 정책은 모델의 '지능'이 발현되는 조건 자체가 됩니다. 이때 모델만 떼어내어 평가하는 것은, 엔진만 떼어내어 자동차 성능을 논하는 것과 다름없어 보입니다.

**[[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]**의 Tool Attention 논문은, 다중 서버 MCP 배포에서 파이프라인 전체의 레이턴시 병목이 "개별 LLM 호출이 아닌 스키마 축적"에 의해 결정됨을 보여줍니다. 이는 하네스 최적화의 병목 지점이 모델 추론의 병목 지점과 **직교함(orthogonal)**을 실증합니다. 모델이 바뀌어도 스키마 축적 문제는 그대로 남고, 하네스가 바뀌어도 모델의 추론 능력은 그대로 남습니다.

**[[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]**에서도, Crab은 에이전트의 의미론적 의존 그래프를 OS의 체크포인트 메커니즘으로 번역할 때 발생하는 비효율을 간극의 본질로 규정합니다. 이는 모델 측(의미론적 의존)과 시스템 측(스냅샷)이 **서로 다른 추상화 계층에 존재**하며, 이 계층 경계를 분석적으로 식별하는 것이 오히려 최적화의 핵심임을 시사합니다.

연구 에이전트 벤치마킹에서 하네스 차이가 동일 모델의 능력을 극적으로 변화시킨다면, 이는 알고리즘(모델 추론)과 시스템(하네스 인프라) 사이의 경계가 **특정 도메인(연구 수명주기)에서 붕괴**함을 의미합니다. 그러나 이 붕괴가 전역적으로 일어나는 것은 아닙니다:

## 4. 결론: 분리가 아닌 **결합체의 분해 가능성**을 묻는 것으로 전환해야

모델-하네스 분리 분석이 무의미해지는 것이 아니라, 평가 대상을 **"모델 단독 능력"과 "하네스 단독 효율"에서 "모델-하네스 결합체의 분해 가능성(decomposability)"**으로 전환해야 합니다.

- 결합체가 **분해 불가능(inseparable)**하다면 → [[concepts/algorithm-system-boundary-collapse.md|algorithm system boundary collapse]] 상태이며, 평가는 결합체 단위로 수행해야 함
- 결합체가 **분해 가능(decomposable)**하다면 → [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]처럼 직교 최적화가 가능하며, 분리 분석이 여전히 유효함

즉, "Act As a Real Researcher"가 보여주는 현상은 분리 분석의 무의미성을 증명하는 것이 아니라, **[[concepts/benchmark-specification-gap.md|benchmark specification gap]]**을 새로운 차원에서 드러내는 것입니다: 벤치마크가 "무엇을 측정하는가"를 명세할 때, 그 측정 대상이 모델인지 하네스인지 아니면 둘의 결합체인지를 명시하지 않으면, 비교 평가 자체가 무의미해집니다. 이 명세 불가능성이야말로 연구 에이전트 평가의 근본적 난제입니다.
