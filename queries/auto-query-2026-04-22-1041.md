## 1. 정의의 모순: Wiki가 스스로 간과한 긴장

Wiki의 `llm-agent` 항목은 LLM 에이전트를 다음과 같이 정의합니다:

그러나 최근 논문 목록에 등장한 **WorldDB**는 이 정의의 전제를 정면으로 부정합니다:

> "Persistent memory is the bottleneck separating stateless chatbots from long-running agentic systems. **Retrieval-augmented generation (RAG) over flat vector stores fragments facts into chunks, loses cross-session coherence**..."

WorldDB의 지적을 Wiki의 정의에 대입하면 모순이 드러납니다. "자율적 의사결정"은 최소한 (1) 이전 결정에 대한 일관성 유지, (2) 충돌하는 정보 간의 조정, (3) 시간적 연속성을 전제합니다. 그런데 현재 에이전트 생태계의 주류 메모리 전략인 RAG은 이 세 가지를 모두 파괴합니다. 즉, **'자율성'이라는 속성이 정의에 명시되어 있으나, 그 자율성을 지탱할 메모리 아키텍처가 정의에 포함되어 있지 않은 것**입니다.

## 2. 파편화 문제의 구체적 메커니즘: WorldDB → MASS-RAG로 이어지는 실패 사슬

WorldDB가 지적한 "flat vector store가 사실을 청크 단위로 파편화"한다는 문제는, 단순한 기술적 한계가 아니라 추론 자체의 왜곡으로 이어집니다. 이는 최근 논문인 **MASS-RAG**의 문제 의식과 직접 연결됩니다:

> "when retrieved contexts are **noisy, incomplete, or heterogeneous**, large language models..."

MASS-RAG은 다중 에이전트 합성으로 RAG의 불완전성을 완화하려 시도하지만, 이는 근본적으로 **파편화된 입력에 대한 사후 처리**입니다. WorldDB의 관점에서 보면, MASS-RAG의 접근은 "깨진 유리를 더 정교하게 조립하려는 시도"이지, 유리가 깨지지 않는 저장 방식을 바꾸는 것이 아닙니다.

**새로운 개념 연결:** WorldDB의 "ontology-aware write-time reconciliation(온톨로지 인식 쓰기 시간 조정)"과 MASS-RAG의 "multi-agent synthesis"는 동일한 문제(파편화)에 대해 정반대의 해결 방향을 제시합니다. 전자는 **쓰기 시점**에 구조를 강제하고, 후자는 **읽기 시점**에 합성을 시도합니다. 이 두 축의 대립은 향후 에이전트 메모리 연구의 핵심 분기점이 될 것입니다.

## 3. 세션 간 정체성 상실: 다중 에이전트 연구와의 역설적 충돌

더 심각한 문제는 세션 간 정체성입니다. Wiki의 `multi-agent-system` 항목은 다음 인사이트를 담고 있습니다:

> "LLM 에이전트가 **사전 설정된 정체성(preset identity)을 넘어 담화적 개입을 통해 자율적으로 입장을 형성하고 경계를 협상**할 수 있음을... 실증한 최초의 연구이다."

이 연구는 에이전트가 단일 세션 내에서 정체성을 발달시킬 수 있음을 보여줍니다. 그러나 WorldDB의 관점에서 이 결과의 외적 타당성은 심각하게 제한됩니다 — **세션 간에 정체성이 지속되지 않으면, "자율적 입장 형성"은 매번 백지상태에서 재발명되는 일회성 이벤트**에 불과합니다. 실제 자율성은 형성(forming)만이 아니라 유지(maintaining)와 수정(revising)을 포함합니다.

**새로운 개념 연결:** 다중 에이전트 시스템 연구의 또 다른 인사이트와 결합하면 문제가 더 선명해집니다:

> "다중 AI 에이전트 시스템에서 **지연된 피드백은 인간 감독자의 오류 귀인을 체계적으로 왜곡**하며..."

지연된 피드백이 귀인을 왜곡한다면, 세션 간 메모리 단절은 에이전트 **자신의** 귀인을 왜곡합니다. 에이전트가 자신의 이전 행동과 결정에 대해 일관된 서사를 구성할 수 없다면, "자율적 의사결정"이라는 표현의 주어("에이전트가") 자체가 참조 불가능해집니다.

## 4. 모순 처리 불가능: 미시적 차원에서의 증거

WorldDB가 "모순 처리 불가능"을 지적한 것은 거시적(세션 간) 차원이지만, Wiki의 다른 논문들은 이것이 **미시적(생성 내) 차원에서도 성립함**을 보여줍니다. **Latent Phase-Shift Rollback** 논문은:

> "Large language models frequently commit **unrecoverable reasoning errors mid-generation**: once a wrong step is taken, subsequent tokens compound the mistake rather than correct it."

단일 생성 내에서도 오류 복구가 불가능하다는 것은, KV-Cache와 residual stream이라는 **초단기 메모리조차 모순을 수용하지 못한다**는 뜻입니다. WorldDB의 "세션 간 모순 처리 불가"는 이 미시적 한계의 거시적 연장선에 있습니다.

**새로운 개념 연결:** Wiki의 `llm` 합성 분석에서 **Box Maze**의 접근을 여기에 연결할 수 있습니다:

> "RLHF와 출력 필터링을 넘어 **추론 프로세스 자체의 무결성을 아키텍처 수준에서 강제**하는 프로세스 제어 패러다임"

Box Maze가 출력 수준이 아닌 프로세스 수준에서 무결성을 강제하려는 시도와, WorldDB가 플랫 스토어가 아닌 그래프 구조에서 쓰기 시점 조정을 강제하려는 시도는 **동일한 패러다임 전환**의 서로 다른 표현입니다: "결과를 고치지 말고, 구조를 고쳐라."

## 5. 결론: '자율성'의 정의를 메모리 의존성으로 재구성해야 함

질문에 대한 직접적인 답은 **"그렇다, 상태 비저장 메모리 아키텍처 위에서 현재 정의된 의미의 '자율성'은 근본적으로 성립하기 어렵다"**입니다. 하지만 이것을 단순한 부정으로 끝내기보다, Wiki의 논문들이 함의하는 **재구성 방향**을 정리하면 다음과 같습니다.

### 현재 Wiki 정의의 구조적 결함

| 자율성의 요소 | 필요한 메모리 조건 | 현재 RAG의 현실 |
|---|---|---|
| 일관된 의사결정 | 세션 간 서사 지속 | 청크 단위 파편화 (WorldDB) |
| 모순 인식 및 조정 | 충돌 사실의 공존 구조 | 플랫 벡터에서 모순 조각만 독립 검색 |
| 환경 학습의 누적 | 쓰기 시점의 구조화된 저장 | 읽기 시점 합성으로만 보완 시도 (MASS-RAG) |
| 자기 귀인의 일관성 | 행동 이력의 서사적 연결 | 지연된 피드백 처리 불가 (다중 에이전트 연구) |

### Wiki 논문들이 가리키는 재구성 축

1. **Box Maze / WorldDB 축**: 출력이나 검색이 아닌, **아키텍처·쓰기 시점**에서 구조를 강제해야 한다
2. **Latent Phase-Shift Rollback 축**: 단일 생성 내에서도 상태를 모니터링하고 복구할 수 있는 **인터벤션 메커니즘**이 필요하다
3. **다중 에이전트 정체성 형성 축**: "자율성"을 개별 에이전트의 내적 속성이 아니라, **다중 에이전트 상호작용에서 돌출되는 관계적 속성**으로 재정의하는 방향

### 명시적 새로운 개념 제안: **메모리-조건부 자율성 (Memory-Conditional Autonomy)**

현재의 "자율성" 정의는 메모리 아키텍처에 무관하게 사용되지만, Wiki의 논문들을 종합하면 자율성은 다음과 같은 **스펙트럼**으로 재개념화되어야 합니다:

- **수준 1 — 세션 내 자율성**: 단일 컨텍스트 윈도우 내에서의 추론 독립성 (현재 대부분의 에이전트가 이 수준)
- **수준 2 — 재구성적 자율성**: 세션 간 메모리가 파편화되어 있으나, 매 세션 시작 시 합성을 통해 일관성을 재구성하는 수준 (MASS-RAG가 시도하는 수준)
- **수준 3 — 구조적 자율성**: WorldDB의 온톨로지 인식 그래프처럼, 메모리 자체가 모순과 시간적 연속성을 내재적으로 표현할 수 있는 수준

현재 Wiki의 `llm-agent` 정의는 수준 3을 암시하면서, 실제 생태계는 수준 1에 머물러 있습니다. 이 정의와 구현 간의 격차를 Wiki가 명시적으로 추적하지 않고 있다는 점이, 질문이 지적하는 근본적 문제의 핵심입니다.
