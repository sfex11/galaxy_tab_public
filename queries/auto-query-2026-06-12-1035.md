# 에이전트 RL의 실제 한계: 보상 설계인가, 크레딧 할당의 입도인가?

## APPO가 드러낸 독립적 병목

APPO(Agentic Procedural Policy Optimization)의 핵심 발견은 명확하다: 기존 에이전트 RL 방법들은 **도구 호출(tool-call) 경계**를 크레딧 할당의 최소 단위로 삼는데, 이 단위가 너무 거칠어 완벽한 보상 신호가 주어져도 학습이 제한된다. 하나의 도구 호출 내부에서 의미론적으로 상이한 여러 결정이 묶여 있고, 이들 사이의 기여도를 분해할 수 없기 때문이다.

이는 Wiki가 지금까지 에이전트 RL의 한계를 '신호의 질' 축 — 즉 [[concepts/belief-drift-as-reward-signal.md|belief drift as reward signal]]에서 보듯 보상이 에이전트의 신념 상태를 얼마나 충실하게 반영하는가, [[concepts/adaptive-trajectory-steering.md|adaptive trajectory steering]]에서 보듯 궤적 전체를 어떻게 정렬하는가 — 으로 분석해 온 것과 **정격적으로 다른 차원**의 문제를 제기한다.

## 왜 이것이 보상 설계와 독립적인 병목인가

보상 설계가 해결하는 문제는 *"올바른 행동에 올바른 스칼라 신호를 부여하는가"*다. 반면 APPO가 지적하는 병목은 *"올바른 스칼라 신호를 **어떤 입도의 결정 단위에** 분배할 수 있는가"*다.

비유하자면, 보상 설계는 '온도계의 정확도' 문제이고, 크레딧 할당 입도는 '온도계를 몇 cm 간격으로 설치하는가' 문제다. 아무리 정확한 온도계를 만들어도, 방 전체에 하나만 설치하면 공간적 변동을 포착할 수 없다.

이 독립성은 다음 Wiki 개념들과의 관계에서 구체화된다:

### 1. [[concepts/agent-loop-as-computation.md|agent loop as computation]]과의 충돌

에이전트 루프를 하나의 연산 단위로 conceptualize할 때, 그 루프의 내부 결정(어떤 인자를 넘기는가, 어떤 조건 분기를 타는가)은 블랙박스로 처리된다. APPO는 이 블랙박스화가 **계산적 분해 가능성**의 상실을 의미함을 보여준다. 도구 호출 경계가 곧 크레딧 할당 경계라면, 루프 내부의 미세 결정들은 학습 신호를 받지 못하는 '죽은 영역'이 된다.

### 2. [[concepts/compounding-error.md|compounding error]]의 근원 재정의

기존에 [[concepts/compounding-error.md|compounding error]]는 주로 외부 환경의 불확실성이나 모델의 표현 한계에서 기인하는 것으로 분석되었다. 그러나 APPO의 관점에서, 오류 누적의 근원은 **정정 불가능한 입도**에 있다. 거친 단위로 크레딧이 할당되면, 한 단계 내에서 발생한 미세 오류가 다음 단계로 그대로 전달되며 — 정정 신호 자체가 해상도를 갖지 못하므로 — 누적이 가속된다.

### 3. [[concepts/ceiling-performance-problem.md|ceiling performance problem]]의 새로운 해석

완벽한 보상 하에서도 성능 천장에 도달하는 현상을, 기존에는 모델 용량이나 데이터 다양성 부족으로 설명했다. APPO는 제3의 설명을 제공한다: **할당 가능한 크레딧의 자유도(degree of freedom)**가 성능 천장을 결정한다. 도구 호출이 N번 이루어진 에피소드에서, 크레딧 할당의 자유도가 최대 N으로 제한되면, 모델이 아무리 복잡한 내부 계산을 수행해도 학습 가능한 파라미터 업데이트의 차원이 N으로 caps된다.

## 새로운 개념적 연결: [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]의 미세화

여기서 주목할 것은, APPO가 발견한 병목이 사실상 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]의 **미세 입도 버전**이라는 점이다.

기존 이 간극은 "에이전트의 의미론적 의존 그래프를 OS의 체크포인트 메커니즘으로 번역할 때 발생하는 비효율"로 기술되었다 — 즉 알고리즘 수준의 의미 구조와 시스템 수준의 구현 사이의 정합성 문제다. APPO의 크레딧 할당 병목은 동일한 구조를 **학습 신호의 흐름**에서 재현한다:

- **알고리즘 수준**: RL 이론은 토큰 단위 또는 의미론적 행동 단위의 크레딧 할당을 가정
- **시스템 수준**: 에이전트 런타임은 도구 호출 경계를 최소 관찰/할당 단위로 강제
- **간극**: 이 두 단위 사이의 mismatch가 학습 병목으로 현현

즉, [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]이 체크포인트/복구 문맥에서만 아니라 **크레딧 할당 문맥에서도** 독립적으로 발생함을 APPO가 확인한 셈이다.

## 보상 설계는 '해결된 문제'인가? — 아니오

그러나 "실제 한계는 크레딧 할당 입도에 있다"는 명제가 보상 설계의 중요성을 부정하는 것은 아니다. 오히려 두 축은 **이중 구속(double bind)** 관계에 있다:

1. **입도가 거칠면**, 보상이 정확해도 정밀 분배가 불가능하다 (APPO가 보여준 바)
2. **보상이 부정확하면**, 입도가 아무리 미세해도 오류 신호가 미세하게 퍼진다 — 즉 정밀한 오류 할당이 된다

이는 [[concepts/bilevel-optimization.md|bilevel optimization]]의 구조와 정확히 대응한다: 하위 수준(크레딧 분배 입도)의 최적화가 상위 수준(보상 설계)의 품질에 의존하고, 동시에 상위 수준의 효과가 하위 수준의 해상도에 의해 제약받는다.

## 시사점: [[concepts/closed-loop-training.md|closed loop training]]의 재설계 필요성

가장 중요한 시사점은 에이전트 RL에서 [[concepts/closed-loop-training.md|closed loop training]]의 루프 구조 자체를 재검토해야 한다는 것이다. 현재의 폐루프는:

```
환경 → 관찰 → 도구 호출 → 결과 → 보상 → (도구 호출 단위) 크레딧 할당 → 업데이트
```

APPO의 발견은 이 루프에서 **'도구 호출 단위'라는 관찰/할당 해상도 고정**이 문제임을 보여준다. 필요한 것은 도구 호출 경계와 독립적인, 의미론적 입도에서의 크레딧 할당 메커니즘이다.

이는 [[concepts/adaptive-inference.md|adaptive inference]]가 추론 시점에서 모델/라우트 선택에서 추측 길이 선택으로 적응의 차원을 확장한 것과 구조적으로 유사하다 — 학습 시점에서도 **적응적 할당 입도(adaptive credit granularity)**라는 새로운 차원이 필요하다.

## 결론

APPO가 보여준 바에 따르면, 에이전트 RL의 한계는 **보상 설계와 크레딧 할당 입도의 이중 구속**에 있으며, 현재 연구 커뮤니티가 후자를 과소평가해 왔다. 완벽한 보상이 주어진 환경에서도 학습이 제한된다는 APPO의 실증은, [[concepts/agent-loop-as-computation.md|agent loop as computation]]의 암묵적 전제 — 루프 경계가 곧 학습 단위 경계라는 가정 — 을 근본적으로 해체해야 함을 의미한다.

**새로운 연결로 명시하자면:** [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]을 크레딧 할당 문맥으로 확장하여 "학습 신호의 의미론적 해상도와 런타임의 구조적 해상도 사이의 간극"으로 재정의할 필요가 있으며, 이 간극을 메우는 메커니즘(예: 도구 호출 내부 프로시저 분해, 의미론적 서브스텝 정의)이 에이전트 RL의 다음 성장 곡선을 결정할 것이다.

## 핵심 통찰

에이전트 RL의 실제 병목은 보상 신호의 질이 아니라 해당 신호를 분배할 수 있는 결정 단위의 입도에 있으며, 이는 algorithm-system-translation-gap이 학습 신호 흐름에서 재현된 형태다.
