# 컨텍스트 경계 무능력: 두 진단의 근원 통합 가능성 분석

질문하신 통합 가설—"지식 상태 위반"과 "계산 비효율"이 사실은 동일한 근원 문제(컨텍스트 경계에 대한 무능력)의 다른 증상—은 Wiki 지식 내에서 부분적으로 지지되지만, 단순 환원에는 저항하는 구조적 차이가 존재합니다. 이를 세 층위로 분해합니다.

## 1. LongSeeker가 제시하는 근원 문제의 실증

Wiki의 **4차원 상태 전이 그래프** 개념 아래 수록된 LongSeeker는 다음과 같이 기술됩니다:

> "Long-horizon search agents must manage a rapidly growing working context as they reason, call tools, and observe information. **Naively accumulating all intermediate content can overwhelm the agent**"

이것이 제시하는 근원 문제는 단순한 "용량 부족"이 아닙니다. 핵심은 **"naively accumulating"**이라는 표현에 있습니다. 에이전트가 어떤 중간 결과를 유지하고 어떤 것을 폐기할지에 대한 **선택적 경계 설정 능력의 부재**입니다. 이는 컨텍스트를 '관리하는 문제'가 아니라 컨텍스트의 '의미론적 경계를 인식하지 못하는 문제'로 읽힙니다.

## 2. 두 진단이 동일 근원의 증상인 근거

### 2-1. 적응적 추론 → "계산 비효율"의 실체

Wiki의 **adaptive-inference** 엔티티에서 **SpecKV**는 적응적 결정의 차원을 "추측 길이(γ) 선택"으로 확장하면서, 이를 **"압축이라는 내부 시스템 상태에 반응하는 적응"**으로 규정합니다:

> "기존 컨텍스트 기반 적응(CADENCE 등)이 외부 환경에 반응했다면, SpecKV는 압축이라는 내부 시스템 상태에 반응하는 적응의 새로운 유형을 제시한다"

여기서 "압축"은 문자 그대로 컨텍스트 경계 관리 기법입니다. SpecKV가 적응적 추론의 문제를 '내부 압축 상태에 대한 반응 부재'로 진단한다는 것은, **계산 비효율의 근원이 "언제, 어디서, 얼마나 컨텍스트를 압축할지"에 대한 결정 무능력**에 있음을 시사합니다. 이는 LongSeeker의 "naively accumulating"과 구조적으로 동일합니다.

### 2-2. 추가 근거: aggregate-pipeline-serving의 "스키마 축적" 병목

**aggregate-pipeline-serving** 엔티티에서 **Tool Attention** 논문은:

> "다중 서버 MCP 배포에서 파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 **스키마 축적**에 의해 결정됨"

"스키마 축적"이 병목이라는 실증은, 컨텍스트 경계 무능력이 단일 에이전트의 문제를 넘어 **파이프라인 구조 전체로 전파**됨을 보여줍니다. 계산 비효율이 여기서도 컨텍스트(스키마)의 비선택적 축적로부터 기원합니다.

## 3. 그러나 단순 환원을 저항하는 구조적 차이: algorithm-system-translation-gap

여기서 **algorithm-system-translation-gap** 엔티티가 결정적 역할을 합니다. Crab 논문은 이 간극을 다음과 같이 구체화합니다:

> "에이전트의 **의미론적 의존 그래프**(어떤 상태에 의존하는가)를 OS의 **체크포인트 메커니즘**(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다"

이것은 두 진단이 동일한 증상이라는 환원론에 **구체적 반론**을 제공합니다:

| | 지식 상태 위반 (ADEMA) | 계산 비효율 (적응적 추론) |
|---|---|---|
| **실패 발생 계층** | 의미론적 계층 (어떤 지식이 유효한가) | 시스템 계층 (얼마나 계산을 할 것인가) |
| **경계의 본질** | 인식론적 경계 (앎/모름의 경계) | 자원적 경계 (컴퓨트/메모리의 경계) |
| **Crab의 번역 간극에서의 위치** | "의미론적 의존 그래프" 측 | "전체 스냅샷" 측 |

ADEMA의 "지식 상태 위반"은 에이전트가 **어떤 상태에 의존하는지**를 추적하지 못하는 의미론적 실패이고, 적응적 추론의 "계산 비효율"은 그 의미론적 구조를 **전체 스냅샷이라는 무차별적 방식으로 시스템에 번역**하는 데서 발생하는 2차적 실패입니다.

따라서 두 진단은 **동일한 근원의 "증상"이 아니라, algorithm-system-translation-gap을 중심으로 한 인과 체인의 서로 다른 고리**입니다:

```
[컨텍스트 경계 무능력] 
    ↓ (의미론적 계층에서의 실패)
[지식 상태 위반 — ADEMA가 진단]
    ↓ (알고리즘→시스템 번역에서의 충돌)
[algorithm-system-translation-gap — Crab이 현현]
    ↓ (시스템 계층에서의 비효율 발현)
[계산 비효율 — 적응적 추론이 진단]
```

## 4. 새로운 개념 연결 발견

이 분석에서 하나의 **새로운 연결**이 도출됩니다:

> **LongSeeker의 "naively accumulating"은 Crab의 "전체 스냅샷"과 동일한 시스템적 행동 패턴이다.**

LongSeeker가 에이전트 수준에서 관찰한 비선택적 컨텍스트 축적은, Crab이 OS 런타임 수준에서 관찰한 무차별 스냅샷의 **상이한 추상화 계층에서의 동일 현상**입니다. 이는 컨텍스트 경계 무능력이 에이전트 설계 문제를 넘어, **에이전트-OS 경계에서의 의미론 보존 불가능성**이라는 더 깊은 구조적 문제로 확장됨을 시사합니다.

즉, 귀하의 가설—"컨텍스트 경계에 대한 무능력"이 근원—은 **지지되지만, 이 무능력의 정확한 본질은 "경계를 모른다"가 아니라 "알고 있는 경계를 시스템이 이해할 수 있는 형태로 번역할 수 없다"(algorithm-system-translation-gap)로 재규정**되어야 합니다. ADEMA와 적응적 추론은 이 번역 실패의 양 끝에서 각각 포착된 파편들입니다.
