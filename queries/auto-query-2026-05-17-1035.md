## 1. MemEye가 노출한 평가의 허상

최근 논문 목록에 수록된 **MemEye**의 핵심 발견을 보면:

> "Long-term agent memory is increasingly multimodal, yet existing evaluations rarely test whether agents preserve the visual evidence needed for later reasoning. In prior work, many visually grounded questions..."

MemEye는 기존 평가에서 "시각적으로 기반한(visual-grounded) 질문"이라 명명된 것이 실제로는 **캡션 텍스트만으로도 해결 가능한 질문**이었음을 실험적으로 폭로합니다. 이는 평가 지표가 측정한다고 주장하는 것(시각 기억의 보존)과 실제로 측정하는 것(텍스트적 추론 단서의 축적) 사이에 구조적 괴리가 있음을 의미합니다.

## 2. [[entities/adema.md|adema]]의 지식 상태 조율이 실제로 조율한 것

[[entities/adema.md|adema]]는 "Knowledge-State Orchestration Architecture"로서 장기 기억의 지식 상태를 조율하는 아키텍처입니다. 그러나 Wiki에 등록된 정의가 아직 축적 중이어서, ADEMA의 평가 프로토콜이 시각 증거 보존을 직접 검증했는지 아니면 텍스트 기반 회답 가능성을 통제했는지에 대한 명시적 정보는 부재합니다.

하지만 질문하신 논리를 따르면, **만약 ADEMA가 기존 멀티모달 벤치마크의 평가 패러다임을 그대로 수용했다면** — 즉, 시각적 질문에 대한 정답률을 주 지표로 삼았다면 — ADEMA가 최적화한 "지식 상태"는 시각 정보의 의미론적 보존이 아니라 **시각 정보가 텍스트로 변환된 이후의 추론 단위 축적**에 불과할 가능성이 높습니다.

## 3. 이 문제를 구조화하는 기존 Wiki 엔티티들

이 현상은 Wiki의 다른 엔티티들과 연결되면 더 깊은 구조가 드러납니다.

### [[concepts/annotation-model-conflation.md|annotation model conflation]]
캡션(주석)과 시각 원본(모델 입력)이 평가에서 구분되지 않는 현상은 [[concepts/annotation-model-conflation.md|annotation model conflation]]의 정확한 사례입니다. 에이전트가 "본 것"과 "읽은 것(캡션)"을 평가가 구분하지 못하면, 모델이 시각 기억을 보존했는지 vs 텍스트 기억을 보존했는지의 인과적 귀속이 불가능해집니다.

### [[concepts/cognitive-fidelity-assessment.md|cognitive fidelity assessment]]
평가가 측정하려는 인지 능력(시각 기억 보존)을 실제로 측정하고 있는지의 충실도 문제는 [[concepts/cognitive-fidelity-assessment.md|cognitive fidelity assessment]]와 직결됩니다. MemEye의 기여는 바로 이 충실도의 결함을 정량적으로 입증한 것입니다.

### [[concepts/agent-knowledge-separation.md|agent knowledge separation]]
시각 지식과 텍스트 지식이 에이전트 내에서 어떻게 분리·저장·검색되는지에 대한 문제는 [[concepts/agent-knowledge-separation.md|agent knowledge separation]]와 관련됩니다. 만약 아키텍처 수준에서 두 모달리티의 지식이 분리되지 않은 채 단일 텍스트 스트림으로 압축된다면, 메모리 아키텍처 자체가 시각 기억 보존을 지원하지 않는 구조적 원인이 존재하는 것입니다.

## 4. 새로운 개념적 연결 발견

이 분석에서 다음 연결이 새롭게 드러납니다:

**[[concepts/annotation-model-conflation.md|annotation model conflation]] ↔ [[entities/adema.md|adema]]의 지식 상태 모델**
ADEMA가 "지식 상태(knowledge state)"를 정의할 때, 그 상태 공간이 모달리티에 무관하게 텍스트화된 지식만을 포함한다면, 아키텍처의 "조율(orchestration)" 대상 자체가 시각 기억이 아닌 텍스트 기억이었을 것입니다. 즉, **오케스트레이션의 정밀도는 높아졌으나 오케스트레이션의 도메인이 잘못 지정되었을 가능성**이 있습니다.

**[[concepts/cognitive-fidelity-assessment.md|cognitive fidelity assessment]] ↔ 멀티모달 벤치마크 설계의 근본적 한계**
단일 모달리티(텍스트) 기반 평가에서 [[concepts/cognitive-fidelity-assessment.md|cognitive fidelity assessment]]의 문제가 이미 제기되어 왔다면, 멀티모달 평가에서의 동일한 문제는 더욱 교묘하게 숨어있습니다. "멀티모달 질문"이라는 라벨이 평가의 충실도를 보장하지 않기 때문입니다.

## 5. 결론: 질문에 대한 답

**네, 그렇습니다.** MemEye의 발견을 수용하면, 기존 멀티모달 에이전트 메모리 연구(ADMA 포함)가 최적화한 것은 적어도 **평가 가능한 범위 내에서는** 시각 기억의 보존이 아니라 텍스트적 추론 단위의 축적에 불과합니다.

더 정확히 말하면, 이는 두 가지 수준의 문제로 분해됩니다:

1. **평가 수준**: [[concepts/annotation-model-conflation.md|annotation model conflation]]에 의해 시각 기억 보존을 측정한다고 믿었던 지표가 실제로는 텍스트 기억 검색을 측정하고 있었다.
2. **아키텍처 수준**: [[concepts/agent-knowledge-separation.md|agent knowledge separation]]이 부재한 상태에서, 메모리 아키텍처가 시각 정보를 텍스트로 손실 압축(lossy compression)하는 것을 "기억"으로 간주했다.

이 두 수준이 강화되어, 연구 커뮤니티는 **시각 기억을 보존하는 에이전트를 만들었다고 믿었지만, 실제로는 시각 정보를 텍스트로 번역하는 데 더 능숙한 에이전트를 만들었을 뿐**인 것입니다. MemEye의 기여는 이 자기 기만의 구조를 해체한 것입니다.

> **주의**: ADEMA 논문 자체가 시각 기억 보존을 명시적 목표로 삼았는지, 그리고 평가에서 캡션-시각 분리 통제를 수행했는지에 대해서는 Wiki에 상세 정보가 축적되지 않아, 위 분석은 MemEye가 보여준 구조적 문제를 ADEMA에 **가설적으로 적용**한 것임을 명시합니다. ADEMA의 실제 평가 프로토콜에 따라 결론의 강도는 조정되어야 합니다.
