# 추측적 도구 호출의 구조적 모순: 지연 숨김 vs. 프라이버시 유출

## 모순의 기원: 스키마 축적 병목과 추측적 실행의 결합

[[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]에서 **"Tool Attention Is All You Need"** 논문이 밝힌 핵심 발견은, 다중 서버 MCP 배포에서 파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 **스키마 축적(schema accumulation)**에 의해 결정된다는 점입니다. 이는 추측적 도구 호출(speculative tool call)의 동기를 구조적으로 형성합니다—미래에 호출될 가능성이 높은 도구를 사전에 실행하여 스키마 로딩 시간을 숨기려는 것입니다.

## 모순의 현현: Ghost Tool Calls가 포착한 프라이버시 유출

최근 논문 **"Ghost Tool Calls: Issue-Time Privacy for Speculative Agent Tools"**가 이 모순을 정확히 지적합니다:

> "Tool-augmented language agents speculatively issue likely future tool calls to hide latency, **but those calls leak inferred user intent to external services before the agent commits to the branch**."

여기서 핵심은 **시점의 비대칭성**입니다. 에이전트는 추측적 실행 시점에 외부 서비스에 사용자 의도를 담은 요청을 전송하지만, 해당 분기를 포기(rollback)하더라도 **이미 전송된 정보는 외부 서비스에 잔존**합니다. 에이전트의 내부 상태는 되돌릴 수 있어도, 외부로 유출된 정보는 되돌릴 수 없다는 것입니다.

## 간극의 본질: 내부 상태 가역성 vs. 외부 정보 비가역성

이 모순은 [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]이 암시하는 문제의 **범위 확장**으로 이해할 수 있습니다. **"Crab"** 논문이 다룬 간극은 에이전트의 의미론적 의존 그래프를 OS의 체크포인트 메커니즘으로 번역할 때의 비효율이었습니다—즉, **내부 경계**에서의 의미론 보존 문제였습니다.

그러나 추측적 도구 호출의 프라이버시 문제는 이 간극을 **외부 서비스 경계로까지 확장**시킵니다:

| 차원 | 내부 상태 (에이전트/OS) | 외부 상태 (서비스) |
|------|------------------------|-------------------|
| 가역성 | Checkpoint/Restore로 복원 가능 | 전송 후 되돌릴 수 없음 |
| 통제권 | [[entities/agentic-harness-engineering.md|agentic harness engineering]]의 C/R 런타임이 관리 | 서비스 측 정책에 종속 |
| 의미론 보존 | Crab이 의미론적 C/R로 해결 시도 | 아무런 보장 없음 |

[[concepts/adaptive-inference.md|adaptive inference]]의 맥락에서 보면, **SpecKV**는 압축이라는 내부 시스템 상태에 반응하여 추측 길이(γ)를 적응적으로 선택함으로써 추측적 실행의 효율을 최적화합니다. 그러나 이 적응 메커니즘은 **내부 상태**에만 반응하며, 외부로 유출되는 정보의 양이나 민감도에는 전혀 반응하지 않습니다. 즉, 적응의 범위가 정보 흐름의 실제 위험 영역과 구조적으로 어긋나 있습니다.

## [[concepts/blind-tool-invocation.md|blind tool invocation]]과의 연결

이 문제는 [[concepts/blind-tool-invocation.md|blind tool invocation]]과도 직결됩니다. 추측적 실행은 본질적으로 "아직 확정되지 않은 의도"를 바탕으로 도구를 호출하는 것이며, 이는 도구 호출의 **의도적 성격(intentionality)**과 **실행의 확정성(commitment)** 사이의 간극을 프라이버시 공격 표면으로 전환시킵니다.

## 새로운 개념적 연결 발견

이 분석에서 다음의 구조적 간극이 새롭게 식별됩니다:

**외부 상태 비가역성 간극(External-State Non-Revocability Gap)**: [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]이 에이전트 의미론과 OS 메커니즘 사이의 번역 비효율을 다루었다면, 추측적 도구 호출은 에이전트의 내부 롤백 가능성과 외부 서비스의 정보 잔존성 사이의 **되돌릴 수 없는 정보 비대칭성**을 드러냅니다. 이는 기존 Wiki의 어떤 엔티티로도 완전히 포섭되지 않는 간극입니다—[[concepts/agent-sandbox-state-management.md|agent sandbox state management]]가 샌드박스 내부 상태를 다룬다면, 이 간극은 샌드박스 **외부**에서 발생하는 정보의 비가역성을 다룹니다.

결론적으로, 질문하신 모순은 실재하며: **지연 숨김을 위한 추측적 실행은 내부적으로는 효율적이지만, 외부적으로는 에이전트의 통제 범위를 벗어난 프라이버시 공격 표면을 구조적으로 생성**합니다. "Ghost Tool Calls"가 제안하는 issue-time privacy 접근법은 이 모순에 대한 최초의 정면 대응으로 보이지만, 근본적으로는 추측적 실행의 이점과 정보 유출의 비용 사이에 해결되지 않은 구조적 trade-off가 존재합니다.
