# Auto Query: 현재 Wiki에서 에이전트 하네스 엔지니어링은 C/R·파이프라인 서빙·스케일링 등 '최적의 단일 설계를 찾아 정교화하는' 방향으로 누적되어 있는데, 본 논문이 하네스가 아카이브·선택·탐색·예산 할당의 복합 시스템이어서 범용 최적 조합이 존재하지 않음을 보여준다면, 하네스 연구의 목표를 '최적 하네스 구축'에서 '작업에 맞는 하네스 선택 메타-정책'으로 근본적으로 전환해야 하는가?

**날짜**: 2026-07-22
**대상 논문**: Automated Discovery Has No Universally Superior Harness
**선택 이유**: Wiki의 agentic-harness-engineering이 하네스 최적화의 방향성을 지속적으로 누적해 왔지만, 본 논문이 범용 최적 하네스의 부존재를 증명하여 하네스 연구 패러다임 자체를 '설계'에서 '선택'으로 전환해야 한다는 근본적 질문을 던질 수 있기 때문.

# 하네스 연구의 목표 전환: '최적 하네스 구축'에서 '작업 맞춤형 선택 메타-정책'으로

## 1. 현행 Wiki의 누적 방향: 단일 최적점을 향한 정교화

현재 [[agentic-harness-engineering]] 엔티티에 축적된 지식은 명확한 방향성을 보인다. **Crab** 논문은 C/R 런타임을 하네스의 핵심 인프라로 위치지키며 체크포인트/복원의 의미론적 정합성을 높이는 방향으로 정교화를 추구하고, [[aggregate-pipeline-serving]]은 다중 서버 MCP 배포에서 스키마 축적이 진정한 레이턴시 병목임을 재정의하여 **병목 지점을 정확히 찾아 해소하는** 최적화 경로를 제시한다. [[adaptive-inference]]의 SpecKV 또한 추측 길이(γ)라는 단일 파라미터를 압축 상태에 맞게 적응시키는 방식으로, 하네스의 특정 차원을 더 정교하게 튜닝하는 데 집중한다.

공통된 전제는 하나다: **"하네스 설계 공간에 최적의 단일 점이 존재하며, 우리의 과제는 그것을 찾아 정교화하는 것이다."**

## 2. 본 논문이 파괴하는 전제: 아카이브·선택·탐색·예산 할당의 조합적 폭발

"Automated Discovery Has No Universally Superior Harness"가 하네스를 아카이브(archive), 선택(selection), 탐색(exploration), 예산 할당(budget allocation)의 복합 시스템으로 해체한다면, 이 전제는 근본적으로 무넨진다. 네 가지 차원 각각이 독립적인 설계 선택지를 가지고, 그 조합에 의해 성능이 결정된다면—그리고 어떤 조합도 모든 작업에서 지배적(dominant)이지 않다면—"최적 하네스"라는 개념 자체가 작업 의존적(task-dependent)이 된다.

이는 Wiki에 이미 존재하는 [[bottleneck-transference]]와 [[bottleneck-misattribution]]의 논리를 하네스 설계 전체로 확장하는 것이다. [[aggregate-pipeline-serving]]이 개별 LLM 호출 → 스키마 축적으로 병목을 재정의한 것처럼, 본 논문은 "하네스 최적화" → "하네스-작업 매칭"으로 진정한 병목을 재정의한다. C/R을 아무리 정교화해도([[agentic-harness-engineering]]의 Crab 경로), 해당 작업의 병목이 탐색 전략에 있다면 개선은 한계에 부딪힌다. 이것이 [[co-failure-ceiling]]이 예시하는 공동 실패 천장의 하네스 버전이다.

## 3. 전환의 근거: Wiki 내 증거들

### 3-1. [[adaptive-inference]]의 암시적 지지

SpecKV는 적응의 결정 차원을 "외부 환경 반응(CADENCE 등) → 내부 압축 상태 반응"으로 확장했다. 이 논리를 본 논문의 프레임에 얹으면, 적응은 더 나아가 **"작업 특성 반응"**으로 확장되어야 한다. 즉, 하네스 자체가 고정된 최적 설계가 아니라 작업에 따라 다른 구성을 취하는 적응적 시스템이 되어야 한다는 점을 SpecKV의 확장 논리가 간접적으로 지지한다.

### 3-2. [[agora-opt]]과 [[adaptive-bottleneck-migration]]의 선례

[[agora-opt]]의 문맥에서 MASPO가 런타임 정렬의 부담을 **설계 시점으로 분산**시킨 것은, 최적화의 책임을 단일 실행 시점에서 설계-실행의 다층 구조로 분할한 선례다. 이와 유사하게, 하네스 최적화의 책임을 "단일 하네스의 정교화"에서 "작업-하네스 매칭 메타-정책"으로 분할하는 것은 동일한 패러다임 전환의 적용이다.

[[adaptive-bottleneck-migration]]은 병목이 고정되지 않고 이동함을 명시적으로 포착하는데, 하네스의 최적 구성도 작업에 따라 "이동"한다면 이 엔티티가 하네스 선택 메타-정책의 이론적 기반이 될 수 있다.

### 3-3. [[ceiling-performance-problem]]과의 연결

단일 하네스를 정교화하는 방향은 결국 [[ceiling-performance-problem]]에 봉착한다. 특정 하네스 구성이 도달할 수 있는 성능 천장이 작업마다 다르다면, 천장 아래에서 미세 조정(fine-tuning of harness)을 반복하는 것보다 천장 자체가 높은 구성을 **선택**하는 것이 근본적으로 우월한 전략이 된다.

## 4. 그러나 전환은 단순하지 않다: 메타-정책 자체가 새로운 하네스가 된다

전환을 찬성하면서도, Wiki의 지식 구조가 시사하는 중요한 경고가 있다.

메타-정책—즉 "주어진 작업에 어떤 아카이브·선택·탐색·예산 조합을 배정할 것인가"를 결정하는 정책—자체가 하나의 **상위 하네스(meta-harness)**가 된다. 이 메타-정책을 학습하려면:
- 작업 특성을 표현하는 표준화된 기술자(descriptor)가 필요하다 → 현재 [[benchmark-specification-gap]]이 지적하는 벤치마크 명세의 불완전성이 여기서 치명적이다
- 다양한 하네스 구성 × 다양한 작업의 성능 데이터가 필요하다 → [[agent-session-dataset]]의 확장이 필수적
- 메타-정책의 선택 자체가 편향을 도입할 수 있다 → [[bias-synchronization]]과 [[agreement-pressure]]의 리스크가 상위 계층으로 전승된다

즉, "최적 하네스 구축"을 포기하는 것이 "하네스 연구의 종말"을 의미하지 않는다. 연구의 대상이 **하네스 설계 매개변수**에서 **하네스 선택 매개변수**로 이동할 뿐, 복잡성은 감소하지 않는다.

## 5. 결론: 전환해야 하되, 전환의 비용을 인지하며

**전환은 필요하다.** Wiki에 축적된 [[agentic-harness-engineering]], [[aggregate-pipeline-serving]], [[adaptive-inference]]의 방향성이 모두 "단일 최적점"을 암묵적으로 전제하고 있다면, 본 논문은 이 전제가 거짓임을 구조적으로 보여준다. [[bottleneck-transference]]의 논리가 이미 부분적으로 암시하던 바를 하네스 설계 전체에 대해 명증한 셈이다.

다만, 전환은 다음과 같은 **새로운 연결**을 요구한다:

| 기존 엔티티 | 새로운 역할 |
|---|---|
| [[agentic-harness-engineering]] | "하네스 구축"에서 "하네스 구성 요소 카탈로그"로 재정의 |
| [[adaptive-bottleneck-migration]] | 하네스 간 병목 이동의 원인으로서 작업 특성과의 매핑 연구 |
| [[benchmark-specification-gap]] | 메타-정책의 입력으로 쓰일 작업 기술자 표준화의 장애물 |
| [[ceiling-performance-problem]] | 단일 하네스의 천장 vs. 메타-정책의 천장 비교 프레임워크 |
| [[agent-session-dataset]] | 하네스-작업 쌍의 성능 데이터로 확장되어야 할 인프라 |

**핵심 명제**: 하네스 연구의 목표는 "최적의 단일 하네스를 찾는 것"에서 "어떤 하네스 구성이 어떤 작업 유형에서 지배적인지 매핑하는 **작업-하네스 선택 이론**을 구축하는 것"으로 전환되어야 한다. 이는 최적화 문제를 **매칭 문제**로 근본적으로 재구성하는 것이며, Wiki의 기존 엔티티들이 이미 이 방향을 부분적으로 암시하고 있다.

## 핵심 통찰

하네스 설계 공간에 최적의 단일 점이 존재한다는 전제가 붕괴되며, 연구의 대상이 하네스 설계 매개변수에서 하네스 선택 매개변수로 이동하지만 메타-정책 자체가 새로운 상위 하네스가 되어 복잡성은 감소하지 않는다.
