# Crab의 의미론적 C/R을 분산 멀티에이전트로 확장하는 문제

## 1. Crab이 해결하는 간극의 정확한 범위

Crab의 핵심 기여는 [[entities/algorithm-system-translation-gap|algorithm-system-translation-gap]]을 **단일 에이전트 샌드박스 내**에서 좁히는 데 있습니다. Wiki가 기록한 바와 같이:

> Crab이 이 간극을 구체적 런타임 문제로 현현한다: 에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다.

즉, Crab의 "의미론적 인식"은 **하나의 의존 그래프를 하나의 스냅샷으로 선택적 직렬화**하는 문제입니다. 분산 시스템으로 확장하면 이 간극이 이항 관계에서 다항 관계로 **조합적으로 폭발**합니다 — 에이전트 A의 의존 그래프가 에이전트 B의 의존 그래프와 어떤 교집합을 가지는지를 정의하는 "상호작용 의미론"이 Crab의 프레임워크에 존재하지 않습니다.

## 2. 권한 위임(authority delegation)의 통합 경로

Crab의 의미론 계층 위에 권한 위임을 올리려면, 현재 Crab이 모델링하는 의존 그래프에 **소유권(ownership)**과 **위임 경계(delegation boundary)**라는 두 메타 속성을 도입해야 합니다.

- Crab의 단일 에이전트 모델에서는 모든 상태 노드가 암묵적으로 하나의 주체에 귀속됩니다.
- 분산 환경에서는 "파일시스템 경로 X에 대한 쓰기 권한이 현재 에이전트 A에게 위임되어 있음"을 의미론 그래프의 엣지 속성으로 표현해야 합니다.
- 복원 시, 이 위임 상태 자체가 체크포인트의 일부로 직렬화되어야 하고, 복원 시점의 위임 일관성이 검증되어야 합니다.

여기서 [[entities/agora-opt|agora-opt]]와 연관된 논문인 **"From Soliloquy to Agora: Memory-Enhanced LLM Agents with Decentralized..."** 가 시사하는 바가 중요합니다. 제목의 대비("독백" → "아고라")는 단일 에이전트 기억 모델에서 다중 에이전트 기억 모델로의 전환이 단순한 확장이 아닌 **구조적 재설계**를 요구함을 암시합니다. Crab의 의미론 계층은 "독백" 수준의 상태 모델이며, "아고라" 수준에서는 기억/상태의 소유권 자체가 분산 협상의 대상이 됩니다.

## 3. 교차 에이전트 상태 일관성(cross-agent state consistency)의 문제

이것이 핵심 난점입니다. 단일 에이전트에서 Crab이 보장하는 것은:

> 체크포인트 시점의 의미론적 의존 그래프가 복원 시점에도 동일하게 재구성된다.

분산 환경에서 이 보장은 **부분 순서(partial order)** 문제로 변형됩니다:

- 에이전트 A가 t=10에 체크포인트 CP_A를 찍고, 에이전트 B가 t=12에 CP_B를 찍었을 때
- A와 B 사이에 t=8에 발생한 메시지 교환이 있다면, CP_A는 이 메시지를 포함하지만 CP_B는 포함합니다
- 두 체크포인트를 동시에 복원하면 **인과적 비일관성(causal inconsistency)**이 발생합니다

Crab의 프레임워크는 이 문제를 수용할 수 없습니다. 이유는 Crab이 해결하는 [[entities/algorithm-system-translation-gap|algorithm-system-translation-gap]]이 **공간적 의존성(어떤 상태가 어떤 상태에 의존하는가)**에 국한되어 있기 때문입니다. 분산 일관성이 요구하는 것은 **시간적 인과성(어떤 상태 전이가 어떤 전이보다 선행하는가)**이며, 이는 근본적으로 다른 형식의 메타데이터입니다.

## 4. 분산 합의 문제: Crab 수용 가능성의 판단

**결론: 근본적으로 다른 협조 메커니즘이 필요합니다.**

Crab의 설계 철학을 분해하면:

| Crab의 설계 원리 | 단일 에이전트에서 | 다중 에이전트에서 |
|---|---|---|
| 의미론적 의존 그래프 | DAG로 모델링 가능 | 에이전트 간 순환 의존(circular dependency) 발생 가능 |
| 선택적 직렬화 | 그래프 트래버설로 결정 | 전역 트래버설 순서 자체가 합의 문제 |
| 검증 가능한 복원 | 단일 기준 상태와 비교 | "기준 상태"가 존재하지 않음(분산 합의 결과물) |

여러 에이전트가 동시에 체크포인트를 찍거나 복원하는 시나리오는 분산 컴퓨팅에서 **원자적 커밋(atomic commit)** 또는 **합의(consensus)** 문제로 귀결됩니다. Crab이 이를 수용하려면 의미론 계층 위에 전혀 다른 프로토콜 계층을 올려야 하며, 이때 Crab은 "개별 에이전트 상태의 의미론적 직렬화"라는 **하위 문제**로 격하됩니다.

## 6. 새로운 개념 연결 제안

이 분석에서 다음 연결이 자연스럽게 도출됩니다:

> **distributed-semantic-checkpoint** (신규 엔티티 제안): 단일 에이전트의 의미론적 의존 그래프(Crab)를 교차 에이전트 인과성 그래프(ADEMA의 오케스트레이션)와 통합하여, 분산 환경에서 검증 가능한 상태 복원을 달성하는 개념. [[entities/algorithm-system-translation-gap|algorithm-system-translation-gap]]이 "의미론→OS 메커니즘" 간극에서 "의미론→분산 합의 메커니즘" 간극으로 진화하는 지점.

요약하면, Crab의 의미론적 직렬화는 분산 확장의 **필요조건**이지만 **충분조건이 아닙니다**. 권한 위임은 의미론 그래프의 메타 속성 확장으로 흡수 가능할 수 있으나, 교차 에이전트 일관성과 동시 체크포인트/복원의 분산 합의는 Crab의 설계 공간 밖에 있으며, [[entities/adema|ADEMA]]의 상태 오케스트레이션과 [[entities/agora-opt|agora-opt]]의 분산 기억 아키텍처를 Crab 위에 계층적으로 통합하는 3계층 구조가 현실적인 확장 경로입니다.

## 핵심 통찰

Crab의 algorithm-system-translation-gap 해결이 단일 에이전트의 '공간적 의존성'에 국한되어 있음을 분명히 했다. 분산 환경이 요구하는 '시간적 인과성'은 근본적으로 다른 형식의 메타데이터이며, 이 간극을 메우려면 의미론 계층 위에 인과적 순서 타임스탬프를 부착하는 상태 전이 로그 계층과, 체크포인트를 원자적 트랜잭션으로 처리하는 합의 계층이 필요하다. '독백에서 아고라로'의 전환이 단순 확장이 아닌 소유권의 분산 협상 대상화라는 구조적 재설계를 요구함을 확인했다.
