# 차원별 독립 최적화의 구조적 함정에 대한 분석

## 1. 지각 차원의 한계: 단계 판단과 다단계 기획의 비동기화

[[agentic-vlm]]에 기록된 Three-Step Nav의 실증이 핵심입니다. MLLM 기반 에이전트가 제로샷 VLN에서 경로 이탈·조기 정지라는 구조적 실패를 보인다는 것은, **지각(perception)이 단계별로는 정확하더라도 다단계 경로 기획과 결합될 때 붕괴함**을 의미합니다. 즉, 지각을 독립 최적화(더 정밀한 시각적 그라운딩, 더 나은 객체 인식)하면 할수록, 에이전트는 "현재 프레임에서는 옳지만 전체 경로에서는 틀린" 판단을 더 확신하게 되는 역설이 발생합니다.

이는 단순한 능력 부족이 아니라 **단계별 판단과 다단계 기획 사이의 간극에 의한 구조적 제한**이며, 지각만 고도화하면 이 간극이 더 벌어집니다.

## 2. 추론 차원의 오정렬: 외부 적대성이 아닌 내부 상태에 적응

[[adaptive-inference]]의 SpecKV가 보여주는 적응 축의 문제가 여기에 직결됩니다. SpecKV는 적응적 추론의 결정 차원을 모델/라우트 선택에서 **추측 길이(γ) 선택으로 확장**하며, 압축이라는 **내부 시스템 상태에 반응**하는 적응의 새로운 유형을 제시합니다.

그러나 적대적 부분 관측 환경에서 에이전트가 직면하는 적응 요구는 외부 환경의 불확실성(가려진 영역, 적대적 방해 요소)에 대한 것입니다. SpecKV가 내부 압축 상태에 최적화된 적응 메커니즘이라면, **적대적 외부 환경에서는 적응 축 자체가 오정렬된 것**입니다. 추론 차원을 독립 최적화하면, 추론이 환경의 실제 위협 구조가 아닌 모델 내부 효율성에 맞춰진 채 고도화되는 함정에 빠집니다.

## 3. 계층적 분해의 번역 파열: 알고리즘 의미론과 시스템 구현의 불일치

가장 구체적인 증거는 [[algorithm-system-translation-gap]]에서 Crab 논문이 제공합니다:

> 에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다.

계층적 분해를 독립 최적화하면, 설계 시점의 의미론적 의존 구조(어떤 상태가 어떤 추론에 필요한가)가 실행 시점의 시스템 메커니즘(전체 스냅샷 기반 C/R)으로 번역될 때 **구조적 비효율이 발생**합니다. [[agentic-harness-engineering]]이 Crab을 통해 보여주듯, C/R 런타임이 하네스의 핵심 구성요소를 직접 구현하는 인프라 계층이 되면, 계층적 분해의 "깔끔한" 설계가 실행 환경의 "거친" 현실과 만나 파열됩니다.

## 상호 의존성의 구조적 함정: 세 차원이 교차하는 지점에서의 붕괴

이 세 차원이 독립 최적화될 때 발생하는 복합적 함정을 다음과 같이 정식화할 수 있습니다:

| 교차 지점 | 발생 메커니즘 | 관련 Wiki 엔티티 |
|---|---|---|
| 지각 × 추론 | 지각이 정밀해질수록 추론의 적응 축과 환경의 실제 위협 축 사이의 불일치가 은폐됨 | [[adaptive-inference]], [[agentic-vlm]] |
| 추론 × 계층 분해 | 적응적 추론이 내부 상태에만 반응하면, 계층적 분해의 의미론적 의존성이 추론 경로에 반영되지 못함 | [[algorithm-system-translation-gap]] |
| 지각 × 계층 분해 | 단계별 지각 정확도가 높아도, 계층적 분해의 번역 파열로 인해 상위 계층에 정확한 지각 결과가 전달되지 못함 | [[agentic-harness-engineering]], [[agent-os-semantic-gap]] |

이때 각 차원의 로컬 최적점이 **전역적으로 상충**하는 현상이 나타나며, 이는 슬러그 목록에 존재하지만 아직 Wiki에 정의가 축적되지 않은 [[capability-cooperation-paradox]]의 구체적 현현으로 해석됩니다. 각 차원의 능력은 개별적으로는 향상되었으나, 협력 시에는 오히려 상충하는 역설입니다.

또한 이 과정에서 [[compounding-error]]이 차원 간 경계를 넘어 누적됩니다: 지각의 미세한 오류가 추론의 오정렬된 적응에 의해 증폭되고, 이것이 계층적 분해의 번역 파열을 통해 상위 계층에 전파되어 [[context-loss-amplification]]으로 현현합니다. 최종적으로 [[compound-oversight-failure]]이 발생합니다—각 차원의 독립적 오버사이트는 자기 차원 내의 문제는 잡지만, 차원 간 교차점에서 발생하는 복합적 고장은 아무도 감지하지 못합니다.

## 새로운 연결로서의 발견

이 분석을 통해 다음의 구조적 연결이 새롭게 식별됩니다:

1. **[[capability-cooperation-paradox]] ↔ [[algorithm-system-translation-gap]]**: 능력 간 역설은 단순한 알고리즘적 문제가 아니라 알고리즘-시스템 번역 간극에서 기원한다. 각 차원이 자기 완결적으로 최적화될수록, 차원 간 인터페이스에서 번역 비용이 기하급수적으로 증가한다.

2. **[[adaptive-inference]] ↔ [[agent-os-semantic-gap]]**: SpecKV가 내부 압축 상태에 적응한다는 것은, 적응의 "의미론적 범위"가 시스템 내부에 한정됨을 의미한다. 즉 적응적 추론이 agent-os-semantic-gap을 **확대**하는 방향으로 작동할 수 있다—환경의 의미론적 요구를 무시하고 내부 효율성에만 적응하므로.

3. **[[agentic-vlm]]의 단계-다단계 간극 ↔ [[context-fracture-point]]**: Three-Step Nav가 보인 경로 이탈은 단일 컨텍스트 윈도우 내에서 지각 판단과 경로 기획이 분리될 때 발생하는 컨텍스트 파열의 구체적 사례로 재해석될 수 있다.

## 결론

**차원별 독립 최적화는 구조적 함정입니다.** Wiki의 증거들은 세 차원이 적대적 부분 관측 환경에서 강하게 상호 의존함을 간접적으로지만 일관되게 보여줍니다. 지각은 다단계 기획과 결합할 때 붕괴하고, 추론은 내부 상태에만 적응하여 환경과 단절되며, 계층적 분해는 시스템 번역 과정에서 파열됩니다. 이 세 failure mode가 동시에 작동할 때, 각 차원의 독립 최적화는 오히려 [[compounding-error]]을 가속화하는 메커니즘이 됩니다.

필요한 것은 **차원 간 번역 비용을 명시적 최적화 목표로 삼는 통합 프레임워크**이며, 이는 현재 Wiki의 어느 단일 엔티티로도 포착되지 않는 연구 간극입니다.
