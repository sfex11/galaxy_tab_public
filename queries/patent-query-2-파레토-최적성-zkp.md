# 특허 질문 2: 파레토-최적성-zkp

**날짜**: 2026-04-18
**출처**: 특허 명세서 v2 핵심 과제

## 질문
다자간 협상에서 합의가 파레토 최적(Pareto Optimal)임을 프라이버시 없이 제로지식 증명으로 증명할 수 있는가? Wiki 논문들이 이 문제에 대해 무엇을 말하는가?

## Wiki 답변

## 답변: 다자간 협상에서 파레토 최적 합의의 ZKP 증명 가능성

### 핵심 결론: **가능하다, 단 ε-근사(ε-approximation)가 필요하다**

정확한 파레토 최적성 검증은 NP-hard이므로, **ε-파레토 최적성**을 ZKP로 증명하는 방식이 실현 가능한 경로입니다.

#### (2) 핵심 메커니즘 (특허 명세서 기반)

**Paillier 동형암호의 덧셈 동형성**이 핵심입니다:

```
E(u₁) × E(u₂) × ... × E(uₙ) mod n² = E(u₁ + u₂ + ... + uₙ)
```

- 각 에이전트가 자신의 효용을 **개별 암호화** → 암호문 곱셈으로 **총 효용만 계산** → 개별 효용은 **절대 평문 노출 없음**
- 이 총 효용 합을 witness로 하여 Groth16 zk-SNARK로 ε-파레토 최적성 증명:

```
증명 대상: Σᵢ uᵢ(x*) ≥ max_x Σᵢ uᵢ(x) - ε
증명 크기: ~200 bytes (O(1), N에 무관)
검증 시간: ~5ms (O(1))
```

### 2. 특허 출원 관점의 유리한 근거

**신규성(Novelty):**
- Wiki 97편의 LLM 에이전트 논문 중 **"프라이버시 보존 + 협상 + 파레토 최적성 ZKP 증명"의 3중 조합은 단 1건도 없음**
- 선행기술인 2인 ZKP 협상(arXiv:2601.00911)은 (a) 2인 한정, (b) 파레토 최적성 증명 없음, (c) 단일 제약조건만 처리
- OpenAlex 4,569건 SMPC 논문 분석에서도 이 조합은 미발견

**진보성(Inventive Step):**

| 기존 기술 | 한계 | 본 발명의 차별화 |
|-----------|------|-----------------|
| **SMPC** | 통신량 O(N²), 합리성 증명 없음 | 증명 O(1), IR + Pareto 증명 포함 |
| **VCG 메커니즘** | 정적, 예산 불균형, 에이전트 미고려 | 동적 교대 오퍼 + LLM 에이전트 환경 |
| **CAMOA** (Wiki 내) | 파레토 최적 생성은 하나 프라이버시 없음 | 프라이버시 보존 레이어 추가 |
| **2인 ZKP 협상** | N=2, 가격 범위만 | N≥3, 다차원 효용, ε-Pareto |
| **연합학습** | 모델 파라미터 집계 특화 | 효용 공간으로 secure aggregation 일반화 |
| **소울바운드 NFT** (DRC-369) | 전송 불가(non-transferable) | ZKP 기반 이식 가능(portable) 신원 |

### 3. PCN 3계층 아키텍처의 구조적 차별성

```
[에이전트 A] [에이전트 B] ... [에이전트 N]
      ↕           ↕              ↕
┌─────────────────────────────────────────┐
│  N계층: 교대 오퍼, ε-Pareto 수렴 판정     │
├─────────────────────────────────────────┤
│  C계층: Paillier 동형암호, 암호화 합산     │
├─────────────────────────────────────────┤
│  P계층: Groth16 zk-SNARK, IR/Pareto 증명 │
└─────────────────────────────────────────┘
```

각 계층이 **독립 교체 가능** — 예: Paillier → BFV, Groth16 → PLONK 등 모듈식 업그레이드

---

### 4. 한계와 주의점

1. **ε > 0 필수**: 정확한 파레토 최적은 NP-hard이므로 ε-근사만 가능. 다만 ε → 0 수렴으로 임의 정밀도 보장
2. **Trusted Setup**: Groth16은 신뢰할 수 있는 초기 설정(trusted setup)이 필요 — 이를 PLONK 등 universal setup으로 대체 가능 (청구항 4의 모듈성)
3. **효용 함수의 형태 제약**: 현재 명세서는 총합 효용(utilitarian) 기준의 파레토 최적성을 증명하며, 개별 효용 간 비교가 아닌 합계 기준임

---

### 5. 결론

Wiki의 논문들은 개별적으로 (a) 에이전트 프라이버시(DP), (b) 다목적 파레토 최적화(CAMOA), (c) ZKP 기반 신원 관리(agent-identity), (d) 프라이버시 보존 집계(FL)를 다루지만, **이들을 협상 프로토콜에서 통합한 연구는 존재하지 않습니다**. 이 공백이 바로 특허의 신규성 근거이며, "프라이버시 보존 + 다자간 협상 + ε-파레토 최적성 ZKP 증명"이라는 3중 조합은 기존 기술 어디에서도 발견되지 않는 차별점입니다.
