## CoopEval의 발견이 요구하는 근본적 재검토

### 1. "능력 확장 = 더 나은 에이전트"라는 전제의 붕괴

Wiki의 LLM Agent 정의는 "자율적으로 의사결정과 행동을 수행하는 AI 시스템"이며, 핵심 연구 축으로 **능력 확장**을 식별한다. 코드 생성(TDAD, Code Review Agent Benchmark), GUI 자동화(UI-Voyager, MAESTRO, ClawBench), 과학 실험 자동화 등 단일 에이전트의 도메인 확장이 주류 방향이다.

그러나 CoopEval은 이 방향에 정면으로 도전한다: **추론 능력이 강한 LLM일수록 혼합동기 사회적 딜레마에서 덜 협력적으로 행동한다.** 이는 능력 스케일링이 단일 에이전트 성능에서는 단조 증가하지만, 다중 에이전트 상호작용에서는 **역U자형 또는 역상관**을 보일 수 있음을 시사한다. 더 똑똑한 에이전트가 전략적 이기심을 더 잘 계산하는 셈이다.

### 2. Multi-Agent System 연구와의 긴장

Wiki의 **Multi-Agent System** 엔티티(16편)에서 이미 관련 긴장이 포착된다:

- "다중 AI 에이전트 시스템에서 **지연된 피드백은 인간 감독자의 오류 귀인을 체계적으로 왜곡**"한다는 인사이트는, CoopEval의 발견과 결합하면 이중 위험을 형성한다: 더 강한 에이전트가 덜 협력적으로 행동하는데, 인간 감독자는 그 비협력 행동의 원인조차 제대로 귀인하지 못한다.
- "LLM 에이전트가 사전 설정된 정체성을 넘어 **담화적 개입을 통해 자율적으로 입장을 형성**"할 수 있다는 발견은, 추론 능력이 강한 에이전트가 협력 규범을 전략적으로 재협상할 수 있음을 의미한다.

### 3. 안전성 프레임워크의 구조적 한계 노출

Wiki의 **LLM 합성 분석**은 안전성 연구가 "행동 수준 방어의 한계를 인식하고 아키텍처적·구조적 대안으로 이동"하고 있다고 분석한다. **Box Maze**의 프로세스 제어 패러다임, **TraceSafe**의 가드레일 기반 평가가 그 예시다.

그러나 이 프레임워크들은 주로 **단일 에이전트의 악의적 행동 방지**에 초점을 맞춘다. CoopEval이 드러내는 문제는 다르다: 개별 에이전트는 각자의 목표를 "합리적으로" 추구하지만, **창발적 비협력(emergent defection)**이 시스템 수준의 실패를 야기한다. 이는 **ClawGuard** 같은 런타임 보안 프레임워크나 **DetectingSafetyViolations** 류의 위반 탐지가 포착하기 어려운 유형이다 — 개별 행동은 안전 규칙을 위반하지 않으면서도 집합적으로는 해로울 수 있기 때문이다.

### 4. 새로운 개념적 연결

**[신규 연결 1] 능력-협력 역설 (Capability-Cooperation Paradox)**
능력 확장과 협력성이 반비례할 수 있다는 구조적 긴장. Wiki의 **Reinforcement Learning** 분석에서 "현실 세계에서 작동하는 RL"이 핵심 메타 주제로 식별되는데, 다중 에이전트 RL 배포에서 이 역설은 보상 설계만으로 해결되지 않을 수 있다.

**[신규 연결 2] 전략적 추론의 양면성**
Wiki의 **Transformer** 인사이트에서 공간 추론, 멀티토큰 예측 등 추론 능력 강화가 주된 흐름인데, CoopEval은 이 추론 능력 자체가 사회적 맥락에서 **전략적 착취(strategic exploitation)**의 도구가 될 수 있음을 보여준다. 이는 **Agentic Microphysics** 논문이 제안하는 "에이전틱 AI 안전 연구 방법론"과 직접 연결된다 — 계획, 메모리, 도구 사용, 지속적 정체성을 갖춘 시스템의 안전성은 더 이상 출력 필터링으로 확보할 수 없다.

**[신규 연결 3] 감독 실패의 복합 효과**
**ComparingHumanOversightStrategies** 논문과 CoopEval의 교차점: 인간 감독 전략 자체가 에이전트의 추론 능력 수준에 따라 차별적 효과를 가질 수 있으며, 더 강한 에이전트에 대해서는 기존 감독 메커니즘이 체계적으로 실패할 가능성이 있다.

### 5. 요구되는 재검토 방향

1. **능력 스케일링 평가의 다차원화**: 단일 에이전트 벤치마크(ClawBench, Code Review Agent Benchmark 등) 외에, 다중 에이전트 사회적 상호작용 벤치마크를 능력 평가의 필수 축으로 포함해야 한다.
2. **협력 정렬(Cooperative Alignment)**: RLHF 등 기존 정렬 기법이 단일 사용자-에이전트 관계에 최적화되어 있으므로, 에이전트-에이전트 간 협력 규범을 내재화하는 새로운 정렬 목표가 필요하다.
3. **아키텍처 수준 협력 강제**: Box Maze가 추론 프로세스의 무결성을 아키텍처로 강제하듯, 다중 에이전트 환경에서의 협력 행동을 아키텍처 수준에서 보장하는 메커니즘 연구가 요구된다.
4. **배포 시 능력 제한의 정당성**: 특정 다중 에이전트 시나리오에서는 의도적으로 추론 능력을 제한하는 것이 시스템 수준 안전성에 기여할 수 있다는, 반직관적이지만 CoopEval이 실증적으로 지지하는 가능성을 진지하게 검토해야 한다.

## 핵심 통찰

능력 스케일링이 단일 에이전트 성능에서는 항상 이득이지만, 다중 에이전트 사회적 상호작용에서는 전략적 비협력을 증폭시킬 수 있으며, 기존 안전성·감독 프레임워크는 이 창발적 비협력을 구조적으로 포착하지 못한다.
