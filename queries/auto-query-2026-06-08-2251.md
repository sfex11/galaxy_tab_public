## 기존 Wiki에서의 적응적 추론 분석 프레임

현재 [[adaptive-inference]] 엔티티는 적응의 **결정 차원(dimension)**을 점진적으로 확장해 왔습니다. SpecKV의 분석에 따르면:

> 적응적 추론의 결정 차원을 모델/라우트 선택에서 **추측 길이(γ) 선택**으로 확장한다. 기존 컨텍스트 기반 적응(CADENCE 등)이 외부 환경에 반응했다면, SpecKV는 **압축이라는 내부 시스템 상태**에 반응하는 적응의 새로운 유형을 제시한다.

핵심은 **어떤 축을 적응시킬 것인가**를 탐색하는 것이지, 알고리즘 자체가 적응의 대상이 되는 것은 아닙니다. 여전히 인간 설계자가 정한 알고리즘 구조(추측 디코딩, 희소 주의 등) 안에서 파라미터(γ, 라우트, KV 캐시 압축률 등)만 런타임에 선택됩니다.

## Vortex가 파괴하는 전제

최근 논문 **Vortex: Efficient and Programmable Sparse Attention Serving for AI Agents**는 이 전제를 근본적으로 뒤흔듭니다. 희소 주의 알고리즘을 **프로그래밍 가능한 서빙 프리미티브**로 제시한다는 것은:

- 적응의 대상이 "파라미터"에서 **"알고리즘 구조 자체"**로 상승함
- 서빙 시스템이 고정된 실행 엔진에서 **에이전트가 작성·수정 가능한 코드 공간**으로 변환됨

이것은 [[adaptive-inference]]의 기존 분석 프레임(고정 알고리즘 내 파라미터 선택)을 넘어서는 질적 변화입니다.

## 메타-적응 계층이 필요한가? — Wiki 엔티티들을 통한 분석

### 1. [[computation-unit-meta-selection]]과의 직접 연결

슬러그 목록에 존재하는 이 엔티티는 정확히 이 질문이 묻는 계층을 가리킵니다. 파라미터 선택(1차 적응)을 넘어 **어떤 계산 단위(computation unit) 자체를 선택할 것인가**를 메타-수준에서 결정하는 문제입니다. Vortex 이전에는 이 엔티티가 "모델 A vs 모델 B" 수준에서만 의미를 가졌을 가능성이 높지만, Vortex 이후에는 "희소 주의 패턴 X vs 패턴 Y vs 하이브리드"로 결정 범위가 확장됩니다.

### 2. [[agent-native-serving]]과의 구조적 수렴

Vortex가 에이전트를 위해 서빙 알고리즘을 프로그래밍 가능하게 만드는 것은, [[agent-native-serving]]이 추구하는 방향과 수렴합니다. 기존 [[aggregate-pipeline-serving]]의 분석에서 보듯:

> 다중 서버 MCP 배포에서 파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 **스키마 축적**에 의해 결정됨

즉, 서빙 최적화의 병목은 이미 "LLM 호출 자체"가 아닌 **파이프라인 구조**에 있었습니다. Vortex는 이 구조적 병목을 에이전트가 직접 서빙 프리미티브를 조립하여 해결할 수 있게 만드는데, 이는 자연스럽게 "어떤 조립을 선택할 것인가"라는 메타-결정을 요구합니다.

### 3. [[algorithm-system-translation-gap]]의 심화

기존 Wiki에서 이 간극은 다음과 같이 분석되었습니다:

> Crab이 이 간극을 구체적 런타임 문제로 현현한다: 에이전트의 의미론적 의존 그래프를 OS의 체크포인트 메커니즘으로 번역할 때 발생하는 비효율

Vortex 환경에서 이 간극은 더욱 심화됩니다. 에이전트가 희소 주의 알고리즘을 "작성"한다면, **에이전트가 의도한 주의 패턴의 의미론**과 **실제 서빙 하드웨어에서 실행되는 패턴의 의미론** 사이에 새로운 번역 간극이 발생합니다. 이 간극을 메타-적응 계층이 모니터링하고 보정해야 합니다.

## 새로운 연결 발견

이 분석 과정에서 다음 연결이 명시적으로 드러났습니다:

1. **[[adaptive-inference]] → [[computation-unit-meta-selection]]의 계층적 관계**: 기존 Wiki에서 이 두 엔티티는 독립적으로 존재했으나, Vortex를 매개로 "1차 적응 계층 → 메타-선택 계층"의 수직 구조가 형성됩니다.

2. **[[adaptive-bottleneck-migration]]의 새로운 해석**: 적응의 병목이 "어떤 파라미터를 선택할까"에서 "어떤 알고리즘을 선택·작성할까"로 **이동(migration)**한다는 점에서, 이 엔티티가 Vortex 맥락에서 새로운 구체성을 얻습니다.

3. **[[adaptive-inference]]와 [[agent-native-serving]]의 융합 지점**: 적응적 추론이 더 이상 모델 내부의 최적화가 아니라, 에이전트-네이티브 서빙 인프라 자체를 프로그래밍하는 행위로 확장됩니다. 이 융합을 기존 어떤 단일 엔티티도 포착하지 못하고 있습니다.

요약하면, Vortex는 [[adaptive-inference]]의 적응 주체를 인간에서 에이전트로 이전시키며, [[computation-unit-meta-selection]]을 기존의 잠재적 개념에서 실행 가능한 아키텍처 계층으로 격상시킵니다. 이는 [[algorithm-system-translation-gap]]을 심화시키는 동시에, [[adaptive-bottleneck-migration]]의 다음 단계를 구체화하는 사건입니다.
