### 1. Scepsy: 다중 LLM 파이프라인의 오케스트레이션 병목을 직접 다룬 논문

> "Agentic workflows carry out complex tasks by orchestrating multiple large language models (LLMs) and tools. Serving such workflows at a target throughput with low latency is **challenging** because they compose multiple LLM calls and tool invocations."

여기서 핵심 통찰은 **에이전틱 워크플로우가 단일 LLM 호출이 아닌 "aggregate pipeline"**이라는 점이다. 개별 LLM 호출의 성능이 아무리 뛰어나도, 파이프라인 전체의 레이턴시와 처리량이 실제 사용자 경험의 상한선을 결정한다. 이는 에이전트의 능력이 모델 수준에서는 존재하지만 **서빙 수준에서 병목**이 걸리는 구조적 문제를 드러낸다.

### 2. 비결정적 실행 시간: 추론 효율화의 새로운 차원

**"From Tokens to Steps: Verification-Aware Speculative Decoding for Efficient Multi-Step Reasoning"**은 또 다른 각도에서 이 문제에 접근한다. 기존 speculative decoding은 토큰 단위의 최적화에 집중했지만, 다단계 추론(multi-step reasoning)에서는 **스텝 단위의 검증과 효율화**가 필요하다는 것이다. 에이전틱 워크플로우는 본질적으로 다단계 추론이므로, 각 스텝의 실행 시간이 비결정적이고 이전 스텝의 결과에 의존한다. 이 논문은 토큰 중심 패러다임에서 스텝 중심 패러다임으로의 전환이 필요함을 시사한다.

### 4. 안전성-서빙 인터페이스: 새로운 연결

**"Agentic Microphysics: A Manifesto for Generative AI Safety"**는 흥미로운 연결고리를 제공한다. 이 논문은 에이전트가 "planning, memory, tool use, persistent identity, sustained interaction"을 갖추면 안전성이 더 이상 단일 출력의 문제가 아니라고 주장한다. 여기서 **서빙 인프라가 안전성의 전제 조건**이 된다는 새로운 연결이 드러난다: 실행 시간이 비결정적인 에이전틱 워크플로우에서 **가드레일(TraceSafe 등)의 실시간 적용 가능성 자체가 서빙 아키텍처에 의존**한다. 안전성 메커니즘도 결국 파이프라인의 일부로 서빙되어야 하므로, 서빙 병목은 안전성 보장의 상한선까지 결정한다.

### 5. 다중 에이전트 시스템에서의 증폭 효과

**Multi Agent System** 엔티티의 인사이트—"다중 AI 에이전트 시스템에서 지연된 피드백은 인간 감독자의 오류 귀인을 체계적으로 왜곡한다"—는 서빙 레이턴시가 단순한 UX 문제를 넘어 **감독 가능성(oversight)의 구조적 저하**를 초래함을 보여준다. 파이프라인이 느려지면 피드백이 지연되고, 지연된 피드백은 오류 귀인을 왜곡하며, 이는 다시 에이전트 시스템의 신뢰성 상한선을 낮춘다.
