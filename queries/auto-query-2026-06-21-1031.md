## 핵심 논점: 실행 상태의 본질적 단위는 무엇인가?

### 1. Wiki가 이미 포착한 통찰: 의존 그래프가 본질이다

[[algorithm-system-translation-gap]] 항목의 Crab 관련 서술을 보면:

> *"에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다."*

이미 Crab 논문 맥락에서 **샌드박스 아티팩트 자체가 아니라, 그 아티팩트 간의 의존 그래프**가 실행 상태의 본질적 구조임을 식별하고 있습니다. Crab의 한계는 이 그래프를 OS 수준의 전체 스냅샷으로 뭉개서 직렬화한다는 점이었고, 이것이 바로 [[agent-os-semantic-gap]]의 현현이었습니다.

즉, Wiki는 "샌드박스 아티팩트 = 실행 상태"로 정의한 것이 아니라, "아티팩트 간 의존 그래프를 전체 스냅샷으로 번역하는 비효율"을 간극의 본질로 진단했습니다.

### 2. Execution-State Capsules의 대척점 수렴

최근 논문에서 Execution-State Capsules는 정반대 레짐에서 동일한 문제를 만납니다:

- **기존 방식**: KV 캐시 하나만 포착 → 단일 풀(pool) 관리 → 고처리량·고동시성에는 적합하지만, **온디바이스 물리적 AI 서빙**(저지연·소규모 배치)에서는 구조적으로 부적합
- **제안 방식**: 실행 상태를 **그래프로 구조화**하여 체크포인트/복원

Crab이 "의존 그래프를 전체 스냅샷으로 뭉개는" 문제를 다뤘다면, Execution-State Capsules는 "KV 캐시라는 단일 노드로 그래프를 축소하는" 문제를 다룹니다. 두 방향 모두 **실행 상태의 그래프적 위상이 단일 포인트로 환원될 때 발생하는 정보 손실**을 지적합니다.

### 3. 그렇다면 본질적 단위는 무엇인가?

두 레짐을 합성하면 다음과 같이 정리됩니다:

| 레짐 | 상태 표현의 문제 | 근본 원인 |
|------|-----------------|----------|
| 샌드박스 C/R (Crab) | 의존 그래프 → 전체 스냅샷으로 과대직렬화 | 그래프 위상을 평면화(flatten) |
| 온디바이스 서빙 (Execution-State Capsules) | 의존 그래프 → KV 캐시 단일 노드로 과소포착 | 그래프 위상을 단일 포인트로 축소 |

**본질적 단위는 샌드박스 아티팩트도, KV 캐시도 아닙니다.** 둘 다 [[agent-loop-as-computation]]이라는 관점에서 볼 때, **계산 그래프의 위상(topology)** — 즉 "어떤 계산 단계가 어떤 상태에 의존하고, 어떤 상태를 생성하는가"하는 방향성 그래프 — 가 실행 상태의 불변량(invariant)입니다. 샌드박스 아티팩트와 KV 캐시는 이 그래프가 특정 배포 레짐에서 구체화된 **매니페스테이션**에 불과합니다.

### 4. 새로운 개념적 연결

이 분석을 통해 다음 연결이 새롭게 드러납니다:

**[[agent-sandbox-state-management]] ↔ [[agent-native-serving]]의 위상 수렴**
기존에는 샌드박스 상태 관리와 에이전트 네이티브 서빙이 별개의 축으로 보였으나, Execution-State Capsules의 그래프 구조화는 두 축이 [[algorithm-system-translation-gap]]를 매개로 동일한 위상적 본질로 수렴함을 보여줍니다. [[checkpoint-restore-efficiency-spectrum]]은 이제 "전체 스냅샷 ↔ KV 캐시 단일 포인트"의 1차원 스펙트럼이 아니라, **"그래프 위상을 얼마나 보존하는가"**라는 2차원 축을 갖게 됩니다.

**[[agent-loop-as-computation]]의 구체화**
에이전트 루프가 "계산"이라는 추상적 명제가, 실행 상태의 본질이 그래프 위상임이 확인되면서 구체적 의미를 얻습니다. 상태 관리는 곧 계산 그래프의 부분적 재개(resumption) 문제이며, 이는 [[adaptive-inference]]의 결정 차원이 모델 선택에서 추측 길이 선택으로 확장된 것과 동일한 패턴 — **추론의 구조를 더 정밀하게 인식하는 방향** — 과 일치합니다.

### 요약

Crab이 [[algorithm-system-translation-gap]]에서 이미 "의존 그래프가 본질"이라는 진단을 내렸고, Execution-State Capsules는 정반대 레질에서 동일한 결론에 도달합니다. 에이전트 실행 상태의 본질적 단위는 **샌드박스 아티팩트가 아니라 계산 그래프의 위상**이며, 기존 Wiki의 [[agent-sandbox-state-management]] 정의는 이 위상이 샌드박스 레짐에서 취하는 특정 형태였을 뿐입니다. 이를 반영하여 Wiki의 해당 엔티티들이 그래프 위상 보존이라는 통일적 관점에서 재구조화될 필요가 있습니다.
