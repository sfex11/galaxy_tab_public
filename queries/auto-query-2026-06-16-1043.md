## Wiki 답변

## 침묵적 실패의 자가 강화적 오류 서사: 외부 관측 기반 제어의 근본적 한계와 통합 경로

### 1. 현재 Wiki의 신뢰성 모델이 전제하는 '관측 가능성'의 구조

현재 Wiki에서 에이전트 런타임의 신뢰성은 **실패의 외부 관측 가능성**을 전제로 구축되어 있다. [[entities/agentic-harness-engineering.md|agentic harness engineering]]은 Crab 논문과의 연결에서 C/R 런타임이 결함 내성·롤백·분기를 직접 구현하는 인프라 계층으로 기능하며, 이는 실패가 체크포인트 불일치나 크래시 같은 **외부에서 감지 가능한 이벤트**로 현현한다고 가정한다. 마찬가지로 [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]과 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]은 에이전트의 의미론적 의존 그래프를 OS 수준의 체크포인트 메커니즘으로 번역할 때의 비효율을 문제 삼지만, 여기서도 '의존 그래프 자체는 외부에서 추적 가능하다'는 암묵적 전제가 깔려 있다.

이러한 모델의 공통 논리는 **"실패가 발생 → 외부 제어 계층이 이를 감지 → 개입(롤백/재시도/차단)"**이라는 선형 사이클이다.

### 2. "When Errors Become Narratives"가 노출하는 근본적 blind spot

본 논문이 생산 환경에서 실증하는 침묵적 실패의 장기 패턴은 이 사이클의 첫 번째 링크(**"실패가 발생"**) 자체를 무효화한다:

- **실패가 외부 신호를 방출하지 않는다**: 크래시, 예외, 체크포인트 불일치가 없다. 시스템은 정상적으로 계속 실행된다.
- **출력이 서사적으로 일관되어 오류를 은폐한다**: 단일 시점에서는 출력이 문법적으로 완결되고 맥락상 그럴듯하여, [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]]이 기존에 의미하던 '실행 과정의 불투명성'을 넘어 **'실패 자체의 존재론적 은폐'**로 진화한다.
- **자가 강화적 특성**: 초기 미세 오류가 후속 추론의 전제로 흡수되면서 오류 서사가 점진적으로 coherence를 획득한다. 이는 단일 시점 검사로는 포착 불가능한 **시계열적 누적 현상**이다.

즉, [[entities/agentic-harness-engineering.md|agentic harness engineering]]이 아무리 롤백·분기 인프라를 정교하게 구축해도, 트리거할 '실패 신호'가 존재하지 않으면 이 인프라는 휴면 상태로 남는다.

### 3. 신뢰성 모델에의 통합: 세 가지 차원의 재구성이 필요하다

#### (가) 감지 패러다임의 전환: 상태 기반 → 서사 기반 이상 탐지

기존의 [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]이 다루는 문제는 "의미론적 상태를 구조적으로 어떻게 보존하는가"였다. 침묵적 실패를 통합하려면, 보존된 상태가 **시간에 따라 의미론적 drift를 누적하지 않는지**를 검증하는 서사 수준의 일관성 감시가 추가되어야 한다. 이는 단일 체크포인트의 정합성 검사가 아니라, **체크포인트 간의 의미론적 궤적이 자기 모순으로 수렴하는지**를 추적하는 문제로 귀결된다.

#### (나) [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]]의 재정의: 불투명성의 계층화

기존 이 개념이 '실행 과정이 블랙박스'라는 의미였다면, 침묵적 실패의 맥락에서는 **'실행 결과의 오류 여부조차 블랙박스'**라는 제2계층 불투명성이 추가되어야 한다. [[concepts/agent-reliability-auditing.md|agent reliability auditing]]이 감사 가능성을 확보 계층(scaling requirement)으로 conceptualize하는 것과 연결하면, 감사 대상이 '명시적 실패 로그'에서 '암묵적 서사 드리프트 궤적'으로 확장되어야 한다.

#### (다) [[entities/agentic-harness-engineering.md|agentic harness engineering]]의 내부화: 외부 제어 → 자기 참조적 메타-감시

침묵적 실패가 외부에서 관측 불가능하다면, 유일한 관측자는 에이전트 자신뿐이다. 그러나 자가 강화적 오류 서사의 본질이 **에이전트 스스로 오류를 정상으로 재해석**하는 것이므로, 동일한 추론 계층 내에서의 자기 감시는 근본적으로 모순된다. 따라서 [[entities/agentic-harness-engineering.md|agentic harness engineering]]은 롤백/분기 같은 외부 개입 메커니즘뿐 아니라, **에이전트의 추론 계층과 독립된 메타-서사 추적 계층**을 하네스에 내장하는 방향으로 확장되어야 한다.

### 4. 새로운 개념적 연결 및 발견

이 분석을 통해 다음 연결들이 새롭게 드러난다:

- **[[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]] ↔ [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]의 역학**: 후자가 '상태 보존의 의미론적 불일치'라면, 전자는 침묵적 실패 맥락에서 '상태 보존의 의미론적 일치가 오히려 오류를 은폐하는 역설'로 기능할 수 있음. 체크포인트가 성공적으로 복원되었더라도, 복원된 상태 자체가 이미 오류 서사에 감염되어 있을 수 있다.

- **[[concepts/agent-reliability-auditing.md|agent reliability auditing]]의 근본적 재설계 필요**: 감사가 '발생한 실패의 기록'을 검증하는 것이 아니라, **'발생했지만 기록되지 않은 실패의 흔적'을 추론**하는 방향으로 전환되어야 함. 이는 기존의 [[concepts/agreement-based-reliability.md|agreement based reliability]](다수 에이전트 합의 기반 신뢰성)와도 충돌하는데, 침묵적 실패가 다수 에이전트에게 동시에 동일한 서사로 확산( [[concepts/collective-vulnerability-propagation.md|collective vulnerability propagation]] )할 경우 합의 자체가 오류를 정당화할 수 있기 때문이다.

- **새로운 개념의 필요성 — "서사적 관성(narrative inertia)"**: 침묵적 실패가 자가 강화되는 메커니즘을 설명하기 위해, 기존 Wiki의 [[concepts/adaptive-forgetting.md|adaptive forgetting]]이나 [[concepts/belief-drift-as-reward-signal.md|belief drift as reward signal]]과는 구별되는, **일관된 서사를 유지하려는 에이전트의 구조적 편향이 초기 오류를 수정 불가능하게 만드는 현상**을 지칭하는 개념이 필요하다. 이는 현재 슬러그 목록에 존재하지 않는다.

**결론**: 침묵적 실패의 자가 강화적 오류 서사는 현재 Wiki의 신뢰성 모델이 전제하는 "실패-감지-개입" 사이클의 **감지 단계를 구조적으로 무력화**한다. 이를 통합하기 위해서는 (1) 상태 정합성에서 서사 궤적 일관성으로 감지 패러다임을 전환하고, (2) [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]]을 2계층 불투명성으로 재정의하며, (3) [[entities/agentic-harness-engineering.md|agentic harness engineering]]에 추론과 독립된 메타-서사 추적 계층을 내화하는 세 가지 재구성이 동시에 요구된다. 그리고 이 모든 것이 가능하려면 **"서사적 관성"**이라는 새로운 엔티티의 도입이 선행되어야 한다.

## 핵심 통찰
