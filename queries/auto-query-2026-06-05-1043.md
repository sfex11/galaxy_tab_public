# 파이프라인 패러다임 전환과 에이전트 경계의 의미론적 재정의

## 1. 기존 병목 분석의 전제: 에이전트 = 원자적 교환 단위

[[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]의 핵심 발견은 다중 서버 MCP 배포에서 **"파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 스키마 축적에 의해 결정된다"**는 것입니다. 이 분석이 암시하는 전제는 명확합니다:

- 에이전트 간 경계는 **완결된 아티팩트(스키마, 결과물)**를 기준으로 정의된다
- 최적화의 대상은 '경계를 넘나드는 아티팩트의 크기와 빈도'다
- 에이전트 내부의 추론 과정은 불투명한 블랙박스로 취급된다

이 전제 하에서 최적화는 [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]이 보여주듯 **"스키마를 얼마나 게으르게(lazy) 로드할 것인가"**로 수렴합니다. 에이전트 경계 자체는 재정의할 필요 없이, 경계 간 전송 효율만 높이면 된다는 가정입니다.

## 2. StreamMA의 파괴적 함의: 전송 단위의 붕괴가 경계의 붕괴를 의미하는 이유

StreamMA가 '생성-후-전송' 패러다임이 파이프라인 깊이에 비례해 선형적 레이턴시를 강제한다고 지적할 때, 이는 단순히 "더 빠르게 보내자"가 아니라 **"전송 단위를 재구성하자"**를 의미합니다. 완성된 추론 체인이 아닌 개별 추론 스텝을 전송 단위로 삼을 때 발생하는 의미론적 변화는 다음과 같습니다:

### 2-1. [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]의 새로운 현현

Wiki에서 [[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]은 이미 [[concepts/agent-os-semantic-gap.md|agent os semantic gap]]과 연결되며, Crab 논문이 구체화한 바 있습니다: **"에이전트의 의미론적 의존 그래프를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율"**이 간극의 본질이라는 것입니다.

StreamMA의 스텝 수준 스트리밍은 이 간극을 정반대 방향에서 다시 현현시킵니다:
- **기존**: 알고리즘적 의존 그래프(에이전트 간) → 시스템적 전송(완결된 체인) → 과도한 원자성으로 인한 번역 손실
- **StreamMA**: 알고리즘적 의존 그래프(스텝 간) → 시스템적 전송(개별 스텝) → 이제는 **에이전트 경계 자체가 번역의 대상이 됨**

즉, 기존에는 "에이전트 A가 끝나면 에이전트 B가 시작한다"는 알고리즘 모델을 시스템 메시지로 번역하는 간극이 문제였다면, 스텝 수준 스트리밍에서는 **"에이전트 A의 어느 스텝까지가 에이전트 B의 입력인가"**라는 더 근본적인 번역 문제가 발생합니다.

### 2-2. [[entities/agora-opt.md|agora opt]]와의 역설적 관계

[[entities/agora-opt.md|agora opt]]는 런타임 메모리와 토론 구조를 통해 **다중 에이전트 정렬을 달성**하고, [[entities/agora-opt.md|agora opt]] 문서 자체에서 MASPO가 "프롬프트 설계 시점에서 사전 정렬을 보장하는 상보적 경로"를 제공한다고 분석됩니다. 즉, Agora-Opt는 이미 에이전트 경계를 런타임 메모리 공유를 통해 '반투명'하게 만드는 시도였습니다.

하지만 Agora-Opt가 다루는 경계 투과성은 **메모리/토론 구조적 수준**에서의 투과성입니다. StreamMA가 제안하는 스텝 수준 통신은 이를 **추론 연산의 가장 미시적 단위**까지 밀어붙입니다. Agora-Opt가 "에이전트들이 같은 기억을 공유하게 하자"라면, StreamMA는 "에이전트들이 같은 추론 호흡을 공유하게 하자"는 것입니다.

### 2-3. [[concepts/agentic-microphysics.md|agentic microphysics]]와의 잠재적 연결

Wiki 슬러그 목록에 [[concepts/agentic-microphysics.md|agentic microphysics]]가 존재합니다. 이 개념이 에이전트 추론의 미시적 동역학을 지칭한다면, StreamMA의 스텝 수준 스트리밍은 **에이전트 미시물리학을 에이전트 간 통신 프로토콜로 격상**시키는 것입니다. 기존에는 미시물리학이 에이전트 내부의 구현 디테일이었으나, 스트리밍 패러다임에서는 이것이 오케스트레이션 계층이 직접 다뤄야 하는 일차적 의미론이 됩니다.

## 3. 경계 재정의가 요구하는 구조적 변화

전송 단위가 '완성된 체인'에서 '개별 스텝'으로 바뀌면, 기존 오케스트레이션이 묵시적으로 전제하던 세 가지 경계 의미론이 붕괴합니다:

| 차원 | 기존 패러다임 (생성-후-전송) | 스텝 스트리밍 패러다임 |
|------|---------------------------|---------------------|
| **계산 단위** | 에이전트 = 원자적 실행 단위 | 에이전트 = 스텝 스트림의 집합체 |
| **의존성** | 에이전트 간 (체인 단위) | 스텝 간 (에이전트 경계를 초월) |
| **최적화 대상** | [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]: 스키마 축적량 | 스텝 간 의존성 해결의 세분도 |

이는 [[concepts/agent-execution-semantic-opacity.md|agent execution semantic opacity]]과도 직결됩니다. 기존에는 에이전트 내부 실행이 불투명해도 경계만 명확하면 오케스트레이션이 가능했습니다. 그러나 스텝 수준 통신에서는 **에이전트 내부의 불투명성이 곧 오케스트레이션의 불가능성**이 됩니다. 내부 스텝을 밖으로 내보내야 하니, 그 스텝의 의미론을 오케스트레이터가 이해할 수 있어야 합니다.

## 4. 새로운 개념적 연결 및 누락된 엔티티

이 분석 과정에서 다음 연결이 새롭게 발견됩니다:

1. **StreamMA → [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]] 간의 패러다임 충돌**: 기존의 스키마 축적 병목 분석은 '생성-후-전송' 전제 하에서만 유효합니다. 스텝 스트리밍에서는 병목이 스키마 크기가 아니라 **스텝 간 의존성 그래프의 동적 해석 비용**으로 이동할 가능성이 높습니다. 이는 아직 Wiki에 없는 연결입니다.

2. **[[concepts/asynchronous-dependency-coordination.md|asynchronous dependency coordination]]과의 관계**: Wiki에 이 슬러그가 존재하므로, 스텝 수준 스트리밍이 비동기 의존성 조정을 어떻게 재정의하는지가 자연스러운 확장 지점입니다. 기존의 비동기 조정이 '에이전트 단위'의 비동기성이었다면, 이제는 '스텝 단위'의 비동기성이 됩니다.

3. **누락된 엔티티**: StreamMA 자체가 아직 Wiki에 등록되지 않았습니다. 또한 질문에서 언급된 APWA도 슬러그 목록에 없습니다. 이 두 논문 간의 관계(파이프라인 병렬화 vs 스텝 수준 스트리밍)를 포괄하는 개념으로, **"통신 입자성(communication granularity)에 의한 에이전트 경계 재정의"**라는 주제가 새로운 엔티티 후보로 보입니다.

## 결론

네, 맞습니다. StreamMA가 제기하는 패러다임 전환은 [[concepts/aggregate-pipeline-serving.md|aggregate pipeline serving]]이 정의한 병목 분석의 틀 자체를 무효화하는 것이 아니라, **그 분석이 가능했던 전제 조건(에이전트 = 완결된 산출물의 교환 단위)을 해체**합니다. 

[[concepts/algorithm-system-translation-gap.md|algorithm system translation gap]]이 이미 지적했던 '알고리즘 의도와 시스템 구현 사이의 간극'이 이제 **에이전트 경계 자체를 둘러싼 간극**으로 확장되며, [[entities/agora-opt.md|agora opt]]가 메모리 수준에서 시도했던 경계 투과성이 추론의 가장 미세한 단위([[concepts/agentic-microphysics.md|agentic microphysics]])까지 침투하게 됩니다. 기존 오케스트레이션 최적화가 '경계 사이의 효율'을 다뤘다면, 새로운 패러다임은 **'경계란 무엇인가'**라는 존재론적 질문을 요구합니다.

## 핵심 통찰

스텝 수준 스트리밍은 단순한 레이턴시 최적화가 아니라 에이전트 경계의 의미론 자체를 해체하여, 에이전트 내부의 미시물리학을 오케스트레이션 계층의 일차적 관심사로 격상시킨다.
