# 📚 오늘의 논문 (2026-03-26)

**제목**: ReqFusion: A Multi-Provider Framework for Automated PEGS Analysis Across Software Domains
**arXiv**: http://arxiv.org/abs/2603.23482v1

## 핵심 요약

ReqFusion은 소프트웨어 요구사항 공학을 자동화하는 멀티 LLM 프레임워크다. OpenAI GPT, Anthropic Claude, Groq 모델을 통합하여 PDF, DOCX, PPTX 문서에서 기능/비기능 요구사항을 자동 추출·분류한다. 핵심은 PEGS(Project, Environment, Goal, System) 기반 프롬프팅으로, 일반 프롬프팅(F1 0.71) 대비 F1 0.88의 우수한 성능을 달성했다. 18개 실제 문서에서 226개 요구사항을 추출했으며, 수작업 대비 분석 시간을 78% 단축했다.

## 💡 3가지 인사이트

1. **PEGS 구조화 프롬프팅**이 일반 프롬프팅보다 F1 점수를 0.17이나 높여, LLM에 도메인 특화 프레임워크를 적용하는 것이 성능 향상의 핵심임을 입증했다.
2. **멀티 LLM 합의(consensus) 및 폴백 메커니즘**을 통해 단일 모델 의존도를 줄이고 추출 결과의 신뢰성과 안정성을 확보했다.
3. 추출된 요구사항의 54.9%가 기능 요구사항, 45.1%가 비기능 요구사항으로, 자동화 시스템이 두 유형 모두를 균형 있게 식별할 수 있음을 보여주었다.

## 🚀 2가지 응용 가능성

1. **대규모 입찰·제안서 분석 자동화** — 공공조달이나 SI 프로젝트에서 수백 페이지의 RFP 문서로부터 요구사항을 자동 추출하여 분석 시간과 인건비를 대폭 절감할 수 있다.
2. **요구사항 품질 검증 파이프라인 구축** — CI/CD와 연계하여 문서 변경 시 요구사항을 자동 재추출·분류하고, 누락이나 불일치를 조기에 탐지하는 지속적 검증 체계에 활용할 수 있다.
