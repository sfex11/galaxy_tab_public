# Comparing Developer and LLM Biases in Code Evaluation

**타입**: 논문  
**출처**: arXiv  
**날짜**: 2026-03-27  
**링크**: None

## 핵심 요약

이 논문은 LLM이 코드 평가자(judge)로 사용될 때 인간 개발자와의 평가 편향 차이를 분석한다. 저자들은 **TRACE**(Tool for Rubric Analysis in Code Evaluation) 프레임워크를 제안하여, LLM 판정자가 인간 선호를 얼마나 잘 예측하는지 측정하고 평가 기준별 체계적 편향을 자동 추출한다. 채팅 기반 프로그래밍, IDE 자동완성, 지시 기반 코드 편집 등 3가지 모달리티에서 13개 모델을 평가한 결과, 최고 성능 모델도 인간 평가자 대비 **12~23% 낮은 일치도**를 보였으며, 인간과 LLM 간 **35개의 주요 불일치 원인**이 발견되었다.

## 인사이트

1. LLM은 긴 코드 설명을 선호하는 반면 인간 개발자는 짧고 간결한 설명을 선호하는 체계적 편향이 존재한다.
2. 발견된 35개 불일치 원인 대부분이 기존 소프트웨어 공학 품질 기준(가독성, 간결성 등)에 해당하여, LLM의 코드 품질 인식이 실무 개발자와 근본적으로 다르다.
3. 채팅, 자동완성, 코드 편집 등 모달리티에 따라 인간-LLM 간 편향 패턴이 다르게 나타나, 단일 평가 기준으로는 LLM 판정자의 신뢰성을 보장할 수 없다.

## 응용 가능성

1. TRACE 프레임워크를 활용하여 LLM 기반 코드 리뷰·평가 시스템의 편향을 사전 진단하고, 인간 선호에 맞게 평가 루브릭을 보정할 수 있다.
2. 코드 생성 모델의 RLHF/보상 모델 학습 시, 발견된 35개 불일치 항목을 반영하여 인간 개발자 선호와 더 정렬된 보상 신호를 설계할 수 있다.
