# Evaluating LLM-Based Test Generation Under Software Evolution

**타입**: 논문  
**출처**: arXiv  
**날짜**: 2026-03-26  
**링크**: None

## 핵심 요약

이 논문은 LLM 기반 자동 단위 테스트 생성이 프로그램의 실제 동작을 이해하여 테스트를 만드는지, 아니면 학습된 표면적 패턴을 재현하는 데 그치는지를 대규모 실험으로 검증한 연구이다. 8개 LLM과 22,374개 프로그램 변이를 대상으로 **의미 변경(SAC)**과 **의미 보존(SPC)** 두 가지 코드 변경 시나리오에서 테스트 성능을 분석했다. 결과적으로 LLM 생성 테스트는 원본 코드에서는 79% 라인 커버리지를 달성하지만, 코드가 진화하면 통과율과 커버리지가 크게 하락하며, **표면적 단서에 의존하고 회귀 인식 능력이 부족**함을 밝혔다.

## 인사이트

1. **SAC 실패 테스트의 99% 이상이 원본 프로그램에서는 통과**하여, LLM이 변경된 의미에 적응하지 못하고 원래 동작에 정렬된 테스트를 생성함을 보여준다.
2. 의미가 보존되는 리팩토링(SPC)에서도 통과율이 79%로 하락하며, 모델이 **의미적 동등성이 아닌 어휘적(lexical) 변화에 민감하게 반응**한다.
3. LLM은 코드 변경 시 기존 테스트를 유지·발전시키기보다 **기존 테스트를 폐기하고 새로 생성하는 경향**이 있어, 회귀 테스트로서의 신뢰성이 낮다.

## 응용 가능성

1. **LLM 테스트 생성 도구의 품질 벤치마크**: 본 논문의 변이 기반 평가 프레임워크를 활용하여 LLM 테스트 생성기의 진화 대응 능력을 체계적으로 측정·비교할 수 있다.
2. **하이브리드 테스트 전략 설계**: LLM 생성 테스트의 한계를 인지하고, 전통적 회귀 테스트와 결합하여 코드 진화 시에도 결함 탐지 능력을 유지하는 보완적 테스트 파이프라인을 구축할 수 있다.
