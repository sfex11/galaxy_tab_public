## 💡 핵심 인사이트

LLM이 코드 평가를 우회하는 위협인 동시에, 대화형 질의를 통한 이해도 검증 도구로도 활용될 수 있다는 이중적 역할이 교육 평가 패러다임의 전환을 요구한다.

## 📖 분석

## Chatbot-Based Assessment of Code Understanding in Automated Programming Assessment Systems

LLM의 코드 생성 능력이 향상됨에 따라, 학생이 코드를 '이해'하지 않고도 기능적으로 올바른 코드를 제출할 수 있는 문제가 대두되고 있다. 본 논문은 대화형(chatbot 기반) 평가 방식을 통해 코드 이해도를 측정하는 접근법을 체계적으로 정리한다. 포화 기반 스코핑 리뷰를 통해 세 가지 아키텍처 패밀리를 식별했다: (1) 규칙 기반/템플릿 시스템, (2) LLM 기반 시스템, (3) 하이브리드 시스템.

이 연구는 LLM이 교육 평가 도구로서 갖는 이중적 역할을 조명한다. LLM은 코드를 생성하여 평가를 우회하는 도구인 동시에, 대화형 질의를 통해 학생의 실질적 이해도를 검증하는 평가자 역할도 수행할 수 있다. 이는 [[concepts/llm-benchmark.md|llm benchmark]] 분야에서 '정답 생성 능력'과 '이해도 측정'의 괴리를 보여주는 사례이며, [[concepts/ai-literacy.md|ai literacy]] 교육에서 AI 도구 활용 역량과 근본적 이해력 간의 균형 문제와 직결된다.

하이브리드 아키텍처가 규칙 기반의 일관성과 LLM의 유연성을 결합한다는 점에서, [[concepts/process-control-architecture.md|process control architecture]]의 구조화된 추론 제어 패러다임과 유사한 설계 철학을 공유한다.
