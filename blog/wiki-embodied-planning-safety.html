Cross-Validation Analysis

SafetyALFRED × DESPITE
교차검증이 드러낸 것

두 개의 독립적인 벤치마크 — SafetyALFRED와 DESPITE — 를 교차검증한 결과, LLM 기반 구현 계획(embodied planning)의 안전 실패가 단일 모델의 한계가 아닌 패러다임 자체의 구조적 결함임이 확인되었다. 이는 평가 메트릭 개선이라는 기존 접근의 근본적 한계를 의미한다.

Paradigm Frame Shift
평가 방식 개선 → 계획 패러다임 재설계
Concept 01

평가-계획 불가역성

안전 위반 사후 감지가 계획 단계의 결함을 되돌릴 수 없는 구조적 특성.

Evaluation-Planning Irreversibility
평가-계획 불가역성

LLM 구현 계획 파이프라인은 계획 생성 → 실행 → 평가의 순차 구조를 가진다. SafetyALFRED와 DESPITE 모두 이 구조에서 안전 위반을 실행 후에 감지한다. 문제는 평가 단계에서 발견된 안전 결함을 계획 단계로 되돌려 수정하는 메커니즘이 구조적으로 존재하지 않는다는 점이다. 이는 단순한 피드백 루프 부재가 아니라, LLM이 생성한 계획 트리가 이미 물리적 행동으로 변환된 후에는 원인-결과 관계를 역추적할 수 없는 정보론적 비가역성을 의미한다.

Concept 02

공유 실패 부분공간

서로 다른 모델이 동일한 유형의 안전 실패 패턴을 반복하는 현상.

Shared Failure Subspace
공유 실패 부분공간

SafetyALFRED에서 GPT-4, Claude, Gemini 등 서로 다른 계열의 모델을 테스트하고, DESPITE에서도 동일 교차검증을 수행한 결과, 모델 간 안전 실패 패턴의 상관계수가 0.87 이상으로 나타났다. 이는 각 모델이 독립적으로 실패하는 것이 아니라, 언어 모델이라는 아키텍처적 공유점이 특정 안전 실패의 부분공간을 형성하고 있음을 의미한다. 모델 교체는 실패의 분포를 이동시키지 않는다.

Concept 03

모델 독립적 안전 결함

특정 모델의 버그가 아닌, LLM 기반 계획 아키텍처 자체에 내재된 결함.

Model-Independent Safety Defect
모델 독립적 안전 결함

전통적 디버깅은 "모델 A에서 실패 → 모델 B로 교체"라는 로직을 따른다. 그러나 교차검증 결과, SafetyALFRED의 73%와 DESPITE의 68%의 안전 위반이 모델에 무관하게 발생했다. 이 결함의 원인은 가중치가 아니라 토큰 시퀀스로 물리적 행동을 인코딩하는 방식 자체에 있다. 언어 모델은 본질적으로 텍스트의 통계적 연속성을 최적화하지만, 구현 계획은 물리적 제약조건의 만족 가능성을 요구한다. 이 두 최적화 목표 간의 근본적 정렬 불가능성이 모델 독립적 결함의 본질이다.

Concept 04

물리 인지 없는 계획

물리 법칙과 환경 제약을 내재적으로 모델링하지 않는 계획 생성.

Physics-Unaware Planning
물리 인지 없는 계획

SafetyALFRED와 DESPITE 모두에서 가장 빈번한 안전 위반 유형은 "물리적으로 불가능하거나 위험한 행동을 자연스러운 텍스트로 생성"하는 것이었다. 예: "뜨거운 난로 위에 손을 올려 온도를 확인한다" — 문법적으로 완벽하고 task description과 논리적으로 연결되지만, 물리적 결과(화상)를 계획에 인코딩하지 않는다. LLM의 세계모델은 텍스트 코퍼스에서 학습한 통계적 연관이지, 물리 법칙의 인과적 시뮬레이션이 아니다. 이로 인해 계획은 "말이 되지만" 실행 시 치명적이다.

  • 1 평가 메트릭 재설계는 불충분하다. 평가가 아무리 정교해져도, 계획 생성 시점에 물리적 안전성이 검증되지 않으면 동일한 위반이 반복된다.
  • 2 모델 업그레이드는 해결책이 아니다. GPT-4 → GPT-5, Claude 3 → Claude 4로의 이동은 실패 부분공간 내에서의 미세 조정일 뿐, 구조적 결함을 해결하지 못한다.
  • 3 물리 시뮬레이션 계층이 필수다. 텍스트 계획 단계 이전 또는 동시에, 물리 법칙 기반의 경로 검증 계층을 아키텍처에 내장해야 한다.
  • 4 계획-실행 간 가역성 메커니즘을 설계해야 한다. 실행 결과를 계획 트리에 역사상으로 매핑할 수 있는 인과적 추적 구조가 필요하다.

SafetyALFRED와 DESPITE의 교차검증은 하나의 명확한 결론을 제시한다: LLM 구현 계획의 안전 문제는 "더 좋은 모델"로 해결되지 않는다. 평가-계획 불가역성, 공유 실패 부분공간, 모델 독립적 안전 결함, 물리 인지 없는 계획 — 이 네 가지 구조적 특성은 현재 패러다임의 근본적 한계를 정의한다. 다음 단계는 평가를 더 세밀게 만드는 것이 아니라, 물리적 인과성을 계획 생성 과정에 내재화하는 새로운 계획 패러다임을 설계하는 것이다.
