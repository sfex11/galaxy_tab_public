🆕 Wiki가 새로 발견한 6개의 개념

1. 계층적 능력 분해 Layered Capability Decomposition

에이전트 능력을 도메인별 열거 대신 3층 구조로 재구성:
L1: 제어·검증 아키텍처 (공통) → L2: 메모리 인프라 (도메인 독립) → L3: 도메인 특화 능력 (응용)

L2가 도메인 독립적으로 구축될 수 있다면, L3은 독립적 연구 축이 아니라 공유 인프라 위의 응용 사례로 재정의된다.

2. 통합 메모리 풀 Unified Memory Pool

이질적 도메인 간 공유 런타임·언어 등의 인프라 기반을 활용하여, 단일 메모리 저장소에서 경험을 도메인 횡단적으로 전이·재활용하는 설계.

3. 제어 자원으로서의 메모리 Memory as Control Resource

메모리를 미분류 아카이브가 아닌 에이전트 제어 루프에 통합된 능동적 자원으로 정의.
체화 상태 · 잠재 동역학 · 에피소딕 메모리 · 관찰 기록이 모두 제어 결정에 영향.

4. 이중 트레이스 인코딩 Dual Trace Encoding

사실 기록에 장면 트레이스(scene trace)를 쌍으로 저장하여 cross-session recall 향상.
인지 과학의 장면 기억(scene memory) 연구에서 유래.

5. 인식론적 충실도 Epistemic Fidelity

검색 충실도(retrieval fidelity)와 구분되는 개념. 약속 강도·모순 상태·조직적 무지 등 지식의 인식론적 속성을 구조화하여 전이 시 정보 손실을 정량화.

6. 세분화 수준 기반 스킬 조직 Granularity-Based Skill Organization

스킬을 도메인이 아닌 세분화 수준(task-level, step-level)으로 조직.
동일 step-level 스킬이 이종 도메인에서 재활용됨.
