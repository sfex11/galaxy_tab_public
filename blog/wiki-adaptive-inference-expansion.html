2026-05-09 RAO · Adaptive Inference

적응적 추론은 줄이는 것이 아니다
— 에이전트가 스스로를 복제하여 확장하는 패러다임

계산 자원의 '절약'에 갇혀 있던 적응적 추론의 정의가, 에이전트가 자신의 복제본을 재귀적으로 생성하는 RAO의 등장으로 근본적으로 뒤집혀야 하는 moment.

01

기존 Wiki의 한계: adaptive-inference는 철저한 '축소' 지향

현재 AI 연구계에서 적응적 추론(Adaptive Inference)이라는 용어가 쓰일 때, 그 내연은 사실상 하나로 수렴한다. "얼마나 덜 계산할 것인가."

모델 라우팅(Model Routing)은 입력의 난이도를 보고 작은 모델로 우회시킨다. 추측 길이 선택(Guess Length Selection)은 문맥을 압축해 토큰 수를 줄인다. SpecKV 같은 최신 연구조차 "내부 시스템 상태(압축)에 반응하는 추측 길이(γ) 선택"이라는 프레임 안에서, 에이전트의 추론 능력을 고정된 틀 안의 최적화로 간주한다.

SpecKV가 보여주는 축소의 극한

SpecKV는 외부 환경 반응에서 내부 시스템 상태 반응으로 발전했다는 점에서 진일보했다. 하지만 방향성은 여전히 단방향 — 계산 가능한 한도 내에서 자원을 '축소'하는 효율성 방향에 머물러 있다. 에이전트의 능력을 '얼마나 적게 써도 되는가'로 측정하는 구조다.

적응적 추론의 본질을 '계산 절약'으로 규정하는 것은, 마치 기업의 '적응력'을 오직 비용 절감 능력으로만 정의하는 것과 같다. 필요할 때 투자를 확대하는 결정 역시 적응이다.

이 시각의 문제는 명확하다. 적응(Adaptation)이라는 생물학적 원형을 떠올려 보라. 환경이 변했을 때 유기체는 에너지를 아끼기도 하지만, 필요하다면 근육을 키우고 세포를 분화시켜 능력 자체를 확장한다. 기존의 적응적 추론은 이 확장의 축을 철저히 누락시켰다.

02

RAO의 도전: 에이전트가 자신의 복제본을 재귀 생성해 하위 작업 위임

Recursive Agent Optimization, 줄여서 RAO는 이 축소 일변도의 패러다임에 정면으로 도전장을 내민다. RAO의 핵심 메커니즘은 한 줄로 요약된다.

에이전트가 자신의 새로운 인스턴스(instance)를 생성(spawn)하고, 하위 작업을 재귀적으로 위임(delegate)한다.

여기서 일어나는 일은 기존의 '적응'과 방향이 완전히 반대다. 문제가 복잡해지면, RAO 에이전트는 자신을 복제하고 복제본에게 하위 문제를 맡긴다. 그 복제본이 또다시 복제될 수 있다. 계산량이 고정된 예산 안에서 재분배되는 것이 아니라, 문제의 복잡도에 비례해서 계산 자체가 늘어난다.

비유: 기업의 조직 확장

기업이 비용 절감만으로는 성장할 수 없다. 프로젝트의 규모가 커지면, 부서를 늘리고 팀을 분할해 업무를 위임해야 한다. RAO는 에이전트가 스스로 '부서장'이 되어 자신의 복제본을 '팀원'으로 배치하는 과정을 자동화한 것과 같다. 이것은 방어적 축소가 아니라 공격적 확장이다.

기존 에이전트 능력 확장이 외부 도구(Nemobot Games의 환경 생성)나 파이프라인(Tool Attention의 집계 서빙)에 의존했다면, RAO의 확장은 순수하게 내부적이다. 외부에 무언가를 요청하는 것이 아니라, 자기 자신의 계산 구조를 물리적으로 복제한다.

03

내생적 계산 확장: 외부 도구 없이 스스로 자원 동적 확장

RAO가 제시하는 패러다임을 정확히 이해하려면, 기존의 '확장'이 어떤 메커니즘이었는지와 비교해야 한다.

기존 확장 vs RAO의 확장

외생적 확장(Exogenous Expansion) — Nemobot Games는 에이전트 외부에 게임 환경을 생성하고, Tool Attention은 외부 스키마를 호출한다. 확장의 주체가 '에이전트 외부 시스템'이다. 에이전트는 요청자일 뿐, 구조적 변화의 주도자가 아니다.

내생적 확장(Endogenous Expansion) — RAO는 에이전트가 자신의 실행 토폴로지(execution topology)를 스스로 변화시킨다. 복제, 위임, 재귀 — 이 모든 것이 에이전트 내부에서 발생한다. 외부 환경 의존도를 낮추면서도 계산량을 문제 난이도에 맞춰 늘리는 구조다.

이 구분이 중요한 이유는, algorithm-system-translation-gap이라는 Wiki의 핵심 개념이 양방향으로 현현되기 때문이다.

  • 축소 측면의 간극 — Crab 논문이 다룬 것처럼, 의미론적 의존 그래프를 OS 스냅샷 메커니즘으로 번역할 때의 비효율.
  • 확장 측면의 간극 (새로운 질문) — "하위 작업 분해라는 알고리즘적 의도를, 새로운 에이전트 인스턴스 생성이라는 시스템적 확장으로 어떻게 비용 효율적으로 번역할 것인가?"

RAO는 두 번째 간극을 새로운 연구 주제로 제시한다. 인스턴스 생성이라는 시스템 레벨의 연산 비용을, 하위 작업 위임이라는 알고리즘적 이득이 상쇠할 수 있는가? 이것이 RAO 패러다임이 직면한 핵심적 공학적 과제다.

04

도구 사용의 내재화: 외부 API 호출에서 자기 복제본을 내부 도구로

현대 에이전트 아키텍처에서 '도구 사용(Tool-use)'은 거의 필수적이다. 검색엔진을 호출하고, 코드 인터프리터를 돌리고, 데이터베이스에 쿼리를 날린다. RunAgent가 다루는 자연어 기반 제약 조건 해석도, 외부 스키마를 호출하는 과정의 일환이다.

RAO는 이 도구 사용의 패러다임을 근본적으로 뒤집는다.

패러다임 전환: 외부 도구 → 내부 복제

기존: "이 문제를 풀기 위해 외부 도구 X를 호출하라." — 도구는 외부에 존재하고, 에이전트는 그 인터페이스에 의존한다.

RAO: "이 문제를 풀기 위해 나 자신의 복제본을 도구로 사용하라." — 도구 호출이 에이전트 내부의 재귀적 구조로 흡수(Internalization)된다.

이 내재화는 여러 함의를 가진다. 첫째, 외부 도구의 가용성에 대한 의존도가 감소한다. API가 다운되거나 스키마가 변경되어도, 자기 복제는 에이전트 내부에서 일어나는 현상이므로 영향을 받지 않는다. 둘째, 도구의 유연성이 극대화된다. 외부 도구는 미리 정의된 인터페이스만 제공하지만, 자기 복제본은 원래 에이전트와 동일한 능력을 가지므로 어떤 형태의 하위 작업이든 처리할 수 있다.

외부 도구를 부르는 대신 '나 자신의 복제본을 도구로 사용'함으로써, 도구 의존적 에이전트 패러다임을 넘어 자율적 계산 패러다임으로 진화시킨다.

이는 단순한 기술적 편의가 아니다. 에이전트가 도구를 '사용하는' 존재에서 도구를 '구성하는' 존재로, 나아가 자기 자신이 도구가 되는 존재로 진화하는 것은, 자율성(Autonomy)의 개념적 경계를 이동시키는 사건이다.

05

적응적 추론의 이중성: 하향식 축소와 상향식 확장의 통합

RAO의 등장으로 우리는 적응적 추론을 단일 메커니즘이 아니라, 두 가지 극단을 연결하는 스펙트럼으로 재정의할 수 있게 되었다.

하향식 적응 적응적 추론의 스펙트럼 상향식 적응
Top-down Adaptation 계산 가능한 한도 내에서 자원을 축소하는 효율성 방향. 모델 라우팅, 추측 길이 선택, SpecKV.
Bottom-up Adaptation 문제의 복잡도에 맞춰 자신을 복제하여 계산을 확장하는 확장성 방향. RAO, 자기 재귀적 위임.

이 스펙트럼이 의미 있는 이유는, 진정으로 적응적인 시스템이란 양극단을 모두 구사할 수 있는 시스템이기 때문이다. 단순한 문제에는 계산을 축소해 에너지를 아끼고(SpecKV 방식), 복잡한 문제에는 자신을 복제해 계산을 확장하는(RAO 방식) — 이 두 동작이 하나의 통일된 프레임 안에서 결정되어야 한다.

Wiki에 산재한 최신 논문들은 이 스펙트럼을 더 풍부하게 만든다.

스펙트럼을 완성하는 보완적 논문들

StraTA — 장기 호라이즌 의사결정을 위해 '전략적 궤적 추상화'로 에이전트의 RL을 유도한다. RAO가 계산의 물리적 복제라면, StraTA는 계산의 시간적·추상적 압축을 다루며, 확장과 압축 사이의 균형점을 제시한다.

SkillOS — 과거 상호작용에서 재사용 가능한 스킬을 큐레이션하여 '자가 진화(Self-Evolving)'하는 에이전트를 제안한다. RAO가 단발성 하위 작업을 위해 복제를 수행한다면, SkillOS는 반복되는 확장을 '스킬'이라는 구조화된 내부 지식으로 체화한다.

StraTA는 확장의 산물을 다시 압축하는 방법을, SkillOS는 확장의 경험을 구조화된 지식으로 보존하는 방법을 각각 제공한다. RAO의 '자기조직적 계산 확장'은 이들과 결합할 때 비로소 지속 가능한 패러다임이 된다.

진정한 적응은 줄이기만 하는 것이 아니다

적응적 추론은 더 이상 "컴퓨팅을 선택적으로 줄이는 메커니즘"으로 정의되어서는 안 된다. 그 정의는 에이전트의 가능성을 절반으로 잘라내는 것이다.

RAO가 보여주듯, 에이전트는 필요할 때 자신을 복제하고, 하위 작업을 위임하고, 계산 구조를 동적으로 확장할 수 있다. 이것도 적응이다. 오히려 문제 복잡도에 맞춰 능력을 늘리는 쪽이, 고정된 예산 안에서만 줄이는 것보다 더 '적응적'인 행동일 수 있다.

적응적 추론은 이제 "문제 복잡도와 내부 시스템 상태에 따라 계산을 동적으로 축소하거나 자기 재귀적 복제를 통해 확장하는 에이전트의 자원 할당 자율성"으로 재정의되어야 한다. 기존의 외부 도구 기반 확장과 구별되는 '내생적 계산 확장(Endogenous Computation Expansion)'이라는 새로운 축이, 적응적 추론의 지도에 그려져야 할 때다.

방어만 하는 것은 적응이 아니다. 공격도 포함해야 한다. 기업이 그렇듯, 에이전트도 그래야 한다.
