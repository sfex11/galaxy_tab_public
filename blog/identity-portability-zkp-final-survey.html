1.3 핵심 선행기술 심층 비교

🔴 DRC-369: Soulbound NFT for AI Agent Identity

기술: Soulbound NFT (전송 불가능한 NFT)로 AI 에이전트의 영구 신원을 증명.
주요 특징:
• Claude (Anthropic)가 공저자 — AI 에이전트 신원에 대한 AI 모델 자체의 관점 반영
• 보안 토큰, 공개키 암호화, 고유 식별자 기반
• 참조 문헌 0개 — Zenodo 사전출판으로 아직 동료 심사 전
중복 위험: 매우 높음 — "AI 에이전트의 영구 신원"이라는 개념 자체가 거의 동일
차별화 필요: NFT 기반 vs 우리 방식의 차이를 명확히 해야 함
🔴 FractalNode: Sovereign AI Identity Architecture

기술: AI 에이전트의 주권적 신원 + 경제 + 거버넌스를 위한 기초 아키텍처.
주요 특징:
• 확장성, 책임성, 공급망 관리 포함
• 아키텍처 수준의 접근 — 단일 기술이 아닌 시스템 설계
중복 위험: 높음 — "주권적 AI 신원 아키텍처"가 포괄적
차별화 필요: 거버넌스+경제가 아닌 이식성(portability)에 초점
🔴 Zero-Trust Agent Gateway (SASE)

기술: SPIFFE 기반 에이전트 게이트웨이, 크로스-엔터프라이즈 접근 제어.
주요 특징:
• 다중 행위자 AI 생태계용 ID 중심 접근 제어
• SASE(Secure Access Service Edge) 원칙 정렬
중복 위험: 중간 — 게이트웨이 접근이지만 "이식성"은 다루지 않음
🟡 MIGT: Who Governs the Machine?

기술: 머신 아이덴티티 거버넌스 분류법.
주요 특징:
• 분류법(taxonomy) 제공 — 실제 구현이 아닌 분석 프레임워크
• CrowdStrike 장애(54~100억 달러 손실) 사례 인용
• 크로스보더 규제 대응 강조
중복 위험: 중간 — 거버넌스 관점, 이식성 기술은 다루지 않음
활용: 우리 특허의 배경/동기 섹션에서 MIGT를 인용하면 설득력 증가

1.5 차별화 전략

✅ 차별화 방향: "Identity Portability" ≠ "Identity Persistence"

DRC-369와 FractalNode는 영구 신원(persistent identity)에 초점 — 에이전트가 "누구인지"를 증명.
우리 특허는 이식성(portability)에 초점 — 에이전트의 신원/자격/평판을 플랫폼 간 이동 가능하게 하는 것.

DRC-369/FractalNode우리 특허 (Identity Portability)
영구 신원 생성신원 이식/이동
단일 플랫폼 내 신원크로스-플랫폼 신원 이식
Soulbound NFT (전송 불가)Transferable credential (전송 가능)
신원 존재 증명신원 + 자격 + 평판 통합 이식
블록체인 기반플랫폼 독립적 (블록체인 불필요)

💡 통합 특허 아이디어:
ZKP 특허의 "프라이버시 보존 증명" + Identity Portability의 "크로스플랫폼 이식"을 결합하면, "에이전트가 자신의 자격을 공개하지 않으면서 다른 플랫폼에서 자격을 증명하는 방법"이라는 완전히 새로운 특허가 됩니다.

이 조합의 선행기술은 0건입니다.

3.3 DRC-369 대응 청구항 전략

DRC-369는 "Identity Persistence" — 우리는 "Identity Portability"

독립항 구조:
1. "에이전트의 신원, 자격, 평판을 제1 플랫폼에서 제2 플랫폼으로 이식하는 방법"
2. "이식 시 신원의 무결성과 연속성을 증명하는 방법"
3. "이식 시 자격의 유효성을 목적 플랫폼에서 검증하는 방법"
4. "이식 시 평판의 이전을 원 플랫폼에서 목적 플랫폼으로 보장하는 방법"
5. "블록체인 없이 분산 합의 기반으로 위 1-4를 수행하는 방법"

핵심: Soulbound NFT는 "전송 불가"가 핵심인데, 우리는 "전송/이식 가능"이 핵심. 정반대의 방향.
