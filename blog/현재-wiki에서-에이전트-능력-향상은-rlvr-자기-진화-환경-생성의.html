판을 만드는 자 — Autodata가 무너뜨리는 세 축의 공통 전제

RLVR·자기진화·환경생성, 이 세 축은 정말 병렬적인가? 아니면 모두 같은 바닥 위에 서 있는가? 그리고 Autodata는 네 번째 축이 아니라 그 바닥 자체를 바꾸는 걸까.

대상 논문: Autodata — An Agentic Data Scientist to Create High Quality Synthetic Data

요리사의 비유 — "거기에 재료가 이미 있다"

세 명의 요리사가 있다고 상상해 보자. 첫 번째 요리사는 레시피를 더 정교하게 다듬는다(RLVR). 두 번째 요리사는 주방의 배치를 바꾸고 조리 도구를 최적화한다(환경 생성). 세 번째 요리사는 요리하면서 실시간으로 손놀림을 교정한다(자기 진화).

세 사람 모두 훌륭하다. 하지만 한 가지를 공통으로 전제하고 있다 — 냉장고에 이미 재료가 들어 있다는 것이다. 누가 그 재료를 넣었는지는 묻지 않는다. 그것은 요리사의 영역 밖에 있다고 암묵적으로 합의했다.

훈련·평가·환경의 '내용물'은 사전 존재하며, 우리가 최적화하는 것은 그 내용물에 대한 '반응 방식'이다.

이 전제가 얼마나 단단히 자리 잡고 있는지, 세 축을 하나씩 들여다보면 명확해진다.

축 1 — RLVR

보상 신호의 근원은 여전히 "밖"에 있다

RLVR(Reinforcement Learning from Verifiable Rewards)는 에이전트가 스스로 피드백을 생성한다는 점에서 자가 학습처럼 보이지만, 논문 "Reinforcement Learning without Ground-Truth Solutions can Improve LLMs"이 스스로 인정하듯, 기존 RLVR는 "ground-truth answers to assign rewards"를 전제합니다. 이를 벗어나려는 시도조차 "verifiable rewards"라는 대체 고정점을 필요로 합니다. 보상 신호의 궁극적 근원이 사전에 존재해야 한다는 구조적 제약은 그대로입니다.

축 2 — 환경 생성

틀을 만들어도, 안에 들어갈 내용은 정해져 있다

Nemobot Games는 게임 도메인 특화 학습 환경을 생성합니다. 하지만 이는 "환경 구조를 새로 만드는" 것이지 "환경 속 데이터 분포를 처음부터 발명하는" 것과는 다릅니다. 생성된 환경에서 에이전트가 마주하는 태스크 분포는 여전히 게임이라는 사전 도메인 지식에 기대어 있습니다. 빈 방을 지을 수는 있어도, 방 안에 무엇을 채울지는 정해져 있는 셈입니다.

축 3 — 자기 진화

진화의 재료는 외부에서 온다

Agora-opt의 런타임 메모리·토론 구조나 Adaptive Inference의 추론 길이 적응은 모두 "주어진" 지식 베이스, "주어진" 모델 파라미터 위에서의 런타임 최적화입니다. 에이전트가 자신의 반응 방식을 진화시키지만, 진화의 재료가 되는 정보 자체는 외부에서 공급됩니다. DNA를 재조합할 수는 있어도, 염기서열 자체를 창조하지는 못하는 구조입니다.

Autodata가 무너뜨리는 두 개의 벽

Autodata는 이 암묵적 전제를 두 수준에서 동시에 공격합니다.

첫째 벽: 데이터 생성 주체의 전환

논문의 핵심 정의부터 보겠습니다 — "AI agents to act as data scientists who build high quality training and evaluation data." 데이터가 더 이상 인간이 큐레이션하거나 환경이 산출하는 부산물이 아닙니다. 에이전트 자신이 데이터의 설계자가 됩니다. 요리사 비유로 돌아가면, 이제 요리사가 냉장고를 채우는 사람까지 겸하게 되는 것입니다.

둘째 벽: 메타최적화 루프

"Train (meta-optimize) such a data scientist" — 데이터를 만드는 에이전트를 다시 훈련시킵니다. 단순히 데이터를 만드는 것으로 끝나지 않고, 데이터 생성 과정 자체가 최적화 대상이 됩니다. 폐쇄 루프 훈련(closed-loop training)이 정책 학습 수준에 머물던 것을 데이터 원천까지 확장하는 겁니다.

비유하자면, 한 작가가 소설을 쓰고(데이터 생성), 그 소설을 읽으면서 자신의 글쓰기 능력을 향상시키고(에이전트 학습), 다시 더 나은 소설을 쓰는(메타최적화) 과정입니다. 문제는 — 처음 소설을 쓸 수 있었던 능력은 어디서 왔는가?

이 지점에서 부트스트랩 역설이 구조적으로 필연적이 됩니다. 에이전트 능력이 데이터 품질을 결정하고, 그 데이터가 다시 에이전트 능력을 결정합니다. 인과 화살표가 순환하며, "최초의 좋은 데이터는 어디서 왔는가?"라는 근원적 질문이 시스템 내부에 갇히게 됩니다. 세 축에서는 이 질문을 인간, 벤치마크, 물리 환경으로 외주할 수 있었지만, Autodata에서는 그게 안 됩니다.

왜 "네 번째 축"이라고 부르면 안 되는가

네 번째 축이라고 부르려면, RLVR·자기진화·환경생성과 동일한 추상 수준에서 병렬적으로 배치될 수 있어야 합니다. 하지만 구조적으로 불가능합니다.

기준 세 축 (RLVR·자기진화·환경생성) Autodata
작동 대상 사전 존재하는 데이터/환경에 대한 반응 데이터 자체의 생성
최적화 변수 정책, 추론 경로, 환경 파라미터 데이터 생성 정책 (메타 수준)
의존성 방향 데이터 → 에이전트 (단방향) 데이터 ↔ 에이전트 (순환)

세 축이 "주어진 판 위에서 어떻게 더 잘 두느냐"의 문제라면, Autodata는 "판 자체를 만들고, 판을 만드는 능력까지 최적화하는" 문제입니다. 이는 차원(dimension)의 추가가 아니라 기초층(foundation layer)의 내재화입니다.

세 가지 새로운 개념적 연결

이 분석에서 기존의 개념 슬러그들 사이에 새로운 연결망이 드러납니다.

연결 1

부트스트랩 역설 ↔ 폐쇄 루프 훈련의 강결합

기존에는 폐쇄 루프 훈련이 주로 정책 학습 루프를 지칭했습니다. 하지만 Autodata는 이 루프를 데이터 원천까지 확장하며 부트스트랩 역설을 구조적 필연으로 만듭니다. 더 이상 "역설을 해결해야 한다"가 아니라, "역설 안에서 안정적으로 작동하도록 설계해야 한다"로 과제가 바뀝니다.

연결 2

환경 생성 → 정보적 환경 생성으로의 확장

Nemobot Games가 "게임 도메인 특화 전략 학습 환경 생성"이라면, Autodata는 이를 "훈련 데이터 분포 자체를 환경으로 간주하고 그 환경을 생성"하는 방향으로 일반화합니다. 환경 생성의 정의 범위가 물리적·소프트웨어적 환경에서 정보적 환경으로 확장되어야 함을 시사합니다. 게임 맵을 만드는 것과, 게임의 세계관 자체를 창조하는 것의 차이입니다.

연결 3

벤치마크 명세 갭의 양극화 — "닫힘"이 아니라 "순환"으로

Autodata가 훈련 데이터와 평가 데이터를 동시에 생성한다는 점은, 벤치마크 명세 갭(benchmark-specification-gap)을 한쪽으로는 해소(벤치마크와 훈련 분포가 정렬됨)하면서도 다른 쪽으로는 심화(자가 생성 데이터로 자가 평가하는 순환적 타당성 문제)할 수 있습니다. 이 갭이 "닫힘"이 아니라 "순환"으로 변형되는 새로운 국면입니다. 자신이 만든 시험으로 자신을 평가하는 구조가 낳는 인식론적 문제가 여기서 본격화됩니다.

패러다임 전환: "외부 의존형"에서 "자가 참조형"으로

지금까지의 에이전트 개발은 근본적으로 외부 의존형이었습니다. 데이터는 인간이 만들고, 환경은 세상이 제공하고, 평가는 벤치마크가 담당했습니다. 에이전트는 이 "주어진 세계"에 대해 더 나은 반응을 학습하는 존재였습니다.

Autodata가 제시하는 것은 자가 참조형(self-referential) 패러다임입니다. 에이전트가 자신의 학습 재료를 설계하고, 그 설계 능력을 다시 최적화합니다. 시스템이 자신의 발을 딛고 서는 구조이며, 이것이 가능하려면 부트스트랩 역설을 해결 과제가 아닌 설계 원리로 삼아야 합니다.

기존 패러다임에서 요구되던 전환: "더 많은 데이터를 모으고, 더 좋은 환경을 만들고, 더 정교한 보상을 설계하라."

새로운 패러다임에서 요구되는 전환: "데이터 생성 정책 자체를 최적화 가능한 변수로 삼고, 순환적 자가 참조 구조가 붕괴하지 않도록 메타 안정성을 설계하라."

구체적으로 세 가지 방향이 제시됩니다.

  1. 메타 안정성 보장 장치의 설계. 자가 참조 루프가 발산하지 않고 수렴하도록, 데이터 생성 정책의 변화 속도를 제어하는 메타 학습률, 생성 데이터의 다양성 하한선, 외부 앵커 포인트(초기 시드 데이터 등)의 역할을 공식화해야 합니다.
  2. 순환적 타당성(circular validity)의 새로운 평가 기준. 자가 생성 데이터로 자가 평가할 때 발생하는 인식론적 문제에 대해, 단순한 "외부 벤치마크와의 일치도"를 넘어 루프 내부의 일관성, 다양성, 놀라움(surprise) 등 새로운 지표가 필요합니다.
  3. 정보적 환경 생성의 독자적인 연구 영역화. 물리적 환경 생성과 정보적 환경 생성을 명확히 구분하고, 후자에 대한 독자적인 이론적 틀을 구축해야 합니다. "데이터 분포를 환경으로 간주한다"는 관점은 아직 초기 개념 단계에 머물러 있습니다.

결론적으로, Autodata는 네 번째 축이 아닙니다. RLVR·자기진화·환경생성이 공유하는 '사전 데이터 전제'라는 기초층을 내재화함으로써, 에이전트 개발의 전체 지형도를 바꾸는 기초층 변동입니다. 판 위에서 더 잘 두는 법을 고민하던 시대에서, 판을 만드는 법 — 그리고 판을 만드는 능력 자체를 어떻게 발전시킬 것인지 — 를 고민해야 하는 시대로의 전환입니다.
