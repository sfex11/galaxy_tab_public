휴리스틱의 세 가지 얼굴, 하나의 결함
— 에이전트 오케스트레이션은 베이즈 합리적이어야 한다

도구 호출 여부, 파이프라인 병목 해결, 지식 상태 관리 — 기존 연구가 각각 독립적인 공학 문제로 다뤄온 세 가지 최적화 축이 사실은 동일한 근원적 결함의 서로 다른 증상이라는 분석.

최근 "agentic AI orchestration should be Bayes-consistent"라는 Position 논문이 제기한 주장은 단순한 방법론 제안이 아니다. 이것은 에이전트 오케스트레이션 연구 전반에 깔린 구조적 진단이다.

현재 Wiki에 축적된 오케스트레이션 관련 논문들은 크게 세 축으로 나뉜다: 도구 호출 여부를 결정하는 adaptive-inference, 파이프라인 레이턴시 병목을 해결하는 aggregate-pipeline-serving, 지식 상태를 관리하는 ADEMA. 각각이 나름의 문제를 식별하고 해법을 제시했지만, Position 논문의 렌즈로 보면 이 세 축은 같은 근원적 결함의 서로 다른 증상에 불과하다.

🩺 비유: 세 환자, 하나의 진단

세 환자가 각각 두통, 시력 저하, 어지러움을 호소하며 신경과·안과·이비인후과를 방문했다. 각 과에서 진통제, 시력 교정, 전정기능 재활을 처방했지만 증상은 낫지 않는다. 왜냐하면 세 증상 모두 같은 뇌종양의 위치에 따른 발현 양상이기 때문이다. 근본 원인을 치료하지 않고 증상별로 처방하면, 병은 결코 낫지 않는다.

에이전트 오케스트레이션에서도 마찬가지다. 과호출/과소호출, 병목 오진, 지식 상태 왜곡은 각각 다른 공학적 처방을 받아왔지만, 실제로는 불확실성 하에서 베이즈 합리성을 결여한 결정 구조라는 하나의 뿌리에서 자란 가지들이다.

01 도구 호출에서 EVI 계산의 부재

"도구 호출이 항상 유익한 것이 아니며, 일부 호출은 중복되거나 심지어 해롭다." — "To Call or Not to Call"

이 진단은 정확하다. 하지만 진단 이후의 처방이 문제다. 현재 시스템은 도구 호출 여부를 정적 임계값이나 규칙 기반 휴리스틱으로 결정한다. "confident가 0.7 미만이면 도구를 호출하라" 식의 규칙은 불확실성이 클수록 체계적으로 과호출(over-calling)하거나 과소호출(under-calling)한다.

베이즈 의사결정론에서 이 결정은 기대 정보 가치(Expected Value of Information, EVI)로 정식화된다:

EVI = P(유익 | 현재 상태) × 이득 − P(해로움 | 현재 상태) × 손실

EVI를 계산하려면 "현재 상태에 대한 불확실성이 얼마인가"를 확률적으로 표현할 수 있어야 한다. 그래야 "이 도구를 호출했을 때 그 불확실성이 얼마나 줄어드는가"를 추정할 수 있기 때문이다. 하지만 기존 휴리스틱은 이 확률을 추정하는 구조 자체가 없다. 확률 없이 "호출하라/말라"를 결정하는 것은, 온도계 없이 "덥다/춥다"를 규칙으로 판정하는 것과 같다.

핵심 결함
도구 호출은 본질적으로 '정보 수집의 가치 판단'이다

베이즈 의사결단 프레임워크 없이는 이 판단을 합리적으로 수행할 수 없다. 정적 임계값은 불확실성의 크기에 반응하지 못하므로, 불확실성이 극단적으로 크거나 작을 때 체계적 편향을 낳는다.

02 파이프라인에서 불확실성 전파의 단절

"파이프라인 전체의 레이턴시 병목이 개별 LLM 호출이 아닌 스키마 축적에 의해 결정된다." — "Tool Attention Is All You Need"

이 발견은 다중 서버 MCP 배포 환경에서의 실증적 관찰이지만, Position 논문의 렌즈로 읽으면 불확실성이 파이프라인을 따라 어떻게 전파되는지에 대한 모델 부재의 증상이 된다.

베이즈 일관성을 만족하는 오케스트레이터라면, 각 단계의 후방 확률(posterior)이 다음 단계의 사전 확률(prior)이 되어 파이프라인 전체에 일관된 불확실성 추적이 유지된다. 1단계에서 얻은 정보가 2단계의 불확실성을 얼마나 줄였는지가 정량적으로 전달되어야, 3단계에서 "추가 정보가 필요한가?"를 합리적으로 판단할 수 있다.

기존 시스템에서는 이 연결이 끊겨 있다. 각 단계는 이전 단계의 불확실성 상태를 모른 채 독립적으로 결정을 내리며, 결과적으로 어디서 불확실성이 누적되는지 인지하지 못한다. 그래서 가시적인 부분 — LLM 호출 지연 — 만 최적화하고, 실제 병목인 스키마 축적을 놓친다.

체계적 오진의 구조
베이즈 일관성 없이는 정보 이론적 비용을 정합하게 할당할 수 없다

병목을 잘못 잡은 것은 엔지니어의 실수가 아니다. 불확실성 전파 모델이 없으면 파이프라인 전체의 정보 이론적 비용을 각 단계에 정합하게 할당할 방법이 구조적으로 존재하지 않는다. 이것은 정보의 흐름을 추적할 수 없는 시스템에서 병목을 찾는 것과 같다.

03 ADEMA에서 베이즈 업데이트의 누적 오차

ADEMA는 "지식 상태 오케스트레이션 구조"를 제안하며, 긴 문맥에서 지식 상태를 조율하는 문제를 다룬다. Position 논문의 렌즈로 보면, 여기서 핵심 질문이 비어 있다.

❓ 빠진 핵심 질문

오케스트레이션된 지식 상태가 실제로 베이즈 업데이트 규칙을 따르는 정합한 사후 확률분포인가?

베이즈 업데이트의 기본 규칙은 명확하다:

P(h|e) ∝ P(e|h) × P(h)

새로운 관찰 e가 들어왔을 때, 기존 믿음 P(h)를 가능도 P(e|h)에 비례하여 갱신해야 한다. 지식 상태 관리가 이 규칙을 위배하면 — 예를 들어 최근 정보에 과도한 가중치를 두거나, 오래된 정보의 소거 없이 축적하면 — 지식 상태 자체가 왜곡된다.

그리고 이것이 치명적인 이유는, 왜곡된 지식 상태로부터 파생되는 모든 downstream 결정이 체계적으로 비합리적이 되기 때문이다. 긴 문맥일수록 베이즈 업데이트의 누적 오차가 폭발하므로, ADEMA가 다루는 도메인에서 이 정합성 문제는 더욱 심각하다.

오염의 파급 경로
지식 상태의 왜곡은 모든 downstream 결정을 오염시킨다

지식 상태 = 오케스트레이션의 "근거". 근거가 왜곡되면, 그 위에서 이루어지는 도구 호출 판단, 다음 단계 우선순위 결정, 사용자 응답 생성 — 모든 것이 체계적으로 비합리적 선택을 산출한다. 이것이 "증상별 처방"이 실패하는 구조적 이유다.

04 Crab, 4차원 그래프, 그리고 다 에이전트 합의

베이즈 관점을 적용하면, Wiki에 산재한 추가 논문들까지 같은 프레임 아래 재분류된다.

Crab과 상태 체크포인트의 비효율

Crab 논문이 식별한 algorithm-system-translation-gap은 "에이전트의 의미론적 의존 그래프를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때의 비효율"이다. 베이즈 관점에서 재해석하면, OS 스냅샷은 에이전트가 현재 무엇을 믿는지(belief state)에 대해 무차별적이다.

베이즈 필터가 유지해야 할 믿음 상태는 불확실성을 정량화한 최소 충분 통계량(sufficient statistic)이다. 전체 스냅샷에서 이것을 추출할 구조적 메커니즘이 없으므로, 상태 복원 시 의미론적 정보가 손실된다. Crab이 구축하는 4차원 상태 전이 그래프는 사실상 베이즈 필터가 유지해야 할 믿음 상태의 구조를 명시화하려는 암묵적 시도로 읽을 수 있지만, 베이즈 업데이트 규칙을 내장하지 않는 한 결정론적 정합성은 보장되지 않는다.

Agora-OPT와 분산 합의 문제

agora-opt는 분산 메모리 기반 에이전트 간 지식 공유를 다룬다. 여기서 새로운 차원의 문제가 발생한다: 각 에이전트의 지식 상태가 베이즈 일관성을 갖추고 있다 하더라도, 에이전트 간 사후 확률의 합의(consensus)가 추가로 필요하다.

기존의 분절적 접근은 "메모리를 공유하면 된다"는 수준이지만, 베이즈 일관성 프레임워크에서는 이것이 다 에이전트 베이즈 합의(multi-agent Bayesian consensus) 문제로 정식화되어야 한다. 그렇지 않으면 분산 자체가 새로운 비합리성의 원천이 된다 — 각 에이전트가 서로 다른 왜곡된 믿음 상태를 공유하며 비합리성을 증폭시키는 구조다.

05 하나의 결함으로 환원

지금까지 살펴본 다섯 가지 최적화 축을 하나의 표로 정리하면, 구조적 패턴이 명확해진다.

최적화 축 구체적 논문 베이즈 관점 결함 산출되는 비합리성
도구 호출 여부 To Call or Not to Call EVI 계산 구조 부재 불확실성 하 과호출/과소호출
파이프라인 서빙 Tool Attention Is All You Need 불확실성 전파 단절 병목 오진 → 잘못된 최적화 대상
지식 상태 관리 ADEMA 사후 확률 정합성 미보장 왜곡된 믿음 → 모든 downstream 오염
상태 체크포인트 Crab / 4D 그래프 최소 충분 통계량 추출 불가 상태 복원 시 의미론적 정보 손실
분산 메모리 Agora-OPT 에이전트 간 합의 메커니즘 부재 분산 비합리성의 증폭

이 표의 오른쪽 두 열이 말하는 것은 하나다. 베이즈 비일관성이라는 단일 원인이, 도메인마다 다른 얼굴로 나타날 뿐이다.

"베이즈 일관성"을 메타-기준(meta-criterion)으로 삼으면, 기존에 미분류로 남아있던 adaptive-inference, aggregate-pipeline-serving, ADEMA, agora-opt, algorithm-system-translation-gap이 모두 "베이즈 비일관성의 도메인별 발현 양상"이라는 하나의 이론적 축 아래 재분류된다. 특히 4차원 상태 전이 그래프는 현재 기술적 데이터구조로만 기술되어 있으나, 이를 "베이즈 필터가 유지해야 할 믿음 상태의 구조적 스키마"로 재해석하는 것이 새로운 연결 지점이다.

핵심 통찰

기존 최적화 축들의 결함은 각기 다른 증상일 뿐, 근원은 모두 '불확실성 하에서 베이즈 합리성을 결여한 결정 구조'라는 하나의 구조적 결함으로 환원된다.

에이전트 오케스트레이션 설계는 더 이상 공학적 휴리스틱의 영역이 아니다. 불확실성을 정량화하고, 정보의 가치를 기대값으로 계산하며, 믿음 상태를 베이즈 업데이트 규칙에 따라 갱신하고, 파이프라인 전체에 불확실성을 일관되게 전파하는 것 — 이것이 오케스트레이션 결정이 합리적이기 위해 충족해야 할 최소 조건이다.

Position 논문이 오케스트레이션 결정의 베이즈 일관성을 요구하는 것은, 가지들을 따로따로 치는 것이 아니라 뿌리를 베이즈 의사결단론으로 교체하자는 패러다임 전환을 선언하는 것이다. 증상이 아닌 원인을, 휴리스틱이 아닌 원칙을, 분절이 아닌 통합을 — 그것이 다음 세대 에이전트 오케스트레이션의 출발점이다.
