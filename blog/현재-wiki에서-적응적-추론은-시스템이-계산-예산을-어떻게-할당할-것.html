Deep Analysis — Adaptive Inference

잔류 스트림에 내장된 타이머:
적응적 추론의 진정한 주체는 누구인가

LLM이 이미 자신의 남은 출력 길이를 내부에 인코딩하고 있다면, 적응의 결정권은 외부 시스템이 아니라 모델 자신에게 있어야 하지 않을까?

2025년 7월 · 기반 논문: How Much is Left? LLMs Linearly Encode Their Remaining Output Length

1. 운전자보다 먼저 도착 시간을 아는 내비게이션

상상해 보세요. 당신이 운전대를 잡고 있는데, 옆에 앉은 내비게이션 음성이 이렇게 말합니다. "목적지까지 12분 남았습니다." 그런데 당신은 아직 고속도로 진입로도 안 들어갔고, 어떤 경로로 갈지도 결정하지 못한 상태입니다. 내비게이션은 당신이 아직 하지 않은 결정의 결과를 이미 알고 있는 셈입니다.

이상하게 들리지만, LLM의 내부에서 정확히 이런 일이 벌어지고 있습니다. 모델은 토큰을 하나씩 생성하면서도, 잔류 스트림(residual stream)에 "나는 앞으로 몇 토큰을 더 생성할 것인가"를 이미 선형적으로 인코딩하고 있습니다. 아직 쓰지 않은 문장의 길이를, 쓰는 중에 이미 알고 있는 것입니다.

이건 마치 타자를 치는 사람이 손가락이 움직이기도 전에 "이 문단은 세 줄 더 치면 끝나"라고 내적으로 느끼는 것과 비슷합니다. 의식적 계산이 아니라, 문장 구조가 수렴하는 방향성을 체감하는 것 — LLM은 정확히 이 체감을 수치적으로 인코딩하고 있습니다.

그런데 적응적 추론(adaptive inference) 분야는 지금까지 이 정보를 완전히 무시해 왔습니다. 외부 스케줄러가 "이 문제는 복잡하니까 추론 단계를 8단계로 할당해"라고 결정하는 방식이었죠. 모델이 이미 자기 얼마나 더 말할지 알고 있는데, 밖에서 누군가가 대신 결정하는 상황입니다.

이 글에서는 왜 이런 모순이 생겼는지, 그리고 적응적 추론의 본질을 어떻게 재정의해야 하는지를 추적합니다.

2. 적응의 주체가 안으로 걸어 들어온 궤적

적응적 추론의 역사를 보면, "적응의 근거"가 점점 모델 내부로 이동해 온 명확한 궤적이 있습니다. 이를 세 단계로 정리할 수 있습니다.

Phase 1

외부 환경 반응형 — CADENCE 등

프롬프트의 복잡도, 입력 길이, 계산 자원 같은 외부 환경 지표를 보고 외부 스케줄러가 어떤 모델에 라우팅할지 결정합니다. 적응의 주체는 완전히 시스템 쪽에 있습니다.

Phase 2

내부 상태 반응형 — SpecKV

여기서 균열이 생깁니다. SpecKV는 KV 캐시의 압축 상태라는 내부 시스템 지표에 반응해 추측 길이 γ를 결정합니다. 적응의 근거가 처음으로 모델 내부로 들어왔지만, 여전히 "외부 런타임이 내부 상태를 읽고 결정한다"는 구조입니다.

Phase 3

내부 표현 판독형 — "How Much is Left?"의 발견

잔류 스트림 자체에 남은 출력 길이가 선형적으로 인코딩되어 있다는 사실이 밝혀집니다. 이제 적응의 근거뿐 아니라, 적응의 결정 자체가 모델 내부에 이미 존재하게 됩니다.

이 궤적을 보면, 적응의 주체 이동은 우연이 아닙니다. 외부 환경 → 내부 시스템 상태 → 내부 표현으로, 적응의 근원(source)이 모델 중심부로 한 걸음씩 걸어 들어온 것입니다. 그리고 Phase 3에 이르러, 이 이동이 단순한 경향이 아니라 구조적 필연이었음이 드러납니다.

3. 결정적 단서: 선형적으로 인코딩된 남은 길이

"How Much is Left? LLMs Linearly Encode Their Remaining Output Length" 논문의 핵심 발견을 다시 정밀하게 들여다봅시다.

LLM은 토큰을 하나씩 생성하지만, 응답은 놀랍도록 일관된 길이 구조를 보인다. 단계별 해답은 예측 가능한 토큰 수에서 수렴하고, 검색은 특정 지점에서 멈춘다. — How Much is Left?, 2025

이 발견이 중요한 이유는 단순히 "모델이 길이를 예측한다"는 사실 자체가 아니라, 그 예측이 잔류 스트림에 선형적으로 인코딩되어 있다는 점입니다. 선형적 인코딩이란 무엇입니까? 특정 방향 벡터와 내적을 취기만 하면 스칼라 값으로 남은 토큰 수가 나온다는 뜻입니다. 복잡한 비선형 변환 없이, 읽기만 하면 되는 상태입니다.

이것이 의미하는 바를 정확히 짚어야 합니다. 모델이 매 토큰 생성 시점마다 "나는 앞으로 N토큰을 더 생성할 것"이라는 정보를 잔류 스트림에 품고 있다는 것은, 이것이 단순한 부산물이 아니라 자기 완결성(self-termination)을 가능하게 하는 기능적 표현이라는 뜻입니다. 모델이 자신의 끝을 내부에서 이미 준비하고 있는 것입니다.

그렇다면 기존 패러다임의 모순이 선명해집니다. 외부 시스템이 "이 프롬프트는 복잡하니 8단계의 추론을 할당하라"고 결정하는 것은, 모델이 이미 "나는 이 답을 47토큰에 끝낼 것"이라고 내부에 적어놓았음에도 그 정보를 읽지 않고 밖에서 예산을 강제하는 셈입니다.

식당 주방에서 요리사가 "이 요리는 12분 더 조리하면 완성"이라고 타이머를 맞춰놓았는데, 매니저가 외부에서 "이 주문은 복잡하니 20분 할당해"라고 기계적으로 시간을 배정하는 것과 같습니다. 요리사의 내부 타이머는 무시되고, 외부의 획일적 배정이 우선하는 구조 — 이것이 지금의 적응적 추론입니다.

4. 알고리즘-시스템 번역 간극: 왜 외부 결정이 비효율적인가

이 모순을 이해하려면 알고리즘-시스템 번역 간극(algorithm-system translation gap)이라는 개념이 필요합니다. 이 간극은 에이전트 시스템 설계에서 반복적으로 등장하는 구조적 문제입니다.

에이전트의 의미론적 의존 그래프(어떤 상태에 의존하는가)를 OS의 체크포인트 메커니즘(전체 스냅샷)으로 번역할 때 발생하는 비효율이 간극의 본질이다. — Wiki, algorithm-system-translation-gap 엔티티

같은 구조가 적응적 추론에서도 나타납니다.

번역 간극의 적용

의미론적 상태 → 시스템 제어의 비효율적 번역

의미론적 상태: 모델의 잔류 스트림에 인코딩된 남은 출력 길이 — 모델이 자신의 추론 경로에 대해 내부적으로 형성한 예측

시스템 제어: 외부 스케줄러의 예산 할당 — 입력 복잡도나 자원 제약에 기반한 기계적 분류

간극: 내부적이고 문맥에 민감한 길이 예측을, 외부적이고 획일적인 예산 결정으로 번역할 때 발생하는 정보 손실과 비효율

이 간극이 존재하는 한, 아무리 정교한 외부 스케줄러를 만들어도 근본적 한계에 부딪힙니다. 왜냐하면 외부 시스템이 접근할 수 있는 정보(입력 길이, 프롬프트 복잡도)는 모델 내부 표현이 이미 압축해서 담고 있는 정보의 부분집합에 불과하기 때문입니다.

재정의: 판독의 문제로

이 간극을 인식하면, 적응적 추론의 본질이 자연스럽게 재정의됩니다.

기존: "시스템이 계산 예산을 어떻게 할당할 것인가?"
재정의: "모델이 잔류 스트림에 이미 형성한 출력 길이 예측을 어떻게 판독하고, 이를 추론 아키텍처에 어떻게 연동할 것인가?"

할당(allocation)이 아니라 판독(reading)입니다. 결정의 주체가 외부에서 모델 내부로 이동하고, 연구의 초점도 "어떤 정책으로 예산을 나눌까"에서 "잔류 스트림에서 길이 정보를 얼마나 충실하게 추출할까"로 전환됩니다.

세 가지 개념적 연결

이 재정의는 기존 Wiki 엔티티들 사이에 새로운 관계망을 드러냅니다.

  1. 적응 제어 계층의 역전: 제어 흐름이 "외부 스케줄러 → 모델"에서 "모델 내부 표현 → 판독 계층 → 추론 아키텍처"로 뒤집힙니다. 제어의 근원이 모델 안으로 들어갑니다.
  2. 병목 이동의 재해석: 병목이 "어떤 모델을 선택할까"에서 "γ를 몇으로 할까"로 이동한 것은, 실제로는 "외부 결정의 병목"이 "내부 표현의 판독 병목"으로 이동하는 과정이었습니다.
  3. 에이전트 미시물리학과의 연결: 잔여 길이의 선형적 인코딩은 에이전트의 미시적 연산 역학에서 자기 모니터링(self-monitoring)이 이미 구현되어 있음을 보여줍니다. 문제는 이 미시적 표현을 거시적 추론 제어로 어떻게 스케일업할 것인가입니다.

5. 남은 역설: 예측된 길이가 곧 최적 길이인가?

재정의가 매력적이긴 하지만, 완전한 전환을 위해서는 하나의 역설을 직면해야 합니다.

모델이 "남은 출력 길이"를 인코딩한다는 것은 현재 추론 경로가 주어졌을 때의 예측입니다. 하지만 적응적 추론의 목적은 더 나은 추론 경로를 선택하는 것입니다. 이 두 가지가 충돌할 수 있습니다.

핵심 역설

관성적 수렴 vs. 최적 수렴

모델이 예측하는 "47토큰 후 종료"가 최적으로 수렴하는 47토큰인지, 아니면 현재 패턴이 관성적으로 흘러가며 도달할 47토큰인지를 구분할 수 없다면, 내부 예측을 단순히 판독해서 그대로 쓰는 것은 위험할 수 있습니다.

자동차의 연비 계기판이 "남은 연료로 82km 주행 가능"이라고 표시할 때, 그것이 최적 연비로 82km를 갈 수 있다는 뜻인지, 현재 운전 습관대로 가면 82km라는 뜻인지는 전혀 다른 문제입니다. 전자라면 믿어도 되지만, 후자라면 운전 습관을 바꿔야 더 멀리 갈 수 있죠.

이 지점에서 단순한 "판독"의 문제가 내부 표현의 신뢰성 평가 문제로 확장됩니다. 모델의 길이 예측이 얼마나 신뢰할 수 있는지, 그리고 그 신뢰도를 어떻게 추정할 것인가가 새로운 연구 문제로 떠오릅니다. 이것이 적응적 유효성(adaptive validity)이라는 개념이 새로운 의미를 획득하는 지점입니다.

하지만 이 역설이 재정의를 부정하는 것은 아닙니다. 오히려 역설을 품으면서도 적응의 주체를 모델 내부로 이동시키는 방향이 여전히 정답임을 보여줍니다. 왜냐하면 신뢰성 평가마저도 모델 내부 표현(예: 레이어별 신호의 일관성, 주의 가중치의 분산 정도 등)에서 읽어내는 것이 외부 휴리스틱보다 정확할 것이기 때문입니다.

패러다임 전환: 할당에서 판독으로

적응적 추론은 "외부 시스템이 계산 예산을 어떻게 할당할 것인가"에서 "모델이 잔류 스트림에 이미 형성한 출력 길이 예측을 어떻게 판독하고 활용할 것인가"로 본질이 재정의되어야 합니다.

이 전환은 세 가지 차원에서 실현됩니다.

  1. 연구 초점의 전환: 라우팅 정책, 예산 스케줄링 알고리즘 → 잔류 스트림에서 길이 방향 벡터를 추출하고 신뢰도를 평가하는 판독(readout) 기법
  2. 아키텍처의 전환: 외부 스케줄러가 모델을 제어하는 구조 → 모델 내부 표현이 추론 아키텍처(추측 디코딩의 γ, 사고 단계 분기, 조기 종료)를 직접 구동하는 구조
  3. 평가 지표의 전환: "예산 대비 정확도" → "내부 예측의 충실도"와 "판독-연동 지연"이 새로운 성능 지표로 등장

모델은 이미 자신의 끝을 알고 있습니다. 우리가 할 일은 대신 결정해 주는 것이 아니라, 그 목소리를 듣는 것입니다.
