# Llm Agent: 합성 분석

**생성일**: 2026-04-14 (업데이트)
**관련 논문**: 115편
**분석 기간**: 2026-03-19 ~ 2026-04-14

## 1. 핵심 연구축

### (1) 평가의 현실화
GymAnything은 에이전트 환경 구축의 수동 병목을 메타 프레임워크로 자동화했고, ClawBench(3회 등장)는 144개 라이브 플랫폼에서 시뮬레이션-현실 격차를 정량화했다. TDAD는 기존 해결률 중심 평가에 **코드 회귀 방지 능력**이라는 새 차원을 추가하여, 에이전트가 "풀면서 망가뜨리지 않는지"를 AST 기반 그래프 분석으로 측정한다. Blindfolded LLMs는 티커 익명화로 암기 편향과 생존자 편향을 제거하여 금융 에이전트의 **진정한 시장 이해도**를 검증하는 방법론을 제시했다.

### (2) 구조·메타인지 설계
ActWisely는 도구 호출 여부를 스스로 판단하는 메타인지를 핵심 과제로 제기했다. 64-Token Creative Agent는 단 64토큰 템플릿이 복잡한 반복적 프롬프트 확장과 동등한 창의적 결과를 낳음을 보여주며, **"Less LLM, More Structure"**에 강력한 실증적 근거를 추가한다. Post-Training Local LLM은 4B 모델이 SFT+RL 2단계 후훈련만으로 Claude Opus 4.6급 보안 태스크 성능(95.8%)을 달성하여, 모델 크기가 아닌 **아키텍처적 설계와 훈련 전략**이 결정적임을 확인했다.

### (3) 보안·거버넌스의 계층 확장
TraceSafe는 안전성 취약점이 중간 실행 궤적에 존재함을 밝혔고, 사회심리학적 공격 벡터가 다중 에이전트를 훼손할 수 있음이 드러났다. Differential Privacy 논문은 기업 데이터 관점의 프라이버시 유출을 수학적으로 정립하여 **정보 이론적 프라이버시 보장과 출력 품질 간 최적 트레이드오프**를 제시한다. Governed Memory는 멀티 에이전트 프로덕션에서 메모리 사일로·거버넌스 파편화·비구조화 메모리 등 5가지 구조적 과제를 식별하고, 공유 메모리의 구조화된 거버넌스가 시스템 신뢰성을 좌우함을 입증했다.

### (4) 자기 진화와 도메인 확장
AgentFactory는 성공 경험을 텍스트가 아닌 **실행 가능한 서브에이전트 코드**로 축적하는 자기 진화 패러다임을 제시하여, 경험의 재현성과 재사용성을 근본적으로 향상시킨다. 동시에 에이전트의 적용 도메인이 급격히 확장되었다: 체화된 내비게이션(AgentVLN), 법률 판단(Traffic Responsibility), 경제 시뮬레이션(MALLES), 창의적 이미지 생성(64-Token Agent), 보안 자동화(Post-Training Local LLM)까지. VLM 기반 에이전트가 2D 의미 이해를 넘어 3D 공간 추론까지 통합하는 흐름(AgentVLN, CADENCE)은 범용 체화 에이전트 구현의 이정표다.

### (5) 통합 아키텍처
PSI의 공유 상태 버스는 독립 모듈 간 고립을 해결하고, Paper Circle은 멀티에이전트 역할 분화를, Maestro는 GUI 동적 적응을 제시했다. VideoAtlas는 장시간 비디오를 계층적 그리드로 변환하여 로그 스케일 연산의 탐색 가능한 환경을 제공하고, **상태 공유·효율적 환경 설계·협조 프로토콜**이 핵심 설계 원리로 부상한다.

## 2. 트렌드

- **평가의 다차원화**: 시뮬레이션 → 라이브 플랫폼(ClawBench) → 회귀 방지(TDAD) → 편향 제거(Blindfolded)로 평가 축이 다각화된다.
- **"Less LLM, More Structure"의 실증화**: 64토큰 템플릿, 4B 모델의 대형 모델급 성능 등 구조적 효율성이 반복적으로 입증된다.
- **경험의 코드화**: 텍스트 기반 리플렉션에서 실행 가능 코드로 경험 저장 패러다임이 전환된다(AgentFactory).
- **보안 관심의 수직·수평 확장**: 출력 → 중간 궤적 → 데이터 접근(차등 프라이버시) → 머신 아이덴티티로 위협 모델이 심화되고, 도메인별 신뢰성 검증(금융, 보안, 코드)이 병행된다.

## 3. 핵심 논점

- **구조 vs. 모델의 비선형 상호작용**: 64토큰 템플릿과 4B 모델의 사례는 구조가 모델을 대체할 수 있음을 보여주지만, 구조적 요소가 스케일링 법칙과 어떻게 비선형적으로 결합하는지는 미해결이다.
- **자율성 vs. 통제의 균형**: 에이전트 자율성 확대(자기 진화, 도메인 확장)와 거버넌스 강화(메모리 거버넌스, 차등 프라이버시, 머신 아이덴티티) 사이의 설계 원칙이 부재하다.
- **효율성의 한계 조건**: 경량화 사례들이 도메인 특화 태스크에서 입증되었으나, 범용 에이전트에서 동일한 효율성이 성립하는지 불투명하다.

## 4. 시사점

LLM 에이전트는 **"모델을 더 크게"에서 "시스템을 더 똑똑하게"**로 전환하는 변곡점에 있으며, 이번 분석은 그 방향을 세 가지로 구체화한다. **(1)** 평가를 해결률에서 회귀 방지·편향 제거까지 다차원화하고, **(2)** 에이전트 경험을 텍스트에서 실행 가능 코드로 전환하며, **(3)** 보안을 출력 수준에서 데이터 접근·메모리 거버넌스까지 계층화해야 한다. Governed Memory의 메모리 거버넌스 패턴과 AgentFactory의 실행 가능 서브에이전트 패러다임은 각각 멀티에이전트 프로덕션과 자기 진화의 표준으로 발전할 가능성이 높다.
