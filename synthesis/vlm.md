# Vlm: 합성 분석

## 1. 핵심 발견: VLM의 공간 맹점과 극복의 두 갈래

26편의 논문이 일치되게 지적하는 VLM의 근본적 한계는 **공간 추론(spatial reasoning)의 부재**다. Loc3R-VLM, 3D-Layout-R1, Meanings and Measurements, Generation Models Know Space 등이 "VLM은 의미적 그라운딩에 강하지만 3D 공간에서의 정밀 추론에는 설계되지 않았다"는 문제 의식을 공유한다. 해결 경로는 두 갈래로 나뉜다.

**암묵적 프라이어 전이** vs **명시적 구조 주입**: Generation Models Know Space는 비디오 생성 모델이 학습 과정에서 획득한 암묵적 3D 프라이어를 전이하는 패러다임을 제안한다. 반면 3D-Layout-R1은 장면 그래프라는 명시적 구조 위에서 추론하고, Loc3R-VLM은 단안 비디오로 3D 추론을 명시적으로 학습시킨다. Meanings and Measurements는 다중 에이전트가 의미·공간·메트릭을 분리 처리하는 구조적 모듈화의 변형을 보여준다. 도메인 특화성이 높아질수록 명시적 구조가 유리하다는 간접적 증거가 나타난다.

## 2. 체화된 배포: 성능에서 신뢰성으로의 전환

NavTrust, DRIVE-Nav, AMIGO, Chameleon 등 10편 이상이 체화된 AI 맥락에서 VLM을 다룬다. 가장 두드러진 공통 논점은 **평가 패러다임의 전환**이다. NavTrust는 이상적 조건 성능이 실배포 신뢰성을 보장하지 않음을 체계적으로 입증하며, AMIGO는 단일 턴 정답률에서 다중 턴 전략적 상호작용 능력으로 평가 축을 이동시킨다.

DRIVE-Nav는 지속적 방향(persistent direction)을 탐색 단위로 사용해 불완전 관측 하의 경로 불안정성을 구조적으로 해소하고, Chameleon은 의미 압축 메모리가 원시 지각 단서를 버려 비마르코프 의사결정에 실패하는 문제를 에피소딕 메모리로 극복한다. 공통 메시지: **체화된 환경에서는 추론 아키텍처의 구조적 강건성이 순수 성능만큼 중요하다.**

## 3. 규모 확장과 효율성의 이중 압력

3DCity-LLM은 실내 수준에서 도시 규모로 스케일을 확장하며 coarse-to-fine 병렬 브랜치 인코딩을 도입한다. VISOR는 반대 방향에서 접근—시각 토큰 축소(token pruning)에 정면 도전하며, 토큰을 버리지 않고 레이어별 cross-attention을 동적으로 희소화하는 직교적 전략을 제시한다. 정보 병목 없는 효율화라는 VISOR의 주장은 도시 규모 처리를 다루는 3DCity-LLM과 시너지 가능하나 직접적 결합은 아직 탐색되지 않았다.

## 4. 단일 모델 vs 다중 에이전트의 긴장

Meanings and Measurements는 다중 에이전트 분담 구조를 채택하고, AMIGO도 에이전트-오라클 상호작용을 사용한다. 이는 VLM 자체를 강화하는 Loc3R-VLM, 3D-Layout-R1의 단일 모델 접근과 구조적 긴장을 형성한다. MARCUS가 의료 도메인에서 에이전틱 VLM의 실현 가능성을 입증한 것은, 도메인 특화 시 다중 에이전트 모듈화가 더 자연스러운 설계가 됨을 시사한다.

## 5. 미래 방향

- **신뢰성-성능 통합 설계**: NavTrust가 평가 측면에서 제기한 신뢰성 문제를 아키텍처 수준에서 해결하는 연구가 부재하다.
- **암묵적-명시적 하이브리드**: 생성 모델의 암묵적 프라이어와 장면 그래프의 명시적 구조를 결합하는 연구가 아직 없다.
