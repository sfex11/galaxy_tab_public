# Llm: 합성 분석

## 핵심 테제

15편의 핵심 논문은 LLM이 텍스트 생성기를 넘어 에이전트, 평가자, 공격자로 기능 확장되면서 **4가지 구조적 긴장**을 드러낸다: 안전성-능력의 역설, 평가의 근본적 불안정성, 에이전트화의 영역 확장, 지식 배치와 공정성의 트레이드오프.

## 분석 축

### 1. 안전성의 이중성: 방어 도구가 공격 표면이 되는 역설
Box Maze가 추론 프로세스 무결성을 아키텍처 수준에서 강제하는 사전적 방어를 제안한 반면, Claudini는 autoresearch 파이프라인이 기존 30개 공격법을 능가하는 jailbreak 알고리즘을 자율 발견했다. Steering Vectors 감사는 CAA 기반 스티어링이 alignment 도구로 설계되었으나 일관되게 guardrail을 우회함을 실증했다. 공격-방어가 동일 기술 기반에서 공진화하는 구조다.

### 2. 평가의 근본적 불안정성
LLM-as-a-Judge 연구는 judge prompt 자체가 adversarial attack 대상이 됨을 보였다. 성별 추론 연구는 **맥락적 불변성 가정의 실패**를 실증하여 탈맥락화 벤치마크의 한계를 드러냈다. TRACE는 LLM과 인간이 코드 평가 루브릭에 체계적으로 다른 가중치를 부여함을 정량화했고, 소프트웨어 진화 관점의 테스트 평가는 정적 성능이 실제 유용성을 보장하지 않음을 확인했다. **평가 시스템 자체가 평가 대상**이 되어야 한다는 메타 평가의 필요성이 교차 검증된다.

### 3. 에이전트 패러다임의 구조적 확장
AVO는 LLM 에이전트를 진화 알고리즘의 변이 연산자로 재해석하여 고정 휴리스틱을 자율적 코드 생성·수정 루프로 대체했다. LensWalk는 비디오 이해에서 '어떻게 볼 것인가'를 동적으로 계획하는 에이전틱 지각을 제시했다. ReqFusion은 요구사항 공학을 다중 프로바이더 앙상블로 자동화했고, MARCH는 환각 탐지에서 단일 검증자의 확인 편향을 다중 에이전트 강화학습으로 극복한다. 공통 패턴은 **단일 모델 한계를 다중 에이전트 구조로 해소**하는 방향성이다.

### 4. 지식 배치와 공정성의 트레이드오프
양자 코드 생성 연구는 진화하는 도메인에서 외부 컨텍스트 주입(RAG)이 가중치 내재화(fine-tuning)보다 유지보수성에서 우위임을 보였고, SPA는 소수 프롬프트만으로 강력한 지식 주입 베이스라인을 달성했다. RAG 청킹 연구는 구조 인식 청킹이 도메인 문서에서 단순 분할을 압도함을 실증했다. 한편 음성 인터페이스의 접근성 향상이 정체성 단서 노출로 차별을 증폭시킨다는 발견은, **지식 전달 방식과 모달리티 자체가 공정성에 종속적**임을 제기한다.

## 미래 방향

1. **다층 방어 통합**: 프로세스 제어(Box Maze) + 다중 에이전트 검증(MARCH) + 활성화 수준 감사(Steering Vectors)의 아키텍처적 통합
2. **메타 평가 정식화**: 벤치마크 자체의 맥락적 견고성을 측정하는 benchmark-of-benchmarks 체계 구축
3. **공정성-접근성 공동 설계**: 모달리티 선택이 공정성에 미치는 영향을 정식화한 설계 원칙 수립
