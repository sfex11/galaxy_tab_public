# Llm: 합성 분석

**생성일**: 2026-06-01  
**관련 논문**: 78편  
**최종 업데이트**: 2026-06-01

## 핵심 테제

15편의 논문은 LLM이 생성기에서 에이전트·평가자·공격자로 기능 확장되면서 **자기 참조적 역설**을 구조적으로 드러낸다: LLM이 자신의 안전성을 보장하려 할수록 새로운 공격 표면이 열리고, 자신의 출력을 평가하려 할수록 평가 자체의 불안정성이 증폭된다. 이 역설은 4개 분석 축으로 분해된다.

## 분석 축

### 1. 안전성의 이중성: 방어 도구가 공격 표면이 되는 역설

Box Maze는 추론 체인을 "Box" 제어 단위로 분리하는 사전적 아키텍처 방어를 제안하지만, Claudini는 autoresearch가 30개 이상의 공격법을 능가하는 jailbreak 알고리즘을 자율 발견한다. Steering Vectors 감사는 CAA 기반 스티어링이 alignment 도구임에도 safety guardrail을 일관되게 우회함을 실증하여, 방어와 공격이 동일 추상화 계층에서 공진화함을 보여준다. 아키텍처 수준 제어(Box Maze)만이 활성화 수준 조작(Steering Vectors)과 알고리즘 수준 자율 공격(Claudini)에 대한 방어층을 제공할 수 있다.

### 2. 평가의 근본적 불안정성: 평가자가 평가 대상이 되는 메타 위기

LLM-as-a-Judge 연구는 judge prompt 자체가 adversarial attack 대상이 됨을 보이고, 성별 추론 연구는 무정보적 최소 맥락 도입만으로 **맥락적 불변성 가정이 무효**임을 실증한다. TRACE는 코드 평가에서 LLM과 인간이 루브릭에 체계적으로 다른 가중치를 부여함을 정량화했고, 소프트웨어 진화 관점의 테스트 평가는 정적 스냅샷 성능이 코드 변경 시 커버리지 감소로 이어짐을 확인했다. 평가 시스템 자체가 평가 대상이 되어야 한다는 메타 평가의 필연성이 네 논문에서 수렴적으로 도출된다.

### 3. 에이전트 패러다임의 구조적 확장: 단일 모델 한계를 다중화로 해소

공통 패턴은 단일 LLM의 인지적 한계를 구조적 다중화로 극복하는 방향성이다. AVO는 에이전트를 진화 연산자로 재해석하여 자율적 생성·수정·자기비판 루프를 구현하고, LensWalk는 추론기가 지각 전략을 동적으로 계획하는 에이전틱 지각을 제시한다. ReqFusion은 다중 프로바이더 앙상블로 단일 모델 의존 리스크를 분산시키고, MARCH는 환각 탐지에서 단일 검증자의 확인 편향을 다중 에이전트 강화학습 기반 교차 검증으로 극복한다. 에이전트화의 핵심은 모델 능력이 아닌 **다중 주체 간 상호 검증 루프의 구조적 설계**에 있다.

### 4. 지식 배치와 공정성의 트레이드오프

SPA는 소수 프롬프트만으로 강력한 지식 주입이 가능함을, 양자 코드 생성 연구는 빠르게 진화하는 도메인에서 외부 컨텍스트 주입이 가중치 내재화보다 유지보수성에서 우위임을 보인다. 반면 음성 인터페이스 접근성 연구는 정체성 단서 노출로 차별을 증폭시킬 수 있어 **접근성과 공정성의 상충**을 제기하고, RAG 청킹 연구는 도메인 특화 구조 인식 청킹이 검색 품질의 핵심 결정 요인임을 보인다. 지식 배치가 단순한 성능 최적화를 넘어 공정성과 신뢰성에 직결됨이 확인되었다.

## 연구 트렌드와 미래 방향

네 축은 하나의 통합적 방향으로 수렴한다: **LLM 시스템의 신뢰성은 단일 모델 능력이 아니라, 다중 주체·다중 계층 간 상호 제약 구조의 설계 질에 의해 결정된다.** 아키텍처 수준 제어(Box Maze), 다중 에이전트 교차 검증(MARCH), 메타 평가(LLM-as-Judge 감사), 다중 프로바이더 분산(ReqFusion)—공통된 설계 원칙은 **단일 점 실패(single point of failure)의 구조적 배제**다. 미래 연구는 이 상호 제약 구조 자체의 안정성과, 방어-공격 공진화 속에서 지속적으로 견고함을 유지하는 적응적 메커니즘 설계에 집중할 것이다.
