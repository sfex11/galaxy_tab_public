## 핵심 테제

78편의 논문은 LLM이 단순한 텍스트 생성기를 넘어 **자율적 에이전트, 평가자, 공격자, 지각 계획자**로 기능 확장되는 과정에서 나타나는 **4가지 구조적 긴장**을 드러낸다: 안전성-능력, 확장성-신뢰성, 접근성-공정성, 내재화-외부주입.

## 6개 분석 축

### 1. 안전성의 이중성: 방어 도구가 공격 표면이 되는 역설
Box Maze가 추론 프로세스 무결성을 아키텍처 수준에서 강제하는 사전적 방어를 제안한 반면, Claudini는 autoresearch 파이프라인이 기존 30개 공격법을 능가하는 jailbreak 알고리즘을 자율 발견했다. 더욱 파격적인 것은 Steering Vectors의 안전성 감사 결과다—CAA 기반 스티어링은 alignment 도구로 설계되었으나, 일관되게 안전성 guardrail을 우회하는 공격 경로로 기능한다. 공격-방어가 동일 기술 기반에서 공진화하는 구조다.

### 2. 평가의 근본적 불안정성
LLM-as-a-Judge의 신뢰성 연구는 judge prompt 자체가 adversarial attack 대상이 됨을 보였다. 성별 추론 연구는 **맥락적 불변성 가정의 실패**를 실증하여, 탈맥락화된 벤치마크에서 측정된 편향이 실제 환경에서 질적으로 달라질 수 있음을 보였다. TRACE 프레임워크는 LLM과 인간이 코드 평가 루브릭에 체계적으로 다른 가중치를 부여함을 정량화했다. 소프트웨어 진화 관점의 테스트 생성 평가는 정적 스냅샷 성능이 실제 유용성을 보장하지 않음을 확인했다. **평가 시스템 자체가 평가의 대상**이 되어야 한다는 메타 평가의 필요성이 교차 검증된다.

### 3. 에이전트 패러다임의 영역 확장
AVO는 LLM 에이전트를 진화 알고리즘의 변이 연산자로 재해석하여 고정 휴리스틱을 자율적 코드 생성·수정 루프로 대체했다. LensWalk는 비디오 이해에서 '어떻게 볼 것인가'를 추론기가 동적으로 계획하는 에이전틱 지각을 제시했다. ReqFusion은 요구사항 공학이라는 상류 단계를 다중 LLM 프로바이더 앙상블로 자동화했다. MARCH는 환각 탐지에서 단일 검증자의 확인 편향을 다중 에이전트 강화학습 교차 검증으로 극복한다. 공통 패턴은 **단일 모델 한계를 다중 에이전트 구조로 해소**하는 방향성이다.

### 4. 지식 배치 전략의 트레이드오프
양자 코드 생성 연구는 빠르게 진화하는 도메인에서 외부 컨텍스트 주입(RAG)이 가중치 내재화(fine-tuning)보다 유지보수성에서 우위임을 보였다. SPA는 정교한 합성 데이터 파이프라인 없이 소수 프롬프트만으로 강력한 지식 주입 베이스라인을 달성했다. RAG 청킹 연구는 구조 인식 청킹이 도메인 특화 문서에서 단순 분할을 압도함을 실증하여, 지식 주입의 효과가 **컨텍스트 전달 방식에 의존적**임을 확인했다.

### 5. 접근성-공정성의 설계 딜레마
음성 인터페이스가 문해력 낮은 사용자에게 접근성을 확대하면서도, 음성에 포함된 정체성 단서(인종, 성별, 억양)가 차별을 증폭시킨다는 발견은 AI 설계에서 두 가치가 상충할 수 있음을 제기한다. 성별 편향의 맥락 민감성과 결합하면, **공정성 보장이 모달리티와 맥락에 종속적**임이 명확해진다.

### 6. 추론 신뢰성의 다층적 보장
Box Maze의 프로세스 제어(사전), MARCH의 다중 에이전트 검증(사후), Steering Vectors 감사(활성화 수준)는 각기 다른 추상화 계층에서 추론 무결성을 보장하려는 시도다. 단일 계층 방어의 한계를 암시한다.

## 미래 방향

1. **다층 방어 아키텍처**: 프로세스 제어 + 다중 에이전트 검증 + 활성화 수준 감사의 통합
2. **메타 평가 정식화**: 벤치마크 자체의 견고성을 측정하는 benchmark-of-benchmarks 체계
3. **모달리티 인지 공정성**: 텍스트 중심 편향 연구를 음성·비디오 등 다중 모달리티로 확장
4. **적응적 지식 배치**: 도메인 진화 속도에 따라 내재화와 외부주입을 동적으로 전환하는 하이브리드 전략
