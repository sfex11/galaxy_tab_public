# Transformer: 합성 분석

**생성일**: 2026-05-18
**관련 논문**: 34편
**최종 업데이트**: 2026-05-18

## 공통 주제: 고치지 않고 뛰어넘고, 경계 없이 확장하다

34편의 논문은 Transformer가 2026년 초 두 가지 명확한 진화 축을 따라 움직이고 있음을 보여준다. 첫째, **아키텍처 수정 없이 추론 효율의 극단적 추구**, 둘째, **비전·언어를 넘어 3D 공간과 물리 법칙으로의 모달리티 무한 확장**. 두 축은 Transformer의 본질적 한계—시퀀스 길이에 대한 이차적 복잡도와 토큰 단위 순차 처리—를 공통의 적으로 삼되, 해법의 방향은 정반대다. 전자는 Transformer **내부**에서 압축하고, 후자는 Transformer **외부**로 입력을 확장한다.

## 토큰 수준 효율화: 프루닝, 프로빙, 계층화의 삼각편대

세 가지 독특한 패턴이 드러난다. **통합 시공간 토큰 스코어링**은 ViT와 LLM의 경계를 허물어 단일 프루닝 파이프라인으로 비디오 VLM의 시간적 중복을 제거한다. **임베딩 공간 프로빙**은 더 파격적—별도 드래프트 모델 없이 mask token으로 speculative decoding을 구현하며, next-token prediction만으로 학습된 Transformer가 내재적으로 다중 토큰 예측 능력을 보유한다는 발견이다. **VideoAtlas**는 토큰 수를 줄이는 대신 비디오를 계층적 그리드로 재구성하여 로그 복잡도 탐색을 달성한다. 세 접근 모두 "Transformer를 고치지 않고 뛰어넘는다"는 철학을 공유하며, 산업적 배포에 가장 현실적인 경로다.

## 모달리티 확장: 3D 기하학과 물리 법칙의 직접 모델링

Transformer는 이제 물리 공간의 구조를 직접 토큰 시퀀스로 변환한다. **GMT**는 3D 장면에서 6-DOF 궤적을 합성하고, **Loc3R-VLM**은 단안 비디오만으로 VLM에 3D 공간 추론을 부여하며, **LoST**는 3D 형상을 의미론적 계층으로 토큰화한다. **Egomotion-Aware** 연구는 IMU 데이터를 MLLM에 통합하여 무거운 3D 재구성 없이 공간 추론을 달성한다.

LoST의 시사점이 특히 중요하다—토큰화 전략이 단순한 압축이 아니라 모델이 학습할 **구조적 사전 지식**을 결정한다는 점은, NLP에서 BPE가 언어 모델 성능을 좌우했던 역사를 3D 도메인에서 그대로 재현함을 보여준다. **RHYME-XT**와 **텍스트 임베딩 보간** 연구는 각각 PDE 기반 물리 시스템과 시각적 속성 제어에서 Transformer 표현 공간이 얼마나 구조화되어 있는지를 확인하는 상보적 사례다.

## 절대적 지표로의 전환: 신뢰성과 하드웨어 효율

**Box Maze**는 추론 신뢰성을 사후 교정(RLHF)이 아닌 프로세스 아키텍처 수준에서 강제하는 패러다임 전환을 제시한다. **SOL-ExecBench**는 GPU 커널 최적화를 상대적 speedup이 아닌 하드웨어 이론 한계(Speed-of-Light) 대비 절대적 효율로 평가한다. 두 논문은 Transformer의 소프트웨어 신뢰성과 하드웨어 효율성에 대해 "상대적 지표"를 "절대적 지표"로 대체하려는 **인식론적 전환**을 공유한다.

## 일치·모순·보완 관계

임베딩 공간 프로빙이 Transformer의 내재적 능력을 긍정적으로 해석하는 반면, Box Maze는 Transformer의 구조적 한계(환각, 추론 불안정성)를 명시적 제어로 보완해야 한다고 본다. "모델을 믿는다" vs "모델을 제어한다"는 긴장이지만, 실제로는 상호보완적이다—내재적 능력을 최대한 활용하되 임계점에서 아키텍처적 가드레일이 개입하는 하이브리드 설계가 실용적이다.

토큰 프루닝(통합 시공간 스코어링)과 계층적 탐색(VideoAtlas) 역시 상호보완적이다. 전자는 Transformer **내부**에서 토큰을 줄이고, 후자는 Transformer에 들어가기 **전**에 입력을 구조화한다. 두 경로가 만나는 지점이 "입력을 얼마나 압축할 것인가"라는 근본적 설계 질문이며, 도메인별 최적 조합을 찾는 것이 다음 과제다.

## 연구 트렌드와 미래 방향

세 가지 방향이 윤곽을 드러낸다. 첫째, **모달리티 특화 토큰화의 체계화**—LoST가 3D에서 보여준 의미론적 토큰화 원칙을 비디오, 오디오, 물리 시뮬레이션 등으로 일반화하는 이론적 프레임워크가 필요하다. 둘째, **효율화 전략의 자동 선택**—프루닝, 프로빙, 계층적 탐색 중 입력 특성에 따라 최적 전략을 자동 선택하는 메타-컨트롤러의 등장. 셋째, **절대적 평가 지표의 확산**—SOL-ExecBench와 Box Maze가 제시한 절대적 기준이 추론 효율, 신뢰성, 멀티모달 이해 등 다른 차원으로 확장되며, Transformer 생태계 전반에 "하드웨어 한계 대비 효율"이라는 새로운 평가 패러다임을 정착시킬 것이다.
