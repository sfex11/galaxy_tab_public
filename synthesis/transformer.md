# Transformer: 합성 분석

**생성일**: 2026-05-11

## 공통 주제: 효율성의 극대화와 모달리티의 무한 확장

포함된 논문들은 Transformer가 2026년 초 두 가지 명확한 방향으로 진화하고 있음을 보여준다. 첫째, **추론 효율화의 극단적 추구**이며, 둘째, **모달리티 경계의 해체**다. 두 축은 상호 독립적으로 전개되면서도 Transformer의 본질적 한계—시퀀스 길이에 대한 이차적 복잡도와 토큰 단위 순차 처리—를 공통의 적으로 삼는다.

## 토큰 수준의 효율화: 프루닝에서 프로빙까지

세 가지 독특한 패턴이 emerges한다. **통합 시공간 토큰 스코어링**은 ViT와 LLM의 경계를 허물어 단일 프루닝 파이프라인으로 비디오 VLM의 중복을 제거한다. **임베딩 공간 프로빙**은 더 파격적—별도 드래프트 모델 없이 mask token으로 speculative decoding을 구현하며, Transformer가 next-token prediction만으로도 내재적으로 다중 토큰 예측 능력을 보유한다는 발견이다. **VideoAtlas**는 토큰 수를 줄이는 대신 비디오를 계층적 그리드로 재구성하여 로그 복잡도 탐색을 달성한다. 세 접근 모두 "Transformer를 고치지 않고 뛰어넘는다"는 철학을 공유하며, 산업적 배포에 가장 현실적인 경로다.

## 모달리티 확장: 3D 공간과 물리 세계로의 진출

Transformer는 비전과 언어를 넘어 **물리 공간의 기하학적 구조**를 직접 모델링한다. **GMT**는 3D 장면에서 6-DOF 궤적을 합성하고, **Loc3R-VLM**은 단안 비디오만으로 VLM에 3D 공간 추론을 부여하며, **LoST**는 3D 형상을 의미론적 계층으로 토큰화한다. **Egomotion-Aware** 연구는 IMU 데이터를 MLLM에 통합하여 무거운 3D 재구성 없이 공간 추론을 달성한다. LoST의 시사점이 특히 중요한데—토큰화 전략이 단순한 압축이 아니라 모델이 학습할 구조적 사전 지식을 결정한다는 점은, NLP에서 BPE가 언어 모델 성능을 좌우했던 역사를 3D 도메인에서 재현함을 보여준다. **RHYME-XT**와 **텍스트 임베딩 보간** 연구는 각각 PDE 기반 물리 시스템과 시각적 속성 제어에서 Transformer 표현 공간의 구조화 정도를 확인하는 상보적 사례다.

## 신뢰성과 하드웨어 인식: 절대적 지표로의 전환

**Box Maze**는 추론 신뢰성을 사후 교정(RLHF)이 아닌 프로세스 아키텍처 수준에서 강제하는 패러다임 전환을 제시한다. **SOL-ExecBench**는 GPU 커널 최적화를 상대적 speedup이 아닌 하드웨어 이론 한계(Speed-of-Light) 대비 절대적 효율로 평가한다. 두 논문은 각각 Transformer의 소프트웨어적 신뢰성과 하드웨어적 효율성에 대해 "상대적 지표"를 "절대적 지표"로 대체하려는 공통된 인식론적 전환을 보여준다.

## 일치·모순·보완 관계

임베딩 공간 프로빙이 Transformer의 내재적 능력을 긍정적으로 해석하는 반면, Box Maze는 Transformer의 구조적 한계(환각, 추론 불안정성)를 명시적 제어 메커니즘으로 보완해야 한다고 본다. "모델을 믿는다" vs "모델을 제어한다"는 긴장이지만, 실제로는 상호보완적이다—내재적 능력을 최대한 활용하되 임계점에서 아키텍처적 가드레일이 개입하는 하이브리드 설계가 실용적이다.

또한 토큰 프루닝(Unified Spatio-Temporal Token Scoring)과 계층적 탐색(VideoAtlas)은 상호 보완적이다. 전자는 Transformer 내부에서 토큰을 줄이고, 후자는 Transformer에 들어가기 전에 입력을 구조화한다. 두 경로가 만나는 지점이 바로 "입력을 얼마나 압축할 것인가" vs "모델을 얼마나 가볍게 만들 것인가"의 균형점이며, 이는 도메인별 연산 예산에 따라 다르게 최적화될 것이다.

## 연구 트렌드와 미래 방향

세 가지 방향이 윤곽을 드러낸다. 첫째, **아키텍처 수정 없는 가속**이 주류로 자리잡고 있다. Training-free, task-agnostic, 설계 변경 없이 기존 모델의 잠재력을 끌어내는 접근이 압도적으로 많다. 둘째, **토큰화의 재발견**—LoST가 3D에서, 임베딩 프로빙이 언어에서 보여주듯, 토큰이란 단순한 입력 단위가 아니라 모델 능력의 스위치다. 셋째, **평가의 절대화**—SOL-ExecBench의 Speed-of-Light, Box Maze의 프로세스 무결성 기준처럼, 상대적 비교에서 하드웨어·아키텍처 이론 한계 대비 절대적 성능 측정으로의 전환이 가속화된다. Transformer는 더 이상 "어떤 구조인가"가 아니라 "어떤 표현 공간을 얼마나 효율적으로 탐색하는가"로 재정의되고 있다.
