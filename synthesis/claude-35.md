## 개요

Claude 3.5 계열 모델(Claude 3.7 Sonnet, Claude 3 Opus, Claude Code 등)은 다국어 사회언어학적 상호작용, 법률 텍스트 추론, 과학 소프트웨어 개발 에이전트 등 광범위한 도메인에서 핵심 평가 대상으로 활용되고 있다. 본 분석은 관련 논문 4종을 바탕으로 Claude 모델 계열의 다각적인 역할과 현실적 한계를 종합적으로 고찰한다.

## 핵심 발견 및 논문 간 관계

### 1. 사회언어학적 컨텍스트 민감성

다국어 정중함 연구에 따르면, Claude 3.7 Sonnet은 영어·힌디어·스페인어 환경에서 사용자의 정중함 및 무례함 정도에 따라 응답 품질과 태도가 변동한다. '보편적 예의'란 존재하지 않으며, 상호작용 이력(raw, polite, impolite)과 언어권에 따라 반응이 달라진다. 이는 다국어 고객 지원 시스템 구축 시 프롬프트 엔지니어링 기준을 문화권별로 차별화해야 함을 시사한다.

### 2. 법률 도메인 적용의 양면성: 평가 깊이에 따른 격차

베트남 법률 텍스트 연구에서는 Claude 3 Opus가 이중 평가 프레임워크 하에서 법률 문장 단순화 및 추론 벤치마크에서 최상위권 성능을 기록했다. 반면, 경쟁법 전문 에이전트(Maat) 연구에서는 일반 Claude 모델이 **전문 도메인 지식 부족**과 **공식 인용 누락**이라는 명백한 한계를 보였다. 이는 모순이 아닌 **평가 깊이의 차이**에서 비롯된다. Claude는 표면적 법률 요약에는 강하지만, 실제 전문가 수준의 판례 인용이 요구되는 태스크에서는 외부 지식 베이스(RAG)와의 결합 없이는 한계가 뚜렷하다.

### 3. 과학 코딩 에이전트로서의 자율성과 한계

"Physics Is All You Need" 연구는 Claude Code(Sonnet/Opus)를 코딩 에이전트로 활용해 물리학 소프트웨어(CLAX-PT)를 12일간 57세션에 걸쳐 개발하는 과정을 정량화했다. 15개 감독 이벤트 중 10개를 오라클 테스트 기반 자율 반복으로 해결했으나, 도메인 특화 지식이 필요한 3개 문제는 인간 개입으로도 해결되지 않았다.

## 세 도메인의 공통 구조: 자율성의 한계선

사회언어학·법률·과학 코딩 세 도메인은 Claude의 능력 스펙트럼을 다르게 조명하지만, 하나의 일관된 결론으로 수렴한다. Claude는 **정형화된 패턴 인식과 일반 추론**에서 뛰어나나, 도메인 깊이가 특정 임계치를 넘어가면 반드시 외부 보완 메커니즘(문화권별 프롬프트, RAG, 인간 감독)이 필요하다. 법률 연구에서의 인용 누락과 과학 코딩에서의 교착 상태는 동일한 구조적 한계의 다른 얼굴이다.

## 연구 트렌드 및 미래 방향

제시된 논문들은 Claude를 '단독 완결형 솔루션'이 아닌 **고도화된 에이전트 시스템의 구성 요소**로 바라보는 추세를 공통적으로 보여준다. Maat 연구가 전문 지식 베이스를 결합한 도메인 특화 에이전트를 제안하고, 물리학 연구가 Human-in-the-loop 감독 모델을 정량화한 것은 같은 방향의 다른 접근이다. 향후 Claude 계열의 실용적 가치는 모델 자체의 성능 향상보다, **적절한 외부 보완 체계와의 통합 설계**에서 결정될 것이다.
