배경과 문제 정의

독일 연방은행의 역할

유로시스템의 핵심 중앙은행으로서 통화정책 실행 및 금융시스템에 유동성 공급. 모든 신용거래는 담보로 보호되어야 하며, 담보 수용 여부는 적격성(Eligibility)에 따라 결정됨.

핵심 문제

  • 수백 페이지의 초록(Prospectus) 수동 검토
  • 연간 수천 건의 증권 발행 처리
  • 반구조화, 이중어(독·영) 문서
  • 극도로 자원 집약적인 과정
적격성 판단은 법적·금융적 기준에 대한 증권 검증이며, 자동화의 최우선 대상임.

복합 기준

기존 접근법의 한계

NER 기반 시스템 (Hänig et al., 2023)

  • Transformer 기반 토큰 분류 모델 사용
  • 대부분의 기준에서 양호한 성능 달성
  • 결정 지원 시스템으로 구현

주요 제약사항

  • 모든 주석 유형에 대한 대규모 수동 주석 필요
  • 엄격한 텍스트 스팬 경계에 민감
  • OCR 노이즈에 취약
  • 훈련 데이터 외 금융 어휘에 불안정

패러다임 전환: 토큰 분류 → 생성적 정보 추출(Generative IE)

제안 방법: LLM 기반 파이프라인

2

3
정규화 (Normalization)

4
해석 (Interpretation)

5
최종 판정

문서 수준 평가

  • 정밀도(Precision): 위양성 최소화 — 중앙은행에 핵심
  • 재현율(Recall): 위음성 최소화

핵심 발견

  • 단순 기준(통화, 유형, 원금, 상환액)에서 높은 정밀도
  • 복합 기준(쿠폰, 우선순위)은 다단계 처리로 개선 여지
  • LLM-as-a-Judge가 퍼지 매칭보다 의미적 평가에 우수
  • 보수적 편향: 부적격을 적격으로 오판하는 경우가 적음
  • 중앙은행 운영 관점에서 바람직한 특성
  • 재현율은 정밀도보다 낮아 개선 필요

논의: 다차원 분석

안전 편향 (Safety Bias)

LLM은 불확실할 때 "모르겠음"으로 응답하는 경향이 있어, 위양성(부적격을 적격으로 판단)보다 위음성(적격을 부적격으로 판단)이 많아짐. 중앙은행 맥락에서는 수용 가능한 편향.

PDF 추출 아티팩트

Docling 변환 과정에서 발생하는 레이아웃 오류, 이중어 인터리빙 왜곡 등이 추출 품질에 직접적 영향. 이중어 처리는 여전히 도전 과제.

LLM 편향 최소화

프롬프트 설계를 통해 LLM의 사전 편향을 줄이고, 정규화 단계에서 규칙 기반 보정을 결합하여 편향 완화 시도.

효율성과 적응성

수동 주석이 불필요한 zero-shot 접근으로 새로운 기준 추가 시 유연하게 대응 가능. 단, 추론 비용은 여전히 높음.

한계점 및 향후 과제

고급 PDF 파싱 및 비전 모델

텍스트 추출 한계를 극복하기 위해 멀티모달 비전-언어 모델(VLM)을 PDF 레이아웃 이해에 통합

의미적 정합 및 RAG 통합

추출값을 규제 문서와 의미적으로 연결(RAG)하여 해석 단계의 신뢰성 향상

LLM-as-a-Judge 메타평가

자동 평가자 자체의 일관성과 신뢰도에 대한 체계적 메타평가 필요

추가 고려사항

· 재현율 향상을 위한 프롬프트 엔지니어링 심화

· 이중어 문서의 언어별 정보 통합 전략

· 대규모 배포 시 추론 비용 최적화

· 실제 업무 환경에서의 인간-루프(Human-in-the-loop) 통합

결론

주요 기여

  • 중앙은행 적격성 검증에 LLM을 적용한 최초 사례 연구
  • 추출-정규화-해석의 다단계 생성적 IE 파이프라인 제안
  • 위치 기반 메트릭을 대체하는 LLM-as-a-Judge 평가 방법론 도입
  • Zero-shot 범용 모델의 금융 도메인 적용 가능성 입증

핵심 인사이트

LLM 기반 시스템은 최대 91%의 문서 수준 정밀도를 달성하며, 보수적 운영 프로필을 통해 위양성 수용을 최소화함.

이는 수동 주석이 불필요하고 새로운 기준에 유연하게 대응할 수 있어, NER 기반 접근법의 근본적 한계를 극복하는 방향을 제시함.

토큰 분류에서 생성적 IE로의 패러다임 전환은 금융 규제 문서 자동화에 새로운 가능성을 열어줌.
