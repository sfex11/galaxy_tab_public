기존 도구 에이전트의 동작 방식

Step-wise Atomic Tool Calls

현재 도구 강화 LLM 에이전트는 각 도구 호출을 원자적 단위로 처리합니다. 호출 → 관측 → 값 전달이 모두 메인 추론 트레이스에 노출됩니다.

  • MCP 스타일 인터페이스 기반
  • 모든 관측값이 트레이스에 기록
  • 단순하고 범용적인 설계

예시: 두 지점 간 거리 계산

1
지점 A 지오코딩 → 좌표 반환
2
지점 B 지오코딩 → 좌표 반환
3
두 좌표 간 거리 계산 → 결과 반환

3단계가 모두 모델에 노출되지만, 실제로 필요한 것은 최종 거리값뿐입니다.

핵심 문제: 실행 입자도 불일치

Execution-Granularity Mismatch

컨텍스트 팽창

긴 도구 관측값이 추론 트레이스에 누적됩니다. 실제로는 몇 개의 필드만 필요하지만, 전체 출력이 컨텍스트를 차지합니다.

추론 파편화

모델이 저수준의 결정적 연산을 위해 반복적으로 메인 결정 루프로 돌아가야 합니다. 고수준 추론과 절차적 상태 전달이 혼재됩니다.

"로컬에서 결정적인 도구 워크플로우가 반복적인 모델 가시적 결정으로 전개되어, 컨텍스트를 소비하고 모델이 저수준 데이터플로우를 관리하도록 강제합니다."

기존 접근법의 한계

접근법 방식 한계
트레이스 자르기/압축 실행 후 트레이스 길이 축소 이후 단계에 필요한 중간값이 손실될 수 있음
캐싱/요약 작업별 상태를_compact하게 유지 워크플로우 특정 가정에 의존, 이기종 도구에 일반화 어려움
서브에이전트 위임 저수준 실행을 메인 컨텍스트에서 숨김 투명성 저하, 중간 제어/복구 어려움
프로그램 기반 에이전트 코드를 일반 액션 언어로 사용 기존 도구 호출을 그루핑하는 래퍼가 아님
워크플로우 컴파일 실행 전 의존성 그래프 구성 워크플로우를 사전에 명시해야 함

공통 문제: 실행 이후에 트레이스를 관리할 뿐, 도구 사용의 실행 입자도 자체는 변경하지 않습니다.

HyperTool: 핵심 아이디어

실행 시점 컨텍스트 관리: 사후 압축이 아닌, 실행 시점 제어

통합 실행 가능 인터페이스

HyperTool은 코드 블록을 입력으로 받는 MCP 호환 도구입니다. 블록 내부에서 기존 도구를 원래 스키마로 호출하고, 중간 출력을 저장·변환·필터링할 수 있습니다.

결정적 서브루틴 접기

로컬에서 결정적인 도구 체인을 하나의 외부 호출로 접습니다. 일시적 관측값과 상태 전달은 메인 트레이스에 노출되지 않습니다.

  • 원자적 호출과 다중 도구 서브루틴을 동일한 코드 블록 형식으로 표현
  • 코드로 도구를 조합하고, 중간값을 변환하며, 임시 헬퍼 함수 정의 가능
  • 에이전트는 간결한 고수준 트레이스를 유지하면서 작업 특화 부분 도구 구성

실행 시점 제어

HyperTool 인터페이스 구조

블록 내부 구성 요소

  • Tool Invocation: 기존 도구를 원래 스키마로 호출
  • Value Assignment: 반환값을 변수에 저장
  • Computation: 중간값에 대한 로컬 연산
  • Control Flow: 조건문, 반복문 등 제어 구조
  • Helper Functions: 임시 보조 함수 정의
  • Return: 메인 트레이스로 반환할 결과 지정

핵심 원칙: 블록 내부의 모든 도구 호출, 연산, 상태 전달은 모델에 보이지 않습니다. 오직 블록의 최종 반환값만 메인 추론 트레이스에 기록됩니다.

조합형 작업 구성

여러 도구를 조합해야 해결 가능한 작업을 자동으로 생성하여, 단일 도구 호출으로는 풀 수 없는 시나리오를 확보합니다.

로컬 수리 + 컨텍스트 압축

롤아웃 중 코드 오류 발생 시 로컬 수리를 시도하고, 트레이젝토리 수준에서 컨텍스트를 압축하여 품질을 높입니다.

트레이젝토리 검증

단순한 수집이 아닌, 두 가지 엄격한 기준을 통과한 트레이젝토리만 사용

실행 정확성 검증

HyperTool 블록이 실제 MCP 환경에서 오류 없이 실행되는지 확인합니다. 코드 문법 오류, 도구 호출 실패, 런타임 예외 등을 검사합니다.

증거 일관성 검증

최종 답변이 블록의 실행 결과(증거)와 논리적으로 일치하는지 검사합니다. 실행은 성공했으나 결론이 잘못된 트레이젝토리를 제거합니다.

이중 검증을 통해 고립된 원자적 호출에서 실행 가능한 도구 서브루틴으로 감독의 패러다임을 전환합니다.

토켄 효율성 분석

메인 트레이스 단축

HyperTool은 중간 관측값을 블록 내부에 유지하여 메인 추론 트레이스의 토큰 수를 상당히 감소시킵니다. 여러 도구 호출의 결과가 개별적으로 트레이스에 기록되지 않습니다.

정확도-효율성 트레이드오프

트레이스 길이 감소에도 불구하고 정확도는 크게 향상됩니다. 이는 컨텍스트 팽창이 단순히 토큰 낭비가 아니라 추론 품질 저하의 원인이었음을 시사합니다.

컨텍스트 절감 + 정확도 향상 = 양립 가능

사후 압축과 달리, 실행 시점 제어는 정보 손실 없이 컨텍스트를 관리합니다.

요약 및 기여

1

문제 식별

과도하게 세분화된 도구 실행이 컨텍스트 팽창과 추론 파편화를 유발함을 밝힘

2

HyperTool 제안

MCP 도구 스키마를 보존하면서, 원자적 호출과 다중 도구 워크플로우를 하나의 코드 블록으로 표현하는 통합 실행 인터페이스

3

검증된 학습 데이터 구축

실제 MCP 환경에서 수집하고 이중 검증(실행 정확성 + 증거 일관성)을 거친 HyperTool 형식 트레이젝토리

4
