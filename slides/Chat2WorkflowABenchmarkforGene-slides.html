01
Problem Statement

왜 이 문제가 중요한가?

실행 가능한 시각적 워크플로우는 산업계에서 주류 패러다임으로 자리 잡았으나, 현재 실무에서는 거의 전적으로 수동 공학에 의존합니다.

개발 비용과 시간

개발자가 워크플로우를 직접 설계하고, 각 단계별 프롬프트를 작성하며, 요구사항 변화에 따라 반복적으로 수정해야 합니다.

오류 발생 가능성

복잡한 제어 흐름과 도구 사용을 자연어에서 정확히 추론하는 것은 매우 어려우며, 수동 구성 시 실수가 빈번합니다.

변화하는 요구사항

사용자 요구사항은 지속적으로 변하며, 기존 워크플로우의 정확성과 일관성을 유지하면서 수정하는 것이 핵심 과제입니다.

04
Task Formulation

과제 정식화

워크플로우 생성을 멀티턴 상호작용 과정으로 정식화합니다. LLM은 Chain-of-Thought 접근법을 통해 구조화된 JSON을 출력합니다.

G(V, E) ← Mθ(q, H)

G: 방향성 비순환 그래프(DAG)  ·  V: 노드 집합  ·  E: 엣지 집합  ·  q: 현재 지시어  ·  H: 대화 기록

<node_selection>

선택된 노드 목록을 추론하여 출력

<design_principle>

설계 원칙과 계획 근거를 서술

<workflow>

nodes_info와 edges 필드를 포함한 JSON 구조

05
Evaluation

2단계 평가 프로토콜

워크플로우의 품질은 표현 형태만으로 판단할 수 없습니다. 실제 실행 결과까지 검증하는 두 단계 평가 방식을 도입했습니다.

Pass Rate
N_success / N_total
CoT 일관성, JSON 파싱, YAML 변환, 변수 일치성, 논리적 유효성 검증을 통과하는 비율
Resolve Rate
N_resolved / N_total
플랫폼에 임포트되어 실제로 실행되고, 지시어의 모든 요구사항을 충족하는 결과를 생성하는 비율

08
Key Findings

핵심 발견

의도 파악 vs 실행 가능성

최신 모델들은 사용자의 고수준 의도를 파악하는 데는 능숙하지만, 이를 정확하고 안정적으로 실행 가능한 워크플로우로 변환하는 데 큰 격차가 존재합니다.

복잡도 증가 시 급격한 성능 저하

태스크가 복잡해지거나 요구사항이 변화할수록 Pass Rate와 Resolve Rate 모두 현저히 감소하여 구조적 추론의 한계를 보여줍니다.

에이전트 루프의 한계

오류 기반 자가 수정 에이전트가 최대 5.34% 향상을 달성했으나, 여전히 실무 수준의 자동화와는 큰 간극이 남아 있습니다.

멀티턴 요구사항 변화의 어려움

사용자가 기존 워크플로우를 수정하거나 추가 기능을 요청할 때, 이전 라운드의 정확성과 일관성을 유지하면서 적응하는 것이 특히 어렵습니다.

09
Conclusion

결론 및 향후 방향

Chat2Workflow는 자연어로부터 실행 가능한 워크플로우를 자동 생성하는 문제를 체계적으로 연구하기 위한 기반을 제공합니다.

기여

실제 프로덕션 환경 기반의 최초 워크플로우 생성 벤치마크를 제안하고, 2단계 평가 프로토콜과 오류 기반 에이전트 프레임워크를 제시했습니다.

현실적 격차

최신 LLM의 성능에도 불구하고 산업급 자동화를 달성하기 위한 구조적 추론과 적응적 워크플로우 합성 능력은 여전히 핵심 병목입니다.

향후 연구

구조적 추론 강화, 적응적 워크플로우 합성, 더 강력한 자가 수정 메커니즘, 그리고 확장된 벤치마크 구축이 필요합니다.
