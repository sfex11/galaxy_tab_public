문제 정의

왜 DRL 안전성 검증이 필요한가?

심층 강화학습은 복잡한 제어 문제에서 뛰어난 성능을 보이지만, 안전한 배포를 위해서는 행동 검증이 필수적입니다.

▶
적응형 비디오 스트리밍
Pensieve 등의 에이전트가 네트워크 상태에 따라 비트레이트를 실시간 조정
📡
무선 자원 관리
CMARS 에이전트가 무선 슬라이스에 자원 블록을 동적 할당
📈
혼잡 제어
Aurora 에이전트가 패킷 손실과 RTT 기반으로 혼잡 윈도우 조정
⚠ 기존 접근법의 한계: 포인트 속성 (Point Properties)
기존 검증 방법은 고정된 단일 입력 상태에서만 속성을 정의 → 입력 공간의 극히 일부만 커버 → 안전하지 않은 영역을 놓칠 위험
2
핵심 개념

포인트 속성 vs 기호적 속성

기호적 속성은 입력 범위 전체에 걸친 기대 행동을 정의합니다.

✕ 포인트 속성
  • 고정 입력에서만 분석
  • 커버리지 극히 제한적
  • 수동으로 포인트 선택 필요
  • 안전하지 않은 영역 누락 가능
단일 점 → 낮은 커버리지
✓ 기호적 속성
  • 입력 범위 전체에 걸친 분석
  • 60~100% 입력 공간 커버
  • 자동화된 분해 알고리즘
  • 비자명한 반례 발견 가능
연속 영역 → 넓은 커버리지
3
수학적 정의

기호적 속성의 정형화

통합된 수학적 프레임워크로 다양한 속성 유형을 표현합니다.

∀x ∈ X, lsi ≤ si ≤ usi : f(π(x), π(x+s)) ≤ d
🛡 기호적 강건성
// 작은 입력 변동 → 출력 안정
lsi = -ε, usi = ε
f(y,z) = |y - z| ≤ d
"작은 입력 섭동이 불균형적으로 큰 출력 변화를 야기하면 안 됨"
📈 기호적 단조성
// 특정 입력 증가 → 예측 가능 방향
lsi = 0, usi = εi
f(y,z) = z - y ≤ d
"손실률이 증가하면 혼잡 윈도우를 감소시켜야 함"
π 결정적 정책   X 입력 상태 공간   s 제한된 섭동 벡터   f 비교 함수   d 허용 임계값

1단계: 비교 인코딩
동일 DNN의 두 결합된 실행을 생성
• 기본 시나리오: x1 ∈ [lx, ux]

5
실현 가능성

네트워킹 도메인의 검증 용이성

시스템/네트워킹 DRL이 기호적 분석에 적합한 3가지 특성

🧠
1. 소형 신경망
컴퓨터 비전(50+ 레이어)과 달리, 구조화된 고수준 입력(지연, 처리량, 버퍼)을 처리 → 2~3 은닉층, 32~128 뉴런
🎯
2. 소규모 행동 공간
차원의 저주 회피를 위해 행동 수를 제한 → Pensieve: 6개, CMARS: 15~30개, Aurora: 연속(분포 매개변수)
🔒
3. 결정적 배포
학습 시 확률적이지만 배포 시 결정적 → Q-학습: 본질적 결정적, 정책 경사법: 평균값만 사용 → argmax 정형화 가능

사례 연구 2

CMARS: 무선 자원 할당

입력 19차원 / 행동 15~30개 자원 블록 / 2~3 레이어

⚖
경쟁 인식 할당
타 슬라이스 수요 증가 → 대상 할당 증가 금지
📡
채널 보상
열악한 채널 → SLA 유지를 위해 할당 증가
🛡
CMARS 강건성
측정 노이즈 → 불균형 할당 변화 금지
⚠ 주요 반례 발견: 강건성 위반
ε = 0.001의 미세한 섭동에서:
• 기본 할당: 3개 자원 블록 → 섭동 후: 29개 자원 블록 (+86.7%)
• 다른 사례: 9개 → 1개 (-88.9% 감소)
• 적대적 조작에 취약한 민감성 노출 — 포인트 분석으로는 발견 불가
네트워크 깊이 vs 안전성
π2,15 (2레이어) 위반 적음    vs    π3,15 (3레이어) 위반 더 많음
→ 깊은 네트워크 ≠ 더 안전한 네트워크

주요 발견

6가지 핵심 발견

1
넓은 커버리지
기호적 속성이 포인트 속성보다 실질적으로 넓은 커버리지를 제공
2
학습 진화 패턴
초기 높은 위반률(30-50%) → 수렴 후에도 일부 위반 지속
3
모델 크기 트레이드오프
소형 모델: 검증 용이성 45% 향상, 성능 동일
4
다중 엔진 시너지
단일 솔버: 40-60% 미해결 → 다중 엔진: 미결정 15-25% 감소
5
의미 있는 반례
ε=0.001 섭동으로 80%+ 할당 변화 — 운영적으로 중요한 코너 케이스
6
깊이 ≠ 안전
네트워크 깊이 증가가 안전성을 보장하지 않음 — 명시적 검증 필수
12
기여 및 향후 연구

학술적 기여 및 향후 방향

✓ 주요 기여

1. DRL 검증을 위한 통합 기호적 속성 정형화
2. 기호적 속성을 처리 가능한 하위 속성으로 분해하는 기법
3. 기존 DNN 검증 도구와 호환되는 실용적 활성화 기법
4. 3개 도메인에 걸친 최초의 체계적 실증 연구 (diffRL)

→ 향후 연구 방향

  • 반례 기반 재학습 (Counterexample-guided retraining)
  • 더 풍부한 시간적 속성 정의
  • 인증된 강건성 학습 기법
  • DRL 최적화 루프와의 통합
  • 확률적 배포 정책으로의 확장
결론: 시스템/네트워킹 DRL의 도메인 특성(소형 네트워크, 소규모 행동 공간, 결정적 배포)을 활용하여
