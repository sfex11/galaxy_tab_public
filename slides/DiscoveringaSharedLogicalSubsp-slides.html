동기

LLM은 여전히 다단계 논리 추론에 취약하다

기존 접근법은 각각 한계가 있다. 자연어 뷰만 다루거나, 외부 기호求解器를 부착하는 방식으로는 근본적인 해결이 어렵다.

자연어 의존 방법
프롬프트 엔지니어링, CoT 세밀화 등. 표면 형태에 강하게 의존하여 논리 구조를 놓친다.
신경-기호 방법
외부 정리기/검증기를 부착. 추가 모듈 필요, 모델 내부 논리를 직접 건드리지 못한다.
핵심 질문
LLM 내부에 자연어 뷰와 기호 뷰 모두에 정렬된 공유 논리 부분공간이 존재하는가? 이를 활용해 추론을 조향할 수 있는가?

방법

논리 부분공간 조향 (LSS)

NL + 기호 쌍 데이터
잔차 스트림 활성화 수집
PCA → CCA → 직교 기저
추론 시 토큰별 조향
핵심 원리
토큰 생성 시, 활성화 벡터를 학습된 부분공간으로 투영하여 증폭한다. 모델 가중치는 변경하지 않는 학습 불필요(training-free) 방식이다.
05 / 09
알고리즘

부분공간 학습 3단계

1
PCA: 노이즈 제거 & 압축
각 뷰의 잔차 활성화에 대해 주성분 분석을 수행하여 고차원 잔차 스트림을 저차원으로 압축하고 노이즈를 제거한다.
2
CCA: 교차 뷰 정렬
정준 상관 분석으로 NL 뷰와 기호 뷰 간의 상관관계를 최대화하는 저차원 부분공간을 학습한다.
3
역투영 & 직교 기저 구성
CCA 부분공간을 원래 잔차 차원으로 역투영하여 직교 기저 U(ℓ) ∈ ℝD×k를 얻는다.
E(ℓ)(r) = ‖r⊤ U(ℓ)‖22 / ‖r‖22 정규화된 투영 에너지: 활성화가 논리 부분공간에 얼마나 놓여 있는지 측정

심층 분석

논리 부분공간이 무엇을 인코딩하는가

의미 + 논리 구조 동시 인코딩 — 부분공간은 단순한 문법 정보가 아니라, 의미적 관계와 논리적 구조를 모두 포착한다.
상위 레이어에서 정렬 강화 — NL-기호 뷰 간 정렬은 하위 레이어보다 상위 레이어에서 현저히 강해진다.
투영 에너지 ∝ 정답률 — 부분공간 내 투영 에너지가 높은 추론 사슬일수록 정답 확률이 높다.
추론 동작 변화 — 조향 후 "since", "so" 등 논리 접속사 사용 증가, "think", "assume" 등 모호한 동사 감소.

결론

공유 논리 부분공간은 실재한다

LLM 내부에 자연어와 기호 언어 양쪽 뷰에 정렬된 저차원 논리 부분공간이 존재하며, 이를 조향하면 최대 11%p의 추론 정확도 향상을 달성할 수 있다.

발견
다중 뷰 논리 부분공간의 존재를 CCA로 최초 발견하고 특성화
조향
학습 불필요한 추론 시 조향으로 교차 뷰 신호 활용
일반화
도메인 외 벤치마크(LogiQA 2.0)에서도 강한 일반화 성능
