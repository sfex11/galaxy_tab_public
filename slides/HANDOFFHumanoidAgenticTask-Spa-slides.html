Humanoid Whole-Body Control Multi-Teacher Distillation

HANDOFF

증류된 상보적 전문가들을 통한
휴머노이드 에이전트적 작업공간 전신 제어

Lizhi Yang, Junheng Li, Nehar Poddar, Yiling Hou, Gio Huh, Robert Griffin, Georgia Gkioxari, Aaron D. Ames

Caltech · IHMC · arXiv:2606.06493v1

문제: 기존 전신 제어기의 한계

휴머노이드 로봇을 실세계에 배치하려면, 명령 공간(command space)의 선택이 결정적이다.

밀도 참조 기반 WBC

전신 관절 각도 스트림(29D)을 컨트롤러 주기마다 요구. 플래너가 이를 합성하기 어려움.

분리 아키텍처

이동은 속도 명령으로 간단하지만, 상체는 여전히 관절 참조(14D) 필요.

결과적으로 플래너는 데이터 재생 엔진으로 전락 → 새 기술마다 새로운 데모 수집 필요

플래너가 진정 원하는 것

인간은 정확한 관절 위치를 추론하지 않는다. 대신 희소한 하위 목표를 설정하고 하위 수준 반사가 나머지를 처리한다.

직관적 Intuitive

인간, 기하학적 플래너, VLM 모두 유효한 명령 생성 가능

범용 General

하나의 인터페이스로 다양한 이동-조작 태스크 서비스

모듈식 Modular

플래너, 인식, 제어기 독립적으로 교체 가능

전신 표현력 Expressive

컴팩트한 명령으로도 조화된 전신 동작 유도

세 가지 상보적 전문가 교사

단일 훈련 체제로 세 가지 요구사항(작업공간 추적, 전신 조작, 낙하 복구)을 모두 달성할 수 없음

전신 모션 추적

  • 공개 리타겟 모션 데이터셋 활용
  • 안전 필터링된 데이터로 훈련
  • 리치, 양손 조작, 스쿼트 담당

이동 전문가

  • 작업 기반 속도 추적 보상
  • 비영 속도 명령 시 하체 담당
  • 다양한 보행 속도/방향 커버

낙하 복구 전문가

  • 적대적 모션 사전(AMP)으로 훈련
  • 특수 상황에서 전신 감독
  • 외란에 대한 강건성 확보

다중 교사 증류 파이프라인

1
전문가 훈련
3개 교사를 각각의 체제에서 독립적으로 학습
2
문맥 조건 게이팅
런타임 체제 신호로 각 교사의 감독 라우팅
3
MoE 증류
PPO + 다중 교사 KL 증류로 단일 학생 통합
4
단일 정책 배포
10-D 인터페이스, 런타임 전환 없음

확장성

새 전문가(접촉/태스크 특화) 추가 시 → 새 교사 헤드 + 새 문맥 채널 하나만 추가. 기존 교사나 명령 인터페이스 변경 불필요.

Mixture-of-Experts 아키텍처

문맥 조건부 게이팅

  • 연속 문맥: 바디 슬라이스 KL (하체/상체 분리)
  • 이산 문맥: AMP KL (낙하 복구 활성화)
  • 차원별 KL: 각 차원 독립적 정규화

훈련 세부사항

  • PPO + DAgger 결합 사용
  • 계수 스케줄링으로 안정적 수렴
  • 도메인 랜덤화 적용

런타임 동작 분담

하체 → 비영 속도 명령 시 이동 교사 감독

상체 → 모션 추적 교사가 전 구간 감독 (리치, 양손 조작, 스쿼트)

전신 → 낙하 복구 전문가가 특수 상황에서 전체 감독 인계

→ 세 교사 모두 단일 정책으로 통합, 런타임 컨트롤러 전환 없음

기존 방법과의 비교

방법 이동 참조 높이 참조 팔 참조 밀집 스트리밍 없음 외부 운동학 참조 없음
Motion-tracking WBCs 운동학 (29D) 운동학에서 상체 전체 (14D) ✗ ✗
HOVER 다중모드 (~80D) 운동학에서 운동학에서 ✗ ✗
HOMIE / FALCON 속도 (2–6D) 스칼라 h (1D) 상체 전체 (14D) ✓ ✗
AMO 속도 (3D) 몸통 자세 (4D) 상체 전체 (14D) ✓ ✗
HANDOFF (Ours) 속도 (3D) 스칼라 h (1D) 손목 목표 (6D) ✓ ✓

HANDOFF만이 컴팩트하고 플래너 친화적인 인터페이스를 제공하면서도 외부 운동학 참조 없이 동작

속도 추적 성능

핵심 결과

  • Unitree G1 하드웨어에서 SOTA 속도 추적 성능 매치
  • 다양한 속도 범위에서 안정적 보행 달성
  • 제로 속도에서 정지 안정성 확보

조작 작업공간

  • 휴머노이드 클래스에서 가장 큰 강건 조작 작업공간 중 하나 제공
  • 양손 조작 및 스쿼트-리치 동시 수행 가능

제거 연구 요약

Ours+Stab. — 안정성 보상 스택 추가로 낙하 복구 성능 향상

Ours+Stab.+Rec. — 낙하 복구 교사까지 포함 시 최고 강건성

Differential-IK 베이스라인 — 손목 작업공간에서 HANDOFF가 압도적 우위

각 전문가의 기여가 명확히 분리되어 확인됨

핵심 특징

  • 태스크 특화 데이터 불필요
  • 컨트롤러 미세조정 불필요
  • 시뮬레이션 및 실제 환경 모두에서 롤아웃 성공

호환성

10-D 베이스-핸드 인터페이스는 VLA 모델, 언어 기반 태스크 플래너, 학습된 엔드이펙터 정책 모두에 합리적인 타겟

한계점

손목 위치 목표

손목 위치만 명령하므로, 손목 방향이 중요한 정밀 조작(예: 드라이버 사용)에는 부적합. 방향 제어가 필요한 태스크 확장 필요.

제한된 인식

현재 정책은 관절/IMU 관측만 사용. 시각 정보가 정책에 직접 주입되지 않아 VLM의 인식 의존도가 높음.

전문가 커버리지

세 전문가가 커버하지 않는 행동(예: 춤, 점프)은 현재 지원 불가. 새 전문가 추가로 확장 가능하지만 완전한 커버리지는 아님.

그러나 문맥 기반 증류 훅은 확장 가능한 설계로, 새 전문가를 플러그인 방식으로 추가할 수 있는 구조적 장점 존재.

결론

HANDOFF는 직관적, 범용, 모듈식, 전신 표현력이라는 네 가지 속성을 모두 만족하는 최초의 휴머노이드 전신 제어 프레임워크이다.

기여 요약

  • 10-D 플래너 친화적 명령 공간 제안
  • 문맥 조건부 다중 교사 KL 증류 기법
  • SOTA 속도 추적 + 최대 조작 작업공간
  • VLM 에이전트와의 태스크 특화 없는 통합

오픈소스

전체 프레임워크를 오픈소스로 공개하여 향후 휴머노이드 전신 제어 연구의 기반으로 제공

확장 가능한 설계: 새 전문가, 새 플래너, 새 하드웨어로의 이식이 구조적으로 용이
