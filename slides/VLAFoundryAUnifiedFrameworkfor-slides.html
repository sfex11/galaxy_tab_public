VLA Foundry
통합 훈련 프레임워크

Vision-Language-Action 모델을 위한 단일 코드베이스 기반의 오픈소스 프레임워크 — LLM, VLM, VLA 훈련을 하나로 통합

02
배경

왜 통합 프레임워크가 필요한가

최근 로봇 학습 분야에서는 대규모 비전-언어 모델(VLM)을 로봇 제어에 적용하는 Vision-Language-Action(VLA) 모델이 주목받고 있습니다.

LLM 기반 확장

대규모 언어 모델의 추론 능력을 로봇 행동 생성으로 확장하는 패러다임 전환

시각 이해의 통합

비전 인코더를 결합하여 시각적 맥락을 이해하고 자연어 명령을 해석

행동 실행

이해한 내용을 바탕으로 로봇의 관절 각도, 그리퍼 상태 등을 직접 출력

03
문제 정의

기존 접근의 한계

대부분의 오픈소스 VLA 프로젝트는 액션 훈련 단계에만 특화되어 있으며, 사전 훈련 파이프라인을 호환되지 않는 조각으로 결합합니다.

항목 기존 방식 VLA Foundry
LLM 사전훈련 별도 코드베이스 통합 지원
VLM 정렬 훈련 파이프라인 불일치 동일 스택 사용
VLA 액션 훈련 주로 이것만 지원 완전 통합
데이터 포맷 일관성 변환 레이어 필요 단일 포맷
코드 재사용성 중복 구현 많음 모듈화 아키텍처

핵심 문제: 파이프라인 간의 불일치가 모델 성능 저하와 개발 복잡도 증가를 초래합니다. "스티칭(stitching)" 방식은 미묘한 호환성 문제를 숨깁니다.

06
훈련 파이프라인

단계별 훈련 흐름

1단계: LLM 사전훈련

대규모 텍스트 코퍼스로 기반 언어 모델을 훈련합니다. 공유 스택의 기저가 되는 단계입니다.

2단계: VLM 정렬 훈련

비전 인코더를 결합하고 이미지-텍스트 쌍으로 시각-언어 정렬을 수행합니다. 동일한 데이터 포맷과 옵티마이저를 사용합니다.

3단계: VLA 액션 파인튜닝

로봇 데모 데이터로 액션 헤드를 추가하고 행동 출력을 학습합니다. 별도 파이프라인 전환 없이 매끄럽게 연결됩니다.

선택 단계: RLHF / DPO

인간 피드백 기반의 강화 학습으로 행동 품질을 추가로 향상시킵니다 (선택적).

07
실습

프레임워크 사용 예시

VLA Foundry의 직관적인 API 설계

# VLA Foundry — 단일 코드베이스 예시 from vla_foundry import Trainer, VLAModel # 모델 구성 (비전 인코더 + LLM 백본 + 액션 헤드) model = VLAModel( vision_encoder="siglip-large", language_backbone="llama-3-8b", action_dim=7, # 6 DOF + 그리퍼 action_head="continuous" ) # 훈련 — 모든 단계가 동일한 Trainer 클래스 trainer = Trainer(model, data="robot_demos") trainer.fit(stage="vla", epochs=100)

설계 철학: 하나의 Trainer 클래스로 LLM → VLM → VLA 모든 단계를 수행할 수 있습니다. 단계 전환 시 모델 구성만 변경하면 되며, 데이터 로더, 옵티마이저, 체크포인트 관리가 일관되게 동작합니다.

파이프라인 일관성

파이프라인 통합

LLM, VLM, VLA 훈련을 단일 코드베이스에서 수행. 호환성 문제를 근원적으로 해결합니다.

모듈식 설계

비전 인코더, 언어 백본, 액션 헤드를 자유롭게 교체 가능한 플러그인 아키텍처를 제공합니다.

통합 데이터 관리

단일 데이터 포맷으로 전체 파이프라인을 지원. 불필요한 변환 레이어를 제거합니다.

실험 재현성

일관된 훈련 환경으로 결과 재현이 용이하며, 연구자 간의 비교를 가능하게 합니다.
