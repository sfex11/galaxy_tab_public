문제 정의

LLM 환각(Hallucination)이란? 왜 위험한가

⚠ 핵심 문제

LLM이 사실과 다른 정보를 그럴듯하게 생성하는 환각 현상은 의료, 금융, 법률 등 실세계 응용에서 치명적인 신뢰성 문제를 일으킵니다.

🔍 RAG의 한계

검색 증강 생성(RAG)도 검색된 문서와 불일치하는 응답을 생성하는 문제에서 자유롭지 못합니다.

🔄 확증 편향

기존 LLM-as-a-Judge 방식은 확증 편향(Confirmation Bias)으로 인해 원래 생성의 오류를 그대로 재생산합니다.

"검증자가 원본 생성의 오류를 무의식적으로 재생산하는 확증 편향이 핵심 병목입니다."

작동 과정

MARCH 파이프라인 상세

1

Solver: RAG 응답 생성

쿼리 + 검색 문서를 입력받아 응답 생성. 제공된 문서에만 충실하며 외부 지식 추론을 금지합니다.

2

Proposer: 원자적 명제 분해

응답의 모든 핵심 주장을 질문-답변 쌍으로 분해. 특히 수치 데이터에 대해 정밀한 질문을 반드시 생성합니다.

3

Checker: 격리 상태 독립 검증

Solver 출력을 보지 않고, 오직 검색 문서만으로 질문에 답변. 증거를 먼저 제시한 후 답변하거나 "답변 불가"를 선언합니다.

4

판정: 교차 검증

Solver의 답변과 Checker의 독립 답변을 비교하여 불일치 발생 시 환각으로 판정합니다.

학습 방법

다중 에이전트 강화학습 (MARL)

🎯 PPO 기반 이중 궤적 최적화

하나의 쿼리에서 두 개의 보상 신호를 생성: 추론 경로(y)와 감사 궤적(λ). 각 에이전트가 독립적 이점 추정과 KL 패널티를 받습니다.

⚡ 무관용 보상 (Zero-Tolerance Reward)
R({(ai, âi)}) = 0 (모든 ai = âi일 때) | -1 (하나라도 불일치 시)

단 하나의 환각이라도 전체 추론 체인을 무효화하는 엄격한 패널티 구조입니다.

📊 학습 설정
  • 기본 모델: Llama3.1-8B-Instruct
  • 배치 크기: 32
  • 단일 에폭 학습
  • Actor LR: 1×10-6
  • Critic LR: 1×10-5
  • Temperature: 0.6
📚 학습 데이터
  • BioASQ: 4,721개 (STEM)
  • 2WikiMultiHopQA: 4,500개
  • MuSiQue: 4,500개
  • 비관련 문서 비율: 24~88%

주요 기여

MARCH의 핵심 기여

1

정보 비대칭 기반 환각 탐지 프레임워크

3개 전문 에이전트 간 정보 격리를 통해 확증 편향을 근본적으로 해결하는 최초의 프레임워크입니다.

2

다중 에이전트 강화학습 공동 진화

PPO 기반 이중 궤적 최적화와 무관용 보상으로 에이전트들이 함께 진화하며 사실 충실도를 극대화합니다.

3

소형 모델의 대형 모델 경쟁력

8B 파라미터 모델이 GPT-4o 등 폐쇄형 대형 모델과 동등하거나 우수한 성능을 달성했습니다.

4

확장 가능한 자기 개선 경로

외부 도구나 인간 주석 없이 LLM의 사실적 자기 개선을 위한 확장 가능한 경로를 제시합니다.

"MARCH는 공동 진화를 통한 LLM의 사실적 자기 개선을 위한 확장 가능한 경로를 보여줍니다."
