임상 진료는 객관식이 아닙니다

"임상 진료는 열거된 옵션 중 답을 선택하는 것이 아니다. 의사는 이질적인 정보를 점진적으로 수집하고, 불확실성 하에서 순차적이고 되돌릴 수 없는 결정을 내린다."

정적 벤치마크의 한계

  • 정보 탐색 불가
  • 순차적 의사결정 없음
  • 되돌릴 수 없는 결정의 부재

실제 임상의 특성

  • 불완전한 정보 → 적극적 탐색
  • 순차적 · 되돌릴 수 없는 결정
  • 약물, 시술, 진단의 통합 관리

ClinEnv가 유일하게 LIS(Longitudinal Inpatient Simulation) 패러다임의 5가지 차원을 모두 충족

ClinEnv 전체 구조

원시 입원 데이터 → 다단계 사례 구성 → 인터랙티브 환경에서 평가

1

자동 사례 구성

MIMIC-IV 원시 입원 데이터를 5단계 파이프라인으로 구조화된 다단계 사례로 변환. 수동 주석 불필요.

2

다중 에이전트 환경

임상 정보는 요청 시에만 제공. 환자, 간호사, 검사, 병력 에이전트에게 적극적으로 질의해야 함.

3

이중 평가 프레임워크

결정의 정확성(온톨로지 기반 결정론적 매칭)과 정보 수집 과정의 질을 동시에 측정.

데이터 전처리: MIMIC-IV 기반

구조화된 EHR 테이블

  • 입원 정보 (admissions)
  • 검사 및 미생물 결과
  • 약물 처방 및 투여 기록
  • ICU 차트 이벤트
  • 수액 입출력 · 시술 이벤트
  • ICD 코드 진단 및 시술

임상 노트

  • 퇴원 요약지 (Discharge Note)
  • 방사선 보고서

각 레코드는 event_time, source_table을 가진 provenance-bearing 이벤트로 변환되어 입원별 단일 타임라인으로 정렬됨

자동 사례 구성 5단계 파이프라인

A
결정 추출
퇴원 요약지에서 주치의 수준의 모든 결정 추출 (약물/시술/계획)
B
시간적 고정
순방향 슬라이딩 윈도우로 각 결정을 타임라인의 특정 이벤트에 위치화
C
단계 구성
결정을 순서화된 임상 단계로 분할하고 유효성 검증
D
진단 검증
각 단계의 결정이 EHR 근거로 정당화 가능한지 스캔
E
분류·강화
ATC/ICD 코드 매핑 및 메타데이터 부착

Claude-Sonnet-4.6을 계획 백본으로 사용하여 전 과정 자동화 — 수동 주석 없음

다중 에이전트 임상 환경

임상 정보는 기본적으로 숨겨져 있으며, 모델이 적극적으로 질의해야만 노출됨

환자 에이전트

현재 증상, 통증 정도, 주관적 호소 제공

간호사 에이전트

환경 관찰, 활력징후, 간호 평가 결과 제공

검사 에이전트

요청된 검사 결과만 반환 (비용 발생)

병력 에이전트

과거 의무기록, 이전 입원, 기저질환 제공

상호작용 모드와 구조화된 제출

Direct Mode

단일 프롬프트로 모든 정보와 결정을 한 번에 요청. 정보 탐색 없이 종합적 출력 평가.

Interactive Mode 핵심

각 단계마다 에이전트에게 자유 형식으로 질의한 후, 구조화된 인터페이스로 결정 제출.

구조화된 제출 인터페이스

각 단계에서 모델은 다음을 제출:

  • 약물 결정 — 약물명, 용량, 경로
  • 시술 결정 — 수행할 시술 목록
  • 진단 결정 — 현재 단계의 진단

자유 텍스트가 아닌 구조화된 형식으로 제출하여 결정론적 평가 가능

이중 평가 프레임워크: 결과 + 과정

임상 결정 정확도

약물
ATC 코드 기반 온톨로지 매칭 — 계층적 부분 점수 부여
진단
계층적 ICD F1 — Hungarian 매칭으로 최적 대응
시술
계층적 ICD-PCS F1 — 동일 매칭 방식

임상 과정 질

정보 탐색 질 — ground-truth 결정에 필요한 정보를 얼마나 커버했는지
검사 비용 효율 — 불필요한 검사 요청 최소화
약물 획득 비용 — 불필요한 약물 처방 최소화

LLM-as-judge 없이 모든 평가가 결정론적으로 수행됨

약물·시술 관리 능력

결과만 평가하는 벤치마크는 임상 준비도를 상당히 과대평가할 수 있다. 진단은 비교적 잘 복원하지만, 실제 관리 행위(약물·시술)에서는 극심한 성능 저하를 보인다.

장기 사례의 난이도 집중

단계 위치에 따른 성능 저하

입원 후반부 단계로 갈수록 결정 정확도가 급격히 하락. 긴 타임라인을 가진 사례일수록 저하 폭이 큼.

이는 LLM 계열 전체에서 일관되게 관찰됨 — 모델 선택의 문제가 아닌 근본적 한계.

결정 유형별 격차

  • 관리 결정이 진단보다 지속적으로 낮은 정확도
  • 결정 유형이 정확도 격차의 지배적 요인
  • 후반부 단계에서 관리 결정의 저하가 가장 극심

후반부에서 관찰되는 일시적 반등은 단계 위치와 결정 유형의 혼합 효과로 설명됨 (부록 G)

정보 탐색과 낭비의 디커플링

핵심 발견

결과 질과 과정 질이 날카롭게 분리됨.

  • 사례가 진행될수록 중복 질의 지속 발생
  • 이미 얻은 정보를 반복적으로 요청
  • 불필요한 검사·약물 비용 증가

시사점

정보 탐색은 임상 역량의 독립적이고 측정 가능한 차원임.

결과만 평가하는 기존 벤치마크에서는 이 정보 획득 격차가 완전히 보이지 않음.

프로세스 지표는 유용한 상호작용을 단순한 질의량과 분리하여 식별할 수 있음.

결론 및 기여

1

자동 사례 구성 파이프라인

원시 입원 데이터를 수동 주석 없이 다단계 사례로 변환

2

인터랙티브 다중 에이전트 임상 환경

각 결정 전 적극적 정보 획득을 요구하는 환경 설계

3

이중 평가 프레임워크

결정 정확도 + 프로세스 질의 동시 측정, LLM-as-judge 없이 결정론적 평가

ClinEnv는 결과만으로는 보이지 않는 정보 획득 격차를 직접 측정 가능하게 만듦. 임상 LLM 평가의 새로운 표준인 Longitudinal Inpatient Simulation 패러다임을 제안한다.
