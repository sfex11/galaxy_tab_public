Co-LMLM: Continuous-Query

연속 벡터 쿼리 기반 지식 외부화 언어 모델

Yair Feldman · Linxi Zhao · Nathan Godey · Dongyoung Go · Yilun Hua · Kilian Q. Weinberger · Jennifer J. Sun · Yoav Artzi

파라메트릭 지식의 한계

기존 LLM 방식

  • 사실적 지식을 모델 파라미터에 암묵적으로 저장
  • 모델 크기와 노출 빈도에 의존적
  • 롱테일 지식 포착 어려움
  • 언어 표현과 사실 연관이 얽혀 있음

발생하는 문제들

  • 개별 사실의 귀속·편집·삭제 불가
  • 할루시네이션 및 오래된 지신 문제
  • 지식 편집 시 일반 능력 저하
  • 머신 언러닝이 추가 학습 필요

"사실적 지식을 모델 가중치 밖으로 분리하여, 해석 가능하고 편집 가능한 형태로 관리할 수 있는가?"

유연·효율

Co-LMLM은 토큰 디코딩 없이 단일 생성 스텝으로 지식을 검색합니다

Rel-LMLM의 핵심 한계

Wikipedia 의존성

각 문서가 특정 엔티티 중심 → 관계형 튜플 자동 주석 가능하지만 확장 불가

관계형 표현 제약

주어-관계-객체 튜플로 표현 가능한 사실만 외부화

추가 토큰 생성 비용

자연어 쿼리를 디코딩하기 위해 여러 토큰 추가 생성 필요

사전 결정된 검색 경로

데이터 전처리에서 쿼리를 고정 → 학습된 검색 메커니즘의 표현력 제한

Co-LMLM 핵심 아이디어

쿼리를 단일 연속 벡터로 발행하여, 관계형 튜플 대신 벡터 키 + 텍스트 값을 저장하는 비구조화 KB를 구성합니다.

단일 스텝 검색

토큰 디코딩 없이 히든 상태에서 직접 쿼리 벡터 생성 → 최소 추론 오버헤드

유연한 쿼리 표현

자연어 관계형 제약 없이 임의의 문맥 정보를 벡터에 인코딩 가능

범용 코퍼스 적용

Wikipedia에 국한되지 않고 FineWeb-Edu 등 임의의 텍스트에 적용

편집 가능한 외부 메모리

LMLM의 핵심 장점 유지: KB 직접 편집, 언러닝, 지식 귀속

모델링 및 추론 과정

1
토큰 생성
일반적인 next-token prediction으로 토큰을 디코딩
2
검색 트리거
특정 토큰에서 검색이 필요하다고 판단되면 히든 상태에서 쿼리 벡터 q 생성
3
KB 검색
q와 가장 가까운 벡터 키를 FAISS 인덱스에서 검색하여 텍스트 값 반환
4
값 인코딩
검색된 텍스트를 모델이 인코딩하고 디코딩 재개

전처리는 언제 검색할지와 어떤 텍스트를 반환할지만 지정하며, 쿼리의 명시적 표현은 규정하지 않습니다. 모델이 스스로 최적의 쿼리 벡터를 학습합니다.

학습: 주석 파이프라인

자유 형식 사실 스팬 태깅

Wikipedia의 엔티티 중심 구조에 의존하지 않고, 임의의 텍스트에서 사실적 스팬을 자유롭게 태깅합니다. 주어-관계-객체 구조를 요구하지 않습니다.

시드 주석 + Gemini

Gemini를 활용해 초기 시드 주석을 생성하고, 인간 주석자 2명이 독립적으로 검증하여 품질을 보장합니다.

대조 학습 손실

합성된 양성 쌍(정확한 KB 항목)과 부정 쌍(무작위 KB 항목)에 대해 대조 손실을 적용하여 쿼리 벡터를 최적화합니다.

왜 주석자 2명?

단일 주석자의 편향을 줄이고, 사실 스팬의 경계와 적절성에 대한 합의 기반 품질 관리를 위해 두 명이 독립 주석 후 합의 도출

실험 설정

모델 구성

  • Decoder-only Transformer 아키텍처
  • 125M, 360M 규모에서 평가
  • FAISS 인덱스 사용 (IVF-PQ)
  • SmolLM2 백본과 동일 구성

사전학습 데이터

  • Wikipedia (Rel-LMLM과 동일 조건)
  • FineWeb-Edu (범용 코퍼스 확장)
  • 대조군: 동일 코퍼스의 일반 LLM
  • Rel-LMLM 재구현 (동일 백본)

평가 범위

Perplexity · FactScore (장문) · T-REx (단문) · PopQA · TriviaQA · SimpleQA · NLU 벤치마크 · 언러닝 (TOFU) · 지식 기억력 분석

Perplexity 결과

모델 (360M) 사전학습 데이터 Perplexity ↓
SmolLM2-360M 2T 토큰 (40× 더 많음) 12.61
표준 LLM (Wiki) Wikipedia 14.58
Rel-LMLM (Wiki) Wikipedia 13.49
Co-LMLM (Wiki) Wikipedia 12.04
Co-LMLM (FW-Edu) FineWeb-Edu 10.87

Co-LMLM은 40배 더 많은 데이터로 학습된 SmolLM2-360M보다 낮은 perplexity를 달성하며, FineWeb-Edu 확장 시 더욱 큰 개선을 보입니다.

모델점수

핵심 관찰

  • Co-LMLM은 Claude Sonnet 4.5보다 높은 SimpleQA 점수 달성
  • GPT-4o-mini와 유사한 수준의 사실 정밀도
  • Rel-LMLM 대비 사실 정밀도 대폭 향상
  • 표준 LLM 대비 약 3배 향상
  • NLU 성능은 동급 모델과 유지

언러닝 및 지식 제어

머신 언러닝 (TOFU)

특정 저자의 지식을 KB에서 직접 삭제하는 것만으로 언러닝이 완료됩니다. 추가 학습이나 특수 목적 함수가 필요하지 않습니다.

  • 목표 지식 완전 제거 확인
  • 비목표 지능 유지율 100%
  • 유틸리티 트레이드오프 없음

지식 기억력 분석

Co-LMLM은 사전학습 데이터에 등장하는 사실을 모델 파라미터가 아닌 KB에 외부화하므로, 모델 자체의 파라메트릭 기억력은 의도적으로 낮게 유지됩니다.

  • 파라메트릭 기억 최소화
  • KB 의존도 검증
  • 지식 소스 명확한 귀속 가능

연속 쿼리 vs 자연어 쿼리 & 추론 효율

연속 쿼리의 우위

  • 단일 포워드 패스로 쿼리 생성 (토큰 디코딩 불필요)
  • 더 풍부한 문맥 정보를 벡터에 인코딩 가능
  • 사전 결정된 쿼리가 아닌 학습된 유연한 검색

추론 오버헤드

  • 검색당 추가 비용: FAISS 쿼리 1회
  • Rel-LMLM 대비 쿼리 토큰 생성 비용 완전 제거
  • 전체 추론 속도에 미미한 영향만

쿼리 타이밍 평가

모델이 언제 검색을 수행해야 하는지를 스스로 학습합니다. 주석된 검색 지점을 기준으로 정확도를 측정하며, Co-LMLM은 관련 문맥에서 적시에 검색을 트리거하는 능력을 보여줍니다.

논의 및 결론

주요 기여

  • 연속 벡터 쿼리 기반 LMLM 패러다임 제안
  • 관계형 제약 없는 자유 형식 사실 외부화
  • Wikipedia 이상의 범용 코퍼스 적용 가능성 입증
  • 360M 규모에서 GPT-4o-mini급 사실 정밀도
  • 40× 더 많은 데이터로 학습된 모델보다 낮은 PPL

향후 방향

  • 더 큰 규모 모델로의 확장
  • 다양한 도메인 코퍼스로의 일반화
  • 멀티모달 지식 외부화 탐색
  • 실제 응용 시나리오에서의 지식 편집 파이프라인
  • 검색 타이밍 학습 메커니즘 심층 분석

Co-LMLM은 지식을 모델 가중치에서 분리하면서도, 연속 쿼리를 통해 표현력과 효율성을 동시에 달성하는 새로운 패러다임을 제시합니다.
