Position-Aware Drafting for
Inference Acceleration in LLM-Based
Generative List-Wise Recommendation

LLM 기반 생성형 리스트 추천에서 추론 속도를 가속화하기 위한
위치 인식 드래프팅 방법론

Jiaju Chen, Chongming Gao, Chenxiao Fan, Haoyan Liu, Qingpeng Cai, Peng Jiang, Xiangnan He
USTC · Zhongguancun Academy

배경

LLM 기반 생성형 리스트 추천의 부상

생성형 추천 패러다임

기존 점수 기반(rank) 추천에서 생성(generate) 기반으로 전환

  • 아이템 → 시맨틱 ID 토큰 시퀀스로 토큰화
  • LLM이 오토레그레시브 방식으로 리스트 생성
  • TIGER, EAGER, LC-Rec 등 활발한 연구

핵심 병목

오토레그레시브 디코딩의 순차적 특성이 지연 유발

  • 아이템당 K=4개 토큰 생성 필요
  • 리스트당 수십~수백 토큰 순차 처리
  • 실시간 추천 서비스에 부적합

"추론 가속은 LLM 기반 생성형 추천의 실제 배치를 위한 가장 시급한 문제이다."

기존 방법

사양 디코딩(Speculative Decoding)

1
드래프트
작은 모델이 여러 토큰을 한 번에 제안
2
검증
타겟 LLM이 배치 호출로 후보 검증
3
수용
가장 긴 일치 접두사를 승인하고 이어서 디코딩

장점

  • 타겟 분포를 정확히 보존 (lossless)
  • 비싼 타겟 호출 횟수 감소
  • EAGLE, HASS 등 지속적 발전

한계

  • 범용 언어 생성에 최적화됨
  • 구조화된 시맨틱 ID 출력을 고려하지 않음
  • 추천 태스크 특유의 특성 무시
문제 정의

생성형 추천에서 SD의 두 가지 과제

추천 출력은 아이템 구조화되어 있음: 각 아이템은 K=4개의 시맨틱 ID 토큰(코드북 레벨)로 표현됨

과제 1 슬롯 조건부 의미 활용 부족

각 아이템 내 고정 위치(슬롯)에 따라 토큰 의미가 다름

  • 기존 드래프트는 토큰을 균일하게 처리
  • 슬롯 식별 정보가 없어 아이템 내 구조 모델링 약화
  • 타겟 헤드와의 정렬 저하

과제 2 깊이 기반 불확실성 미관리

드래프트 깊이가 깊어질수록 불확실성 증가

  • 이후 토큰은 더 긴 히스토리에 의존 (오류 누적)
  • 조기 거부(rejection) 확률 증가
  • 기존 학습은 단계 무관(step-agnostic)

결과: 수용률(acceptance rate) 저하 → 실제 가속 효과 제한

제안 방법

PAD-Rec 개요

Position-Aware Drafting for generative Recommendation

표준 드래프트 모델에 플러그인되는 경량 모듈로, 두 가지 보완적 위치 신호를 제공

IPE

아이템 위치 임베딩

토큰의 아이템 내 슬롯 위치를 명시적으로 인코딩하여 구조 인식 강화

SPE

스텝 위치 임베딩

현재 드래프트 스텝을 인덱싱하여 깊이 조건부 동작 학습

기존 드래프트에 쉽게 통합

무시할 수준의 추론 오버헤드

학습 가능한 게이트로 신호 조화

IPE

아이템 위치 임베딩 (Item Position Embeddings)

동기

RQ-VAE 기반 시맨틱 ID에서 각 아이템은 K개 토큰(코드북 레벨 1→K)으로 구성됨

  • 레벨 1: coarse 카테고리
  • 레벨 2~K: 점진적으로 세분화
  • 슬롯별 의미 체계가 상이함

슬롯 정보 없이는 드래프트가 토큰 간 의미 차이를 구분하지 못함

설계

각 슬롯 s ∈ {1, ..., K}에 대해 학습 가능한 임베딩 벡터 es를 정의

구분자(separator) 토큰 위치에는 영벡터 할당

핵심 효과: 드래프트가 아이템 내부 구조를 인식하고, 타겟 LLM의 출력 분포와 더 잘 정렬됨

SPE

스텝 위치 임베딩 (Step Position Embeddings)

동기

SD의 "첫 불일치에서 중단" 검증 메커니즘으로 인해:

  • 스텝 1: 비교적 높은 수용률
  • 스텝 D: 오류 누적으로 낮은 수용률
  • 모든 스텝에서 동일한 제안 전략 사용 → 비효율

설계

각 드래프트 스텝 d ∈ {1, ..., D}에 대해 학습 가능한 임베딩 pd 정의

드래프트가 스텝별로 다른 제안 전략을 학습 가능

핵심 효과: 얕은 스텝에서는 공격적 제안, 깊은 스텝에서는 보수적 제안 → 전체 수용률 향상

게이팅

게이팅 메커니즘

두 위치 신호를 기존 특징과 조화롭게 융합하기 위한 단계별 게이트

Stage 1: 슬롯 인식 융합

IPE를 기본 특징에 결합

αs — 학습 가능한 스칼라 계수
슬롯 임베딩의 기여도를 조절

파라미터 수: K개 (매우 적음)

Stage 2: 깊이 인식 융합

SPE를 슬롯 융합 결과에 결합

gd — 컨텍스트 구동 게이트
MLP + 시그모이드로 스텝 임베딩 적응적 제어

입력 문맥에 따라 스텝 신호의 중요도 동적 조절

두 게이트 모두 단순하고 경량이며, 추론 시 추가 연산량 무시 가능

학습

학습 파이프라인

인과적 마스킹 (Causal Masking)

드래프트 스텝 d에서 이전 스텝의 예측 토큰만 참조하도록 마스킹

  • 추론 시 드래프트 조건과 정확히 일치
  • 정보 누설 방지

점진적 대체 (Progressive Replacement)

학습 중 드래프트 예측을 점진적으로 실제 토큰으로 대체

  • 초기: 교사 강제(ground truth)
  • 점진적: 드래프트 자체 예측 사용
  • 추론-학습 갭 최소화

PAD-Rec 모듈은 드래프트 모델과 공동 학습되며, 타겟 LLM은 동결

추천 품질 보존

SD는 타겟 분포를 정확히 보존하므로:

  • N@K, HR@K 기본적으로 동일
  • PAD-Rec 추가로 품질 저하 없음
  • Lossless acceleration 달성

HASS 대비 4개 데이터셋에서 평균 5%의 추가 wall-clock 가속을 달성하며, 추천 성능은 완전히 보존됨

RQ3 & RQ4

하이퍼파라미터 & 스케일링 분석

드래프트 길이 D의 영향 (RQ3)

  • D가 증가할수록 잠재적 가속은 커지지만, 깊은 스텝의 수용률 저하
  • PAD-Rec은 D가 클수록 SPE의 효과가 더 두드러짐
  • 최적 D 범위에서 가장 큰 이득

모델 스케일링 (RQ4)

  • 타겟 모델이 클수록 SD의 기본 가속 효과 증가
  • PAD-Rec의 추가 이득은 모델 크기에 비례
  • 더 큰 타겟 → 더 큰 드래프트-타겟 갭 → 위치 인식의 가치 증가

PAD-Rec은 다양한 드래프트 길이와 모델 규모에서 일관된 개선을 보임
