Background

왜 언어 독립성인가?

다국어 LLM의 내부 표현이 언어 식별 정보와 개념 정보를 분리하여 인코딩한다는 증거가 축적되고 있습니다. 하지만 작업 수준의 표현인 Function Vector에서도 같은 현상이 나타나는지는 미지수였습니다.
표현 수준
mBERT, 자기회귀 LLM에서 언어 간 개념 정렬이 자발적으로 발현됨
문법·회로 수준
수·성·시제 등 형태통사 특성과 주어-동사 일치 회로가 언어 간 공유됨
작업 수준 (FV)?
ICL로 추출된 Function Vector도 언어 독립적인가? → 본 연구가 최초로 검증
What are Function Vectors?

FV 추출 절차

In-Context Learning 시 정상 프롬프트 vs 레이블 셔플 프롬프트를 비교하여, 각 어텐션 헤드의 인과적 간접 효과(CIE)를 측정합니다.
CIEℓj(p̃) = P(y | p̃ ; aℓj ← āℓjt) − P(y | p̃)
CIE가 높은 상위 헤드들의 평균 출력을 합산하여 FV를 구성합니다.
vt = Σ(ℓ,j) ∈ A āℓjt   →   잔차 스트림에 가산적 개입
번역 FV의 경우: 영어 단어 → 타겟 언어 번역의 ICL 데모를 사용하며, 타겟 측 레이블을 무작위 순열하여 교란 프롬프트를 생성합니다.

Shared Mechanism

FV 헤드 집합의 방향 간 중첩

상위 10개 헤드 중 6~8개가 세 번역 방향에서 공통으로 선택됩니다. 이는 FV가 공유 번역 회로를 포착하고 있음을 시사합니다.
3방향 공통 (6~8개)
2방향 공유
방향 특이적
이 결과는 Zhang et al.(2025a)이 FV 프레임워크 외부에서 보고한 번역 관련 헤드 중첩과 일치하는 발견입니다.

Conclusions

핵심 발견

교차언어 전이
단일 EN→X 방향 FV가 10개 미지시 언어에서 정답 번역 토큰 순위를 일관되게 상승시킴
공유 회로
상위 10개 FV 헤드 중 6~8개가 3방향에서 공통 → 언어 독립적 번역 부분 회로 존재
인과적 중요성
FV 방향 제거 시 번역 성능 저하, 관련 없는 작업에는 제한적 영향
모델 간 전이
베이스 모델 FV가 인스트럭션 튜닝 모델로도 전이 가능
단어→문장 수준
단어 수준 FV가 문장 수준 번역에 부분적으로 일반화
시사점
FV는 언어 특이적 매핑이 아닌 언어 독립적 번역 "의도"를 인코딩할 가능성
