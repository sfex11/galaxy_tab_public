동기

왜 다목표 다에이전트 문제가 어려운가?

목표 간 충돌

효율성, 안전성, 에너지 소비, 공정성 등 여러 목표가 서로 충돌하는 상황에서 단일 스칼라 보상으로는 충분하지 않음

에이전트 간 충돌

서로 다른 관측, 역할, 기여도를 가진 에이전트들이 동일한 목표 하에서도 갈등을 유발

신호등 없는 교차로에서 두 자율주행차가 모두 효율성을 선호하면 충돌하고, 모두 안전성을 선호하면 교착 상태에 빠짐 — 조율된 선호도가 필요함

핵심 아이디어

선호도 조율의 직관

공유 선호도 (기존 방식)

모든 에이전트가 동일한 선호도 벡터 w를 사용 → 파레토 전선 상의 단일 지점에 집중 → 행동 충돌 발생

조율된 선호도 (제안 방식)

각 에이전트가 서로 다른 선호도 w₁, w₂, w₃를 사용 → 파레토 전선 상의 다양한 지점을 커버 → 보완적 역할 분담

단일 에이전트 = 전선 상 1점  ·  단일 목표 = 전선 자체가 없음  ·  다에이전트 + 다목표 + 조율 = 전선 전체를 커버

배경지식

다목표 분산 POMDP

G = ⟨N, S, {Aᵢ}, P, Ω, O, r_team, {rᵢ}, γ⟩

N

에이전트 수

K

목표 수 (벡터 보상 차원)

γ

할인 인자

팀 보상 r_team

스칼라 값 · 모든 에이전트가 공유 · 팀 목표 완료 측정 (예: 승리 시 +1)

가이드 벡터 보상 rᵢ ∈ ℝᴷ

에이전트별 · 효율성, 비용, 위험 등 보조 다목표 피드백 제공

문제 정식화

팀 최적 균형 문제

목표 함수 정의

J_team(θ) = 𝔼[Σ γᵗ r_team(sₜ, aₜ)]   — 팀 스칼라 목표

Jᵢ(θ) = 𝔼[Σ γᵗ rᵢ(sₜ, aₜ)]   — 에이전트 i의 벡터 목표

선호도 조건부 페이오프

Uᵢ(θ; pᵢ) = J_team(θ) + pᵢᵀ Jᵢ(θ)

선호도 프로파일 p = (p₁, ..., pₙ)이 주어지면 확률 게임 G(p)가 유도됨

핵심 질문: 고정된 선호도 하의 균형이 반드시 팀 목표를 최대화하지 않음 → 선호도 프로파일 자체를 조율하여 유도되는 균형이 높은 팀 성능을 달성하도록 만들어야 함

이론적 분석 ①

1차 팀 개선과 선호도 다양성

주요 결과 (정리 4.2)

적절한 조건 하에서, 선호도 다양성이 1차 팀 목표 개선에 긍정적으로 기여함을 수학적으로 증명

팀 개선의 분해

1
공통 개선 항
모든 에이전트가 공유하는 방향의 기여
2
선호도 다양성 항
서로 다른 선호도로 인한 보완적 개선
Σ
전체 1차 팀 개선
두 항의 합으로 분해 가능

선호도가 동일하면 다양성 항이 0이 되어 잠재적 개선 기회를 상실함

이론적 분석 ②

균형 추적 정리

주요 결과 (정리 4.6)

서로 다른 선호도 프로파일 하의 균형들이 국소적으로 부드러운 정상 경로로 연결됨을 보임

시사점

  • 선호도를 점진적으로 변화시켜도 균형이 불연속적으로 튀지 않음
  • 선호도 최적화를 그래디언트 기반으로 안전하게 수행 가능
  • 기존 균형에서 출발해 선호도를 조율하며 새로운 균형로 점진적 이동 가능

이론적 결과는 PCMA 알고리즘이 선호도 공간에서 점진적 탐색을 통해 팀 성능을 개선할 수 있는 근거를 제공

제안 방법

PCMA: 선호도 조율 다에이전트 정책 최적화

Preference Coordinated Multi-agent Policy Optimization — CTDE(중앙집중 훈련, 분산 실행) 프레임워크 기반

확률적 선호도 기획자

  • 에이전트별 선호도를 잠재 조율 변수로 모델링
  • 팀 개선과 로컬 관측 기반으로 선호도 샘플링
  • 학습된 선호도 분포에 정규화하여 동일 선호도로의 수렴 방지

선호도 조건부 액터

  • 샘플링된 선호도 pᵢ를 조건으로 정책 실행
  • 단일 정책으로 다양한 트레이드오프 달성
  • 실행 시 새로운 선호도에 즉시 적응 가능
제안 방법

선호도 조건부 정책 최적화

1

선호도 샘플링

각 에이전트의 기획자가 관측 oᵢ 기반으로 선호도 pᵢ ~ q(·|oᵢ)에서 샘플링

2

조건부 행동 선택

액터 네트워크 π(aᵢ|oᵢ, pᵢ)가 샘플링된 선호도를 조건으로 행동 결정

3

스칼라 보상 구성

Uᵢ = r_team + pᵢᵀ rᵢ 로 스칼라 페이오프 계산 → 표준 PPO 업데이트 적용

4

선호도 분포 업데이트

팀 목표 J_team의 그래디언트로 기획자 q(·|oᵢ) 업데이트 + 다양성 정규화

제안 방법

조율된 선호도 기획

다양성 정규화의 핵심 역할

정규화 항 없이 학습하면 모든 에이전트의 선호도 분포가 동일한 방향으로 수렴(collapse)하는 문제 발생

정규화 없음

모든 에이전트가 최적 단일 방향으로 수렴
→ 선호도 다양성 상실
→ 공유 선호도와 동일한 한계

다양성 정규화 적용

에이전트 간 선호도 분포의 차이를 장려
→ 의미 있는 특화 달성
→ 파레토 전선 상 보완적 커버

실험 ①

선호도 조율이 협력에 미치는 영향

Particle-World: 선호도 특화

PCMA가 에이전트 간에 뚜렷한 선호도 특화 패턴을 학습함을 확인. 각 에이전트가 다른 목표 축을 우선시하며 팀 전체의 파레토 전선을 커버

SMAC: 역할 명세

스타크래프트 미니게임에서 PCMA가 유닛의 전술적 역할을 자동으로 분담. 공격형/방어형 선호도가 유닛 특성에 따라 자연스럽게 분리

정성적 분석 결과: 선호도 조율이 단순한 성능 향상을 넘어, 에이전트 간 역할 분담과 행동 다양성을 촉진

실험 ③

절제 연구 및 실제 시나리오 검증

절제 연구

  • 다양성 정규화 제거 시 성능 저하
  • 선호도 기획자를 결정론적으로 변경 시 특화 소실
  • 에이전트별 가이드 보상 제거 시 조율 품질 하락

  • PCMA가 실제 도로 환경에서도 유의미한 개선 달성

모든 구성 요소(확률적 기획, 조건부 액터, 다양성 정규화)가 성능에 필수적임을 확인

결론

요약 및 향후 과제

주요 기여

  • 협력적 MOMARL을 팀 최적 균형 문제로 정식화
  • 선호도 다양성이 1차 팀 개선에 기여함을 이론적 증명
  • 선호도 조건부 균형 간의 국소 연결성(균형 추적) 증명
  • PCMA 알고리즘 제안: 확률적 기획자 + 조건부 액터 + 다양성 정규화
  • MPE, SMAC, MOMAland, CARLA에서 성능 및 조율 품질 개선 확인

향후 과제

  • 에이전트 수와 목표 수가 대규모로 확장될 때의 선호도 조율 메커니즘
  • 비정상 환경 및 에이전트 진입/퇴장 상황에서의 적응적 선호도 재조율
