SkillOpt

자기 진화형 에이전트 스킬을 위한 실행 전략

Yifan Yang, Ziyang Gong, Weiquan Huang, Qihao Yang 외 · Microsoft, 상교대, 동제대, 복단대

스킬 문서를 외부 상태로 취급하고, 딥러닝 옵티마이저와 동일한 규율로 훈련하는 최초의 체계적 텍스트 공간 옵티마이저

기존 스킬 접근법의 한계

수동 작성

도메인별로 수작성 → brittle, 확장 불가

원샷 생성

LLM이 한 번에 생성 → 피드백 없이, 개선 불가

느슨한 자기 수정

제어 없는 자기 수정 → 방향 불일치, 누적 오류

공통 문제: 딥러닝 옵티마이저처럼 동작하지 않으며, 피드백 하에서 시작점보다 안정적으로 개선되지 않음

핵심 아이디어

스킬을 동결된 에이전트의 외부 상태로 취급하고, 가중치 공간 최적화를 재현 가능하게 만드는 동일한 규율로 훈련하라

딥러닝 옵티마이저

  • 배치 샘플링으로 노이즈 제어
  • 학습률로 스텝 크기 제한
  • 검증셋으로 수용 여부 결정
  • 모멘텀으로 장기 방향 유지

SkillOpt (텍스트 공간)

  • 롤아웃 배치 → 반성 미니배치
  • 텍스트 학습률 예산
  • 홀드아웃 검증 게이트
  • 에포크별 느린/메타 업데이트

순전파: 롤아웃 증거 수집

문제 설정

  • 스킬 s: 에이전트 컨텍스트에 삽입되는 자연어 정책
  • 타겟 모델 M: 동결됨, 가중치 변경 없음
  • 하니스 h: direct chat / Codex / Claude Code

스킬은 시스템 프롬프트에 prepend되거나, 툴 사용 하니스에서 지속적 절차적 메모리로 동작

롤아웃 배치

현재 스킬 st로 태스크 배치를 실행하여 성공/실패 궤적을 수집

배치 크기는 증거의 노이즈 수준을 제어 → 가중치 공간의 배치 크기와 동일한 역할

역전파: 미니배치 반성과 유한 편집

미니배치 반성

프론티어 옵티마이저 모델이 성공 궤적과 실패 궤적을 각각 분석하여 원인과 패턴을 도출

구조화 편집 제안

자유 형식 재작성이 아닌 add / delete / replace 연산으로만 스킬 문서 수정 제안

텍스트 학습률 예산

스케줄된 편집 예산 하에서 후보 편집을 병합·랭킹 → 한 에포크에서 허용된 편집 수만 적용

유한 업데이트: 연속 스킬 수정이 이전 버전에서 너무 멀어지는 것을 방지하여 최적화 이력의 의미를 보존

검증 게이트와 거부 편집 버퍼

검증 게이트

  • 홀드아웃 selection split 사용
  • 후보 스킬이 검증 점수를 엄격히 개선할 때만 수용
  • 딥러닝의 validation set과 동일한 역할
  • 유해한 제안의 누적 방지

거부 편집 버퍼

  • 게이트에서 거부된 편집을 저장
  • 이후 업데이트에 부정적 피드백으로 제공
  • "무엇이 실패했는지"를 옵티마이저가 학습
  • 무작위 탐색이 아닌 지향적 개선 가능

에포크별 느린/메타 업데이트와 배포

느린/메타 업데이트

모멘텀과 유사하게, 여러 에포크에 걸쳐 안정적인 편집 방향을 보존

  • 장기 horizon의 규칙성 포착
  • 배포되는 스킬의 크기 부풀림 방지
  • 단기 변동과 장기 패턴 분리

하니스 무관 배포

최종 산출물: best_skill.md

  • 300~2,000 토큰 (매우 컴팩트)
  • 1~4회 수용된 편집만으로 완성
  • 추론 시 추가 모델 호출 제로
  • 텍스트이므로 감사 가능

절제 연구와 학습된 스킬 분석

주요 절제 발견

  • 유한 텍스트 학습이 무제어 재작성보다 우수
  • 홀드아웃 게이트가 유해 제안 누적 방지
  • 거부 편집 버퍼가 실패를 부정적 피드백으로 변환
  • 느린/메타 업데이트가 장기 정밀화 개선

학습된 스킬의 특성

  • 300~2,000 토큰으로 매우 컴팩트
  • 1~4회 수용 편집만으로 완성 (편집 경제성)
  • 인스턴스별이 아닌 절차적(procedural) 내용
  • 사람이 읽고 감사 가능

벤치마크별 사례 연구: ALFWorld에서는 객체 상호작용 순서 규칙 학습, SpreadsheetBench에서는 함수 선택 및 참조 전략 발견

기여 및 결론

1

공식화

에이전트 스킬 학습을 외부 자연어 상태에 대한 최적화로 공식화하고, 딥러닝 스타일 제어기를 갖춘 최초의 체계적 옵티마이저 제안

2

포괄적 실증

6 벤치마크 × 7 모델 × 3 하니스에서 52/52 셀 최고 성능, 기존 6종 베이스라인 전체 우위

3

전이 검증

교차 모델, 교차 하니스, 교차 벤치마크 전이로 최적화된 스킬 산출물의 컴팩트성·재사용성·배포 가능성 입증

전망: 스킬 최적화를 프론티어 에이전트 도메인 적응의 표준 패러다임으로 확장
