teLLMe Why
(Ain't Nothing but a Jam)

도시 주행 데이터의 탐색적 인과 분석 시스템

Qiwei Li, Jorge Ortiz · Rutgers University
arXiv:2607.15254v1 · 2026년 7월

배경과 동기

도시는 대시캠, CCTV, 모바일 기기 등을 통해 대규모 비디오 데이터를 생성합니다. 이 데이터는 교통 안전과 혼잡 연구에 풍부한 정보원이 됩니다.

분석가들은 종종 인과적 질문을 던집니다:

  • "비가 오면 교차로 교통 밀도가 줄어드는가?"
  • "혼잡 시간대가 특정 구간의 수요를 증가시키는가?"

핵심 문제

대부분의 비디오 데이터는 관찰 데이터(observational)로 수집됩니다. 개입(intervention)이 없으므로, 진짜 인과 효과를 교란(confounding)과 노출 차이로부터 구별하기 어렵습니다.

해결해야 할 과제들

1

불균형과 교란
날씨, 시간대, 장면 유형, 수요 패턴이 설계에 의해 통제되지 않은 채 상호작용하며, 일부 조합은 극히 드물게 발생합니다.

2

다단계 파이프라인
객체 탐지 → 이벤트 특징 집계 → 인과 구조 추론 → 처리 효과 추정으로 이어지는 복잡한 단계를 거쳐야 합니다.

3

소통의 간극
추정 결과를 인과 그래프나 조정 규칙을 해석할 수 없는 실무자에게 전달해야 합니다.

teLLMe 시스템 개요

관찰적 도시 주행 데이터에서 탐색적 인과 분석을 수행하는 통합 시스템

1
오프라인 구축
대시캠 주석 → 이벤트 테이블 → PC 알고리즘으로 인과 그래프 학습
2
자연어 질의
사용자가 자연어로 질문 → 스키마 인식 LLM이 구조적 인과 질의로 변환
3
효과 추정
DAG 기반 백도어 조정 집합 선택 → OLS 및 DoWhy로 ATE 추정
4
Causal Card
효과 추정값, 조정 집합, 가정, 자연어 설명을 요약 출력

데이터: 대시캠 주석 기반 이벤트 테이블

BDD100K와 같은 대시캠 말뭉치에서 구조화된 이벤트 테이블을 구축합니다.

  • 원시 탐지 및 메타데이터를 고정 길이 윈도우로 집계
  • 날씨 라벨, 장면 유형, 교통 밀도, 혼잡 시간대, 시간대 구분 포함
  • 교통 밀도 = 분당 탐지된 차량 수
  • 결측 라벨 제거 및 극단값 클리핑
  • 계층적 샘플링으로 균형 부분집합 생성
변수유형예시
weather범주비, 맑음, 흐림
is_peak_hour이진0 / 1
traffic_density연속차량 수/분
scene_type범주교차로, 고속도로
weekday이진평일 / 주말
time_bin범주오전, 오후, 야간

인과 그래프 학습: PC 알고리즘 + 부트스트랩

변수 집합 V = {weather, is_peak_hour, traffic_density, scene_type, weekday, time_bin}에 대해 DAG G를 학습합니다.

PC 알고리즘 (pgmpy 구현)

완전 무방향 그래프 → 조건부 독립성 검정으로 엣지 제거 → 논리 규칙으로 방향 지정

도메인 제약

예: traffic_density → weather 엣지 금지 (비현실적 방향 차단)

부트스트랩 안정성 분석

B = 20회 부트스트랩 리샘플링 후 각 리샘플에서 PC 알고리즘 실행

안정성 점수 s(e) ∈ [0, 1]: 각 방향 엣지 e가 리샘플에서 출현한 비율

높은 s(e) → 안정적인 엣지
낮은 s(e) → 주의 필요한 엣지

자연어 질의 파싱과 조정 집합 선택

자유 텍스트를 데이터셋 스키마에 매핑하기 위해 스키마 인식 LLM을 활용합니다.

구조적 인과 질의 Q = (T, t_treated, t_control, Y, C)

T: 처리 변수 · t_treated/t_control: 처리값 · Y: 결과 변수 · C: 조건 제약

LLM 프롬프트에 (i) 데이터셋 스키마(변수명, 유형, 허용값)와 (ii) 사용자 질문을 제공하고, 제약된 JSON 형식으로 Q를 출력하도록 지시합니다.

검증 계층

  • 엄격한 타입 검사
  • 정의되지 않은 변수 참조 거부
  • 유효하지 않은 범주값 폐기
  • 실패 시 규칙 기반 파서로 폴백

백도어 조정 집합 Z 선택: 학습된 DAG G에서 T나 Y의 부모 중 T의 자손이 아닌 공변량을 포함하는 휴리스틱 적용

효과 추정 방법

필터링된 데이터셋 D_Q에 대해 평균 처리 효과(ATE) 추정:

τ_Q = E[Y | do(T=t_treated), C] − E[Y | do(T=t_control), C]

선형 회귀 (OLS)

Y = α + βT + γᵀZ + ε

계수 β를 τ_Q의 추정값으로 사용. 점추정값, 표준오차, 신뢰구간 보고.

DoWhy 추정기

CausalModel(D_Q, T, Y, G의 DOT 표현)

백도어 추정량(estimand)을 식별하고 선형 회귀로 추정. OLS와의 일관성을 위해 동일 조정 집합 사용. 선택적 부트스트랩으로 안정성 추가 확인.

Causal Card: 결과 요약 형식

각 질의에 대해 Causal Card라는 압축 요약을 반환하여 실무자도 결과를 이해할 수 있도록 설계했습니다.

Causal Card 구성 요소

요소내용
파싱된 변수처리 T, 결과 Y, 처리/대조 값, 필터 조건
효과 추정τ_Q 점추정값 및 신뢰구간
표본 크기처리군 / 대조군 샘플 수
DAG 정보직접 T→Y 엣지 존재 여부 및 안정성 점수 s(e)
조정 집합선택된 백도어 조정 변수 Z
가정 및 경고모델링 선택, 한계, 주의사항
자연어 설명비전문가도 이해할 수 있는 짧은 요약

사례 연구 1: 날씨와 교차로 교통 밀도

질의: "비가 오면 도시 교차로의 교통 밀도가 어떻게 변하는가?"

  • 처리: weather (비 vs 맑음)
  • 결과: traffic_density
  • 필터: scene_type = intersection

시스템은 날씨와 교통 밀도 간의 그럴듯한 인과 관계를 탐색하며, 조정 집합에 포함된 변수들이 교란을 어느 정도 통제했는지를 Causal Card에 명시합니다.

핵심 발견

비가 교차로 교통 밀도에 미치는 효과가 추정되지만, 날씨-시간대-장면 유형 간의 복합적 교란이 존재합니다. 부트스트랩 안정성 점수를 통해 어떤 엣지가 신뢰할 수 있는지 명확히 구분됩니다.

이 결과는 "확정적 인과 주장"이 아닌 "가설 생성" 목적임을 명시

사례 연구 2 & 3

혼잡 시간대와 고속도로 밀도

"혼잡 시간대가 고속도로 교통 밀도에 미치는 영향은?"

  • 처리: is_peak_hour (1 vs 0)
  • 결과: traffic_density
  • 필터: scene_type = highway

혼잡 시간대의 효과가 긍정적 방향으로 추정되며, 시간대 빈(time_bin)과 평일/주말 여부가 주요 조정 변수로 선택됩니다.

조정 선택에 대한 민감도

조정 집합과 균형화 선택을 달리할 때 추정치가 어떻게 변하는지 분석합니다.

  • 다양한 조정 집합 조합 비교
  • 추정치의 변동 폭 관찰
  • 모델링 선택의 영향을 명시적으로 보고

→ teLLMe는 단일 "정답" 대신 불확실성을 투명하게 전달

논의와 한계

기여점

  • 비디오 파생 데이터에서 인과 추론까지의 통합 워크플로우 제시
  • 인과 가정을 명시적으로 만들고 재현 가능하게 유지
  • Causal Card로 비전문가도 결과 이해 가능
  • 부트스트랩 안정성으로 불확실성 정량화

한계

  • 관찰 데이터만 사용하므로 미관측 교란 문제 불가피
  • PC 알고리즘은 숨겨진 공통 원인을 식별하지 못함
  • LLM 파싱의 환각(hallucination) 위험 — 스키마 검증으로 부분 완화
  • 백도어 조정 집합 선택이 휴리스틱에 의존
  • 확정적 인과 주장이 아닌 가설 생성 도구로 설계됨
