인용되었지만 검증되지 않았다

LLM 딥 리서치 에이전트에서 출처 귀속의 파싱과 평가

Hailey Onweller, Elias Lumer, Austin Huber, Pia Ramchandani, Vamse Kumar Subbiah, Corey Feld

PricewaterhouseCoopers, Commercial Technology and Innovation Office

문제 제기

딥 리서치 에이전트의 확산

Perplexity AI, ChatGPT 웹 검색, Google Gemini 등이 수백 개의 웹 출처에서 정보를 종합하여 인용된 리포트를 생성합니다.

검증 불가능한 인용

생성된 인용은 접근 불가한 URL, 무관한 콘텐츠, 출처 왜곡 등의 문제를 포함하며 신뢰할 수 없습니다.

상업 배포 모델의 인용 환각률은 11%~57%에 이르며, 기존 프레임워크는 이진 속성 분류 이상의 평가를 수행하지 못합니다.

기존 연구의 한계

1

다차원 평가 부재

URL 접근성, 주제 관련성, 사실 정확성을 동시에 평가하는 종단간 프레임워크가 없습니다.

2

모델 간 비교 누락

딥 리서치 설정에서 주요 LLM 제공자 간의 체계적인 비교가 존재하지 않습니다.

3

검색 깊이 관계 미탐색

검색 깊이가 인용 품질에 미치는 영향이 탐구되지 않았습니다.

AttributionBench · CiteME · CiteEval

이진 분류, 인용 매칭, 패러다임 비교에만 집중

CiteAudit · RefChecker

학술 논문 위조 참고문헌에 초점, 라이브 웹 인용 미대상

연구 질문

RQ1. 프론티어 LLM은 얼마나 사실적으로 정확한 인용을 생성하는가? 표면적 인용 품질(링크 유효성, 주제 관련성)이 더 깊은 사실적 실패를 가리는가?

RQ2. 검색 깊이를 늘리면 이 세 가지 차원의 인용 품질에 어떤 영향을 미치는가?

핵심 가설

표면적 지표(링크 작동, 관련성)는 안정적으로 유지되면서도, 사실 정확성은 검색 깊이 증가에 따라 저하될 것이다 — 정보 과부하가 사실적 종합을 방해하는 현상.

평가 프레임워크 개요

1
리포트 생성
딥 리서치 에이전트가 인라인 인용이 포함된 Markdown 리포트 생성
2
AST 파싱
Markdown AST 파서로 인용-주장 쌍을 구조적 추출 (LLM 불필요)
3
출처 검색
각 인용 URL의 실제 콘텐츠를 검색하여 추출
4
3차원 평가
Link Works · Relevant Content · Fact Check

설계 원칙

  • 모델 수정 불필요 — 어떤 Markdown 출력 LLM도 평가 가능
  • 문장 단위 세밀 분석
  • 수천 개 인용 페이지로 확장 가능

재현성

  • 결정론적 AST 파서 — LLM 추론 없이 구조적 추출
  • 루브릭 기반 LLM 판사 — 인간 검토로 보정
  • 평가 기준 명시적 제약으로 편향 완화

Markdown AST 파서

동작 원리

1

Markdown → AST 변환

구조화된 문법 트리로 파싱

2

링크 노드 추출

인라인 인용 마크다운 패턴 식별

3

주장-인용 쌍 구성

각 링크를 포함하는 문장과 매핑

4

AttributionDocument 생성

구조화된 평가 입력 산출

핵심 장점

LLM 추론이 전혀 필요 없는 결정론적 추출

  • 파싱 비용 최소화
  • 추출 일관성 보장
  • 다양한 인용 형식 지원 (각주, 인라인 등)
  • 문장 단위 세밀한 귀속 분석 가능

세 가지 평가 차원

Link Works

URL 접근성 검증

인용된 URL이 실제로 접근 가능한지 HTTP 요청으로 확인. 자동

Relevant Content

주제 관련성 측정

검색된 출처 콘텐츠가 주장과 주제적으로 일치하는지 LLM 판사가 평가. LLM 판사

Fact Check

사실 정확성 검증

주장이 출처 콘텐츠에 의해 사실적으로 지지되는지 인간 보정 LLM 판사가 평가. 인간 보정

루브릭 기반 LLM-as-a-Judge 설계

개방형 품질 평가가 아닌 구체적 사실 기준으로 평가를 제약하여 위치 편향, 수다증 편향, 자기 강화 효과 등 체계적 판사 편향을 완화합니다. 인간 검토를 통한 보정 절차를 거칩니다.

실험 설정

벤치마크 대상 모델

발견 1: 표면적 품질이 사실적 실패를 가린다

평가 차원 프론티어 모델 성능 해석
Link Works 94% 이상 대부분의 URL이 정상 접근 가능
Relevant Content 80% 이상 출처가 주제와 관련성 있음
Fact Check 39~77% 사실적 지지 수준 크게 저하

링크가 작동하고 내용이 관련성이 있어도, 실제로 주장을 사실적으로 지지하지 않는 경우가 빈번합니다. 표면적 지표와 사실적 신뢰성 사이에 결정적 단절이 존재합니다.

발견 2: 인용 양과 사실 정확성의 트레이드오프

폐쇄소스 프론티어 모델

83~100%

인용 리포트 생성 성공률

오픈소스 모델

17~40%

인용 리포트 생성 성공률

핵심 시사점

  • 오픈소스 모델의 절반 이상이 원샷 설정에서 인용 리포트를 생성하지 못함
  • 리포트를 생성하더라도 인용의 사실 정확성은 프론티어 모델 대비 현저히 낮음
  • 인용량을 늘리는 것은 사실 정확성을 보장하지 않으며, 오히려 저하시킬 수 있음

발견 3: 더 많은 검색이 사실 정확성을 저하시킨다

도구 호출 수 Link Works Relevant Content Fact Check

관찰 결과

Link Works와 Relevant Content는 검색 깊이 증가에도 안정적으로 유지되지만, Fact Check는 지속적이고 큰 폭으로 저하합니다.

정보 과부하 가설

더 많은 검색이 더 나은 인용을 만드는 것이 아니라, 정보 과부하가 사실적 종합 능력을 손상시킵니다. 출처가 늘어날수록 모델이 정보를 정확하게 통합하기 어려워집니다.

오류 분석

주요 오류 유형

  • 출처 왜곡: 출처의 의미를 잘못 해석하여 주장과 불일치
  • 세부 사항 과장: 출처의 수치나 범위를 부풀리거나 축소
  • 혼합 인용: 여러 출처의 정보를 결합하여 원문에 없는 주장 생성
  • 접근 불가 URL: 링크가 작동하지 않거나 콘텐츠 없음

오류 패턴 특징

  • 출처 왜곡이 가장 빈번한 Fact Check 실패 원인
  • 검색 깊이가 깊어질수록 혼합 인용 오류 증가
  • 표면적 차원에서는 감지되지 않는 오류가 대부분
  • 사실 정확성이 인용 품질을 구별하는 가장 중요한 차원

이 오류들은 Link Works와 Relevant Content에서는 전혀 감지되지 않으므로, 사실 정확성 검증 없이 인용 품질을 평가하는 것은 심각하게 불충분합니다.

한계점

평가 방법론

  • Relevant Content와 Fact Check는 LLM-as-a-Judge에 의존하므로 본질적 편향 가능성 존재
  • 인간 보정으로 완화했으나 완전한 제거는 불가

웹 환경의 동적성

  • URL 접근성은 시간에 따라 변동 가능
  • 웹 콘텐츠 변경으로 인한 재현성 제약

리포트 형식

  • Markdown 형식에만 적용 가능
  • HTML, PDF 등 다른 형식은 추가 파서 필요

범위 제한

  • 절제 연구는 2개 프론티어 모델에만 수행
  • 특정 도메인(과학, 뉴스 등)에 대한 분석 미포함

기여 요약

최초의 종단간 출처 귀속 평가 프레임워크

결정론적 인용 추출 + 링크 접근성 · 주제 관련성 · 사실 정확성의 다차원 평가를 결합

14개 LLM 대규모 벤치마크

사실 정확성이 인용 품질을 구별하는 가장 도전적이고 차별화된 차원임을 입증

정보 과부하 효과의 실증적 증거

검색 깊이 증가가 사실 정확성을 지속적으로 저하시키면서 표면적 지표는 안정적으로 유지됨

"더 많은 검색이 더 나은 인용을 만드는 것이 아니다." — 이 연구는 LLM 리서치 종합에서 정보 과부하가 사실적 신뢰성을 훼손한다는 체계적 증거를 제시합니다.
