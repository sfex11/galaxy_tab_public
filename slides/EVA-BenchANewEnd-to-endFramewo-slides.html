방법론 · 시뮬레이션

대화 시뮬레이션과 검증 게이트

봇-투-봇 오디오 시뮬레이션

  • 사용자 시뮬레이터가 실제 오디오로 발화
  • 음성 에이전트와 실시간 오디오 교환
  • 동적 다중 턴 대화 — 단일 교환 아님
  • 억센트 · 배경 소음 섭동 적용 가능

검증 게이트 (Validation Gate)

  • 대화 유효 종료 — 시나리오 목표 달성 여부
  • 사용자 행동 충실도 — 자연스러운 대화 행동
  • 사용자 음성 충실도 — 발화 품질 검증
  • 검증 실패 시 자동 재생성

"시뮬레이터가 시나리오에서 이탈하거나 비현실적으로 행동하면, 모든 다운스트림 평가의 타당성이 훼손된다."

방법론 · 지표

EVA-A: 정확도 복합 지표

작업 완수, 충실도, 음성 수준 엔티티 충실도를 결합한 정확도 측정

작업 완료

사용자의 의도가 올바르게 해결되었는가? 인증 게이트를 통과한 후 도구 호출 결과 기반 판정

충실도 (Faithfulness)

에이전트가 정책을 위반했는가? 도구 출력과 다르게 말했는가? 적대적 요청에 응했는가?

음성 충실도

오디오 레벨에서 핵심 엔티티(확인 코드, 금액 등)가 정확히 발화되었는가? 트랜스크립트만으로는 불충분

핵심 포인트: 잘못된 확인 코드나 금액은 프로덕션에서 치명적이지만, 텍스트 수준 평가에서는 감지 불가

방법론 · 지표

EVA-X: 경험 복합 지표

대화 진행, 구어 간결성, 턴테이킹 타이밍을 결합한 사용자 경험 측정

대화 진행

턴 간에 의미 있는 진전이 있는가? 이전 질문 반복, 진전 없는 루프 등을 감지

구어 간결성

음성 전달에 적합한가? 과도한 옵션 나열로 사용자 작업 기억을 압도하는지 평가

턴테이킹

적시에 응답하는가? 인터럽트 비율과 타이밍을 측정 (τtt = 0.8 임계값)

"에이전트가 낮은 응답 지연을 달성하더라도, 턴 간에 의미 있는 진전을 만들지 못하거나 이전 질문을 반복하면 사용자 경험은 저하된다."

방법론 · 통합 지표

pass@1, pass@k, pass^k: 최고 vs 신뢰 성능

pass@1

단일 시도 성공률
최고 성능 (Peak)

pass@k

k회 중 1회 이상 성공
최적 조건 하 역량

pass^k

k회 모두 성공
신뢰 성능 (Reliable)

왜 세 가지가 모두 필요한가?

pass@1과 pass@k의 차이가 클수록 시스템의 행동 일관성이 낮음을 의미합니다. 실제 배포에서는 pass^k (모든 시도에서 성공)가 더 의미 있는 지표이지만, 대부분의 벤치마크는 pass@1만 보고합니다.

EVA-Bench 핵심 발견: EVA-A에서 중앙값 pass@k–pass^k 격차가 0.44에 달해, 최고 성능과 신뢰 성능이 실질적으로 분리됨

실험 설정

실험 환경과 범위

세 가지 기업 도메인

항공 고객 서비스 (CSM)

항공편 변경, 환불, 좌석 업그레이드 등

기업 IT 서비스 관리 (ITSM)

장애 보고, 계정 잠금 해제, 라이선스 요청

의료 HR 서비스 (HRSD)

휴가 요청, 복지 혜택, 정책 문의

평가 대상 아키텍처

  • Cascade: STT → LLM → TTS 파이프라인
  • Hybrid: 대형 오디오 언어 모델(LALM) + TTS
  • Speech-to-Speech (S2S): 엔드-투-엔드 오디오 처리

시나리오 유형

  • 단일 의도 (Single-intent)
  • 다중 의도 (Multi-intent)
  • 적대적 (Adversarial) — 정책 위반 유도
핵심 결과

주요 실험 발견

0.5

동시 달성 불가

12개 시스템 중 어느 것도 EVA-A pass@1과 EVA-X pass@1을 동시에 0.5 초과 달성하지 못함 — 정확도와 경험 간 근본적 트레이드오프 존재

0.44

최고 vs 신뢰 성능 격차

EVA-A에서 pass@k와 pass^k의 중앙값 격차가 0.44 — pass@1만 보고하는 기존 벤치마크가 성능을 과대평가함

0.314

섭동 민감도 — 억센트 및 소음

억센트 및 배경 소음 섭동 시 최대 Δ 0.314의 성능 저하 발생. 효과는 아키텍처, 시스템, 지표에 따라 비균일하게 나타남 — Cascade, Hybrid, S2S 간 취약 패턴이 상이함

섭동 분석

억센트 및 소음 견고성 분석

억센트 섭동

  • 다국적 억센트로 오디오 변환 적용
  • STT 성능 저하가 하위 파이프라인에 연쇄적 영향
  • Cascade 시스템이 S2S보다 더 큰 타격

오디오 열화 섭동

  • 배경 소음, 반향, 비대칭 채널 조건 적용
  • 지표별 민감도가 상이 — 턴테이킹은 타격 적음
  • 작업 완료와 충실도에서 가장 큰 저하

"섭동 효과는 아키텍처, 시스템, 지표에 따라 비균일하게 나타난다. 이는 단일 조건 평가로는 실제 배포 견고성을 예측할 수 없음을 시사한다."

실패 모드 분석

음성 특유의 실패 패턴

정확도 측면 실패

  • 엔티티 대체 — "A123"을 "B123"으로 발화
  • 엔티티 생략 — 확인 코드 누락
  • 엔티티 삽입 — 존재하지 않는 정보 추가
  • 음성적 혼동 — 동음이의어 오인
  • 인증 게이트 무시 후 도구 호출
  • 비활동 타임아웃으로 조기 종료

경험 측면 실패

  • 턴 간 진전 없이 동일 질문 반복
  • 과도한 옵션 나열로 사용자 압도
  • 부적절한 인터럽트 발생
  • 응답 지연으로 대화 흐름 단절

아키텍처별 패턴

Cascade는 음성적 혼동에 취약하고, S2S는 엔티티 대체에 취약한 경향을 보임

평가 신뢰성

분산 분해와 시도 횟수 정당성

분산 분해

혼합 효과 모델로 평가 점수의 분산을 세 가지 출처로 분해:

  • 시나리오 분산 — 시나리오 간 난이도 차이
  • 모델 × 시나리오 상호작용 — 특정 시나리오에서 특정 모델이 약한 현상
  • 판정기 확률성 — 동일 입력에 대한 LLM 판정기의 일관성

시도 횟수 정당성

각 시나리오별 반복 시도 횟수가 통계적으로 충분한지 분석:

  • 신뢰구간 기반 충분성 검증
  • 판정기 일관성을 인간 합의와 교차 검증
  • 시뮬레이터 검증 게이트가 에이전트 변동성과 시뮬레이터 변동성 분리
결론

기여와 핵심 시사점

네 가지 기여

  • 검증 게이트가 적용된 봇-투-봇 시뮬레이션 프레임워크
  • EVA-A / EVA-X 복합 지표 — 기존 벤치마크에서 보이지 않는 실패 모드 포착
  • 3개 도메인 213개 시나리오의 음성 특화 벤치마크 데이터셋
  • Cascade vs Audio-native 비교, 섭동 민감도, 시도 간 일관성에 대한 실증 분석

핵심 시사점

  • 정확도와 경험의 동시 만족은 현재 불가능
  • pass@1만으로는 신뢰 성능을 과대평가함
  • 섭동 견고성은 아키텍처별로 비균일 — 일반화 불가
  • 음성 수준 평가는 텍스트 평가로 대체 불가
