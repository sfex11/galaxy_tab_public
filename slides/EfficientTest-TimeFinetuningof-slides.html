테스트 시간 파인튜닝(TTFT)이란?

대규모 언어 모델은 웹 규모 코퍼스에서 학습되어 전역 최적점에 도달하지만, 특정 입력에 대해서는 지역 최적이 아닐 수 있습니다.

TTFT는 추론 시점에서 각 프롬프트마다 관련 시퀀스를 검색하고, 모델을 업데이트한 뒤 평가하는 패러다임입니다.

TTFT 파이프라인

1
검색
프롬프트 q와 관련된 N개 시퀀스 검색
2
파인튜닝
검색된 시퀀스 각각에 대해 그래디언트 스텝
3
추론
파인튜닝된 모델로 q 평가

"단 20개의 이웃만으로도 파라미터 수가 10배 이상 차이나는 모델 간의 격차를 상당히 좁힐 수 있다."

기존 방법의 한계

데이터 선택은 쿼리마다 추론 시점에서 수행되므로, 사용자 체감 지연에 직접적인 영향을 줍니다.

kNN 검색 [Sun et al., 2024]

  • FAISS 인덱스로 빠른 검색
  • 중복에 맹목 — 유사 시퀀스가 중복 선택됨
  • 그래디언트 스텝이 동일한 신호 반복

SIFT [Muennighoff et al., 2025]

  • 정보이론적 기준으로 중복 페널티
  • bpb 개선 효과 유의미
  • 탐욕적 선택으로 쿼리당 오버헤드 큼
  • 낮은 N에서 특히 비효율

핵심 문제: 속도와 품질 사이의 트레이드오프 — 빠른 검색은 중복되고, 다양성 인식 선택은 느립니다.

핵심 아이디어: 볼록 기하학적 접근

임베딩 공간에서 방향이 의미를 가집니다:

  • 서로 다른 방향의 시퀀스 → 더 넓은 의미적 특성 커버
  • 거의 같은 방향의 시퀀스 → 대부분 중복

쿼리 q를 후보 시퀀스들의 희소 볼록 결합으로 근사하면:

  • 관련성(relevance)은 자연스럽게 보장
  • 다양성(diversity)은 기하학에서 내재적으로 파생
  • 명시적 페널티나 탐욕적 휴리스틱 불필요

근사 Carathéodory 정리

q ≈ Σ wᵢ pᵢ   (wᵢ ≥ 0, Σ wᵢ = 1)

‖q − Pw‖² ≤ ε

ε-정확한 해가 O(1/ε²)개의 점으로 항상 존재하며, 앰비언트 차원과 무관합니다.

연산 비용 상각

방법 1: Frank–Wolfe 볼록 근사

쿼리 q를 K개 후보 {p₁, …, pₖ}의 볼록 결합으로 근사하는 투영 없는(projection-free) 최적화

최적화 문제

minw∈Δₖ ‖q − Pw‖²

Δₖ = 확률 심플렉스

목적함수의 그래디언트:

∇f(w) = −2Pᵀ(q − Pw)

Frank–Wolfe 알고리즘

1

그래디언트 계산 후 선형 오라클: 가장 낮은 내적을 갖는 후보 선택

2

현재 해와 선택된 점 사이를 보폭 γ만큼 이동

3

반복마다 서포트 집합이 최대 1개씩 성장 → 자연스럽게 희소

4

‖q − Pw‖² ≤ ε 이면 조기 종료

투영이 불필요하므로 심플렉스 제약 하에서도 효율적 — O(Kd) per iteration

방법 2: 기하학적 정수화

볼록 근사는 분수 가중치를 반환하지만, TTFT는 정확히 N개의 샘플이 필요합니다.

가중치 비례 패딩 (Weight-Proportional)

가중치 wᵢ를 정수 카운트 cᵢ로 변환:

cᵢ = ⌊N · wᵢ⌋ 또는 ⌈N · wᵢ⌉

Σ cᵢ = N을 만족하도록 조정

높은 가중치의 예제가 자연스럽게 여러 번 등장하는 멀티셋 생성

정수화의 중요한 부수 효과

분수 가중치 → 정수 카운트 변환 시:

  • 일부 예제가 멀티셋에서 다중 등장
  • 이 반복 구조가 Gradient Reuse를 가능하게 함
  • 기하학적 근사 품질을 최대한 보존

대안: FW 계열 기하학적 정수화 (Frank–Wolfe 가중치에 대한 기하학적 충실도 더 높음)

방법 3: Gradient Reuse

정수화로 생성된 반복 예제를 활용하여 순방향–역방향 연산을 상각합니다.

일반적인 TTFT

N개 예제 각각에 대해 독립적으로 forward–backward:

F+B₁ F+B₂ F+B₃ ... F+Bₙ

총 N회 forward–backward

HullFT Gradient Reuse

동일 예제의 연속 등장 시 그래디언트 재사용:

F+B₁ 재사용₁ F+B₂ F+B₃ 재사용₃ 재사용₃

총 |S|회 forward–backward (|S| ≪ N)

정수화가 단순히 기하학을 TTFT에 맞게 변환하는 것을 넘어, 연산 효율화의 구조적 기반을 제공합니다.

세 가지 기여 요약

볼록 근사 기반 선택

kNN 후보 풀 위에서 Frank–Wolfe로 희소 볼록 근사 → 관련성과 다양성을 기하학에서 내재적 획득

Alg. 3

기하학적 정수화

분수 가중치를 N개 정수 멀티셋으로 변환하면서 근사 품질 보존, 반복 구조 자연 생성

Alg. 1

Gradient Reuse

멀티셋의 반복 예제에서 캐시된 그래디언트 재사용 → 파인튜닝 연산 비용 상각

Alg. 4

세 가지 아이디어가 상호 의존적으로 결합: 볼록 근사 → 정수화 → 반복 구조 → Gradient Reuse

실험 설정

벤치마크

The Pile의 12개 서브셋에서 평가

평가 지표

  • Bits-per-byte (bpb) — 낮을수록 우수
  • 총 런타임 — 선택 + 파인튜닝 + 추론
  • 품질–효율 트레이드오프 곡선

비교 대상

방법 선택 전략 다양성
kNN 최근접 이웃 없음
SIFT 정보이론적 탐욕 명시적
HullFT 볼록 근사 (FW) 내재적

파인튜닝 예산 N ∈ {5, 10, 20, 50}에서 비교

주요 실험 결과

HullFT는 12개 서브셋 모두에서 비교 방법을 엄격히 우위하는 품질–효율 트레이드오프 달성

품질 측면

  • kNN 대비 일관된 bpb 개선
  • SIFT와 동등하거나 우수한 품질
  • 특히 낮은 N(5, 10)에서 개선 폭 큼

효율 측면

  • SIFT 대비 상당히 낮은 총 런타임
  • Gradient Reuse로 파인튜닝 단계 가속
  • 지연 민감 구간에서 가장 큰 이점

핵심 발견: HullFT는 품질을 유지 또는 향상하면서 런타임을 크게 단축하여, 기존 방법들이 타협해야 했던 트레이드오프를 개선합니다.

추가 분석: 선택 기하학

t-SNE 시각화에서의 차이

  • kNN: 선택된 예제가 임베딩 공간에서 국소적으로 클러스터링
  • HullFT: 서포트 집합이 쿼리 주변에 넓게 분산
  • 볼록 근사가 의미적으로 다양한 방향을 자동 커버

Frank–Wolfe 파라미터 민감도

  • 허용오차 ε: 작을수록 더 정확한 근사, 더 많은 서포트 포인트
  • kNN 풀 크기 K: 충분히 크면 성능 포화, 작으면 후보 부족
  • 실용적 범위에서 robust한 성능 확인

정수화 방식 비교: FW 기반 기하학적 정수화가 가중치 비례 패딩보다 FW 가중치에 대한 충실도가 더 높음을 확인 (부록 C.7)

결론 및 한계

결론

  • TTFT의 두 병목(선택 + 파인튜닝)을 기하학적 관점에서 동시 해결
  • 볼록 근사가 관련성과 다양성을 단일 최적화에서 내재적 획득
  • 정수화가 기하학과 TTFT를 연결하며 Gradient Reuse의 구조적 기반 제공
  • The Pile 12개 서브셋에서 SOTA 대비 엄격한 우위

한계

  • 임베딩 품질에 의존 — 임베딩이 의미를 잘 담지 못하면 볼록 근사의 이점 감소
  • Frank–Wolfe가 국소 최적해에 도달 가능 — 글로벌 최적 보장 아님
  • 대규모 코퍼스에서 kNN 풀 구축 비용은 여전히 존재
  • Generation task가 아닌 perplexity 기반 평가에 집중
