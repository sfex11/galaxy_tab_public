EMO

Pretraining Mixture of Experts for
Emergent Modularity

Ryan Wang, Akshita Bhagia, Sewon Min

UC Berkeley · Allen Institute for AI

문제 정의

단일체(monolithic) 배포의 한계

  • 코드·수학·특정 도메인 등 일부 능력만 필요한 경우에도 전체 모델 로드
  • 불필요한 연산 비용과 메모리 소모
  • 특정 기능만 개선·업데이트 불가

기존 MoE의 현실적 한계

  • 토큰마다 다른 전문가를 활성화 → 결국 대부분의 전문가 사용
  • 전문가 부분 집합으로 제한 시 성능 급감
  • 모델이 커지고 희소해질수록 문제 심화
MoE는 모듈식 배포를 약속하지만, 실제로는 여전히 전체 모델이 필요한 단일체이다.

기존 MoE의 전문가 특화 분석

기존 MoE의 특화 양상

  • 저수준 구문적(syntactic) 패턴에 특화
  • 전치사, 구두점, 토큰 ID 등 표면적 패턴
  • 소수 전문가에만 특화가 집중됨
  • 의미적 도메인 구분 불가

EMO가 달성하는 특화

  • 고수준 의미적(semantic) 패턴에 특화
  • 수학, 코드, 생물학 등 도메인 수준
  • 전문가 부분 집합의 독립적 사용 가능
  • 인간 직관과 일치하는 활성화 패턴

이러한 질적 차이가 모듈성(modularity)의 근본 원인이다.

EMO의 핵심 아이디어

유사한 도메인의 토큰은 유사한 전문가 부분 집합을 활성화하도록 유도하라.
1
가정
동일 문서 내 토큰은 동일 도메인에 속할 가능성이 높음
2
제약
문서 내 모든 토큰이 공유 전문가 풀에서만 선택
3
결과
문서 간 다른 풀 사용 → 의미적 전문가 군집 자발적 형성

  사전 정의된 도메인 라벨 불필요 — 문서 경계만으로 자기지도 학습

EMO 아키텍처

기본 MoE 구조

  • 총 128개 전문가, 토큰당 8개 활성화
  • Top-k 라우팅 기반 게이트
  • 1B 활성 / 14B 총 파라미터

EMO의 추가 제약

  • 문서별 공유 전문가 풀(d=32) 설정
  • 해당 문서의 모든 토큰은 풀 내에서만 k개 선택
  • 문서 간에는 서로 다른 풀 사용 가능
문서 경계로 토큰 그룹화
문서당 전문가 풀 할당
풀 내에서 Top-k 라우팅

주요 기술적 고려사항

고려 1: 부하 균형 (Load Balancing)

전문가 풀 제약으로 인해 특정 전문가 과부하 위험

  • 보조 손실(auxiliary loss)로 균형 유지
  • 문서 경계 기반 그룹화가 자연스럽게 분산 촉진
  • 표준 MoE와 동일한 균형 메커니즘 적용 가능

고려 2: 전문가 풀 크기 (d) 선택

d=32 (총 128개 중)로 설정

  • d가 너무 작으면 모델 표현력 제한
  • d가 너무 크면 표준 MoE와 유사해져 모듈성 약화
  • 총 전문가의 25% 수준이 적절한 스위트스팟

  공유 전문가(shared experts) 아키텍처도 검토하였으나, 문서 수준 풀 제약이 모듈성 유도에 더 효과적임

평가 방식

  • 전체 모델 평가: MMLU, MMLU-Pro 등 벤치마크
  • 선택적 전문가 사용: 도메인별 전문가 부분 집합만 로드하여 평가
  • 메모리 매칭 모델(동일 파라미터 크기로 처음부터 학습)과 비교
  • 다양한 전문가 선택 방식에 대한 강건성 검증

표준 MoE는 전문가를 줄이면 급격히 성능이 붕괴하지만, EMO는 메모리-정확도 파레토 프론티어를 획기적으로 개선한다.

전문가 선택 방식에 대한 강건성

EMO는 다양한 전문가 부분 집합 선택 방식에서 일관된 성능을 보임

검증된 선택 방식

  • 검증 데이터 기반 빈도수 선택
  • 토큰 클러스터링 기반 선택
  • 무작위 선택 (baseline)
  • 순차적 선택

추가 발견

  • 소수의 검증 데이터로 효율적으로 전문가 선택 가능
  • 표준 MoE에 어닐링으로 모듈성 부여 시도 → 실패
  • 사전학습부터 모듈성을 목표로 해야 함

의미적 특화의 자발적 출현

토큰 클러스터링 분석

전문가 활성화 패턴 기반 토큰 클러스터링 결과

  • 클러스터가 의미적 주제와 정렬
  • 수학, 코드, 생물학, 사회과학 등 도메인별 군집 형성
  • 문서 첫 토큰이 특별한 클러스터 형성

도메인별 활성화 패턴

MMLU 카테고리별 전문가 활성화 분석

  • 도메인 간 활성화 패턴이 뚜렷하게 구분
  • 유사 도메인(예: 물리-수학)은 겹치는 전문가 사용
  • 인간 직관과 일치하는 계층적 구조
라벨 없이 문서 경계만으로 도메인 수준의 전문가 특화가 자발적으로 출현함

결론

  • 모듈성을 1순위 목표로 한 MoE 사전학습 방법 EMO 제안
  • 문서 경계만으로 의미적 전문가 특화 자발적 출현
  • 전체 모델 성능 유지하면서 선택적 전문가 사용 가능
  • 25% 전문가만으로 1% 감소, 메모리 매칭 모델 상회
  • 모듈식, 메모리 효율적 LLM 배포로의 새로운 경로 개척
