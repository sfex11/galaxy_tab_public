Self-Improving Language Models with Bidirectional Evolutionary Search

Guowei Xu, Zhenting Qi, Huangyuan Su, Weirui Ye
Himabindu Lakkaraju, Sham M. Kakade, Yilun Du

Harvard University · MIT

왜 탐색(Search)이 중요한가?

학습 시간

고품질 샘플은 사후 학습(Post-training)과 자기 개선의 효율을 결정적으로 좌우한다.

추론 시간

테스트 시간 스케일링(Test-time Scaling)의 핵심 메커니즘으로, 모델의 한계를 넘어설 수 있다.

모델 능력의 프론티어에서는 단순한 샘플링으로 올바른 답을 얻기 위해 너무 많은 샘플이 필요하거나, 아예 실패할 수 있다.

기존 방법의 근본적 한계

한계 1

희소한 검증 신호

Best-of-N, Tree Search 모두 이진(binary) 또는 거친 피드백에만 의존한다.

RLVR 사후 학습 등에서 검증기가 제공하는 정보량이 극히 제한적이다.

한계 2

모델 분포 내 갇힘

자기회귀적 확장만으로 후보를 생성하므로, 모델의 확률 질량이 작은 영역에 도달 불가능하다.

난문제에서 올바른 해답은 종종 낮은 확률 영역에 존재한다.

핵심 통찰: 진화생물학에서의 유추

무성 생식 (기존 방법)

자손은 부모의 직접적인 연장일 뿐이다. 서로 다른 개체에서 독립적으로 발생한 유익한 변이를 결합할 수 없다.

유성 생식 (BES)

염색체 재조합을 통해 서로 다른 계통의 유전자 세그먼트를 결합하여 어느 부모도 가지지 못한 새로운 조합을 생성한다.

단일 모델 롤아웃으로는 얻기 어려운 후보를, 서로 다른 궤적의 부분들을 재결합하여 생성하자.

BES: 양방향 진화 탐색 프레임워크

F
순방향 탐색
진화 연산자로 후보 진화
B
역방향 탐색
목표 분해로 조밀한 피드백
최종 해답
y* 도달
  • 순방향: 단순 확장뿐 아니라 궤적 재결합으로 모델 분포 밖의 후보 생성
  • 역방향: 원래 작업을 검증 가능한 하위 목표로 재귀적 분해
  • 몇 번의 순방향 탐색 후 역방향 탐색 1회 수행하여 교대 진행

순방향 탐색: 5가지 진화 연산자

확장

정책이 새로운 단계를 자기회귀적으로 생성

결합

공통 접두사를 공유하는 두 경로의 접미사 연결

삭제

궤적 내에서 가장 취약한 내부 단계 제거

전위 (Translocation)

경로 A의 한 단계를 경로 B의 해당 단계로 교체

교차 (Crossover)

경로 A의 접합점 이후를 경로 B의 꼬리로 대체

역방향 탐색: 목표 분해를 통한 조밀한 피드백

재귀적 목표 트리 분해

원래 작업을 검증 가능한 하위 목표들로 계층적으로 분해한다.

각 하위 목표는 개별적으로 만족 여부를 확인할 수 있다.

희소 → 조밀 신호

최종 답안에 대한 이진 피드백 대신, 각 중간 단계에 대한 세밀한 점수를 제공한다.

순방향 탐색이 어떤 경로를 우선 탐색할지 방향을 제시한다.

핵심 효과: 역방향 탐색이 필요한 샘플 수를 지수적으로 감소시킨다고 이론적으로 증명

이론적 동기: 엔트로피 껍질 탈출

정리 4.4(a)

확장 전용 탐색의 한계

자기회귀적 확장만으로 생성된 모든 후보는 좁은 엔트로피 껍질(Entropy Shell) 내에 갇힌다.

탐색 범위가 해 공간의 아주 작은 영역으로 제한된다.

정리 4.4(b)

진화 연산자의 탈출

결합, 전위, 교차 등의 진화 연산자가 엔트로피 껍질을 벗어나 새로운 영역을 탐색할 수 있게 한다.

단일 롤아웃으로는 도달 불가능한 후보를 생성한다.

세 가지 가정(유한 놀람, 감쇠 의존성, 선형 블록 상관관계) 하에, 진화 연산자가 껍질 탈출을 수학적으로 보장한다.

실험: 사후 학습 (Post-Training)

기존 사후 학습 알고리즘이 개선을 실패하거나 오히려 성능이 저하되는 도전적 과제에서 평가

논리 추론

GRPO, MaxRL, Tree-GRPO가 충분한 고품질 학습 샘플을 찾지 못해 기저 모델보다 저하

BES는 일관되게 유효한 학습 샘플을 발견하여 유의미한 성능 향상 달성

다중 홉 추론

여러 단계의 추론이 필요한 복잡한 질문에서 동일한 패턴 관찰

베이스라인이 거의 실패하는 상황에서 BES만이 안정적 개선

핵심 원인: 진화 연산자가 고품질 샘플의 커버리지를 극대화하고, 역방향 탐색이 효율적인 선별을 가능하게 함

실험: 추론 시간 오픈 문제 해결

3개의 오픈 문제 해결 벤치마크에서 기존 오픈소스 프레임워크와 비교

방법 평균 성능 최고 성능 안정성

벤치마크: 원 패킹(Circle Packing), 하일브론(Hailbronn) 등 수학적 오픈 문제

절제 연구 및 비용 분석

절제 연구 결과

  • 진화 연산자 제거 시 성능 급감
  • 역방향 탐색 제거 시 효율성 저하
  • 결합, 전위, 교차 각각이 독립적 기여
  • 양방향 결합이 단방향보다 우월

비용 분석

소요 시간(Wall-clock) 및 API 비용 측면에서 베이스라인과 비교 분석.

진화 연산자의 추가 계산 비용은 샘플 효율성 향상으로 상쇄되며, 동일 예산 내에서 더 높은 품질의 해답을 발견한다.

결론 및 핵심 요약

1
기존 탐색 방법(Best-of-N, Tree Search)은 희소 신호와 모델 분포 갇힘이라는 두 가지 근본적 한계를 가진다.
2
BES는 순방향 진화 탐색(궤적 재결합)과 역방향 목표 분해(조밀 피드백)를 결합하여 이를 해결한다.
3
이론적으로 확장 전용 탐색의 엔트로피 껍질 갇힘을 증명하고, 진화 연산자의 탈출 가능성을 보였다.
4
사후 학습과 추론 시간 모두에서, 기존 방법이 실패하는 도전적 과제에서 일관된 성능 향상을 달성했다.
