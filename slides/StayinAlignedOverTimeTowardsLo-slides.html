시간이 지나도 정렬을 유지하는법

맥락적 반성과 프라이버시 보호 행동 데이터를 통한
장기적 Human-LLM 정렬 방법론

Simret Araya Gebreegziabher · Allison E Sproul · Yinuo Yang · Chaoran Chen · Diego Gómez-Zará · Toby Jia-Jun Li

문제 정의

현재 정렬 방법의 시간적 한계

기존 RLHF 접근법

  • 상호작용 직후의 선호도 수집
  • 선호를 정적으로 암묵적으로 가정
  • 단기 만족에 과적합 위험
  • 실제 결과 반영 불가

현실의 LLM 사용

  • 제품 선택, 여행 계획, 중요 이메일 작성
  • 결과는 시간이 지나야 드러남
  • 경험 후 재평가 발생
  • 선호는 맥락에서 구성됨

"기존 정렬 방법은 시간적으로 얕다 — 단일 순간의 피드백에만 기반하여, 사용자가 모델 출력을 실제로 행동으로 옮긴 후의 평가는 포착하지 못한다."

핵심 통찰

선호는 고정된 것이 아니라 시간에 따라 구성된다

1

즉시 판단
가용한 단서에 의존
(유창성, 일관성)

2

행동 전개
결과 경험, 기회비용 확인
추가 정보 획득

3

재평가
초기 판단과 달라질 수 있음
장기 목표와 정렬 여부

심리학·행동경제학 연구에 따르면, 순간적 평가는 문맥 의존적 구성물이며,
새로운 정보나 실제 경험 후에는 체계적으로 달라질 수 있다.
단일 순간의 선호도 데이터만으로는 실제 정렬을 과대평가할 위험이 있다.

설계 목표

4가지 설계 원칙

DG1
시간에 따른 반성 포착

LLM 상호작용에 대한 사용자의 지연된 재평가를 측정할 수 있어야 함

DG2
사용자 부담 없는 맥락 인식 반성

추가적인 부담 없이, 관련 결과가 발생했을 때만 반성을 유도해야 함

DG3
점진적·맥락적 동의 지원

행동 데이터 공유에 대해 사용자가 통제 가능한 동의 모델 제공

DG4
자연 환경에서의 LLM 사용 연구

실험실이 아닌 일상적 맥락에서 LLM 사용을 관찰해야 함

시스템

BITE 시스템 개요

1
LLM 대화
사용자가 LLM과 대화 수행
2
초기 평가
즉시 선호도 캡처
3
분류·관찰
대화 분류 → 옵저버 생성 → 하류 행동 추적
4
후속 평가
결과 감지 시 맥락과 함께 재평가 유도

BITE는 브라우저 확장 기반 시스템으로, LLM 상호작용의 분류 → 하류 행동 모니터링 → 결과 인식 후속 반성의 전체 파이프라인을 자동화한다.

시스템

BITE의 핵심 기능

Feature 1

이벤트 트리거 반성 프롬프트

대화를 의미 있는 카테고리(제품, 여행, 건강, 금융 등)로 자동 분류하고, 관련 웹 행동을 감지하면 후속 평가를 트리거

Feature 2

결과 인식 후속 반성

단순히 시간이 지난 후가 아니라, 사용자가 실제 결과를 경험한 시점에 맥락 데이터와 함께 재평가 요청

Feature 3

점진적 동의 모델

브라우저 히스토리 접근에 대해 대화 단위로 세분화된 동의를 요청하고, 사용자가 언제든 철회 가능

핵심 차별점: 시간 경과만이 아닌 결과 경험을 기준으로 재평가를 트리거하여, 반성의 의미성을 극대화

연구 설계

2주간 In-the-Wild 배포 연구

연구 구성

  • 기간: 2주
  • 참여자: 8명 (다양한 LLM 사용 패턴)
  • 캡처된 대화: 182건
  • 사후 인터뷰 진행

평가 차원

  • 정확성 (Accuracy)
  • 관련성 (Relevance)
  • 신뢰 (Trust)
  • 유해성 (Harmfulness)
  • 유용성 (Usefulness)

참여자는 자유롭게 일상에서 LLM을 사용하며, BITE가 대화를 자동 감지하고 맥락에 따라 후속 반성을 유도

발견 1

즉시 판단과 지연 판단의 발산

평가 차원별로 즉시 평가와 후속 평가 간의 변화 정도가 달랐다.

평가 차원 발산 정도 해석
정확성 가장 큰 발산 실제로 행동한 후 검증 결과가 반영됨
관련성 큰 발산 하류 맥락에서 제안의 실제 적합성 재평가
유용성 보통 부분적 변화
신뢰 방향적 비대칭 대체로 상향 수정 (80% 증가 vs 20% 감소)
유해성 반대 방향 시간이 지나며 유해성 인식 증가 경향

신뢰와 반대 방향의 비대칭

"사용자는 실제 결과를 경험한 후, LLM의 유용성에 더 신뢰를 갖는 반면, 잠재적 유해성에 대해서는 더 비판적으로 재평가한다."

발견 3

반성은 의미 구성 과정이다

선호 수정은 시간 경과 자체가 아닌, 결과 검증과 맥락 재구성에 의해驱动되었다.

결과 검증 패턴

  • 외부 소스에서 LLM 답변 교차 검증
  • 추천 옵션을 실제로 비교해 본 후 재평가
  • 행동 결과가 기대와 달랐을 때 정확성 재평가

탐색 행동 = 맥락

  • 브라우저 히스토리가 재평가의 맥락으로 기능
  • 어떤 페이지를 방문했는지가 판단에 영향
  • 탐색 깊이와 재평가 강도의 연관성

핵심: 행동 추적 데이터 없이는 선호 변화의 원인을 해석할 수 없다 — 단순 재평가만으로는 "왜" 바뀌었는지 알 수 없음

논의

시야를 넓히기: 장기 정렬의 함의

정렬의 재개념화

Human-AI 정렬을 정적 라벨링 작업이 아닌 시간적으로 전개되는 과정으로 재개념화해야 한다. 즉시 피드백만 최적화하면 사용자가 궁극적으로 적용하는 기준을 놓친다.

반성 중심 설계

사용자가 행동으로 옮긴 후의 반성-in-action을 정렬 데이터에 통합하는 설계 패러다임이 필요하다. 결과 인식 트리거가 핵심 메커니즘이 될 수 있다.

프라이버시 인식 정렬 데이터 수집

행동 데이터는 필수적이지만 프라이버시 침해 위험이 크다. BITE의 점진적 동의 모델은 맥락별 세분화된 통제를 통해 생태적 타당성과 프라이버시 보호 사이의 긴장을 완화하는 하나의 방향을 제시한다.

한계와 향후 과제

인정할 한계, 나아갈 방향

현재 한계

  • 8명 참여자로 대표성 제한
  • 브라우저 내 행동만 추적 가능
  • 모바일 환경 미지원
  • 2주라는 비교적 짧은 기간
  • 특정 LLM 플랫폼에 종속

향후 방향

  • 대규모 장기 연구 확장
  • 다양한 플랫폼·디바이스 지원
  • 지연 피드백을 RLHF 파이프라인에 통합
  • 선호 변화 예측 모델 개발
  • 차등 프라이버시 등 강화된 보호 기법 적용
