01 문제 정의 동기

🛠

GUI 에이전트의 보상 함수 문제

강화학습(RL)은 GUI 에이전트의 견고성을 향상시킬 수 있으나, 학습 품질은 보상 함수의 정확도에 극도로 민감합니다. 부정확한 보상은 정책 경사(policy gradient)의 품질을 저하시킵니다.

⚖

기존 접근법의 한계

규칙 기반: 새로운 태스크마다 수동 설계 필요 → 확장성 부족
단일 VLM 판정: 전체 궤적을 한 번에 평가 → 증거 희석(evidence dilution) 발생
DigiRL/ZeroGUI: 확장성과 정밀도 사이의 트레이드오프 존재

🎯

핵심 과제

확장성(Scalability)과 정확도(Accuracy)를 동시에 달성하는 보상 모델이 필요합니다. 특히 RL에서는 정밀도(Precision)가 재현율(Recall)보다 중요합니다 — 거짓 양성이 학습을 크게 해칩니다.

03 프레임워크 아키텍처

🔍 마일스톤 검증 모듈 (MVM)

Selector 에이전트

궤적 τ = {(st, at, mt)} 에서
마일스톤 후보 M = {(ti, di, ri)} 를 추출
→ 단계 인덱스, 상태 변화 설명, 필요성 근거

➔

Verifier 에이전트

각 마일스톤에 대해 이진 검증 수행
vi ∈ {0, 1} + 피드백 fi
→ 사전/사후 상태, 액션, 메타데이터 활용

↻ 반복 개선 루프

⚖ 판정 교정 모듈 (VCM)

Reviewer 에이전트

증거 체인의 완전성/적절성 감사:
✓ 누락된 핵심 마일스톤 식별
✓ 과도하게 관대한 기준 탐지
✓ 포착되지 않은 실패 모드 발견
→ M0 → M1 반복 개선

➔

Judge 에이전트

모든 정보를 종합하여 최종 보상 산출:
r = J(I, τ, {M0,...,M}, V, F)
→ 전체 심의 이력 고려, 이진 보상 r ∈ {0,1}

💡 핵심 설계 원칙

정밀도 우선 전략: RL에서 재현율(Recall) 감소를 감수하더라도 정밀도(Precision) 향상이 더 나은 경사 품질을 제공

11 학습 세부사항

⚙ RL 학습 구성

  • 알고리즘: GRPO (Verl 프레임워크)
  • 학습률: 1 × 10-6
  • 가중치 감쇠: 1 × 10-2
  • 경사 클리핑: 1.0
  • 샘플링 온도: 1.0
  • 궤적 후보: 상태당 n=4
  • 최대 궤적 길이: 50 스텝
  • 학습 에피소드: 4회

📊 스케일링 구성

  • 학습 태스크: 96개 (기본) / 1,024개 (스케일)
  • 검증 세트: 64개 (기본) / 128개 (스케일)
  • 궤적 롤아웃: 태스크당 4회
  • KL 정규화: low_var_kl, coef=0.005
  • 옵티마이저: AdamW
  • 태스크 생성: Qwen3-VL-235B (AndroidGen 방법론)

12 한계점 및 향후 과제

⚠ 온라인 RL 스케일링 제약

가상 환경 프로비저닝을 위한 하드웨어 한계, 환경 조율 효율성 부족, 불완전한 초기화 파이프라인이 대규모 실험을 제약

🔬 보상 세분화 미탐구

마일스톤별 세밀한 보상, 고급 합성 전략, 프로세스 수준 증거 합성 등 프레임워크의 풍부한 감독 능력이 아직 충분히 탐구되지 않음

🤖 VLM 의존 위험

암묵적 의미 이해에 의존 → 시맨틱 보상 해킹 가능성. 정책이 VLM 추론 약점을 악용하는 적대적 시각 상태를 발견할 수 있음

🚫 편향 전파 우려

VLM의 고유 편향을 상속하며, 다중 에이전트 아키텍처가 이를 완화하지만 완전히 제거하지는 못함. 접근성 지향 또는 비표준 GUI 레이아웃에 대한 체계적 불이익 위험
