You Only Index Once:
Cross-Layer Sparse Attention
with Shared Routing

Yutao Sun1,2 · Yanqi Zhang1 · Li Dong1 · Jianyong Wang2 · Furu Wei1
1Microsoft Research · 2Tsinghua University
단일 라우팅 인덱스를 계산하여 여러 레이어에 걸쳐 재사용함으로써,
토큰 희소 주의력의 정밀도를 유지하면서 디코딩 오버헤드를 획기적으로 감소시킨다.

문제 정의: 긴 문맥 추론의 병목

최신 LLM에서 긴 문맥 추론은 점차 디코딩 효율성에 의해 제약받고 있다. 특히 추론 중심 설정(Chain-of-Thought, Test-time Scaling)에서 모델은 긴 중간 추론 사슬을 생성하며 반복적으로 큰 문맥에 주의를 기울여야 한다.
병목 1
프리필링 비용
문맥 길이 증가에 따른 입력 처리 비용 급증
병목 2
KV 캐시 저장
128K 토큰에서 메모리 압력이 심각해짐
병목 3
긴 문맥 디코딩
매 토큰 생성 시 전체 캐시 스캔 필요

기존 희소 주의력 방법의 한계

방법 가속 효과 품질 핵심 한계
블록 희소 주의력 강한 가속 품질 손실 큼 구조적 희소성이 GPU 실행에 유리하나, 거친 근사로 인한 정확도 하락
토큰 희소 주의력 가속 제한적 높은 정확도 전체 캐시에 대한 top-k 라우팅이 비용이 크고, 레이어마다 재계산
토큰 희소 주의력은 정밀한 선택 능력을 가지지만, 매 레이어에서 독립적으로 top-k 라우팅을 수행하는 것이 실제 엔드투엔드 가속을 제한하는 핵심 병목이다.

핵심 아이디어: 인덱스도 공유하자

YOCO 아키텍처에서 여러 크로스-디코더 레이어가 동일한 KV 캐시를 읽는다는 점에 착안한다. KV 캐시를 공유하는데, 라우팅 인덱스는 왜 매 레이어마다 따로 계산하는가?
기존 YOCO (Dense)
  • KV 캐시: 공유됨
  • 라우팅 인덱스: 레이어별 독립 계산
  • 크로스-어텐션: 전체 캐시 밀집 검색
CLSA (제안)
  • KV 캐시: 공유됨
  • 라우팅 인덱스: 단일 계산 후 전체 레이어 공유
  • 크로스-어텐션: 선택된 토큰만 희소 검색

CLSA 아키텍처 전체 구조

1
Self-Decoder
입력 시퀀스를 공유 은닉 상태 H로 인코딩하고, 단일 KV 캐시 구축
2
Shared Indexer
H에서 Q_idx, K_idx 계산 → top-k 라우팅 인덱스 단일 계산
3
Cross-Decoder ×L
각 레이어가 공유 KV + 공유 인덱스로 희소 크로스-어텐션 수행
Self-Decoder의 역할
YOCO와 동일하게 효율적인 어텐션으로 공유 은닉 상태를 생성. 프리필링과 KV 캐시 저장의 효율성을 그대로 계승.
Cross-Decoder의 역할
각 레이어는 자체 Query 상태와 FFN을 유지하면서, 선택된 k개 토큰에만 어텐션을 수행하여 디코딩 가속.

크로스-레이어 희소 주의력 수식

공유 은닉 상태 H ∈ ℝn×d와 공유 KV 캐시 K, V ∈ ℝn×d가 주어졌을 때:
1
인덱스 질의/키 프로젝션 (단일 헤드)
Q_idx = H·W_idx^Q,   K_idx = H·W_idx^K   (단일 헤드, d_idx 차원)
2
라우팅 인덱스 계산 (한 번만!)
I = Q_idx · K_idx^T,   S_t = TopK(I_t, k)   → 각 쿼리 토큰별 k개 선택
3
희소 크로스-어텐션 (모든 레이어 l에 대해 동일한 S_t 사용)
O_t^(l) = Attn(Q_t^(l), K_{S_t}, V_{S_t})   → k ≪ n 이므로 메모리 접근大幅 감소
핵심: 라우팅 인덱스를 공유 KV 캐시에 바인딩함. 여러 레이어가 같은 메모리를 읽으면, 같은 라우팅 결정도 공유해야 함.

다중 레이어 증류 학습

공유 인덱서가 모든 레이어에 동시에 유용한 토큰을 선택하도록 훈련해야 한다. 밀집 어텐션의 합의된 중요 토큰을 타겟으로 사용.
타겟 어텐션 (모든 레이어·헤드 평균):
Ā = (1/LH) Σ_l Σ_h softmax(Q^(l,h) · K^(h)T)
KD 손실:
L_KD = (1/n) Σ_t KL(sg[Ā_t] ∥ softmax(I_t))
Stage 1 인덱서 웜업
백본 파라미터 고정, L_KD만으로 인덱서 안정적 라우팅 패턴 학습
Stage 2 결합 희소 적응
L_LM + 0.1·L_KD로 백본이 희소 어텐션 분포에 적응하도록 파인튜닝

추론 효율성: 세 가지 병목 동시 해결

추론 단계 Dense Transformer 블록 희소 토큰 희소 (기존) CLSA (제안)
프리필링 O(N²LD) O(NW₁LD) O(N²LD) + 라우팅 YOCO 효율 계승
KV 캐시 저장 O(NLD) O(NLD) O(NLD) YOCO 효율 계승
디코딩 (토큰당) O(NLD) O(W₁LD) O(kLD) + N·L 라우팅 O(kLD) + N 라우팅 (1회)
핵심 차이: 기존 토큰 희소는 라우팅 비용이 O(N·L)이지만, CLSA는 O(N)으로 L배 감소. 이 오버헤드 amortization이 실제 가속의 핵심.

실험 결과: 모델 품질 유지

짧은 문맥과 긴 문맥 벤치마크 모두에서 밀집 베이스라인에 거의 무손실인 성능을 달성.
일반 벤치마크 (짧은 문맥)
  • MMLU, ARC, HellaSwag 등 다영역 평가
  • Dense YOCO 대비 품질 손실 무시할 수준
  • 블록 희소 방법 대비 일관되게 우수
긴 문맥 벤치마크
  • 128K 문맥에서 LongBench, RULER 등 평가
  • 긴 문맥에서도 품질 유지 확인
  • 희소성에도 불구하고 핵심 정보 검색 능력 보존
어텐션 패턴 분석 결과: 하나의 라우팅 인덱스를 레이어 간에 공유해도 어텐션 거동에 미미한 영향만 있음을 확인.

문맥 길이가 길어질수록 가속 효과가 더 커진다. 라우팅 오버헤드 amortization과 희소 검색의 이점이 문맥 길이에 비례하여 증가하기 때문. 기존 토큰 희소 방법은 레이어별 라우팅 재계산으로 인해 이 스케일링 이점을 충분히 활용하지 못함.

어텐션 희소성 분석

공유 인덱서가 실제로 어떤 토큰을 선택하는지, 그리고 이것이 레이어 간 합의와 얼마나 일치하는지 분석.
주요 발견
  • 레이어 간 어텐션 분포가 높은 합의를 보임 → 공유 인덱스의 근거 확인
  • 선택된 토큰이 실제로 문맥적으로 중요한 위치에 집중
  • k 값 변화에 따른 품질-효율 트레이드오프가 완만한 곡선 형태
설계 가정 검증
"하나의 라우팅 인덱스를 여러 레이어가 공유해도 문제없다"는 핵심 가정이 어텐션 패턴 분석을 통해 실험적으로 지지됨. 레이어별 독립 라우팅과 비교했을 때 어텐션 거동 차이가 미미함.

관련 연구와의 차별성

희소 주의력 연구
  • Longformer, BigBird: 고정 패턴, 적응성 부족
  • FlashAttention-2/3: 하드웨어 최적화, 구조적 희소성 없음
  • DaBa, MoBA: 블록 희소, 품질 손실 존재
  • CLSA: 토큰 희소 + 라우팅 공유로 두 장점 결합
하이브리드 아키텍처
  • YOCO: KV 공유로 프리필링·캐시 효율화, 디코딩은 밀집
  • Cross-Attention 기반 모델들: 각 레이어 독립 연산
  • CLSA: YOCO 위에 희소성을 추가하여 디코딩까지 효율화

결론 및 핵심 기여

CLSA는 "메모리를 공유하면 라우팅도 공유하라"는 단순하면서 강력한 설계 원칙을 제시한다.
  • 통합적 효율 해결책: 프리필링 + KV 캐시 + 디코딩 세 병목을 동시 개선
  • 품질-효율 양립: 거의 무손실 품질로 최대 17.1× 전체 처리량 향상
  • 설계 단순성: 단일 인덱서 추가만으로 기존 YOCO에 적용 가능
  • 확장성: 문맥 길이가 길어질수록 가속 효과 증가
