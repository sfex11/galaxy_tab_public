세 가지 장벽

조기 중단을 실현하기 위해 넘어야 할 과제들

1

초기 신호의 부재

행동 기반 스코어러는 첫 라운드에서 거의 무작위 수준이며, 3~4라운드에야 유의미해짐 — 그때는 이미 유용한 연산량의 대부분이 소진됨

2

배포 가능한 보장의 부재

성공할 에피소드를 잘못 중단하면 보상이 파괴됨. 배포 전에 위조 중단율의 상한을 바인딩해야 함

3

순차적 위험 누적

각 게이트의 위조 중단 확률이 낮아도, 성공 궤적은 모든 게이트를 통과해야 하므로 에피소드 수준 보장이 불가능

핵심 발견: 내부 표현이 실패를 예측한다

에이전트의 숨겨진 활성화(hidden activations)에서 추출한 경량 probe가 행동 기반 스코어러보다 훨씬 일찍 실패를 감지

라운드 행동 기반 스코어러 내부 활성화 Probe
Round 1 거의 무작위 행동 스코어러의 최고점에 이미 도달
Round 2 점진적 개선 probe 자체의 최고점 도달
Round 3–4 유의미해짐 안정적 고성능 유지

행동 특징을 probe에 추가해도 성능 향상이 없음 — 내부 상태가 행동이 드러내는 정보를 이미 포착하고 있다.

방법론 개요: Recall-제어 Abort Cascade

에피소드 수준 성공 recall을 보장하는 다단계 게이트 시스템

1
Probe 추출
라운드별 잔차 스트림 숨겨진 상태에서 로지스틱 회귀 probe 학습
2
임계값 보정
Clopper–Pearson 하한으로 라운드별 분포 자유 recal 보장
3
예산 탐색
검증 분할에서 전역 recall ≥ ρ* 제약 하에 연산 절감량 최대화
4
배포 또는 보류
마진/인증 규칙 통과 시 배포, 불가 시 중단 없이 전체 실행

게이트 수 Rg = 6, 에피소드 최대 라운드 Rfull = 20

라운드별 실패 스코어러

특징 추출

  • 에이전트가 생성한 액션의 마지막 토큰에서 잔차 스트림 숨겨진 상태 추출
  • Teacher-forced forward pass로 궤적 재현
  • 모델당 단일 레이어 선택 (사전 AUC 스윕)

Qwen-2.5-7B: 28층 중 20층
Llama-3.2-3B: 28층 중 14층

Probe 구조

  • 표준화 특징 위의 로지스틱 회귀
  • L2 정규화 적용
  • 과적합 방지를 위해 task-grouped 교차 적합

경량화 핵심: 복잡한 분류기 없이도 선형 probe가 내부 상태에서 강한 실패 신호를 포착

Recall-보정 게이트와 예산 탐색

라운드별 Recall 보정

각 임계값 τr은 성공 에피소드의 라운드 r 생존율에 대한 Clopper–Pearson 하한 신뢰 구간이 tr 이상이 되도록 설정

→ 분포 자유(distribution-free) 보장

전역 Recall 제약

예산 벡터 (t1, ..., tRg)를 검증 분할에서 전역 recall ≥ ρ* 제약 하에 연산 절감량 최대화하도록 탐색

→ 라운드 간 예산 분배가 핵심

두 가지 안전 변형

마진 기반

검증 recall이 목표 ρ* + δ 이상인 후보만 선택 → 검증 노이즈에 강건

인증 기반

전역 recall 자체에 대한 분포 자유 인증 제공 → 사전 검증 가능한 보장

Cascade vs. Single Gate

recall 예산을 여러 라운드에 분배하는 것이 핵심적인 가치 창출원

Single Gate

  • 단일 결정 지점에서 recall 예산 전체 소진
  • 초기 라운드의 강한 신호를 활용 불가
  • 낮은 절감률 (90% recall에서 약 28%)

Cascade (본 연구)

  • 라운드별로 예산을 유연하게 분배
  • 신호가 강한 라운드에 더 많은 예산 집중
  • 1.6–1.7× 더 높은 절감률

"recall 예산을 여러 결정 지점에 분배할 수 있는 자유도가 실질적 가치의 상당 부분을 창출한다."

논의 및 한계

고려 사항

  • 마진 vs. 인증: 마진 변형이 실용적이지만, 인증 변형이 더 강한 보장 제공
  • 탐색 입도: 예산 그리드의 세밀함과 마진 크기 간 트레이드오프
  • 절감된 연산의 활용: 재시도, 추가 궤적 샘플링, 서빙 비용 절감 등

한계

  • TextCraft 환경에만 실험 — 다른 환경에서의 일반화 필요
  • 단일 레이어, 단일 토큰 특징 — 더 풍부한 표현 탐색 여지
  • 이진 성공/실패만 고려 — 부분 보상 설정으로의 확장 미해결

결론

1

LLM 에이전트의 내부 활성화에서 실패가 첫 라운드부터 예측 가능하며, 행동 기반 스코어러는 3~4라운드까지 무의미

2

최초의 에피소드 수준 recall 제어 중단 정책을 제안 — 분포 자유 게이트의 recall 예산을 전역 제약 하에 공동 최적화

3

90% recall에서 최대 47.1% 연산 절감, 단일 게이트 대비 1.6–1.7× 성능

4

인증의 표본 복잡도를 정직하게 특정화하여, 가용 데이터가 뒷받침할 수 없는 약속을 사전에 식별 가능

실패가 내부에 이미 쓰여 있음을 읽고, 성공이 보장될 때만 계속하라.
