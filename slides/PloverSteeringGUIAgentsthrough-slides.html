문제 정의

GUI 자동화의 근본적 난제

동적 인터페이스 환경

  • 레이아웃 변화, 예상치 못한 대화상자
  • 진화하는 인터페이스 상태
  • 에이전트가 사용자 의도에서 점진적 이탈

자율주행 모델의 한계

  • 계획·재계획이 내부에만 존재
  • 의도를 은밀히 재해석 가능
  • 사용자가 감독·수정 불가
"더 큰 자율성은 유연성을 높이지만, 불확실성 하에서 계획이 변할 때 시스템 행동을 모호하게 만들고 조향 가능성을 저하시킨다."
기존 접근의 한계

세 가지 설계 격차

1

일회성 계획 출력

작업 계획이 지속적이고 편집 가능한 산출물이 아닌, 한 번 출력된 텍스트로만 존재

2

텍스트 중심 인터페이스

GUI 작업의 시각적·공간적 특성을 무시하고 여전히 텍스트에만 의존

3

내부적 재계획

재계획이 검사·상호작용 불가능한 내부 복구 메커니즘으로 처리됨

자연어 수정만으로는 부족합니다 — GUI 오류는 상황 의존적이고 공간적이며, 여러 유사 요소가 존재할 때 모호해집니다.

계획의 외부화

  • 작업 계획을 지속적·검사 가능·수정 가능한 산출물로 표면화
  • 계획 변경을 명시적으로 노출
  • 이전 진행 상황 보존

국소화된 수정

  • 편집 가능한 계획을 통한 국소 수정
  • 자연어 안내 + 스크린샷 기반 개입
  • 광범위한 재생성 대신 대상 단계만 수정

다중 모달 개입 방식

계획 직접 편집 · 자연어 안내 · 스크린샷 기반 주석 — 상황에 맞는 최적의 수정 경로 제공

핵심 메커니즘

지능형 재계획 (Intelligent Replanning)

사용자 주도 IR

  • 사용자가 계획을 직접 편집하거나 자연어로 수정 지시
  • 스크린샷에 주석을 달아 특정 요소 지정
  • 수정된 계획 단계만 재실행, 완료된 부분 보존

시스템 주도 IR

  • 실행 중 오류 감지 시 자동으로 재계획 트리거
  • 재계획 결과를 사용자에게 명시적으로 노출
  • 사용자가 승인/수정/거부 가능
핵심 차이: 기존 시스템은 재계획을 숨겨진 내부 복구로 처리하지만, Plover는 재계획을 명시적이고 검사 가능한 상호작용으로 만듭니다.
설계 세부사항

Planner 아키텍처와 계획 표현

계획 표현

  • 단계별 구조화된 계획 형식
  • 각 단계에 목적, 예상 상태, 액션 기술
  • 상태 불변량(state invariants) 정의

Executor

  • 비전 기반 스크린샷 해석
  • 마우스/키보드 액션 생성
  • 크로스 플랫폼 지원 아키텍처

상태 불변량의 역할

계획의 각 단계가 완료되었을 때 인터페이스가 충족해야 하는 조건을 명시적으로 정의

불변량 위반 시 자동으로 시스템 주도 재계획이 트리거되어, 사용자에게 계획 수정 기회를 제공

이를 통해 "암묵적 상태 추론"을 "명시적 상태 검증"으로 전환

기여 요약

핵심 기여와 함의

계획 중심 상호작용 설계: 비전 기반 GUI 자동화를 위한 계획 외부화 패러다임 제안
Plover 시스템: 편집 가능한 계획 산출물, 지능형 재계획, 다중 모달 개입을 통합한 실제 구현
경험적 특성화: 벤치마크 실패 사례와 시나리오 기반 워크플로우에서 전문가 주도 복구 가능성 실증

실무적 함의

모든 GUI 작업에 지속적 감독이 필요한 것이 아님. 장기적·고위험·오류 prone한 워크플로우에서 필요한 순간의 감독을 정밀하고 국소적으로 만드는 것이 핵심

향후 방향

위임을 수동 제어로 대체가 아닌, 필요한 감독의 순간을 정밀·국소·복구 가능하게 만드는 방향으로 GUI 자동화 연구 확장
