MemEye: 멀티모달 에이전트 메모리를 위한
시각 중심 평가 프레임워크

A Visual-Centric Evaluation Framework for Multimodal Agent Memory

Minghao Guo, Qingyue Jiao, Zeru Shi, Yihao Quan 외 · Rutgers, Notre Dame, Princeton, UMN

왜 시각 중심 평가가 필요한가?

기존 벤치마크의 시각 기반 질문은 캡션이나 대화 텍스트만으로도 답변 가능한 경우가 많아, 에이전트가 실제로 시각 증거를 보존하는지 검증할 수 없다.

시각 정보의 손실

이미지가 텍스트로 압축되는 과정에서 영역 레이아웃, 객체 식별자, 미세 텍스처 등 핵심 시각 세부 정보가 쉽게 사라진다.

상태 변화 추론 부재

시각 상태가 시간에 따라 변하는 상황을 텍스트로 기술하면, 에이전트가 실제로 시각적 업데이트를 추적하는지 평가 불가능하다.

기존 벤치마크의 한계

MemEye는 기존 멀티모달 대화 벤치마크 대비 시각 정보 의존도가 압도적으로 높다.

MemEye 2차원 평가 프레임워크

멀티모달 메모리 과제를 두 개의 직교 축으로 분류하여 체계적으로 평가한다.

X축: 시각 증거 입자도

X₁
장면 수준 — 전체 장면 파악
X₂
객체 수준 — 객체 존재 및 속성
X₃
인스턴스 수준 — 개별 인스턴스 구분
X₄
픽셀 수준 — 미세 색상, OCR 등

Y축: 메모리 추론 깊이

Y₁
원자 검색 — 단일 증거 직접 답변
Y₂
교차 링킹 — 세션/모달리티 간 연결
Y₃
상태 개정 — 충돌·갱신 하에서 추론
Y₄
진화 종합 — 시간적 변화 추적 및 합성

시각 증거 입자도 (X축) 상세

X₁

장면 수준 (Scene-level)

전체 장면의 유형이나 분위기를 파악하는 수준. 캡션으로도 충분히 대체 가능한 영역.

X₂

객체 수준 (Object-level)

어떤 객체가 존재하는지, 기본 속성(색상, 형태)을 식별하는 수준.

X₃

인스턴스 수준 (Instance-level)

동일 카테고리 내 개별 인스턴스를 구분. 영역 배치, 객체 식별자 바인딩이 필수.

X₄

픽셀 수준 (Pixel-level)

미세 색상 차이, 텍스트(OCR), 작은 패턴 등 원본 이미지 없이는 불가능한 수준.

메모리 추론 깊이 (Y축) 상세

Y₁
원자 검색
단일 세션의 단일 증거로 직접 답변
Y₂
교차 링킹
여러 턴·세션·모달리티의 정보를 연결
Y₃
상태 개정
충돌, 갱신, 오버라이드 하에서 유효 상태 복구
Y₄
진화 종합
시간적 변화를 추적하고 최종 상태 합성
Y축이 높아질수록 단순 검색을 넘어 메모리 시스템의 추론 능력을 직접적으로 검증하게 된다. 특히 Y₃–Y₄ 영역은 기존 벤치마크에서 거루 다루어지지 않는 핵심 구간이다.

8개 생활 시나리오 태스크:

  • 객체 식별 추적, 미세 속성 비교, 미세 색상 기억
  • 교차 세션 신원 추적, 객체 이동 추적, 게임 상태 추적
  • 서사 아크 추적, 상태 진화 하의 믿음 개정

검증 게이트 (Validation Gates)

숏컷 방지와 시각 필수성을 보장하기 위해 4단계 필터링을 적용한다.

Caption-Proof 게이트

캡션만으로는 답변 불가능한 항목만 통과. VLM이 캡션 설정에서 오답을 낸 항목만 유지하여 원본 이미지의 대체 불가능성을 보장한다.

Oracle-Evidence 게이트

정답에 해당하는 시각 증거를 오라클로 제공했을 때 VLM이 올바르게 답변하는지 검증. 메모리가 아닌 추론 능력 자체의 한계를 분리한다.

숏컷 저항 게이트

대화 맥락이나 답변 옵션에서 텍스트적 단서를 통한 우회 답변을 차단한다.

택소노미 구조 감사

각 항목의 X축·Y축 라벨이 정확히 분류 체계와 일치하는지 인간 검토를 거친다.

실험 설정

평가 대상

  • 메모리 방법 13종
  • VLM 백본 4종
  • Full Context, Semantic RAG, A-Mem, Reflexion, Generative Agents, MemoryOS, SimpleMem, MIRIX, MMA, M2A 등

평가 지표

  • 다지선다 정확도 (MC Accuracy)
  • LLM-as-a-Judge 개방형 평가
  • 부트스트랩 신뢰 구간
  • 진단 지표: 캡션 견고성, 검색 진단, 유효 시각 정보 분석

RQ1: 어디서 메모리 시스템이 실패하는가?

MemEye 매트릭스 상에서 X₃–X₄ (인스턴스·픽셀 수준)과 Y₃–Y₄ (상태 개정·진화 종합) 영역에서 성능이 급격히 하락한다.

시각 입자도 측면

X₁(장면) → X₄(픽셀)로 갈수록 모든 메모리 방법의 정확도가 지속적으로 하락. 픽셀 수준에서는 최고 성능 모델도 큰 폭의 성능 저하를 보인다.

추론 깊이 측면

Y₁(원자 검색)은 비교적 잘 수행하나, Y₃–Y₄에서는 과거 상태와 새로운 관찰의 충돌을 해결하지 못하고 구식 증거를 유지하는 경향이 뚜렷하다.

결론: 현재 시스템은 미세 시각 세부 정보 보존과 시간적 상태 변화 추론 모두에서 심각한 한계를 보인다.

RQ2 & RQ3: 왜 시각 정보가 손실되는가?

RQ2 시각 정보 손실 원인

  • 텍스트 추상화 과정: 이미지를 캡션으로 변환할 때 미세 속성이 누락
  • 임베딩 한계: 시맨틱 검색이 미세 시각 차이를 구분하지 못함
  • 저장 선택 오류: 중요한 시각 세부 정보가 저장 단계에서 필터링됨

RQ3 진화 상태 추적 실패 원인

  • 시간적 유효성 식별 실패: 새 관찰이 이전 상태를 오버라이드해야 함을 인지 못함
  • 구식 증거 유지: 갱신된 시각 상태 대신 과거 이미지를 검색
  • 종합 추론 부재: 여러 시점의 증거를 통합하여 현재 상태를 재구성하지 못함

핵심 발견: 트레이드오프

텍스트 기반 메모리

  • 상태 전이와 업데이트 관리에 유리
  • 추상화 과정에서 미세 시각 세부 정보 소실

이미지 기반 메모리

  • 시각 증거를 직접 보존하여 세부 정보 유지
  • 어떤 시각 상태가 시간적으로 유효한지 식별 실패

교차 토픽 확장 실험

대화 길이와 주제 다양성이 증가할수록 전문화된 메모리 메커니즘의 중요성이 더욱 커진다. 단순한 전체 컨텍스트 방식은 스케일업 시 급격히 성능이 저하된다.

메모리 아키텍처 설계 시사점

1
시각 정보 저장
원본 이미지 또는 고해상도 시각 표현을 유지. 텍스트 추상화만으로는 불충분
2
유효 상태 선택
시간적 유효성을 추적하여 구식 시각 증거를 식별하고 제거하는 메커니즘 필요
3
증거 라우팅
질문의 입자도와 추론 깊이에 따라 적절한 메모리 계층을 동적으로 선택
효과적인 멀티모달 메모리는 시각 세부 정보 보존, 시간적 유효성 추적, 적절한 증거 선택의 세 가지를 동시에 달성해야 한다.

요약 및 기여

  • 다차원 평가 프레임워크: 시각 증거 입자도(X축)와 메모리 추론 깊이(Y축)의 두 직교 축으로 멀티모달 메모리 과제를 체계적으로 분류
  • 시각 중심 벤치마크: 371개 미러 질문, 8개 실생활 시나리오, 4단계 검증 게이트를 통과한 엄격한 데이터셋 구축
  • 포괄적 평가와 실증적 통찰: 13종 메모리 방법 × 4종 VLM 백본 평가를 통해 텍스트 vs 이미지 메모리의 근본적 트레이드오프 발견
  • 아키텍처 설계 방향 제시: 증거 라우팅, 시간적 추적, 세부 정보 추출의 통합이 필요함을实证적으로 입증

MemEye는 멀티모달 에이전트 메모리 연구가 "시각 정보를 진정으로 보존하고 활용하는가?"라는 핵심 질문에 집중하도록 이끈다.
