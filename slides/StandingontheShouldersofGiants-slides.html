문제 정의: 크로스 언어 코드 클론 탐지

코드 클론이란?

복제되거나 유사한 코드 조각으로, 유지보수 비용 증가, 버그 전파, 소프트웨어 품질 저하를 유발합니다.

  • Type-1/2: 구문적 유사성 보존
  • Type-3/4: 의미적 동등성 필요

X-CCD의 핵심 난제

서로 다른 언어로 작성된 의미적으로 동등한 프로그램은 표면적 유사성이 거의 없습니다.

  • 전통 기법(Type-1/2) 성능 급감
  • 딥러닝 기법의 OOD 한계
  • 의미적 추론이 필수적

동기: LLM의 한계와 기회

기존 LLM 활용의 문제점

블랙박스 사용 — 수정 불가, 프롬프트 의존

비용·개인정보·재현성 문제

소규모 오픈소스 모델 불안정한 출력

응답 불완전 — 이진 라벨 매핑 실패

지식 증류(KD)의 기회

"강력한 교사 모델의 추론 능력을 컴팩트한 학생 모델로 전이"

  • Phi 시리즈가 "교과서 스타일" 데이터로 성공 입증
  • 코드 생성·분류·요약 등에 KD 적용
  • X-CCD에 KD를 적용한 최초의 연구

단점: 생성 기반이므로 추론 시간 여전히 김

이진 분류 헤드

100%

응답률 달성

장점: 추론 시간 극단적 단축
단점: 62% 시나리오에서 성능 향상

대조 분류 헤드

100%

응답률 달성

장점: 분포 변화에 강건
단점: 50% 시나리오에서 성능 향상

세 가지 방법 모두 100% 응답률을 달성. 분류 헤드 변형은 추론 시간을 수 시간에서 수 분으로 단축하여 실용성을 크게 높임.

핵심 논의: 성능 vs 추론 시간 트레이드오프

방법 응답률 추론 시간 추론 보존 실용성
기본 생성 불안정 수 시간 낮음
강제 결론 100% 수 시간 중간
이진 분류 헤드 100% 수 분 높음
대조 분류 헤드 100% 수 분 높음 (DD 강건)

실무 권장: 추론 과정이 필요하면 강제 결론, 속도가 중요하면 분류 헤드 (DD 환경이면 대조 분류 헤드)

결론

추론 지향 지식 증류와 응답 안정화의 결합은 컴팩트 오픈소스 모델을 X-CCD 탐지에 대해 더 실용적이고 신뢰할 수 있게 만든다.

  • DeepSeek-R1의 System 2 추론을 Phi-3, Qwen-Coder로 성공적 전이
  • 지식 증류가 응답률과 F1 성능을 동시에 향상, 특히 분포 변화 환경에서 두드러짐
  • 강제 결론은 추론을 보존하면서 100% 응답률 달성
  • 분류 헤드는 추론 시간을 수 시간에서 수 분으로 단축하여 실배치 가능하게 함
  • API 의존 없이 로컬 배포 가능한 고성능 X-CCD 모델 구축 경로 제시
