Generative AI-Driven Middleware for Dynamic Runtime Interoperability

👤 Oliver Aleksander Larsen & Mahyar T. Moghaddam arXiv: 2603.28731v1 · 2026년 3월 30일 제출

핵심 아이디어

대규모 언어 모델(LLM)을 활용하여 이기종 분산 시스템 간의 스키마 불일치를 런타임에서 자동으로 감지하고 해결하는 FastAPI 기반 미들웨어

02 / 13

⚡ 문제 정의

🌐
이기종 시스템의 현실

현대 분산 시스템은 REST API, GraphQL 엔드포인트, IoT 장치 등 서로 다른 스키마와 프로토콜을 사용하는 서비스를 통합합니다.

💥
런타임 상호운용성 실패

컴포넌트 간 데이터 교환 시 구조적 차이로 인한 실패 — 중첩 객체 불일치, 타입 비호환성, 스키마 위반 등이 발생합니다.

🔧
기존 접근법의 한계

전통적 정적 어댑터는 모든 스키마 쌍에 대해 수동 코딩이 필요하며, 사전에 모든 형식 변형을 알아야 합니다.

🎯
SAGAI-MID의 목표

LLM을 활용하여 사전 정의된 매핑 없이 런타임에서 스키마 불일치를 발견하고 적응하는 자율적 미들웨어를 구현합니다.

🔄 5계층 파이프라인 상세

1
요청 가로채기 및 검증
모든 컴포넌트 간 통신을 포착하고, 예상 스키마 대비 수신 데이터를 검증하여 구조적 불일치를 플래그
2
이상 감지
DeepDiff 라이브러리를 사용한 구조적 비교 — 필드 유무, 타입 불일치, 중첩 구조 위반, 배열 구조 비호환성 분류
3
LLM 기반 분석
구조화된 프롬프트 엔지니어링으로 호환성 문제에 대한 가설 생성, DIRECT 또는 CODEGEN 경로 결정
4
해결 실행
DIRECT: 변환 규칙 직접 적용 | CODEGEN: 변환 코드 생성 후 AST 분석 검증 및 샌드박스 실행
5
응답 및 피드백
변환된 데이터 반환, 해결 성공/실패 로깅, 컴포넌트 상호작용 프로필 업데이트

⚔️ 해결 전략 비교: DIRECT vs CODEGEN

DIRECT 전략

단순 변환에 최적화

  • 필드 이름 변경 (userId → user_id)
  • 타입 캐스팅 및 검증
  • 선택 필드 기본값 주입
  • Python/JS 직접 실행
평균 pass@1: 0.77
VS

CODEGEN 전략

복잡한 변환에 최적화

  • 조건부 로직 및 알고리즘 변환
  • 타입 힌트 포함 코드 생성
  • AST 분석을 통한 사전 검증
  • 재사용 가능한 어댑터 코드
평균 pass@1: 0.83

결론: CODEGEN이 DIRECT를 약 6%p 상회. 복잡한 변환에서의 재사용성과 구조화된 출력이 핵심 장점.

07 / 13

🛡️ 3단계 안전장치 스택

✅
1단계: 검증
  • • 타입 체크: 출력이 예상 스키마와 일치하는지 검증
  • • 코드 분석: 위험한 연산 스캔 (eval, 시스템 호출, 파일 I/O)
🗳️
2단계: 앙상블 투표
  • • 다중 LLM 응답 비교
  • • 속도 제한으로 캐스케이드 실패 방지
  • • 샌드박스 환경에서 격리 실행
🔙
3단계: 규칙 기반 폴백
  • • 원본 데이터 보존
  • • 실패한 변환 자동 롤백
  • • 규칙 기반 대안 경로 제공
파이프라인 흐름:
요청 → 구조적 diff (DeepDiff) → LLM 의미 분석 → [DIRECT | CODEGEN] → 검증 → 앙상블 투표 → 폴백 → 응답

⚙️ 구현 세부사항

기술 스택

  • FastAPI 프레임워크 (REST API)
  • OpenAI GPT-4 & GPT-3.5-turbo
  • 샌드박스 Python/JS 환경
  • 구조화된 JSON 로깅

프롬프트 엔지니어링

  • 역할 기반 프롬프팅
  • Few-shot 변환 예시 포함
  • 명시적 제약 조건 지정
  • JSON 스키마 구조화 출력

FastAPI 아키텍처 특징

미들웨어 레이어 요청/응답 가로채기 비동기 처리 (논블로킹) WebSocket 스트리밍 지원 자동 OpenAPI 문서 생성
