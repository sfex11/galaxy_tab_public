에이전트 및 멀티모달 LLM을 위한 접근법

Context-Aware RL for Agentic and Multimodal LLMs
Peiyang Xu, Bangzheng Li, Sijia Liu, Karthik R. Narasimhan, Pramod Viswanath, Prateek Mittal, Xingyu Fu
Princeton University · UC Davis

문제 정의: 맥락 무인식Context Unawareness

에이전트 설정
긴 툴 트레이스 내에서 단일 결정적 줄을 식별해야 하는데, 모델이 이를 놓침. 예: 코드 편집 시 주변 맥락과의 일관성을 유지하지 못해 변수 정의를 삭제하는 등의 런타임 오류 발생.
멀티모달 설정
이미지 내 미세한 시각적 단서를 놓쳐 잘못된 답변 생성. 예: 그래프에서 y값을 3이 아닌 2로 잘못 읽어 예측 오류 발생. 관련 정보는 존재하나 모델이 이를 활용하지 못함.
"관련 정보가 맥락에 존재함에도, 모델의 예측이 그 정보에 정밀하게 정합되지 않는 현상"

진단: 대조적 맥락 탐색Contrastive Context Probe

질문, 후보 답변, 두 개의 유사한 맥락을 제시하고 답변을 지지하는 맥락을 선택하도록 함
모델 정확도(%) 비고
GPT-5.4 ~90 안정적 성능
Claude Opus 4.7 ~88 안정적 성능
Qwen-3 (VL) 8B ~50 거의 무작위 선택
Qwen-3.5 9B ~50 거의 무작위 선택
약 40%p의 격차 — 오픈소스 모델들은 표준 벤치마크에서 강한 성능을 보이지만, 맥락 정합 능력은 심각하게 부족

에이전트 대조적 맥락 쌍 구축

SWE-smith의 66K 트레이젝토리에서 조건 필터링으로 1K 쌍 구축
단계적 필터링 조건
  • 동일 리포지토리 & 커밋
  • 동일 파일 수정
  • 동일 함수/클래스 타겟
  • 의미적으로 관련되지만 구별되는 이슈
  • 패치 내용 마스킹 (<PATCH_MASKED>)
결과적 특성
토큰 수준에서 거의 동일한 쌍 생성. 동일 리포, 커밋, 파일, 함수 — 작은 결정적 코드 영역만 다름. 따라서 표면 통계가 아닌 맥락 이해가 필수적.
GPT-5.4로 자동 검증: 단축로 노출 쌍 적극적 거부

멀티모달 대조적 맥락 쌍 구축

다양한 소스에서 생성적 편집과 유사도 검색으로 7K 쌍 구축
생성적 편집 경로
원본 이미지를 기반으로 최소 변경으로 답변이 바뀌도록 편집. 두 이미지는 시각적으로 매우 유사하지만 서로 다른 답변을 지지.
유사도 검색 경로
임베딩 공간에서 고도로 유사하지만 의미적으로 구별되는 이미지 쌍을 검색. 시각적 유사성과 의미적 차이를 동시에 만족.
품질 관리
  • 아티팩트/단축로 필터 적용 — 표면적 단서에 의존하는 쌍 적극 거부
  • 구성상 어려운 쌍(hard by construction)으로 보장
  • GPT 자동 검증으로 인위적 단서 노출 최소화

맥락 인식 강화학습 목표

표준 GRPO 목표와 맥락 인식 손실 ℒ_CA를 공동 최적화
표준 GRPO 손실
원본 태스크 데이터에서 최종 답변에 대한 결과 보상(outcome reward)으로 정책 최적화. 기존 RL 사후 학습과 동일.
맥락 인식 손실 ℒ_CA
(Q, A)를 조건으로, 로짓 수준 대조 손실로 C⁺(지지 맥락)의 로짓을 C⁻(혼동 맥락)보다 높이도록 학습. 정답 생성과 별개로 맥락 식별 능력 향상.
최종 목표 = GRPO 손실 + λ · ℒ_CA (맥락 인식 손실)
λ: 맥락 인식 손실 계수 — ablation으로 최적값 탐색

데이터 증강 vs ContextRL

동일한 대조적 데이터를 사용하더라도, 학습 목표가 다르면 결과가 극적으로 다름
방법 데이터 사용 방식 긴 호라이즌 결과
SFT 증강 대조 쌍을 일반 Q-C-A 예시로 변환 resolve rate 0%로 붕괴
결과 보상 RL 증강 대조 쌍에 결과 보상 적용 사실상 무효
ContextRL (제안) 맥락 선택 보조 목표 적용 일관된 향상
향상의 원인은 대조적 데이터 자체가 아니라, 맥락 선택 목표를 어떻게 학습에 통합하는지에 있음

왜 데이터 증강은 실패하는가?

SFT 증강의 문제
대조 쌍을 일반 지시-맥락-답변 형태로 변환하면, 모델이 맥락 간의 미세한 차이를 무시하고 표면적 패턴에 과적합. 결과적으로 기존 능력까지 붕괴(collapse).
결과 보상 RL의 한계
최종 답변의 정오만 보상하면, 어떤 맥락이 답변을 지지하는지에 대한 신호가 전달되지 않음. 대조적 데이터를 추가해도 기존 RL과 동일한 그래디언트 방향.
ContextRL의 핵심: "답변 생성"과 "맥락 지지 식별"을 명시적으로 분리하여 로짓 수준에서 정합 신호 제공
