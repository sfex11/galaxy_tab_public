핵심 개념

LLM 네이티브 그림이란?

사람이 읽을 수 있으면서 동시에 기계가 접근 가능한
데이터 기반 연구 산출물

그림은 더 이상 최종 결과물이 아닌, 발견을 위한 인터페이스입니다. 데이터 하위 집합, 분석 연산, 코드, 시각화 사양을 모두 내장하여 LLM이 출처를 추적하고, 코드를 생성하여 분석을 확장할 수 있습니다.

프레임워크 구조

4대 핵심 구성 요소

Ft = { Vt , Ct , Dt , Mt }
Vt - 시각화
렌더링된 이미지, 텍스트 요약, Vega-Lite JSON 사양
Ct - 코드
시각화를 생성하는 실행 가능한 SQL, Python 연산
Dt - 데이터
렌더링에 사용된 기본 데이터 하위 집합과 스키마
Mt - 메타데이터
타임스탬프, 버전, 사용자 연산 설명, 산출물 링크

이 네 가지 구성 요소가 결합되어 완전한 출처 정보(provenance)를 제공하며, LLM이 그림을 단순 이미지가 아닌 구조화된 인터페이스로 활용할 수 있게 합니다.

양방향 매핑

분석과 시각화의
양방향 변환

💬
분석 연산
자연어 / SQL / Python
순방향 → ← 역방향
📊
시각화
차트 / 대시보드 / 상호작용
순방향: 연산 → 시각화
자연어를 테이블 선택, 필터링, 차트 유형 지정, 시각적 인코딩 등 일련의 동작으로 파싱
역방향: 시각화 → 연산
브러싱, 클릭 등 시각적 상호작용을 데이터 행에 자동 매핑하여 새로운 분석 시퀀스를 트리거

이 왕복 변환(round-trip conversion) 덕분에 사용자와 LLM이 시각화를 통해 데이터를 탐색하고, 다시 데이터에서 새로운 시각화를 생성하는 혼합 주도형 워크플로가 가능합니다.

설계 원칙

핵심 설계 원칙 4가지

  • 1️⃣
    이중 가독성 (Dual-Legibility)
    그림은 인간에게 직관적이면서 동시에 기계에 출처 정보를 노출해야 합니다.
  • 2️⃣
    조합적 해석 가능성 (Compositional Interpretability)
    시각적 인식(인간) + 구조화된 데이터/코드 접근(기계)에서 의미가 도출됩니다.
  • 3️⃣
    출처를 인터페이스로 (Provenance-as-Interface)
    완전한 분석 계보가 사후 투명성과 감사 추적을 가능하게 합니다.
  • 4️⃣
    혼합 주도형 워크플로 (Mixed-Initiative Workflows)
    인간과 LLM이 자연어 지정과 직접 조작을 번갈아 수행합니다.
