분산 에이전트 공격 탐지

Stateful Online Monitoring Catches Distributed Agent Attacks

arXiv: 2605.31593v1

배경

LLM 에이전트의 악용이 급증하고 있다

수천 개

언어 모델이 단일 세션에서 발견할 수 있는 심각한 소프트웨어 취약점의 수

지속 증가

에이전트 기반 사이버 공격의 실제 사례가 빠르게 늘고 있음

언어 모델은 수천 개의 심각한 소프트웨어 취약점을 찾을 수 있으며, 에이전트는 점점 더 사이버 공격에 악용되고 있다.

문제 정의

분산 공격: 탐지를 회피하는 새로운 전략

공격자는 유해한 작업을 여러 사용자 계정에 분산시켜, 개별 대화 기록은 정상으로 보이도록 조작한다.

1
공격 계획
해킹 목표 설정
2
작업 분할
여러 계정에 나누어 할당
3
개별 실행
각 계정은 양호해 보임
4
결합 공격
분산 조각이 합쳐져 공격 완수

각 개별 전사(transcript)만 보면 양성으로 판별됨

핵심 과제

기존 탐지 방식의 한계

무상태(Stateless) 모니터링

  • 각 대화를 독립적으로 검사
  • 계정 간 연결성 파악 불가
  • 분산 공격은 자연스럽게 통과

사후 분석(Post-hoc) 접근

  • 공격 완료 후에야 감지
  • 실시간 차단 불가
  • 피해가 이미 발생한 상태

필요한 것: 계정 간 상태를 추적하는 실시간 모니터링

제안 방법

상태 유지 온라인 모니터링

Stateful Online Monitoring — 여러 계정에 걸친 대화의 전역 상태를 실시간으로 유지하고 추론한다.

상태 연결

서로 다른 계정의 대화를 논리적으로 연결

실시간 추론

대화가 진행되는 동안 온라인으로 판단

조기 차단

공격 완수 전에 실시간으로 개입

기술 세부사항

모니터링 아키텍처

다중 계정 스트림 수집

여러 사용자 계정에서 생성되는 대화 스트림을 동시에 수신

상태 저장소 업데이트

각 계정의 컨텍스트와 계정 간 관계를 상태 그래프에 지속적 갱신

분산 패턴 매칭

여러 계정에 분산된 조각들이 합쳐져 공격 패턴을 형성하는지 추론

실시간 개입

공격으로 판단되면 즉시 해당 계정들에 대한 조치 실행

핵심 개념

무상태 vs 상태 유지 모니터링 비교

특성 무상태 모니터링 상태 유지 모니터링제안
검사 단위 개별 대화 계정 간 대화 집합
시간성 사후 또는 배치 온라인 실시간
분산 공격 탐지 불가능 가능
거짓음성율 높음 (분산 공격 누락) 낮음
연산 비용 낮음 중간~높음
공격 시나리오 예시

분산 공격은 어떻게 진행되는가

계정 A

"웹 서버의 최신 취약점 CVE-2024-XXXX에 대해 설명해줘"

→ 단독으로 보면 일반 정보 요청

계정 B

"Python으로 HTTP 요청을 보내는 스크립트를 작성해줘"

→ 단독으로 보면 일반 코딩 요청

계정 C

"특정 IP 주소에 대해 포트 스캔을 수행하는 방법은?"

→ 단독으로 보면 학습 목적 질문

상태 유지 모니터링 관점: 세 계정의 요청을 결합하면 "취약점 식별 → 익스플로잇 작성 → 표적 탐색"이라는 완전한 공격 체인이 됨

제안 방법

상태 추적의 핵심 메커니즘

상태 그래프 구성

  • 각 계정을 노드로 표현
  • 대화 내용을 노드 속성으로 저장
  • 계정 간 의미적 연관성을 에지로 표현
  • 시간적 패턴을 메타데이터로 기록

온라인 추론 규칙

  • 새 대화가 들어올 때마다 그래프 갱신
  • 서브그래프가 공격 패턴과 매칭되는지 검사
  • 임계값 초과 시 즉시 알림 발생
  • 지연 없이 스트리밍 방식으로 처리
실험 설정

평가 방법

다중 계정

공격자가 2~10개 계정에 작업 분산

다양한 공격

취약점 탐색, 익스플로잇, 피싱 등

베이스라인

무상태 필터와 사후 분석 비교

평가 지표

  • 탐지율 (Detection Rate)
  • 거짓양성율 (False Positive Rate)
  • 탐지 지연 시간 (Time to Detect)
  • 차단 시점 (공격 완수 전/후)
결과

실험 결과

방법 분산 공격 탐지율 거짓양성율 차단 시점
무상태 필터 ~0% 낮음 탐지 불가
사후 분석 중간 중간 공격 완료 후
상태 유지 온라인 모니터링 높음 낮음 공격 완수 전

상태 유지 온라인 모니터링은 분산 공격을 실시간으로 탐지하고 사전에 차단하는 유일한 방법

논의

한계점 및 향후 과제

현재 한계

  • 상태 저장소의 스케일러빌리티 요구
  • 정당한 다중 계정 사용과 공격의 구별 난이도
  • 공격자가 더 미세하게 분산할수록 탐지 난이도 증가
  • 프라이버시 보장과 모니터링의 균형 문제

향후 방향

  • 효율적인 상태 압축 및 만료 메커니즘
  • 학습 기반 의미적 연결 추론
  • 다양한 플랫폼 간 분산 공격 추적
  • 프라이버시 보존 기술(차분 프라이버시 등) 통합
결론

핵심 메시지

분산 에이전트 공격은 개별 대화 수준에서는 보이지 않는다. 따라서 계정 간 상태를 추적하는 온라인 모니터링이 필수적이다.
