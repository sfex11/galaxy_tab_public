Statistical Self-Consistency in Language Models

동기: In-Context Learning의 해석

In-Context Learning(ICL)은 프롬프트가 문맥을 지정하고, 모델의 출력이 해당 조건부 분포의 추정치로 처리되는 조건부 추론의 한 형태로 해석된다.

전통적 해석

프롬프트 → 모델 출력 ≈ P(y|x, context)

출력을 조건부 확률 분포의 근사로 간주

핵심 질문

이 해석이 정말로 유효한가?

기본 확률론적 성질을 LLM이 만족하는가?

통계적 자기일관성이란?

확률 분포로서 만족해야 할 기본적 성질들:

Σ

정규화

모든 결과의 확률 합 = 1

⊆

여백화 일관성

부분집합의 합은 전체 확률과 일치

⊥

독립성 보존

독립 사건의 결합 확률 = 곱

이러한 성질을 위배하면 "조건부 분포 추정"이라는 해석이 근본적으로 의심된다.

문제 제기: LLM은 자기일관성을 만족하는가?

단일 프롬프트로는 하나의 샘플만 얻을 수 있다.

하지만 분포의 성질을 검증하려면 여러 샘플이 필요하다.

근본적 딜레마

  • 프롬프트를 바꾸면 → 조건부 분포 자체가 바뀔 수 있음
  • temperature 조절로 샘플링 → 분포 왜곡 가능
  • 같은 프롬프트 반복 → 상관된 샘플, 독립성 불만족

⟹ 동일한 조건부 분포에서 독립적인 샘플을 어떻게 얻을 것인가?

제안 방법: Partition, Prompt, Aggregate

동일한 조건부 분포에서 독립 샘플을 추출하기 위한 세 단계 프레임워크

1
Partition
데모 데이터를
여러 부분집합으로 분할
2
Prompt
각 부분집합으로
개별 프롬프트 구성
3
Aggregate
출력들을 집계하여
분포 추정치 도출
핵심 아이디어: 동일한 전체 데이터의 다른 부분집합은 동일한 조건부 분포를 정의해야 한다.

Step 1: Partition — 분할 전략

전체 데모셋 D를 K개의 부분집합 {D₁, D₂, ..., Dₖ}으로 나눈다.

무작위 분할

D를 무작위로 K개로 나눔

각 Dᵢ는 전체 분포의 대표성 보존

교차 검증형 분할

K-fold 방식으로 순차적 분할

중복 없이 전체 데이터 활용

핵심 가정: Dᵢ ⊂ D이고 D가 동일한 데이터 생성 과정에서 왔다면, P(y|x, Dᵢ) ≈ P(y|x, D)

Step 2: Prompt — 개별 프롬프트 구성

각 분할 Dᵢ에 대해 독립적인 프롬프트를 생성하고 모델에 입력

1

D₁으로 프롬프트 구성 → 모델에 입력 → 출력 y₁ 획득

2

D₂으로 프롬프트 구성 → 모델에 입력 → 출력 y₂ 획득

...

반복...

K

Dₖ으로 프롬프트 구성 → 모델에 입력 → 출력 yₖ 획득

각 yᵢ는 동일한 조건부 분포 P(y|x, D)에서의 근사 독립 샘플로 간주

Step 3: Aggregate — 집계와 검증

K개의 독립 샘플 {y₁, ..., yₖ}을 바탕으로 분포 추정 및 자기일관성 검증

경험적 분포 추정

  • 빈도 기반 확률 추정 P̂(y|x, D)
  • 기댓값, 분산 등 통계량 계산

자기일관성 검증

  • 정규화: ΣP̂(y) ≈ 1 인가?
  • 여백화: 부분집합 합이 일관된가?

보정(Calibration) 분석

추정된 분포가 실제 빈도와 얼마나 일치하는지 정량적 평가

자기일관성 위배 사례

PPA 프레임워크로 발견한 주요 위배 패턴

위배 1

과대 확신

특정 출력에 확률이 과도하게 집중됨

위배 2

비정규화

추정 확률의 합이 1에서 벗어남

위배 3

문맥 민감성

분할 방식에 따라 분포가 크게 변동

LLM의 ICL 출력은 엄밀한 조건부 분포 추정이 아닐 가능성이 높다.

실험 설계

벤치마크

  • 분류 작업 (Sentiment, Topic 등)
  • 객관식 QA 작업
  • 다중 선택 추론 과제

설정

  • 분할 수 K = 10~100 가변
  • 다양한 모델 규모 비교
  • 데모 수에 따른 민감도 분석

평가 지표

지표의미이상값
정규화 오차|ΣP̂(y) − 1|0
여백화 오차부분집합 합 vs 전체0
보정 오차신뢰도 vs 정확도0

주요 실험 결과

발견 1

정규화 위배

대부분의 모델에서 추정 확률 합이 1에서 유의미하게 벗어남

→ 단일 샘플 해석의 근거 약화

발견 2

보정 부족

높은 확률을 부여한 출력의 실제 빈도와 불일치

→ 과대 확신 현상 확인

발견 3

규모 효과

더 큰 모델이 자기일관성을 더 잘 만족하는 경향

→ 스케일링이 부분적 해결책

발견 4

PPA의 개선 효과

PPA 집계가 단일 프롬프트보다 더 나은 보정 제공

→ 실용적 가치 입증

시사점과 함의

이론적 시사점

  • ICL = 조건부 추론이라는 해석에 경고
  • LLM 출력을 확률로 취급할 때 주의 필요
  • 자기일관성은 새로운 평가 축 제안

실용적 시사점

  • PPA를 활용한 신뢰도 향상
  • 불확실성 정량화에 PPA 활용 가능
  • Self-consistency 기법의 통계적 근거 제공
"LLM이 무엇을 하는지 이해하려면, 출력이 어떤 수학적 객체인지 명확히 해야 한다."

한계 및 향후 방향

한계
  • 분할이 "진정으로 동일한 분포"를 보장한다는 가정
  • 주로 분류/객관식에 집중된 실험
  • 생성 작업(자유 형식)으로의 확장 미해결
  • 추론 비용 증가 (K배 프롬프트)
향후 방향
  • 개방형 생성에서의 자기일관성 정의
  • 분할 독립성에 대한 이론적 보장 연구
  • PPA 기반 보정 기법 개발
  • 다른 모달리티(비전 등)로 확장

요약

LLM의 In-Context Learning 출력은 엄밀한 조건부 분포가 아닐 가능성이 높으며, PPA 프레임워크는 이를 체계적으로 검증하고 개선하는 방법을 제공한다.
