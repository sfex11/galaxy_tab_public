01 — 문제 정의

왜 기존 예측 시스템이
부족한가?

이진 예측(Binary Forecasting)은 "사건 X가 일어날까?"에 대한 확률을 추정하는 과제입니다. 기존 접근법은 구조적 한계를 가집니다.

단일 숫자 확률

단일 확률값만 출력하여 불확실성과 근거를 잃어버림. 왜 그 확률인지 설명 불가

정적 추론

새로운 증거가 등장해도 사후 확률을 갱신하지 못함. 일회성 판단에 머뭄

수동 정보 수집

사람이 직접 뉴스와 자료를 찾아야 함. 자동화된 탐색 부재

해석 불가

블랙박스 모델이라 예측 근거를 추적할 수 없음

03 — 아이디어 ①

베이지안 언어적
믿음 상태

기존 방식은 단일 숫자(예: 0.73)만 출력합니다. BLF는 수치 확률 + 언어적 근거 + 불확실성 서술을 하나의 상태로 유지합니다.

기존 방식
P = 0.73

← 근거 없음, 불확실성 없음

BLF 방식
P = 0.73
"최근 3개의 여론조사가 모두 찬성 우세를 보이며,
역사적으로 유사 사건의 68%가 성공했음.
다만 야당 반발 리스크가 남아있어
불확실성은 중간 수준으로 평가."
믿음 상태 구조
Belief = {
  probability: 0.73,
  evidence: [조사1, 조사2, ...],
  reasoning: "유사 사건 대비...",
  uncertainty: "medium"
}
수치 확률 증거 목록 불확실성

순차적 베이지안 갱신

새로운 증거 e₁, e₂, e₃, ...를 순차적으로 발견할 때마다, 믿음 상태를 베이지안 규칙에 따라 갱신합니다.

베이지안 갱신 규칙
P(H|e) ∝ P(e|H) · P(H)
각 단계에서 언어적 근거 갱신

확률뿐 아니라 증거 목록과 추론 서술도 함께 갱신

모순되는 증거 처리

반대 증거를 만나면 확률을 하향 조정하고 불확실성 표시 증가

멈춤 조건 (Stopping Criterion)

불확실성이 충분히 낮아지면 탐색 루프 종료

05 — 아이디어 ③

에이전틱 탐색 루프

BLF는 자율적인 에이전트로, 질문을 분석하고 정보를 탐색하며 믿음을 갱신하는 루프를 반복합니다.

1
질문 분해
이진 질문을 하위 질문으로 분해
2
정보 탐색
웹 검색, 문서 조회, 도구 호출
3
증거 추출
관련 정보에서 증거 eᵢ 추출
4
베이지안 갱신
Beliefₜ → Beliefₜ₊₁
5
예측 출력
최종 확률 + 근거 서술

단계 2–4를 불확실성이 충분히 낮아질 때까지 반복합니다
