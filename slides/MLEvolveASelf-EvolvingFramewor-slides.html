MLEvolve: 자가 진화형

A Self-Evolving Framework for Automated Machine Learning Algorithm Discovery

상하이 인공지능 연구소 · 화동사범대학교

Shangheng Du, Xiangchao Yan, Jinxin Shi, Zongsheng Cao 외

배경: AI for Science와 LLM 에이전트

장기 수평 자율 과제의 부상

  • 과학적 발견, 자동 실험, 엔드투엔드 알고리즘 설계
  • 열린 탐색 공간에서 지속적 솔루션 생성·평가·조정 필요
  • 자가 진화(self-evolution) 능력이 핵심

머신러닝 엔지니어링(MLE)의 과제

  • 고성능 AI 시스템 설계는 전문가 지식과 수동 반복에 의존
  • 기존 AutoML은 개별 단계 최적화에 국한
  • 데이터 준비→모델 학습→추론의 전체 파이프라인 커버 필요

기존 MLE 에이전트의 세 가지 한계

분기 간 정보 격리

트리 기반 탐색으로 분기 간 성공 전략 전송 불가, 고정된 탐색 전략으로 비효율적 자원 할당

기억 없는 탐색

스칼라 보상만 전파되어 과거 시도의 인사이트 재사용 불가, 매 계획 결정이 독립적으로 이루어짐

계층적 통제 부재

계획과 코드 구현이 일회성 생성으로 결합, 매 반복마다 전체 재작성으로 낮은 효율과 불안정한 수정

MLEvolve 프레임워크 개요

세 가지 핵심 컴포넌트의 통합으로 장기 수평 자가 진화 실현

Progressive MCGS
그래프 기반 분기 간 정보 흐름과 엔트로피 기반 점진적 탐색 스케줄
Retrospective Memory
도메인 지식 베이스(콜드스타트) + 동적 글로벌 메모리(경험 누적·검색)
계층적 적응형 코드 생성
전략 계획과 코드 구현 분리, 상태에 따른 적응적 편집 모드 선택

강한 솔루션은 단일 선형 정제가 아닌, 축적된 경험과 다중 후보 경로의 참조에서 비롯된다

Progressive MCGS: 그래프 기반 탐색 공간

기존 MCTS의 한계

  • 트리 구조로 분기 간 정보 격리
  • 동일 상태 압축 목적의 기존 MCGS와 달리, LLM 코드 생성은 개별 솔루션 노드
  • 고정된 탐색-활용 비율

MLEvolve의 그래프 확장

  • 참조 에지(Reference Edge)로 분기 간 정보 흐름 허용
  • 성공적 궤적의 재사용 및 솔루션 구성 가능
  • 상태 공간 압축이 아닌 크로스-브랜치 전송이 목적

기존 MCGS(계획/RL용)와 달리, 본 방법은 오픈엔디드 LLM 코드 생성을 타겟으로 하며, 각 노드는 고유 후보 솔루션을 나타냄

Progressive MCGS: 점진적 탐색 스케줄

엔트로피 기반 스케줄로 탐색 단계를 적응적으로 제어

초기: 넓은 탐색 (Exploration)

높은 엔트로피 → 다양한 분기 동시 확장, 다각도 솔루션 시도

중기: 균형 단계 (Transition)

엔트로피 점진적 감소, 유망한 분기에 자원 집중 시작

후기: 집중적 활용 (Exploitation)

낮은 엔트로피 → 최적 경로에 집중, 정밀한 솔루션 개선

제한된 시간 예산 하에서 고정 전략보다 효율적인 자원 할당 달성

Retrospective Memory: 회고적 기억 체계

정적 도메인 지식 베이스

  • 사전 큐레이션된 MLE 도메인 지식
  • 콜드스타트 문제 해결: 초기 탐색 시 가이드 제공
  • 추가 LLM 호출 없이 직접 검색 활용

동적 글로벌 메모리

  • 탐색 과정에서 자동 경험 누적
  • 태스크 특화 인사이트의 검색 및 재사용
  • 명시적 반성을 위한 추가 LLM 호출 불필요

기존 방법(ROME, MARS, ML-Master 2.0)과 달리, 추가 LLM 기반 반성/요약 없이 자동으로 경험을 축적하고 검색

계층적 계획과 적응형 코드 생성

Planner-Coder 분리: "무엇을 수정할지" vs "어떻게 구현할지"

전략적 계획과 코드 구현을 명확히 분리하여 안정적인 장기 반복 보장

세 가지 적응형 코딩 모드

Full Rewrite

초기 단계 또는 대규모 변경 시 전체 재작성

Stepwise

단계별 점진적 수정, 중간 규모 변경에 적합

Diff-based Edit

후기 정밀 단계, 국소적 타겟 수정에 활용

현재 탐색 상태에 따라 모드 자동 선택 → 불필요한 전체 재작성 방지, 반복 효율 극대화
