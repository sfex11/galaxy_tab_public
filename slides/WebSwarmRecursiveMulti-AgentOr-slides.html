WebSwarm

순차적 재귀적 다중 에이전트 오케스트레이션을 통한
깊고 넓은 웹 검색

Xiaoshuai Song, Liancheng Zhang, Kangzhi Zhao, Yutao Zhu,
Zhongyuan Wang, Guanting Dong, Jinghan Yang, Han Li, Kun Gai, Ji-Rong Wen, Zhicheng Dou

중국인민대학교 · 콰이쇼우

웹 검색 에이전트의 진화

LLM 기반 웹 검색 에이전트가 정보 탐색 패러다임을 변화시키고 있습니다.

단순 사실 QA

정보 탐색의 초기 단계

깊은 검색 (Deep)

다홉 의존성 및 제약 조건 해결

넓은 검색 (Wide)

후보 엔티티·페이지·소스 전반의 포괄성 확보

복잡한 연구 수준의 검색과 보고서 지향 조사를 수행하려면 깊이(Depth)와 너비(Width)를 동시에 지원해야 합니다.

기존 방법의 한계

① 얕은 재귀 깊이

루트 수준에서만 분해하므로, "연도→브랜드→모델→속성" 같은 다단계 구조가 결국 단일 에이전트의 긴 ReAct 궤적으로 압축됨

② 제한된 협업 적응성

단일 전역 협업 패러다임(병렬 분할, 직렬 인계, 다경로 샘플링 등)만 사용하여, 서로 다른 하위 작업의 특성에 맞게 전환 불가

③ 약한 증거 기반 확장

쿼리의 표면 의미로만 분해하여, 웹 정보 구조와 정렬되지 않으면 과도 분해 또는 잘못된 차원 분할 발생

WebSwarm 제안

점진적 재귀 위임(Progressive Recursive Delegation) 프레임워크

증거가 축적됨에 따라 분해, 확장, 협업 구조를 동시에 구성합니다.

동적 에이전트 검색 노드 생성

각 노드는 지역 목표(Local Objective) + 검색 모드(Search Mode) 결합

재귀적 위임 & 다양한 협업 구조

병렬 분할 정복, 순차적 검색-검증, 다경로 샘플링-집계 등 모드별 협업

상향식 결과 반환 & 확장 결정

하위 노드 완료 후 상위 노드가 확장·수정·종료 여부 판단

WebSwarm 전체 구조

DeepWideSearch 벤치마크 예시를 통한 WebSwarm 실행 흐름

1
루트 에이전트
원본 태스크 수신 및 초기 검색 노드 위임
2
웹 구조 탐색
관련 정보의 집중/분산 여부를 경량 프로빙으로 파악
3
노드 확장
웹 구조에 맞춰 자식 노드를 재귀적으로 생성
4
경험 전이
선행 노드의 궤적 경험을 동질 노드에 증류
5
상향 집계
증거와 결과를 상위로 반환하여 최종 응답 생성

재귀적 검색 위임 메커니즘

목표 분해 & 위임

상위 노드가 자식 노드에 지역 목표와 검색 모드를 함께 할당합니다. 검색 모드는 해당 노드가 독립 해결할지, 자식을 생성할지, 어떤 협업 구조를 조직할지 결정합니다.

검색 노드 해결 & 증거 갱신

노드는 자체 ReAct 검색으로 목표를 해결하거나 자식에게 위임합니다. 완료 후 증거와 결과를 상위로 반환하고, 상위 노드는 이를 바탕으로 추가 확장, 수정, 또는 종료를 결정합니다.

핵심 아이디어: 고정된 분해 구조가 아닌, 중간 증거가 새로운 엔티티·제약·목표를 드러낼 때마다 점진적으로 트리를 확장합니다.

검색 모드와 협업 구조

각 노드의 지역 목표에 따라 적응적 협업 패턴을 선택합니다.

사실 조회 모드

빠르고 신뢰할 수 있는 단일 경로 실행

넓은 검색 모드

다수 항목에 대한 병렬 분할 정복

깊은 검색 모드

순차적 단서 발견-검증 반복

오픈셋 열거 모드

다경로 샘플링-집계로 재현율·정밀도 균형

웹 구조 기반 & 경험 기반 안내

웹 구조 기반 확장

경량 프로빙으로 관련 정보가 소수 집계 페이지에 집중되어 있는지, 시간축·엔티티·속성 차원으로 분산되어 있는지 파악합니다.

  • 집중 → 단일 노드 직접 검색
  • 분산 → 해당 차원으로 자식 노드 분해

경험 기반 노드 해결

동일 부모 노드 아래의 동질적(Homogeneous) 형제 노드 간에 선행 노드의 궤적 경험을 증류하여 후속 노드의 검색을 안내합니다.

  • 효과적인 검색 전략 재사용
  • 불필요한 시행착오 감소

실험 설정

벤치마크

  • BrowseComp-Plus — 깊은 검색
  • WideSearch — 넓은 검색
  • DeepWideSearch — 깊+넓은 검색
  • GISA — 교차 깊+넓은 검색

베이스라인

  • ReAct (단일 에이전트)
  • MindSearch, HiRA (계층적 계획)
  • InfoSeeker, A-MapReduce (수평 분해)
  • Table-as-Search, Web2BigTable (테블릿 완성)
  • Kimi-Swarm (다중 에이전트)

기본 모델: Claude Sonnet 4 · 웹 도구: Bing Search + 브라우저 렌더링

절제 분석

각 구성 요소의 기여도 분석

구성 요소 제거 효과

  • 재귀 위임 제거 → 성능 급락
  • 웹 구조 안내 제거 → 넓은 검색에서 감소
  • 경험 전이 제거 → 동질 노드에서 중복 오류
  • 검색 모드 분화 제거 → 협업 비효율

핵심 발견

  • 재귀 깊이가 깊을수록 WebSwarm의 이점이 극대화됨
  • 웹 구조 안내는 특히 DeepWideSearch에서 효과적
  • 경험 전이는 검색 효율을 ~20% 향상

추가 분석

난이도별 성능

태스크 난이도가 높을수록 단일 에이전트와의 격차가 더욱 벌어집니다. 재귀적 분해의 이점이 복잡한 태스크에서 극대화됨.

웹 도구 효율

웹 구조 안내를 통해 불필요한 검색 호출을 감소시키면서도 더 높은 재현율을 달성합니다.

모델 일반화

Claude, GPT, Gemini 등 다양한 LLM 백엔드에서 일관된 성능 향상을 확인했습니다.

결론

  • 점진적 재귀 위임 프레임워크를 제안 — 지역 목표와 검색 모드를 결합한 에이전트 검색 노드를 동적으로 인스턴스화하여 분해·확장·협업을 공동 구성
  • 웹 구조 기반 재귀 위임을 도입 — 웹 정보 조직에 기반하여 확장을 정당화하고, 동질 검색 노드 간에 하위 태스크 경험을 전이
  • 4개 벤치마크에서 깊은 검색, 넓은 검색, 깊+넓은 교차 검색 태스크에 대해 일관된 성능 우위 검증

WebSwarm은 고정된 분해가 아닌 증거 기반의 점진적 트리 확장을 통해, 복잡한 웹 정보 탐색에서 단일 에이전트 및 기존 다중 에이전트 방법의 한계를 극복합니다.
