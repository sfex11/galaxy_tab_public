LLM 지식 인코딩의 근본적 한계

파라메트릭 지식

LLM은 세계 지식을 수십억 파라미터 내에 암묵적으로 인코딩

  • 불투명 — 어떤 지식이 어디에 있는지 알 수 없음
  • 정적 — 업데이트가 어려움
  • 환각 위험 — 사실성 보장 불가

지식 그래프 (KG)

구조적, 해석 가능, 업데이트 가능한 사실 정보 표현

  • 인간이 읽고 편집 가능
  • 정확한 쿼리와 검증 가능
  • 효율적인 증분 업데이트

핵심 과제: 그래프 구조와 LLM이 소비하는 토큰 시퀀스 사이의 표현 간극을 어떻게 좁힐 것인가?

기존 접근법의 한계

접근법 대표 방법 주요 한계
텍스트 기반 주입 RAG, KG-RAG, K-BERT 토큰 비효율성 하나의 사실이 수십 토큰 소모
PEFT 어댑터 K-Adapter, KnowLA 재훈련 필요 새 정보마다 비용 발생
그래프 토큰 임베딩 GraphToken, TEA-GLM, GQT 그래프 특화 일반 LLM 추론에 미검증
컨셉 벡터 주입 ConceptFormer 소규모 모델 한정 GPT-2 0.1B에서만 평가

KoRe가 해결하고자 하는 네 가지 연구 공백

  • 토큰 효율적 지식 표현
  • GNN을 통한 사실 전달
  • 미검증 엔티티 일반화
  • 대규모 LLM 백본 적용

쿼리 엔티티 c를 중심으로 WikiData에서 추출한 1-hop 스타 서브그래프

Gx = (Vx, Ex, Rx)

쿼리 x와 서브그래프 Gx가 주어졌을 때, Gx 내의 팩트 트리플에 기반한 정답 y를 생성해야 함

이는 지식 조건부 언어 모델링 목표로, 조건 신호는 LLM 컨텍스트에 들어맞는 Gx의 압축 표현입니다.

그래프 추출 및 GNN 인코딩

그래프 추출

  • 쿼리에서 핵심 엔티티 식별
  • WikiData에서 1-hop 스타 그래프 추출
  • PageRank 점수로 이웃 순위 매김
  • Tri-REx: 최대 100 엣지
  • QA 데이터셋: 최대 10,000 엣지 (노이즈 조건 평가)

GNN 인코딩

  • 동결된 문장 인코더(Qwen-Embeddings)로 노드/엣지 임베딩 초기화
  • 초기 표현이 텍스트 도메인에 정렬됨
  • TransformerConv + GraphNorm + 잔차 연결
  • 중앙 노드 풀링으로 단일 벡터 g 생성

핵심 차별점: 사전 훈련된 KGE 모델 없이, 문장 인코더 초기화만으로 임의의 지식 그래프를 인코딩 가능 → 미검증 엔티티에 일반화

방향성 잔차 벡터 양자화 (Directional RVQ)

기존 RVQ의 문제와 해결

표준 RVQ: rt+1 = rt − et → 코드북 붕괴 발생

방향성 업데이트: rt+1 = rt − ⟨rt, ct⟩ ct

선택된 코드 방향의 성분만 제거 → 코드들이 직교 부분공간을 커버하도록 유도

훈련 손실

LRVQ = β Σ(1 − cos(rt, ct)) + ||rQ||²

방향성 커밋먼트 항 + 최종 잔차 노름

코드북 관리

  • ℓ₂ 정규화 코드북 + 코사인 유사도 선택
  • EMA로 입력 잔차의 이동평균 업데이트
  • Dead-code 리셋 메커니즘
  • STE로 인코더에 기울기 전파

훈련 목표

Ltotal = LLLM + λrvq · LRVQ

Qwen3-8B의 자기회귀 언어모델 손실 + RVQ 압축 손실의 가중합. LoRA 어댑터와 4개 모듈만 훈련.

결론 및 시사점

주요 기여

  • KG를 이산 지식 토큰으로 최대 10배 압축
  • 방향성 RVQ로 코드북 붕괴 해결
  • 사전 KGE 없이 미검증 엔티티 일반화
  • 대규모 LLM(Qwen3-8B)에서 경쟁적 성능 검증

한계점

  • 1-hop 서브그래프에 한정 (다중 홉 확장 필요)
  • 엣지 수 증가 시 노이즈 처리 한계
  • 추가 훈련 모듈에 대한 학습 비용 존재
  • 다양한 도메인에 대한 추가 검증 필요

"압축 이산 KG 표현은 현대 LLM을 그라운딩하는 데 효율적이고 효과적으로 사용될 수 있다."
