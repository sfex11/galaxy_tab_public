조향 벡터란 무엇인가?

핵심 문제

조향 벡터(Steering Vectors)를 LLM에 적용하면 모델 정렬(alignment)을 효율적으로 달성할 수 있지만, 내부적으로 어떤 메커니즘이 작동하는지 해석 가능한 설명이 부족하다.

입력 프롬프트
➔
+ 조향 벡터
➔
변경된 출력

연구 질문: 조향 벡터가 영향을 미치는 내부 메커니즘은 무엇이며, 이것이 어떻게 다른 모델 출력을 만들어내는가?

회로 교환 가능성

서로 다른 조향 방법론들이 기능적으로 동등한 회로를 동일한 레이어에서 활성화한다. 한 방법의 회로를 다른 방법의 벡터에 적용해도 충실도가 유지됨.

핵심 통찰: 조향은 모델 전체에 분산되지 않고, 소수의 특정 경로를 통해 작동한다. 세 가지 방법론 모두 공유된 메커니즘적 백본을 활용한다.

결론 & 의의

핵심 기여와 한계점

✓ 주요 기여

  • 조향 벡터는 주로 OV 회로를 통해 어텐션 메커니즘과 상호작용 (QK 회로는 무시)
  • 서로 다른 조향 방법론이 동일한 내부 회로를 활용함을 입증
  • 조향 벡터를 90~99% 압축해도 성능 유지 가능
  • SVV 분해를 통해 의미론적 해석 가능성 확보

⚠ 알려진 한계

  • MLP에 대한 깊은 분석 부족 (회로 내 존재감에도 불구하고)
  • 단일 조향 레이어만 평가 (DIM 최적 레이어)
  • 거부 특화 연구 — 다른 개념으로의 일반화는 불확실
  • 화이트박스 모델 접근이 필요

윤리적 고려: 거부 조향은 모델 가중치에 대한 화이트박스 접근이 필요하므로, 블랙박스 공격보다 악용 위험이 제한적이다. 메커니즘적 이해는 오히려 더 나은 방어 체계를 가능하게 한다.
