Symbal: 모델 생성 캡션에서
체계적 오정렬 탐지

다중모달 대형 언어 모델이 생성한 캡션에서
반복적 텍스트 오류와 시각적 특징 간의 연관성을 자동으로 식별하는 방법

Maya Varma · Jean-Benoit Delbrouck · Sophie Ostmeier · Akshay Chaudhari · Curtis Langlotz
Stanford AIMI

왜 이 문제가 중요한가?

캡션 오정렬의 심각성

MLLM이 생성한 캡션은 종종 이미지에 없는 내용을 잘못 기술합니다. 의료 분야 같은 안전 중심 도메인에서는 잘못된 진단으로 이어질 수 있습니다.

기존 방법의 한계

CLIPScore, CHAIR 등 기존 지표는 단일 이미지-캡션 쌍의 오류만 평가합니다. 데이터셋 전체의 체계적 패턴을 찾는 방법은 없었습니다.

심장 비대(cardiomegaly) 진단 오류가 이미지 내 심박동조율기(pacemaker) 존재와 강하게 연관되는 현상 — 이를 어떻게 자동으로 발견할 것인가?

체계적 오정렬이란?

체계적 오정렬(Systematic Misalignment)이란, MLLM 생성 캡션에서 반복적으로 나타나는 텍스트 오류가 짝지어진 이미지의 특정 시각적 특징과 밀접하게 연관된 현상을 말합니다.
핵심 특징
  • 허위 상관관계나 학습 편향에서 기인한 오류 클래스
  • 현실에서 자주 함께 등장하지만 결정적 인과 관계는 아닌 특징 쌍
  • 오류가 매우 그럴싸하게 보여 수동 발견이 매우 어려움
  • 단일 샘플 수준이 아닌 데이터셋 전체에서 패턴으로만 드러남

의료 예시 심박동조율기가 보이는 X-ray → 모델이 "심장 비대"라고 잘못 진단 (두 조건은 빈번히 공존하나 인과적이지 않음)

작업 정의: 체계적 오정렬 탐지

이미지-텍스트 쌍으로 구성된 비전-언어 데이터셋을 입력으로 받아, 반복적 텍스트 오류와 연관된 시각적 특징을 자연어로 출력하는 태스크
두 가지 핵심 난제
01
데이터셋 규모

수천 개의 이미지-캡션 쌍에서 전역 오류 패턴을 식별해야 하며, 이는 최첨단 모델의 추론 능력을 초과하는 규모입니다.

02
벤치마크 부재

체계적 오정렬 탐지 능력을 종합적으로 평가할 수 있는 기존 벤치마크가 존재하지 않았습니다.

Symbal: 개요

기존 파운데이션 모델만을 활용한 구조화된 2단계 파이프라인

1
텍스트 오류 탐지
캡션에서 반복적 텍스트 오류를 클러스터링하고 오정렬 정도를 평가
2
시각적 특징 탐지
1단계 결과를 활용해 연관된 시각적 특징을 식별하고 기술
3
자연어 요약
발견된 오정렬을 자연어로 종합 보고

핵심 통찰: 거대 데이터셋을 한 번에 처리하지 않고, 의미적으로 유사한 사실을 그룹화하여 문제를 분해한다.

Stage 1: 텍스트 오류 탐지

캡션에서 반복적으로 등장하는 잘못된 사실(fact)을 식별

A
의미적 유사 사실 그룹화

LLM을 활용해 각 캡션에서 사실을 추출하고, 임베딩 기반 클러스터링으로 의미적으로 유사한 사실들을 그룹화

B
오정렬 정도에 따른 그룹 점수화

VLM(비전-언어 모델)을 사용해 각 사실이 짝지어진 이미지와 얼마나 불일치하는지 평가하여 그룹별 점수 산출

C
최상위 그룹 요약

가장 높은 오정렬 점수를 받은 그룹의 사실들을 LLM이 하나의 통합 개념으로 요약 (예: "cardiomegaly")

Stage 2: 시각적 특징 탐지

1단계에서 발견한 텍스트 오류와 연관된 시각적 특징을 식별

A
의미적 유사 이미지 그룹화

VLM로 각 이미지를 기술하고 임베딩 기반 클러스터링으로 시각적으로 유사한 이미지들을 그룹화

B
오정렬 정도에 따른 그룹 점수화

1단계에서 식별된 텍스트 오류가 특정 이미지 그룹에서 얼마나 집중적으로 나타나는지 평가

C
최상위 그룹 요약

가장 높은 점수의 이미지 그룹에 공통으로 나타나는 시각적 특징을 VLM이 자연어로 요약 (예: "pacemaker")

설계 이점 두 단계가 대칭적 구조를 가지며, 각 단계는 독립적으로 최적화 가능

구성 방식
  • 각 데이터셋에 주석이 달린 체계적 오정렬 ground-truth 레이블 포함
  • 모델 예측과 ground-truth 간 정렬도로 정량적 평가
  • 자연 이미지(COCO 등)와 의료 이미지(CheXpert 등) 모두 포괄

최근접 베이스라인 대비 성능 향상

구조화된 2단계 접근이 단일 프롬프트로 전체 데이터셋을 처리하는 방식보다 압도적으로 우수

단계별 성능 분석

Stage 1: 텍스트 오류 탐지
  • 의미적 그룹화에서 임베딩 기반 방식이 LLM 직접 그룹화보다 우수
  • VLM 기반 오정렬 점수화가 핵심 성능 요인
  • 요약 단계에서 LLM 선택이 결과 품질에 큰 영향
Stage 2: 시각적 특징 탐지
  • 1단계 결과 품질이 2단계 성능에 직접적 영향
  • 이미지 그룹화 품질이 연관 특징 식별의 핵심
  • VLM의 이미지 기술 능력이 시각적 특징 요약에 결정적

제거 연구(Ablation) 각 하위 작업의 구성 요소를 순차적으로 제거하며 성능 기여도 분석 — 구조화된 분해 없이는 성능이 급감함을 확인

실제 환경에서의 검증

4개 MLLM 캡션 감사

GPT-4o, Gemini, LLaVA 등 4개 모델이 생성한 캡션에서 Symbal이 체계적 오정렬을 정확히 발견. 각 모델마다 고유한 오정렬 패턴을 식별 가능.

기존 데이터셋 감사

MLLM 생성 캡션이 포함된 기존 공개 데이터셋을 블랙박스 설정(모델 접근 불가)에서 감사하고 체계적 오류를 발견.

실제 활용 시나리오
  • 개발자: 모델의 실패 모드를 파악하여 더 강건한 모델 구축에 활용
  • 최종 사용자: 배포 전 모델 한계를 사전 파악 (예: "심박동조율기가 있으면 심장 비대 진단을 의심")
  • 데이터 커뮤니티: 차세대 모델 학습용 데이터셋의 품질 보증

새로운 태스크

체계적 오정렬 탐지 — 데이터셋 수준에서 반복적 캡션 오류와 시각적 특징 간 연관성을 식별하는 새로운 문제 정의

Symbal 방법
