Reflective Context Learning

컨텍스트 공간의 최적화 원시(Optimization Primitives)에 대한 체계적 연구

컨텍스트 학습 최적화 에이전트 반영적 학습

01 · 연구 동기

학습의 근본적 문제는
어디에나 존재한다

범용 에이전트는 다양한 태스크와 환경에서 경험으로부터 학습해야 합니다. 그러나 학습의 근본적 문제들은 파라미터 공간이든 컨텍스트 공간이든 동일하게 발생합니다.

⚠
신용 할당 (Credit Assignment)
긴 행동 시퀀스에서 어떤 결정이 결과에 기여했는지 파악하기 어려움
📈
과적합 (Overfitting)
특정 태스크에 과도하게 특화되어 일반화 능력 상실
🧠
망각 (Forgetting)
새로운 학습이 이전에 학습한 능력을 덮어씀
🌀
지역 최적해 (Local Optima)
전역 최적이 아닌 국소적 최적점에 갇히는 문제
🎲
높은 분산 신호
학습 신호의 노이즈가 커서 안정적 업데이트가 어려움

핵심 관찰: 이러한 문제들은 컨텍스트 공간 최적화에서 체계적으로 연구된 적이 없다.

03 · 프레임워크

RCL의 3단계 업데이트 루프

1단계
실행 (순전파)
τ = A(x, Ct)
r = R(τ, y*)
➡
2단계
반영 (역전파)
Δ = g(τ, r, Ct)
➡
3단계
변형 (옵티마이저)
Ct+1 = f(Ct, Δ)

반영기 (Reflector)

궤적과 결과를 분석하여 무엇이 실패했고 왜 실패했는지 기술하는 진단 신호를 생성합니다. 그래디언트와 유사한 방향 정보를 제공합니다.

변형기 (Mutator)

진단 신호를 받아 컨텍스트 아티팩트에 구조적 업데이트를 적용합니다. 업데이트(수정), 추가, 삭제 연산을 수행합니다.

// 전체 합성 업데이트 Ct+1 = f(Ct, {g(τi+, τi-, τiann, ri, Ct)}i∈Bt, St)
04 · 최적화 원시

5가지 최적화 원시

전통적 ML 최적화 기법을 컨텍스트 공간에 체계적으로 적용

① 배칭 & 그룹 롤아웃
다수의 실패 궤적에서 반영 · 대조적 그룹으로 분산 O(1/B) 감소
② 개선된 신용 할당
이중 궤적 메커니즘 — 표준 실행 + XML 주석 실행으로 정밀 진단
③ 보조 손실 & 구조적 편향
플레이북 매개변수화 · 다중 헤드 반영 스키마 (귀인, 근본원인, 격차)
④ 실패 재생 & 커리큘럼
리플레이 버퍼 관리 · 한계 수익이 가장 높은 곳에 집중 학습
⑤ 옵티마이저 상태 & 모멘텀
롤링 상태 문서 St — 변경 원장, 가설 추적, 최적화 단계 관리
05 · 원시 상세

배칭 & 그룹 롤아웃

태스크 배칭

반복 당 B개의 실패 궤적을 반영하여 반복 패턴을 식별합니다.

Ct+1 = f(Ct, Δ1, ..., Δk) // k ≤ B // 분산 감소: O(1/B)

그룹 롤아웃

각 태스크를 G번 실행하여 대조적 그룹을 생성합니다. 성공/실패 궤적이 모두 포함될 때, 반영기가 결과 차이를 만든 결정 지점을 분리합니다.

Δi = g(τi+, τi-, ri+, ri-, Ct) // 성공 궤적 τ⁺와 실패 궤적 τ⁻ 비교

핵심: 그룹 롤아웃은 특히 준결정적 태스크에서 대조적 신호를 제공하여 가장 큰 단일 성능 향상을 달성합니다.

06 · 원시 상세

보조 손실 & 반영 스키마

플레이북 매개변수화

컨텍스트를 명명된 섹션의 개별 엔트리 e1,...,eN으로 구성하여 변형기를 국소적 연산으로 제한합니다.

Update(ej, ej') Add(eN+1) Delete(ek)

다중 헤드 반영 스키마

반영기가 구조화된 진단을 생성합니다:

Δ = (Δattr, Δroot, Δgap) // Δᵃᵗᵗʳ ∈ {actionable gap, execution variance, intractable} // Δʳᵒᵒᵗ: 근본 원인 분석 // Δᵍᵃᵖ: 현재 플레이북의 커버리지 격차

효과: 보조 손실은 추가 실행 없이 반영 품질만 개선하여 7/8 조건에서 성능 향상을 달성했습니다.

07 · 원시 상세

실패 재생 & 옵티마이저 상태

실패 재생 & 커리큘럼 전략

리플레이 버퍼 Bt를 유지하며, 균등 샘플링과 실패 재생을 혼합합니다.

{x1, ..., xB} ~ (1-ρ)·Uniform(D) + ρ·Uniform(Bt) // ρ = 0.5 (재생 비율)
ngrad회 연속 성공 → 졸업 nevict회 연속 실패 → 퇴출

옵티마이저 상태 (모멘텀)

롤링 상태 문서 St가 매 반복 후 업데이트됩니다.

St+1 = h(St, Δ1, ..., Δk, Ct, Ct+1) // 추적 항목: // - 변경 원장 (Change Ledger) // - 플레이북 평가 // - 열린 가설 // - 최적화 단계

* 상태는 변형기에만 주입, 반영기에는 제외 — 모멘텀이 그래디언트가 아닌 옵티마이저 스텝에 작용하는 것과 동일

14 · 핵심 통찰

4가지 핵심 통찰

1
진단 정밀도가 핵심

반영 품질 원시(옵티마이저 상태, 보조 손실)가 컴퓨팅 당 가장 큰 수익을 제공합니다. 추가 실행 없이 성능이 향상됩니다.

2
영역 의존성

어떤 원시가 도움이 되는지는 태스크 구조에 따라 다릅니다.
좁은 격차 → 미세 조정 원시 | 넓은 격차 → 전략 학습 | 준결정적 → 대조적 신호

3
비가산적 합성

개별 가치 ≠ 합성 역할. 그룹 롤아웃과 실패 재생만이 일관되게 필수적이며, 나머지는 영역 의존적 상호작용을 보입니다.

4
파라미터 공간 역학의 재현

진동, 모멘텀 안정화 수렴, 안정성-재학습 트레이드오프 등 파라미터 공간 최적화와 유사한 역학이 컨텍스트 학습에서도 관찰됩니다.

16 · 결론

결론 & 미래 방향

핵심 결론

컨텍스트 기반 학습은 원칙적 연구와 전이 가능한 메커니즘을 통해 개선 가능한 체계적 최적화 문제입니다. 고전적 최적화 원시들이 컨텍스트 공간에서도 효과적으로 작동하며, 그 상호작용은 비자명하고 영역 의존적입니다.

미래 연구 방향

적응적 원시 선택
훈련 단계나 태스크 특성에 따라 자동으로 원시를 선택
2차 상태 추적
옵티마이저 궤적의 2차 도함수 정보 활용
지속적 배포
분포 변화(distribution shift) 하에서의 지속적 학습
모델 능력 확장
더 강력한 모델에서의 원시 효과 재검증
