EurekAgent

Agent Environment Engineering is All You Need
For Autonomous Scientific Discovery

Amy Xin, Jiening Siow, Junjie Wang, Zijun Yao, Fanjin Zhang, Jian Song, Lei Hou, Juanzi Li

Tsinghua University · Zhipu AI

연구 배경

LLM 기반 에이전트는 최적화 가능한 지표와 실행 환경이 주어지면,
가설을 제안·검증·반복하여 인간 설계 방법을 능가하는 결과를 만들어냄

급성장하는 에이전트 능력

  • 수학, 알고리즘, 커널 설계, ML 엔지니어링 등 다양한 도메인에서 강력한 성과
  • Claude Code, Codex 같은 범용 에이전트가 연구 특화 시스템을 능가

신뢰성의 부재

  • 평가 오염, 아티팩트 조작, 절차적 제약 위반 등 리워드 해킹 보고
  • 프롬프트만으로는 복잡한 에이전트 설정에서 신뢰성 보장 불가

핵심 통찰

"자율 과학 발견의 병목은 에이전트 워크플로우를 규정하는 것에서,
에이전트가 작동하는 환경을 설계하는 것으로 이동하고 있다."

기존 접근: 워크플로우 중심

진화적 시스템, 솔루션 트리, 구조화된 토론 등 고정된 연구 절차를 에이전트에 강제

제안 접근: 환경 엔지니어링

Gibson의 수용성(affordance) 이론에 기반 — 환경이 행동의 가능성을 형성하며, 생산적 행동은 증폭하고 해로운 행동은 억제

비유: 유능한 박사과정 학생의 생산성은 분 단위 지시가 아니라, 책임감, 자율성, 정확한 피드백, 동료 협력, 지도 감독에서 비롯됨

EurekAgent의 핵심 설계 원칙 — 에이전트 행동이 아닌 환경을 설계

Permissions Engineering

유용한 능력과 리소스는 노출하되, 연구 무결성 위반(평가 코드 접근 등)은 방지. 격리된 평가 환경 제공

Artifact Engineering

솔루션, 로그, 평가 결과를 파일시스템과 Git 기반의 공유 진행 메모리로 구조화

Budget Engineering

런타임 및 API 비용 경계 내에서 예산 인식적 탐색 지원. 비용 추적 및 할당 관리

Human-in-the-loop Engineering

저마찰(low-friction) 인간 감독 및 개입 지원. 불필요한 수동 승인은 최소화

Permissions Engineering

연구 무결성을 보장하면서 에이전트의 생산적 행동은 허용하는 권한 설계

차단되는 행동

  • 평가 스크립트 소스 코드 접근
  • 평가 환경 내 파일 시스템 수정
  • 네트워크를 통한 외부 정보 검색

허용되는 행동

  • 작업 디렉토리 내 자유로운 파일 조작
  • 제출 형식에 맞는 솔루션 실행 및 평가 요청
  • Git 기반 버전 관리 및 협업

핵심 원칙: 에이전트는 "무엇을 할 수 있는가"가 아니라 "무엇을 할 수 없는가"에 의해 행동이 형성됨

Artifact & Budget Engineering

Artifact Engineering

  • 파일시스템 기반 구조화된 솔루션 저장
  • Git 커밋으로 제출 이력 관리 — 재현성 보장
  • 평가 로그를 공유 진행 메모리로 활용
  • 에이전트 간 세션 간 정보 전달 매체

이전 라운드의 성공/실패 경험이 다음 Propose 세션의 입력으로 자동 제공됨

Budget Engineering

  • API 비용 및 런타임 상한선 설정
  • 실시간 비용 추적 및 남은 예산 제공
  • 예산 고갈 시 자동 종료 메커니즘
  • 에이전트가 탐색 전략에 예산을 반영 가능

26-원 패킹 문제: 총 API 비용 $11 미만으로 SOTA 달성

Human-in-the-loop Engineering

높은 마찰(low-friction)의 인간 감독 — 불필요한 개입은 최소화, 필요한 개입은 용이하게

실시간 모니터링

에이전트 진행 상황, 비용, 점수를 대시보드에서 실시간 확인

선택적 개입

매 스텝 승인이 아닌, 필요시에만 인간이 개입하는 방식

롤백 및 재시작

문제 발생 시 특정 라운드로 롤백하거나 시스템 재시작 가능

"환경 엔지니어링은 인간 감독을 부담이 아닌 가치로 만든다 — 정확히 필요한 순간에, 최소한의 노력으로."

핵심 시사점

워크플로우보다 환경

범용 에이전트의 능력이 충분해진 시점에서, 복잡한 워크플로우를 설계하는 것보다 환경을 잘 설계하는 것이 더 효과적. Claude Code 같은 기성 에이전트를 그대로 사용하여 SOTA 달성

탐색과 무결성의 균형

Permissions Engineering이 에이전트의 자유로운 탐색을 제한하지 않으면서도, 평가 조작과 아티팩트 오염이라는 치명적 실패 모드를 환경 수준에서 차단

비용 효율성

Budget Engineering을 통해 $11~$17의 API 비용으로 수학 SOTA 달성. 환경 설계가 단순한 안전장치를 넘어 효율성의 원천이 됨

인간-에이전트 협업 모델

인간은 방향 설정과 메트릭 정의에 집중하고, 에이전트는 대규모 방법론 탐색을 실행. Human-in-the-loop Engineering이 이 협업을 실용적으로 만듦

한계점 및 논의

인정된 한계

  • 최적화 가능한 메트릭이 있는 과제에만 적용 가능
  • 기성 CLI 에이전트 의존 — 기반 모델의 능력에 상한 존재
  • 완전 자율 연구가 아닌 지표 주도 발견에 한정
  • 환경 엔지니어링 자체의 설계 원칙이 아직 경험적

향후 방향

  • 환경 엔지니어링을 독립된 연구 방향으로 정립
  • 더 다양한 도메인으로 확장 (생물학, 화학 등)
  • 에이전트-환경 공진화(co-evolution) 탐색
  • 환경 설계 원칙의 이론적 기반 구축

결론

"에이전트에게 무엇을 하라고 지시하는 것보다,
에이전트가 어떤 환경에서 행동하는가가
자율 과학 발견의 핵심이다."

EurekAgent는 4차원 환경 엔지니어링만으로 기성 에이전트를 조정하여
수학, 커널, ML 과제에서 새로운 SOTA를 달성함
