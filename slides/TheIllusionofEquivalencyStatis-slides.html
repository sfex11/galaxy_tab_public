양자화 체계: Legacy vs K-Quantization

Legacy 양자화

  • 블록별 균일 선형 양자화
  • 고정 블록 크기 k=256
  • 블록당 스케일링 인자 적용
  • Q8_0, Q5_0, Q4_0 등

K-Quantization

  • 통계적 정보 기반 블록 양자화
  • 지역 평균·표준편차로 정규화
  • 서브블록 단위 처리
  • Q6_K ~ Q2_K (8비트→2비트)

분석 대상: 각 레이어의 Q, K, V, O 투영 행렬에 대해 개별적으로 수행

핵심 지표: Correctness Agreement

정의

기반 모델과 양자화 변형 모델 간에 동일한 예시에서 올바르게 예측한 비율의 겹침을 측정

기존 지표의 한계

  • 정확도: 절대적 성능만 측정
  • Perplexity: 분포 핏만 측정
  • 동일한 문제에 다른 답을 해도 감지 불가

CA의 장점

  • 절대 정확도와 독립적인 행동 측정
  • 예시 수준 결정 일치성 직접 평가
  • 숨겨진 행동 발산을 드러냄

통계적 변화 분석

각 어텐션 레이어의 투영 행렬(WQ, WK, WV, WO)에 대해 다음 통계량을 측정:

모멘트 기반 측정

  • 평균 변화: 가중치 중심 이동
  • 분산 변화: 산포도 왜곡
  • 왜도 변화: 비대칭성 변화
  • 첨도 변화: 꼬리 두께 변화

주요 발견

  • Q, K 투영이 일관적으로 더 민감
  • V, O 투영은 상대적으로 강건
  • 낮은 비트에서 비선형적 급증 관찰
  • 레이어별 민감도에 편차가 큼

분포 발산 분석

기반 모델과 양자화 모델 간 가중치 분포의 거리를 측정

W1

Wasserstein 거리

Earth Mover's Distance

DKL

KL 발산

Kullback-Leibler

JS

Jensen-Shannon 발산

대칭적 거리 측정

8비트→4비트 구간에서는 발산이 완만하지만, 4비트→2비트 구간에서 비선형적 브레이크포인트가 나타남

실험 결과: 성능 지표

Perplexity

  • 8비트~4비트: 안정적 유지
  • 3비트 이하에서 급격한 열화
  • K-Quant이 Legacy 대비 우수

다운스트림 정확도

  • 중간 양자화에서도 대체로 보존
  • 태스크별 민감도 차이 존재
  • 표면적 지표만으로는 불충분

PPL과 정확도가 "안정적"으로 보이는 구간에서도 Correctness Agreement는 하락하기 시작함

실험 결과: Correctness Agreement

정확도가 유지되는 구간에서도 행동 발산이 감지됨

8비트: 기반 모델과 거의 완전한 일치 (CA ≈ 1.0)

6~5비트: CA 점진적 하락 시작 — 정확도는 여전히 안정

4비트: CA 유의미한 하락 — 동등성 환상의 시작

3~2비트: CA 급격한 붕괴 — 행동적 동등성 상실

레이어별 민감도 분석

투영 행렬별 민감도 순서

WQ > WK > WV > WO

Query > Key > Value > Output

Q와 K는 어텐션 점수 계산에 직접 관여하여 양자화 노이즈에 더 취약

레이어 위치 효과

  • 초기 레이어: 비교적 강건
  • 중간 레이어: 가장 큰 변화
  • 후기 레이어: 민감도 편차 큼
  • 모델 아키텍처별로 패턴 상이

핵심 발견: 비선형 브레이크포인트

비선형적 구조 왜곡

비트 폭 감소가 단조적·선형적인 왜곡을 만들지 않음. 특정 임계점(주로 4→3비트)에서 불연속적인 급증이 발생

동등성의 환상

정확도·Perplexity가 안정적인 구간(5~4비트)에서도 내부 구조와 결정 행동은 이미 유의미하게 발산하고 있음

구조-행동 연결

어텐션 가중치의 통계적/분포적 발산 정도가 Correctness Agreement 하락과 상관관계를 보임 — 내부 왜곡이 행동 변화로 전이됨

결론 및 시사점

  • 정확도와 Perplexity만으로는 양자화 모델의 행동 보존 여부를 판단할 수 없음
  • Correctness Agreement는 기존 지표와 상보적인 행동 평가 도구로 작동
  • Q, K 투영이 양자화에 가장 취약하므로, 선택적 고정밀도 유지 전략 가능
  • 비선형 브레이크포인트 존재 → 양자화 비트 선택 시 구간별 세심한 평가 필요
  • LLM 배포 시 행동적 동등성 검증을 표준 프로세스로 도입해야 함

"양자화된 모델이 원본과 '동등하다'는 믿음은 종종 환상일 뿐이다."
