Background & Motivation

Shannon의 질문, 그리고 자가 프로그래밍의 비전

"컴퓨터가 스스로 작성한 명령어를 따르게 프로그래밍할 수 있는가? 결국 컴퓨터가 직접 99%의 명령어를 작성하게 되는 것이 아닌가?"

— Claude Shannon, 자동화 프로그래밍에 관하여 (1950)

지능 vs 프로그래밍

LLM에 게임을 직접 프롬프트하면 비결정적이고, 재현 불가능하며, 디버깅이 어렵다. Shannon의 분류법은 결정적·알고리즘적 접근에 기반했다.

구조화된 프로그래밍 패러다임의 부재

LLM의 능력을 재현 가능하고 교육적이며 확장 가능한 프레임워크 안에서 체계적으로 활용하는 방법이 필요하다.

Minsky: 프로그래밍의 구조화된 규칙은 제약이 아니라, 불분명한 아이디어를 정제하는 도구다.

LLM Extension

Collaborative Prompt Engineering

크라우드소싱 기반 협력적 프롬프트 엔지니어링

  • 프로그래밍 가능한 프롬프트 — LLM 함수로 전략을 코드처럼 작성

08 / 10
Tiered Curriculum

단계별 교육 커리큘럼

단순한 사전 기반 게임에서 고급 강화학습 에이전트까지, 점진적 난이도 상승 구조를 제공한다.

Tier 1

사전 기반 — 틱택토

상태-행동 매핑을 효율적으로 압축하여 일반화된 모델 구축. LLM이 전체 게임 트리를 학습하지 않고 핵심 패턴만 추출.

Tier 2

수학적 해결 — Nim

수학적 공식(Nim-sum)을 활용하여 완벽한 전략을 계산. LLM이 기호적 추론으로 최적 해를 도출하고 사람이 읽을 수 있는 설명 생성.

Tier 3

휴리스틱 — Mancala

고전적 Minimax 알고리즘과 크라우드소싱된 플레이 데이터를 결합하여 휴리스틱 전략을 종합.

Tier 4

학습 기반 — 강화학습 에이전트

RLHF와 자기 비판, 모방 학습을 통해 시행착오로 반복적 전략 정제. 가장 고도화된 자가 프로그래밍 단계.

원격 학습 환경 평가

Nemobot을 실제 교육 환경에 배포하여, 학생들이 협력적 프롬프트 엔지니어링을 통해 AI 기초 개념을 효과적으로 습득하는 것을 확인.

게임보드 = 마이크로월드

게임판은 이산적이고 관찰 가능한 세계 상태로, LLM이 합법적 수, 시간 의존성, 숨겨진 상태, 상대 전략을 학습하는 구조화된 시험대다.
