문제: 에이전틱 MLLM의 병목

연쇄적 도구 호출의 순차 오버헤드

에이전틱 MLLM(OpenAI o3, Gemini Agentic Vision 등)은 반복적 시각 도구 호출로 뛰어난 추론 능력을 달성하지만, 인지→추론→도구 호출의 연쇄 루프가 심각한 순차 오버헤드를 발생시킵니다.

문제 1

🕐 지연 시간 폭발

에이전틱 깊이(D)에 비례하여 지연 시간이 선형 증가합니다. 각 쿼리가 여러 단계의 도구 호출을 순차적으로 실행해야 합니다.

L = Σ(d=1→D) [c_reason(d) + c_tool(d)]
문제 2

📉 동시성 붕괴

쿼리별 상태 변이로 인해 GPU 배칭이 무효화됩니다. 엄격한 데이터 의존성이 병렬 처리를 근본적으로 차단합니다.

💡 핵심 통찰

"에이전틱 MLLM으로 전달되는 쿼리의 상당 부분은 실제로 깊은 도구 기반 추론을 필요로 하지 않습니다. 대신, 경량 비도구 비전 모델이 원본 이미지만으로 올바르게 답할 수 있습니다."

Phase I–II: 판단 & 투기적 예측

Phase I — 휴리스틱 도구 사용 판단

대형 모델의 이진 분류

도구 호출 없이 대형 모델(ML)이 쿼리와 이미지를 받아 도구 필요 여부를 판정합니다.

g(q, I) = M_L(q, I; P_judge) ∈ {0, 1}

g = 0 → Phase II 투기적 예측으로 진행
g = 1 → Phase IV 에이전틱 폴백으로 직행

Phase II — 투기적 예측

소형 비도구 모델의 병렬 추론

Qwen3-VL-2B와 같은 경량 모델(MS)이 원본 이미지에서 직접 답변을 생성합니다.

ŷ_S, {logits} = M_S(q, I)
  • 완전 무상태(Stateless): 도구 호출이 없어 상태 의존성 없음
  • 배치 병렬화 가능: GPU에서 다수 쿼리 동시 처리
  • 전체 로짓 분포 출력: Phase III 게이팅에 활용

Phase III: 인지적 게이팅 — 답변 분리성

토큰 수준 분리성 점수

상위 로짓과 나머지 후보 간의 간격을 측정하여, 모델의 확신도를 스케일 불변(Scale-Invariant) 방식으로 평가합니다.

S_sep(n) = (l[1](n) − μK(n)) / (σK(n) + ε)

l[1]: 최고 로짓  |  μK, σK: 상위 K개 로짓의 평균/표준편차  |  K=64 기본값

Mean 집계

전체 토큰의 평균 점수

일반적 성능

Min 집계 ⭐

최소 점수 기준 (최악 토큰 기반)

최고 성능 달성

Bottom-r 집계

하위 r% 토큰의 평균

유연한 조절

Ssep ≥ τ → 투기적 답변 수용  |  Ssep < τ → 에이전틱 폴백

4가지 핵심 기여

기여 1

상태 의존 병목의 형식화

에이전틱 MLLM에서 데이터 의존성이 지연 시간과 동시성에 가하는 근본적 장벽을 수학적으로 증명했습니다.

기여 2

최초의 에이전틱 수준 투기적 가속 프레임워크

투기적 가속을 토큰 수준에서 에이전트 파이프라인 전체 수준으로 끌어올린 최초의 연구입니다.

기여 3

답변 분리성 기반 인지적 게이팅

레이블이 필요 없고 스케일 불변인 기준으로, 소형 모델이 자신의 예측을 신뢰할 수 있는지 판단합니다.

기여 4

이종 병렬 퍼널 설계

소형 모델의 무상태 특성을 활용하여 동시 쿼리 처리에서 처리량을 극대화하는 이종 병렬 아키텍처를 설계했습니다.
