Shayla Sharmin, Mohammad Fahim Abrar, Mohammad Al-Ratrout, Roghayeh Leila Barmaki

LLM 문제 만들기 스캐폴딩 컴퓨팅 교육

문제 만들기란 무엇인가?

학습자가 주어진 문제를 푸는 것이 아니라, 스스로 문제를 구성·정제·반성하는 능동적 학습 활동

문제 해결 vs 문제 만들기

  • 정답 도달 → 의미 있는 질문 구성
  • 알고리즘 구조, 엣지 케이스, 조합 관계에 대한 추론 요구
  • 더 깊은 개념 이해와 고차원 사고 촉진

핵심 난관

  • 학생은 자신이 만든 문제를 어떻게 개선할지 피드백 부족
  • 대형 강의에서 개별화된 피드백 제공의 현실적 제약
  • 문제 만들기에 익숙하지 않은 학생의 높은 인지 부하

연구 동기: LLM 스캐폴딩의 딜레마

LLM은 즉각적이고 대규모 피드백을 가능하게 하지만, 인지 지원과 인지 외주화(cognitive offloading) 사이의 긴장 존재

과도한 직접 지원의 위험

학생이 생성된 출력을 충분히 평가 없이 수용하게 됨. 시간이 지날수록 더 단순한 질문만 하게 되는 경향

과도한 간접 지원의 위험

개방형 안내만으로는 학생이 불확실성을 느끼고, 개선 방향을 파악하지 못해 좌절할 수 있음

실질적 설계 질문: "도움을 주는가?"가 아니라 "도움을 시스템과 학습자 사이에 어떻게 분배하는가?"

연구 질문 및 기여

연구 질문

서로 다른 형태의 LLM 스캐폴딩이 컴퓨팅 문제 만들기 과정에서 학생의 참여와 문제 품질에 어떻게 영향을 미치는가?

블룸 분류법 루브릭 기반 LLM 스캐폴딩 시스템 2종의 설계 및 프롬프트 공개
2

3
참가자 인식 분석 및 직접·간접 스캐폴딩의 순차적 활용에 대한 실천적 지침 도출

시스템 설계: 두 스캐폴딩 조건

두 시스템은 동일한 블룸 분류법 기반 평가 프레임워크 사용 · 출력 형태만 다름

간접 스캐폴딩 (Indirect)

안내 질문(Guiding Questions) 제공

  • "이 문제의 엣지 케이스는 무엇인가요?"
  • "제약 조건을 어떻게 구체화할 수 있나요?"
  • 학습자가 직접 개선 방향을 탐색
  • 반성과 자기 수정 촉진

직접 스캐폴딩 (Direct)

모범 예시(Worked Examples) 제공

  • 개선된 문제 버전의 구체적 예시 제시
  • "다음과 같이 제약을 추가할 수 있습니다: ..."
  • 즉시 실행 가능한 개선안 제공
  • 인지 부하 감소 효과

블룸 분류법 기반 평가 프레임워크

학생이 만든 문제를 블룸 분류법의 하위 수준에서 상위 수준으로 이끄는 기준 적용
블룸 수준 문제 만들기에서의 의미 스캐폴딩 목표
기억/이해 단순 재생형 문제 더 높은 수준으로 견인
적용 주어진 알고리즘을 활용한 문제 분석적 요소 추가 유도
분석 구조·관계·엣지 케이스를 포함한 문제 평가 기준 명확화 지원
평가/창조 다중 해결 경로, 창의적 제약이 있는 문제 정교화 및 심화 촉진
→ 두 조건 모두 동일한 루브릭으로 문제를 평가하되, 피드백의 형태만 다르게 설계

결과 1: 문제 품질 개선

조작 검증 통과 — 참가자가 두 조건의 차이를 정확히 인지
Direct

더 큰 즉각적 개선

모범 예시를 통한 구체적 개선안 제공로
첫 수정에서 유의미한 품질 상승

Indirect

개선은 있으나 보다 점진적

안내 질문을 통해 자율적 탐색 유도
즉각적 변화는 덜하지만 과정적 학습 기대

시나리오별 효과

낮은 개방성 시나리오(데이터 구조)에서 두 조건 모두 더 큰 개선 효과 · 높은 개방성 시나리오(알고리즘)에서는 개선 폭이 상대적으로 작음 — 작업 구조가 스캐폴딩 효과를 조절

결과 2: 품질 차원별 분석

품질 차원 Direct 스캐폴딩 Indirect 스캐폴딩
명확성 높은 개선 — 모범 예시로 직접 시연 보통 — 스스로 재구성 필요
복잡도 높은 개선 — 구체적 확장 예시 제공 보통 — 방향은 잡으나 구체화 어려움
실현 가능성 보통 — 예시가 항상 학생 맥락에 부합하지 않음 높은 개선 — 학생이 자신의 맥락에 맞게 조정
창의성 보통 — 모범 예시에 의존 경향 높은 개선 — 자율적 탐색으로 독창적 결과
→ Direct는 명확성·복잡도에서 강점, Indirect는 실현 가능성·창의성에서 강점

결과 3: 사용자 경험 및 인터뷰

Direct에 대한 인식

  • 더 효율적이라고 느낌
  • 개선 방향이 명확하여 불안감 감소
  • 다만 "AI가 만든 것 같다"는 우려도 존재
  • 예시에 과의존하는 경향 보고

Indirect에 대한 인식

  • 더 깊은 반성을 촉진한다고 평가
  • 문제 설계 과정에 대한 메타인지 향상
  • 일부 참가자는 "무엇을 해야 할지 모르겠다"고 호소
  • 시간이 더 오래 걸린다고 느낌
"직접 스캐폴딩은 빠르게 고쳐주지만, 간접 스캐폴딩은 내가 왜 이 문제를 이렇게 만들었는지 생각하게 해줬어요."

논의: 상보적 역할과 시간적 역학

서로 다른 학습 과정 지원

Direct는 수정 실행에 유리하고, Indirect는 반성과 메타인지에 유리. 두 조건은 경쟁이 아닌 보완 관계

시간적 역학의 중요성

스캐폴딩 효과는 작업 구조와 시점에 따라 달라짐. 개방성 정도, 학습자 숙련도, 세션 내 시간 위치가 조절 변수

예상치 못했던 점

높은 개방성 시나리오에서 두 조건 모두 개선 폭이 작았음 — 스캐폴딩만으로는 제한적이며, 시나리오 자체의 구조화가 필요할 수 있음

소유권 문제

LLM 스캐폴딩이 학습자의 문제 소유감을 반드시 감소시키지는 않음 — Indirect 조건에서는 오히려 소유감이 강화됨

한계 및 주의점

  • 장기 학습 효과 미측정
  • 자기 보고 편향 가능성

결론

  • 동일한 루브릭을 사용하더라도 스캐폴딩 출력 형태가 학습 경험을 상이하게 만듦
  • Direct 스캐폴딩은 즉각적 품질 개선에 효과적 (명확성, 복잡도)
  • Indirect 스캐폴딩은 더 깊은 반성과 메타인지를 촉진 (실현 가능성, 창의성)
  • 두 방식은 상보적이며, 순차적 활용이 실천적 전략으로 제안됨
  • 작업 구조(개방성 정도)가 스캐폴딩 효과를 조절하는 중요한 변수
LLM 기반 학습 지원의 설계 핵심 질문: "도움을 주는가?"가 아니라 "도움을 시스템과 학습자 사이에 어떻게 분배하는가?"
