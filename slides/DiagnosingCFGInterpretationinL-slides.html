Background

왜 CFG 해석 능력을 평가하는가?

에이전트 시스템의 핵심 요구

LLM은 도구 호출, API 오케스트레이션, DSL 생성을 위해 동적으로 정의된 기계 인터페이스(JSON 스키마, 함수 시그니처)를 준수해야 함

"거의 맞음"은 충분하지 않다

단일 구분자 누락이나 중첩 루프의 논리 불일치가 치명적 실행 실패를 유발할 수 있음 (Yehudai et al., 2025)

핵심 질문

LLM은 제공된 문법의 형식적 논리를 내면화하는가, 아니면 메모리에서 표면 규칙성만 검색하는가?

Grammar

EBNF 기반 문법 규격

start : stmt+ stmt : action_stmt | loop | if_stmt action_stmt : DO action END loop : LOOP INT TIMES LBR stmt+ RBR if_stmt : IF cond THEN LBR stmt+ RBR (ELSE LBR stmt+ RBR)? action : MOVE MOVE_DIR INT? | TURN TURN_DIR | GRAB ITEM cond : HOLDING ITEM
DO : "exec" END : "end" LOOP : "loop" TIMES : "times" IF : "when" THEN : "after" LBR : "[" RBR : "]"

재귀적 합성

loop, if_stmt이 stmt+를 재귀적으로 포함하여 중첩 허용

명시적 구분자

DO/END, LBR/RBR이 재귀 수준 추적을 위한 앵커 제공

동적 단말 기호

"loop" → "v_xkqm" 등 불투명 토큰으로 치환하여 의미적 선험지식 차단

인컨텍스트 해석기

EBNF가 추론 시점에 동적 프로토콜로 작동

핵심 발견

급격한 비선형 감소: 깊이 증가에 따라 성능이 급격히 붕괴하며, 이는 LLM의 깊은 추론 사슬 유지 한계와 일치 (Rodkin et al., 2025)

Alien 토큰 효과: 익숙한 어휘를 불투명 토큰으로 교체하면 성능이 급감하여, 순수 기호 귀납이 아닌 의미적 선험지식 의존을 입증

CoT의 부분적 완화: 사고사슬(CoT) 추론이 어느 정도 도움이 되지만, 구조적 밀도(깊은 재귀, 높은 분기) 앞에서는 여전히 붕괴

Ablation

절제 연구 분석

계층 깊이와 재귀 스케일링

D=2에서는 높은 성능을 보이나, D≥6에서 비선형 감소 시작

Else 분기 확률(p) 증가 시 분기 밀도에 비례하여 BER, SCR 추가 저하

술어 복잡도와 논리 해결

표현식 깊이(E) 증가는 지역적 논리 해결 실패로 이어짐

깊은 재귀 실패와 구분 가능: 중첩 And/Or/Not 술어에서 독립적 저하 패턴 관찰

구조적 구분자와 어휘 선험지식

Block → C-style → S-expr 전환 시 격식 차이 발생

Alien 어휘 사용 시 모든 스타일에서 극적인 성능 하락 → 순수 기호 귀납 한계

Prompting

프롬프팅 전략과 ICL 효율성

새로운 CFG에서 CoT의 지배적 우위

Zero-shot 대비 CoT가 새로운 문법 해석에서 가장 큰 개선을 제공하나, 이는 표면 구문 수준에 국한

Few-shot 프롬프팅의 한계적 수익

예제 수 증가가 초기에는 도움이 되나, 수확 체감이 빠르게 나타남 — 구조적 이해가 아닌 패턴 모방에 의존하는 것으로 추정

깊이에서의 보편적 취약성

모든 프롬프팅 전략이 깊은 중첩(D ≥ 8)에서 보편적으로 붕괴 — 이는 프롬프트 엔지니어링으로 극복 불가한 근본적 한계를 시사

Failure Modes

실패 모드 분류와 사례

구문 취약성

Syntactic Fragility

  • 어휘 매핑 실패: 불투명 토큰 혼동
  • 계층 스코핑 실패: 중첩 구분자 불일치
  • 연산자 혼동: 유사 토큰 간 오인식
  • 조기 스코프 종료: 닫는 괄호 누락

행동 불일치

Behavioral Misalignment

  • 논리 술어 반전: And/Or 혼동
  • 산술 시뮬레이션 붕괴: 계산 오류 누적
  • 계획 수평 붕괴: 다단계 목표 달성 실패

의미 드리프트

Semantic Drift

  • 산술 AST 평탄화: 중첩 구조 손실
  • 불리언 AST 평탄화: 논리 깊이 붕괴
  • 자기회귀 에코: 구조적 중복 생성

핵심 통찰: 구문적으로 유효한 출력이라도 내부 AST 구조가 평탄화되어 원래 논리를 보존하지 못하는 현상이 광범위하게 관찰됨

Conclusion

결론과 시사점

핵심 메시지

LLM은 CFG를 표면적으로는 따를 수 있으나, 복잡도가 증가하면 논리적 충실도가 붕괴한다. 이는 순수 기호 귀납이 아닌 의미적 선험지식에 의존하기 때문이다.

계층적 상태 추적의 한계

깊은 재귀와 높은 분기에서 계층적 상태 추적 능력이 보편적으로 실패하며, 이는 신뢰 가능한 문법 무관 에이전트 구현의 핵심 병목이다.

향후 방향

  • 순수 기호 귀납 능력 향상을 위한 아키텍처 연구
  • 깊은 중첩 구조 처리를 위한 체인-오브-사고 이상의 추론 메커니즘
  • 문법 무관 에이전트를 위한 검증-수정 루프 통합
