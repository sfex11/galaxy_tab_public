생성형 AI 에이전트에서의
차분 프라이버시 분석과
최적 트레이드오프

LLM 기반 AI 에이전트가 기업 데이터베이스에 접근할 때 발생하는 프라이버시 누출을 차분 프라이버시 프레임워크로 분석하고, 생성 파라미터의 최적 설계를 제시합니다.

Ya-Ting Yang Quanyan Zhu

Cryptography and Security (cs.CR) · Artificial Intelligence (cs.AI)

Problem Definition

문제 정의: AI 에이전트의 프라이버시 위험

🏢
기업 데이터 접근
LLM과 AI 에이전트가 내부 데이터베이스에 접근하여 맥락 인식 응답을 생성하는 구조가 확산되고 있습니다.
⚠️
민감 정보 노출 위험
모델 출력이 의도치 않게 민감한 기업 데이터를 드러낼 수 있으며, 이에 대한 체계적 분석이 부족합니다.
🔍
기존 연구의 한계
대부분의 기존 연구는 사용자 프롬프트의 프라이버시 보호에 초점을 맞추고 있어, 기업 데이터 관점의 분석이 미흡합니다.
📊
정량적 프레임워크 부재
생성 파라미터(온도, 메시지 길이 등)가 프라이버시 누출에 미치는 영향을 정량화할 수 있는 이론적 도구가 필요합니다.
🎯
핵심 연구 질문
AI 에이전트의 응답 생성 메커니즘을 확률적 프로세스로 모델링하고, 차분 프라이버시(Differential Privacy) 기반으로 프라이버시-유용성 간 최적 트레이드오프를 어떻게 달성할 것인가?
Approach

제안하는 해결책

🧮 이론적 프레임워크
확률적 메커니즘 모델링 — 응답 생성을 프롬프트와 데이터셋을 토큰 시퀀스 분포로 매핑하는 확률적 메커니즘으로 정의
토큰 수준 DP — 개별 토큰 생성 단계에서의 차분 프라이버시를 분석하여 미시적 프라이버시 보장 도출
메시지 수준 DP — 전체 응답 메시지에 대한 차분 프라이버시를 정의하여 거시적 프라이버시 보장 제공
⚙️ 최적화 설계
프라이버시 바운드 도출 — 프라이버시 누출과 온도(temperature), 메시지 길이 간의 관계를 수학적으로 규명
프라이버시-유용성 최적화 — 프라이버시 보호와 응답 품질 사이의 최적 균형점을 찾는 설계 문제 공식화
최적 온도 선택 — 주어진 프라이버시 예산 하에서 유용성을 극대화하는 최적 온도 파라미터 결정
핵심 수학적 개념
Pr[M(D, x) ∈ S] ≤ eε · Pr[M(D', x) ∈ S] + δ
(ε, δ)-차분 프라이버시: 인접 데이터셋 D, D'에 대한 메커니즘 M의 출력 분포 차이를 ε로 제한

토큰 수준 분석: 각 토큰 생성 단계에서 softmax 확률 분포의 차분 프라이버시를 분석하고, 온도 τ가 프라이버시 보장에 미치는 직접적 영향을 정량화
메시지 수준 합성: 토큰 수준 DP를 조합 정리(composition theorem)를 통해 전체 메시지 수준으로 확장하여, 메시지 길이에 따른 프라이버시 열화를 분석
최적화 공식화: 프라이버시 예산 ε 하에서 유용성 함수를 최대화하는 최적 온도 τ*를 구하는 볼록 최적화 문제로 정의
실용적 가이드: 기업 환경에서 AI 에이전트 배포 시, 프라이버시-유용성 트레이드오프를 체계적으로 관리하기 위한 설계 원칙 제공

균형점 달성
  • 온도-프라이버시 관계: 높은 온도(τ)는 출력 분포를 평탄화하여 프라이버시 보호를 강화하지만, 응답 품질(유용성)이 저하됨
  • 길이-프라이버시 관계: 긴 응답은 더 많은 토큰을 생성하므로 조합 효과(composition effect)에 의해 프라이버시 예산이 빠르게 소진됨
  • 최적 설계 존재: 주어진 프라이버시 제약 조건 하에서 유용성을 극대화하는 유일한 최적 온도 τ*가 존재함을 증명
  • 실용적 가이드라인: 기업이 AI 에이전트를 배포할 때, 데이터 민감도에 따라 온도와 응답 길이를 체계적으로 조절할 수 있는 방법론 제시

본 연구는 생성형 AI 에이전트의 프라이버시를 차분 프라이버시 프레임워크로 최초 분석하고,
온도와 메시지 길이라는 실질적 파라미터를 통한 최적 프라이버시-유용성 트레이드오프 설계를 가능하게 합니다.
