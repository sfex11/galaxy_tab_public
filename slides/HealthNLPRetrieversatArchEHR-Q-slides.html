문제 제기

환자는 EHR에 접근할 수 있지만,
이해할 수는 없다

환자의 질문

  • 장문적이고 비형식적인 표현
  • 감정적 표현이 많음
  • 의학적 용어 부족

임상 기록

  • 밀집되고 파편화된 구조
  • 전문 기술 용어 사용
  • 증거가 여러 구간에 분산
접근성 ≠ 이해도. 환자 언어와 임상 문서 간의 의미론적 격차를 해소해야 한다.

Stage 1

Few-shot 질문 해석

검색 전 단계에서 환자 의도를 임상적으로 재해석

설계 전략

  • 임상 행정 보조원 페르소나 부여
  • 훈련셋 3개 샘플 기반 Few-shot
  • 전문문 금지, 대명사 표준화
  • 진단적 언어 사용 금지
  • 후처리: 최대 15단어 강제 truncation

변환 예시

입력: "제가 아침에 일어났을 때 너무 어지러워서 걸을 수도 없었고, 간호사님한테 말했더니 혈압이 낮다고 하셨는데 이게 제 약 때문인가요?"
출력: "아침 어지럼증과 저혈압의 약물 관련성"
감정적 맥락을 보존하면서 검색 노이즈를 최소화하는 것이 핵심
Stage 2

휴리스틱 기반 증거 평가

문장 수준 Likert 척도 점수 부여 + 동적 리콜 편향 필터링

문장 관련성 점수 척도

점수 등급 기준
5 핵심 직접적 답변, 확정 진단, 특정 약물 결과
4 높은 맥락 핵심 소견의 한정 정보(검사 결과 맥락 등)
3 관련 배경 정보, 입원일, 임상 병력
1–2 무관/약함 정상 소견, 행정 헤더, 무관 계통

동적 폴백 메커니즘

1
엄격 모드: 점수 ≥ 4 문장만 선별
2
엄격 모드 결과 없음 → 관대 모드: 점수 ≥ 3으로 확장
3
API 오류 시 → 노트 첫 3문장을 폴백으로 사용
Stage 3

접지된 답변 생성

임상 문서 전문가 페르소나 + 3블록 RAG 구조

제약 조건

  • 최대 75단어 제한
  • 객관적 임상 어휘만 사용
  • 대화형 서문 완전 제거
  • 외부 의학 지식 삽입 금지

Soft-cut Truncation

75단어 초과 시 마지막 완전한 문장 경계에서 자르는 휴리스틱 적용으로 불완전한 문장 방지

3블록 RAG 프롬프트 구조

Block 1
질문의 임상적 해석
Stage 1 출력
Block 2
필터링된 증거 텍스트
Stage 2 출력 (최우선)
Block 3
전체 임상 노트
보조 컨텍스트
Stage 4

답변-증거 정렬

다대다(Many-to-Many) 정밀 매핑 프레임워크

설계 철학

  • 단일 문장이 여러 증거를 지지 가능
  • 단일 증거가 여러 답변 구절에 기여 가능
  • 정밀도(Precision) 최우선 설계
  • 거짓 양성(FP) 최소화 목표

왜 다대다인가?

임상 답변은 복합적 정보를 압축하여 표현하는 경우가 많음. 예: "혈압 강하제를 조정하고 수액을 추가 투여함"이라는 한 문장은 약물 기록과 수액 기록 두 개의 독립적 증거에 동시에 정렬됨.

표준 RAG는 문서/구절 수준이지만, 임상 증거 접지는 문장 수준 정밀도를 요구한다.

질문 해석에서 1위를 달성한 것은, 검색 전 해석 단계가 파이프라인 전체 성능의 핵심 동력임을 확인시킨다.
분석

절제 연구 및 논의

절제 연구 핵심 발견

  • Stage 1(질문 해석) 제거 시 전체 성능 하락 가장 큼
  • 동적 폴백이 증거 기아 현상 방지에 핵심적 역할
  • 전체 노트(Block 3) 보조 컨텍스트의 추가 효과 확인
  • Soft-cut truncation이 답변 완결성에 유의미

논의

  • 태스크 분해가 모델 규모보다 더 큰 영향력
  • 정렬 순위(9위) 개선이 향후 주요 과제
  • CUNI 연구와 일치: 프롬프트 설계 > 모델 크기
  • 환자 대면 건강 커뮤니케이션 품질 향상 확인

핵심 인사이트

단일 LLM 호출로 모든 것을 해결하려는 접근보다, 각 단계를 명확한 역할로 분해한 캐스케이드 구조가 임상 QA에서 접지, 정밀도, 전문성을 동시에 달성하는 데 더 효과적이다.

한계 및 윤리

제한 사항과 윤리적 고려

기술적 한계

  • MIMIC-III 중환자 데이터에만 검증
  • 다국어 환자 질문 미지원
  • 단일 모델(Gemini) 의존성
  • 정렬 정밀도 향상 필요
  • 실시간 환자 포털 배치 미검증

윤리적 고려

  • 환자 데이터는 비식별화된 공개 데이터셋 사용
  • 생성 답변은 정보 제공 목적, 진단 대체 불가
  • LLM 환각 위험에 대한 인지와 경고 필요
  • 임상 실무 적용 전 의료 전문가 검토 필수
자동화된 QA 시스템은 의사를 대체하는 것이 아니라, 환자-의사 간 소통의 질을 높이는 보조 도구로 설계되어야 한다.
결론

핵심 요약

1
페르소나 기반 질문 해석 — 검색 전 단계에서 환자 의도를 임상적으로 재구성하는 것이 파이프라인 성능의 핵심 동력
2
리콜 편향 동적 필터링 — Likert 척도 기반 점수화와 3단계 폴백으로 의료 분야에서 치명적인 위음성 문제 완화
3
3블록 구조화된 RAG — 필터링 증거를 최우선으로, 전체 노트를 보조로 활용하는 계층적 컨텍스트 설계
4
다대다 정밀 정렬 — 임상 답변의 복합적 특성을 반영한 Many-to-Many 매핑으로 정밀도 극대화
구조화된 다단계 파이프라인이 단일 패스 LLM 생성보다 임상 QA에서 접지, 정밀도, 전문성을 동시에 향상시킨다.
