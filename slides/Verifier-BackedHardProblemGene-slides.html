Verifier-Backed Hard Problem Generation
for Mathematical Reasoning

검증기 기반 난제 생성 프레임워크 VHG

Yuhang Lai, Jiazhan Feng, Yee Whye Teh, Ning Miao

City University of Hong Kong · Peking University · University of Oxford

동기: LLM이 풀 수는 있지만, 낼 수는 있는가?

LLM은 GPQA-Diamond, AlphaGeometry 등에서 전문가 수준의 문제 해결 능력을 보여주었지만, 유효하고 도전적인 새로운 문제를 생성하는 데에는 큰 어려움이 있습니다.

문제 생성의 중요성

  • 과학 연구에서 새로운 질문을 제기하는 것은 해결만큼 중요
  • LLM 훈련 데이터의 난이도를 지속적으로 높이는 핵심 경로
  • 완전한 자율 연구 주기를 달성하기 위한 필수 조건

현재 상황의 한계

  • 정적 인간 작성 데이터셋(MATH 등)에 의존
  • 도메인 전문가 고용은 비용과 시간이 많이 소요
  • LLM의 능력이 인간 수준으로 제한됨

기존 접근법과 치명적 결함

스캐폴드 기반

PromptCoT, CHASE, MathSmith 등 인간 설계 구조에 의존 → 인간 설계로 상한 존재

변환/융합 기반

MathFusion 등 기존 문제를 결합하는 방식 → 여전히 인간 설계 전략에 종속

순수 자가 대결

Setter-Solver 이인자 게임 → 보상 해킹(Reward Hacking) 문제

핵심 문제: 순수 자가 대결에서 Setter는 무효한 문제를 생성하여 Solver가 틀리게 만들면 높은 보상을 받습니다. 즉, 어려운 문제를 만든 것이 아니라 풀 수 없는 문제를 만드는 것으로 보상을 해킹합니다.

VHG 프레임워크: 세 주체 자가 대결

기존 Setter-Solver 이인자 구조에 독립적인 Verifier를 추가하여, 유효성과 난이도를 분리하고 보상 해킹을 원천 차단합니다.

Setter 측 효과

높은 보상은 문제의 진정한 난이도에서만 비례됨 → 보상 해킹 불가

세 구성 요소의 상세 역할

Setter Q

문제 x와 참조 해답 y*을 함께 생성. 기존 자가 대결과 달리 해답도 출력에 포함하여 검증 가능성 확보. 시드 문제-정답 쌍을 조건으로 한 LLM으로 구현.

Solver S

생성된 문제 x에 대해 해결 시도. 고정 샘플링 예산 하에서 경험적 정확도 Acc_S(x, y*)를 난이도 신호로 제공. 핵심: Verifier가 먼저 승인한 경우에만 Setter 보상에 기여

Verifier V

생성된 (x, y*) 쌍이 사전 정의된 수용 기준을 만족하는지 검증. Hard Verifier(기호적 검증)와 Soft Verifier(LLM 기반 검증) 두 가지 변형 존재.

훈련 메커니즘: 보상 함수

RQ(x, y*) = 𝟏[V(x,y*)=1] × (1 − AccS(x, y*))

지시 함수의 역할

V(x, y*) = 0일 때, 즉 Verifier가 거부하면 Solver의 실패가 보상에 전혀 기여하지 않음. 유효성과 난이도의 명시적 분리.

난이도 신호

Solver 정확도가 낮을수록 보상이 높아짐. 단, 반드시 유효한 문제에 대해서만. 무효 문제로 높은 보상을 얻는 경로가 차단됨.

훈련 루프 요약

  • Setter가 문제-정답 쌍 배치 생성
  • Verifier가 각 쌍의 유효성 판정 → 거부된 쌍은 폐기
  • 승인된 쌍에 대해 Solver가 정확도 측정
  • 보상 함수에 따라 Setter 강화학습 업데이트
  • 승인된 쌍은 Solver 훈련 데이터로도 활용

두 가지 Verifier 변형

Hard Verifier

기호적 검증 매커니즘 활용

  • SymPy 라이브러리로 부정적분 검증
  • 제안된 부정적분을 미분하여 피적분함수와 일치하는지 확인
  • 거의 100% 신뢰성 보장
  • 적용 가능 도메인이 제한적
적용: 부정적분

Soft Verifier

LLM 기반 단계별 검증

  • 문제 생성 과정의 단계별 정확성 검사
  • Hard Verifier보다 검증 정확도는 낮음
  • 광범위한 수학 도메인에 적용 가능
  • 기호적 검증이 불가능한 일반 수학에 필수
적용: 일반 수학

실험 설정

부정적분 실험

  • Hard Verifier (SymPy) 사용
  • 벤치마크: AntiderivBench Qualifier / Competition, Integration Stress Test
  • 기본 모델: Qwen3-4B (Setter & Solver 동일)

일반 수학 실험

  • Soft Verifier (LLM 기반) 사용
  • 벤치마크: MATH, AMC, Minerva, Olympiad, AIME24-26
  • 기본 모델: Qwen3-4B (Setter & Solver 동일)

비교 대상 (Baselines)

R-Zero (자가 대결 SOTA), 오프라인 데이터 증강 방법들 (MetaMath, WizardMath, ToRA, DeepSeekMath), 순수 자가 대결 (Verifier 없이)

약한 모델이 강한 모델을 위한 데이터 생성

Setter와 Solver 모두 Qwen3-4B 기반이지만, VHG가 생성한 문제는 Qwen3-8B, 14B, 32B 같은 더 큰 모델도 도전하게 만듦. 약한 모델이 강한 모델의 훈련에 기여할 수 있음을 시사.

분석: VHG가 작동하는 이유

Setter 학습 역학

Verifier가 있는 경우, Setter는 점진적으로 진정으로 어려운 문제 생성 방향으로 학습. Verifier가 없는 경우, 초기에는 난이도가 올라가다가 무효 문제 비율이 급증하며 붕괴.

생성 분포 분석

VHG는 유효성-난이도 영역에서 균형 잡힌 분포를 유지. 순수 자가 대결은 고난이도-무효 영역으로 편향된 분포를 보임.

핵심 발견

  • Verifier는 단순한 필터가 아니라 Setter의 학습 방향을 결정하는 핵심 요소
  • 유효성과 난이도의 분리가 보상 해킹 방지의 근본 원인
  • 생성된 문제의 신규성(Novelty)도 확보 — 기존 데이터셋과의 중복 낮음

주요 기여 및 시사점

1

새로운 삼인자 자가 대결 프레임워크

Verifier를 추가한 세 주체 구조로 무효 문제 생성으로 인한 훈련 파이프라인 붕괴를 방지하는 최초의 설계.

2

Hard/Soft Verifier를 통한 포괄적 실증 연구

기호적 검증이 가능한 부정적분부터 일반 수학까지, VHG가 신규·난이도 높은 문제를 생성하여 다중 벤치마크 성능을 향상시킴을 입증.

3

Verifier의 작동 메커니즘 심층 분석

Verifier가 생성 문제 품질 보장에 미치는 결정적 역할을 학습 역학과 분포 분석을 통해 규명.

약은 모델(4B)이 강한 모델(32B)을 위한 유용한 훈련 데이터를 생성할 수 있다는 발견은, 자율적이고 확장 가능한 데이터 생태계 구축의 가능성을 시사합니다.

한계점 및 향후 방향

현재 한계

  • Soft Verifier는 완벽하지 않은 검증 정확도 가짐
  • 수학 외 도메인(물리, 화학 등)으로의 일반화 미검증
  • Setter와 Solver의 규모가 동일한 설정만 실험
  • 생성 문제의 다양성 측정에 추가 연구 필요

향후 방향

  • 더 강력한 검증기 아키텍처 탐색 (formal verification 통합)
  • 과학 전반으로의 확장 (정리 증명, 실험 설계 등)
  • 다규모 Setter-Solver 구성 실험
  • 자율 연구 주기 전체 구현을 향한 통합 시스템
