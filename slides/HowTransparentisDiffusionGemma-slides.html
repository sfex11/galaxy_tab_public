투명성의 두 가지 구성 요소

변수 투명성 (Variable Transparency)

모델의 중간 계산 상태 스냅샷을 이해할 수 있는가?

  • 중간 상태가 자연어로 표현 가능한가?
  • 정보 병목을 해석 가능한 형태로 매핑할 수 있는가?
  • 매핑 시 성능 저하가 발생하는가?

알고리즘 투명성 (Algorithmic Transparency)

스냅샷을 통해 출력에 도달하는 과정을 재구성할 수 있는가?

  • 토큰 예측의 순서와 의존성을 추적할 수 있는가?
  • 분산 알고리즘이 어떻게 실행되는지 이해할 수 있는가?
  • 디노이징 과정 중 의사결정 흐름을 파악할 수 있는가?

본 연구는 이 두 축을 따라 DiffusionGemma의 투명성을 체계적으로 분석합니다.

불투명 직렬 깊이 (Opaque Serial Depth)

해석 가능한 노드를 포함하지 않는 가장 긴 직렬 계산 경로의 길이. 모델이 얼마나 많은 비투명 계산을 수행할 수 있는지의 상한을 나타냄.

정의

신경망(샘플링 절차 포함)과 동일한 함수를 계산하는 불리언 회로에서, 해석 가능한 노드를 포함하지 않는 가장 긴 경로

정보 병목

디노이징 스텝 사이의 정보 병목은 C개 자연어 토큰 + C개 자기조건화 벡터로 구성됨

핵심 질문: 이 정보 병목을 "해석 가능하다"고 가정할 수 있는가?

불투명 직렬 깊이: 핵심 결과

28.6X

병목 미해석 가정 시

자기조건화 벡터를 해석 불가능하다고 가정하면, DiffusionGemma의 불투명 직렬 깊이는 Gemma 4 대비 28.6배

1.1X

병목 해석 가정 시

중간 상태를 해석 가능하다고 가정하면, 불투명 직렬 깊이는 Gemma 4와 거의 동일한 1.1배 수준

이 결과는 중간 병목의 해석 가능성 여부가 투명성 평가의 핵심 분기점임을 보여줍니다. 다음 섹션에서 이 가정을 검증합니다.

변수 투명성: 병목 절제 실험

디노이징 스텝 사이의 정보 흐름을 O(c)개 자연어 토큰으로 축소해도 성능 저하가 최소인지 검증

실험 설계

  • 자기조건화 벡터 S^t를 최근접 임베딩 토큰으로 매핑
  • 매핑된 토큰 수를 점진적으로 축소 (C → O(c))
  • 다운스트림 태스크 성능 측정

핵심 발견

정보 흐름을 O(c)개 자연어 토큰으로 축소해도 하위 성능 감소 없이 유지됨. 이는 병목에 포함된 정보의 상당 부분이 자연어로 표현 가능함을 시사

변수 투명성: 상위 토큰 분석

매핑된 O(c)개 토큰이 실제로 무엇을 의미하는지 분석

관측 결과

  • 상위 토큰들은 주로 인근 위치 토큰에 대한 예측
  • 자기조건화 벡터가 임베딩 공간으로 편향되도록 훈련됨
  • 캔버스의 특정 위치에 대한 "현재 추측"을 인코딩

해석 가능성 의미

매핑된 토큰이 단순한 잡음이 아니라 의미 있는 자연어 추측이므로, 중간 상태를 인간이 해석 가능한 병목으로 간주할 수 있는 근거가 됨

이로써 불투명 직렬 깊이 1.1X 가정이 정당화됨

모니터링 가능성 (Monitorability)

투명성의 핵심 응용 지표: 모델 출력이 하위 태스크에 유용한 정보를 포함하는가?

평가 방법

Guan et al.의 모니터링 가능성 평가 제도를 DiffusionGemma에 적용. 모니터 모델이 사고과정에서 정보를 추출하는 능력을 측정.

모델 모니터링 가능성 비교 결과
Gemma 4 (자가회귀) 기준선 —
DiffusionGemma (확산) 기준선과 유사 거의 동등

DiffusionGemma의 사고과정이 모니터링 관점에서 자가회귀 모델과 동등한 정보량을 제공함

알고리즘 투명성: 확산 모델의 고유 현상

자가회귀 모델과 근본적으로 다른 디노이징 과정의 특성으로 인해 발생하는 새로운 현상들

비시계열적 추론

시간 순서를 따르지 않는 추론: 조기 응답 길이 예측, 소급 자기 교정, 비자가회귀 코드 생성

토큰·시퀀스 번짐

단일 토큰의 정보가 여러 위치에 분산되거나, 시퀀스 전체에 걸쳐 정보가 퍼지는 현상

중간 맥락 추론

디노이징 중간 단계에서 캔버스 내용을 맥락으로 활용하는 추론 방식

비시계열적 추론 사례 연구

조기 응답 길이 예측

디노이징 초기 단계에서 이미 최종 응답의 길이를 예측하고 캔버스 크기를 조정. 자가회귀 모델은 토큰을 순차적으로 생성하므로 불가능한 현상

소급 자기 교정

이미 확정된 것처럼 보이는 앞부분 토큰을 나중 디노이징 스텝에서 수정. 자가회귀 모델에서는 한 번 생성된 토큰이 변경되지 않음

비자가회귀 코드 생성

함수 정의 이전에 함수 호출부를 먼저 생성한 뒤, 이후 스텝에서 정의를 채워 넣는 방식. 순차적 생성이라는 제약에서 자유로움

이러한 현상들은 알고리즘 투명성을 자가회귀 모델보다 더 어렵게 만드는 원인이 됨

토큰·시퀀스 번짐 현상

토큰 번짐 (Token Smearing)

단일 토큰의 정보가 캔버스의 여러 위치에 분산되어 나타남. 자가회귀 모델에서는 한 위치 = 하나의 토큰이 명확히 대응되지만, 확산 모델에서는 이 대응 관계가 모호해짐

시퀀스 번짐 (Sequence Smearing)

토큰 시퀀스 전체의 정보가 캔버스 전체에 걸쳐 확산. 특정 문장이나 구절의 경계가 디노이징 과정에서 흐려질 수 있음

모든 토큰 예측이 매 디노이징 스텝마다 변경될 수 있다는 사실이 모델이 복잡한 분산 알고리즘을 구현할 수 있는 능력의 원천이자, 해석 가능성의 장애물이 됨

남은 문제와 한계

미해결 문제

  • DiffusionGemma 행동의 추가 이해
  • 표준 기계론적 해석가능성 도구의 적용
  • 사고과정 연구의 확산 모델로의 확장
  • 모델 생물체 연구 (Model Organism)

주요 한계

  • 결과가 DiffusionGemma의 특정 아키텍처에 의존
  • 자기조건화 벡터가 임베딩으로 편향되는 훈련 방식에 기인
  • 결과 기반 RL 대량 적용 시 잠재 공간이 불투명해질 수 있음
  • 미래 모델에 대한 일반화 불보장

모델 개발자는 본 연구의 방법론을 활용해 향후 모델의 투명성 및 모니터링 가능성 퇴보를 지속적으로 평가해야 한다.

결론

1

불투명 직렬 깊이

중간 병목을 해석 가능하게 매핑하면 28.6X → 1.1X로 급감. Gemma 4와 거의 동등한 수준

2

변수 투명성

자기조건화 벡터를 O(c)개 자연어 토큰으로 축소해도 성능 유지. 매핑된 토큰은 의미 있는 추측

3

모니터링 가능성

DiffusionGemma가 Gemma 4와 동등한 모니터링 가능성을 보여줌

4

알고리즘 투명성

비시계열적 추론, 번짐 현상 등 확산 특유 현상 발견. 자가회귀보다 해석 난이도 높으나, 초기 연구 결과 제시
