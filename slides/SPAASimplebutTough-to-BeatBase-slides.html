01 / 문제 정의

LLM의 지식은 불완전하다

🧠
LLM의 한계
대규모 사전학습에도 불구하고, 전문 도메인에서의 지식 커버리지가 부족
📊
데이터 부족
특수 분야는 학습 데이터가 극도로 희소하여 직접 파인튜닝 시 과적합 위험
핵심 질문

소규모 도메인 코퍼스로부터 어떻게 효과적으로 대규모 합성 데이터를 생성하여 LLM에 지식을 주입할 수 있는가?

기존 접근법들의 문제:

  • RL 기반 방법 (SEAL) — 스케일업 시 다양성 붕괴 발생
  • 다단계 프롬프팅 (Active Reading) — 복잡도 대비 이점 불분명

04 / 기존 방법과 비교

  • 수익 체감 심화
SPA의 장점
  • 구현이 단순
  • 스케일에 강건
  • 도메인 전이 용이

11 / 결론

핵심 테이크어웨이

① 단순함의 승리
고정된 7개 프롬프트 템플릿만으로 복잡한 RL 기반/다단계 방법을 능가
② 스케일링이 핵심
SPA는 스케일에 따라 꾸준히 성능이 향상되나, RL 방법은 다양성 붕괴로 정체
③ 프롬프트 설계의 중요성
복잡한 파이프라인보다 신중하게 설계된 프롬프트가 더 효과적
SPA
