문제 제기: 서구 중심의 가치 정렬

현실적 문제

  • LLM의 가치 정렬이 서구 규범에 편향됨
  • 다언어·다민족 사회에서 지역 가치를 오인 처리
  • 스리랑카 공용어인 싱할라어 기준 벤치마크 부재
  • 일반 도덕 추론과 국가 특정 가치 민감도 구분 불가

스리랑카의 특수성

  • 다민족·다언어 국가 (싱할라, 타밀, 영어)
  • 종교과 갈후 사회 역학이 공적 생활 형성
  • 공식 국가 가치 체계 없음
  • 기존 벤치마크는 언어 이해에만 집중

"모델 규모, 최신성, 다언어 능력이 스리랑카 가치 민감적 추론을 보장하지 않는다"

발란스 제약 (Valence Constraint)

각 가치에 대해 긍정·부정·중립 시나리오 균형을 유지하여 모델이 가치를 맹목적으로 긍정적으로만 반영하지 않도록 설계

핵심 아이디어: 가치 정렬 데이터만이 아닌 일반 저자원 명령어 데이터와 혼합하여 언어 능력과 가치 정렬을 동시에 달성

주요 결과: 벤치마크 평가

상업 모델

  • 전반적으로 강한 성능이나 균일하지 않음
  • 일부 프론티어 모델은 언어 간 균형 있으나 완벽하지 않음
  • 싱할라에서 주요 난점 지속

오픈 가중치 모델

  • 추론 및 포맷 측면에서 약점 명확
  • 언어 간 격차와 포맷 취약성 더 큼
  • 무효 출력 비율 높음

핵심 발견

모델 크기와 다언어 능력이 스리랑카 가치 정렬을 보장하지 않는다

  • 싱할라 = 주요 스트레스 테스트 언어
  • 영어 성능이 높아도 싱할라에서 급락
  • 가치 카테고리별로 편차 존재
  • 프롬프트 프레이밍에 민감한 모델 존재

파인튜닝 효과

Qwen 계열: 큰 개선

  • 싱할라 정확도 상승
  • 무효 출력 대폭 감소
  • 영어-싱할라 간 격차 유의미하게 축소
  • 미세조정된 소형 모델이 대형 모델과 경쟁

Aya-Expanse: 제한적 전이

  • 동일 LoRA 설정이 균일하게 전이되지 않음
  • 모델 패밀리 의존성 확인

시사점

저자원 가치 정렬에는 다음 3요소가 필요:

  • 문화적으로 근거한 감독 데이터
  • 일반 저자원 명령어 데이터 혼합
  • 모델별 맞춤 튜닝 (에포크, 학습률, 적응 방법)

가치별 정렬 분석

특정 가치(예: 다원주의)에서 더 큰 격차 관찰

프롬프트 민감도

프레이밍 변화에 따라 모델 응답이 불안정한 경우 존재

진단 도구로 활용 가능

언어 수준 분석

싱할라가 지속적 스트레스 테스트

LKValues 파인튜닝으로 Qwen 계열 싱할라 행동 개선

자유 형식 생성 평가

LKvaluesIT 테스트셋으로 자유 생성 후 인간 평가 + 자동 메트릭 병행 수행

오류 분석

포맷 오류, 가치 오인, 언어 혼용 등 체계적 오류 패턴 분류

결론 및 시사점

핵심 메시지

저자원, 다언어, 다민족 국가에서 LLM 가치 정렬은 문화적으로 근거한 지역 데이터와 모델 특화적 튜닝 전략 없이는 달성될 수 없다.

성과 요약

  • 스리랑카 최초 서베이 기반 가치 정렬 리소스
  • 40개 사회적 가치 체계화
  • 150K 훈련 + 1K 평가 데이터셋 공개
  • Qwen 계열에서 유의미한 개선 입증

향후 방향

  • 다른 국가로의 재현 가능한 파이프라인 적용
  • 타밀어 확장 및 3개국어 완전 커버
  • 모델 패밀리 간 전이성 연구 심화
  • RLHF/DPO 기반 가치 정렬 결합
