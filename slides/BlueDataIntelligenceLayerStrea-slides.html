문제 정의

NL2SQL의 한계를 넘어서

기존 NL2SQL 시스템은 단일 데이터베이스의 폐쇄 세계 가정에 머물러 있습니다. 그러나 현실의 정보 요구는 훨씬 복잡합니다.

01

반복적 질의

사용자는 하나의 완성된 질의가 아닌 여러 발화에 걸쳐 반복적으로 질문을 표현합니다.

02

다중 데이터 소스

질문이 여러 데이터 소스에 걸쳐 존재하며 단일 DB 가정을 벗어납니다.

03

외부 지식 의존

쿼리가 DB에 없는 상식 또는 외부 지식에 의존하는 경우가 빈번합니다.

핵심 과제: 사용자 의도와 가용 정보 사이의 의미적 격차(Semantic Gap)를 해소하는 것

해결책

Blue DIL: 데이터 인텔리전스 레이어

Blue는 에이전트와 데이터를 오케스트레이션하는 복합 AI 시스템이며, DIL은 그 핵심 데이터 인텔리전스 계층입니다.

  • LLM, 웹, 사용자를 1급 데이터 소스로 격상

  • 에이전트의 자율적 데이터 발견
  • 비용, 품질 기반 최적화

플랫폼 개요

Blue 플랫폼 아키텍처

Blue는 스트림(Streams)과 세션(Sessions) 두 가지 핵심 개념으로 구성됩니다.

S

스트림 (Streams)

데이터와 제어 메시지를 위한 채널. 에이전트 간 조율과 결과 전달을 지원합니다. 에이전트는 특정 스트림을 구독하고, 처리 후 새 스트림을 생성합니다.

N

세션 (Sessions)

에이전트들이 참여하여 작업을 수행하는 맥락적 경계. 데이터, 상태, 컨텍스트를 공유하며 작업을 분담합니다.

04 / 12
데이터 소스

확장된 데이터 소스 추상화

DIL은 전통적 DB를 넘어 LLM, 사용자, 웹을 1급 데이터 소스로 취급합니다.

LLMDB

LLM을 구조화된 데이터 소스로 모델링

  • 상식 및 세계 지식 질의
  • 쿼리 재작성 및 파티셔닝
  • 자동 스키마 설계
  • 카디널리티 추정
  • 캐싱 및 분산 실행

UserDB

사용자 데이터의 구조화된 관리 및 검색

  • 개인 문서 수집 및 구조화
  • 대화형 세션 기반 추출
  • 데이터 유효성 및 신선도 판단
  • 불확실 시 사용자 확인

WebDB

웹에서 온디맨드 구조화 데이터 추출

  • 웹 검색 및 소스 식별
  • 구조화 데이터 스크래핑
  • 스키마 설계 및 엔티티 해소
  • 데이터 보강 및 분산 실행

모든 소스는 네이티브 쿼리 언어(SQL, Cypher 등)와 자연어 인터페이스를 모두 지원하며, 구조화된 출력을 통해 이종 소스 간 통합 가능

데이터 플래닝

DAG 기반 데이터 계획 수립

데이터 플래너는 사용자 쿼리를 연산자의 비순환 방향 그래프(DAG)로 변환합니다.

계획 수립 과정

1

인스턴스화

연산자 레지스트리에서 추상 연산자(예: QuestionAnswer) 생성

2

재귀적 정제 (Refine)

추상 연산자를 NL2SQL, NL2LLM, QueryBreakdown 등 실행 가능한 하위 계획으로 분해

3

최적화 (Optimize)

연산자 수준: 하이퍼파라미터 조정
계획 수준: DAG 재구성, 병렬화, 자원 적응

데모 시나리오 1

아파트 검색 시나리오

여러 웹 사이트의 비정형 데이터를 통합 데이터베이스로 구축하고 자연어로 탐색합니다.

1

웹 기반 DB 구축

여러 웹사이트에서 아파트 매물과 보조 데이터를 스크래핑하여 SQL 데이터베이스 생성 및 보강

2

파일 기반 DB 구축

로컬 파일에서 아파트 정보 또는 맥락 정보를 추출하여 데이터베이스 보강

3

DB 변환

레지스트리 내 기존 데이터베이스를 변환 또는 재구성

4

자연어 -> SQL

자연어 질문을 SQL 쿼리로 변환하여 정확한 답변 제공

5

데이터 탐색 & 시각화

속성 분포, 결측값, 품질 이슈를 프로파일링하고 임대료 추이 등을 시각화

이종 데이터의 동적 통합, 사용자 상호작용, 에이전트 조율을 통해 실행 가능한 인사이트 생성

핵심 인사이트

멀티 에이전트 개발의 과제와 방향

발견된 과제

  • 에이전트 간 제어 흐름과 데이터 흐름 추적의 어려움
  • 병렬 실행 시 에러가 비동기적으로 전파되어 루트 원인 파악 어려움
  • 에이전트 조율, 실행 상태 모니터링, 데이터 일관성 유지의 어려움
  • 스트리밍 기반 통신 모델 학습에 시간 소요

제안된 방향

  • 에이전트 상호작용 시각화 도구 강화
  • 멀티 에이전트 실행의 로깅 및 추적 개선
  • 워크플로우의 결정론적 재생(Replay) 지원
  • 비동기 연산의 오류 메시지 개선
  • 배포 전 테스트용 시뮬레이션 / 샌드박스 모드

결론

결론 및 전망

Blue DIL은 관계형 데이터베이스, 웹 데이터, LLM, 사용자를
1급 데이터 소스로 통합하는 프레임워크로,
보다 신뢰할 수 있고, 확장 가능하며, 의미적으로 기반이 탄탄한
데이터 인텔리전스 시스템을 향한 진전입니다.

R

레지스트리 기반

데이터, 에이전트, 모델, 도구, 연산자를 통합 관리하는 카탈로그 아키텍처

O

확장 가능한 연산자

논리-물리 분리 계층으로 다양한 모달리티와 실행 전략 지원

D

선언적 DAG 플래닝

복잡한 요청을 분해, 실행, 최적화하는 실행 가능한 데이터 워크플로우
