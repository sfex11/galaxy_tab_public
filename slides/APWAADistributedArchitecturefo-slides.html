Manager: 워크플로우 계획과 병렬 분해

에이전트 조직 구조

  • LLM 기반 Manager 에이전트가 전역 워크플로우 계획 수립
  • 복잡한 쿼리를 비간섭 서브태스크로 동적 분해
  • 컨텍스트 형성을 통해 관련 정보만 선택적 포함
  • Worker에게 출력 계약(output contracts) 명시

Manager 도구

  • 병렬 계획 구조 생성
  • 서브태스크 위임 명령 발행
  • 데이터 테이블 접근 및 관리

출력 계약 (Output Contracts)

Worker가 반환해야 할 데이터 형식과 구조를 사전에 정의하여 결과 통합의 일관성 보장

병렬 계획 구조

각 병렬 단계에서 실행할 서브태스크 집합과 의존성 관계를 트리 형태로 표현

Worker: 독립적 서브태스크 실행

논리적 실행 모드

  • 데이터-병렬: 동일한 연산을 데이터 부분집합에 적용
  • 태스크-병렬: 서로 다른 연산을 동일 입력에 적용
  • 복제-병렬: 유사 태스크를 통해 솔루션 공간 탐색 가속

핵심 특징

  • 전역 조정과 로컬 상태 완전 분리
  • 개별 에이전트의 자율적 탐색 허용
  • 동적 에이전트 능력 및 프리셋 지원
  • 데이터 의존적 이질적 처리 경로 탐색

핵심 원칙: Worker 간 교차 통신(cross-communication) 없이 독립 자원으로 비간섭 하위 문제를 처리 → 병렬화의 극대화

분산 에이전트 컴퓨팅을 위한 핵심 추상화

병렬 계획 수립

워크플로우를 병렬 실행 가능한 단계들로 구조화. 각 단계는 독립적으로 실행 가능한 서브태스크 집합을 포함.

서브태스크 위임

Manager가 Worker에게 명령을 발행. 출력 계약을 통해 결과 형식 사전 정의. 비동기적 분산 실행.

데이터 테이블

대규모 분산 데이터셋을 위한 추상화. 단일 컨텍스트에 들어가지 않는 데이터도 테이블 단위로 논리적 접근 가능.

동적 능력 레지스트리

에이전트의 도구와 능력을 동적으로 등록. 이질적 데이터에 맞춰 컨텍스트 의존적 실행 경로 자동 선택.

구현: Ray 기반 확장 가능한 패브릭

상태 관리

  • State Store: 워크플로우 실행 상태, 태스크 메타데이터 관리
  • Object Store: 대규모 데이터 객체의 분산 저장소

클러스터 관리

  • 노드별 자원 관리 및 동적 할당
  • 공유 노드 서비스로 중복 기능 통합
  • Executor 자동 재시도 메커니즘

Executor의 확장성

Ray 액터 기반으로 구현된 Executor는 클러스터 전체에 분산 배치되어 수백~수천 개의 병렬 LLM 호출을 처리

기존 시스템: 오케스트레이터가 10~100개 에이전트만 관리 가능
APWA: 분산 패브릭에서 수천 개 Executor 동시 실행

설계 과제와 해결 방안

설계 과제 APWA의 해결 방안
LLM의 분산 인프라 추론 능력 부족 에이전트 전용 프로그래밍 추상화 제공으로 분산 리소스 추론 가능하게
대규모 데이터셋에 대한 추상화 부재 데이터 테이블 추상화로 컨텍스트 외부 데이터도 논리적 접근 가능
메타데이터조차 컨텍스트 초과 테이블 단위 분할 접근으로 백만 단위 객체도 처리
정적 병렬화 패턴의 한계 데이터/태스크/복제 병렬 3종 패턴 동적 선택
이질적 데이터에 대한 단일 규칙 불충분 컨텍스트 및 데이터 의존적 실행 경로 자동 발견

실험 결과: 핵심 발견

성능 우위

  • 다수 벤치마크에서 베이스라인 대비 향상된 출력 품질 달성
  • 복잡한 쿼리를 자동으로 병렬 워크플로우로 동적 분해 성공
  • 이질적 데이터와 병렬 처리 패턴 모두 수용

확장성 (가장 중요)

  • 대규모 태스크에서 기존 시스템이 완전히 실패하는 상황에서도 정상 동작
  • 태스크 크기 증가에 따른 선형적 확장 달성
  • 컨텍스트 윈도우 초과 문제를 분산 아키텍처로 극복

"APWA는 기존 시스템이 완전히 실패하는 설정에서도 확장 가능하다."

기여 요약 및 결론

주요 기여

1
병렬 가능한 에이전트 워크로드를 위한 최초의 분산 아키텍처 제안
2
LLM 에이전트에 맞춘 새로운 프로그래밍 추상화 (데이터 테이블, 출력 계약, 능력 레지스트리)
3
Ray 기반 확장 가능한 구현체 제공
4
다양한 벤치마크에서 베이스라인 대비 우수한 확장성 입증

핵심 메시지

MapReduce가 데이터 처리에 가져온 패러다임 전이를 APWA가 에이전트 워크플로우 영역에서 달성한다.

비간섭 하위 문제 분해 + 교차 통신 없는 독립 실행 + 분산 데이터 추상화 = 고도 병렬 에이전트 워크로드의 실현
