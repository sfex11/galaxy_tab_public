왜 체크포인트/롤백이 필요한가?

LLM 기반 AI 에이전트는 테스트 시간 트리 탐색과 강화 학습을 위해 고빈도 상태 탐색을 수행합니다.

트리 탐색 (MCTS)

  • 다수 후보 경로 병렬 탐색
  • 실패 시 이전 상태로 롤백
  • 중간 체크포인트 필수

강화 학습 (RL)

  • 매 훈련 스텝마다 k개 롤아웃
  • 각 롤아웃은 동일한 웜 스타트 상태 필요
  • 포크 지연이 훈련 처리량 직접 제한

rm은 파일을 삭제하고, pip install은 패키지 트리를 변경하며, sed은 소스 코드를 재작성합니다. 롤백과 포크는 완전한 샌드박스 상태의 복구를 요구합니다.

기존 접근의 두 가지 핵심 과제

과제 1 높은 체크포인트 지연

서버리스 컴퓨팅 중심의 최적화로 복원 지연만 고려, 체크포인트 지연은 간과됨

  • E2B: RAM 1GiB당 ~4초
  • CRIU: multi-GiB 프로세스에 수 초
  • Docker commit: 수 초
  • Firecracker 스냅샷: 수백 ms~수 초

과제 2 파일 시스템 상태 관리 부재

파일 복사 기반 접근은 지연이 크고, 확장성이 부족함

  • Git stash/branch: 100ms~1s
  • shutil.copytree: 100ms~10s
  • Btrfs/LVM 스냅샷: 10~100ms
  • 쓰기 증폭이 O(dir) 또는 O(layer)

결과: SWE-bench MCTS에서 상태 관리 오버헤드가 트레이젝토리 시간의 47~77%를 차지

DeltaFS: 동적 오버레이 파일 시스템

OverlayFS에서 영감을 받아, 런타임 핫 레이어 스위칭을 도입

레이어 구조

  • 하위 레이어: 읽기 전용 기반 상태
  • 상위 레이어: 쓰기 가능, 하위 덮어씀
  • 체크포인트 시 현재 레이어 동결 후 새 레이어 삽입
  • 언마운트 없이 레이어 교체 가능

핵심 최적화

  • 지연 스위치: per-inode 생성 카운터로 활성 FD 투명 재지정
  • 파일 수준 CoW: 업데이트가 복사 없이 COW로 축소
  • 롤백 = 레이어 스위치: 복사 연산 없이 즉시 복원
  • 쓰기 증폭 O(4KB) 수준

OverlayFS와의 핵심 차이: OverlayFS는 정적 마운트만 지원하지만, DeltaFS는 런타임에 레이어를 동적으로 추가·전환하며 활성 파일 디스크립터를 투명하게 유지합니다.

Firecracker 마이크로VM 기반 에이전트 샌드박스에 통합

커널 공간 DeltaFS

  • OverlayFS 기반 독립 Linux 파일시스템 모듈
  • 런타임 레이어 핫 스위칭 지원
  • per-inode 생성 카운터 기반 지연 FD 재지정
  • 약 1,200줄 커널 코드

유저 공간 DeltaCR 데몬

  • CRIU 확장 (증분 덤프 + 템플릿 포크)
  • 유한 템플릿 풀 + LRU 추방
  • 백그라운드 async-warm 스레드
  • MCTS 인식 가비지 컬렉션

호환성 및 배포 모델

에이전트 프레임워크와의 통합은 StateManager 결합 프로토콜을 통해 이루어지며, 기존 에이전트 코드 수정 없이 샌드박스 래퍼만 교체하면 배포 가능합니다.

상세 평가 지표

쓰기 증폭 (Write Amplification)

DeltaBox는 파일 시스템과 프로세스 모두 O(4KB) 수준의 쓰기 증폭 달성

  • Docker commit: O(layer) — 레이어 전체 재작성
  • VM 스냅샷: O(VM) — 전체 메모리 덤프
  • Git: 낮지만 OS 상태 복원 불가

누적 저장 오버헤드

MCTS 탐색 깊이에 따른 저장소 증가량

  • 전체 복제: 깊이에 비례하여 선형 증가
  • DeltaBox: 증분만 저장하여 크게 완화
  • MCTS GC로 불필요한 체크포인트 사전 정리

MCTS 탐색 처리량 및 포크 성능

고정 시간 예산 하에서 DeltaBox는 기존 방식보다 현저히 더 많은 탐색 노드를 방문할 수 있으며, RL 팬아웃 마이크로 벤치마크에서도 포크 지연이 크게 감소하여 훈련 처리량이 개선됩니다.

결론 및 기여 요약

문제 정의

AI 에이전트 샌드박스에서 전체 상태 복제 기반 C/R이 트리 탐색과 RL 워크로드에 치명적 병목이라는 점을 명확히 식별

핵심 통찰

연속 체크포인트 간 유사성이 높다는 관찰 → 변화량만 관리하는 DeltaState 추상화 제안

DeltaFS

언마운트 없는 레이어 스위칭과 지연 파일 디스크립터 재지정을 지원하는 오버레이FS 확장

DeltaCR

증분 덤프 + 웜 템플릿 fork() 복원 + 비동기 핫페이지 웜업의 결합 설계

DeltaBox는 체크포인트 14ms, 복원 5ms를 달성하여, 고정 시간 예산 하에서 에이전트가 현저히 더 많은 탐색 노드를 방문할 수 있도록 합니다.
