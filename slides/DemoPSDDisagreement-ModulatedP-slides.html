배경: RLVR와 온폴리시 자기 증류

RLVR의 한계

검증 가능한 보상 기반 강화학습(RLVR)은 롤아웃 수준의 보상을 모든 토큰에 균일하게 분배하여 세부 토큰 기여도를 구분하지 못하는 근본적인 크레딧 할당 병목을 겪음

GRPO 등은 여러 롤아웃을 샘플링해 결과 정답 여부만 보상으로 활용 → 조잡한 토큰 수준 신호

OPSD의 등장

온폴리시 자기 증류(OPSD)는 단일 모델이 교사와 학생을 동시에 수행. 교사는 정답 등 특권 정보를 조건으로, 학생은 질문만으로 밀집 토큰 수준 감독 제공

  • 외부 교사 불필요
  • GRPO 대비 수배율 토큰 효율 향상
  • Qwen3, DeepSeek-V4 등에서 채택

핵심 문제: 특권 정보 유출

OPSD에서 교사는 학생이 테스트 시점에 결코 관찰할 수 없는 특권 정보 y*를 조건으로 출력을 생성함. 이로 인해 학생 모델이 정답 의존적 단축경로를 내재화하는 특권 정보 유출(Privileged Information Leakage) 발생

조건부 상호정보량 갭

I(yt ; y* | x, y<t) > 0

질문과 생성 접두사를 조건으로 해도 특권 정보가 여전히 다음 토큰에 대한 추가 정보를 제공함

증상

  • 초기 성능 향상 후 점진적 성능 저하
  • 도메인 내 패턴에 과적합
  • 탐색 억제 및 교차 도메인 일반화 저하
  • 이전 가능성 붕괴(Policy Entropy Collapse)

기존 접근법의 한계

기존 방법들은 "모든 토큰이 동등하게 신뢰할 수 없다"는 공통 직관을 공유하지만, 특권 정보의 영향을 직접 측정하지 않고 간접 프록시에 의존

방법 핵심 메커니즘 한계

핵심 질문: 증류 타겟 자체를 어떻게 설계하여 교사 안내 학습과 학생의 독립적 추론을 균형 있게 조절할 것인가?

DemoPSD: 핵심 아이디어

선택적 교사 안내 채택(Selective Adoption of Teacher Guidance)

교사와 학생의 분포가 합리적으로 일치하면 교사의 안내를 채택하고, 분포가 크게 diverge하면 (특권 정보 과도 영향 시) 학생 자신의 추론에 더 의존

1
분포 불일치 측정
교사-학생 분포 간 차이를 토큰마다 정량화
2
적응적 αt 결정
불일치에 기반한 토큰별 유출 억제 계수 산출
3
바리센터 타겟 구성
교사-학생 분포의 기하학적 가중 결합 타겟 생성
4
선택적 증류 학습
유출이 적은 토큰은 교사 따르고, 많은 토큰은 학생 보존

Reverse-KL 바리센터 타겟

전체 교사 분포를 맞추는 대신, 교사와 학생 분포의 가중 기하학적 결합을 타겟으로 사용

πtargett(v | x, y*, ŷ<t) ∝ πteacher(v | x, y*, ŷ<t)1−αt · πstudent(v | x, ŷ<t)αt

αt ≈ 0 (불일치 작음)

특권 정보가 교사 분포를 크게 변화시키지 않음 → 타겟이 교사 분포에 가까움 → 교사 안지를 적극 수용

αt → 1 (불일치 큼)

특권 정보가 교사 분포를 강하게 왜곡 → 타겟이 학생 분포로 이동 → 유출 억제 및 학생 추론 보존

이는 KL-divergence 하에서 두 분포 간의 기하학적 중간점(바리센터)에 해당하는 자연스러운 보간

불일치 측정과 적응적 αt

교사-학생 불일치 측정

토큰 위치 t에서 교사 분포와 학생 분포 간의 역방향 KL 발산을 측정하여 불일치 정도를 정량화

  • 특권 정보의 직접적 영향을 측정하는 최초의 접근
  • 간접 프록시(엔트로피, 정답 여부 등) 불필요
  • 토큰 수준의 세밀한 제어 가능

αt 결정 메커니즘

측정된 불일치를 시그모이드 함수를 통해 [0, 1] 범위의 αt로 변환

불일치 작음
αt ≈ 0 → 교사 분포 가중치 높음
불일치 큼
αt → 1 → 학생 분포 가중치 높음

시그모이드의 온도 파라미터로 민감도 조절 가능

손실 함수와 학습 절차

손실 함수 구성

학생은 바리센터 타겟에 대해 역방향 KL 발산을 최소화하도록 학습. 표준 OPSD의 전체 교사 분포 매칭을 바리센터 타겟 매칭으로 대체하는 것이 유일한 변경사항으로, 기존 파이프라인과 완전히 호환

특권 정보 주입과 학습 과정

1
롤아웃 생성
학생이 질문 x로부터 ŷ 샘플링
2
교사 로짓 계산
동일 모델에 특권 정보 y* 주입 후 로짓 획득
3
불일치 → αt
토큰별 역방향 KL → 시그모이드 → αt
4
바리센터 손실
보간 타겟에 대한 역방향 KL 최소화

구현 이점: 외부 교사 불필요, 추가 모델 호출 없음, 기존 OPSD 코드에 최소 변경만으로 적용 가능

이론적 보장

정리 5.1 유출 억제(Leakage Attenuation)

불일치 가중 역방향 KL 바리센터 타겟은 특권 정보 유출의 비율을 효과적으로 감소시킴

  • 특권 정보와 학생 출력 간 상호정보량 상한 감소
  • 불일치가 클수록 억제 효과 강화
  • 수학적으로 엄밀한 증명 제공

정리 5.2 탐색 보존(Exploration Preservation)

DemoPSD는 밀집 토큰 수준 증류 하에서도 학생의 탐색 능력을 유지

  • 정책 엔트로피의 하한 보장
  • 표준 OPSD의 엔트로피 붕괴 문제 해결
  • αt가 엔트로피 보존의 자연스러운 조절자 역할

두 가지 이론적 결과는 유출 억제와 탐색 보존이 상충되는 것이 아니라, 불일치 기반 αt를 통해 동시에 달성됨을 보임

결론

DemoPSD는 온폴리시 자기 증류에서 특권 정보 유출이라는 근본적 문제를 교사-학생 분포 불일치를 직접 측정하여 해결하는 새로운 패러다임을 제시

1

유출 억제

불일치 가중 바리센터로 특권 정보 유출 비율 감소

2

탐색 보존

밀집 증류 하에서도 정책 엔트로피 붕괴 방지

3

견고한 일반화

도메인 내외 모두에서 일관된 성능 향상

핵심 메시지

"교사의 안내를 맹목적으로 따르는 것이 아니라, 언제 따르고 언제 자신의 추론을 유지할지를 분포 불일치라는 직접적 지표로 판단하는 것이 자기 증류의 성공적 적용을 위한 핵심 원칙이다."
