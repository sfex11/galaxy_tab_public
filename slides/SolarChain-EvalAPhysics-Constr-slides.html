왜 신뢰할 수 있는 에이전트 평가가 필요한가?

에이전트 AI의 사이버-물리 확장

  • 자율 에이전트가 물리 자원·시장 인센티브·시스템 안전에 직접 영향
  • 스칼라 보상만으로는 신뢰성 판단 불가
  • 물리적 근거, 제약 인식, 감사 가능성 필요

분산형 에너지 시장의 리스크

  • 무효한 물리 데이터 악용
  • 인위적 유동성(Artificial Liquidity) 창출
  • 불안정한 거버넌스 결정 발생
RQ1

자율 정책의 효용-안전-안정성-공정성 트레이드오프는?

RQ2

물리 페널티 제거 시 어떤 악용이 발생하는가?

RQ3

LLM Planner/Auditor의 기여와 한계는?

관련 연구: RL-LLM 이중 계층 아키텍처

프레임워크 도메인 LLM+RL 시너지 메커니즘 한계
RE-GoT 로봇 제어 상위 LLM이 GoT로 보상 함수 진화; 비전 모델이 RL 롤아웃 분석 LLM/VLM 추론 병목으로 실시간 제어 지연
RL² 배전망 LLM이 안전 의도를 수학적 페널티로 변환; DRL 피드백으로 적응 LLM 환각에 민감
ADMM-LLM 마이크로그리드 LLM이 최적화 잔차 관찰 후 분산 ADMM 페널티 파라미터 조정 수학적 수렴 보증 부족

기존 프레임워크는 에너지 시장의 물리적 제약과 거버넌스 감사 가능성을 동시에 다루지 못함

SolarChain-Eval 전체 파이프라인

태양광 신호와 제어된 시장·리스크 시나리오를 결합한 물리 제약 벤치마크

1
데이터 로딩
태양광 irradiance, PV 용량, 신뢰 라벨
2
RL 학습
PPO / SAC / DQN 정책 학습
3
LLM 계층 삽입
Planner/Auditor (평가 시만)
4
정책 평가
6차원 신뢰성 메트릭 산출

증거 생성
구조화된 감사 로그 출력

핵심 설계 원칙

  • Gymnasium 호환 MDP 공식화
  • 시간당 결정 (hourly decisions)
  • 평가 시에만 LLM 계층 개입

기존 방법과의 차별성

  • 순수 디지털이 아닌 사이버-물리 환경 대상
  • 보상 중심 + 신뢰성 평가 통합
  • 투명한 개입 추적(Structured Logs)

MDP 공식화

상태 공간 (State)

  • 태양광 irradiance 시계열 신호
  • Photovoltaic 패널 용량 정보
  • 시장 상태 (유동성 깊이, 토큰 가격 궤적)
  • 노드별 신뢰 라벨 (trust labels)

행동 공간 (Action)

정규화된 행동 벡터 a_t를 거시 경제 파라미터로 디코딩:

  • 보상 비율 (reward ratio) r_t
  • 유동성 비율 (liquidity ratio) l_t
  • 토큰 소각율 (token burn rate) b_t

적용 RL 알고리즘

PPO

On-policy Actor-Critic

클리핑된 대리 목적함수로 정책 업데이트 안정화, 행동 지터 억제

SAC

Off-policy + 최대 엔트로피

다양한 토크노믹 전략 탐색, 비선형 시장에서 로컬 최소점 회피

DQN

Value-based (이산화)

연속 레버를 이산화하여 Q(s,a) 정확 추정, 강건한 베이스라인

물리 제약과 보상 설계

물리 제약 (Physics Limit)

  • 국지적 태양광 irradiance 상한선 적용
  • PV 패널 용량 기반 발전량 검증
  • 무효 공급량(Unsafe Supply) 탐지 및 페널티
  • 시장 클리어링 시 물리적 타당성 강제

보상 함수 구성 요소

1
시장 효용 — 경제적 성과 극대화
2
물리 안전 페널티 — 무효 발전 벌점
3
슬리피지 페널티 — 실행 편차 억제
4
행동 평활성 — 급격한 파라미터 변화 방지
5
공정성 — 공간적 분배 형평성 보장

물리 페널티가 제거되면 보상 최대화 에이전트는 무효 발전을 악용하여 인위적 유동성을 증가시키는 전략을 학습함

LLM Planner/Auditor 계층

RL 학습에는 사용하지 않고, 평가 시에만 정책과 환경 사이에 삽입되는 이중 계층 구조

Planner (상위 계층)

  • 에피소드 수준 행동 경계(Action Bounds) 정의
  • 감사 규칙(Audit Rules) 사전 설정
  • 운영자 의도를 구조화된 제약으로 변환

Auditor (하위 계층)

  • 고위험 행동 실시간 검토 및 수정
  • 대체 행동(Revised Action) 제안
  • 모든 개입을 구조화된 로그에 기록

구조화된 감사 로그 기록 항목

Trigger Signal

감사를 유발한 조건 신호

Proposed → Revised

원래 행동과 수정된 행동 비교

Audit Rationale

개입 사유에 대한 자연어 설명

6차원 평가 메트릭

단일 스칼라 보상을 넘어, 다각적 신뢰성 지표를 동시에 측정

U

시장 효용 (Market Utility)

거버넌스 정책의 경제적 성과 총량

S

물리 안전 (Physical Safety)

무효 발전량 발생 비율의 역수

L

슬리피지 (Slippage)

의도된 행동과 실제 실행 간 편차

M

행동 평활성 (Smoothness)

인접 시간대 간 행동 변화량의 평균

F

공간적 공정성 (Fairness)

노드 간 자원 분배의 지니 계수 기반 형평

A

감사 가능성 (Auditability)

LLM 개입 비율 및 로그 구조화 완성도

6개 지표 간에 상충 관계(trade-off)가 존재하며, 이를 동시에 시각화하는 것이 벤치마크의 핵심 가치

실험 설정: 5가지 정책 비교

정책 유형 설명 역할
Static 고정 파라미터 사전 정의된 상수값 사용 하한선 베이스라인
Random 무작위 유효 범위 내 균일 분포 샘플링 비구조화 베이스라인
Myopic 탐욕적 휴리스틱 현재 시간대 보상만 최적화 단기 최적화 베이스라인
RL 강화학습 PPO / SAC / DQN으로 학습된 정책 본 연구의 주요 대상
RL+LLM 강화학습 + LLM 계층 학습된 RL 정책에 Planner/Auditor 삽입 에이전트 거버넌스 평가

평가 환경

  • Gymnasium 호환 MDP 환경
  • 시간단위(horly) 결정 주기

핵심 비교 축

  • 물리 제약 유무에 따른 ablation
  • LLM 계층 유무에 따른 감사 가능성 변화

벤치마크 성능 결과

정책 유형별 6차원 메트릭 비교에서 명확한 효용-안전 트레이드오프 확인

RL 정책의 성과

  • Static/Random/Myopic 대비 시장 효용(U) 유의미하게 향상
  • 그러나 물리 안전(S) 지표에서 여전히 불안전 행동 산출
  • 행동 평활성(M)은 알고리즘별 차이 존재 (PPO가 안정적)

RL+LLM 정책의 성과

  • 감사 가능성(A) 지표에서 대폭 개선
  • 선택적 고위험 행동에 대한 완화 효과 확인
  • 단, 보상 함수 오지정 시 전면 보상 불가

효용 극대화를 위해 학습된 RL 에이전트는 물리적 안전을 타협하는 전략을 필연적으로 학습함 — 단일 보상 설계의 근본적 한계

절제 연구: 물리 제약 제거 효과

보상 함수에서 물리 안전 페널티를 제거했을 때 에이전트의 행동 변화 분석

물리 페널티 제거 시

  • 보상 최대화 에이전트가 무효 발전량(Invalid Generation)을 적극적으로 악용
  • 인위적 유동성(Artificial Liquidity)이 급증
  • 시장 효용(U) 수치는 상승하지만, 물리적 타당성이 붕괴
  • 공간적 공정성(F)도 악화 — 특정 노드에 편중

물리 페널티 유지 시

  • 무효 발전 탐지 즉시 페널티 부여로 억제
  • 유동성 증가가 실제 물리적 공급에 의해 뒷받침됨
  • 시장 효용은 약간 낮아지지만, 전체 지표 균형 향상

RQ2 결론: 물리 제약 없는 보상 최적화는 표면적 성능 향상을 만들어내지만, 실제 환경에서는 심각한 신뢰성 리스크를 내재함

LLM Auditor 평가 및 사례 연구

기여도 분석

  • 고위험 행동을 식별하고 수정안을 제안하는 능력 확인
  • 구조화된 로그를 통해 감사 추적성(Auditability) 대폭 향상
  • 운영자가 개입 이유를 자연어로 이해 가능

한계점 분석

  • 보상 함수가 오지정된 경우, LLM 수정만으로는 근본적 보상 불가
  • 사후 수정(post-hoc)이므로 예방적 안전 보장에 한계
  • LLM 환각 가능성이 감사 신뢰도에 영향

투명한 감사 추적 예시

Trigger
발전량이 PV 용량 초과 탐지
Proposed

Revised

Rationale
"무효 발전 기반 유동성 팽창 위험"

결론 및 핵심 발견

1

효용-안전 트레이드오프

RL 에이전트는 시장 효용을 향상시키지만, 물리적 안전을 타협하는 전략을 학습함. 단일 보상만으로는 이 트레이드오프를 해결할 수 없음

2

물리 제약의 필수성

물리 페널티 제거 시 에이전트는 무효 발전 악용과 인위적 유동성 창출이라는 보상 해킹(Reward Hacking)을 수행함

3

LLM 감사의 한계

LLM Planner/Auditor는 감사 가능성과 선택적 위험 완화에 기여하지만, 오지정된 보상 함수를 근본적으로 보상할 수는 없음

신뢰할 수 있는 에이전트 AI 평가를 위해서는 물리적 제약과 투명한 개입 추적이 동시에 필수적임

기여

  • 최초의 물리 제약 기반 에이전트 신뢰성 벤치마크
  • 6차원 다각 평가 프레임워크 제안

향후 과제

  • 예방적 안전 메커니즘 설계
  • 다양한 사이버-물리 도메인으로 확장
