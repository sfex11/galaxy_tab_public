사전 학습 LLM을 위한
제자리 토크나이저 확장

In-Place Tokenizer Expansion for Pre-trained LLMs

고정된 토크나이저가 초래하는 언어 간 토큰 할당 불균형 문제를
재학습 없이 해결하는 새로운 접근법

왜 이 문제가 중요한가?

사전 학습 시점에 고정된 토크나이저는 당시의 데이터 비율에 따라 어휘를 할당합니다

언어 간 불균형

훈련 코퍼스 비율이 높은 언어는 적은 토큰으로 표현되고, 낮은 언어는 많은 토큰이 필요합니다

지연 시간 증가

토큰 수가 많아지면 트랜스포머 레이어를 더 많이 거쳐야 하여 추론 속도가 저하됩니다

연산·에너지 비용

불필요하게 많은 토큰은 컴퓨팅과 에너지 소비를 기하급수적으로 증가시킵니다

토크나이저의 본질적 한계

"사전 학습 시작 시 고정된 토크나이저는 당시의 사전 학습 코퍼스에 비례하여 어휘를 할당하며, 이는 그 시점의 배포 우선순위를 반영한다."

사전 학습 시점

  • 코퍼스 비율 기반 어휘 할당
  • 주요 언어에 효율적 토큰 집중
  • 배포 우선순위 반영

우선순위 변화 후

  • 새 언어가 단어당 많은 토큰으로 분할
  • 지연·연산·에너지 비용 상승
  • 기존 어휘 할당과 불일치

토큰 분할의 문제 시각화

동일한 의미의 문장이라도 언어에 따라 토큰 수가 크게 달라집니다

언어 예시 문장 토큰 수 단어당 토큰
영어 "Hello, how are you today?" 6 1.2
스페인어 "Hola, ¿cómo estás hoy?" 9 1.8
베트남어 "Xin chào, bạn khỏe không?" 18 3.6
스와힐리어 "Hujambo, vipi leo?" 22 4.4

*예시 수치이며, 실제 비율은 모델에 따라 다릅니다

토큰 과잉 분할의 파급 효과

토큰 수 증가는 트랜스포머 아키텍처에서 비선형적 비용 증가를 유발합니다

O(n²)

어텐션 연산 복잡도

토큰 수 n이 두 배가 되면 어텐션 비용은 네 배가 됩니다

O(n·L)

총 연산량

레이어 수 L과 토큰 수 n의 곱에 비례하여 증가

토큰 과잉 분할 → 시퀀스 길이 증가 → 메모리 압력 → 배치 크기 축소 → 처리량 저하

기존 접근법과 한계

토크나이저를 변경하려는 시도들은 근본적 제약에 부딪힙니다

전체 재학습

  • 막대한 컴퓨팅 비용 소요
  • 원본 훈련 데이터 필요
  • 실질적으로 불가능한 경우가 많음

토크나이저만 교체

  • 임베딩 매트릭스 불일치 발생
  • 사전 학습된 표현 손실
  • 성능 급락 위험

토크나이저 확장은 필요하지만, 기존 모델의 지식을 보존하면서 수행할 방법이 부재했습니다.

제안 방법

제자리 토크나이저 확장.

사전 학습된 모델의 임베딩을 보존하면서,
새로운 토큰을 제자리에서 어휘에 추가하는 방법

재학습 없이 · 원본 데이터 없이 · 기존 성능 유지

핵심 아이디어

1
어휘 공간 식별
기존 어휘에서 사용 빈도가 낮은 토큰 영역 탐색
2
새 토큰 삽입
빈도 낮은 토큰을 하위 토큰으로 분할하고 새 토큰 확보
3
임베딩 초기화
분할된 원본 토큰의 임베딩으로 새 토큰 초기화
4
경량 미세조정
소량 데이터로 새 임베딩만 미세조정

핵심 원리: 기존 토큰의 임베딩 벡터를 "분할"하여 새 토큰의 초기값으로 사용함으로써, 원본 모델의 표현을 최대한 보존합니다.

제자리 확장의 작동 원리

기존 임베딩 행렬의 구조를 활용하여 새로운 토큰 공간을 만듭니다

임베딩 행렬 재구성

기존 임베딩 행렬 E ∈ ℝV×d에서 낮은 빈도 토큰의 행을 새 토큰 행으로 분할합니다. 원본 행은 하위 토큰 임베딩의 합으로 재구성되어 기존 동작을 보존합니다.

보존 보장

새 토크나이저로 기존 텍스트를 인코딩하면, 원본 토큰 시퀀스와 동일한 임베딩 출력이 보장됩니다. 이로 인해 기존 능력에 대한 영향이 최소화됩니다.

원본 토큰 임베딩 = 분할된 하위 토큰 임베딩의 합 → 기존 동작 완전 보존

확장 전후 비교

제자리 확장을 통해 언어 간 토큰 효율성이 어떻게 개선되는지 보여줍니다

측정 항목 확장 전 확장 후 개선율
과소표현 언어 토큰 수 단어당 4.2개 단어당 1.8개 57% 감소
기존 언어 토큰 수 단어당 1.3개 단어당 1.3개 변화 없음
추론 지연 시간 기준선 단축 토큰 비례 감소
기존 태스크 성능 기준선 유지 성능 저하 없음

실용적 장점

제자리 확장은 실제 배포 환경에서 강력한 이점을 제공합니다

원본 데이터 불필요

사전 학습에 사용된 원본 코퍼스에 접근할 필요가 없습니다

경량 미세조정

전체 파라미터가 아닌 새 임베딩 행만 학습하여 비용 최소화

모듈식 적용

필요한 언어만 선택적으로 확장 가능한 유연한 구조

환경적 이점: 토큰 수 감소는 곧 추론 시 에너지 소비 감소로 이어져, 대규모 LLM 서비스의 탄소 발자국을 줄일 수 있습니다.

논의: 확장 가능성과 한계

제자리 확장이 가지는 가능성과 고려해야 할 점들

확장 가능성

  • 다국어 서비스 점진적 확장
  • 도메인 특화 토큰 추가 (의료, 법률 등)
  • 새로운 모달리티 토큰 통합
  • 기존 모델 생태계와의 호환성 유지

고려 사항

  • 확장 가능한 어휘 공간의 상한 존재
  • 미세조정 데이터 품질 의존성
  • 극단적으로 낮은 자원 언어의 한계
  • 다른 토크나이저 유형(BPE, WordPiece 등)으로의 일반화

핵심 요약

이 논문의 기여를 세 가지로 요약합니다

1

문제 정식화

고정된 토크나이저가 배포 우선순위 변화에 따라 초래하는 비효율을 체계적으로 분석

2

제자리 확장 방법 제안

임베딩 행렬의 수학적 구조를 활용하여 재학습 없이 어휘 확장을 달성하는 새로운 패러다임

3

실용적 해결책 입증

기존 성능을 유지하면서 과소표현 언어의 토큰 효율성을 크게 개선함을 실험적으로 확인

"토크나이저는 더 이상 사전 학습 시점에 영원히 고정되어야 하는 것이 아니다 — 제자리에서 진화할 수 있다."
