전략 추상화

정답을 맞힌 문제에서 핵심 추론 패턴을 식별하여 재사용 가능한 전략으로 요약

주의 추상화

오답 문제에서 실수 패턴을 식별하여 피해야 할 함정으로 요약

추상화 라이브러리 구축 상세

추출 과정

입력 구성

문제 + 풀이 추적 + 정답 + 정오 라벨

추출 규칙

문제 특정 수치·변수·정답 제외, 1~2문장으로 재사용 가능하게 작성

정오 조건부 추출

정답 → 전략 추상화 / 오답 → 주의 추상화

라이브러리 정리

  • all-MiniLM-L6-v2로 384차원 임베딩

  • 임계값 0.85에서 근접 중복 제거

추상화 강화 RL 사후 학습

추상화를 추론 시간뿐만 아니라 RL 미세조정 단계에 통합하여 정책이 (q, ℛ_q) 쌍에 조건화되도록 학습

GRPO 적용 과정

1

학습 문제 q에 대해 추상화 집합 ℛ_q 검색

2

π_θ(·|q, ℛ_q)에서 그룹별 후보 해답 샘플링

3

결과 보상 r(y, y*)로 스코어링 후 그룹 내 정규화 → 어드밴티지 Â 계산

4

Â(q,y) · log π_θ(y|q, ℛ_q) 최대화로 정책 업데이트

표준 GRPO와의 차이

  • 보상 신호와 최적화는 표준 GRPO와 동일
  • 정책 입력이 q 단독이 아닌 (q, ℛ_q) 쌍
  • 강한 교사의 시연 없이 결과 보상만으로 학습

두 가지 변형: 학습 시에만 추상화 사용 / 학습+테스트 모두 사용

RL에서의 효과

RL 단계에서는 프롬프트 구조 효과와 추상화 내용 효과가 더 명확히 분리되며, 추상화 강화 GRPO가 일반 GRPO를 유의미하게 상회

핵심 발견과 분석

자기 추출의 파워

학생 LLM이 자신의 풀이 흔적에서 추출한 추상화가 강한 교사 모델의 추출물과 거의 동등한 성능을 달성함. 프론티어 모델 의존 없이 자가 향상 가능함을 시사

전이 가능성

추상화 사용 파이프라인이 수학(MATH) 외에 논리 추론 데이터셋으로 전이 가능. 한 모델의 흔적에서 추출한 추상화가 다른 타겟 모델에도 효과적

내면화 가설 불성립

RL 학습 후 추상화 없이도 성능이 유지될 것이라는 가설은 기각됨. 추상화는 학습 중 가이드 역할을 하지만, 모델이 이를 완전히 내면화하지는 않음

약점 보완, 강점 간섭

추상화는 모델이 약한 영역에서는 도움을 주지만, 이미 강한 영역에서는 오히려 성능 저하를 유발할 수 있음. 선택적 적용의 필요성 시사

결론

LLM은 인간처럼 자신의 추론 경험에서 경험적 추상화를 추출하고, 이를 활용하여 점진적으로 자가 향상할 수 있다.

자기 추출 가능

교사 모델 없이도 추상화 추출 가능

두 가지 활용 모드

추론 시 검색 + RL 학습 통합

범용 전이 가능

다른 데이터셋·도메인·모델로 전이

향후 방향

  • 모델 강점/약점에 따른 선택적 추상화 적용 메커니즘
  • 추상화의 자동 갱신 및 진화 메커니즘 탐색
  • 수학·논리 외 다양한 도메인으로의 확장
