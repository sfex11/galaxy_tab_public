왜 PCB 회로도 자동 생성인가?

PCB(인쇄회로기판)는 소비자 전자제품부터 IoT 기기, 로봇 하드웨어까지 거의 모든 전자 하드웨어의 기반입니다. 맞춤형 PCB 설계 수요는 급증하고 있지만, 설계 워크플로우는 여전히 수동에 의존합니다.

회로도 설계는 하드웨어 개발의 첫 단계이자 가장 중요한 단계로, 시스템 아키텍처를 정의합니다. 그러나 방대한 설계 공간과 도메인 전문지식에 대한 의존으로 인해 자동화가 가장 뒤처진 영역입니다.

기존 PCB 설계 워크플로우
  • 사용자 요구사항 분석
  • 회로도 설계 (컴포넌트 선정·배치·배선)
  • 넷리스트 추출
  • PCB 레이아웃 및 제조

회로도 설계 단계가 가장 자동화가 안 됨

두 가지 핵심 과제

01
효율적인 표현 방식의 부재

기존 회로도 포맷은 도구 특화적 메타데이터로 가득 차 있어 LLM 생성에 부적합합니다. 이미지 기반 표현은 편집 불가하고 기계 판독도 어렵습니다. 또한 일관된 컴포넌트 배치와 가독성 있는 배선을 요구하나, 기존 언어 모델은 이를 충족하기 어렵습니다.

02
대규모 데이터셋의 부재

오픈소스 하드웨어 설계는 공개되어 있으나, 이기종 포맷으로 제공되거나 이미지 형태로만 공개되어 학습 데이터로 재사용하기 어렵습니다. 자연어 프롬프트와 회로도가 짝지어진 데이터셋은 존재하지 않습니다.

디지털/아날로그 IC 설계에서 생성형 AI의 성공에도 불구하고, 자연어 기반 PCB 회로도 생성은 거의 탐구되지 않은 분야입니다.

기존 표현 방식의 한계

표현 방식 LLM 적합성 편집 가능 주요 문제점
Raw KiCad 파일 매우 낮음 가능 도구 특화 메타데이터 과다, 포맷 에러 빈발
이미지 기반 낮음 불가 기계 판독 불가, 편집 불가
SKiDL 코드 보통 가능 공간 배치 정보 누락, 시각적 회로도 생성 불가
SchGen 코드 표현 높음 가능 —

핵심 문제: 기존 방식은 절대 좌표 기반의 기하학적 예측을 요구하지만, SchGen은 의미론 기반 매칭으로 문제를 전환합니다.

의미론적 기반 코드 표현

전문가가 회로도를 그리는 방식을 모방한 컴팩트한 편집 프리미티브 집합을 정의합니다.

심볼 추가

add_symbol()로 컴포넌트를 배치. 상대 좌표 오프셋으로 공간 추론 부담 감소.

핀 이름 기반 배선

connect_pins()로 절대 좌표 대신 VCC, TXD 등 의미적 핀 이름으로 연결.

핀 위치 조회

query_pin()으로 이미 배치된 심볼의 핀 위치를 동적으로 확인 가능.

핵심 설계 원칙  절대 좌표 → 상대 배치: 앵커 컴포넌트 기준 로컬 좌표계 사용  |   기하학 예측 → 의미론 매칭: 핀 이름 기반 연결로 LLM 친화적 변환

코드 표현의 구조적 특징

기존 Raw 파일 vs SchGen 코드
Raw KiCad (문제점)

(symbol (lib_id "Device:R") ...
(at 142.5 95.25 0) ...
(uuid "a1b2c3d4-...")
... [수십 줄의 메타데이터]

SchGen Code (해결책)

u1 = add_symbol("ESP32-WROOM")
r1 = add_symbol("R", right_of=u1)
connect_pins(u1.VCC, r1.pin1)

표현 설계의 세 가지 원칙
1
메타데이터 제거
UUID, 버전 정보 등 불필요 데이터 완전 제거
2
상대 좌표계
right_of, below 등 의미적 방향 지시자 사용
3
핀 이름 기반 연결
VCC, GND, TXD 등 시맨틱 이름으로 배선

결과적으로 생성된 코드는 실행 가능한 Python 스크립트로, KiCad에서 직접 편집 가능한 회로도를 생성합니다.

표현 방식의 영향

Code-L1 표현을 사용한 SchGen은 Raw KiCad 베이스라인 대비 유효 회로도 비율 82% vs 32%로 2.5배 이상 향상. 표현 설계가 성능의 핵심 임을 입증.

모델 크기 대비 효율성

20B 파라미터의 SchGen이 훨씬 큰 GPT-5.2보다 높은 성능 달성. 동일한 API를 제공했음에도 파인튜닝의 중요성과 표현의 효과를 확인.

광범위한 시사점

SchGen의 성공은 복잡한 하드웨어 설계 과제에서 표현 설계(representation design)의 결정적 역할을 강조합니다. 단순히 더 큰 모델을 사용하는 것보다, 문제 구조에 맞는 표현을 설계하는 것이 핵심입니다.

이 접근법은 PCB를 넘어 기계 설계, 건축 설계 등 시각적·공간적 설계 과제에도 확장 가능합니다. 기하학 예측을 의미론적 매칭으로 변환하는 아이디어는 범용적 적용 가치가 있습니다.
