설명

왜 부정 무시가 발생하는가?

귀납적 편향(Inductive Bias) 가설

LLM은 주장을 확신 있게 참으로 표현하는 해을 선호하는 귀납적 편향을 가짐. 부정을 포함하는 해도 학습 가능하지만, 추가 학습 시 불안정하여 참 표현 해로 되돌아감

소프트 제약 실험

학습 중 부정 준수 해에 추가 보상을 주면 모델이 주장을 거짓으로 보고함

불안정성

제약 없이 추가 학습하면 즉시 "참" 해로 되돌아감 — 부정 해가 솔루션 공간에서 국소적임

부정 무시는 모델이 부정을 "이해 못 하는" 것이 아니라, "더 안정적인 해를 선택하는" 것

모델 호환성

모든 테스트 모델에서 관찰됨

모델 베이스라인 부정 문서 학습 후
Qwen3.5-397B-A17B 2.5% 88.6%
Kimi K2.5 낮음 높음
GPT-4.1 낮음 높음
Qwen3.5-35B-A3B 낮음 높음

특정 모델의 문제가 아닌 LLM 아키텍처 전반의 구조적 특성으로 추정

시사점

합성 문서 파인튜닝에 대한 시사점

  • 경고문, 면책 조항, "거짓" 라벨만으로는 모델이 잘못된 믿음을 형성하는 것을 막을 수 없음
  • 부정을 아무리 반복해도 효과가 제한적 — 토큰 비율을 12%에서 40%로 늘려도 신뢰율 88.6%→84.4%
  • 주장 자체에 부정을 내재시키는 "국소 부정" 표현이 효과적
  • 교정 정보(실제 사실)를 함께 제공하면 부분적 완화 가능
  • 안전 관점: 유해 행동에 "하지 마라" 주석을 다는 것만으로는 부족할 수 있음

합성 문서 파인튜닝은 가치 주입과 안전 연구에 점점 더 사용되고 있으나, 한정어가 내재화되지 않는 조건을 인식해야 함
