Claw-Eval-Live

진화하는 실제 워크플로우를 위한
라이브 에이전트 벤치마크

Chenxin Li, Zhengyang Tang, Huangxin Lin, Yunlong Lin, Shijue Huang 외 · CUHK, CUHK-Shenzhen, SCUT, XMU, HKUST, PKU, HKU

문제 정의

기존 에이전트 벤치마크의 두 가지 근본적 한계

정적 태스크 집합의 고착화

출시 시점에 큐레이션된 태스크 믹스를 고정합니다. 실제 워크플로우 수요는 지속적으로 변하지만, 벤치마크는 점차 현실과 괴리됩니다.

  • 툴 스택의 진화 미반영
  • 엔터프라이즈 병목 변화 누락
  • 자동화 우선순위 전환 무시

최종 응답만 채점하는 방식

"그럴듯한 보고서" ≠ "실제 작업 수행". 에이전트가 필수 쓰기를 누락하거나, 잘못된 파일을 수정하거나, 부정확한 레코드를 근거로 추천해도 통과됩니다.

  • 실행 궤적 검증 불가
  • 상태 변경 여부 확인 불가
  • 복제 가능성 부족
핵심 아이디어

이중 접지(Dual Grounding)

워크플로우 에이전트 평가는 두 가지 측면에서 "접지"되어야 합니다.

1

수요 측 접지

출시 태스크 분포가 작성자의 일회성 스냅샷이 아닌, 진화하는 실제 워크플로우 수요에 가깝게 유지되어야 합니다.

2

증거 측 접지

각 태스크의 점수가 최종 텍스트의 그럴듯함이 아닌, 관측 가능한 실행 증거에 기반해야 합니다.

"워크플로우 설정에서 '맞는 것 같다'와 '실제 작업을 했다'의 차이는 단순한 미적 문제가 아니다."

아키텍처

Claw-Eval-Live 전체 구조

1
공개 수요 신호
ClawHub Top-500 스킬에서 워크플로우 수요 패턴 수집
2
신호→태스크 변환
클러스터링, 가중치 부여, 시드 확장, 판별력 기반 선택
3
타임스탬프 스냅샷
고정된 픽스처, 서비스, 워크스페이스, 채점기로 재현 가능
4
실행 & 기록
제어된 환경에서 실행, 궤적·감사로그·상태·아티팩트 기록
5
행동 기반 채점
결정론적 검사 + 구조화된 LLM 판정의 하이브리드

분기별 리프레시: 신호와 모델이 진화함에 따라 파이프라인을 재실행하여 새로운 릴리스 생성

구성 파이프라인

신호에서 태스크로: 5단계 파이프라인

1
신호 수집 (Signal Collection)

ClawHub Top-500 공개 워크플로우 스킬에서 수요 패턴 추출

2
패턴 클러스터링 (Pattern Clustering)

유사한 워크플로우 수요를 패밀리로 군집화

3
패밀리 가중치 부여 (Family Weighting)

실제 수요 빈도와 중요도에 따라 패밀리별 가중치 할당

4
시드 확장 & 구현 (Seed Expansion)

각 패밀리에서 후보 태스크를 생성 → 157개 실행 가능한 후보 풀 구성

5
판별력 기반 공개 릴리스 선택

혼합 정수 선형 계획(MILP)으로 모델 간 판별력이 최대화되는 105개 태스크 선택

실행 환경

두 가지 실행 표면(Surface)

서비스 백업 워크플로우

에이전트가 레코드를 조회하고, 시스템 간 교차 참조하며, 상태 변경 액션(초안, 캘린더 업데이트, 태스크 생성 등)을 커밋해야 합니다.

  • 18개 제어된 비즈니스 서비스
  • HR, 관리, 다시스템 워크플로우 포함
  • 승인 체인, 트랜잭션 조정 등

로컬 워크스페이스 복구

에이전트가 로그를 검사하고, 파일을 편집하며, 명령을 실행하고, 복구가 실제로 작동했음을 증명해야 합니다.

  • 샌드박스된 개발 환경
  • 로그 분석, 파일 수정, 명령 실행
  • 복구 검증을 위한 사후 아티팩트 점검

고가치 워크플로우는 종종 이 두 경계를 동시에 넘나듭니다. 기존 벤치마크는 한쪽만 다룹니다.

채점 방법론

행동 기반 하이브리드 채점

최종 응답이 아닌 실행 궤적, 감사 로그, 서비스 상태, 워크스페이스 아티팩트에서 증거를 수집합니다.

P1

증거+판정

분석적 워크플로우 태스크
결정론적 증거 검사 + 의미적 차원에 대한 구조화된 LLM 판정

P2

작업 검증

작업 수행 위주 태스크
의미적 구성요소가 적고, 주로 작업 완료 여부를 검증

P3

스크립트 우선

워크스페이스 태스크
스크립트 실행 결과를 먼저 검증하고, 추가 의미 판정은 최소화

원칙: 증거가 충분하면 결정론적 검사를 사용하고, 의미적 차원만 구조화된 LLM 판정에 위임합니다.

심층 분석

패밀리 수준 이질성

모델 성능은 태스크 패밀리에 따라 크게 편차를 보입니다. 특정 패밀리에서는 모든 모델이 어려워하거나 쉽게 통과하는 현상이 관찰됩니다.

지속적 병목 패밀리

  • HR 워크플로우 — 직원 기록 조작, 온보딩 자동화
  • 관리 워크플로우 — 승인 체인, 권한 위임
  • 다시스템 조정 — 크로스 시스템 데이터 일치

순위 발산 현상

합격률이 유사한 모델이라도 전체 완료도에서 크게 diverge할 수 있습니다.

리더보드 순위만으로는 모델의 실제 워크플로우 역량을 충분히 파악할 수 없습니다.

추가 발견

임계값 효과와 태스크 판별력

임계값 효과 (Threshold Effects)

많은 태스크에서 모델들이 "거의 맞춤" 상태에 도달하지만, 마지막 한 단계(정확한 레코드 수정, 올바른 파일 편집 등)에서 실패합니다. 이는 워크플로우 자동화의 성숙도가 특정 임계값 아래에 있음을 시사합니다.

중간 대역 판별력 집중

태스크 수준 판별력은 중간 난이도 대역에 집중됩니다. 너무 쉬운 태스크는 모든 모델이 통과하고, 너무 어려운 태스크는 모든 모델이 실패하여 모델 간 차별력을 제공하지 못합니다.

효율성과 자원 사용

합격률이 높은 모델이 항상 효율적인 것은 아닙니다. 일부 모델은 불필요한 API 호출, 반복 조회, 장황한 궤적으로 높은 자원 소비를 보이며, 이는 실제 배포 환경에서 비용 문제를 야기할 수 있습니다.

결론

핵심 메시지

워크플로우 에이전트 평가는 신선한 외부 수요와 검증 가능한 에이전트 행동이라는 두 가지 축에 접지되어야 합니다.

기여 1

공개 워크플로우 신호에서 태스크를 소싱하고, 서비스 백업 워크플로우와 워크스페이스 복구를 단일 릴리스에서 평가하는 라이브 벤치마크 스냅샷 제시

기여 2

신호에서 태스크로의 재실행 가능한 릴리스 파이프라인 정의 — 클러스터링, 가중치, 확장, 판별력 기반 MILP 최적화 포함

기여 3

행동 기반 하이브리드 채점 도입 및 결과 보고 — 최고 모델 66.7%, 서비스 워크플로우가 복구보다 상당히 어려움

기여 4

리더보드 순위만으로 불충분함을 입증하고, 태스크 패밀리·실행 표면별 구조화된 실패 분석 프레임워크 제공
