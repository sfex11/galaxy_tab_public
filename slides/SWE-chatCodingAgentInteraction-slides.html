Background

왜 이 연구가
필요한가?

AI 코딩 에이전트는 급격히 도입되고 있지만, 실제 사용 방식에 대한 실증적 증거는 거의 없습니다.

기존 벤치마크의 한계 SWE-bench 등은 큐레이팅된 문제만 평가 — 실제 반복적 사용과 차이 큼
에이전트 능력은 급성장 중 12시간 인간 작업의 50%를 성공한다는 보고 (METR 2026)
근본적 질문 미해결 사용자가 얼마나 에이전트 출력을 실제로 사용하는가? 실패 시 어떻게 대응하는가?
SWE-chat으로 gap 해소 실제 오픈소스 개발자의 완전한 상호작용 추적 + 코드 작성자 귀속

Key Findings

핵심 발견

SWE-chat이 밝혀낸 실제 코딩 에이전트 사용의 5가지 핵심 인사이트

코딩 패턴은 극단적으로 바이모달 41% 바이브 코딩 vs 23% 인간 전담 — 중간이 없음
에이전트 코드의 절반이 폐기됨 44%만 커밋 생존 — 자율성 높을수록 폐기율 증가
바이브 코딩이 보안에 위험 인간 작성 코드 대비 9배 더 많은 취약점 도입
사용자가 지속적으로 에이전트를 견제 44% 턴에서 푸시백, 5%에서 직접 중단
자율성이 감독을 앞지름 에이전트는 길게 작동하지만 사용자에게 거의 질문하지 않음

큐레이팅된 문제가 아닌, 실제 개발자의 반복적이고 다양한 작업 흐름을 반영한 평가 체계 필요

적응적 상호작용 설계

사용자 시뮬레이터
