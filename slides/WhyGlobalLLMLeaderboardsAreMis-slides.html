왜 글로벌 LLM 리더보드가
오도하는가

이질적 지도 학습을 위한 작은 포트폴리오 접근법

Jai Moondra, Ayela Chughtai, Bhargavi Lanka, Swati Gupta

Carnegie Mellon University · MIT Sloan

arXiv:2605.06656v1

LLM 평가의 현재 상황

리더보드의 부상

  • Arena(LMArena) 등 개방형 평가 플랫폼
  • 인간 피드백 기반 쌍대 비교
  • 투명한 '골드 스탠다드'로 인정
  • WSJ 등 주요 언론에서 인용

숨겨진 문제

  • 글로벌 단일 순위가 대표성 없음
  • 다양한 사용자 의견이 하나로 압축
  • 상충하는 선호가 지워짐
  • 특정 집단 선호에 지배됨

"글로벌 순위는 이질적인 투표 집단 사이에서 헤징(hedging)하며, 모든 집단에서 예측 성능을 잃게 된다."

분석 대상: Arena 데이터셋

52

분석된 LLM 수

116

언어 수

~89K

쌍대 비교 건수

Bradley-Terry (BT) 모델이란?

각 LLM ℓ에 대해 Elo 점수 θℓ를 추정하는 표준 모델. 점수가 높을수록 사용자가 해당 모델을 더 선호할 확률이 높다고 가정한다.

Arena는 비슷한 순위의 모델을 더 자주 매칭하여 순위를 분리하려 하지만, 언어 집단 간 선호 차이는 설명하지 못한다.

핵심 발견: 글로벌 순위의 붕괴

투표 상쇄 현상

72.3%

결정적(non-tied) 투표 중 서로 반대되는 투표가 상쇄된 비율

"A가 B보다 낫다" vs "B가 A보다 낫다"가 거의 같은 빈도로 발생

구별 불가능한 상위 모델

0.53

상위 50개 모델 간 최대 승리 확률 — 거의 무작위

1위 모델 vs 50위 모델 = 50점 Elo 차이에 불과

글로벌 BT 순위가 승자를 ≥70% 확률로 예측할 수 있는 투표는 전체의 단 10.3%에 불과 (학습 데이터 기준)

원인: 구조화된 이질성

사용자 선호는 언어, 작업 유형, 시간에 따라 강하게 구조화된 이질성을 보인다.

상위 5개 모델의 순위 변동

  • 언어어족별 순위에서 평균 5~14위 변동
  • Grok-4: 아프로아시아어족에서 52위로 추락
  • 글로벌 1위가 모든 하위 집단에서 1위가 아님

시간에 따른 변동

  • 시간별 그룹핑에서도 이질성 존재
  • 하지만 언어가 가장 강력한 구조화 요인
  • 작업 유형 단독 분류는 여전히 비대표적

"전역적으로 보이는 노이즈는 사실 일관되지만 충돌하는 하위 집단의 혼합이다."

언어가 핵심이다

언어(및 언어어족)로 그룹화하면 투표 일치도가 극적으로 증가한다.

Elo 점수 스프레드 비교 (max - min)

그룹핑 방식 스프레드 수준 순위 일관성
글로벌 (전체) 매우 낮음 거의 무작위
무작위 그룹핑 낮음 비대표적
시간별 낮음 제한적
작업 유형만 보통 부분적
언어별 매우 높음 우수
언어어족 × 작업 최고 최우수

언어어족 기반 순위는 글로벌 순위보다 Elo 스프레드가 최대 2자릿수(100배) 더 크다.

시각적 증거: 하위 집단 vs 글로벌

승자 예측 확률

388개 하위 집단 BT 순위 vs 글로벌 순위

  • 언어/어족 기반 순위: 관찰된 승자에 더 높은 확률 할당
  • 글로벌 순위: 항상 가장 낮은 예측력
  • 작업별 순위: 글로벌과 유사한 수준

로그 손실 (Log Loss)

낮을수록 더 나은 예측 성능

  • 언어/어족 기반 순위: 훨씬 더 낮은 로그 손실
  • 글로벌 순위: 항상 가장 높은 로그 손실
  • 일관된 결과: 언어가 예측력의 핵심

심슨의 역설(Simpson's Paradox)과 유사: 집단을 합치면 추세가 역전되어 글로벌 순위가 모든 하위 집단에서 통계적 유의성을 잃는다.

해결책: (λ, ν)-포트폴리오

단일 글로벌 모델 대신, 작은 모델 집합이 대부분의 사용자를 '커버'하도록 한다.

λ

허용 가능한 예측 오류 상한

모델이 개별 투표를 얼마나 잘 예측해야 하는가?

ν

커버해야 할 사용자 비율

전체 중 최소 몇 %를 만족시켜야 하는가?

핵심 아이디어

"대규모 ML 모델 앙상블이 주어졌을 때, 각 사용자가 포트폴리오 내 최소 하나의 모델로부터 합리적인 서비스를 받을 수 있는 최소 크기의 부분집합을 찾아라."

알고리즘: 부분 집합 커버 문제

(λ, ν)-포트폴리오 문제는 부분 집합 커버(Partial Set Cover)의 변형으로 공식화된다.

1
입력
모델 앙상블 + 사용자 투표
2
커버 집합 구성
각 모델이 λ 이하 오류로 커버하는 사용자 집합 계산
3
탐욕적 선택
가장 많은 미커버 사용자를 추가하는 모델 반복 선택
4
출력
최소 크기의 (λ, ν)-포트폴리오

MIP 정식화

혼합 정수 계획(MIP)으로도 정확히 해결 가능. 소규모 인스턴스에 적합.

VC 차원 보장

집합 커버-VC 차원 연결(Brönnimann & Goodrich, 1994)을 활용하여 포트폴리오 크기에 대한 이론적 상한 제공.

확장: COMPAS 사례 연구

LLM뿐 아니라 일반적인 이질적 지도 학습에 포트폴리오 접근법이 적용됨

COMPAS 데이터셋

  • 형사 피고인 2년 내 재범 예측
  • 공정성 정규화된 분류 모델 앙상블
  • 다양한 공정성-정확도 트레이드오프

발견된 사각지대

  • 커버하기 어려운 피고인 특성 식별
  • 이전 전과 적음, 연령이 적음
  • 데이터 내 blind spot 탐지 가능

포트폴리오가 커버하지 못하는 개체는 데이터의 문제점이나 모델 부족을 시사하며, 정책 입안자에게 독립적으로 유용한 정보를 제공한다.

결론 및 시사점

1

글로벌 순위는 본질적으로 한계가 있다

구조화된 이질성 하에서 단일 순위는 정보를 지우고 헤징한다

2

언어가 LLM 평가 이질성의 핵심 요인

언어어족별 분석이 순위 일관성을 극적으로 향상시킨다

3

포트폴리오가 글로벌 순위를 대체할 대안

소수의 모델로 다수의 사용자를 커버 — 단일 모델보다 훨씬 효과적

4

일반 지도 학습으로의 확장 가능

COMPAS 등 공정성 문제에서 사각지대 탐지에 활용

"리더보드는 단일 순위가 아니라 포트폴리오를 제시해야 한다. 충돌하는 선호를 보존하는 것이 정보를 지우는 것보다 낫다."
