Long-Context Aware Upcycling:
새로운 하이브리드 LLM 확장의 지평

사전 학습된 Transformer를 하이브리드 아키텍처로 전환하면서
단문 품질을 유지하고 장문 능력을 향상시키는 실용적 방법

Parsa Ashrafi Fashi, Utkarsh Saxena, Mehdi Rezagholizadeh 외 · AMD

문제 정의

왜 롱컨텍스트 업사이클링이 필요한가?

기존 하이브리드 모델의 한계

  • 대부분 처음부터 사전 학습 (비용 폭발)
  • 기존 Transformer 체크포인트 재사용 불가
  • Jamba, Samba, Qwen3-Next 등 모두 Scratch 학습

기존 업사이클링의 blind spot

  • MambaInLlama, Zebra-Llama 등 단문 성능에만 집중
  • 장문(롱컨텍스트) 능력 간과
  • 일반적으로 24K 토큰 수준까지만 학습
장문 보존을 업사이클링의 핵심 목표로 삼아야 한다 — 단문 품질과 병행하여.

하이브리드 구조 설계

MLA 대 선형 블록 비율이 품질-효율성 트레이드오프를 결정

Multi-Head Latent Attention (MLA)

  • 잠재 표현을 통한 KV-캐시 압축
  • 소프트맥스 어텐션의 표현력 유지
  • 주의력 집중이 필요한 레이어에 배치

예: 4MLA12M2 = 4개 MLA + 12개 Mamba-2 레이어

선형 순환 블록

Mamba-2

Gated DeltaNet

게이트 델타 규칙 기반 선형 어텐션

  • KV-캐시 오버헤드 제로
  • 장문 시퀀스 효율적 처리

Llama 및 Qwen 두 가지 Transformer 계열 × Mamba-2 및 GDN 두 가지 선형 블록 타입에서 레시피의 일반성 검증

초기화

SVD 기반 가중치 초기화

업사이클링의 핵심: 사전 학습된 지식을 하이브리드 블록으로 어떻게 전이할 것인가?

1
순수 MLA 모델 + 순수 선형 모델 구성

기존 Transformer의 모든 어텐션 블록을 각각 MLA 또는 Mamba-2/GDN으로 교체

2
MLA 초기화 (X-EcoMLA 방식)

Query 경로: SVD 분해로 KV 압축 행렬 도출 · Key-Value 경로: 결합 SVD로 잠재 KV 생성

3
GDN 초기화 (본 논문 제안)

GQA 확장: H_kv < H_q 경우 KV 헤드 반복 확장 → 프로젝션 가중치 SVD 분해 → GDN 파라미터로 매핑

4
MLP & LayerNorm 그대로 복사

SwiGLU MLP 가중치와 RMSNorm 파라미터는 원본 Transformer에서 그대로 유지

차원 불일치 문제를 SVD 분해로 해결하여, 정보 손실을 최소화하면서 구조 변환을 달성
학습 전략

2단계 미세조정 + 장문 증류

Stage 1: 단문 미세조정 (Enhanced-ILD)

  • 중간 레이어 증류 (Intermediate Layer Distillation)
  • 순수 모델들의 숨겨진 상태를 결합하여 타겟 하이브리드 모델 지도
  • 8K 시퀀스 길이로 단문 품질 안정화

Stage 2: 장문 학습 + 교사 증류

  • 단계적 시퀀스 확장: 8K → 32K → 64K
  • 교사 모델의 Chunk-wise KL 발산으로 지도
  • 장문 일반화 능력 획득

메모리 효율적 장문 증류 기법

Fused Linear CE

크로스 엔트로피 커널 퓨전

Chunked KL

청크 단위 KL 발산 계산

Hidden-State KL

로짓 없는 증류 (Logit-Free)

Triton 기반 커스텀 커널로 교사 모델의 메모리 오버헤드를 최소화하여 장문 학습이 가능해짐

절제 연구

설계 선택의 영향 분석

학습 시퀀스 길이의 효과

  • 8K만 학습: 64K에서 급격한 성능 저하
  • 64K까지 단계적 확장: 장문 일반화 크게 향상
  • NIAH 벤치마크에서도 64K 학습이 8K 대비 우세
장문 미세조정은 업사이클링에서 선택이 아닌 필수

Position Interpolation vs HyLo

  • PI만 적용: 단문 성능은 유지되나 장문에서 불안정
  • HyLo의 단계적 학습 + 증류가 PI보다 일관되게 우수
  • 선형 블록 자체가 외삽에 더 유리한 구조
구조적 적응이 위치 보간보다 근본적인 해결책

선형 블록 타입 비교

Mamba-2와 GDN 모두 HyLo 레시피와 호환되며, GDN이 일부 장문 태스크에서 더 안정적인 경향을 보임. 두 타입 모두 기존 Transformer 대비 KV-캐시 오버헤드가 0이므로 동일한 메모리 이점 확보.

Needle-in-Haystack

합성 장문 검색 벤치마크

다양한 컨텍스트 깊이에서 타겟 정보를 검색하는 능력 평가

8K 시퀀스 학습 모델

  • 짧은 깊이에서는 양호한 성능
  • 깊이가 증가할수록 성능 급락
  • 컨텍스트 길이 외삽에 한계 노출

64K 시퀀스 학습 모델

  • 전체 깊이 범위에서 안정적 성능
  • Llama-3.2-1B 원본과 동등한 검색 능력
  • Zebra-Llama 대비 현저히 우수
HyLo의 4MLA12M2 모델은 원본 대비 단 3.9% KV-캐시로 Llama-3.2-1B와 동등한 장문 검색 성능 달성

이 결과는 하이브리드 아키텍처가 업사이클링 후에도 Transformer의 장문 정보 검색 능력을 보존할 수 있음을 실증적으로 보여줌

핵심 요약

HyLo의 4가지 핵심 기여

1
롱컨텍스트 인지 업사이클링

기존 업사이클링이 간과했던 장문 보존을 핵심 목표로 설정, Zebra-Llama 대비 장문 성능 크게 향상

2
확장된 장문 학습 체제

기존 24K 수준에서 8K → 32K → 64K로 단계적 확장, 시퀀스 길이가 장문 일반화에 미치는 영향 체계적 분석

3
교사 지도 장문 증류

Chunk-wise KL 지도로 장문 성능 크게 향상, 메모리 효율적 커널(Triton)로 학습 가능 수준의 메모리로 구현

4
고처리량 추론 서빙

vLLM 통합으로 8× MI300X에서 2M 토큰 서빙, Llama 대비 30× 컨텍스트 확장

결론

시사점 및 향후 방향

핵심 시사점

  • 처음부터 학습 없이도 장문 하이브리드 모델 구축 가능
  • 단문 품질 유지와 장문 확장은 양립 가능
  • 적은 데이터(10B 토큰)로도 대규모 학습(400B) 모델을 능가
  • 레시피가 모델 계열(Llama/Qwen)과 선형 블록(M2/GDN)에 걸쳐 일반화

한계 및 향후 과제

  • 1B-3B 규모에서 검증, 더 큰 규모(7B+)에서의 스케일링 미확인
  • 교사 증류의 최적 청크 크기 및 스케줄링 추가 연구 필요
  • MLA-선형 레이어 비율의 자동 탐색 방법 미탐색
  • 멀티모달 확장 및 RLHF와의 결합 가능성
HyLo는 업사이클링이 단순한 "비용 절감"을 넘어, 새로운 능력(장문 처리)을 부여하는 강력한 패러다임임을 보여줌
