문제 정의

워크플로우 실행은 자동화되었지만,
의미 변환은 여전히 수동

"유럽 및 아프리카 인구의 1~5번 염색체 돌연변이 패턴을 비교하라" — 이 연구 질문을 실행 가능한 워크플로우 DAG로 변환하려면?

진입 장벽

인프라 지식이 없는 과학자는 워크플로우 시스템을 혼자 사용할 수 없습니다

오류 전파

잘못된 어휘 매핑(예: 인구 코드 혼동)이 결과에 직접 영향을 미칩니다

재현성 저해

변환 논리가 암시적이라 실험 반복 시 문서화되지 않은 결정을 재구성해야 합니다

Pegasus, Nextflow, Snakemake, Galaxy — 모두 실행만 자동화합니다

핵심 제안

3계층 분해로 비결정론을 격리하다

LLM의 비결정성을 의도 추출에만 한정 — 동일한 인텐트는 항상 동일한 워크플로우를 생성

SEMANTIC LAYER

의미 계층

LLM이 자연어를 구조화된 인텐트로 해석

DETERMINISTIC LAYER

결정론 계층

검증된 생성기가 인텐트를 실행 가능한 DAG로 변환

KNOWLEDGE LAYER

지식 계층

도메인 전문가가 작성한 Skills — 마크다운 문서

핵심 원리: LLM은 "무엇을"만 결정하고, "어떻게"는 결정론적 코드가 담당

구현

1000 Genomes 인구 유전학 워크플로우

도메인

  • 1000 Genomes Project VCF 데이터
  • 인구 그룹 간 돌연변이 패턴 비교
  • 염색체 단위 영역 지정 분석
  • 병렬 VCF 처리 파이프라인

인프라

  • HyperFlow WMS 워크플로우 엔진
  • Kubernetes 클러스터 실행 환경
  • 동적 병렬도 조절 (vCPU 기반)
  • bgzip 압축 데이터 스테이징
Deferred Generation: 프로비저닝 후 실제 데이터 크기와 가용 vCPU를 측정 → 그 이후에 최적화된 workflow.json 생성

엔드투엔드 흐름 예시

연구자: "유럽 인구의 1~3번 염색체에서 돌연변이 빈도를 분석하라" ↓ Conductor (도메인 분류: population_genomics) ↓ Workflow Composer + Skills → intent: {populations: [EUR], chromosomes: [chr1,chr2,chr3], analysis: frequency} ↓ [Human Validation Gate] — "이 분석이 맞습니까?" ↓ Deployment Service — Kubernetes 파드 프로비저닝 ↓ 측정: 데이터 크기 2.1GB, 가용 vCPU 8 ↓ Workflow Composer — 병렬도 4로 설정, 선택 추출 적용 ↓ Execution Sentinel — HyperFlow DAG 실행 및 모니터링 → 결과 완료 (LLM 오버헤드 <15s, 비용 <$0.001)
기여

네 가지 핵심 기여

01

하이브리드 에이전트 아키텍처

의미·결정론·지식 3계층 분해로 LLM 비결정성을 인텐트 추출에 한정. 동일 인텐트 → 항상 동일 워크플로우 보장

02

Skills: 도메인 전문가 작성 지식

마크다운 텍스트 파일. ML 지식 불필요, 버전 관리 가능, 도메인 전문가가 직접 감사

03

엔드투엔드 에이전트 파이프라인

자연어 → Kubernetes 실행. 4개 에이전트, Human-in-the-Loop 검증, 지연 생성(Deferred Generation)

04

Skills 기반 실행 시간 최적화

어휘 매핑뿐 아니라 최적화 전략도 인코딩. 선택적 데이터 추출로 전송량 92% 감소 달성
