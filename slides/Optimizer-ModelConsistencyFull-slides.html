Optimizer-Model Consistency: Full Finetuning with the Same Optimizer as Pretraining Forgets Less

Yuxing Liu, Jianyu Wang, Tong Zhang

UIUC · Apple

배경: 학습-잊기 트레이드오프

사전학습 (Pretraining)

  • 대규모 혼합 코퍼스 학습
  • 일반적인 언어 능력 획득
  • AdamW, Muon 등 옵티마이저 사용

지도 미세조정 (SFT)

  • 소규모 고품질 데이터로 학습
  • 새로운 지식·기술 습득
  • 사전학습 지식 잊지 않아야 함
핵심 질문: SFT 단계에서 어떤 학습 알고리즘이 학습-잊기 트레이드오프를 최적으로 달성하는가?

핵심 관찰: 옵티마이저-모델 일관성

사전학습에 사용한 동일한 옵티마이저로 Full Finetuning을 수행하면, 다른 옵티마이저나 LoRA보다 더 나은 학습-잊기 트레이드오프를 달성한다.

주요 결과: 동일 옵티마이저의 우위

Pareto frontier 분석 (x축: 잊기량, y축: 학습량 — 둘 다 낮을수록 좋음)

AdamW로 사전학습 시

  • AdamW로 SFT가 최우위 Pareto
  • Muon으로 SFT는 잊기 증가
  • SGD는 학습 가능하나 잊기 심함

Muon으로 사전학습 시

  • Muon으로 SFT가 최우위 Pareto
  • AdamW로 SFT는 잊기 증가
  • 역방향도 동일하게 일관성 관찰

수학, 지시수행, 코딩 모든 태스크에서 동일한 패턴 확인

Full Finetuning vs LoRA

LoRA는 기존 연구에서 "잊기가 적다"고 알려져 있으나, 학습률 탐색을 포함한 전체 Pareto 관점에서 다른 결론

LoRA 최적 체크포인트는 확실히 잊기가 적음 (기존 결과와 일치)
그러나 모든 LoRA 체크포인트에 대해 더 적게 잊으면서 같거나 더 잘 학습하는 Full Finetuning 체크포인트가 존재
LoRA rank가 클수록 트레이드오프가 개선되나, 여전히 Full Finetuning에 미달
LoRA는 파라미터 효율성 측면에서 훌륭하지만, 학습-잊기 트레이드오프를 개선한다는 가정은 성립하지 않음

7B 모델에서의 검증

Llama-2-7B + MetaMathQA 설정

Biderman et al. (2024) 설정을 따라 Llama-2-7B (원래 AdamW로 사전학습됨)를 MetaMathQA로 1 epoch SFT 수행

  • AdamW Full Finetuning이 가장 우측 상단 (최소 잊기 + 최대 학습)
  • Weight decay 없이도 다른 옵티마이저보다 우수
  • LoRA 및 Muon Full Finetuning보다 일관되게 나은 트레이드오프

작은 모델(GPT-2)에서의 관찰이 큰 모델(7B)에서도 견고하게 유지

왜 이런 현상이 발생하는가?

단계 1: 옵티마이저가 모델을 형성한다

LLM 학습은 비볼록 문제이므로, 서로 다른 옵티마이저는 특정 속성을 가진 해(solution)을 선호한다. 옵티마이저에 내재된 노름(norm) 기반 정규화 효과가 모델의 활성화(activation) 패턴을 형성한다.

단계 2: SFT 업데이트 구조가 잊기에 영향

옵티마이저가 형성한 활성화 정규화에 맞춰 가중치 업데이트가 특정 구조를 따를 때, 사전학습 지식의 잊기가 최소화된다.

결론: 동일 옵티마이저가 정확한 구조 제공

사전학습 시와 동일한 옵티마이저를 사용하면, 그 정규화에 이미 적응된 업데이트 구조가 자연스럽게 얻어진다.

이론적 설명: 활성화 정규화

핵심 아이디어 (정리화)

1
옵티마이저 O는 특정 노름 ‖·‖O와 연관
2
O로 사전학습된 모델의 활성화는 ‖·‖O에 의해 정규화됨
3
SFT에서 가중치 업데이트 ΔW가 이 정규화 구조와 일치하면 잊기 최소화
동일 옵티마이저 사용 시 ΔW가 자연스럽게 이 구조 따름

Pethick et al. (2025)의 행렬 유도 노름 프레임워크 기반 + 본 논문의 이론적 증명 (Theorem 1, 부록 C)

사례 연구: Muon vs AdamW

옵티마이저 일관성 관점에서 Muon과 AdamW를 사전학습~SFT 전체에 걸쳐 비교

장점 Muon 사전학습

  • 일반적으로 더 강한 사전학습 체크포인트
  • 여러 다운스트림에서 낮은 loss

단점 Muon SFT

  • 추론 태스크(수학)에서 SFT 후 성능 하락
  • AdamW보다 반드시 좋지 않음
옵티마이저 평가 시 사전학습 성능만이 아닌 SFT 후 성능까지 포함한 종합적 지표가 필요하다

합성 실험: 암기 vs 패턴 학습

Muon이 추론 태스크에서 약한 이유를 파악하기 위해 합성 언어 모델링 실험 수행

Muon의 경향

  • 암기(memorization)에 매우 강함
  • 소량 데이터로 패턴 학습에 취약
  • 업데이트 공식의 특성에서 기인

AdamW의 경향

  • 패턴 학습에 더 적합
  • 소량 데이터(SFT) 환경에서 유리
  • 추론 태스크에서 SFT 후 우수

Muon의 암기 편향은 대규모 데이터인 사전학습에서는 장점이지만, 소량 데이터인 SFT에서는 패턴 습득을 방해할 수 있음
