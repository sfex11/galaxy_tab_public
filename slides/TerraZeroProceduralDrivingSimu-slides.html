TerraZero

Zero-Demonstration Self-Play를 위한
절차적 주행 시뮬레이션

인간 시연 제로(Zero) · 로그 데이터 제로 · 폴백 플래너 제로
오직 강화학습만으로 강건한 자율주행 정책을 학습

문제 정의: 로그 데이터의 한계

루틴 주행의 지배

실제 주행 데이터의 대부분은 정속 차선 유지, 완만한 커브, 일반적인 정지 등 학습 가치가 적은 상황으로 구성됨

안전 핵심 롱테일의 희소성

밀집 고속도로 합류, 공격적 끼어들기, 보행자 근접 상황 등 배포 준비도를 결정하는 사건은 극히 드묾

모방 학습은 로그에 거의 없는 행동을 가르칠 수 없고, 대규모 수집으로도 분포 격차는 느리게만 좁혀짐 — 시뮬레이션이 보완 경로가 되어야 함

기존 접근법의 한계

범주 대표 시스템 장점 한계
픽셀 레벨 CARLA, SMARTS 사실적 렌더링 초당 수십 프레임으로 RL 불가
경량 오브젝트 PufferDrive 높은 처리량 단일 에이전트, 고정 역학, 신호등 없음
학습된 교통 모델 SimNet, BITS 현실적 궤적 비반응적 교통으로 폐루프 개선 불가
셀프플레이 Gigaflow 반응적 다중 에이전트 소수 합성 맵, 단일 역학 모델

TerraZero는 처리량-충실도 트레이드오프를 동시에 해결하는 것을 목표로 함

TerraZero 개요: 세 가지 기둥

1

고속 오브젝트 레벨
시뮬레이터

C 엔진, CPU 시뮬레이션 + GPU 추론, 제로 카피 경로

2

절차적 시나리오
생성기

로그 데이터를 맵 지오메트리로만 사용, 무한한 시나리오 공간 조립

3

강건한 셀프플레이
학습 레시피

PPO, 도메인 랜덤화, 팝플레이, 분산 학습

핵심 철학: 로그 데이터는 시작 분포일 뿐 학습 분포가 아님

고속 시뮬레이터 설계

CPU/GPU 분업 구조

  • CPU: 물리 시뮬레이션, 교통 규칙 집행
  • GPU: 정책 추론 (제로 카피 경로)
  • 밀집 특징 패킹 + 16비트 관측
  • NUMA 인식 오케스트레이션

구성 가능한 시뮬레이션 기질

  • 이종 에이전트: 차량, 보행자, 자전거
  • 클래스별 다중 역학 모델
  • 충돌·오프로드·차선 방향·신호등 규칙
  • 컴팩트 바이너리 시나리오 포맷
단일 서버급 GPU에서 1.3M agent-steps/sec 8-GPU 노드에서 2.8M SPS

절차적 시나리오 생성

1
맵 로딩
WOMD, nuPlan(4개 도시), CARLA(5개 타운)의 실제 지오메트리
2
장면 초기화
랜덤 에이전트 배치, 밀도·차원 랜덤화, 목표 할당
3
NPC 구성
반응형 차량, 정적 차량, 추돌 클러스터, 공사 구역, 보행자·자전거
4
신호 제어
3종 랜덤화된 신호 컨트롤러로 교차로 동작 생성

에피소드마다 모든 축(역학, 보상, 크기, 밀도, 신호, NPC 구성)이 변하므로, 하나의 맵에서 사실상 무한한 학습 시나리오가 파생됨

셀프플레이 학습 레시피

컴퓨트 효율적 RL

  • 샐리언시 우선 샘플링
  • 컴팩트 피드포워드 정책
  • V-trace 오프폴리시 보정
  • PopArt 가치 정규화

도메인 랜덤화 & 팝플레이

  • 보상 가중치 랜덤화
  • 운동학 파라미터 랜덤화
  • 목표 드롭아웃
  • 팝플레이로 셀프플레이 대칭성 파괴

"Zero"의 의미: 인간 시연 제로, 모방 손실 제로, 로그 궤적 제로, 추론 시 폴백 플래너 제로 — 오직 강화 신호만으로 학습

시뮬레이션 처리량 비교

시스템 에이전트 유형 역학 모델 신호등 처리량 (SPS)
CARLA 단일 단일 O ~10-50
SMARTS 다중 단일 O ~1K
PufferDrive 단일 고정 X ~100K
GPUDrive 다중 단일 제한 ~1M
TerraZero 이종 다중 다중 O 1.3M ~ 2.8M

충실도(이종 에이전트, 다중 역학, 신호등)를 유지하면서도 기존 시뮬레이터보다 한두 자릿수 빠른 처리량 달성

플래너 벤치마크: nuPlan val14

루틴 주행 평가 — TerraZero는 최고 수준의 성능을 기록하며, 특히 안전성 지표에서 1위

1위

충돌률
(Collision Rate)

1위

충돌까지 시간
(Time-to-Collision)

Top-tier

전체 드라이빙 점수
(Driving Score)

더 큰 학습 플래너보다 높은 안전성을 보이며, 인간 시연 없이 오직 RL로 달성

플래너 벤치마크: InterPlan

롱테일 주행 평가 — 완전 학습된 정책으로서 최초로 1위

InterPlan의 특징

  • 안전 핵심 롱테일 시나리오 집중
  • 밀집 합류, 공격적 끼어들기 등
  • 정확히 TerraZero가 절차적 생성으로 목표하는 상황들

TerraZero의 결과

  • 더 큰 학습 플래너들 앞서는 성능
  • 폴백 플래너 없이 엔드투엔드 동작
  • 절차적 시나리오 생성의 효과 입증

Sim Agent 벤치마크: WOSAC

Waymo Open Sim Agents Challenge — 하나의 스택이 주행 정책과 시뮬 에이전트를 모두 수행

시뮬 에이전트로서의 역할

차량, 보행자, 자전거를 동시에 제어하는 현실적인 교통 흐름 생성

결과

시연 없는(demonstration-free) 방법론 중 최고 성능, 레퍼런스 앵커 셀프플레이 방법과 경쟁적

하나의 레시피로 자동차·트럭 주행 정책과 다중 모달리티 시뮬 에이전트를 모두 학습 가능 — 광범위한 구성 가능성

일반화: 제로샷 전이

도시 간 전이

학습하지 않은 도시에서도 제로샷 동작

데이터셋 간 전이

WOMD에서 학습 후 nuPlan 평가 성공

좌측 통행 등장

명시적 감독 없이 좌측 통행 자발적 출현

nuPlan의 4개 도시 중 우측·좌측 통행이 혼재함에도, 명시적 라벨 없이 맵 구조로부터 통행 방향을 스스로 학습 — 절차적 생성과 도메인 랜덤화의 강건성 증명

결론 및 한계

핵심 기여

  • 충실도를 유지한 최고 수준의 시뮬레이션 처리량
  • 로그 데이터를 맵 지오메트리로만 활용하는 절차적 생성
  • 시연 제로의 셀프플레이로 SOTA 달성
  • 하나의 스택으로 주행 정책 + 시뮬 에이전트 모두 수행

한계점

  • 오브젝트 레벨 추상화로 인해 시각적 인식 파이프라인과의 갭 존재
  • 레이더, 카메라 등 센서 모델링 미포함
  • Rule-based NPC로 인한 행동 다양성의 상한
  • 실차 전이(sim-to-real) 검증 필요

향후 방향: 학습된 NPC 도입, 멀티모달 센서 통합, 실차 전이 연구
