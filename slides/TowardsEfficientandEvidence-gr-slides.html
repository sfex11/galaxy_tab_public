Towards Efficient and Evidence-grounded Mobility Prediction with LLM-Driven Agents

Linyao Chen, Qinlao Zhao, Zechen Li, Mingming Li, Likun Ni, Jinyu Chen, Yuhao Yao, Xuan Song, Noboru Koshizuka, Hiroki H. Kobayashi

The University of Tokyo · HUST · UNSW · LocationMind Inc. · SUSTech · Jilin University

문제 정의와 배경

개인 수준의 다음 위치 예측은 도시 시뮬레이션, 교통 계획, 정책 분석의 핵심 과제입니다.

지도 학습 시퀀스 모델

RNN, Transformer 등은 높은 정확도를 달성하지만 태스크 특화 학습이 필요하고 의사결정 수준의 투명성이 제한적입니다.

기존 LLM 기반 방법

해석 가능성은 개선되지만, 정적 프롬프트와 단일 추론에 의존하여 이동 신호가 약하거나 충돌할 때 추가 증거를 탐색할 수 없습니다.

"모든 이동 인스턴스가 동일하게 어려운 것은 아니다. 신뢰할 수 있는 예측기는 단일 패스 후에 결정을 내려서는 안 된다."

기존 패러다임의 한계

01

고정 추론 절차

모든 샘플이 동일한 프롬프트와 단일 생성 단계를 거치며, 난이도에 따른 추론량 조절이 불가합니다.

02

사전 스크립트 워크플로

에이전트형 프레임워크도 구조화된 메모리나 추론 모듈을 도입하나, 증거의 양과 유형을 인스턴스별로 적응하지 못합니다.

03

검증·수정 불가

고정 컨텍스트로 예측 후, 결정을 검증하거나 수정할 구조화된 메커니즘이 존재하지 않습니다.

AgentMob: 핵심 아이디어

다음 위치 예측을 적응적 증거 통제 의사결정으로 재정의합니다.

1
빠른 경로 판별
강한 역사적 규칙성이 있으면 즉시 반환
2
적응적 도구 호출
모호한 사례는 도구를 반복적으로 호출하여 증거 수집
3
증거 기반 종료
충분한 증거가 모이면 예측 결정

LLM을 직접 예측기가 아닌 컨트롤러로 사용 — 각 인스턴스에 필요한 증거량을 결정

학습 불필요(training-free) · smolagents 기반 구현

빠른 경로 예측

전체 도구 호출 루프 진입 전에 경량 규칙성 검사를 수행합니다.

판별 조건

사용자의 과거 기록에서 동일 요일·시간대에 특정 위치가 지배적으로 나타나면, 해당 위치를 바로 예측으로 반환합니다.

  • 루틴 사례를 효율적으로 해결 — 불필요한 LLM/도구 연산 방지
  • 역사적 신호가 이미 강한 경우 추론 비용 최소화
  • 모호한 사례만 도구 호출 모드로 진입

적응적 도구 호출 예측

강한 역사적 규칙성이 없을 경우, AgentMob은 도구 호출 모드로 진입합니다.

증거 수집

최근 궤적, 역사적 행동, 체류-이동 가능성, 지리적 증거를 도구로 탐색

신호 충돌 해결

최근 맥락, 장기 루틴, 체류-이동 성향, 지리적 타당성이 다른 후보를 가리킬 때 교차 검증

증거 기반 종료

충분한 증거가 축적되면 추가 도구 호출을 중단하고 순위 예측 생성

최종 예측은 불투명한 프롬프트 생성이 아닌 명시적 도구 출력에 기반합니다.

이동성 분석 도구상자

이동 맥락 검색기

대상 시간 직전의 최근 궤적 패턴(방문 위치, 시간 간격, 이동 방향)을 검색하여 단기 맥락 제공

지리 정보 검색기

후보 위치 간의 거리, 도시 기능 설명, 공간적 인접성 등 지리적 증거를 제공

체류-이동 추정기

현재 위치에서 체류할 가능성 vs 이동할 가능성을 추정하여 예측 방향을 안내

역사적 행동 검색기

사용자의 장기 방문 빈도, 요일/시간대별 통계, 선호 위치 프로필을 제공

예측과 감사 가능성

AgentMob의 최종 예측은 타임스탬프 바운드 도구 출력으로 추적 가능합니다.

감사 가능한 예측 추적

  • 어떤 도구가 호출되었는지 기록
  • 각 도구의 입력·출력 시간戳 보존
  • LLM의 추론 체인 전체 재현 가능

기존 방법과의 차이

  • 지도 학습 DNN: 블랙박스, 의사결정 근거 불가
  • 프롬프트 LLM: 고정 컨텍스트, 검증 불가
  • AgentMob: 명시적 증거 기반, 전체 추적 가능

주요 결과

AgentMob은 학습 불필요 LLM 기반 방법 중 최강 성능을 달성합니다.

방법BW Acc@1YJMob100K Acc@1Shanghai ISP Acc@1
LLM-Mob (GPT-4o)44.60%27.46%26.45%
LLM Urban Residents51.58%29.21%28.98%
AgentMove (GPT-4o)58.26%29.12%29.21%
AgentMob (GPT-5.4)71.42%33.14%33.50%

BW 데이터셋에서 같은 도구를 사용한 통계적 베이스라인 대비 비빠른경로 사례에서 Acc@1이 30.65% → 48.62%로 향상 — LLM 컨트롤러의 핵심 가치는 모호한 예측의 해결에 있음

난이도 계층 분석

이동 인스턴스를 난이도에 따라 분류하여 적응적 증거 수집의 효과를 분석했습니다.

쉬움

강한 역사적 규칙성 → 빠른 경로로 해결

불필요한 연산 최소화

보통

부분적 신호 → 1~2회 도구 호출로 해결

적절한 증거 수집

어려움

신호 충돌 → 다중 도구 교차 검증 필요

LLM 컨트롤러가 가장 큰 이점 발휘

추론 과정 분석 결과, AgentMob은 모호한 사례에서 체류-이동 추정기와 역사적 행동 검색기를 전략적으로 조합하여 충돌하는 신호를 해결하는 패턴을 보였습니다.

효율성 및 비용 분석

빠른 경로의 비용 절감

루틴 사례를 빠른 경로로 처리하여 LLM API 호출 및 도구 실행 비용을 크게 절감합니다. 난이도에 따른 추론량 할당이 핵심입니다.

비빠른경로의 추가 비용

모호한 사례에서는 다중 도구 호출이 발생하지만, 정확도 향상(30.65% → 48.62%)에 비해 추가 비용은 정당화됩니다.

백본 LLM 민감도

GPT-5.4 > Claude 3.5 Sonnet > GPT-4.1 > GPT-4o 순으로 성능이 높으며, 오픈소스 모델(Llama 3.1 70B, Qwen 2.5 72B)에서도 폐쇄소스 프롬프트 방법보다 우수한 성능을 보입니다.

절제 연구

각 도구 구성요소의 개별 기여도를 분석했습니다.

제거 구성요소영향역할
체류-이동 추정기성능 하락이동 vs 체류 방향 결정에 핵심
역사적 행동 검색기성능 하락장기 패턴 기반 예측에 필수
이동 맥락 검색기부분 하락단기 궤적 맥락 제공
위치 프로파일러부분 하락후보 위치의 도시 기능 의미 제공

체류-이동 추정기와 역사적 행동 검색기의 기여가 가장 크며, 이 두 도구가 신호 충돌 해결에 핵심적인 역할을 합니다.

결론 및 향후 과제

  • 적응적 증거 통제 공식화 — 다음 위치 예측을 인스턴스 수준 의사결정 과정으로 재정의
  • 학습 불필요 도구 증강 에이전트 — 선택적 도구 호출로 감사 가능한 예측 추적 생성
  • 최강 학습 불필요 성능 — 3개 데이터셋, 다중 LLM 백본에서 일관된 우수 성능

한계점

LLM 백본의 추론 지연, 도구 호출 비용, 도시 외 지역으로의 일반화 미확정

향후 과제

더 작은 오픈소스 모델에서의 효율적 도구 사용, 멀티모달 증거(이미지, 센서) 통합, 실시간 도시 시뮬레이션 적용
