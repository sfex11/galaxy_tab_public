Agentic Web Browsers(AWBs)란?

Computer-Use Agents(CUAs)의 브라우저 전용 특화 하위 집합

  • LLM 기반 자율 웹 탐색
  • DOM 구조 및 실시간 스크린샷 해석
  • 자연어 지시를 인터페이스 동작으로 변환
  • 자율적 행동 계획 및 실행

발견 1: 높은 대화 유창성

참가자가 가장 높이 평가한 강점

  • 대화의 자연스러운 흐름
  • 상호작용의 전반적 유연성
  • 문맥에 따른 응답 상세도 조절

시스템이 사용자의 상황적 필요에 맞춰 응답의 세밀함 수준을 자동 조절하는 능력을 높이 평가

웹 접근성의 미래를 위한
상호작용 패러다임으로서의
명확한 증거

발견 2: 비시각적 피드백 부재

시스템이 자율적으로 요청을 처리할 때 시각적 피드백에만 의존

  • 에이전트 작동 시 색상 애니메이션 제공
  • 하지만 청각적·촉각적 피드백 없음
  • 응답 지연 시 불확실성 증가

핵심 문제

참가자는 잔존 시력과 기술 지식으로 상황을 이해했으나, 일반 저시력 사용자에게는 심각한 투명성 문제가 될 수 있음

"화면에서 무슨 일이 일어나고 있는지 알 수 없다"

대화 유창성 강점 문맥 적응형 응답, 자연스러운 흐름
상호작용 유연성 강점 전통적 AT 대비 뛰어난 유연성
비시각적 피드백 한계 시각적 애니메이션에만 의존, 투명성 부족
사용자 통제 한계 자율 행동 시 확인 없음, 임의 선택
전반적 잠재력 긍정적 한계 극복 시 웹 접근성 패러다임 전환 가능

결론 및 향후 과제

결론

  • AWB의 AT로서 초기 긍정적 전망 확인
  • 단일 사례이므로 일반화에는 주의 필요
  • 참가자의 기술 전문성이 긍정적 평가에 영향

향후 과제

  • 전맹 사용자 포함 다양한 참가자 그룹 확대
  • 스크린 리더 등 기존 AT와 비교 평가
  • 투명성, 통제, 신뢰 중심 추가 연구

시사점: 포용적 설계의 관점

본 연구에서 식별된 한계들 — 제한된 사용자 통제, 투명성 부족 — 은 시각 장애 여부와 무관하게 모든 사용자가 에이전트에 작업을 위임할 때 발생하는 보편적 문제일 수 있다.

핵심 메시지

AWB는 시각 장애 사용자를 위한 새로운 보조 기술로서
웹 탐색을 유창한 대화 경험으로 변환할 잠재력을 지닌다.

하지만 그 잠재력을 실현하려면
비시각적 피드백, 사용자 통제, 투명성 문제를 해결해야 한다.

접근성을 위한 설계가 곧 모두를 위한 설계다.
