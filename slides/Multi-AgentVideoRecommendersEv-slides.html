왜 멀티 에이전트인가?

기존 단일 모델 추천의 한계와 새로운 접근

비디오는 고차원 · 시간적 · 다중모달 데이터이다. 단일 파운데이션 모델은 장기 시청 이력을 픽셀 수준에서 처리할 수 없다. 멀티 에이전트 시스템은 인지와 추론을 분리하여 이 문제를 해결한다.

⚠

기존 접근의 한계

  • 정적 참여 지표만 최적화
  • 콜드 스타트 문제
  • 데이터 희소성
  • 동적 사용자 의도 포착 실패
✨

멀티 에이전트 해결책

  • 전문 에이전트 간 역할 분담
  • 비디오 이해 + 추론 + 메모리 분리
  • 설명 가능한 추천
  • 동적 피드백 통합

미해결 과제

MAVRS 실용화를 가로막는 5가지 핵심 과제

⚡ 1. 계산 비용 & 확장성

Agent4Rec(수천 에이전트)은 대부분의 연구소에 비용 면에서 비실용적. 실시간 프로덕션 추천에는 경량 증류 에이전트 또는 효율적 토큰 공유 메커니즘 필요

🎬 2. 다중모달 그라운딩 & 추론

현재 텍스트 중심 LLM은 풍부한 다중모달 데이터에 대한 추론이 어려움. IP 에이전트의 의미 요약은 손실 압축이며, 비디오 스트림에 대한 직접적 교차 모달 추론이 필요

📊 3. 평가의 어려움

오프라인 메트릭(nDCG, MRR)은 대화형 추천의 이점을 포착 불가. 시뮬레이션 기반 평가의 행동 충실도 검증 문제 미해결

🛡 4. 제어 가능성 & 신뢰성

에이전트 자율성 증가에 따른 인간 가치 정렬 보장 과제. 하위 에이전트의 목표 이탈 및 조용한 오류 전파 위험

⚖ 5. 인센티브 정렬

에이전트 간 목표 충돌(예: WatchTime vs Likes). 계산 경제학(경매, 계약 이론) 차용한 명시적 협력 메커니즘 설계 필요

하이브리드 RL-LLM 아키텍처
