문제 정의

기존 VLN 에이전트의 비효율적 탐색 문제

❌ 기존 접근법의 한계

  • 탐욕적 프론티어 선택 — 즉각적 의미 점수만으로 다음 위치 결정
  • 수동적 공간 기억 — 정적이고 이력을 활용하지 않는 맵
  • 국소 진동(Local Oscillation) — 두 지점 사이를 반복 이동
  • 중복 재방문 — 이미 탐색한 영역을 불필요하게 재탐색

✓ 필요한 능력: 메타인지

  • 탐색 진행 모니터링 — 현재 얼마나 진전했는지 인식
  • 전략 실패 진단 — 왜 목표에 도달하지 못하는지 분석
  • 행동 적응 — 실패로부터 학습하여 전략 수정
  • 효율적 자원 사용 — 불필요한 VLM 호출 최소화

핵심 통찰: 에이전트에게 "자기 자신의 탐색 과정을 성찰하는 능력"을 부여하면
탐색 효율성과 성능을 동시에 개선할 수 있다

TSDF 융합

Truncated Signed Distance Function을 활용하여 기하학적 일관성을 유지하는 볼류메트릭 매핑 수행

매 프레임마다 깊이 정보를 TSDF 볼륨에 융합하여 누적적 3D 재구성

프론티어 추출

프론티어 = { v ∈ V_free | ∃ v' ∈ N(v), v' ∈ V_unknown }

자유 공간과 미지 공간의 경계 → 탐색해야 할 후보 지점

D2 상세

이력 기반 휴리스틱 계획: 통합 효용 함수

U(fi) = α · sem(fi) − β · geo(fi) − γ · ep(fi)
🎯
sem(fi)
의미적 관련성 점수
VLM이 프론티어와 목표 지시문의 관련성을 0~1로 평가
📏
geo(fi)
기하학적 비용
정규화된 2D 거리 — 가까운 프론티어에 가중치 부여
🚫
ep(fi)
에피소딕 페널티
과거 방문 이력에 기반한 시간 감쇠 가우시안 반발 필드

에피소딕 페널티 수식

pep(fi) = Σ λ(t−τ) exp(−‖pfi − pτ‖² / 2σ²)
  • λ ∈ (0,1) — 시간 감쇠 인자: 최근 방문일수록 강한 페널티
  • σ — 공간적 반발 반경: 방문 지점 주변의 영향 범위
  • 막다른 골목과 국소 최솟값 재방문 방지

거시적 전략 개요 유지

정체 감지 메커니즘

Δt = 𝕀(Σ 𝕀(gτ < ε) ≥ Nstag ∧ (t − tlast) ≥ Tcool)
  • gt = |Vunkt-1| − |Vunkt| — 탐색 이득
  • 이득이 임계값 미만 Nstag회 연속 → 정체 판정
  • 쿨다운 Tcool로 과도한 반성 호출 방지

LLM 수정 출력

회피(Avoid) 비효율적 영역/전략 건너뛰기
시도(Try) 유망한 대안 방향 제시
근거(Evidence) 실패로부터의 논리적 추론
알고리즘

MetaNav 실행 사이클

Phase 1: 목표 인식 내비게이션 — 시맨틱 맵에서 목표 이미 탐지?
Phase 2: 정체 감지 / 반성 — 탐색 이득 모니터링 → 정체 시 LLM 반성 수행
Phase 3: 이력 기반 프론티어 선택 — U(fi) 계산 → 최적 프론티어 선택
Phase 4: 메모리 갱신 — 위치 로깅 + 에피소딕 버퍼 업데이트

핵심 설계 원칙

  • 비용이 큰 VLM 추론을 연속적 공간 실행에서 분리
  • Nreplan=3 간격으로 VLM 재호출 — 저수준 실행은 연속적
  • 반성은 정체 감지 시에만 선택적 호출
  • 에피소드당 평균 1.9회 반성 + 1.8회 요약만 수행
