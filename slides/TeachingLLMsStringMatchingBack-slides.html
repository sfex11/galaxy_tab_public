문제 정의

비트 조작 퍼즐이란?

숨겨진 논리 규칙을 역추론하여, 새로운 입력에 대한 출력을 예측하는 과제

사용 가능한 연산: 비트 시프트, 회전, XOR, AND, OR, NOT, 다수결/선택 함수

예제입력출력
예제 11010001111011001
예제 20110011010001101
예제 31111011011011011
타겟01001010????????

단 3~8개의 예제만으로 숨겨진 규칙을 완벽히 복원해야 함

핵심 난제

왜 기존 LLM은 실패하는가?

산술 추론의 한계

LLM에게 불리언 대수와 비트 연산을 "머릿속에서 시뮬레이션"하도록 강제하면 심각한 환각(Hallucination) 발생

조합 폭발

22개 기본 변환 중 3개를 선택하고 문법 트리를 구성하면 330,000+개의 조합 탐색 필요

자동 회귀(Auto-Regressive) 아키텍처는 수천 가지 이론적 경로를 동시에 시뮬레이션할 내부 작업 메모리가 부족하다.

핵심 아이디어

패러다임 전환: 산술에서 문자열 매칭으로

기존 접근

  • 논리 게이트 문법 트리 역추론
  • 복잡한 불리언 대수 시뮬레이션
  • 폭발적 탐색 공간

제안 접근

  • 이산적 특징 선택 문제로 재구성
  • 문자열 유사성(최소 비트플립) 활용
  • 백트래킹 DFS + 자율 오류 복구

핵심 통찰: 산술 논리를 완전히 포기하고 문자열 유사성, 구조적 탐색, 자율 오류 복구로 대체

핵심 개념 1

베이시스(Bases): 공간 포인터

8비트 문자열을 8개의 독립적인 1비트 변환으로 분해

x (원본)

대상 위치 i의 입력 비트를 직접 조회

R1~R7 (우시프트)

좌측 방향 공간 포인터, i-k 위치 조회

L1~L7 (좌시프트)

우측 방향 공간 포인터, i+k 위치 조회

C1~C7 (순환 회전)

경계를 넘으면 반대쪽으로 래핑하는 포인터

총 22개의 불리언 특징으로 동적 비트 이동을 정적 피처 공간으로 변환
f(x, R₁, ..., R₇, C₁, ..., C₇, L₁, ..., L₇) = y

핵심 개념 2

경험적 진리표 & 최소 비트플립

경험적 진리표

복잡한 산술 공식 대신, 각 베이시스 조합에 대해 예제에서 실제 관찰된 입력-출력 쌍을 기록

8비트 × 8예제 = 64행 불리언 논리 테이블

최소 비트플립

서로 다른 출력 간의 해밍 거리(비트플립 수)를 추적하여, 출력 변화를 실제로 유발하는 베이시스를 격리

산술 없이 문자열 유사성만으로 핵심 변수 식별

게이트 시뮬레이션 없이, "어떤 베이시스가 출력 변화를 일으키는가?"라는 단순한 질문으로 귀결

솔버 알고리즘

3단계 솔버 파이프라인

1
특징 추출
입력-출력 쌍에서 22개 베이시스 값 추출 및 문자열 매칭 수행
2
백트래킹 탐색
후보 베이시스 조합을 DFS로 탐색하고 충돌 검증 후 백트래킹
3
진리표 합성
최종 베이시스 조합의 경험적 진리표를 구성하여 타겟 예측

결정론적(Set Cover 기반) 솔버로 상한선(Baseline)을 확립

Phase 2 상세

백트래킹 DFS와 충돌 검증

1
가설 설정

후보 베이시스 부분집합을 선택하여 가설 형성

2
전역 충돌 검증

모든 예제에 대해 동일한 진리표가 성립하는지 검사

3
충돌 시 백트래킹

논리적 충돌(Logical Collision) 감지 시 이전 분기로 되돌아가 대안 탐색

4
수용 및 진행

충돌이 없으면 해당 분기를 확정하고 다음 베이시스 탐색
