⚠ 문제: CoT 추론의 비효율성

LLM의 Chain-of-Thought 추론 현황

LLM은 CoT(사고의 연쇄) 추론을 통해 높은 성능을 달성하지만, 과도한 토큰 소비로 인해 추론 비용이 급증합니다.

기존 방법들의 한계

✗ 명시적 길이 페널티

L1, ShorterBetter, ALP 등

정확도 최대화와 길이 최소화 사이에 적대적 그래디언트 생성 → 최적화 붕괴

✗ 적응형 추론

ARM, SelfBudgeter, Thinker 등

보조 난이도 추정 모델 필요, 인간이 설계한 분류 체계에 의존

✗ 다단계 커리큘럼

DeepScaleR, FastCuRL 등

3-8단계 파이프라인으로 단계별 하이퍼파라미터 튜닝 필요

핵심 통찰

LLM은 이미 효율적 추론 능력을 보유하고 있으나, 기존 단일 문제 훈련이 이를 활성화하지 못할 뿐이다.

BCR: 핵심 아이디어

Batched Contextual Reinforcement

하나의 컨텍스트 윈도우에서 N개 문제를 동시에 풀도록 훈련
보상은 문제별 정확도만 사용 — 길이 페널티 없음

1
문제 그룹 구성

데이터셋을 K개 그룹으로 분할, 각 그룹에 N개 문제 포함
Gk = {(qk,1, ak,1), ..., (qk,N, ak,N)}

↓
2
단일 프롬프트로 결합

시스템 지시 + N개 문제를 하나의 프롬프트로 연결, 공유 토큰 예산 Bmax 설정

↓
3
모델이 순차적으로 풀기

하나의 응답에서 N개 문제를 모두 해결 → 문제 간 토큰 경쟁 발생

↓
4
정확도 기반 보상

각 문제의 정답 여부만으로 보상 계산 — 길이 관련 보상 성분 없음

암묵적 길이 제어의 핵심: 앞 문제에 토큰을 낭비하면 뒤 문제의 예산이 줄어들어, 모델이 스스로 불필요한 추론을 제거하게 됩니다.

𝑓 수학적 프레임워크

GRPO 목적 함수

maxθ 𝔼G~𝒢 𝔼y~πθ [R(y,G) − β · DKL(πθ ∥ πref)]

그룹 G에 대해 S개 후보 완성을 샘플링하고 그룹 상대 어드밴티지 계산

정확도 보상

racc(y,G) = (1/N) ∑i=1N 𝕀[verify(âi, ai)]
  • 기호적 동치 검증 (SymPy)
  • 수치 비교 (ε = 10-6)
  • LaTeX 정규화 + 문자열 매칭

형식 보상

rfmt(y) = 𝕀[∀i: hasFormat(y, i)]
  • 각 답변이 지정 형식인지 확인
  • Answeri: \boxed{...}
  • 스택 기반 파서로 중첩 괄호 처리

토큰 예산 설정 알고리즘

Bmax ≈ N × Lavg × λ     (λ ≈ 0.5)

프로브 세트에서 평균 출력 길이(Lavg) 추정 후, 목표 압축 비율 λ를 적용
예) JustRL-1.5B, N=3: Bmax = 5,120 토큰 (문제당 ~1,707)

🎯

직접 전략 선택

기존 모델은 여러 풀이 접근법을 탐색 후 결정. BCR 모델은 최적 경로를 즉시 식별 — 다중 문제 훈련으로 문제 인식 능력 향상.

🛡

퇴화 방지

어려운 문제에서 기존 모델은 토큰 예산을 소진하고 반복적 문자열 생성. BCR은 퇴화 전에 답을 제출하도록 학습.

✂

교육적 서술 제거

기본 개념의 불필요한 설명 억제. 필수 추론 단계에 예산을 집중.

모든 압축은 구문적(syntactic)이며, 의미적(semantic) 추론 단계는 제거되지 않습니다.
