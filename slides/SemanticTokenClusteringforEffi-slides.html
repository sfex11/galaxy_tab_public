🔍 문제: LLM의 신뢰성 한계

과신(Overconfidence) 문제

  • LLM은 그럴듯하지만 틀린 답변을 자신있게 생성
  • 출력의 진실성이 보장되지 않음
  • 사용자가 잘못된 정보를 신뢰할 위험

불확실성 정량화의 필요성

  • 모델이 "잘 모르겠다"고 말할 수 있어야 함
  • 잠재적 오답을 사전에 식별
  • 신뢰할 수 있는 AI 시스템 구축의 핵심

기존 방법의 한계

▸ 다중 샘플링 기반: 5~20회 반복 생성 → 막대한 비용

▸ 외부 NLI 모델: 추가 모델 필요 → 복잡도 증가

▸ Semantic Entropy: 높은 성능이지만 600%+ 오버헤드

▸ CCP: 단일 생성이지만 여전히 62~220% 오버헤드

토큰 확률의 분산 문제

다음 토큰을 예측할 때, 동일한 의미를 가진 여러 토큰에 확률이 분산됩니다.
개별 토큰 확률만 보면 모델의 진짜 확신도를 과소평가하게 됩니다.

STC의 해결책

의미적으로 동등한 토큰들의 확률을 하나로 합산하여, 모델이 특정 의미 개념에 대해 실제로 얼마나 확신하는지 정확히 측정합니다.

단계 1: 오프라인 사전 계산

  • 입력·출력 임베딩을 결합하여 토큰 벡터 생성
  • 응집 클러스터링(코사인 거리) 수행
  • 전체 어휘를 16,000개 클러스터로 분류
  • 기능어(불용어)는 제외
  • 1회만 수행 → 추론 지연 없음

단계 2: 온라인 추론

  • 임베딩 클러스터링: 같은 클러스터 토큰 식별
  • 접두사 매칭: 서브워드 변형 포착
  • 두 집합의 합집합에서 확률 합산
  • 클러스터 확률로 불확실성 점수 계산

📐 수학적 정의

임베딩 클러스터링 집합

생성된 토큰과 같은 클러스터에 속하는 토큰들:

Tᵢᵉ = { t ∈ V | cluster(t) = cluster(yᵢ) }

접두사 매칭 집합

나머지 출력 문자열의 접두사가 되는 토큰들 (서브워드 처리):

Tᵢᵖ = { t ∈ V | norm(yᵢ₊) starts with norm(t) }

클러스터 확률 & 불확실성 점수

pᶜ(yᵢ | x, y<ᵢ) = Σ p(t | x, y<ᵢ)    [t ∈ Tᵢᵉ ∪ Tᵢᵖ]
S(x, y) = 1 − ∏ᵢ pᶜ(yᵢ | x, y<ᵢ)

클러스터 확률의 곱이 높을수록 → 불확실성이 낮음 (모델이 확신)

🔬 소거 연구 & 방법 비교

소거 연구 결과

  • 임베딩 클러스터링 제거 시: 0.3~5.3% AUROC 하락
  • 접두사 매칭 제거 시: 0.1~1.2% AUROC 하락
  • 둘 다 제거 (확률만): 1.5~5% AUROC 하락
  • 두 구성 요소는 상호 보완적

민감도 분석

  • 클러스터 수, 거리 척도, 알고리즘 변경에 안정적
  • "Single" 연결 방식만 성능 저하
  • 임베딩 유형에도 견고한 성능
