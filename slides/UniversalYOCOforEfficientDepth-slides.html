기존 트랜스포머의 한계

테스트 타임 스케일링은 LLM의 추론 능력을 크게 향상시켰지만, 표준 트랜스포머는 효율적 확장에 어려움을 겪습니다.

KV 캐시 폭증

루프 반복 시 KV 캐시가 모델 깊이에 비례하여 증가
O(L·T·N·D) — 시퀀스 길이가 길어질수록 메모리 폭발

연산 오버헤드

전통적 루핑은 모든 레이어를 재실행하여 글로벌 어텐션 비용 급증
O(L·T·N²·D) — 이차 복잡도 프리필링

⚠ 핵심 딜레마: 깊이를 늘려 성능을 높이면 → 추론 비용이 기하급수적으로 증가 → 실용적 배포가 불가능해지는 악순환

⬆
입력 토큰 시퀀스

유니버설 셀프 디코더 (USD)

  • 슬라이딩 윈도우 어텐션 (W=512)
  • 파라미터 공유로 T회 반복
  • 얕은 레이어에서만 재귀
  • RetNet, Mamba, DeltaNet 호환

크로스 디코더

  • 표준 Multi-Head Attention
  • USD의 KV 캐시를 재사용
  • NoPE 위치 임베딩
  • 루핑 없음 — 정적 실행

계산 복잡도 비교

N=시퀀스 길이, L=레이어 수, D=차원, T=반복 횟수, W=윈도우 크기

구성요소 Transformer Universal TRM YOCO-U
KV 캐시 메모리 O(LND) O(LTND) O((N+WTL)D)
프리필링 시간 O(LN²D) O(LTN²D) O(L/2·TND)
디코딩 시간 O(LND) O(LT(N+W)D) O(L/2·(N+WT)D)

프리필링: 이차 → 선형

Transformer: O(LN2D) — 이차
YOCO-U: O(L/2·TND) — 선형!

효율적 어텐션으로 N²을 N으로 감소

KV 캐시: 상수 유지

글로벌 캐시: O(ND) — T와 무관!
로컬 캐시: O(WTL) — W≪N이므로 무시 가능

핵심 발견

얕은 레이어에서 재귀가 깊은 레이어보다 효과적
크로스 디코더 루핑은 KV 전략과 무관하게 성능 저하
모델 레이아웃(깊이 vs 너비)에 거의 영향 없음
💬 "깊은 레이어에서의 재귀적 연산은
KV 캐시 전략과 관계없이
수확체감을 보인다"

표현 분석

유니버설 셀프 디코더 내 연속 레이어 간 각도 거리는 루프 간 일관된 패턴을 보이며, 셀프-크로스 디코더 인터페이스에서 급격한 스파이크가 관찰됨 → 두 모듈이 상이한 기능을 수행한다는 증거
