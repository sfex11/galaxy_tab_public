방법론: 세 가지 핵심 수정

데이터 합성 파이프라인에 대한 세 가지 단순하지만 매우 효과적인 수정

그래프 크기 확장 — 더 풍부한 탐색
확장 예산 k → K로 증가. 더 큰 하위 그래프에서 합성된 질문은 여러 노드의 증거를 통합해야 하므로 구조적으로 다홉 탐색이 필수적으로 요구됨
도구 세트 확장 — 더 넓은 기능성
사용 가능한 도구 𝒜를 확장하여 에이전트가 더 다양한 전략을 학습하고 더 광범위한 쿼리를 처리할 수 있도록 함
엄격한 저스텝 필터링 — 최소 난이도 보장
T(τ) < T_min인 궤적을 모두 제거. 직접 조회나 얕은 키워드 매칭으로 해결 가능한 단순 사례를 배제하여 지속적 추론 강제

그래프 확장의 수학적 공식화

소스 그래프 𝒢 = (𝒱, ℰ)에서 시드 노드 v_seed 주변의 국소 하위 그래프를 구성합니다.

𝒢_sub⁽ᴷ⁾ = Expand(𝒢, v_seed, K)  where K > k

q ~ P_gen(q | 𝒢_sub⁽ᴷ⁾)

확장된 하위 그래프 위에서 조건부로 질문을 생성하여, 단일 소스 의존이 아닌 다중 노드 증거 집계를 요구하는 질문을 합성합니다.

ReAct 궤적 구조

τ = (r₁, a₁, o₁, r₂, a₂, o₂, …, r_T, a_T, o_T, r_{T+1}, y)

r: 추론 단계 · a: 도구 호출 (a_t ∈ 𝒜) · o: 관측 결과 · y: 최종 답변

저스텝 필터링의 역할

𝒟_v2 = { (q, τ) ∈ 𝒟_raw | T(τ) ≥ T_min }

T_min 이하의 도구 호출 단계로 해결되는 궤적은 모두 폐기합니다.

제거 대상
  • 직접 조회로 해결되는 질문
  • 얕은 키워드 매칭으로 답변 가능
  • 단일 홉 탐색으로 충분한 사례
보장되는 특성
  • 최소 난이도 임계값 강제
  • 장기적 지속 추론 학습
  • 견고한 정보 추출 능력
그래프 확장은 문맥의 풍부함과 다홉 의존성을 증가시키고, 저스텝 필터링은 최소 난이도 바닥을 강제합니다. 두 수정이 결합하여 고품질 SFT 데이터를 생성합니다.

왜 SFT만으로 충분한가?

데이터 품질의 힘

  • 다홉 추론이 구조적으로 필수인 질문 설계
  • 단순 사례의 철저한 제거로 난이도 바닥 보장
  • 다양한 도구 사용 패턴 학습
  • 10.6k로 고도로 압축된 고품질 데이터셋

산업계 파이프라인의 필요성 재검토

  • CPT 단계가 반드시 필수적인 것은 아님
  • RL 단계 없이도 SOTA 달성 가능
  • 데이터 품질이 파이프라인 복잡도를 상쇄할 수 있음
  • 학계에서도 프론티어 에이전트 개발 가능
핵심 발견: 복잡한 다단계 학습 파이프라인이 아니라, 데이터의 난이도와 정보 밀도가 검색 에이전트 성능의 결정적 요인입니다.
