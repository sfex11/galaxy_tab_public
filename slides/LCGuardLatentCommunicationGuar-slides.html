LCGuard: Latent Communication Guard for Safe KV Sharing in Multi-Agent Systems

다중 에이전트 시스템에서 KV 캐시 기반 잠재 통신의 안전성을 보장하는 프레임워크

Sadia Asif · Rensselaer Polytechnic Institute Mohammad Mohammadi Amiri · Rensselaer Polytechnic Institute Momin Abbas, Prasanna Sattigeri, Karthikeyan Natesan Ramamurthy · IBM Research

arXiv:2605.22786v1 · May 2026

배경: LLM 기반 다중 에이전트 시스템

기존: 텍스트 기반 통신

자연어로 내부 상태를 직렬화하여 전달

  • 유연하고 해석 가능
  • 비효율적 — 반복적 디코딩/토큰화
  • 정보 손실 — 의미 구조 파괴

최근: 잠재 통신 (Latent Comm.)

Transformer KV 캐시를 직접 전달

  • 불필요한 계산 회피
  • 더 풍부한 의미 구조 보존
  • 에이전트 간 긴밀한 결합
KV 캐시를 공유 작업 메모리로 활용하면 효율성은 높아지지만, 민감 정보가 표면에 드러나지 않은 채 전파될 수 있다.

문제 제기: KV 캐시는 새로운 공격 표면

텍스트와 달리 KV 캐시는 고차원·밀집 표현으로, 내용을 검사하거나 제약하기 어렵다.

1
에이전트 입력 처리
민감 정보 sᵢ가 KV 캐시에 암묵적으로 임베딩
2
KV 캐시 전달
텍스트 출력없이 표현 공간 채널로 전파
3
적대적 재구성
공격자가 디코더로 민감 입력 복원
고차원

T×dₖ, T×dᵥ 크기의 밀집 텐서

불투명

텍스트와 달리 외부 검사 불가

추론 시

명시적 텍스트 노출 없이 유출 발생

위협 모델

적대적 접근 (Adversarial Access)

  • 손상된 에이전트가 공유 KV 캐시 접근
  • 로깅 인프라를 통한 캐시 가로채기
  • 보조 모델을 활용한 재구성 시도

적대적 지식 (Adversarial Knowledge)

  • 통신 아티팩트 mᵢⱼ 관찰 가능
  • 에이전트 모델 아키텍처 파악
  • 민감 입력 sᵢ는 모름

공격 목표 (Attack Objective)

공유된 캐시 아티팩트 mᵢⱼ로부터 에이전트별 민감 입력 sᵢ를 재구성하는 것

유출 범위

유형설명
Local단일 에이전트의 민감 입력 유출
System-level전체 시스템의 민감 정보 종합 유출

LCGuard: 개요

KV 캐시 전달 이전에 표현 수준 변환을 학습하여, 과제 관련 정보는 보존하되 재구성 가능한 민감 정보는 억제

핵심 아이디어

공유 캐시 아티팩트가 "안전하다" = 적대적 디코더가 민감 입력을 복원할 수 없다

학습 가능한 변환 함수

gᵢⱼ(Kᵢ, Vᵢ) → mᵢⱼ : 원시 KV 캐시를 안전한 통신 아티팩트로 변환

미니맥스 최적화

재구성 적대자는 민감 정보를 복원하려 하고, 가드는 이를 억제하면서 과제 성능 유지

토폴로지 무관

순차적, 계층적, 그래프 기반 통신 구조 모두에 적용 가능

적대적 학습 공식화

각 에이전트 aᵢ는 과제 입력 xᵢ와 민감 입력 sᵢ를 받아 KV 캐시 (Kᵢ, Vᵢ)를 생성한다.

통신 아티팩트: mᵢⱼ = gᵢⱼ(Kᵢ, Vᵢ)

목적 함수

과제 손실 ℒ_task(𝓜)

전달된 아티팩트 기반으로 시스템이 원래 과제를 수행하는 능력 측정

재구성 손실 ℒ_recon(mᵢⱼ, sᵢ)

적대적 디코더가 mᵢⱼ에서 sᵢ를 복원하는 정도 측정

LCGuard 전체 목적 함수

ming maxA [ ℒ_task(𝓜) + β · ℒ_recon(mᵢⱼ, sᵢ) ]

β: 프라이버시-유용성 트레이드오프 제어 파라미터

학습 절차

1
순전파
에이전트들이 KV 캐시 생성 후 gᵢⱼ로 변환하여 전달
2
적대자 업데이트
고정된 g 아래에서 재구성 손실 최소화 (민감 정보 복원 시도)
3
가드 업데이트
고정된 A 아래에서 과제 손실 + β×재구성 손실 최소화
4
반복
수렴할 때까지 2-3 단계 교대 반복

추론 시 효율성

학습된 gᵢⱼ는 추론 시 단일 순전파로 적용되므로 오버헤드가 매우 낮음. 적대자 모델은 학습에만 사용

β 파라미터 역할

β↑ → 프라이버시 강화 (재구성 어려움) / 과제 성능 저하 가능
β↓ → 과제 성능 강화 / 프라이버시 약화 가능

주요 결과: 프라이버시-유용성 트레이드오프

LCGuard는 경쟁적 과제 성능을 유지하면서 재구성 기반 유출을 일관되게 감소

↓ 유출률

Vanilla KV 대비 재구성 기반 Leak Rate 및 ASR 대폭 감소

민감 입력이 공유 표현에서 복원 불가능하도록 억제

≈ 성능

Task Accuracy에서 베이스라인과 경쟁력 있는 수준 유지

β 조정으로 유용성-프라이버시 균형 제어 가능

베이스라인 대비 요약

방법프라이버시 보호과제 성능적용 방식
Vanilla KV무보호기준선—
ADAPT중간저하노이즈 추가
PrivAct중간중간정렬 기반
LCGuard우수경쟁적표현 변환 학습

심층 분석 결과

시스템 수준 vs 로컬 보호

시스템 수준 LCGuard가 에이전트별 로컬 변형보다 우수한 프라이버시 보호 달성

여러 에이전트의 아티팩트를 종합하여 재구성하는 공격에 더 강력

통신 토폴로지 영향

순차적, 계층적, 그래프 토폴로지 모두에서 일관된 보호 효과

그래프 토폴로지(N=5)에서도 확장성 확인

모델 패밀리 효과

Qwen3, Gemma-2, LLaMA 전체에서 일관된 프라이버시-유용성 트레이드오프

모델 크기(3B~14B)에 따른 스케일링 거동 분석 완료

에이전트별 재구성 난이도

에이전트마다 민감 정보의 재구성 난이도가 상이

입력 특성과 처리 깊이에 따른 차이 분석

핵심 인사이트

1
표현 수준 유출은 텍스트 유출과 근본적으로 다르다

출력 필터링이나 도구 액션 제어로는 방어 불가 — 잠재 공간에서 직접 규제 필요

2
재구성 기반 정의가 실용적 프라이버시 지표다

추론적 유출의 "실제 복원 가능성"을 직접 측정하므로, 추론적 공격의 실질적 위험을 포착

3
학습 기반 변환이 노이즈 주입보다 우수하다

차분 프라이버시(ADAPT)는 일괄적 노이즈로 유용성 저하가 크지만, LCGuard는 과제 관련 정보를 선택적으로 보존

4
β는 실제 배포 시나리오에 맞게 조정 가능한 제어 knob이다

높은 프라이버시 요구 환경과 높은 성능 요구 환경 모두 대응 가능

5
추론 시 오버헤드는 무시할 수준이다

적대자 모델은 학습에만 필요하며, 추론 시 단일 순전파 변환만 수행

결론 및 향후 과제

결론

  • KV 캐시 잠재 통신을 새로운 공격 표면으로 식별
  • LCGuard 프레임워크로 표현 수준 정보 흐름 규제
  • 재구성 기반 유출 정의와 미니맥스 최적화 공식화
  • 다중 모델·벤치마크에서 일관된 프라이버시-유용성 트레이드오프 달성

향후 과제

  • 더 다양한 공격자 모델에 대한 강인성 평가
  • 실시간 스트리밍 환경으로의 확장
  • 공식적 프라이버시 보장(ε-차분 프라이버시 등)과의 연결
  • 다중 모달 에이전트 시스템으로의 일반화
LCGuard는 다중 에이전트 시스템에서 효율적인 잠재 통신과 에이전트별 프라이버시 보호을 동시에 달성하는 체계적인 접근법을 제시한다.
