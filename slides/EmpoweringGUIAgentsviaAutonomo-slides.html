자율 경험 탐색과 사후 경험 활용을 통한
GUI 에이전트 태스크 계획 강화

Empowering GUI Agents via Autonomous Experience Exploration
and Hindsight Experience Utilization for Task Planning

Tianyi Men, Zhuoran Jin, Pengfei Cao, Yubo Chen, Kang Liu, Jun Zhao

중국과학원 자동화연구소 · 중국과학원대학교 인공지능학부

문제 정의 및 동기

멀티모달 웹 에이전트의 필요성

반복적 GUI 작업을 대신 수행하며, 복잡한 태스크를 실행 가능한 액션으로 분해하는 태스크 계획(Task Planning)이 핵심 역량

  • 익숙하지 않은 웹사이트 조작 지원
  • 반복적인 GUI 작업 자동화
  • 사용자 시간 및 비용 절감

소형 오픈소스 MLLM의 한계

약한 계획 능력 — 복잡한 태스크를 올바르게 분해하지 못함

제한된 교차 웹사이트 일반화 — 학습하지 않은 웹사이트에서 성능 저하

상용 대형 모델에 비해 비용 효율적·프라이버시 보존되나, 성능 격차 존재

기존 방법론의 한계

접근 1 원자 단위 태스크 학습

환경 관찰 변화를 비교하여 클릭, 타이핑, 스크롤 등 원자적 액션 경험 추출

→ 원자 단위 학습이 고수준 태스크로 일반화되는지 불명확

접근 2 거친 고수준 태스크 학습

탐색 궤적을 바탕으로 "제약 조건이 있는 스노우보드 찾기" 등 거친 고수준 태스크로 학습

→ 궤적과 태스크 간 정렬 불량 및 엄격한 제약 부족

인간은 환경과의 상호작용·탐색을 통해 경험을 축적하고, 사후 경험(Hindsight Experience)을 요약하여 미래 의사결정에 활용한다. 에이전트도 이와 같은 학습 패러다임이 필요하다.

PEEU 방법론 개요

Planning Experience Exploration and Utilization — URL만 제공하면 자율 탐색·경험 추출·정렬된 학습 데이터 구축을 자동 수행

1
Planning Tree Exploration
웹사이트 기능에 적응한 목표 자율 설정 및 목표 주도 탐색으로 탐색 트리 구축
2
Planning Experience Utilization
궤적에서 경험 추출 → 사후 경험으로 태스크-궤적 쌍 역정렬 → 엄격히 정렬된 고수준 학습 데이터 합성
3
Cross-Site Generalization
합성된 데이터로 소형 MLLM 미세조정 → 학습하지 않은 실제 웹사이트에서 일반화

핵심 차별점: 사전 정의된 목표에 궤적을 무작위 매칭하는 대신, 사후 경험을 활용해 궤적에 태스크를 역으로 정렬하여 고수준 데이터 품질을 크게 향상

단계 1: Planning Tree Exploration

자율 목표 설정

웹사이트 URL과 홈페이지 상태 s₀를 MLLM에 입력하여, 해당 웹사이트 기능 특성에 적응된 태스크 리스트 자동 생성

𝒟 = M(s₀, URL)

𝒟 = {d₁, d₂, ..., dₙ}: 탐색할 태스크 집합

목표 주도 탐색

태스크 리스트 기반으로 환경을 자율 탐색하여 홈페이지를 루트로 하는 방향성 탐색 트리 구축

ℛ = (V, E) = Explore(M, 𝒟, Env, URL)

V: 웹사이트 스크린 집합, E: 액션 집합

  • 모든 궤적 τᵢ는 동일한 루트 노드(홈페이지)를 공유
  • 재귀적 탐색 과정으로 웹사이트의 다양한 기능 영역 커버
  • 사용자 개입 없이 URL만으로 전체 과정 자동화

단계 2: Planning Experience Utilization

기존 거친 고수준 태스크의 두 가지 한계

한계 1 정렬 불량

태스크는 "4.5성 이상"을 요구하지만, 궤적은 4성까지만 도달하는 등 목표-결과 불일치

한계 2 제약 부족

웹사이트는 부분 관측 가능 환경이므로, 홈페이지 정보만으로는 성분·조리 방법 등 엄격한 제약을 알 수 없음

PEEU의 해결책: 사후 경험 기반 역정렬

1

탐색 트리에서 궤적 τᵢ를 요약하여 가치 있는 경험 추출

2

추출된 경험을 바탕으로 궤적에 엄격히 정렬되고 제약이 포함된 태스크를 역으로 합성

3

정렬된 (태스크, 궤적) 쌍을 고수준 학습 데이터로 활용

TDHAF: 태스크 분해 계층 분석 프레임워크

Task Decomposition Hierarchical Analysis Framework — 구성적 일반화(Compositional Generalization)를 체계적으로 분석

Low

저수준 태스크

클릭, 타이핑, 스크롤 등 원자적 단일 액션

Mid

중수준 태스크

여러 액션의 조합으로 하위 기능 수행

High

고수준 태스크

다단계 계획이 필요한 복합 태스크

세 가지 분석 관점

ID 하향 일반화

저→고수준 도메인 내 일반화 가능 여부

ID 상향 일반화

고→저수준 도메인 내 일반화 가능 여부

OOD 다계층 일반화

어떤 입자도의 태스크가 도메인 외 일반화에 유리한가

TDHAF 분석 결과

발견 1

저수준 개별 기능을 마스터해도 해당 고수준 태스크를 반드시 마스터하지는 않음

원자적 스킬만으로는 복잡한 계획 능력이 보장되지 않음

발견 2

고수준 태스크 학습 시 도메인 내 하향 일반화가 더 용이하며 전체 커버리지가 높음

상위에서 하위로의 전이가 하위→상위보다 효과적

발견 3

고수준 태스크 학습이 OOD 다계층 일반화 능력을 더 강하게 부여

미지 웹사이트에서의 다단계 계획에 고수준 학습이 필수적

포스트트레이닝 단계에서 저수준 태스크만으로는 고수준 태스크로 효과적으로 일반화할 수 없다. 이는 PEEU가 고수준 경험에 집중해야 하는 근본적 이유를 제공한다.

실험 설정

평가 환경

  • 7개 학습하지 않은 실제 웹사이트에서 OOD 평가
  • 모든 방법에 대해 동일한 데이터 규모로 엄격 통제
  • 동일한 학습 설정으로 공정 비교

베이스라인

  • Instruct Model (기본 지시 모델)
  • 원자 단위 태스크 학습 방법
  • 거친 고수준 태스크 학습 방법
  • 검색 기반 경험 활용 방법

사용 모델

Qwen2.5-VL-7B — PEEU의 타겟 소형 모델

Qwen2.5-VL-32B — 비교 대상 대형 모델

핵심 평가 지표

태스크 완료 정확도(Accuracy) — 실제 웹사이트에서 태스크를 올바르게 수행한 비율

주요 실험 결과

방법 모델 크기 정확도 비고
Instruct Model 7B 7.8% 기준선
원자 단위 학습 7B 낮음 고수준 일반화 불량
거친 고수준 학습 7B 중간 정렬 불량으로 한계
검색 기반 경험 활용 7B 중간 소형 모델에 비효율
PEEU (제안) 7B 30.6% 최고 성능
Qwen2.5-VL 32B 30.6% 미만 4.5배 더 큰 모델

PEEU 기반 7B 모델이 32B 모델을 능가 — 사후 고수준 태스크 구성과 경험 활용이 소형 MLLM의 OOD 계획 능력에 결정적임을 입증

추가 분석 인사이트

경험-궤적 정렬의 효과

수동 설계된 프롬프트 파이프라인 없이도, 직접 학습(Direct Training)이 소형 모델에서 검색(Retrieval) 기반 방법보다 효과적

교차 웹사이트 일반화

고수준 태스크가 실제 웹사이트에서 저수준 태스크보다 우수한 교차 웹사이트 일반화를 달성

계층적 전이 방향

고→저수준 전이가 저→고수준 전이보다 더 넓은 커버리지를 제공하며, 하위 기능 습득률이 높음

데이터 효율성

동일 데이터 규모 조건에서 PEEU의 사후 정렬 방식이 데이터 품질 측면에서 압도적 우위

결론 및 기여 요약

PEEU 방법론

자율 탐색과 사후 경험 활용으로 웹 에이전트의 계획 일반화 능력 강화

TDHAF 프레임워크

멀티모달 웹 네비게이션에서 구성적 일반화를 체계적으로 분석하는 계층적 프레임워크

실증적 우수성

동일 데이터 규모에서 7B 모델이 32B 모델을 능가하는 교차 웹사이트 OOD 일반화 달성

소형 MLLM의 OOD 계획 능력을 위해 사후 고수준 태스크 구성과 경험 활용이 필수적이다. 원자적 스킬 학습만으로는 복잡한 계획에 도달할 수 없다.
