LLM 판별자 신뢰성 진단

01 — 문제 제기

LLM 판별자, 정말 믿을 수 있을까?

🔍

현재 상황

LLM-as-Judge 프레임워크가 자동 NLG 평가에 널리 사용되고 있으나, 개별 인스턴스 단위의 신뢰성은 충분히 이해되지 않고 있습니다.

⚠

핵심 문제

  • 집계 지표가 개별 문서의 실패를 은폐
  • 어떤 평가가 신뢰할 수 있는지 불분명
  • 기준(criterion)별 신뢰도 차이 미탐구
🎯

연구 목표

SummEval 데이터셋에 적용되는 이중 진단 도구를 제시하여, 집계 수준이 아닌 개별 입력 단위에서 LLM 판별자의 신뢰성을 정밀 진단합니다.

진단 1: 이행성 분석

쌍별 비교에서 방향성 순환(3-cycle)을 탐지하여 논리적 비일관성을 측정합니다.

예시: A > B, B > C 이면 A > C 여야 하나, C > A가 관찰되면 이행성 위반

진단 2: 적합 예측 집합

분할 적합 예측(Split Conformal Prediction)으로 이론적 보장이 있는 예측 집합을 생성합니다.

집합 크기가 클수록 해당 입력에 대한 판별자의 불확실성이 높음을 의미

04 — 수학적 프레임워크

적합 예측 집합 구성

비적합성 점수 (Nonconformity Score)

s = |ŷ − y*|

판별자 점수(ŷ)와 반올림된 사람 평균 점수(y*) 간의 절대 잔차

적합 임계값 (Conformal Threshold)

q̂ = s(⌈(1−α)(n+1)⌉)

보정 데이터의 경험적 분위수로, 커버리지 보장을 제공합니다.

예측 집합 (Prediction Set)

C(x) = { y ∈ {1, ..., 5} : |ŷ − y| ≤ q̂ }

커버리지 ≥ (1−α)를 보장하는 가능한 점수들의 집합. 집합 크기 = 신뢰도 지표

핵심: 20회 무작위 50/50 보정/테스트 분할로 평균과 95% 신뢰구간을 보고합니다.

06 — 결과 2

MFAS 순위 복구 실험

최소 피드백 호 집합 (Minimum Feedback Arc Set)

이행성 위반을 자동으로 수정하기 위해 MFAS 기반 순위 복구를 시도했습니다.

결과: 개선 효과 없음

  • Kendall τ 범위: 0.071 ~ 0.714 (불안정)
  • 단순 Win Rate 대비 일관된 개선 없음
  • 인간 판단과의 일치도 향상 실패
해석: 위반이 소수 문서에 희소하게 분포(ρ̄ < 5%)되어 있어, 토너먼트 그래프가 거의 비순환적이므로 복구할 여지가 거의 없습니다. 따라서 자동 수정보다는 문제 문서를 탐지하여 인간 검토에 위임하는 것을 권장합니다.

10 — 핵심 발견

기준이 판별자보다 중요하다

통합 발견: 기준(Criterion) > 판별자(Judge)

이행성 분석과 적합 예측 분석 모두에서 동일한 결론이 도출되었습니다: 어떤 기준을 평가하느냐가 어떤 모델을 사용하느냐보다 신뢰도에 더 결정적인 영향을 미칩니다.

11 — 실용적 시사점

배포 시 권고사항

1. 선택적 인간 검토 위임

예측 집합 크기 ≤ 2이면 자동 판정을 신뢰하고, = 5이면 인간 전문가 검토로 위임하세요.

2. 기준별 차별 전략

일관성/관련성 평가는 LLM 판별자에게 위임해도 안전하지만, 유창성/일치성은 인간 평가를 병행하는 것을 권장합니다.

3. 문제 문서 탐지

이행성 위반이 집중된 문서를 자동 탐지하여 인간 후속 검토(human follow-up)로 안내하세요. MFAS 자동 수정보다 효과적입니다.

4. 집계 지표에 의존하지 마세요

집계 위반율 0.8~4.1%가 낮아 보여도, 개별 문서의 33~67%에 문제가 숨어 있을 수 있습니다. 반드시 개별 수준 진단을 병행하세요.
