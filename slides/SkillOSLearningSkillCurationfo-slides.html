Siru Ouyang, Jun Yan, Yanfei Chen 외 · Google Cloud AI Research · UIUC · MIT

문제 정의: 일회성 문제 해결자의 한계

LLM 에이전트는 실제 환경에서 연속적인 작업을 처리해야 하지만, 대부분 과거 경험에서 학습하지 못하는 일회성 해결사에 머물러 있다.

수동 큐레이션
전문가 지식 의존, 확장 불가
히스틱 기반
고정 규칙, 성능 피드백 부재
단기 적응 RL
짧은 스트림 내 적응, 복잡 관리 불가
핵심 병목: 재사용 가능한 스킬의 고품질 큐레이션 — 기존 방법은 간접적이고 지연된 피드백에서 복잡한 장기 큐레이션 정책을 학습하지 못함

핵심 아이디어: 운영체제처럼 스킬을 관리

SkillOS는 에이전트의 절차적 기억을 파일 시스템 형태의 스킬 저장소(SkillRepo)로 구성하고, 전용 큐레이터가 파일 I/O 연산으로 스킬을 관리한다.

1
작업 수행
Executor가 SkillRepo에서 스킬 검색 후 실행
2
경험 축적
실행 궤적(trajectory)을 경험으로 수집
3
스킬 큐레이션
Curator가 삽입/갱신/삭제 연산으로 SkillRepo 업데이트
4
자기 진화
향후 관련 작업의 성능이 큐레이션 품질로 평가

스킬 형식: 단일 Markdown 파일 (Anthropic 설계 철학 따름)

시스템 아키텍처: 모듈형 다중 에이전트

Agent Executor Frozen
  • SkillRepo에서 관련 스킬 검색
  • 스킬을 참조하여 작업 실행
  • 실행 궤적 생성
  • 학습 중 파라미터 고정
Skill Curator Trainable
  • 축적된 경험 분석
  • 함수 호출로 SkillRepo 편집
  • 삽입 / 갱신 / 삭제 연산 수행
  • RL로 큐레이션 정책 학습
모듈형 설계로 Curator와 Executor를 독립적으로 교체 가능 → 높은 범용성

스킬 표현: Markdown 파일

각 스킬은 단일 Markdown 파일로 저장되어, 사람이 읽고 감사할 수 있으며 조립 가능한 형태를 갖춘다.

# 스킬: WebShop 가격 비교 전략
## 적용 조건
사용자가 특정 제품 구매를 요청할 때
## 실행 단계
1. 검색어로 제품 검색
2. 옵션 페이지에서 가격 비교
3. 최저가 옵션 선택 후 장바구니 추가
## 주의사항
- 무료 배송 옵션이 있는지 반드시 확인

파일 I/O 연산 (write, update, delete)으로 관리 → 운영체제(OS)의 파일 시스템과 유사한 패러다임

학습 방법: 그룹 기반 스트리밍 인스턴스 구성

각 훈련 인스턴스는 스킬 관련 의존성을 기반으로 그룹화된 작업 스트림으로 구성된다.

1
잠재 속성 주석 (Stage 1)
LLM이 각 작업의 스킬 관련 속성(도메인, 하위 작업 유형 등)을 주석
2
그룹 구성 (Stage 2)
구문 유사도 + 의존성 게이트 + 후보 검색 점수로 관련 작업들을 그룹화
3
스트리밍 시뮬레이션
초기 궤적 → SkillRepo 업데이트 → 이후 관련 작업이 업데이트 평가
테스트 시 스트리밍 환경을 모방하여, 스킬 큐레이션을 장기적 유용성에 정박시킨다.

학습 방법: 복합 보상 설계

지연되고 간접적인 환경 피드백을 큐레이션 결정에 귀인하기 위해 다중 신호 복합 보상을 설계한다.

작업 성과 보상
하위 작업의 성공률, WebShop 점수 등 실행자의 직접 성능 피드백

Curator의 함수 호출이 문법적으로 유효하고 실행 가능한지 여부
스킬 품질 보상

SkillRepo의 크기를 적정하게 유지하여 검색 효율성 확보

범용성 분석: Executor 교차 일반화

학습된 Skill Curator는 서로 다른 Executor 백본과 작업 도메인에 걸쳐 일반화한다.

Executor 교체 효과
  • Gemini-2.5-Pro를 Executor로 사용해도 성능 향상
  • Gemini-3.1-Flash-Lite에서도 일관된 개선
  • Curator 학습 시 본 적 없는 백본에도 적용 가능
8B vs Gemini-2.5-Pro
  • SkillOS의 8B Curator가 Gemini-2.5-Pro를 직접 Curator로 사용한 것보다 우수
  • 전문 프롬프트 기반 큐레이션보다 학습된 정책이 더 효과적
  • 작은 모델로도 강력한 큐레이션 능력 달성
모듈형 설계의 핵심 이점: Curator를 한 번 학습하면 다양한 Executor에 플러그앤플레이로 적용 가능

심층 분석: SkillRepo의 진화

학습된 Curator는 시간이 지남에 따라 더 풍부한 내부 구조와 고차원 메타 스킬을 발전시킨다.

정밀한 스킬 활용
불필요한 스킬 검색 감소, 관련 스킬만 선택
구조적 발전
단순 팁 → 계층적 섹션을 가진 구조화된 문서
메타 스킬 등장
여러 하위 작업을 아우르는 고수준 전략 스킬 자발적 형성

제거(Delete) 연산도 적극 활용 → 과도한 스킬 축적을 억제하고 저장소의 건전성 유지

제한사항 및 향후 방향

현재 제한사항
  • 단순한 검색 메커니즘 (의미적 매칭 한계)
  • 단일 Markdown 형태의 단순화된 스킬 표현
  • Executor 파라미터 고정으로 인한 상호 적응 불가
향후 연구 방향
  • 경험 메모리 위한 에이전틱 검색
  • 계층적 및 조합적 스킬 구조
  • 다중 에이전트 간 공유 메모리
SkillOS는 실용적이고 모듈형이며 경험 주도적인 RL 훈련 레시피로, 자기 진화 에이전트 구축의 실질적인 방향을 제시한다.
