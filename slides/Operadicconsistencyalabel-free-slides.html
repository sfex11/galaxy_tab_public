Operadic Consistency: LLM의
구성적 추론 실패를 탐지하는
라벨 없는 신호

Nathaniel Bottman · Yinhong Liu · Kyle Richardson

Incubilate · University of Cambridge · Allen Institute for AI

문제 정의

현대 LLM은 유창하고 자신감 있어 보이는 추론 사슬을 생성하지만, 실제로는 올바른 답으로 구성되지 않는 경우가 흔함.

추론 시점에서 정답 라벨 없이 이러한 실패를 탐지하는 것이 핵심 과제.

핵심 질문

모델의 추론이 내부적으로 구성적(compositional)인지 — 부분 결론들을 진지하게 받아들였을 때, 모델이 직접 제시한 최종 답과 동일하게 조합되는지 — 묻는 기존 방법은 없음.

기존 방법론과 한계

Self-Consistency

K개 온도 샘플의 다수결 투표 또는 평균 쌍별 일치도

프로브: 샘플 다양성

Semantic Entropy

의미적 클러스터 위의 샤논 엔트로피 계산

프로브: 샘플 다양성

P(True)

모델이 자신의 답이 올바를 확률이라고 평가하게 함

프로브: 자기 평가

어느 것도 모델의 추론이 내부적으로 구성적인지 — 부분 결론들이 실제로 최종 답과 일치하는지 — 검사하지 않음. 또한 외부 검증 기반 방법(Jacovi et al. 2024 등)은 작업 특화 도구나 부분 감독에 의존함.

핵심 아이디어

1
복합 질문
구성적 구조를 가진 질문 (예: multi-hop QA)
2
두 경로로 답변
직접 답변 vs. 분해 후 조합 답변
3
일치성 검사
두 답이 일치하는가? → Operadic Consistency

모델이 복합 질문에 직접 답한 결과와, 동일 질문을 명시적으로 분해하여 부분 답변을 조합한 결과가 일치해야 내부적으로 일관된 추론이라고 간주.

이는 Press et al. (2023)의 인구 수준 구성성 격차(compositionality gap)를 개별 질문 수준으로 가져온 것.

오퍼라드 이론과 질문 오퍼라드 𝒬

오퍼라드(Operad)란?

반복적 치환(iterated substitution)으로 구축된 시스템을 연구하는 대수적 구조. 다양한 입출력 수를 가진 연산들이 결합법칙을 만족하며 함께 구성됨.

질문 오퍼라드 𝒬

  • 𝒬(k): 빈칸 k개를 가진 질문 템플릿의 집합
  • ∘_i: 한 연산의 출력을 다른 연산의 i번째 입력에 치환
  • 합성 결과 = 하위 질문으로의 분해에 해당

언어 모델은 𝒱에 대한 대수(Algebra)

질문 템플릿에 답변을 채우는 함수로 언어 모델을 해석. 오퍼라드 구조가 일관성 조건(operadic consistency)을 자연스럽게 제안함.

본 논문은 깊이-2(depth-2) 트리 인스턴스만 사용하며, 일반 정의는 동반 논문(Bottman & Richardson, 2026)에서 전개.

Operadic Consistency (OC) 정의

직접 답변 경로

복합 질문 Q를 모델에 직접 입력 → 답변 A_direct

분해-조합 경로

Q를 하위 질문 {q₁, q₂, ...}로 분해 → 각각에 답변 → 조합하여 A_composed 획득

OC 판정

A_direct ≈ A_composed이면 OC-consistent (신호=1), 아니면 OC-inconsistent (신호=0)

분해는 외부에서 제공되거나 (비사고 모델) 모델의 CoT에서 추출됨 (사고 모델). 비용: 직접 답변 1회 + 하위 질문 답변 n회 = 총 K=3회 추론 호출 (depth-2, 2개 하위 질문 기준).

개별 질문 수준에서의 보완성

OC는 CoT-SC, Semantic Entropy, P(True)가 잡지 못하는 추가 정보를 제공하는가?

회귀 분석 결과

  • CoT-SC + Semantic Entropy 통제 후 OC 계수: p ≤ 10⁻¹⁶ (전 데이터셋)
  • 분해 인식 기준선(Decomposed SC, PRM 등) 추가 통제 후에도: p ≤ 10⁻¹³
  • Cluster-robust 표준오차 사용으로 모델 간 의존성 통제

해석

OC는 기존 신호들이 이미 포착한 변동성과 통계적으로 독립적인 정보를 제공. "샘플 다양성"과 "자기 평가"를 넘어서는 제3의 차원 — 구성적 일관성 — 을 프로브.

예시: OC가 포착하는 실패 패턴

OC-consistent & 정답

MuSiQue 예시: 직접 답변과 분해-조합 답변이 모두 올바르고 일치함. 내부 구성이 정상 작동.

OC-inconsistent & 오답

MuSiQue 예시: 직접 답변은 하나의 하위 질문 답변만 반영하고, 다른 하위 질문의 정보를 누락. 두 경로가 불일치 → OC가 실패 탐지.

OC-consistent & 오답 (위양성)

두 경로가 같은 잘못된 답에 도달. 구성은 일관되나 근본적으로 잘못된 추론. OC 한계 사례.

OC-inconsistent & 정답 (위음성)

직접 답변이 맞으나 분해 과정에서 오류 발생으로 불일치. OC는 보수적 신호.

기여 요약 및 향후 방향

핵심 기여

  • 프레임워크: 질문 오퍼라드 𝒬 정의 및 언어 모델을 𝒬-대수로 해석. 오퍼라드 일관성(OC) 신호 도출
  • 실증 신호: 12개 모델 × 4개 데이터셋에서 r ∈ [0.86, 0.94]. 유일하게 전 데이터셋 r ≥ 0.85 달성
  • 배포 효과: 동일 비용(K=3)에서 CoT-SC 대비 AUARC +0.086~+0.096, AUROC +0.092~+0.164 향상
  • 사고 모델 확장: CoT에서 추출한 분해로도 16/16 셀 양성 향상

향후 방향

더 깊은/비선형 질문 트리로의 일반화

CoT에 내재된 구조의 직접 활용
