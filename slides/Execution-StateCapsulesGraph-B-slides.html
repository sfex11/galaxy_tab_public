주류 서빙 시스템
  • vLLM PagedAttention — KV 캐시 페이징
  • SGLang RadixAttention — 접두어 radix 트리 매칭
  • 관리 객체 = 위치 기반 KV 캐시 (단편)
  • 목표 = 높은 동시성, 높은 처리량
근본적 문제
  • 블록 테이블 간접 참조로 그래프가 자가완비 불가
  • 재귀 상태·합성곱 상태는 관리 대상 외
  • 단일 스트림·온디바이스에서는 가정이 성립하지 않음
  • 세션 복원·분기·인터럽트에 대한 제어 불가
위치적 KV 캐시는 실행 상태의 일부일 뿐, 그 상태를 제어하기 위한 전체 제어면이 아니다.

결과 1: 실행 기반(low floor)

결과 2: 상태 동결 가능

이를 복사하는 것만으로 체크포인트/복원이 완료된다.
두 기능은 독립적이지 않다. 동일한 설계 선택이 런타임을 빠르게 만들고 상태를 동결 가능하게 만든다.

Execution-State Capsule: 정의

캡슐이란, 커밋된 토큰 경계에서 그래프 바인딩 실행 상태의 완전한 스냅샷이다.
캡처된 계산 그래프가 아닌, 그래프가 의존하는 폐쇄된 명명 디바이스 버퍼 집합이다.
캡슐의 네 가지 동사
State (저장)
현재 버퍼 집합을 캡슐로 동결
Restore (복원)
저장된 버퍼를 런타임에 복사하여 재개
Fork (분기)
캡슐을 복사하여 독립 세션 생성
Rollback (롤백)
이전 경계 상태로 되돌리기
비용 모델: 재계산(compute-bound)이 아닌 대역폭 바운드 복사-재구축(bandwidth-bound copy-and-rebuild).
GPU 내 스냅샷/복원은 서브밀리초 수준.

재사용 단위의 전환

기존: 토큰 주소 지정 KV 단편
  • 블록 테이블이 토큰 위치 → KV 페이지 매핑
  • Radix 트리가 토큰 접두어 → 캐시 서브트리 매칭
  • 관리 객체 = KV 캐시 (위치적 단편)
  • 재귀·합성곱 상태는 관리 외
캡슐: 그래프 바인딩 실행 상태 경계
  • 토큰 경계는 메타데이터 (위치 + 접두어 다이제스트) 로만 사용
  • 재사용 객체 = 해당 경계의 전체 실행 상태
  • KV + 재귀 폴드 + 합성곱 상태 포함
  • 제3의 관리 객체 도입
캡슐은 더 나은 KV 캐시가 아니다. 지연 우선 런타임 기판과 실행 상태라는 제3의 관리 객체가 함께 정의하는, 처리량 우선 스택이 의도적으로 최적화하지 않는 서빙 설계점이다.

실행 계약: 하나의 계약, 세 가지 시나리오

캡슐은 최소 실행 계약 위에 구축된다: 3개 핸들 타입, 1개 불투명 셰이프 키, 엄격한 메커니즘-정책 경계.

LLM 코딩 에이전트
대규모 시스템 프롬프트 + 도구 스키마 + 저장소 인덱스의 웜 스타트.
캡슐 복원으로 매 턴의 콜드 프리필 회피.
Snapshot/Restore
로봇 RL 에피소드 리셋
VLA 확산 정책의 에피소드 초기 상태 스냅샷/복원.
계산 상태 복구, 물리적 역전이 아님.
Snapshot/Restore
계획자–실행자 인계
계층적 플래너에서 액터로의 제로카피 버퍼 전달.
복사 없이 실행 상태 소유권 이전.
Zero-Copy Pass
실행 메커니즘 수준에서, 코딩 에이전트의 웜 스타트와 로봇 RL 롤아웃의 에피소드 리셋은 동일한 스냅샷/복원 동사이다.

정확성: 바이트 상태 복원과 청크 정렬

세 가지 정확성 보장
  • 바이트 동일 저장 상태 — 스냅샷과 복원 후 버퍼가 비트 단위로 일치
  • 토큰 동일 탐욕 디코드 — LLM에서 캡슐 복원 후 동일 토큰 시퀀스 생성
  • 바이트 동일 액션 리플레이 — VLA 정책에서 코사인 유사도 1.0
청크 정렬 조건
청크형 선형 어텐션 스캔의 정확한 재사용은 청크 정렬 조건을 만족해야 한다.

이 조건은 캡슐이 토큰 경계를 메타데이터로 유지하는 이유이며, 단순한 KV 복사로는 보장되지 않는다.
결정적 증거: KV-전용 제거(ablation)
하이브리드 모델에서 위치적 KV만 복원하고 재귀 폴드(recurrent fold)를 누락하면 첫 토큰부터 발산한다.
전체 캡슐 복원은 토큰-정확하다. → 이득은 런타임만이 아닌 상태 메커니즘 자체에서 온다.

평가: 런타임 기반층 (Layer 1)

범위: TTFT/TTFA, 디코드 처리량이 아님. 동일 하이브리드 모델, 동일 GPU, 단일 스트림.

요약 및 핵심 메시지

캡슐은 재사용 단위를 토큰 주소 지정 KV 단편에서 그래프 바인딩 실행 상태 경계로 이동한다.
기여
  • 지연 우선 화이트박스 런타인 FlashRT
  • 그래프 바인딩 실행 상태 캡슐 메커니즘
  • 최소 실행 계약으로 3개 도메인 통합
  • 구조적 정확성 증명 (KV-전용 ablation)
  • 온디바이스 (Thor, Spark) 복제 검증
범위 (명시적 한계)
  • 고동시성 처리량 최적화 불주장
  • 분산/클러스터 서빙 불주장
  • 동적 임의 셰이프 배칭 불주장
  • 크로스 노드 KV 재사용 불주장
처리량 우선 스택(vLLM, SGLang)의 보완 설계점이지, 대체재가 아니다.
단일 스트림·온디바이스·Physical-AI라는 반대 레짐을 위해 존재한다.
