법률 자문

판례 검색과 법리 해석, 변호사 없이도 법적 조언을 제공받는 상황

보안 분석

취약점 탐지와 위협 분석, 잘못된 판단이 치명적인 결과로 이어짐

LLM과 LRM은 중요한 임무에 점점 더 많이 사용되지만, 해답의 정확성에 대한 어떠한 보장도 제공하지 않는다.

핵심 문제: 허위 신뢰(False Trust)

사용자는 모델의 답변을 신뢰할지 결정해야 하며, 이때 추론 과정(trace), 요약, 또는 사후 생성 설명(post-hoc explanation)에 의존한다.

허위 신뢰가 발생하는 메커니즘

1
LLM 답변 생성
정답 또는 오답 출력
2
설명 첨부
추론 trace / 요약 / post-hoc
3
사용자 판단
설명의 "그럴듯함"에 현혹
4
허위 신뢰 형성
오답인데도 신뢰하게 됨

LLM 설명의 세 가지 유형

01

추론 과정 (Reasoning Traces)

Chain-of-Thought 등 모델이 답에 도달하는 과정을 단계별로 기록한 출력

CoT 단계별 추론
02

요약 (Summaries)

긴 추론 과정을 압축하여 핵심 근거만 제시하는 형태의 설명

압축 핵심 근거
03

사후 설명 (Post-hoc)

답변 생성 후 별도로 근거를 구성하는 설명으로, 실제 추론과 무관할 수 있음

위험 허구 근거

사후 생성 설명은 모델의 실제 추론 과정을 반영하지 않을 가능성이 높으며, 그럴듯하게 들리지만 근거 없는 논리를 구성할 수 있다.

연구 질문

RQ1

LLM 설명이 사용자의 신뢰 보정(trust calibration)에 미치는 영향은 무엇인가?

사용자가 올바른 답변과 잘못된 답변을 얼마나 구별하는지 측정

RQ2

설명 유형(trace, 요약, post-hoc)별로 허위 신뢰의 정도가 다른가?

각 설명 형태가 신뢰에 미치는 차별적 효과 분석

RQ3

LLM과 LRM(추론 모델) 간에 허위 신뢰 패턴이 어떻게 다른가?

o1, o3 등 추론 특화 모델에서의 신뢰 보정 효과 비교

RQ4

설명의 품질 지표가 실제 신뢰 보정과 얼마나 상관하는가?

설명의 "그럴듯함(plausibility)"이 신뢰의 대리 지표로 유효한지 검증

연구 방법론 개요

실험 설계

  • 다양한 도메인의 문제 집합 구성
  • 정답/오답 쌍을 포함한 실험 자료 준비
  • 설명 유형별 조건 통제
  • 사용자 신뢰도 측정 프로토콜 설계

평가 지표

  • 신뢰 보정 점수 (Trust Calibration Score)
  • 오답 수용률 (False Acceptance Rate)
  • 정답 거부률 (False Rejection Rate)
  • 설명 품질-신뢰 상관계수

LLM

GPT-4, Claude 등
일반 대규모 언어 모델

LRM

o1, o3 등
추론 특화 모델

Baseline

설명 없는 답변만
제공하는 조건

신뢰 보정의 핵심 개념

신뢰 보정이란 사용자의 주관적 신뢰도가 모델 답변의 객관적 정확도와 얼마나 일치하는지를 의미한다.

상황 모델 답변 사용자 반응 보정 상태
올바른 보정 정답 신뢰 적절
올바른 보정 오답 불신 적절
허위 신뢰 오답 신뢰 과신뢰
과소 신뢰 정답 불신 과소신뢰

허위 신뢰가 가장 위험한 이유: 사용자가 오답을 정답으로 받아들이고, 이를 바탕으로 실제 의사결정을 수행한다.

주요 발견 ① 설명이 신뢰 보정을 저해한다

오답 수용률 증가

설명이 제공된 오답을 사용자가 설명 없는 오답보다 훨씬 더 높은 비율로 수용

신뢰 보정 점수 하락

설명 제공 조건에서 전반적인 신뢰 보정 점수가 베이스라인 대비 유의미하게 감소

핵심 인사이트

설명이 "투명성"을 높인다고 믿었지만, 실제로는 설명의 그럴듯함(plausibility)이 정확도와 무관하게 작동하여 신뢰 보정을 오히려 악화시킨다. 이는 설명 가능성(XAI)의 근본적 역설을 보여준다.

주요 발견 ② 설명 유형별 허위 신뢰 차이

설명 유형 오답 수용률 신뢰 보정 허위 신뢰 위험
설명 없음 (Baseline) 기준치 가장 높음 낮음
추론 과정 (Traces) 증가 감소 중간
요약 (Summaries) 더 증가 더 감소 높음
사후 설명 (Post-hoc) 최대 증가 최대 감소 최고 위험

왜 사후 설명이 가장 위험한가?

  • 답변에 맞춰 논리를 역구성하므로 "가장 그럴듯하게" 들림
  • 실제 추론 과정과의 단절이 사용자에게 감지되지 않음
  • 정답·오답 여부와 무관하게 일관된 설명 품질을 보여줌

주요 발견 ③ LLM vs LRM 비교

LLM (GPT-4, Claude)

  • 오답 시 설명의 논리적 결함이 상대적으로 더 드러남
  • 추론 과정이 단순하여 사용자가 오류를 감지할 확률이 더 높음
  • 그럼에도 설명 유무에 따른 허위 신뢰 증가는 확실히 관측됨

LRM (o1, o3)

  • 길고 정교한 추론 과정이 사용자에게 "전문성"의 인상을 줌
  • 오답이라도 추론의 복잡성 덕분에 더 높은 신뢰를 획득
  • LRM의 설명이 허위 신뢰를 더 강하게 유발하는 역설적 결과

모델의 추론 능력이 향상될수록 설명의 그럴듯함도 함께 증가하여, 오답 상황에서의 허위 신뢰가 오히려 심화되는 안전성 역설(safety paradox)이 존재한다.

주요 발견 ④ 설명 품질 지표의 한계

기존 설명 품질 평가 지표(완결성, 일관성, 간결성 등)가 실제 신뢰 보정과 낮은 상관관계를 보인다.

기존 지표가 포착 못하는 것

  • 설명이 "틀린 답을 옹호"하는 방식의 미묘한 차이
  • 논리적 오류가 존재하지만 그럴듯하게 포장된 구조
  • 사용자의 인지 편향을 자극하는 수사적 기법

필요한 새로운 평가 방향

  • 정답·오답 구별 능력 기반의 보정 지표
  • 사용자 수준에서의 실증적 검증
  • 설명의 "신뢰 유발 효과"를 직접 측정하는 메트릭

즉, "좋은 설명"을 평가하는 기존 기준이 "신뢰 보전에 도움이 되는 설명"을 평가하는 기준과 다르다.

시사점 및 논의

사용자 측면

  • 설명의 "그럴듯함"을 신뢰의 근거로 삼지 않도록 교육 필요
  • 길고 복잡한 추론일수록 더 비판적으로 검토해야 한다는 인식 부여
  • 다중 출처 교차 검증 습관 형성의 중요성

시스템 설계 측면

  • 불확실성을 명시적으로 표시하는 인터페이스 설계
  • 오답 가능성이 높은 영역에서 설명 제공 방식 재설계
  • 사후 설명보다 실시간 추론 노출을 우선하는 아키텍처

정책 및 규제 측면

  • LLM 설명에 대한 "신뢰 보정 테스트"를 안전성 평가의 필수 항목으로 채택
  • EU AI Act 등 기존 규제 프레임워크에 설명 유해성 지표 반영 필요
  • 설명 제공이 의무화되는 상황에서의 역효과에 대한 정책적 고려

한계점 및 향후 연구

연구 한계

  • 특정 도메인과 문제 유형에 집중된 실험 환경
  • 참여자 풀의 다양성 제한 (주로 영미권 기술 중심)
  • 실제 임상/법률 환경이 아닌 실험실 설정
  • 장기적 노출 효과(Long-term exposure) 미측정

향후 연구 방향

  • 도메인 특화 허위 신뢰 측정 벤치마크 구축
  • 전문가 vs 비전문가 간 신뢰 보정 차이 연구
  • 허위 신뢰를 완화하는 설명 프레이밍 기법 개발
  • 멀티모달 환경(이미지, 코드 등)으로의 확장

결론

LLM의 설명은 투명성을 제공한다는 기대와 달리, 사용자의 신뢰 보정을 체계적으로 저해한다. 특히 사후 생성 설명과 LRM의 복잡한 추론 과정은 오답 상황에서 더 강력한 허위 신뢰를 유발한다.

설명 유무가 신뢰 보정에 미치는 영향을 최초로 체계적으로 정량화

설명 유형별 위험도 순서를 명확히 규명 (Post-hoc > 요약 > Trace)

LRM의 안전성 역설을 식별: 더 나은 모델이 더 큰 허위 신뢰를 유발

기존 설명 품질 지표의 신뢰 보정 예측 한계를 실증적으로 입증
