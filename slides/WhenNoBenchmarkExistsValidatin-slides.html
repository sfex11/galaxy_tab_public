When No Benchmark Exists: Validating Comparative LLM Safety Scoring
Without Ground-Truth Labels

Sushant Gautam, Finn Schwall, Annika Willoch Olstad, Fernando Vallecillos Ruiz,
Birk Torpmann-Hagen, Sunniva Maria Stordal Bjørklund, Leon Moonen, Klas Pettersen, Michael A. Riegler

Simula Metropolitan · Oslo Metropolitan University · University of Oslo · Simula Research Laboratory · Norwegian Directorate of Health

문제 상황

"노르웨이 공공기관이 국민 서비스 안내용으로 로컬 배포 LLM 파일럿을 준비한다. 후보 모델 중 어느 것이 노르웨이어·정책·데이터 처리 관점에서 더 안전한가?"

  • 해당 언어·도메인에 대한 안전성 벤치마크가 존재하지 않음
  • 벤치마크 구축은 예산·일정·데이터 제약으로 불가능
  • 모델·프롬프트·가드레일이 업데이트될 때마다 재평가 필요
  • 상용 API 경로 차단 (규제 환경)

→ 기존 패러다임(정적 벤치마크, 자동 레드팀, LLM-as-judge)만으로는 해결 불가

정적 벤치마크

Ground-truth 레이블 필요

  • 구축 비용 매우 높음
  • 영어 중심
  • 출시 시점에 고정

자동 레드팀

발견 중심, 점수 아님

  • 전문가 검토용 산출물
  • 반복 가능한 점수 부재
  • 불확실성 정량화 안 됨

LLM-as-judge

판별 자체의 신뢰성 문제

  • 절대 점수 불안정
  • 위치·길이·자기향상 편향
  • 도구 수준 검증 필요

노르웨이의 경우: NorEval(24개 데이터셋)에 안전성 구성요소 없음 · 다국어 안전 벤치마크에서 노르웨이어 제외

벤치마크리스 비교 안전성 점수화

정답 레이블이 없는 상황에서 후보 모델 간 안전성을 비교하는 새로운 평가 범주를 공식화

1
측정 기기 고정

시나리오 팩, 루브릭, 감사자, 판사, 샘플링, 재실행 횟수

2
후보 타겟 모델에 대해 재실행

동일한 기기로 각 모델 평가

3
명시적 클레임 계약하에 보고

점수, 델타, 임계율, 불확실성, 사용된 판사·감사자

"다른 기기의 점수는 직접 비교할 수 없다. 점수는 고정된 구성 하에서만 유효하다."

세 가지 독립 역할

T

타겟 모델

평가 대상 LLM. 안전성이 비교되는 후보 모델.

A

감사자 (Auditor)

시나리오를 프롬프트로 변환하여 타겟에 입력. 멀티턴 대화 생성.

J

판사 (Judge)

전체 대화 기록을 읽고 루브릭에 따라 심각도 레이블 부여.

독립성이 핵심: 세 역할을 분리하면 타겟, 감사자, 판사를 실험 요인으로 변동시켜 점수가 타겟 행동을 반영하는지, 장치 인공물인지 정량화 가능

도구적 타당성 연쇄

Ground-truth 대신 세 가지 검증을 연쇄적으로 통과해야 점수 유효

1

2
타겟 지배성

3
재현성

안전 vs 소거(Ablated) 대조

거부 방향(ablation)을 제거한 능력 일치 변형. 안전 모델과 소거 모델이 분리되어야 함.

분산 분해

총 분산을 타겟·감사자·판사·잔차 성분으로 분해. 타겟이 지배적이어야 함.

검증 결과: 타겟 지배성과 재현성

타겟 지배성

η² ≈ 0.52

총 분산 중 약 52%가 타겟 모델 간 차이에서 설명됨

  • 감사자·판사 인공물보다 타겟이 압도적으로 지배적
  • 대조(contrast) 제거 후에도 지배성 유지
  • 장치 성분은 무시할 수 없지만 2차적

재현성

10회

재실행 10회부터 심각도 프로필 안정화

  • 부트스트랩 분석으로 안정화 지점 확인
  • 안정화 이후 추가 재실행은 점수 변동 최소
  • 불확실성 범위 축소 가능

판사·감사자 선택과 기본 구성

판사 선택의 핵심 지표

임계 미스(Critical-miss) 일치도

가장 심각한 위험(임계 등급)을 놓치는 판사는 배제. 전반적 일치도보다 임계 미스 동의가 운영상 중요.

감사자 선택의 핵심 설계점

감사자는 타겟보다 능력이 충분히 높아야 함. 감사자-타겟 간 능력 격차가 부족하면 타겟의 안전 거부를 회피하지 못해 점수 왜곡.

기본 구성: J = A

판사와 감사자에 동일 모델 사용을 기본값으로 권장

  • 구성 복잡도 감소
  • 분산 성분 단순화
  • J ≠ A일 때도 연쇄 통과 가능하나 추가 검증 필요
  • AUROC는 J=A 구성에서 가장 높음

Petri에 대한 연쇄 검증

동일한 타당성 연쇄를 Petri(Anthropic의 에이전트 감사 도구)에 적용

연쇄 통과

  • 안정성: 재실행 간 안정화
  • 분산 분해: 타겟 지배적
  • 안전 vs 소거 분리

→ 연쇄는 특정 도구가 아닌 도구 클래스를 식별

본질적 차이는 상류에

  • 클레임 계약 강제 정도
  • 배포 적합성 (로컬 vs API)
  • 발견 vs 점수화 설계 목표
  • 다차원 루브릭의 구성 분할 문제

노르웨이 공공조달 사례 연구

Borealis vs Gemma 3 비교 — 노르웨이어 안전 시나리오 팩 적용

핵심 발견

  • 더 안전한 모델은 시나리오 카테고리에 따라 다름
  • 위험 측정 지표(점수 vs 임계율)에 따라도 순위 변동
  • 단일 순위로 축약 불가

보고해야 할 항목

  • 점수 (scores)
  • 매치 델타 (matched deltas)
  • 임계율 (critical rates)
  • 불확실성 (uncertainty)
  • 사용된 판사·감사자 식별

"점수, 델타, 임계율, 불확실성, 사용된 판사와 감사자를 함께 보고해야 하며, 순위로 축약해서는 안 된다."

논의 및 한계

논의

  • 도구적 타당성 ≠ 구성 타당성
  • 점수가 "실제 세계 안전성"을 증명하는 것은 아님
  • 점수가 노이즈나 장치 인공물이 아닌 타겟 행동에 반응한다는 선행 주장만 확립
  • 배포 타당성은 배포 팀의 기여로 남음

한계

  • 소거 대조는 거부 방향이라는 가정 의존
  • 노르웨이어 팩에만 검증 (다른 언어 미검증)
  • 시나리오 팩 자체의 타당성은 별도 문제
  • 장치 분산 성분이 무시 불가능
  • 멀티턴 상호작용의 복잡성 일부 미포착 가능
