  • 사용자 히스토리에서 개인화 루브릭 항목을 자동 추론

  • 다양한 모델에 범용적으로 적용 가능

  • 기존 baseline보다도 낮은 정확도
  • 정보 과부하로 인한 추론 방해
핵심 발견

주요 시사점

1. 모델 규모 ≠ 개인화 성능

파라미터 수 증가가 개인화 성능 향상으로 직결되지 않음. 규모보다 학습 방식이 중요

2. 해결 가능한 문제

오라클 성능(~98%)이 보여주듯, 올바른 루브릭만 주어지면 보상 모델은 개인화 판단이 가능. 핵심은 루브릭 추론 능력

3. 벤치마크 신뢰성 검증

NDCG 0.93으로 다운스트림 성능과 높은 상관관계 — 기존 벤치마크(0.66)를 크게 상회

4. 학습-평가 불일치 주의

단순 프로필 주입은 역효과. 구조화된 플래너를 통한 선호도 추론이 필수
