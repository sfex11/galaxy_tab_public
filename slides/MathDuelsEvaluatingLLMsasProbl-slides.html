출제자 역할

적대적 프롬프팅을 통해 수학 문제를 생성하고, 난이도 증폭 단계를 거쳐 다른 모델이 풀기 어려운 문제를 만든다.

해결자 역할

자신이 출제하지 않은 모든 문제에 도전하며, 기호적 등가성 검사로 정답 여부가 자동 판정된다.

파이프라인

3단계 문제 생성 파이프라인

최소한의 인간 개입 — 난이도와 경쟁적 맥락만 지정, 내용과 전략은 모델 자율에 맡긴다.

1
메타 프롬프팅

주어진 수학 도메인 δk에 대해 문제 출제용 프롬프트를 먼저 작성하게 한다. 모델이 "어떤 문제가 어려운지"를 먼저 추론하도록 유도하여, 다양한 난이도 전략을 자연스럽게 확보한다.

2
문제 생성

1단계에서 생성한 프롬프트를 동일 모델에 입력하여, 문제陈述 qk과 단일 스칼라 수식 형태의 금정답 gk을 출력한다.

3
난이도 증폭

출제 모델이 자신의 문제를 한 번 이상 변형하여 더 깊은 추론을 요구하도록 강화한다. 경쟁적 맥락("상대가 풀지 못하게 만들어라")이 추가 난이도를 견인한다.

자동 검증

문제 유효성 검증 시스템

"의견이 다를 경우 공개 토론이 열려야 한다" — Toscano, 2020

정상 문제

문제陈述에 유일하고 잘 정의된 해가 존재함을 확인. 검증자가 후보 답안 중 올바른 것을 선택하면 v = 1, 해당 답안이 새로운 금정답으로 대체된다.

결함 문제

모호하거나 부정확하게 정식화된 문제는 v = 0으로 판정되어 점수 계산에서 완전히 제외된다. 이 과정으로 ill-posed 문제가 평가를 오염시키는 것을 방지한다.

판정 트리거 조건
적어도 한 명의 solver가 오답을 낸 경우에만 검증이 실행됨
∃i : yi = 0 ⟹ Verify(qp, gp, Ap)
평가 모델

Rasch 모델 기반 복합 평점

해결 능력과 문제 난이도를 동시에 추정하여, 단일 척도에서 모델과 문제를 배치한다.

Rasch 모델 최적화 목표
max Σ(m,p) [ ymp log σ(sm − dp) + (1−ymp) log σ(dp − sm) ] − λ Σp dp2
Solver Rating
Rasch 추정치를 Elo 척도로 변환. 앵커 모델 기준 보정.
Author Rating
해당 모델이 출제한 문제들의 난이도 평균. 높을수록 더 어려운 문제를 만듦.
Composite Rating
가중 선형 결합으로 최종 순위 산출. Rm = wsolveRsolve + wauthorRauthor
핵심 발견

출제와 해결 능력의 부분적 분리

강한 해결자가 반드시 강한 출제자는 아니다 — 단일 역할 벤치마크에서는 보이지 않는 능력 축이 존재한다.

해결자 상위 모델

Solver Rating이 높지만 Author Rating이 중간인 모델들이 존재 — 문제를 잘 풀지만, 다른 모델을 압도할 만한 문제를 만들지 못함.

출제자 상위 모델

Author Rating이 높지만 Solver Rating이 상대적으로 낮은 모델 — 어려운 문제를 만들 수 있지만, 타 모델의 문제를 모두 풀지는 못함.

기여

세 가지 핵심 기여

1
셀프 플레이 평가 프로토콜 제안

모델이 출제자와 해결자 이중 역할로 평가받으며, 자동화된 문제 유효성 필터링을 포함하는 완전 자동화된 재현 가능한 프로토콜을 정식화.

2
19개 프론티어 모델 대규모 실험

이중 역할 평가가 단일 역할 벤치마크에서 보이지 않는 능력 분리를 드러내고, 출제와 해결 능력이 부분적으로 분리됨을 실증적으로 입증.

3
공개 리더보드 운영

새로운 모델이 출시될 때마다 업데이트되는 공개 리더보드를 제공하여, 지속적으로 진화하는 수학 추론 평가 생태계를 구축.
