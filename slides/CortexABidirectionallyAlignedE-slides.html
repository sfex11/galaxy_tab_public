Cortex: 양방향 정렬된
로봇 조작 에이전트 프레임워크

장기 수평(long-horizon) 조작을 위한 인지 오케스트레이터와 반응형 실행기의 양방향 정렬

Jiaqi Peng, Xiqian Yu, Delin Feng, Yuqiang Yang 외 · Tsinghua University · Shanghai AI Lab

문제 정의

단일 VLA 모델의 근본적 한계

마르코프적 단기 시야

현재 관측값만으로 행동을 결정하는 마르코프 성질로 인해, 장기 작업에서 중간 상태 추적이 불가능합니다.

행동 루프 함정

진행 상황을 판별하지 못해 동일한 행동을 반복하며, 오류가 누적되어 최종적으로 작업이 실패합니다.

제한된 컨텍스트 윈도우

비주얼 프레임 버퍼링 접근법은 문맥 길이에 제약받아 논리적 계획에 필요한 의미론적 기억을 형성하지 못합니다.

의미론-운동학적 격차

기존 계층적 방법은 고수준 계획 의미론과 저수준 실행 운동학 사이의 정렬 부족 문제를 겪습니다.

핵심 아이디어

Cortex: 인지 오케스트레이터 × 실행 정렬

고수준 VLM과 저수준 VLA 간에 실행 가능성(executability)과 추적 가능성(tractability)을 보장하는 맞춤형 하위 작업 인터페이스로 양방향 정렬을 달성합니다.

"Cortex는 고수준 VLM이 저수준 VLA의 능력을 고려하고, VLA가 VLM의 출력에 적응할 수 있도록 양방향 정렬을 달성하는 최초의 프레임워크입니다."

실행 가능성 (Executability)

32개 정규화된 스킬 원시형

조작 하위 작업을 32개의 표준화된 스킬로 정규화하여, VLM 출력이 VLA가 직접 실행할 수 있는 형태를 보장합니다.

인터페이스 정규화 예시

전역 명령: "비커 씻기"

  • s₁: [Pick] 테이블에서 비커 집기
  • s₂: [Move] 수도꼭지 근처로 이동
  • s₃: [Open] 수도꼭지 열기
  • s₄: [Wash] 비커 헹구기
  • s₅: [Close] 수도꼭지 닫기

자동 주석 파이프라인

  • 단조 하위 작업 분할 알고리즘
  • 의사 주석(pseudo-annotation) 자동 생성
  • 4,000시간 이상 오픈소스 비디오 활용
  • VLM이 이해 가능한 구조화된 형식
추적 가능성 (Tractability)

물리 원리 기반 데이터 생성

하위 작업 계획이 실제 물리 환경에서 실행 가능하도록 데이터 생성 파이프라인에 물리적 원리를 직접 주입합니다.

대표 객체 속성

객체의 위치, 자세, 크기 등 물리적으로 의미 있는 속성을 계획에 명시

궤적 도달 가능성

생성된 궤적이 로봇의 운동학적 제약 내에서 실제 도달 가능한지 검증

시뮬레이션 데이터

30시간 이상의 고품질 시뮬레이션 데이터를 추적 가능성 원리에 따라 생성

추적 가능성 원리가 없다면 VLM은 "멀리 있는 물체 집기"와 같은 물리적으로 불가능한 계획을 생성할 수 있습니다.

의미론적 기억

시간적 브릿지: 동적 기억 갱신

고립된 하위 작업 실행 시 발생하는 루프 문제를 해결하기 위해, 완료된 마일스톤을 누적하는 의미론적 기억 M(t)을 도입합니다.

M(t) = M(0) ⊕ Φ(s₁) ⊕ Φ(s₂) ⊕ ... ⊕ Φ(sₖ₋₁)

⊕: 의미론적 연결 · Φ(sᵢ): i번째 하위 작업의 완료 상태 인코딩

초기 기억 M(0)

"첫 번째 하위 작업, 이전 완료 항목 없음"

점진적 갱신

각 하위 작업 완료 시 Φ(sᵢ)가 기억에 누적됨

루프 방지

이전 완료 상태를 참조하여 중복 행동 회피

진행 상황 인식

현재 시점에서 전체 작업의 어느 단계인지 파악

학습 전략

이벤트 균형 샘플링

하위 작업 전환 단계에서 발생하는 계획 모호성을 해결하기 위해, 이벤트 유형별로 균형 잡힌 샘플링 전략을 설계합니다.

문제: 불균형 데이터

일반적으로 하위 작업 내부 실행 샘플이 전환 샘플보다 압도적으로 많아, 모델이 전환 시점을 제대로 학습하지 못합니다.

해결: 이벤트 균형 샘플링

  • 하위 작업 실행(Execution) 샘플 균형화
  • 하위 작업 전환(Transition) 샘플 균형화
  • 모호성이 높은 전이 구간 과샘플링
  • 통일된 명령 인터페이스로 학습 일관성 확보
추론 파이프라인

하니스 엔지니어링 기반 비동기 추론

추론 시 계획자와 실행기 간 정렬을 보장하기 위해, 작업 문맥과 스킬 제약을 명시적으로 전달하는 하니스(harness) 엔지니어링을 적용합니다.

4

시행착오 피드백

하위 작업 수준에서 실패 감지 시 VLM이 재계획을 수행하여 자가 복구

실세계 평가

제로샷 실세계 장기 작업 수행

Cortex의 범용 VLM은 파인튜닝된 VLA와 결합만으로 본적 없는 실세계 장기 작업을 제로샷으로 완수할 수 있습니다.

오븐 가열 실험

용기를 오븐에 넣고 가열하는 다단계 작업

쓰레기 분리 배출

비커 세척

수도꼭지 제어와 세척 동작의 조합

화학 액체 교반

다단계 화학 실험 — 복잡한 다단계 시퀀스

엔드투엔드 VLA: 성공률 0% · Cortex: 최대 65% 성공률 — VLA 파인튜닝만으로는 달성 불가능한 수준이며, VLA 정책을 사용하는 인간과 비슷한 성능

오픈루프 VLM 평가

계획 능력만으로도 범용 모델 능가

Cortex의 VLM 구성요소만 단독으로 평가해도, GPT-5, Gemini 등 최신 범용 모델의 계획 능력을 능가합니다.

Gemini 범용 수준 비정규화 출력 물리적 고려 없음

이는 양방향 정렬을 위한 맞춤형 인터페이스와 데이터 구성이 범용 모델의 일반적 지식보다 로봇 조작 계획에 훨씬 효과적임을 시사합니다.

결론

핵심 기여 및 시사점

양방향 정렬의 새로운 패러다임

VLM 계획과 VLA 실행 간 단방향 의존이 아닌, 상호 고려하는 양방향 정렬을 최초로 제안

실행 가능성 + 추적 가능성 인터페이스

32개 스킬 원시형과 물리 원리 주입으로 계획의 실현 가능성을 데이터 수준에서 보장

확장 가능한 데이터 구성

4k시간 오픈소스 비디오 자동 주석 + 30시간 시뮬레이션 데이터로 스케일러블한 메타데이터 파이프라인 구축

실세계 제로샷 일반화

다단계 화학 실험 등 본적 없는 실세계 작업을 VLA 파인튜닝 없이 제로샷으로 완수

한계: 현재 단일 로봇 플랫폼에서 검증되었으며, 교차 체현(cross-embodiment) 일반화와 더 복잡한 환경에서의 평가가 향후 과제입니다.
