검색 증강 에이전트의 현주소

대부분의 검색 증강 에이전트는 검색을 블랙박스로 취급한다 — 탐색적 쿼리를 발행하고, 반환된 스니펫을 검사하며, 유용한 증거가 나타날 때까지 반복적으로 재구성한다.

초보자 방식

익숙하지 않은 데이터베이스를 탐색하듯, 여러 라운드의 검색을 통해 말뭉치를 학습

전문가 방식

용어와 증거 패턴에 대한 강한 사전 지식을 바탕으로 직관적으로 검색

불필요한 검색 라운드, 증가된 지연시간, 낮은 재현율 문제

왜 기존 방식이 한계에 부딪혔는가

1

학습 신호의 붕괴

AI 요약 제공 시 사용자 클릭이 크게 감소(Pew 연구). 클릭 기반 학습이 불안정해짐

2

조합적 쿼리의 등장

단기 키워드에서 제약·배제·다단계 의도가 결합된 복합 쿼리로 이동

3

밀집 검색의 병목

고정 차원 임베딩은 제약 구조를 표현할 수 없고, 구조적 제어가 불가능

순수 유사도 검색은 구조적 제어(must-include/must-not-include, 속성 필터, 명시적 분해)에 부적합

BM25의 과소평가된 강점

  • 투명성 — 에이전트가 키워드 가중치 조절, 제약 강제, 쿼리 분해를 예측 가능하게 수행
  • IDF 가중치 — 희귀하고 판별력 있는 용어를 자연스럽게 보상하여 도메인 전문 용어가 강력한 신호로 작용
  • 감사 가능 — 어떤 키워드가 매칭되었는지 정확히 추적 가능, 밀집 인덱스의 지연/메모리 비용 없음
  • 구조적 제어 — 가중치, 제외, 분해를 명시적으로 지정할 수 있는 검색 원시 기능 제공

결핍된 요소: 올바른 희귀 용어와 제약을 발굴하는 메커니즘 → LLM의 방대한 파라메트릭 지식이 이 역할을 채움

LLM 에이전트가 모르는 검색의 어려움

검색은 단순한 의미 관련성 문제가 아니다 — 말뭉치 수준의 혼란 문서들 사이에서 정답 증거를 상위로 올리는 비교 순위화 문제다.

독립적으로 타당한 쿼리의 실패

  • 용어가 많은 오답 문서도 매칭
  • 가장 전문적으로 보이는 용어가 실제로는 흔하거나 판별력 없음
  • 타겟 인덱스에 용어 자체가 부재

검색-문맥 이점의 함정

  • 이전 검색 결과를 LLM 컨텍스트에 누적
  • 긴 컨텍스트에서 관련 증거가 묻힘
  • 비용이 높고 노이즈가 많음

핵심 간극: LLM은 관련 증거의 모습을 알지만, 인덱스 통계와 명시적 제어 없이는 판별력을 만들 수 없다.

검색에서의 초지능 정의

다중 라운드 탐색을 단일 말뭉치-판별적 검색 행동으로 압축하는 능력

1
도메인 사전 형성
관련 증거의 모습에 대한 기대치 구성
2
인덱스 통계 활용
문서 빈도(DF)로 기대치 검증 및 가지치기
3
검색 프로그램 컴파일
가중 키워드·제약·구조적 조합 생성
4
효율적 실행
단일 BM25 호출로 투명하게 검색

SIRA: 전체 구조

SIRA는 새로운 말뭉치에 대해 전문가처럼 검색하는 두 단계 프레임워크

오프라인 — 말뭉치 측

어휘 풍부화

LLM이 각 문서에 대해 누락된 검색 어휘를 오프라인으로 보강

  • 문서의 핵심 개념에 대한 동의어·변형 추가
  • DF 필터로 너무 흔한 용어 제거
  • 인덱스에 풍부화된 어휘 반영
온라인 — 쿼리 측

예상 응답 스케치

LLM이 쿼리에서 누락된 증거 어휘를 예측

  • 관련 증거에 등장할 개념·엔티티·판별 용어 가설
  • DF 툴 호출로 제안 용어 검증
  • 검색 여백을 만들 용어만 선택

핵심 메커니즘: DF 필터

SIRA는 단순히 "어떤 용어가 관련 있는가"가 아니라 "어떤 용어가 원하는 증거를 말뭉치 혼란자와 분리할 수 있는가"를 질문한다.

부재 용어

인덱스에 존재하지 않는 용어 → DF=0 → 즉시 제거

과다 빈도 용어

말뭉치 전체에 너무 흔한 용어 → 판별력 없음 → 제외

약한 판별 용어

검색 여백을 만들 수 없는 용어 → 가중치 하향 또는 제거

최종 검색: 원본 쿼리 + 검증된 확장 용어를 결합한 단일 가중 BM25 호출

검색 프로그램의 예시

원본 쿼리: "기계학습 모델의 편향을 줄이는 방법"

예상 응답 스케치: adversarial debiasing, counterfactual fairness, parity constraints, pre-processing interventions, representation learning...

DF 필터링 후: "adversarial debiasing" (DF=47✓), "counterfactual fairness" (DF=23✓), "representation learning" (DF=12,890✗ — 너무 흔함), "pre-processing" (DF=8,400✗)

컴파일된 BM25: 기계학습^1.0 모델^1.0 편향^1.0 줄이는^1.0 방법^1.0 "adversarial debiasing"^2.5 "counterfactual fairness"^2.0

가중치 부여, 제외어 지정, 구조적 조합 모두 단일 BM25 호출로 실현

실험 설정

벤치마크

  • BEIR 벤치마크 10개 데이터셋
  • 하위 질의응답(QA) 태스크
  • 다양한 도메인(과학, 뉴스, 웹 등)

비교 대상

  • 밀집 검색기 (Contriever, DPR 등)
  • 신경 희소 검색 (SPLADE)
  • 다단계 에이전트 (ReAct, IRCoT)
  • RL 기반 검색 에이전트

평가 설정

엄격한 1샷

1회 LLM 추론 + 선택적 DF 확인 + 1회 BM25

학습 여부

Training-free

검색 모델에 대한 지도 학습 없음

검색 문맥

Zero accumulation

이전 검색 결과 스니펫 읽기 불가

주요 결과

RQ1

지도 학습 없이 학습된 검색기 초과

SIRA는 BEIR 10개 벤치마크에서 학습된 밀집 검색기 및 SPLADE를 상회하는 성능 달성

단일 BM25 호출 + LLM 인지가 대규모 라벨 학습을 대체

RQ2

LLM 추론을 검색 고유 순위화로 변환

다단계 에이전트 기반선(ReAct, IRCoT, RL 에이전트)보다 우수한 NDCG/Recall

다중 라운드 검색 없이도 더 강력한 판별력 실현

RQ3

최고의 검색 에이전트가 최강의 QA 증거 산출

하위 QA 태스크에서 SIRA가 검색한 증거로 구축한 RAG 시스템이 다른 모든 검색 기반 QA를 상회 — 검색 품질이 최종 답변 품질을 결정적으로 좌우함을 입증

핵심 시사점

잘 설계된 단일 어휘적 쿼리가 비용이 많이 드는 다단계 검색을 능가할 수 있다

LLM 인지와 가벼운 말뭉치 통계(DF)의 결합이 검색에서 강력한 조합

해석 가능성과 효율성을 희생하지 않고 최고 성능 달성 — 어떤 키워드가 매칭되었는지 추적 가능

Training-free 패러다임 — 검색 모델에 대한 지도 학습 없이 새로운 말뭉치에 즉시 적용

검색 인터페이스의 설계가 LLM 에이전트 성능의 상한을 결정 — 블랙박스에서 제어 가능한 원시 기능으로 전환 필요

결론 및 한계

기여

  • 검색에서 초지능의 새로운 정의 제시
  • DF 기반 용어 검증 메커니즘 도입
  • 10개 BEIR 벤치마크에서 SOTA 달성
  • 학습 없이, 단일 호출로, 해석 가능하게

한계 및 향후 과제

  • 오프라인 말뭉치 풍부화에 LLM 호출 비용 발생
  • 다중 쿼리 허용 시 추가 성능 향상 가능성 미탐색
  • 매우 대규모 말뭉치에서의 확장성 추가 검증 필요
  • 멀티모달 검색으로의 확장
