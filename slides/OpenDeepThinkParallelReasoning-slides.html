UC San Diego · Princeton · UW · UC Berkeley

문제 정의: 테스트 시간 연산 스케일링의 병목

깊이 스케일링 (기존)

  • 단일 추론 경로를 연장
  • 본질적으로 순차적 (sequential)
  • 초기 오류가 전체를 망침

너비 스케일링 (Best-of-N)

  • 다수 후보를 병렬 샘플링
  • 자연스럽게 병렬화 가능
  • 선택 병목 발생
정답 검증자(verifier) 없이 다수 후보 중 최적을 선택하는 것 — 이것이 핵심 병목이다.

기존 방법의 한계

Pointwise LLM 판정

긍정 편향(positive bias) 심각 — 정확도 59%에 불과

Self-Refine / Reflexion

단일 궤적만 수정 → 인구 수준 선택 압력 부재, "인지 웅덩이" 문제

FunSearch / AlphaEvolve

프로그램적 검증자(programmatic verifier) 필수 → 도메인 종속적

Self-Consistency (다수결)

투표 가능한 최종 답안이 있는 작업에만 한정

핵심 통찰: 쌍대 비교의 힘

절대 평가(self-evaluation)는 어렵지만, 상대 평가(pairwise comparison)는 훨씬 쉬운 판별 과제다. 두 후보를 비교하면 긍정 편향이 상쇄된다.

선택 메커니즘: Bradley–Terry 집계

1. 무작위 쌍대 비교

후보당 K회 무작위 짝을 구성하여 LLM이 승자를 판정. 자연어 비평(critique)을 함께 생성.

2. Bradley–Terry 최대우도 추정

각 후보 i의 능력 파라미터 γᵢ를 추정. P(i beats j) = γᵢ / (γᵢ + γⱼ). 상대 강도를 자동으로 보정.

3. 엘리트 보존과 폐기

폐기

변이(Mutation) 메커니즘

피드백 조건부 재생성

쌍대 비교에서 생성된 자연어 비평을 프롬프트에 주입. 기존 접근법을 완전히 포기하고 새로운 방향으로 전환할 수 있도록 허용.

주요 발견: 부정 피드백(negative feedback)이 변이 신호의 거의 전부를 담당한다. 패배 이유에 대한 비평이 다음 세대의 개선 방향을 결정한다.

Self-Refine과의 결정적 차이

Self-Refine은 단일 궤적의 자기 평가로 오류를 탐지하지만, OpenDeepThink는 후보 간 비교로 오류를 드러낸다 — 판별 과제로서 훨씬 신뢰 가능.

논의: 주요 발견 세 가지

1

쌍대 선택이 pointwise를 압도

생성 품질이 동일한 조건에서도 쌍대 비교가 pointwise 판정보다 우수한 선택 신호 제공

2

부정 피드백이 변이의 핵심

패배 사유에 대한 비평(negative critique)이 거의 모든 변이 신호를 담당 — 긍정 피드백의 기여는 미미

3

진화는 전방 집중, 밀집 비교가 잔여 추출

대부분의 개선은 초기 세대에서 발생하며, 최종 밀집 비교 라운드가 잔여 성능을 끌어올림

핵심 메시지

훈련 없이, 검증자 없이, 도메인 특화 없이 — 쌍대 Bradley–Terry 집계와 피드백 변이만으로 강력한 병렬 추론이 가능하다.

한계

  • 주관적 도메인에서는 효과가 역전됨 (쌍대 판정 신뢰도 의존)
  • 단일 모델 계열(Gemini)에서만 검증
  • 진화가 전방 집중되어 후기 세대의 한계적 개선
  • 연산 예산이 고정되어 있어 스케일링 특성 추가 연구 필요
