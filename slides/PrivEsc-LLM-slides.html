Motivation

왜 로컬 LLM 보안 에이전트가 필요한가?

🔒 현재 한계

  • ❌ 클라우드 API 의존 → 민감 데이터 유출 위험
  • ❌ 높은 추론 비용 (건당 ~$0.62)
  • ❌ 네트워크 격리 환경에서 사용 불가
  • ❌ 소형 모델의 보안 태스크 성능 부족

✅ 제안 솔루션

  • ✓ 4B 로컬 모델로 오프라인 실행
  • ✓ 2단계 후훈련 (SFT + RL)
  • ✓ 검증 가능한 보상으로 자동 평가
  • ✓ 100배 비용 절감

Linux 권한 상승이 이상적인 이유

자동 검증 가능: root 쉘 획득 여부로 성공/실패를 이진 판단 → RL 보상 설계에 최적

다단계 추론 필요: 정찰 → 취약점 식별 → 익스플로잇 실행의 복합 의사결정 체인

절차적 환경 생성: Docker 컨테이너로 무한한 훈련 시나리오 자동 생성

실용적 가치: 레드팀/펜테스트 자동화의 핵심 역량

SFT 상세

교사 모델: Arcee Trinity-Large (398B MoE, 13B active)
LoRA: rank=8, alpha=32 (17.75M params, 0.44%)
LR: 1.5×10⁻⁴ (log-sweep 선택)
최적화: AdamW 8-bit, β₂=0.95
시퀀스: 32,768 토큰, Batch 8
선택: 10 에폭 중 4번째 체크포인트

RL (AIPO) 상세

알고리즘: Token-level Async Importance-Weighted PO
프레임워크: Prime-RL
LR: 1.5×10⁻⁵ (SFT의 1/10)
배치: 80 (인스턴스당 8 rollout)
스텝: 1,000 · 훈련 호라이즌 12라운드
샘플링: Round-robin + staleness bounds
Reward Design

다중 성분 보상 함수

R(e) = R_out + R_speed + R_recon − R_pen
R_out · 결과
2·𝟙{root} − 1
성공: +1 / 실패: −1
R_speed · 속도
𝟙{root}·(1 − τ/τ_max)
빠른 탈취에 보너스
R_recon · 정찰
min(0.30, 0.05·u)
고유 정찰 패턴 보상
R_pen · 패널티
Σ λᵢ · nᵢ
반복/오류 행동 억제

패널티 가중치

λ_rep = 0.15 · 반복 도구 호출

Insights

핵심 기술적 통찰

라운드 예산 민감도
PrivEsc-LLM은 제한된 예산(R=20)에서 95%를 달성하지만, 베이스 Qwen3-4B는 42.5%에 불과. RL이 단순 성능 향상이 아닌 상호작용 효율성을 학습함을 시사.
SFT vs RL 기여도
SFT가 ~38% 성능 향상 (42.5→80.8%), RL이 추가 ~15% (80.8→95.8%) 향상. SFT는 기초 능력, RL은 효율적 전략을 부여.
비동기 오프폴리시 학습
AIPO + bounded staleness로 rollout 생성 지연 문제 해결. 각 rollout이 다단계 LLM-환경 상호작용이므로 비동기 안정화가 필수.
절차적 일반화
시나리오별 문자열(유저명, 비밀번호, 경로)을 랜덤화하여 패턴 매칭이 아닌 정책 학습 유도. 벤치마크 풀이 경로를 명시적으로 제외하여 암기 방지.
실패 모드 분석
Sudo GTFOBins (6/10): tar 벡터 식별 후 오류 복구 실패 — 14~22회 잘못된 명령 반복.
Docker Escape (9/10): 그룹 멤버십 발견 후 미행동, 비밀번호 추측으로 전락.
DeepSeek V3.2 특성
높은 예산 민감도 (R=5: 17% → R=60: 94%) — 열거 중심 정책으로 초기 라운드 비효율적. 반면 Claude Opus는 모든 예산에서 일관된 성능, 천장 효과 근접.
Conclusion

한계점 & 결론

⚠️ 한계점

• 단일 베이스 아키텍처 (Qwen3-4B)만 테스트
• RL 훈련에 4×H100 필요 (추론만 로컬)
• 일반적 오설정만 커버, 실제 long-tail 취약점 미포함
• 모델별 프롬프트 최적화 없이 단일 시스템 프롬프트
• 터미널 출력 잘림 → 평가 아티팩트 가능
• 12개 시나리오 벤치마크의 규모 한계

🔬 평가 프로토콜

• R=5, 10, 20, 60 다중 예산 평가
• 모델-시나리오당 10회 독립 실행 (총 120회)
• 실행당 새 Docker 컨테이너
• Wilson 95% CI + Bootstrap CI
