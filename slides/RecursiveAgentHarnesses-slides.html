Recursive Agent Harnesses

모델 재귀에서 하니스 재귀로: 긴 컨텍스트 추론을 위한 새로운 패러다임

Elias Lumer, Sahil Sen, Kevin Paul, Vamse Kumar Subbiah

PricewaterhouseCoopers, U.S.

문제 제기

현대 코딩 에이전트는 수천 개의 독립 항목에 대해 항목별 LLM 추론이 필요한 작업에서 한계에 부딪힌다. 핵심 질문은 다음과 같다.

에이전트가 긴 컨텍스트 작업을 재귀적으로 처리할 때, 재귀 단위는 도구 없는 모델 호출이어야 하는가, 아니면 완전한 에이전트 하니스이어야 하는가?

코딩 에이전트

정규식 휴리스틱에 의존 → 항목별 추론 생략

재귀 언어 모델 (RLM)

모델 재귀는 유효하나 도구 접근 부재

기존 접근법의 한계

코딩 에이전트의 맹점

  • 항목별 추론을 정규식으로 축소
  • 컨텍스트 윈도우에 수천 항목 불가
  • LLM 추론 자체를 건너뜀

RLM의 맹점

  • 파일 시스템 접근 불가
  • 도구 사용 없는 순수 모델 호출만 재귀
  • 정규식 교차 검증 불가

Oolong 벤치마크는 이 간극을 명확히 드러낸다: 코딩 에이전트는 정규식 한계로, RLM은 도구 부재로 낮은 점수를 기록한다.

Recursive Agent Harness (RAH)

RAH는 완전한 에이전트 하니스를 재귀 단위로 삼는 패턴이다. 이를 하니스 재귀(Harness Recursion)라 정의하며, RLM의 모델 재귀에 대한 코드 중심 확장이다.

스포닝 로직은 고정된 재귀 호출 규약이 아닌 일반 프로그램 코드이므로, 동시성, 항목별 지시사항, 출력 경로를 동일한 언어로 매개변수화할 수 있다.

두 가지 스폰닝 경로

코드 실행 스폰닝

  • 부모가 실행 가능한 스크립트 작성
  • Task() 객체로 서브에이전트 인스턴스화
  • asyncio.gather로 병렬 실행
  • API 턴당 툴콜 제한 우회
  • 수천 개 서브에이전트 스케일링

세밀한 작업(다수 항목)에 적합

JSON 툴콜 스폰닝

  • 구조화된 함수 호출로 직접 Task 호출
  • 중간 스크립트 생성 불필요
  • 항목 내용과 지시사항을 인자로 전달
  • 1~5개 항목 소규모 하위 작업에 최적

소규모 작업에 적합 (1~5 항목)

7+

실행 흐름

1
서브에이전트는 실행 전 계획 단계(Planning)를 수행
2
도구를 활용하여 항목별 추론 수행 (정규식 + 추론 교차 검증 가능)
3
완료 시 공유 출력 파일에 결과 기록
4
부모는 모든 서브에이전트 완료 후 집계된 stdout만 수신

실험 설정

벤치마크

  • Oolong-Synthetic (Cao et al., 2026)
  • 199개 무작위 추출 인스턴스
  • 13개 컨텍스트 길이 버킷 (1K~4M 토큰)
  • 계층적 추출 (stratified sampling)

백본 모델

  • 통제 실험: GPT-5 (Codex 베이스라인과 동일)
  • 강화 실험: Claude Sonnet 4.5
  • 모델 고정 → 성능 차이는 하니스에 기인

핵심 설계 원칙: 백본을 고정함으로써 성능 향상이 모델이 아닌 하니스 아키텍처에서 비롯됨을 입증한다.

컨텍스트 길이별 분석

RAH의 향상은 모든 컨텍스트 길이 버킷에서 일관되게 나타나며, 4M 토큰에서도 유의미한 성능 유지를 보인다.

핵심 관찰

  • 짧은 컨텍스트(1K~32K): 기존 방법과 유사하나 RAH가 안정적으로 우위
  • 중간 컨텍스트(64K~512K): 항목 수 증가로 RAH의 병렬 스폰닝 이점 극대화
  • 긴 컨텍스트(1M~4M): 코딩 에이전트의 정규식 한계가 심화되나 RAH는 항목별 추론 유지
  • 재귀 깊이 제한은 설정 가능하며, 대부분 1~2수준으로 충분

하니스 재귀는 단일 컨텍스트 윈도우 제약을 구조적으로 우회하며, 컨텍스트 길이가 늘어날수록 그 이점이 더욱 명확해진다.

비용·지연시간 및 실패 모드

비용 및 지연시간

  • 서브에이전트 병렬 실행으로 지연시간 감소
  • 총 토큰 사용량은 항목 수에 비례하여 증가
  • 코드 실행 경로가 JSON 툴콜보다 오버헤드 적음
  • 재귀 깊이 설정으로 비용 상한 통제 가능

주요 실패 모드

  • 서브에이전트 출력 형식 불일치
  • 집계 단계에서의 답변 추출 오류
  • 과도한 재귀 깊이로 인한 비용 폭증
  • 공유 파일 동시 쓰기 충돌

재귀 전략 선택 가이드

1~5 항목 → JSON 툴콜 스폰닝 (간편, 낮은 오버헤드)
6+ 항목 → 코드 실행 스폰닝 (스케일링, 병렬성)
복잡한 항목 → 서브에이전트가 재귀적으로 추가 분해

관련 연구와의 위치

연구 재귀 단위 도구 접근 스포닝 방식
RLM (2026) 모델 호출 없음 고정 규약
Minions (2025) 워커 모델 제한적 오케스트레이션
AGENTHIVE (2026) 에이전트 있음 스키마 정의
Anthropic 동적 워크플로우 에이전트 있음 코드 실행
RAH (본 연구) 하니스 전체 실행 코드

RAH는 Anthropic의 동적 워크플로우와 동일한 코드 중심 스폰닝을 연구 프레임으로 체계화하고, 모델 재귀에 대한 최초의 통제된 벤치마크 비교를 제공한다.

한계점

  • 비용 증가: 항목별 독립 하니스 생성으로 총 토큰 소비가 선형적으로 증가
  • 단일 벤치마크: Oolong-Synthetic에만 평가되어 일반화 가능성 추가 검증 필요
  • 구현 의존성: LangChain 프레임워크 기반으로 구현 언어 의존성 존재
  • 원시 기반: 새로운 메커니즘 발명이 아닌 기존 원시기(primitive)의 조합 및 명명
  • 재귀 깊이 설정: 최적 깊이를 자동으로 결정하는 메커니즘 부재
  • 오류 전파: 서브에이전트 실패가 상위로 전파되는 패턴에 대한 분석 부족

본 연구의 기여는 원시기를 발명하는 것이 아니라, 패턴을 명명하고 통제된 평가를 제공하는 데 있다.

결론

RAH는 재귀 단위를 도구 없는 모델 호출에서 완전한 에이전트 하니스로 격상시킴으로써, 긴 컨텍스트 추론에서 기존 접근법의 양쪽 맹점을 동시에 해결한다.

세 가지 기여

1
하니스 재귀(Harness Recursion) 명명 및 정의 — RLM 모델 재귀의 코드 중심 확장
2
백본 고정 통제 실험 — 성능 차이를 하니스에 귀인
3
Oolong-Synthetic 전체 버킷에 걸친 종합 평가 및 Anthropic 동적 워크플로우와의 연결

에이전트 하니스에 대한 재귀는 단일 컨텍스트 윈도우를 초과하는 작업에서 점점 기본 전략이 되고 있으며, RAH는 이 패턴을 체계적으로 연구한 첫 작업이다.
