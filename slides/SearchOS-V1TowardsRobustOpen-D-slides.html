문제 인식

도구 통합 LLM의 발전으로 웹 검색은 정보 탐색 에이전트의 핵심 능력이 되었습니다. 그러나 장기 수행 태스크에서는 심각한 한계가 드러납니다.

  • 상호작용 기록이 길어지면 진행 상황 추적 실패
  • 검색 실패 시 유사 쿼리 반복 — 검색 예산 낭비
  • 증거가 컨텍스트에 묻혀 누락·중복·충돌 탐지 불가

단순히 에이전트를 추가해도 해결되지 않는 이유

  • 병렬 워커 간 작업 중복
  • 필드 해석 불일치
  • 가장 느린 태스크 대기로 슬롯 유휴
검색 상태는 에이전트가 매번 대화 기록에서 추론하는 것이 아니라 시스템이 유지해야 합니다.

문제 정식화: 관계형 스키마 완성

개방 도메인 정보 탐색을 근거 인용이 포함된 관계형 스키마 완성으로 정식화합니다.

관계형 검색 스키마 𝒮

  • 다중 테이블 {Tₘ} — 기본키로 엔티티 식별
  • 외래키 관계 ℛ — 테이블 간 연결
  • 정규화로 중복·불일치 감소

출력 산출물 𝒪

  • 엔티티 집합 ℰₘ 발견
  • 값 행렬 𝐘ₘ 채움
  • 인용 행렬 𝐂ₘ — 모든 값에 출처 URL + 발췌문 앵커링

SearchOS 아키텍처 전체 구조

SearchOS는 상태 기반 폐루프(stateful closed loop)로 설계됩니다. 오케스트레이터가 미해결 스키마 갭을 분배하고, SOCM이 관찰을 커밋하며, 미들웨어가 모든 상호작용을 근거화합니다.

핵심 원칙: 계획, 진행, 증거, 실패를 일시적인 대화 내용이 아닌 시스템 수준의 지속 상태로 관리한다.

SOCM: 검색 지향 컨텍스트 관리

진화하는 검색 상태를 에이전트 대화 외부에 영속적 공유 상태 ℳ로 외부화합니다. 잠금 기반 읽기-수정-쓰기로 일관성을 보장합니다.

Frontier Task ℱₜ

현재 검색해야 할 구체적인 스키마 갭 — 어떤 테이블의 어떤 속성이 아직 채워지지 않았는지 명시

Evidence Graph 𝒢ₜ

수집된 증거를 노드-엣지 그래프로 구성 — 엔티티, 속성 값, 출처 URL 간의 관계를 명시적으로 추적

Coverage Map 𝒞ₜ

스키마 전체의 커버리지를 테이블·속성 수준에서 시각화 — 어떤 영역이 해결되었고 어떤 갭이 남았는지 한눈에 파악

Failure Memory 𝒲ₜ

실패한 검색 패턴을 기록 — 동일한 실패 경로 재탐색 방지, 세션 간 실패 패턴 학습

파이프라인 병렬 다중 에이전트 오케스트레이션

동기화된 배치가 아닌 연속 디스패치 방식 — GPU 훈련의 파이프라인 병렬성 원리를 에이전트 오케스트레이션에 적용합니다.

에이전트 역할

  • Explore 에이전트 — 엔티티 발견 및 스코프 확장
  • Search 에이전트 — 특정 속성 값 검색
  • Writer 에이전트 — 최종 보고서 생성

연속 디스패치 메커니즘

  • 에이전트 슬롯 해제 즉시 새 갭 할당
  • 가장 느린 태스크 대기 시간 최소화
  • 검색 처리량(throughput) 극대화

배치 vs 파이프라인 병렬 비교

배치 방식 전체 완료 후 다음 배치 → 느린 작업이 병목
파이프라인 슬롯 해제 즉시 재할당 → 유휴 시간 최소화

도구 설계

브라우저 인터페이스, 스키마/엔티티 관리 도구, 태스크 큐, 라이터 도구를 통합 제공

검색 도구 미들웨어 하니스

에이전트가 기억해서 호출해야 하는 통제가 아니라, 미들웨어가 자동으로 보장하는 실행 불변성을 설계합니다.

1

컨텍스트 미들웨어

모델 호출 전 SOCM에서 관련 상태를 주입하여 에이전트가 현재 검색 맥락을 인지하도록 보장

2

증거 추출 미들웨어

도구 응답에서 근거 발췌문을 자동 추출하고 Evidence Graph에 앵커링하여 인용 무결성 유지

3

센서 미들웨어

반복/정체 궤적 감지, 예산 소진 감시, 실패 패턴 식별 — 에이전트 프롬프트 외부에서 실행 제어

에이전트는 로컬 검색 결정에 집중하고, 미들웨어가 글로벌 실행 불변성을 유지합니다.

계층적 검색 스킬 시스템

재사용 가능한 검색 전략과 사이트별 접근 절차를 분리하여 계층화하고, 성공·실패 궤적에서 지속적으로 정제합니다.

오케스트레이터 스킬

스키마 분해, 태스크 우선순위 결정, 커버리지 기반 갭 선택

전략 스킬

어떻게 검색할 것인가 — 쿼리 재구성, 다단계 검색, 교차 검증 전략

접근 스킬

특정 출처에 어떻게 도달하고 정보를 추출할 것인가 — 사이트별 세부 절차

스킬 선택 및 실행 흐름

  • 태스크 유형과 출처에 따라 자동 라우팅
  • Failure Memory와 연동하여 실패 패턴 회피
  • 세션 간 스킬 재사용 — 매 태스크마다 재발견 불필요
  • 성공/실패 궤적에서 스킬 자동 정제

절제 연구 분석

고정 스키마 vs 검색 시 스키마 계획

검색 시점에 스키마를 동적으로 계획하는 것이 고정 스키마보다 유연성 측면에서 우수하지만, 관계형 구조의 사전 정의가 커버리지 추적의 안정성을 제공함

파이프라인 병렬 스케줄링 절제

배치 방식 대비 파이프라인 병렬이 유휴 시간을 크게 감소시키며, 특히 태스크 간 실행 시간 편차가 클 때 효과가 극대화됨

미들웨어 통제 실행 분석

미들웨어 없이 에이전트에만 통제를 맡기면 반복 검색과 예산 초과가 빈번히 발생 — 미들웨어가 실행 품질을 결정적으로 향상

스킬 절제 및 효율성

전략 스킬과 접근 스킬 모두 효과에 기여하며, 특히 접근 스킬이 사이트별 접근 성공률을 크게 개선하고 불필요한 재시도를 감소

사례 연구

사례 1 Spotify 2024 랭킹

다중 국가·다중 장르의 음악 랭킹을 수집하는 장기 태스크

  • SOCM이 국가별 커버리지를 명시적으로 추적
  • Failure Memory가 차단된 출판사 페이지 자동 우회
  • 파이프라인 병렬로 국가 간 검색 동시 진행

사례 2 베이징 여행 계획

관광지, 숙소, 교통, 비용 등 다차원 정보 통합

  • 관계형 스키마로 엔티티(장소)·속성 분리 관리
  • 거짓 포화 감지 후 스코프 복구 자동 수행
  • 인용 행렬로 모든 추천에 출처 부착
교차 사례 관찰: SOCm의 명시적 상태가 에이전트의 자기 진단 능력을 크게 향상하며, Failure Memory가 실패 경로의 조기 차단을 가능하게 합니다.

핵심 기여 요약

관계형 검색 정식화

개방 도메인 정보 탐색을 근거 인용이 포함된 관계형 스키마 완성으로 정식화 — 엔티티 발견, 속성 완성, 증거 귀속의 통일된 목표 제공

상태 기반 파이프라인 병렬 협업

SOCM과 연속 오케스트레이션으로 공유 검색 상태를 통한 전문 에이전트 조정 및 미해결 갭 기반 동적 작업 할당

미들웨어 통제 검색 실행

증거 처리, 컨텍스트 관리, 예산 집행, 행동 안전장치를 에이전트 프롬프트 외부로 이동

계층적 검색 스킬

재사용 가능한 검색 전략과 사이트별 접근 절차를 세션 간 공유하여 에이전트 검색 과정 강화
