02 / 12 — 문제 정의

LLM은 자신감 있게 틀린다

핵심 문제

  • LLM은 틀릴 때에도 높은 신뢰도로 답변
  • 기권(abstention)이 더 안전한 상황에서도 답변을 강행
  • 기존 평가 프로토콜은 항상 응답을 요구

기존 지표의 한계

  • 대칭적 페널티만 부여 (과신 ≒ 과소신뢰)
  • 위험 선호도(risk preference)를 고려하지 않음
  • 의사결정 관점의 평가 부재

결과: 정확도가 비슷한 모델도 실제 의사결정 신뢰성은 극적으로 다를 수 있다

04 / 12 — 수학적 프레임워크

선택적 효용 함수

위험 의존 효용 모델 — 임계값 t ∈ [0,1)

St(Z, a) = { 1 (정답&응답) , −t/(1−t) (오답&응답) , 0 (기권) }

위험 허용도 t에 따라 오답의 비용이 비대칭적으로 증가합니다.
의사결정 정책: 신뢰도 s ≥ t 이면 응답, 아니면 기권

기대 BAS 효용

∫₀¹ 𝔼[St(Z, πs(t))] dt

모든 위험 허용 수준에 걸쳐 적분하여 종합적 신뢰성 측정

폐쇄형 표현식

U(s,Z) = { s (정답) , s + ln(1−s) (오답) }

ln(1−s)는 s→1일 때 음의 무한대로 발산 — 과신 오류 심각 페널티

05 / 12 — 최종 지표 & 정리

BAS 최종 공식

BAS = (1/N) Σi=1N U(si, Zi)

정리 2.1 — 최적성

진실된 신뢰도 추정치가 기대 BAS 효용을 유일하게 최대화한다.

  • p < 1인 경우: s = p (실제 확률)일 때 최대
  • p = 1인 경우: 효용은 s→1로 갈수록 단조 증가

→ BAS는 의사결정 최적 행동에 부합하는 적절한 스코어링 목표(proper scoring rule)

+s
정답일 때 보상
(신뢰도가 높을수록 높은 점수)
−∞
과신 오답일 때 페널티
(s→1이면 ln(1-s)→−∞)

핵심 발견: 동일 ECE/AURC ≠ 동일 BAS

SimpleQA에서 Llama 3.3과 Mistral은 거의 동일한 ECE(68.1% vs 68.8%)와 AURC(0.75 vs 0.74)를 보이지만,
BAS는 −2.97 vs −1.26으로 극적인 차이 → Llama의 극단적 과신 오류를 BAS만 포착

10 / 12 — 확장

안전 중요 환경을 위한 가중 BAS

일반화된 가중 BAS

기대 가중 BAS = ∫₀¹ 𝔼[St(Z, πs(t))] · w(t) dt

균일 위험 사전분포를 임의의 가중 함수 w(t)로 대체하여 특정 위험 수준을 강조

균일 가중 (기본)

w(t) = 1

모든 위험 임계값에 동일한 중요도

이차 가중 (안전 중요)

w(t) = 3t²

높은 임계값(안전 중요) 시나리오에 집중

정리 B.1 — 일반 최적성

임의의 양수 가중 함수 w(t) 하에서도 진실된 보고가 여전히 최적으로 유지됨

실용적 시사점

1

간단한 개입이 효과적

Top-k 신뢰도 추출은 단일 포워드 패스만 필요하지만 신뢰성을 크게 향상시킴

2

샘플 효율적 보정

50~100개의 검증 예제만으로 거의 최적의 의사결정 수준 효용 회복 가능

3

프론티어 모델도 예외 아님

GPT-4o도 어려운 추론 과제에서 심각한 과신을 보임 — 규모만으로 해결 불가

4

보완적 지표 필요

BAS는 보정/순위 지표에서 보이지 않는 의사결정 수준 위험을 포착

핵심 메시지: LLM 배포 시 정확도만이 아닌 신뢰도의 의사결정 품질을 반드시 평가해야 한다

결론
