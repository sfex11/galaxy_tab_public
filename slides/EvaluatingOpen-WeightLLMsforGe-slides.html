Evaluating Open-Weight LLMs for Generating Structured Threat Information for Autonomous Vehicle Vulnerabilities

arXiv:2607.16175v1 · 2026년 7월

자율주행차 보안의 현실

연결형 자율주행차(CAV)는 센서, ECU, 인포테인먼트 시스템, 텔레매틱스 등 수많은 소프트웨어·하드웨어 구성 요소로 동작합니다.

  • LiDAR, 레이더, 카메라 등 센서
  • 전자 제어 유닛(ECU) 및 CAN 버스
  • 인-차량 인포테인먼트 시스템
  • 텔레매틱스 제어 유닛 및 V2X 통신

2024년 기아 취약점: 번호판만으로 원격 차량 제어 가능

2025년 스바루 STARLINK: 차량 제어, 고객 데이터, 위치 기록 노출

모든 복잡한 소프트웨어 시스템에는 버그가 존재하며, CAV의 방대한 코드베이스도 예외가 아닙니다. 이 중 일부는 보안 취약점으로 악용될 수 있습니다.

문제 정의: 비정형 → 구조화

1
CVE 기록
자연어 평문으로 작성된 취약점 설명
2
수동 분석
보안 실무자가 직접 자산, 약점, 공격 행위 식별
3
STIX 구조화
엔티티, 관계, 약점·공격 매핑을 JSON 형식으로 변환

현재의 한계

  • 수동 분석은 시간·비용 집약적
  • CAV 도메인 전용 STIX 데이터셋 부재
  • LLM 생성 STIX의 신뢰성 검증 부족

연구 목표

  • CAV 전용 CVE→STIX 데이터셋 구축
  • 오픈 웨이트 LLM 11개 모델 평가
  • 프롬프트 전략 및 멀티 에이전트 분석

핵심 개념

CVE Common Vulnerabilities and Exposures

공개적으로 알려진 취약점에 대한 고유 식별자와 자연어 설명을 제공하는 데이터베이스 (예: CVE-2023-28901)

CWE Common Weakness Enumeration

소프트웨어·하드웨어 약점 유형의 표준 분류 체계 (예: 접근 제어 부재, 버퍼 오버플로우)

ATT&CK MITRE ATT&CK

전술(Tactics), 기법(Techniques), 절차(Procedures)로 adversary 행위를 분류하는 체계

STIX Structured Threat Information eXpression

사이버 위협 정보를 JSON 기반으로 표현하는 표준 형식 — SDO(도메인 객체)와 SRO(관계 객체)로 구성

CVE 평문 → STIX(SDO + SRO + CWE 매핑 + ATT&CK 매핑)으로 구조화하여 기계 판독 가능한 형태로 변환

연구 질문

RQ1

데이터셋 구축

CAV 도메인의 비정형 CVE 설명으로부터 STIX 표현 데이터셋을 어떻게 구축할 수 있는가?

RQ2

오픈 웨이트 LLM 평가

다양한 프롬프팅 전략 하에서 오픈 웨이트 LLM들이 CVE→STIX 생성을 얼마나 효과적으로 수행하는가?

RQ3

CWE 및 ATT&CK 기법 분석

CAV 관련 취약점에서 빈번하게 나타나는 CWE와 ATT&CK 기법은 무엇이며, 어떤 기법들이 함께 발생하는가?

CAV-STIXGen 데이터셋 구축 (RQ1)

1
CVE 수집
CAV 관련 CVE 기록 선별
2
수동 주석
STIX 엔티티, 관계, CWE, ATT&CK 매핑
3
검증
주석 간 일관성 검토 및 수정
4
STIX JSON
검증된 주석을 Ground-Truth STIX 번들로 변환
SDO

공급자, 클라우드, 서비스, 취약점, 공격 패턴, 공격자 등 도메인 객체

SRO

객체 간 관계 (예: 취약점이 서비스에 영향, 공격자가 취약점 악용)

CWE+ATT&CK

약점 유형 및 공격 행위 분류 매핑

데이터셋, 프롬프트, 재현 스크립트 공개: figshare.com/s/29f4ceff6bb1c6de5700

실험 설계

평가 대상 모델

오픈 웨이트 LLM 11개 (4B ~ 120B 파라미터)

  • Gemma 계열 (2B, 9B, 27B, 31B)
  • Llama 계열 (8B, 70B, 120B)
  • Mistral 계열 (7B, Codestral-22B)
  • Qwen2.5-Coder-32B 등

평가 차원

SDO 추출 — 도메인 객체 식별 정확도

SRO 추출 — 관계 객체 식별 정확도

CWE 매핑 — 약점 유형 분류 정확도

ATT&CK 매핑 — 공격 기법 분류 정확도

다양한 프롬프팅 전략(Zero-shot, Few-shot, Chain-of-Thought 등) 및 온도 설정 비교 · 멀티 에이전트 설정도 추가 평가

주요 결과: 단일 모델 성능 (RQ2)

평가 항목 최고 F1 점수 해석
SDO 추출 0.94 도메인 객체 식별에 매우 우수
SRO 추출 0.63 관계 객체 식별은 중간 수준
CWE 매핑 0.99 약점 유형 분류에 거의 완벽
ATT&CK 매핑 낮음 전체 매핑은 여전히 도전적

강점

SDO와 CWE 매핑에서 높은 정확도 — LLM이 취약점 설명에서 핵심 엔티티와 약점 유형을 잘 파악함

한계

SRO 추출은 객체 간 관계 유추가 어려워 F1 0.63에 머물며, ATT&CK 전체 매핑은 아직 불완전

프롬프트 전략 비교 (RQ2)

전략별 성능 경향

Z

Zero-shot

기본 지시만으로도 SDO/CWE에서 준수한 성능

F

Few-shot

예시 제공으로 SRO 추출 성능 향상

C

Chain-of-Thought

단계적 추론이 복잡한 관계 추출에 효과적

모델 수준 발견 사항

  • 파라미터 크기가 클수록 SRO 성능 향상 경향
  • 코딩 특화 모델(Codestral, Qwen2.5-Coder)이 구조화 JSON 생성에 유리
  • 온도 설정이 출력 일관성에 큰 영향
  • SDO 유형별로 식별 난이도 차이 존재

작은 모델(4B~9B)도 SDO와 CWE에서는 경쟁력 있으나, SRO와 ATT&CK에서는 큰 모델이 압도적 우위

멀티 에이전트 STIX 생성 (RQ2)

단일 모델의 한계를 극복하기 위해 전문 역할을 분담하는 멀티 에이전트 구성을 실험

Gemma-4-31B 기반

0.91

SDO F1 점수

단일 모델 최고(0.94)와 근접한 수준으로, 역할 분담이 효과적으로 작동

Codestral-22B 기반

0.43

SRO F1 점수

코딩 모델의 JSON 구조화 강점을 활용하나, 관계 추출은 여전히 어려움

멀티 에이전트 설정은 SDO에서 단일 모델에 근접한 성능을 보였으나, SRO에서는 단일 모델 최고 성능(0.63)에 미치지 못함 — 에이전트 간 조율 메커니즘 개선이 필요

CWE 및 ATT&CK 기법 분석 (RQ3)

빈번한 CWE 유형

  • CWE-200: 민감 정보 노출
  • CWE-287: 부적절한 인증
  • CWE-862: 부적절한 접근 제어
  • CWE-20: 부적절한 입력 검증

접근 제어와 인증 관련 약점이 CAV 취약점의 주요 원인

ATT&CK 기법 공발현 패턴

  • T1190: 공개 웹 애플리케이션 취약점 악용
  • T1609: 시스템 서비스를 통한 실행
  • T1530: 클라우드 계정에서 데이터 추출

초기 접근(T1190) → 실행(T1609) → 데이터 추출(T1530)의 연쇄 패턴이 반복적으로 나타남

AI 지원 취약점→STIX 변환이 교통 보안 분야의 위협 인텔리전스 자동화와 방어 우선순위 결정에 기여할 수 있음을 시사

논의 및 시사점

실무적 함의

  • LLM이 CVE→STIX 변환의 보조 도구로 실질적으로 활용 가능
  • SDO/CWE는 자동화 신뢰도 높음, SRO는 인간 검토 필요
  • CAV 도메인 특화 프롬프트 설계가 성능 핵심
  • 공발현 패턴 분석으로 방어 우선순위 자동화 가능

남은 과제

  • ATT&CK 전체 매핑 정확도 향상 필요
  • 할루시네이션(허위 객체/관계 생성) 감소
  • 멀티 에이전트 간 조율 메커니즘 개선
  • 폐쇄형 모델(GPT-4 등)과의 비교 연구 부재

"LLM 생성 STIX를 신뢰할 수 있는 보안 분석에 사용하려면, 도메인 특화 Ground-Truth 데이터셋에 기반한 체계적인 평가가 필수적입니다."

결론 및 향후 연구

CAV-STIXGen 데이터셋 구축 — CAV 관련 CVE를 STIX SDO, SRO, CWE, ATT&CK 매핑으로 수동 주석 후 Ground-Truth JSON 변환

11개 오픈 웨이트 LLM 종합 평가 — SDO F1 0.94, CWE F1 0.99 달성, SRO와 ATT&CK는 개선 여지 존재

멀티 에이전트 설정 실험 — Gemma-4-31B SDO 0.91, Codestral-22B SRO 0.43 달성

ATT&CK 공발현 패턴 분석 — 초기 접근→실행→데이터 추출의 반복 패턴 식별

향후 연구 방향

  • 폐쇄형 LLM과의 비교 평가
  • 파인튜닝 기반 도메인 적응 연구
  • 데이터셋 규모 확장 및 다른 도메인(의료, IoT 등)으로 일반화
  • 실제 보안 워크플로우에 통합한 사용자 연구
