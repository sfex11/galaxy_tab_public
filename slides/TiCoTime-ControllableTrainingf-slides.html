문제 정의
왜 시간 제어가 필요한가?
🗣
실시간 음성 시스템
음성 비서와 대화형 에이전트는 응답 시간 제약이 필수적입니다. "15초 안에 답해줘"와 같은 지시를 따라야 합니다.
⚠️
기존 모델의 한계
현재 상용 및 오픈소스 SDM은 시간 인식 능력이 없어 목표 길이를 맞추지 못합니다.
💬
텍스트 ≠ 음성
단어 수는 실제 발화 시간의 근사치일 뿐입니다. 같은 단어도 음소 구성과 운율에 따라 시간이 달라집니다.
🎯
핵심 목표
응답 품질은 유지하면서 목표 시간에 맞는 음성 응답을 생성하는 것이 과제입니다.
핵심 아이디어
Spoken Time Markers (STM)
생성 과정 중간에 누적 발화 시간을 나타내는 특수 토큰을 삽입하여
모델이 실시간으로 시간을 추적하고 남은 내용을 조절합니다
한국의 수도는 서울이며, <3.2초> 인구는 약 천만 명이 넘는 대도시입니다. <6.8초> 서울은 경복궁, 남산타워 등 다양한 관광지가 있고, <10.6초> 현대적인 도시와 전통 문화가 조화를 이루고 있습니다. <14.2초>

"STM은 중간 의미 계획(z)과 실현된 음성 응답(y_sp) 사이의 이산화된 정렬 신호 역할을 합니다."

  • 문장 부호(쉼표, 마침표 등) 뒤에 마커 삽입
  • 누적 시간을 추적하여 목표 시간까지 내용 조절
  • 음성 합성 전 정규식으로 마커 제거

주요 인사이트

단-장시간 모두 안정적
대부분의 SDM은 단시간에서 상대 오차가 크지만, TiCo는 모든 범위에서 낮은 오차를 유지합니다.
🚀
외삽 능력
학습 시 최대 41초였지만, 추론 시 45-60초 범위까지 성공적으로 외삽합니다.
💰
품질 유지
GPT-score 3.56으로 기본 모델(3.57)과 동등한 응답 품질을 유지합니다.
⚡
효율적 학습
4대의 A6000 GPU로 하루 이내에 전체 2단계 파이프라인 학습이 완료됩니다.

"캐스케이드 시스템은 강력한 LLM을 사용하더라도 텍스트 수준의 시간 계획만으로는 불충분합니다. 최종 시간은 발화 속도와 음성 실현에 의존하기 때문입니다."

기여
TiCo의 주요 기여
  • STM (Spoken Time Markers) - 음성 생성 과정에서 실시간 시간 추적을 가능하게 하는 새로운 메커니즘 제안
  • 2단계 학습 파이프라인 - SFT로 시간 인식을 부여하고, GRPO+CHORD로 정밀한 시간 제어를 학습하는 효율적 방법
  • TiCo-Bench - 음성 및 텍스트 질의, 단-장시간 범위를 포괄하는 1,440개 샘플의 종합 벤치마크 구축
  • 최소 데이터 학습 - 단 4,000개의 음성 질문만으로 추가 QA 데이터 없이 학습 가능
  • 도메인 간 일반화 - 음성 질의로만 학습했음에도 텍스트 질의에서도 우수한 성능 달성
  • 보상 해킹 방지 - 5가지 보상 함수의 조합으로 안정적인 강화 학습 실현
