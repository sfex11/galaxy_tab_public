베이스라인 대비 절대 향상

추론 시 압축 대비 RL 학습 효과

문제 정의

장기 수행 에이전트의 상호작용 궤적이 컨텍스트 윈도우 한계를 초과하여, 작업 완료 전에 실행이 중단된다.

핵심 과제

  • 소프트웨어 엔지니어링, 터미널 기반 문제 해결 등 장기 작업
  • 도구 출력, 오류 메시지, 부분 해결책이 누적되어 컨텍스트 소진
  • 컨텍스트 길이 확장만으로는 비용 문제와 긴 시퀀스 활용도 저하 해결 불가

자연스러운 해결책

  • 문맥 압축(Context Compaction): 이전 상호작용을 요약하고 압축된 문맥 하에 계속 수행
  • 고정된 컨텍스트 예산 내에서 무한히 긴 수행 가능
  • 하지만 강화 학습에 압축을 통합하는 방법은 미탐구 분야

기존 접근의 한계

추론 시 휴리스틱

기존 압축은 추론 시 허리스틱이나 외부 메모리로 처리

요약 품질이 학습되지 않음

그룹 정규화 불적합

GRPO 등 그룹 기반 방법은 가변 세그먼트 수에 대응 불가

고정 크기 그룹 정규화 가정 위배

시간적 신용 할당 왜곡

세그먼트를 독립 샘플로 처리하면 손실 가중치와 신용 할당 왜곡

동일 궤적 내 세그먼트 길이·시간 거리 불균형

압축이 발생하면 요약이 이후 모든 행동에 사용 가능한 정보를 결정하므로, 작업 성공은 실행 정책뿐 아니라 압축 정책의 품질에도 의존한다.

CompactionRL 개요

PPO 기반 강화 학습 프레임워크로, 학습 가능한 문맥 압축을 롤아웃 수집에 통합

1
롤아웃 수집
고정 컨텍스트 예산 하에 에이전트-환경 상호작용
2
압축 트리거
잔여 예산이 T_comp 미만 시 정책이 요약 생성
3
문맥 재구성
요약 + 최근 상호작용으로 새 트레이스 시작
4
공동 최적화
실행 액션과 요약 액션을 동일한 작업 보상으로 학습

핵심: 요약을 외부 도구가 아닌 학습 가능한 정책의 일부로 만들어, RL 학습 수평을 최대 컨텍스트 길이 이상으로 확장

학습 가능한 문맥 압축

롤아웃 중 압축

  • 잔여 컨텍스트 예산이 임계값 T_comp 미만 시 트리거
  • 정책이 이전 궤적을 요약 토큰으로 출력
  • 요약 + 최근 k턴의 상호작용으로 컨텍스트 재구성
  • 단일 롤아웃이 가변 수의 세그먼트로 분할됨

요약 품질의 영향

  • 요약이 원본 히스토리를 대체하면 이후 모든 정보원이 됨
  • 정보 누락 → 이후 행동 품질 저하 → 최종 보상 감소
  • 따라서 요약 생성을 보상 신호로 학습해야 함
  • 추론 시 허리스틱과 근본적으로 다름

훈련 세그먼트 구조

하나의 롤아웃 → [실행 세그먼트₁, 요약 세그먼트₁, 실행 세그먼트₂, 요약 세그먼트₂, ..., 실행 세그먼트ₙ] → 모든 세그먼트가 동일한 최종 작업 보상 공유

압축 롤아웃 하의 최적화

그룹 기반 방법의 부적합성

GRPO는 완전한 롤아웃 그룹을 고정 크기로 정규화하지만, 압축으로 인해:

  • 롤아웃마다 세그먼트 수가 가변적
  • 그룹 내 샘플 수가 불균형
  • 그룹 상대적 기준선이 왜곡됨

→ 개별 세그먼트를 학습하는 PPO 공식으로 전환

토큰 수준 손실 정규화

세그먼트 간 길이 불균형으로 인한 가중치 편향 해결:

  • 긴 세그먼트가 손실을 지배하는 문제 방지
  • 각 세그먼트의 손실을 토큰 수로 정규화
  • 모든 세그먼트가 균등하게 기여

→ 세그먼트 길이에 무관한 공정한 업데이트

궤적 간 일반화 우위 추정 (Cross-Trajectory GAE)

압축 경계를 넘어 시간적 신용 할당을 보존하는 핵심 기법

문제: 경계에서의 TD 잔차 단절

표준 GAE는 단일 궤적 내에서 연속적인 TD 잔차를 계산:

δ_t = r_t + γV(x_{t+1}) − V(x_t)

압축 경계에서 컨텍스트가 불연속적으로 변하므로, 세그먼트 경계에서 가치 함수의 불일치가 발생

해결: 궤적 간 GAE

  • 크리틱이 세그먼트 단위가 아닌 궤적 수준의 가치를 추정하도록 학습
  • 압축 경계를 넘어서도 보상 신호가 적절히 전파
  • 초기 세그먼트의 요약 품질이 최종 보상과 연결됨
  • 요약 생성 행동에 대한 신용 할당 가능

토큰 수준 손실 정규화 + 궤적 간 GAE의 조합이 압축 롤아웃에서의 안정적인 PPO 학습을 가능하게 함

절제 연구

요약 학습의 효과

요약을 RL로 학습하지 않고 고정 템플릿 사용 시 성능 저하

  • 학습된 요약이 고정 요약보다 작업 관련 정보를 더 잘 보존
  • 요약 품질이 최종 성능에 직결됨을 확인

토큰 수준 손실 + 궤적 간 GAE

각 구성요소의 개별 및 결합 기여도 분석

  • 토큰 수준 손실 정규화: 길이 불균형으로 인한 학습 불안정성 해소
  • 궤적 간 GAE: 압축 경계를 넘는 신용 할당 개선
  • 두 기법의 결합이 최적 성능 달성

압축 행동 분석

학습된 모델은 작업 진행 상황, 발견된 버그, 시도한 해결책 등 작업 완료에 필수적인 정보를 선택적으로 보존하는 요약을 생성함을 관찰

학습 역학 분석

훈련 진행에 따른 변화

  • 초기: 요약 품질이 낮아 압축 후 성능 급감
  • 중기: 요약이 점진적으로 작업 관련 정보를 보존
  • 후기: 압축 세그먼트와 비압축 세그먼트 간 성능 격차 축소

압축 빈도와 성능

  • 압축 임계값 T_comp 조정에 따른 민감도 분석
  • 너무 잦은 압축 → 정보 손실 증가
  • 너무 드문 압축 → 컨텍스트 소진 위험
  • 적절한 범위 내에서 안정적 성능

학습 초기에는 압축이 오히려 독이 되지만, 충분한 RL 훈련 후에는 압축이 없을 때보다 더 나은 성능을 달성함 — 이는 학습된 요약이 원본 히스토리의 노이즈를 제거하는 효과도 가짐을 시사

결론 및 한계

주요 기여

  • 문맥 압축을 장기 수행 에이전트 RL 훈련에 최초로 통합
  • 그룹 기반 → PPO 기반 개별 세그먼트 최적화로 전환
  • 토큰 수준 손실 정규화로 길이 편향 해결
  • 궤적 간 GAE로 압축 경계 신용 할당 보존
  • SWE-bench, Terminal-Bench에서 일관된 향상
  • GLM-5.2 (750B-A40B) 파이프라인에 실제 배치

한계점

  • 코딩 작업에만 평가 → 웹, 일반 에이전트로의 일반화 미검증
  • 압축 임계값 T_comp에 대한 민감도 존재
  • 요약 품질의 자동 평가 메트릭 부재
  • 다중 압축-복원 주기에서의 누적 정보 손실 분석 부족

CompactionRL은 최대 작업 컨텍스트 길이를 늘리지 않고도 에이전트 LLM의 유효 학습 수평을 확장하는 실용적 접근법을 제시한다.
