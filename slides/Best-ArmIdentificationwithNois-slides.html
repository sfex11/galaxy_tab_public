문제 정의

다중 슬롯머신(MAB) 모델

밴딧 모델

  • K개의 팔: a ∈ {0,1,...,K-1}
  • 각 팔의 보상 평균: μa
  • 최적 팔: a* = argmax μa
  • 최적성 격차: Δa = μa* - μa

고정 신뢰도 BAI

  • 목표: Pr(â = a*) ≥ 1 - δ
  • 성능 측정: 물리적 라운드 수 τ
  • 정확한 최적 팔을 높은 확률로 식별

무잡음 벤치마크

Nclean(δ, μ) = Θ(∑a≠a* Δa-2 · log(1/δ))

잡음 없는 이상적 환경에서 필요한 샘플 수

통신 모델

이산 무기억 채널 (DMC)

채널 정의

W: X → Y,   W(y|x) = Pr(Yt = y | Xt = x)

팔 채널 제약: X = Y = K (입출력 알파벳 = 팔 집합)

타자기 채널 예시

Y = X (확률 1-ε) 또는 X+1 mod K (확률 ε)

사례 1

디코딩 없음: ε-의존적 성능 저하

보조정리 1: 비식별 불가능성

만약 혼합 매핑 W가 단사(injective)가 아니라면, 어떤 알고리즘도 모든 인스턴스에 대해 δ-정확성을 달성할 수 없습니다.

∃ μ ≠ μ' : Wμ = Wμ'  but  argmax μi ≠ argmax μ'i

명제 1: 단일샷 + 역혼합 기준선

W가 가역(invertible)이고 σmin(W)를 알 때:

Tbaseline(δ, μ) = Õ(σmin(W)-2 · Nclean(δ, μ))

타자기 채널에서의 의미

σmin(W) = |1 - 2ε|

ε → 1/2일 때 성능이 발산 (짝수 K의 경우)

→ 곱셈적 오버헤드가 채널 잡음에 따라 무한히 증가

6 / 12
사례 2

영-오류 블록 코드: 상수 오버헤드

기법 1: 코드북 기반

블록 길이 nu인 영-오류 코드 사용

τ ≤ nu · τclean

상수 곱셈적 오버헤드 → ε에 무관!

기법 2: 패리티 스케줄

독립 집합 스케줄로 단일샷 영-오류 디코딩

τ ≤ 2 · τclean + α

C6에서 짝수/홀수 집합 교대 활용

보조정리 2: C5에서의 영-오류 코드

길이-2 코드북 φ(i) = (i, 2i mod 5)는 용량-최적 (2,5) 코드를 달성합니다.

보조정리 3: 독립 집합 영-오류 디코딩

인코더가 X ∈ S (독립 집합)로 제한하고 디코더가 S를 알면, 단일 수신으로 영-오류 디코딩 가능

사례 3: PSE 알고리즘

패킷화 순차 제거법 (PSE)

1
초기화: 모든 팔을 활성 집합 S1 = K에 포함, 대기 팔 h 선택
2
설치 단계: 길이 nr인 영-오류 패킷으로 실행 계획 전송
(에이전트는 설치 중 대기 팔 h를 당김)
3
실행 단계: 활성 집합의 각 팔을 mr = 2r-1번 당김
4
제거 단계: UCB/LCB 계산 후 열등한 팔 제거

신뢰 반경

β(t) = √(2 log(8Kt²/δ) / t)

UCBa = μ̂a + β(t)    LCBa = μ̂a - β(t)

핵심 특성: 통신은 페이즈 경계에서만 발생 → 오버헤드가 덧셈적으로 축소!

핵심 정리

정리 1: PSE의 정확성과 복잡도

정리 진술

1-subGaussian 보상을 가정할 때, PSE는 δ-정확하며:

τ ≤ NSE(δ, μ) + ∑r=1R nr

통계적 비용

NSE = Õ(∑a≠a* Δa-2 log(K log(1/Δa)/δ))

무잡음 순차 제거법과 동일한 복잡도

통신 비용

∑r=1R nr   (R = O(log(1/Δmin)))

영-오류 용량에만 의존하는 덧셈적 오버헤드

타자기 채널 응용 (따름정리 2 & 3)

C5
τ ≤ NSE + 6R
C6
τ ≤ NSE + 4R

결론 및 향후 연구

요약 및 전망

주요 결론

  • 잡음 있는 실행 채널에서 최적 팔 식별 문제를 최초로 체계적 분석
  • 영-오류 용량이 ε-의존적 / ε-독립적 성능을 나누는 자연스러운 분계점
  • 에이전트 능력에 따라 질적으로 다른 세 가지 성능 특성 규명
  • 상태 유지 실행(PSE)이 가장 효율적: 오버헤드를 O(R)로 축소

향후 연구 방향

  • 겹치는 독립 집합 스케줄의 체계적 설계
  • 타자기 채널 너머 일반적 DMC로의 확장
