스킬 생성

런타임 루프 내 skill_create 툴 호출로 생성–사용 불일치 해소

다층 메모리

단기·장기 기억에 더해 스킬별 기억 도입

평가 서브시스템

단위 테스트 + 실행 피드백으로 신뢰성 보장, 실패 시 자동 정제

컨텍스트 관리

적응적 압축 + 세션 간 상태 지속성으로 긴 과제 대응

스킬 카탈로그

파일 시스템 기반 구조화 저장, 메타데이터 기반 검색

에이전트 간 전이

생성된 스킬을 다른 에이전트에 주입하여 이식 가능한 지식 자산으로 활용

다층 메모리 시스템

단기 기억

현재 세션의 대화 이력과 관찰 결과. 적응적 압축으로 컨텍스트 윈도우 초과 방지

장기 기억

세션 간 지속되는 에이전트 수준의 경험. 파일 시스템에 구조화 저장

스킬별 기억 핵심 제안

각 스킬이 독자적으로 축적하는 경험 로그. 성공/실패 사례, 엣지 케이스, 정제 이력을 보관하여 향후 호출 시 활용

스킬별 기억은 개별 스킬이 "자신이 어떤 상황에서 잘 작동하고 어떤 상황에서 실패했는지"를 기억하게 하여, 단순한 코드 스니펫을 경험을 갖춘 지능적 자산으로 변환합니다.

스킬 생성과 평가·정제 루프

생성 과정

  • 에이전트 런타임 루프 내에서 skill_create 툴 호출
  • 성공적인 실행 궤적을 역추적하여 스킬 추출
  • 스킬 패키지: 지시문 + 코드 + 메타데이터 + 단위 테스트
  • 카탈로그에 자동 등록 및 색인

평가·정제 순환

  • 스킬 호출 전 단위 테스트 자동 실행
  • 런타임 실행 피드백 수집
  • 테스트 실패 또는 실행 오류 시 자동 정제 트리거
  • 정제 결과를 스킬별 기억에 기록

이 순환을 통해 스킬은 일회성 생성물이 아닌 테스트 가능하고 진화하는 인프라가 됨

컨텍스트 관리 전략

적응적 압축

컨텍스트 윈도우 임계값 도달 시 자동 트리거

  • 단순 truncate가 아닌 의미 보존 압축
  • 중요도 기반 우선순위 할당
  • 감사 추적(audit trail) 유지

세션 간 상태 지속성

세션이 끝나도 에이전트 상태를 보존

  • 파일 시스템 기반 구조화 저장
  • 에이전트 홈 디렉토리 + 세션 워크스페이스
  • 샌드박스 내 안전한 실행 환경

"왜 그냥 truncate하지 않는가?" — 단순 절단은 과제 완료에 필요한 결정적 정보를 잃게 하며, 압축 알고리즘은 정보 손실을 최소화하면서 비용을 절감함

핵심 메시지

스킬을 모델 가중치 안에 갇힌 불투명한 행동이 아닌, 테스트 가능하고 전이 가능하며 경험을 축적하는 외부화 인프라로 만들어야 한다.

에이전트가 스스로 스킬을 작성, 검증, 정제하고, 그 스킬이 다른 에이전트에도 이식될 수 있다면 — 이는 모놀리식 모델 가중치에서 모듈형 지식 인프라로의 패러다임 전환을 의미합니다.
