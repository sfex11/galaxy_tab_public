문제 정의

추론 시간 스케일링의 비용 문제

Self-Consistency (SC)

LLM에서 여러 후보 답변을 샘플링하고 가장 빈도가 높은 답변을 선택하는 기본 방식

  • 단순한 다수결 투표
  • 추가 LLM 호출 불필요

CISC ("Think Twice")

Critic LLM이 각 후보의 신뢰도 점수를 평가하고 가중 다수결 투표 수행

  • SC보다 높은 정확도
  • 후보마다 추가 LLM 호출 필요

CISC는 모든 후보 답변에 대해 Critic을 호출하므로 추론 오버헤드가 크게 증가하며, 환각·퇴화 샘플까지 구분 없이 평가합니다.

핵심 아이디어

VecCISC: 의미적 유사성 기반 후보 필터링

추론 경로의 임베딩을 클러스터링하여 중복·퇴화·환각 샘플을 제거하고, 대표 샘플만 Critic에 전달

47%

평균 토큰 사용량 감소

≥CISC

정확도 유지 또는 향상

경량 임베딩 모델만 추가

의미적으로 동등한 추론 경로를 하나로 묶고, 클러스터 내에서 가장 중심적인 추론 경로를 대표로 선택합니다.

방법론

단계 1–3: 샘플링, 임베딩, 그룹화

1

샘플링

질문 q에 대해 LLMgen에서 n개의 (추론 경로 ri, 답변 ai) 쌍을 샘플링하여 후보 풀 P에 저장

2

임베딩 생성

각 추론 경로 ri를 임베딩 모델 Emb을 통해 d차원 벡터로 변환. 의미적 세부 정보를 포착

3

답변별 그룹화

동일한 답변 a를 도출한 임베딩들을 그룹 Ga로 묶어 고유 답변 집합을 보존

답변별로 그룹화하는 이유: 서로 다른 답변 간의 클러스터링은 의미가 없으며, 동일 답변 내에서의 의미적 중복을 제거하는 것이 목표입니다.

방법론

단계 4–5: 클러스터링과 대표 선택

클러스터링

각 그룹 Ga 내에서 K개의 클러스터를 형성합니다.

  • KMeans 및 HAC 분석
  • DBSCAN 불사용 (고차원 공간에서 임계값 D 선정 어려움)
  • 데이터셋·모델별 K값 그리드 서치

대표 선택

클러스터의 중심(centroid)에 가장 가까운 추론 경로를 대표로 선택합니다.

  • 코사인 유사도를 거리 척도로 사용
  • 벡터 각도 기반이므로 고차원 공간에 적합
  • 가장 의미적으로 이상적이지 않은 경로 제외 효과

핵심 통찰: 클러스터 중심에 가장 가까운 추론 경로가 이상적인 추론 오류를 포함할 확률이 가장 낮다는 관찰에 기반합니다.

방법론

단계 6–7: 신뢰도 평가와 최종 답변 선택

신뢰도 평가

각 답변 a의 대표 추론 경로들만 Critic LLM에 전달하여 0~1 범위의 신뢰도 점수를 획득합니다.

프롬프트 구성: [지시문] ∘ 질문 ∘ 추론경로 ∘ 답변

Softmax 정규화 (온도 T)를 적용하여 점수를 정규화합니다. T값도 그리드 서치로 선정합니다.

가중 다수결 투표

정규화된 신뢰도 점수를 가중치로 사용하여 최종 답변을 선택합니다.

Afinal = arg maxa Σ [aj = a] · ĉa,j

동일 답변의 여러 대표 경로 점수가 합산되므로, 클러스터가 많은 답변이 유리합니다.

실험 결과

정확도 성능

VecCISC는 CISC 대비 정확도를 유지하거나 초과하는 성능을 보입니다

주요 관찰

  • 대부분의 (데이터셋, 모델) 조합에서 CISC와 동등 이상
  • 일부 경우 CISC 대비 정확도 향상 — 퇴화 샘플 제거 효과
  • SC보다 항상 우수한 정확도
  • Random-K보다 일관되게 높은 정확도

클러스터링 방법 비교

  • KMeans와 HAC 모두 유효
  • Random-K 대비 클러스터링 기반 선택이 체계적으로 우수
  • 단순 무작위 축소보다 의미적 필터링의 중요성 입증

퇴화/환각 샘플이 Critic에 도달하기 전에 클러스터링으로 사전에 분리되어, Critic이 더 높은 품질의 샘플에 집중할 수 있습니다.

인사이트

클러스터링의 부가 효과

의미적 중복 제거

동일한 의미의 추론 경로가 여러 번 샘플링되는 경우가 빈번합니다. 클러스터링은 이를 하나로 묶어 불필요한 중복 평가를 방지합니다.

→ Appendix A.3: 유사 의미 경로 클러스터링 사례

퇴화 경로 격리

결함이 있는 추론, 환각, 퇴화된 응답은 임베딩 공간에서 고유한 클러스터를 형성하는 경향이 있습니다. 이를 분리하여 Critic 평가 대상에서 제외할 수 있습니다.

→ Appendix A.4: 퇴화 경로 격리 사례

이러한 부가 효과는 단순한 비용 절감을 넘어, Critic의 평가 품질 자체를 향상시키는 메커니즘으로 작동합니다.

한계점

논의 및 한계

고려사항

  • 클러스터 수 K와 온도 T에 대한 그리드 서치 필요 — 추가 검증 비용 발생
  • 임베딩 모델의 품질이 클러스터링 품질에 직접 영향
  • 매우 복잡한 추론 경로에서는 의미적 유사성 포착이 어려울 수 있음

확장 가능성

  • 다른 "Think Twice" 방법론에도 범용 적용 가능
  • 더 큰 생성 모델(GPT-4 등)과의 결합 미탐색
  • 클러스터 수 K의 자동 결정 메커니즘 연구 필요

코드는 공개 예정으로, 향후 연구에서의 재현 및 확장을 지원합니다.

결론

요약 및 기여

VecCISC 제안 — 추론 경로 클러스터링을 통해 "Think Twice" 패러다임의 비용을 획기적으로 절감하는 경량 프레임워크

광범위한 실험 검증 — 5개 도메인, 4개 모델에서 전체 토큰 사용량 47% 감소하면서 CISC와 동등 이상의 정확도 달성

클러스터링 방법 비교 — KMeans, HAC 모두 Random-K보다 일관되게 우수한 성능 입증

공개 계획 — 코드 공개를 통해 재현성 및 후속 연구 지원

"더 많이 평가하는 것이 아니라, 더 잘 선택하여 평가하는 것이 핵심입니다."
