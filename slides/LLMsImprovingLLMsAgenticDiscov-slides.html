기존 Test-Time Scaling의 한계

수동 설계 의존

연구자가 분기, 가지치기, 중단 등의 휴리스틱을 직관에 의해 직접 설계하고 임계값을 조정

탐색 공간 축소

연산 할당 공간의 극히 일부만 수동으로 탐색되며, 다수의 잠재적 전략이 미탐색 상태로 남음

단편적 접근

각 방법이 너비 또는 깊이 축의 일부만 개별적으로 조정하며, 통합적 관점 부재

반복적 비용

새로운 전략을 평가할 때마다 LLM을 반복적으로 호출하여 높은 비용 소모

핵심 통찰: TTS를 알고리즘 탐색으로 재정의

기존 TTS 방법들은 너비–깊이 할당 공간에서의 수동으로 지정된 정책으로 해석될 수 있다.

각 방법은 공간에서 고유한 궤적을 그린다:

  • SC@64 — 고정된 전체 예산 코너 점유
  • ASC / ESC — 최대 깊이에서 너비 축만 적응
  • Answer Consistency — 단일 체인에서 깊이 축만 적응
  • ST-BoN — 넓게 확장 후 하나로 가지치기, 이후 심화
  • Parallel-Probe — 넓게 시작하며 점진적으로 가지치기+심화

AutoTTS 프레임워크 개요

연구자의 역할을 전략 설계에서 환경 구성으로 전환

1
환경 구성
상태, 행동, 피드백, 목표를 정의하여 발견 환경 설계
2
에이전트 탐색
탐색 LLM이 후보 컨트롤러 반복적으로 제안
3
오프라인 평가
사전 수집된 궤적으로 저비용·결정론적 평가
4
피드백 기반 개선
스케일링 곡선 + 실행 추적으로 실패 원인 진단

핵심: 발견 환경이 제어 공간을 다루기 쉽게 만들고, 저렴하고 빈번한 피드백을 제공해야 함

리플레이 환경 구성

오프라인 데이터 수집

  • 각 문제에 대해 사전에 추론 궤적 수집
  • 고정 길이 생성 구간별 중간 프로브 신호 기록
  • 한 번의 수집으로 여러 컨트롤러 재사용 가능

오프라인 리플레이 평가

  • 컨트롤러가 사전 수집 궤적 위에서 의사결정 리플레이
  • LLM 반복 호출 없이 저비용·결정론적 평가
  • 정확도–비용 트레이드오프로 후보 비교

컨트롤러의 상태–행동 공간

너비–깊이 TTS를 5가지 행동으로 형식화

행동 설명 비용
BRANCH 새로운 추론 분기 생성 후 첫 구간 진행 1 구간
CONTINUE(i) 활성 분기 i를 한 구간 더 심화 1 구간
PROBE(i) 분기 i의 현재 프로브 신호 확인 (진행 없음) κ_probe
PRUNE(i) 분기 i를 활성 집합에서 제거 0
ANSWER 추론 종료 후 집계 규칙으로 최종 답변 생성 0

상태: (문제, 분기 수, 활성 분기 집합, 각 분기 깊이, 공개된 프로브 피드백 집합, 잔여 예산)

발견 루프: 에이전트 주도 제안

1
컨트롤러 제안

탐색 LLM이 코드 형태로 후보 컨트롤러 작성 (조건문, 임계값 포함)

2
히스토리 구성

이전 제안, 평가 결과, 실행 추적을 구조화된 히스토리로 정리

3
컨트롤러 선택

Pareto 프론티어 기반으로 탐색 집합에서 최적 컨트롤러 선택

4
반복 개선

피드백을 바탕으로 다음 라운드에서 개선된 제안 생성

베타 매개변수화: 탐색 공간 축소

문제

자동 발견된 컨트롤러가 과도한 하이퍼파라미터를 도입하여 탐색 공간이 기하급수적으로 확장되고, 제한된 라운드 내에서 과적합 위험 증가

해결: β 매개변수화

컨트롤러가 단일 스칼라 파라미터 β만 노출하고, 모든 내부 하이퍼파라미터를 β로부터 결정론적으로 도출

β

단일 스칼라 → 모든 임계값 결정론적 도출
탐색 집합 과적합 방지 · 미세 조정 가능한 정확도–비용 트레이드오프

실행 추적 피드백: 실패 진단

스칼라 정확도–비용 피드백만으로는 컨트롤러가 왜 실패했는지 알 수 없다.

스케일링 곡선 피드백

다양한 예산 수준에서의 정확도 변화를 곡선으로 제공하여 전체적인 트레이드오프 패턴 파악

실행 추적 피드백

컨트롤러가 시간에 따라 연산을 어떻게 할당했는지 상세히 기록하여 구체적 실패 모드 식별

예: "예산을 초반에 모두 소모하여 후반 프로브 불가"와 같은 원인 수준 진단이 가능해짐

베이스라인

모델

평가 지표: 정확도–비용 Pareto 프론티어 (다양한 예산 수준에서의 정확도)

주요 결과: Pareto 프론티어 개선

  • AutoTTS가 발견한 컨트롤러가 모든 강력한 수동 베이스라인보다 정확도–비용 트레이드오프를 개선
  • 동일 비용에서 더 높은 정확도 달성, 또는 동일 정확도에서 더 낮은 비용 달성
  • 탐색에 사용된 벤치마크에서 학습된 전략이 검증용 벤치마크로 일반화
  • Qwen-1.7B에서 발견된 전략이 다른 모델 규모로도 전이

수동 설계 전략들은 너비–깊이 공간에서 각각 특정 궤적만 탐색하지만,

저비용의 핵심 원인

  • 오프라인 리플레이: 사전 수집된 궤적을 재사용하여 LLM 반복 호출 제거
  • 결정론적 평가: 동일한 컨트롤러는 항상 동일한 결과를 반환
  • β 매개변수화: 탐색 공간 축소로 필요한 라운드 수 감소
  • 일회성 투자: 발견된 컨트롤러는 이후 무한히 재사용 가능

핵심 시사점

패러다임 전환

TTS 전략 설계에서 환경 설계로 인간의 역할을 이동. 올바른 환경만 구성하면 에이전트가 전략을 자동 발견

재사용성

발견된 전략은 타 벤치마크와 모델 규모로 일반화되며, 소량의 일회성 비용으로 영구적 가치 창출

확장 가능성

너비–깊이 공간은 개념 증명이며, 트리 탐색·검증자 기반 정제 등 더 풍부한 구조로 확장 가능

LLM 자가 개선

LLM이 자신의 추론 전략을 개선하는 메타 루프로, "LLM이 LLM을 개선하는" 자기 참조적 개선 패러다임 실현
