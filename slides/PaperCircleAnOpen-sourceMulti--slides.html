01 — 문제 정의

연구자들이 직면한 도전

과학 문헌의 폭발적 증가로 인해 연구자들이 관련 논문을 효율적으로 발견하고, 평가하고, 종합하기가 점점 어려워지고 있습니다.

🔍
논문 발견의 어려움
방대한 학술 문헌 속에서 관련 논문을 효율적으로 검색하기가 점점 더 힘들어지고 있음
📈
평가 및 종합의 한계
개별 논문의 기여도 평가와 여러 연구 결과의 종합적 이해가 시간 소모적
🔗
통합 시스템 부재
기존 도구들은 논문 발견과 심층 분석을 하나의 프레임워크에서 제공하지 못함
📄
구조화된 지식 부족
비정형 PDF를 구조화된 지식 그래프로 변환하는 자동화된 방법이 필요
"단일 소스로는 검색할 수 없는 논문이 전체의 36.9% (Google Scholar) ~ 70.9% (arXiv)에 달합니다."
02 — 솔루션 개요

Paper Circle 아키텍처

Paper Circle은 발견(Discovery)과 분석(Analysis) 두 개의 상호보완적 멀티에이전트 파이프라인으로 구성됩니다.

🔎
발견 파이프라인
오프라인/온라인 검색 통합
다중 기준 점수화 & 다양성 인식 랭킹
6개 전문 에이전트 활용
4개 학술 데이터 소스 연동
🔬
분석 파이프라인
PDF → 구조화된 지식 그래프 변환
유형별 노드: 개념, 방법, 실험, 시각자료
4단계 추출 프로세스
그래프 기반 Q&A 및 커버리지 검증
기술 스택
smolagents 프레임워크 FastAPI + Supabase React + TypeScript TailwindCSS + Vite CodeAgent (CoA) ToolCallingAgent (ToCA)
03 — 발견 파이프라인

6개 에이전트 기반 논문 발견

🧠
의도 분류 에이전트
사용자 쿼리를 구조화된 제약조건으로 파싱 (검색 모드, 컨퍼런스 필터, 연도 범위, 랭킹 선호도 추출)
🔍
논문 검색 에이전트
오프라인/온라인 검색 실행, arXiv · Semantic Scholar · OpenAlex · DBLP 통합, 중복 제거 수행
📊
정렬 에이전트
최신성, 인용수, 유사도, 참신성, BM25 점수의 가중 조합 + 교차 인코더 재랭킹
📈
분석 에이전트
집계 통계 계산 — 출처 분포, 연도 추세, 상위 저자, 학회별 분석, 키워드 빈도
📦
내보내기 에이전트
동기화된 다형식 출력: JSON, CSV, BibTeX, Markdown, HTML 대시보드
🌐
웹 검색 에이전트
보조 온라인 조회 도구 — 최신 논문 및 추가 정보 검색 지원
04 — 점수화 및 랭킹

다중 기준 점수화 시스템

5가지 점수 차원을 가중 결합하여 최적의 논문 순위를 도출합니다.

💰
유사도 점수
TF-IDF 코사인 유사도로 쿼리와 논문 텍스트 간 의미적 유사성 측정
📅
최신성 점수
출판 연도 기반 점수 — 최신 논문에 가중치 부여
💡
참신성 점수
제목 토큰 빈도 분석 기반 독창적인 연구 탐지
📖
BM25 점수
Okapi BM25 알고리즘 기반 어휘적 관련성 랭킹
📐
인용 수
원시 인용 지표로 학술적 영향력 반영
🎯
MMR 다양성
최대 한계 관련성(MMR)으로 관련성과 다양성 균형 확보
추가 뷰: "숨겨진 보석" (관련성 높으나 인용 낮은 논문), "정전 논문" (고인용), "소스 분포" 뷰 제공
05 — 분석 파이프라인

PDF → 지식 그래프 변환

4단계 프로세스를 통해 비정형 PDF를 구조화된 쿼리 가능한 지식 그래프로 변환합니다.

1
PDF 수집 및 청킹
PyMuPDF 기반 추출 → 계층적 섹션 인식 → SemanticChunker로 구조 정렬 청크 생성 (기본 1500자)
▼
2
지식 그래프 스키마 정의
노드 유형: 논문, 섹션, 개념, 방법, 실험, 데이터셋, 시각 요소 / 엣지 유형: 계층, 정의, 제안, 사용, 평가, 설명, 의존성
▼
3
멀티에이전트 추출
4개 전문 추출기 순차 실행: 개념 → 방법 → 실험 → 시각 요소 연결 → 개념 간 관계 매핑
▼
4
Q&A 및 검증 레이어
벡터 검색 + 그래프 순회 결합 / CoverageChecker로 그림, 표, 섹션, 수식 커버리지 검증
06 — 리뷰 프레임워크

7개 전문 리뷰 에이전트

NeurIPS/ICML 수준의 체계적 논문 리뷰를 자동으로 생성합니다.

🤖 심층 분석기
수학적 기초, 방법론 구성요소, 핵심 발견에 대한 기술적 핵심 분석
⚖️ 비평가
시니어 리뷰어 에뮬레이션 — 강점/약점 평가, 저자 질문 생성, 참신성·명확성·중요도 점수화
📚 문헌 전문가
연구 분야 내 위치 매핑, Semantic Scholar/arXiv 연동, 인용 검증
🏆 기여도 분석기
명시적 주장과 검증된 기여도 분리, 과대 주장 및 누락된 베이스라인 식별
🔧 재현성 검사기
소스 코드 가용성, 하이퍼파라미터 명세, 데이터셋 접근성, 계산 요구량 평가
📝 요약기
다양한 추상화 수준의 다중 충실도(multi-fidelity) 요약 생성
🎓 오케스트레이션 관리자
다단계 파이프라인을 통한 전체 생명주기 조율 — ThreadPoolExecutor를 활용한 병렬 실행 지원

11 — 한계 및 미래

한계점 및 향후 연구

⚠ 리뷰 점수 예측 한계
인간 판단과의 상관관계가 낮음 (r < 0.25) — 논문 순위 매기기의 신뢰할 수 있는 메커니즘으로 사용 불가
⚠ 모델 의존성
최적 성능을 위해 대형 오픈/클로즈드 소스 모델 필요 — 코드 특화 모델은 일관된 리뷰 워크플로에 취약
💡 코드 특화 모델 한계
Qwen3-Coder 변형은 기술적 차원에서 합리적이나 전체 평점에서 더 큰 오류를 보임
🔬 향후 연구 방향
파이프라인 통합 최적화에 집중 — 발견과 분석 파이프라인의 더 긴밀한 연결 추진
"전체적으로 리뷰 품질은 더 큰 모델에서 향상되며, 이는 용량과 지시 따르기 능력이 중요함을 시사합니다."
핵심 시사점
➤ 채팅 스타일 LLM이 코드 지향 모델보다 일관된 리뷰를 생성
➤ 멀티소스 접근이 단일 소스 대비 검색 커버리지를 크게 향상
➤ BM25 기반 어휘적 매칭이 여전히 강력한 기준선으로 작용
➤ 하드웨어: 4× 40GB Nvidia GPU, Ollama + fastllm 라이브러리 활용
