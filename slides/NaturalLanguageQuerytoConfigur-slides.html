Natural Language Query to Configuration for Retrieval Agents

Melissa Z. Pan, Negar Arabzadeh, Mathew Jacob, Fiodar Kazhamiaka, Esha Choukse, Matei Zaharia

UC Berkeley · University of Washington · Microsoft Azure Research

문제 배경: 복잡해진 검색 파이프라인

현대 AI 파이프라인은 단일 LLM 호출을 넘어 복잡한 검색·합성 파이프라인으로 진화했습니다. 고객 지원, 연구 보조, 웹 검색 엔진 등 다양한 도메인에서 활용되며, 각 단계마다 수많은 설정 옵션이 존재합니다.

LLM 선택

모델 종류, 크기, 비용

리트리버 설정

검색기 종류, 문서 수

합성 전략

홉 수, 요약 방식, 에이전트 루프

현재 이러한 파이프라인은 워크로드 단위로 수동 튜닝되며, 질의별 최적화의 여지가 크게 남아 있습니다.

관찰 1: 워크로드 내 질의별 분산

동일한 워크로드(동일 코퍼스上的 질의 집합) 내에서도 각 질의마다 최적 설정이 다릅니다. 단일 설정으로 전체 워크로드를 커버할 수 없습니다.

단순 질의 예시

"해당 프로그램의 감독 이름은?"

→ 저비용 파이프라인으로도 충분

낮은 복잡도

복잡 질의 예시

"A회사가 인수한 기업의 CEO가 이전에 근무했던 ..."

→ 고비용 멀티홉 파이프라인 필요

높은 복잡도
~3개 오더 단일 워크로드에서 파이프라인 비용 편차 범위

관찰 2 & 3: 전체 파이프라인과 워크로드 특성

관찰 2

LLM만 고르면 충분할까?

아닙니다. FinanceBench 실험에서 합성 방식과 리트리버를 함께 변경하는 것이 LLM만 바꾸는 것보다 훨씬 더 넓은 비용-품질 범위를 커버합니다.

LLM 라우팅만으로는 최적화 여지의 일부만 활용

관찰 3

범용 특성의 한계

기존 방법은 need_joint_reasoning 같은 소수의 범용 특성을 사용합니다. 하지만 이는 워크로드 내 대부분 질의를 동일 값으로 매핑하여 구분력이 부족합니다.

예: "지역 요리 관련", "인물 관련" 등 워크로드 특화 특성 필요

핵심 시사: 전체 파이프라인을 대상으로, 워크로드 특화 특성을 기반으로 질의별 설정 선택이 필요하다.

Query2Conf: 문제 정식화

자연어 질의와 정확도 목표(또는 비용 예산)가 주어졌을 때, 사전 정의된 파이프라인 카탈로그에서 비용을 최소화(또는 정확도를 최대화)하는 설정을 추론 시점에 동적으로 선택합니다.

입력

  • 자연어 질의 q
  • 정확도 목표 또는 비용 예산
  • 사전 프로파일링된 설정 카탈로그

출력

  • 최적 파이프라인 설정 c*
  • 목표 충족 시 최소 비용 보장
  • λ 조절로 비용-품질 트레이드오프 제어

선택 규칙: argmax_c [ p̂_c(𝒻(q)) − λ · cost(c) ] — λ ≥ 0을 조정하여 파레토 프론티어 상의 임의 지점 도달 가능

BRANE 프레임워크 개요

1
설정 프로파일링
소수 질의를 모든 후보 설정에서 실행하여 정확도·비용 기록
2
특성 추출 & 예측기 학습
LLM이 워크로드 특화 이진 특성 제안 → 전체 질에 라벨링 → 설정별 경량 예측기 학습
3
추론 시 설정 선택
신규 질의 특성화 → 파레토 설정 스코어링 → λ 기반 최적 설정 선택

핵심 아이디어 1

LLM이 질의를 워크로드 특화 이진 특성 𝒻(q)으로 변환 — 자연어 의미와 파이프라인 공간 사이의 번역 계층 역할

핵심 아이디어 2

설정별 경량 예측기 p̂_c가 ℙ(정답 | 𝒻(q)) 추정 — 재학습 없이 λ 조절만으로 목표 전환 가능

워크로드 특화 질의 특성화

범용 특성(예: requires_multihop)은 워크로드 내 구분력이 부족합니다. BRANE은 프론티어 LLM이 소수 질의 샘플에서 워크로드 특화 이진 특성을 자동 제안하고, 저비용 LLM이 전체 질의에 라벨링합니다.

범용 특성 예시 (구분력 낮음)

  • requires_multihop_reasoning
  • requires_numerical_answer
  • involves_temporal_comparison

→ 워크로드 내 대부분이 동일 값

워크로드 특화 특성 (구분력 높음)

  • involves_regional_cuisine
  • requires_financial_ratio_calc
  • involves_people_relationships
  • references_specific_regulation

→ 워크로드 구조를 반영한 미세 분류

프론티어 LLM이 특성 제안 → 저비용 LLM이 라벨링 → 퍼지 파레토 가지치기로 설정 수를 1/10 수준으로 축소

예측기 학습 & 퍼지 파레토 가지치기

설정별 경량 예측기

파레토 프론티어 상의 각 설정 c에 대해 독립적인 분류기 학습

  • 입력: 질의의 이진 특성 벡터 𝒻(q)
  • 출력: 정답 확률 p̂_c(𝒻(q))
  • 학습 데이터: 오프라인 프로파일링 결과

퍼지 파레토 가지치기

전체 설정(예: 60개) 중 파레토 우위 설정만 선별

  • 엄격 파레토 대신 퍼지 임계값 적용
  • 예측기 수를 약 1/10로 축소
  • 학습·추론 비용 동시 절감

추론 시 재학습 없이 λ ≥ 0 하나만 조절하면 비용 절约 모드 → 최고 정확도 모드까지 자유롭게 이동 가능

실험 설정

벤치마크

  • MuSiQue — 멀티홉 추론
  • BrowseComp-Plus — 웹 검색
  • FinanceBench — 재무 질의

설정 공간

  • LLM: 4종 (GPT-4o 등)
  • 리트리버: 3종
  • 문서 수: 3단계
  • 합성 방식: 3종
  • 총 60개 설정 프로파일링

비교 대상

  • 최고 고정 설정 (Murakkab)
  • Carrot (LLM 라우터)
  • METIS (규칙 기반)
  • Adaptive-RAG (T5 라우터)
  • Fine-tuned BERT / Qwen3-4B
526 프로파일링 트레이스 전체 공개 (벤치마크당 150~600 질의)

주요 결과: 비용-품질 파레토 프론티어

BRANE은 세 벤치마크 모두에서 비용-품질 파레토 프론티어를 밀어붙이며, 고정 설정 및 모든 베이스라인을 지배합니다.

벤치마크 최고 고정 설정 대비 Carrot 대비 Fine-tuned Qwen 대비
MuSiQue 동일 정확도 at 89% 비용 절감 우위 우위
BrowseComp-Plus 정확도 +0.7% at 81.7% 비용 절감 우위 우위
FinanceBench 동일 정확도 at 91% 비용 절감 우위 우위

BRANE의 질의별 파레토 궤적(노란 다이아몬드)이 고정 파레토 프론티어(검은 점선)를 전 비용-품질 범위에서 지배

절제 실험: 워크로드 특화 특성 vs 임베딩

LLM이 제안한 워크로드 특화 이진 특성이 범용 임베딩을 예측기 입력으로 사용하는 것보다 모든 벤치마크에서 우수합니다.

LLM 제안 이진 특성 승리

  • 자연어 → 파이프라인 공간의 효과적 변환
  • 워크로드 구조를 직접 반영
  • 희소 이진 벡터로 계산 효율적

범용 임베딩 열위

  • 질의의 대부분 정보가 설정 선택과 무관
  • 의미적으로 유사한 질의라도 최적 설정이 다름
  • 고차원 밀집 벡터의 노이즈 문제

추가 발견: 특성 수 증가에 따른 수익 체감 — 약 15~20개 특성에서 수렴

절제 실험: 고전적 예측기 vs 엔드투엔드 파인튜닝

놀랍게도 설정별 경량 고전적 분류기가 엔드투엔드 파인튜닝된 대형 모델보다 일관되게 우수합니다.

방법 아키텍처 학습 데이터 요구량 성능
BRANE (본 논문) 설정별 경량 분류기 수백 개 프로파일링 최고
Fine-tuned BERT 판별적 멀티레이블 10⁴~10⁵ 라벨 열위
Fine-tuned Qwen3-4B 생성적 엔드투엔드 10⁴~10⁵ 라벨 열위

이유: 이진 특성이 이미 의미적 추상화를 수행하므로, 예측기는 단순한 패턴 매칭만 수행하면 충분

BRANE은 전체 파이프라인 설정 공간을 대상으로 질의별 동적 선택을 수행하는 최초의 프레임워크입니다.

한계점 및 확장 방향

한계점

  • 오프라인 프로파일링 비용 발생
    (60설정 × 수백 질의 ≈ $11,000)
  • 사전 정의된 설정 카탈로그 내에서만 선택 가능
  • 새로운 워크로드 적용 시 추가 프로파일링 필요
  • 코퍼스나 설정 공간 변화에 대한 적응 비용

확장 가능성

  • 전이 학습으로 프로파일링 비용 감소
  • 설정 공간 자동 탐색과 결합
  • 온라인 학습으로 코퍼스 변화 적응
  • 다양한 도메인(코드, 의료 등)으로 확장

프로파일링은 일회성 투자이며, 526개 트레이스 전체를 오픈소스로 공개하여 재사용 가능성을 극대화

결론

  • Query2Conf 정식화: 질의별 전체 파이프라인 동적 설정 선택 문제
  • LLM 제안 워크로드 특화 이진 특성이 범용 임베딩 압도
  • BRANE 프레임워크: 최고 고정 설정 동일 정확도 at 최대 89% 비용 절감
  • 3 벤치마크에서 LLM 라우터, 규칙 기반, 파인튜닝 모델 모두 상회
