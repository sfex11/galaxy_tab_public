문제 제기

LLM 에이전트는 ReAct, Reflexion 같은 스캐폴드를 통해 추론과 행동이 가능하지만, 에피소드 간에 실질적으로 활용 가능한 지식을 거의 보존하지 못한다.

특히 확률적, 부분 관측 가능한 환경에서는 반복 상호작용을 통해서만 정책을 발견해야 하며, 파인튜닝은 비용과 실현 가능성 측면에서 제약이 크다.

세 가지 핵심 질문

  • 무엇을 기억해야 하는가? (표현 방식)
  • 메모리를 어떻게 전파하는가? (개별 vs 집단)
  • 방법이 다양한 LLM 패밀리에 걸쳐 일관되게 동작하는가?

FORGE 개요

FORGE는 단계적(staged), 집단 기반(population-based) 프로토콜로, 가중치 업데이트 없이 프롬프트 주입 방식의 자연어 메모리를 진화시킨다.

내부 루프 (Inner Loop)

Reflexion 스타일의 실패 트리거 반성: 동일 기반 LLM을 사용하는 반성 에이전트가 실패 궤적을 재사용 가능한 지식 아티팩트로 변환

외부 루프 (Outer Loop)

단계 간 챔피언 브로드캐스트: 최고 성능 인스턴스의 메모리를 집단에 전파 · 수렴 인스턴스는 졸업(Graduation)으로 조기 종료

핵심 원칙: 더 강한 모델에서의 증류(distillation) 없이, 동일 기반 LLM만으로 자기 향상

에이전트 아키텍처

계층적 ReAct (Hierarchical ReAct) + 동적 메모리 주입

단계적 학습 (Staged Training)

챔피언 브로드캐스트

각 단계 종료 시 최고 성능 인스턴스의 메모리를 전체 집단에 복제. 고립된 학습의 한계 극복

졸업 및 조기 종료

수렴 기준을 만족한 인스턴스를 동결(graduation). 불필요한 계산 절감

No-Graduation ablation에서 확인: 브로드캐스트만으로도 성능 향상 달성

실험 환경: CybORG CAGE-2

CAGE-2는 확률적 네트워크 방어 POMDP로, 프롬프트 전용 적응을 위한 엄격한 스트레스 테스트로 설계됨

  • 긴 호라이즌: 30스텝 + 부분 관측 가능
  • 재난적 제로샷 성능: 메모리 기반 학습이 필수적
  • 스칼라 보상만: 자연어 피드백 없음
  • 실용적 도메인: 공개 리더보드 존재

브로드캐스트 vs 고립 반성

Reflexion (고립 학습)

각 인스턴스가 자신의 실패로부터만 학습. 단일 스트림으로 제한된 탐색으로 인해 중요한 실패 모드를 놓칠 수 있음.

FORGE (집단 브로드캐스트)

최고 인스턴스의 메모리가 전체 집단에 전파. 병렬 탐색의 이점을 모든 인스턴스가 공유하여 학습 가속화.

No-Graduation ablation 결과: 브로드캐스트가 성능 향상의 핵심 원동력이며, 졸업은 주로 계산 비용 절감에 기여

→ 브로드캐스트 없이 졸업만 적용해도 성능 향상은 제한적

핵심 발견 요약

1
집단 브로드캐스트가 핵심

고립 반성 대비 29–72% 향상. 브로드캐스트가 성능 견인, 졸업은 비용 절감

2
표현 선택은 목적에 따라

Examples → 최고 수익 / Rules → 최고 비용 효율 (토큰 40% 절감) / Mixed → 중간

3
약한 모델이 더 큰 이득

FORGE는 능력 격차를 완화할 가능성. 강한 모델의 이득을 증폭하는 것이 아님

4
가중치 업데이트 불필요

동일 기반 LLM만으로 스칼라 보상에서 자연어 메모리를 자가 진화. 최고 점수 −3.60 (DRL −3.47에 근접)

제한점 및 향후 과제

제한점

  • 모든 증거가 CAGE-2 B_line에 한정 (단일 공격자, 30스텝)
  • 다른 공격자 유형 및 환경으로의 일반화 미검증
  • 교차 패밀리 발견은 방향적 증거로 해석 필요
  • 메모리 크기 증가 시 컨텍스트 압력 가능성

향후 과제

  • 다양한 공격자 유형(CAGE 4 등)으로 확장
  • TextGrad 등 텍스트 기반 최적화 방법과의 비교
  • 메모리 압축 및 선택적 삭제 메커니즘
  • 다른 도메인(로보틱스, 게임 등)으로의 전이
