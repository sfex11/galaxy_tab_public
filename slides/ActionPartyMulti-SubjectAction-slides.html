01 — 문제 정의

비디오 월드 모델의
핵심 한계

  • 현재 비디오 확산 모델은 단일 에이전트 설정에 제한되어, 장면 내 여러 에이전트를 동시에 제어할 수 없음
  • 행동 바인딩(Action Binding) 문제: 특정 행동을 해당 주체에 올바르게 연결하지 못함
  • Veo 3 같은 최신 모델도 "삼각형은 아래로, 사각형은 왼쪽으로" 같은 기본적인 다중 주체 조합에 실패
  • 기존 멀티뷰 접근법(Multiverse, Solaris)은 에이전트별 별도 스트림 생성 → 선형적 비용 증가
핵심 질문: 하나의 통합된 장면에서 N명의 주체를 각각 독립적으로 제어할 수 있는가?

04 — 핵심 메커니즘

3D RoPE 공간 바이어싱

  • 비디오 토큰 위치 (h,w): 회전 R(t, h, w) 적용
  • 주체 토큰 zti: 이전 시점의 위치 사용 → R(t, ht-1i, wt-1i)
  • 주체를 현재 공간 영역에 고정 → 전역 탐색을 지역 정밀화로 축소
  • 연속 프레임 간 주체는 비교적 정적이라는 가정 활용
주체 토큰 zti ← R(t, ht-1i, wt-1i)   |   비디오 토큰 x(h,w) ← R(t, h, w)
왜 이전 시점 위치를 사용하는가?
현재 시점의 위치는 아직 예측되지 않았으므로, 이전 시점의 알려진 위치를 앵커로 사용하여 공간적 접지(grounding)를 수행합니다.

11 — 한계 및 향후 과제

한계점

📍
좌표 드리프트
예측된 zt 좌표가 실제 위치에서 벗어날 수 있음. 공간 바이어싱이 약간의 오정렬은 허용
👻
주체 소실
주체가 장면에서 사라질 수 있으며, 이후 상호작용이 불가능해짐
⏱️
실시간 미달
완전한 실시간은 아님. 증류(distillation)와 확산 강제(diffusion forcing)로 해결 가능
🎲
2D 환경 제한
2D 타일 기반 게임에 한정. 3D 장면의 가려짐(occlusion)은 미해결
필요 조건: 초기 프레임과 주체 위치가 필수 — 처음부터 주체를 자동 발견하는 것은 불가능합니다.
12 — 결론

ActionParty의 기여

  • 최초의 다중 주체 비디오 월드 모델 — 최대 7명의 플레이어를 46개 환경에서 동시 제어
  • 주체 상태 토큰 + 공간 바이어싱 — 프레임 렌더링과 주체 업데이트의 우아한 분리
  • 이동 정확도 5배 향상 — 텍스트 기반 대비 압도적 성능, 뛰어난 시각적 품질
  • 확장 가능한 아키텍처 — 주체 수 증가에도 토큰 6%만 추가, 안정적 성능 유지
