일반 디코딩

  • 전체 어휘에서 자유롭게 토큰 선택
  • 구조적 오류 발생 가능
  • 동일 질문에 다른 SQL 출력

템플릿 제약 디코딩

  • 템플릿 구조에 허용된 토큰만 선택
  • 구조적 정합성 보장
  • 동일 질문에 일관된 SQL 출력

핵심: 템플릿의 슬롯(플레이스홀더) 위치에서만 LLM이 값을 채우도록 제약

확장성과 템플릿 커버리지

커버리지 성장

쿼리 로그가 누적될수록 템플릿 커버리지가 자연스럽게 상승

  • 100개 로그: ~45% 커버리지
  • 500개 로그: ~72% 커버리지
  • 1000개 로그: ~85% 커버리지

추론 비용

템플릿 매칭 시 토큰 생성량 감소로 지연 시간 단축

  • 평균 30~40% 빠른 응답
  • 추론 비용 절감 효과

결론 및 시사점

  • 실제 환경에서 질문은 반복되며, 이를 활용하면 정확도를 극적으로 향상 가능
  • 템플릿 제약 디코딩은 SQL 구조의 정합성을 보장하는 간단하면서도 강력한 방법
  • 반복 질문에서 약 18%p 정확도 향상, 일관성 34% → 96%
  • 신규 질문에 대한 유연성도 하이브리드 라우팅으로 유지
  • 사용이 누적될수록 자연스럽게 성능이 향상되는 자가 강화 시스템

"완벽한 모델을 만드는 대신, 실제 사용 패턴에서 신뢰성을 확보하는 것이 실용적 접근"
