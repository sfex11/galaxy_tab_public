시스템 구조

5개 에이전트 아키텍처

공유 메모리 원장 위에서 협력하는 다중 에이전트 시스템

1

오케스트레이터

자연어 → 공간 기술절(SDC) 파싱. 앵커 객체, 공간 술어, 거리 제약 추출

2

그라운딩 에이전트

시맨틱 씬 그래프 Γ=(V,E)에서 참조 객체 해결. 문자열·CLIP·공간 현저성 3중 매칭

3

공간 에이전트

방향 + 거리 커널을 합성하여 3D 연속 확률 밀도 P(x|r) 생성

4

검증 에이전트

그라운딩 결과의 일관성 검증, 필요시 수정 재시도 트리거

5

플래너 인터페이스

P(x)에서 중요도 샘플링 → 상위 K 웨이포인트 → RRT* 경로 계획

🗄️ 공유 메모리 원장 — 씬 그래프 Γ, 참조 객체 신뢰도 Bt, 활성 공간 우도 필드를 모든 에이전트가 공유

수학적 기반

공간 커널 확률 합성

🧭 방향 커널

von Mises-Fisher 분포로 방향 제약 모델링

Pdir(x) = (1/Z) exp(κ · (Ro m(θ₀,φ₀))ᵀ · x̂)

κ: 방향 집중도 | Ro: 객체→세계 변환 | m: 방향 벡터

📏 거리 커널

방사형 가우시안으로 거리 제약 모델링

ℓmet(x) = -1/(2σ²m) · (‖x-to‖ - d₀)²

d₀: 목표 거리 | σm: 허용 오차 | to: 앵커 위치

합성 로그 우도

log P(x) = ℓmet(x; d₀, σm) + ℓpred(x; θ₀, φ₀, κ)

로그 공간에서 합산 → 수치적 안정성 확보 + 다중 제약의 자연스러운 다봉 분포 형성

계단식 합성: 복합 명령("싱크대 근처이면서 전자레인지 왼쪽에 컵 놓기")의 경우, 각 제약에 대한 커널을 독립적으로 생성한 뒤 전역 좌표에서 합성 → 제약이 강화되거나 균형을 이루는 곳에서 다봉 밀도가 출현합니다.

한계점과 향후 방향

⚠️ 현재 한계

  • 씬 그래프 의존성: 정확하고 완전한 3D 시맨틱 씬 표현이 필수. 가려진 객체가 누락되면 성능 저하
  • 기준 프레임 모호성: "앞"/"뒤" 같은 술어에서 객체 고유 방향이 불분명할 때 부분적으로만 완화
  • 실세계 확장: 현재 소규모 개념 증명에 한정. 체계적 대규모 평가 미실시
  • 상식 QA: 의미적 상태 추론이나 화용적 추론에는 미설계

🚀 향후 연구 방향

  • 씬 그래프 구축의 강건성 향상
  • 객체 고유 방향 처리 개선
  • 대규모 실세계 주석 데이터셋 구축
  • SLAM과의 통합 (동적 환경)
  • 시간적 쿼리로의 확장 (객체 상태 변화)

GraphEQA의 근본적 실패 원인

GraphEQA는 "최근접 만족 편향": 방향은 충족하지만 거리를 강제하지 않아 5.82m 오차 발생. 반면 MAPG는 확률적 커널로 거리를 명시적으로 강제하여 0.07m 달성.

결론

3가지 핵심 기여

1. MAPG 프레임워크

온라인 3D 씬 그래프 + 분석적 공간 커널을 결합한 다중 에이전트 확률적 프레임워크. 메트릭-시맨틱 명령에서 플래너 사용 가능 목표 분포 생성.

2. MAPG-Bench

메트릭-시맨틱 목표 그라운딩 전용 최초의 벤치마크. 30개 장면, 100개 쿼리, 3D 세계중심 정답. 시점 일관성·부분 관측성 평가.

3. 실증적 검증

거리 오차 0.07m (98.8% 개선), 각도 일관성 0.3° Yaw. 제거 연구 + 실세계 로봇 시연으로 전이성 확인.

98.8% 거리 오차 감소율
0.3° 최소 Yaw 오차
0.98 앵커 식별률

MAPG는 합성적 확률 그라운딩이 단일 단계 VLM 예측보다 근본적으로 더 신뢰할 수 있는 메트릭-시맨틱 내비게이션 인터페이스를 제공함을 입증했습니다.
