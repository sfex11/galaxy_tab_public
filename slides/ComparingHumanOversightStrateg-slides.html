CUA의 등장

실제 위험

개인정보 유출, 프롬프트 인젝션, 다크 패턴 등 에이전트의 문제 행동이 사용자에게 직접적 피해를 줄 수 있음

감독 전략 A

단계 수준 전략

🛡️

위험 게이트 (Risk-Gated)

에이전트 주도 단계 수준 (선택적)

에이전트가 자율적으로 실행하며, 위험하다고 판단된 단계에서만 사용자에게 승인을 요청합니다. 예: 주문 제출, 인증 정보 입력 시 모달 승인 대화상자 표시.

참고 시스템: OpenAI Operator

✅

행동 확인 (Action Confirmation)

인간 통제 단계 수준 (필수)

모든 행동에 대해 명시적 승인이 필요합니다. 최대 직접 통제권을 보장하지만, 상호작용 비용이 가장 높습니다.

참고 시스템: Anthropic Claude Computer Use

감독 전략 B

계획 수준 전략

🔄

감독 공동실행 (Supervisory Co-Execution)

에이전트 주도 계획 수준

사용자가 실행 전 작업 계획을 검토·수정하고, 에이전트가 그 구조 내에서 자율 실행합니다. 계획, 현재 상태, 중간 추적 정보가 지속적으로 표시됩니다.

참고 시스템: Microsoft Magentic-UI

🔬

구조 강화형 (Structurally Enriched)

하이브리드 계획 + 단계 (유연)

계획 수준과 단계 수준의 참여를 유연하게 결합합니다. 실행 전 계획 검토, 실행 중 단계별 근거 확인, 위험 정보가 특정 단계에 부착되어 필요 시 검토를 유발합니다.

질적 발견

감독 실패의 3가지 메커니즘

1

국소적 위임 경계

참가자들은 전역적 신뢰 판단 대신, 행동 유형별로 국소적 위임 경계를 설정했습니다. 시스템 정의와 다른 기준으로 행동을 그룹화했습니다.

"승인 유사 버튼을 누르면 결제로 넘어가지 않을 거라 생각했어요" — P001
2

합리화, 부주의가 아님

감독 실패는 단순한 놓침이 아니라 합리화에서 비롯되었습니다. 참가자들은 문제 행동을 익숙한 자동화나 웹사이트 특성으로 정상화했습니다.

"웹사이트가 요구한 거라서, 에이전트는 그냥 자기 일을 한 거죠" — P006
3

인터페이스 가독성

위험 라벨과 하이라이트가 있을 때만 감독 관련 순간으로 인식되었습니다. 시각적 단서 없이는 위험한 행동이 정상적 흐름에 섞여 버렸습니다.

"모든 것이 중립적으로 수용 가능해 보이게 만들어요" — P014

노출 vs 교정 분리

계획 기반 전략이 노출을 줄이지만 교정 능력을 보장하지 않음을 인식하고 설계

👁️

인식 병목 해소

통제 메커니즘보다 문제 순간의 인식을 돕는 맥락적 단서에 집중

⚖️

맥락 적합성

작업 결과의 중대성에 따라 감독 깊이를 조정 — 획일적 적용 금지

💡

판단 지향 투명성

왜 중요한지 설명하되, 과도한 정보로 주의를 분산시키지 않기

🔀

유연한 참여

고정 깊이가 아닌, 불확실성에 따라 경량 모니터링↔심층 검토 전환 지원

⏱️

결정적 순간 중심

통제나 가시성 극대화가 아닌, 판단이 필요한 순간과 그 의미를 부각

핵심 결론: 효과적인 CUA 감독은 인간 관여의 극대화로 달성되지 않습니다. 핵심은 사용자가 중요한 결정 순간을 인식하고 적절히 개입할 수 있도록 시스템을 설계하는 것입니다.
