DashAttention

미분 가능하고 적응적인 희소 계층적 어텐션

Differentiable and Adaptive Sparse Hierarchical Attention

동기 부여

LLM의 컨텍스트 길이가 길어짐에 따라, 전체 어텐션의 연산 비용은 기하급수적으로 증가합니다. 이를 해결하기 위해 계층적 희소 어텐션 방법들이 제안되었으나, 근본적인 한계가 존재합니다.

전체 어텐션

O(n²) 복잡도 — 긴 시퀀스에서 비실용적

기존 희소 어텐션

Top-K 기반 선택 — 고정된 토큰 수 가정

기존 방법: 계층적 어텐션

NSA, InfLLMv2 등은 두 단계로 어텐션을 수행합니다.

핵심 문제: Top-K의 한계

Top-K 연산은 모든 쿼리에 대해 관련 토큰의 수가 동일하다고 가정합니다.

  • 단순한 질문은 소수의 토큰만 필요 — 과도한 선택
  • 복잡한 질문은 많은 토큰 필요 — 부족한 선택
  • Top-K는 미분 불가능 — end-to-end 학습 불가
  • 하이퍼파라미터 K에 대한 민감도 높음

DashAttention: 핵심 아이디어

미분 가능한 블록 선택 메커니즘을 통해, 각 쿼리마다 적응적으로 관련 KV 블록의 수와 중요도를 결정합니다.

D

Differentiable

미분 가능한 선택

A

Adaptive

쿼리별 적응적 선택

S

Sparse

희소성 유지

방법론: 미분 가능한 블록 선택

Hard top-K 대신 소프트 게이팅 함수를 사용하여 블록의 중요도를 연속적으로 결정합니다.

기존 Top-K

  • 이진 선택 (0 또는 1)
  • 기울기 전파 불가
  • K값 수동 튜닝

DashAttention

  • 연속적 가중치 (0~1)
  • End-to-end 학습 가능
  • 자동 적응적 결정

적응적 희소성 메커니즘

쿼리의 복잡도에 따라 참조할 KV 블록의 수가 자동으로 조절됩니다.

단순 쿼리

소수의 블록에 높은 가중치 집중 → 낮은 연산량

복잡 쿼리

다수의 블록에 분산된 가중치 → 높은 연산량 (필요시)

희소성 유지

스파르시티 정규화로 불필요한 블록 자동 억제

학습 목표

DashAttention은 다음 세 가지 목표를 동시에 최적화합니다.

어텐션 품질

표준 어텐션과의 출력 차이를 최소화하여 모델 성능 유지

희소도 분석

DashAttention이 학습한 쿼리별 희소도 분포를 분석하면 의미 있는 패턴이 관찰됩니다.

희소도 분포

쿼리별 활성 블록 수가 넓은 범위에 걸쳐 분포 — 고정 K가 부적합한 이유를 시사

의미적 상관관계

복잡한 추론 질문이 단순 질문보다 더 많은 블록을 활성화 — 적응적 동작 확인

핵심 요약

  • Top-K의 미분 불가능성과 고정 K의 비적응성을 동시에 해결
  • 소프트 게이팅 기반 미분 가능한 블록 선택으로 end-to-end 학습 실현
  • 쿼리 복잡도에 따라 자동으로 참조 블록 수 조절
  • 스파르시티 정규화로 희소성 유지하면서 품질 저하 최소화
  • NSA, InfLLMv2 등 기존 방법 대비 일관된 성능 우위 달성
