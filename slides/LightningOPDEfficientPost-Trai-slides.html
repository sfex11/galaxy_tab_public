🔑 교사 일관성 (Teacher Consistency)

Lightning OPD 성공의 핵심 조건

정의

SFT 단계의 교사 모델(πTSFT)과 OPD 단계의 교사 모델(πTOPD)이 반드시 동일해야 함

❌ 위반 시

  • πref는 πTSFT의 궤적에 집중
  • 어드밴티지 함수는 πTOPD의 선호도 반영
  • 환원 불가능한 기울기 편향 발생
  • 최적이 아닌 점에 수렴

✅ 충족 시

  • 표준 OPD와 동일한 최적점 공유
  • 기울기 불일치가 제한됨
  • 자동 암묵적 정규화 작동
  • 동등한 성능 달성
실제 사례: OpenThoughts-3 파이프라인은 SFT에 QwQ-32B, OPD에 Qwen3-32B 사용 → 교사 일관성 위반으로 2~4점 하락

📜 이론적 보장

정리 3.5: 기울기 불일치 상한

||∇Jon(θ) − ∇Joff(θ)||₂ ≤ G · σA · √χ²(πθ||πref)

초기화 시점(πθ = πref)에서 χ² = 0 → 기울기 완전 일치

정리 3.7: 동일한 고정점

Jon과 Joff는 동일한 정상점을 공유 → 모든 정상점 θ*는 모델 클래스 ΠΘ 위에서 KL(πθ||πT)를 최소화

정리 3.9: 암묵적 정규화

∇Joff(θ) = ∇Jon(θ) − Covπref[w(x;θ), f(x;θ)]

공분산 항이 자동 트러스트 리전 페널티로 작용 → 학생 정책의 과도한 이탈 방지 (하이퍼파라미터 조정 불필요)
