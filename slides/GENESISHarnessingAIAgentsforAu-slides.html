문제 정의

RAN R&D를 지연시키는 6가지 구조적 병목

각 병목은 반복당 수개월의 수작업 공학이 필요하며, 이들이 누적되어 혁신의 속도를 제한한다.

1

기능 합성

표준/논문 → 프로덕션 코드

2

적합성 테스트

상호운용성 및 회귀 테스트

3

하드닝

필드 이상 대응 및 환경 다양성

4

데이터 기반 최적화

네트워크 기능 알고리즘 튜닝

5

신규 기능 발견

파형, 기능,Capability 프로토타입

6

보안 강화

스택 취약점 식별 및 대응

5G 스택 분석: 첫 코드 변경 → 안정 브랜치 병합 평균 74일, 90th percentile 207일

동기

LLM만으로는 RAN 공학에 부족하다

일반 소프트웨어 공학에서의 성과

  • SWE-bench에서 강력한 성능
  • AutoGen, MetaGPT, Claude Code 등 다중 에이전트 프레임워크 활성
  • 엔지니어링 리드타임 며칠 → 몇 분으로 단축

RAN 환경에서의 한계

  • API 환각(Hallucination) → 상호운용성 즉시 파괴
  • 모호한 사양 오해독해
  • 시뮬레이션에서는 동작하나 실제 하드웨어 전환 시 실패
  • RAN 특화 스캐폴딩 부재

이 간극은 프롬프트 엔지니어링만으로 해결될 수 없다. 폐루프 테스트 하니스와 RAN 특화 스캐폴딩이 필수적이다.

제안

GENESIS: 의도(Intent) → 실증(OTA) 자율 프레임워크

사양 조항, 텔레메트리 이상, 연구 가설 등의 고수준 의도를 실제 무선(OTA) 실험으로 검증된 솔루션으로 변환

아키텍처

4계층 아키텍처 + Synapse 지식 평면

Intent Layer (의도 계층)

고수준 의도가 프레임워크 진입점. 사양 조항, KPI 이상, 연구 가설 등

Agentic Layer (에이전트 계층)

오케스트레이터가 의도를 6개 파이프라인으로 라우팅. 전문가 에이전트 풀(DevOps, RAN, Radio, UE, Testbed, Emulation)에서 런타임 구성

Deterministic Execution Layer (결정론적 실행 계층)

~23개 매개변수화된 Skills(build, run, configure, deploy, experiment…) + Hooks(관찰, 정책 게이트, 감사)

Substrate Layer (기반 계층)

3계층 검증 연속체: RFSIM → Colosseum(HIL 에뮬레이션) → OTA 테스트베드(X5G, Arena)

Synapse가 모든 계층을 수직으로 관통 — 사양 검색의 진실 공급원이자, 모든 실행 아티팩트의 수신처

설계 원칙

3가지 합성 가능한 기본 요소(Primitives)

Agents

추론자 + 도구 인터페이스

  • 도메인 전문 페르소나
  • 결정 권한과 도구 부분집합
  • 런타임 파이프라인 조립
  • 에이전트 프레임워크 이식 가능

Skills

에이전트 호출 명령 패키지

  • ~23개 매개변수화된 절차
  • 명시적 성공 기준 정의
  • build, run, configure, deploy, experiment 등
  • 에이전트 추론 기반 실행

Hooks

관찰·정책·감사 플레인

  • 이벤트 구조화 로깅 (관찰)
  • 우회 불가 안전 검사 (정책)
  • 변경 출처 기록 (감사)
  • 컨텍스트 윈도우 바이패스

Agent / Skill / Hook 분리가 GENESIS를 합성 가능, 관찰 가능, 이식 가능하게 만든다.

지식 평면

Synapse: 이중 역할 지식 베이스

진실의 공급원 (Source of Truth)

  • 3GPP / O-RAN 사양 하이브리드 검색
  • 에이전트가 API를 환각하는 것을 방지
  • 모호한 사양 조항의 정확한 해석 제공

아티팩트 수신처 (Sink of Artifacts)

  • 모든 실행의 트레이스, 로그, 코드 diff 저장
  • 사양→코드 추적성(traceability) 유지
  • 실행 간 Capability 복합 효과

Claude Opus 4.7 / Sonnet 4.6

상용 클라우드 모델

gpt-oss, Llama 4

로컬 서빙 오픈 웨이트

역량 파이프라인

R&D 라이프사이클 전체를覆盖하는 6개 파이프라인

Synthesize

사양 조항 → 프로토타입 및 프로덕션 코드 자동 생성

Test

적합성, 상호운용성, 회귀 테스트 자동 실행

Harden

이상 감지 → 수정 루프 자율 수행

Optimize

데이터 기반 알고리즘 학습 및 배포

Discover

신규 파형, 기능, Capability 연구 및 프로토타입

Secure

스택에 대한 적대적 분석 및 취약점 식별

각 파이프라인은 런타임에 전문가 에이전트 풀에서 동적으로 조립되며, 동일한 인터페이스로 진입

사례연구 1

3GPP RRC.ConnMean KPM 구현 및 테스트

의도: "TS 28.552에 따라 RRC.ConnMean Key Performance Measurement를 구현하라"

수행 절차

  • Synapse에서 TS 28.552 사양 검색
  • RAN 에이전트가 E2SM-KPM 인터페이스 코드 합성
  • DevOps 에이전트가 빌드 및 배포
  • Testbed 에이전트가 UE 연결 후 KPM 리포트 검증

검증 결과

  • RFSIM에서 정확한 KPM 리포트 생성 확인
  • 에뮬레이션 환경에서 교차 검증 완료
  • 사양→코드 추적성 자동 기록
  • 100% 성공률

기준(Claude Code Opus 4.7)은 동일 작업에서 매번 일관되게 실패 — API 환각으로 인해 빌드 단계에서 차단

사례연구 2

Conditional Handover + xApp 폐루프 최적화

의도: E2SM-RC 인터페이스 위에 CHO를 위한 폐루프 xApp을 합성, 테스트, 하드닝하라

1
합성
CHO 로직 + E2SM-RC xApp 코드 생성
2
테스트
RFSIM → 에뮬레이션 다계층 적합성 검증
3
하드닝
필드 이상 시나리오 감지 후 자율 수정 루프

주요 성과

  • Synthesize + Test + Harden 3개 파이프라인 연속 실행
  • 이상 시나리오에서 자율적으로 에러 원인 식별 및 패치
  • 하드닝 후 재테스트 자동 수행

기준선의 실패 원인

  • E2SM-RC 인터페이스 API 환각
  • 다중 벤더 스택 인터페이스 오해
  • 에러 발생 시 자가 복구 불가
사례연구 3

스케줄러 발견(Discovery): 연구 가설 → 실증

의도: 새로운 RAN 스케줄링 변형을 연구하고, 기존 SOTA와 비교하라

Discover 파이프라인 동작

  • 연구 가설을 코드로 변환
  • 5G 스택에 통합
  • 벤치마크 스케줄러와 비교 실험
  • 결과를 Synapse에 기록

R&D 라이프사이클 시범

3개 사례연구가 합쳐 R&D 전체 라이프사이클을 커버:

  • CS1: Synthesize + Test
  • CS2: Synthesize + Test + Harden
  • CS3: Discover (연구 → 코드 → 비교)

기존에는 연구 가설 하나를 실제 라디오에서 검증하는 데 수개월이 소요되었으나, GENESIS는 이를 수 시간 내에 완수

성능 평가

GENESIS vs SOTA 코딩 에이전트

평가 항목 GENESIS Claude Code (Opus 4.7)
스택 기능 구현 성공률 100% 0% (매번 실패)
API 환각 방지 Synapse 기반 검증 환각으로 빌드 실패
에러 자가 복구 폐루프 테스트 기반 복구 불가
실제 하드웨어 검증 OTA까지 수행 시뮬레이션 단계 불통
사양→코드 추적성 자동 기록 미지원

다수의 통계적으로 독립적인 실험에서 GENESIS는 일관되게 100% 성공, 기준선은 일관되게 0% 성공

비용 분석

단계별 비용·시간 분석 및 모델 선택

단계별 토큰/비용 프로파일링

  • 사양 검색(Synapse): 검색 비용 최소화
  • 코드 합성: 토큰 소비 최대 단계
  • 빌드/테스트: 에이전트 추론 비용
  • 하드닝 루프: 반복 횟수에 비례

전체 파이프라인: 수작업 수개월 → 수 시간 (토큰 비용은 수~수십 USD 수준)

모델 선택: 속도 vs 성공당 비용

  • Claude Opus 4.7: 최고 성공률, 높은 비용
  • Claude Sonnet 4.6: 균형잡힌 속도/비용
  • Llama 4 / Phi: 저비용, 단순 작업에 적합
  • 파이프라인 단계별 모델 혼합 가능

RAN 작업의 복잡도에 따라 모델을 동적 선택하는 전략이 효과적

결론

요약 및 향후 과제

핵심 기여

  • RAN R&D 6가지 병목을 커버하는 최초의 에이전트 프레임워크
  • Agent/Skill/Hook 합성 가능 아키텍처
  • Synapse: 이중 역할 지식 평면으로 환각 방지
  • 시뮬레이션→에뮬레이션→OTA 3계층 폐루프 검증
  • 3개 사례연구에서 100% 성공률 달성
  • R&D 주기: 수개월 → 수시간 단축

향후 과제

  • 더 광범위한 3GPP Release 적용
  • 다벤더 상호운용성 확장
  • Secure 파이프라인 심화 (적대적 테스트)
  • 에이전트 간 협업 프로토콜 고도화
  • 실제 오퍼레이터 필드 데플로이먼트
  • 비용 최적화를 위한 모델 자동 선택

GENESIS는 셀룰러 R&D가 "수작업 공학"에서 "의도 주도 자율 합성"으로 전환하는 패러다임을 제시한다.
