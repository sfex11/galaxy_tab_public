모델 크기 × 프롬프트 상호작용

💪 대형 모델 (≥32B)
  • 복잡한 프롬프트(CoT, 상세 기준)에서 최적 성능
  • 충분한 파라미터로 긴 지시사항 통합 가능
  • 추론 능력으로 미묘한 판정 수행
🤏 소형 모델 (≤14B)

안전 정렬의 양날의 검

🚫 콘텐츠 필터 차단
OpenAI o-시리즈(o1, o1-mini, o3-mini)는 API 레벨의 콘텐츠 필터가 유해/불법 입력을 차단하여 완전한 평가 불가능 → 연구에서 제외
⚠ 판정 거부 현상
안전 정렬이 강한 모델(Llama 3.3, GPT-4o)이 유해 콘텐츠 판정 작업 자체를 거부하거나 포맷 오류를 발생시키는 경우 확인
💡 시사점
보안 취약점 평가처럼 유해 콘텐츠를 다뤄야 하는 경우, 안전 정렬 모델만으로는 충분하지 않음. 특화 탐지기와 대화형 판사의 병행 사용이 필요.
📊 탈옥 공격 카테고리: 32B 이하 모델의 CoT 정확도 74-85%로 하락
📊 일부 모델이 이진 판정 대신 "somewhat_similar" 등 정의되지 않은 판정 생성
