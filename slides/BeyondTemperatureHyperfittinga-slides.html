문제: LLM의 반복적 생성 붕괴

결정적 디코딩의 한계

  • Greedy / Beam Search 사용 시 반복 루프에 빠짐
  • 개방형 텍스트 생성에서 특히 심각
  • 다수의 최근 연구에서 지속적으로 보고

확률적 방법의 대안?

  • Top-k, Top-p, Nucleus Sampling 등
  • 반복은 완화되나 일관성 저하 위험
  • 전체 텍스트 품질을 저해할 수 있음
결정적 디코딩에서 반복 없이 고품질 생성을 달성할 수 있는가?

Hyperfitting: 역설적 현상

소규모 데이터셋에서 과적합(Train Loss → 0)까지 학습하면,
오히려 개방형 생성 품질이 향상되고 반복이 감소한다

핵심 질문: 그냥 Temperature Scaling인가?

Temperature Scaling 가설

확률 분포를 날카롭게 만들되
토큰 순위는 변하지 않음

반복 토큰(Token A)이 여전히 Top-1 유지

Rank Reordering 가설

토큰 순위 자체를 재배열
반복 후보 억제, 다양한 후보 승격

새로운 토큰(Token B)이 Top-1로 승격

이 두 가지를 구별하는 것이 Hyperfitting이 새로운 학습 역학인지
단순한 확률 스케일링인지 판단하는 핵심

오히려 지속적 향상

Perplexity는 Greedy Decoding 설정에서 생성 품질의 부적절한 대리 지표다.
모델은 확률 추정에서는 "나빠졌지만" 텍스트 생성에서는 "나아졌다".

결론

핵심 발견

  • Hyperfitting은 Temperature Scaling과 근본적으로 다름
  • 동적 순위 재배열이 품질 향상의 인과적 메커니즘
  • 최종 레이어의 기하학적 확장(ΔDim ≈ +80.8)이 원인
  • Late-Stage LoRA로 최소 파라미터로 동일 효과 달성

시사점

  • Perplexity ≠ 생성 품질 (Greedy 설정)
  • 과적합이 항상 나쁜 것은 아님
  • 메커니즘 이해가 효율적 개입을 이끔
  • 반복 문제 해결의 새로운 패러다임
"Hyperfitting은 학습된 형태의 Temperature Scaling이 아니다.
이는 최종 레이어에서의 기하학적 확장을 통한 동적 순위 재배열이다."
