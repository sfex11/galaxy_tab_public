FASE: Fast Adaptive Semantic Entropy
for Code Quality

LLM 생성 코드의 기능적 정확도를
추론 없이, 신속하게, 적응적으로 평가하는 방법

Shizhe Lin, Ladan Tahvildari · University of Waterloo

다중 에이전트 코드 생성의 현실과 한계

유망한 패러다임

  • 요구사항 분석, 설계, 코딩, 테스트, 리뷰 등 역할 분담
  • MetaGPT, CodeCoR, AdaCoder 등 실제 프레임워크 등장
  • 단일 모델 한계를 넘어 협업적 워크플로우 구현

근본적 한계

  • LLM 환각(Hallucination)으로 인한 신뢰도 저하
  • 에이전트 간 오류 전파 → 연쇄적 실패
  • 기존 완화 기법의 높은 계산 비용

초기 단계의 환각이 불신뢰 에이전트(빨간색)에서 시작되어, 일관된 에이전트(파란색)의 신뢰성마저 훼손하며 전체 워크플로우로 전파된다.

기존 불확실성 추정 방법의 세 가지 한계

1

정확성 의존

LLM 기반 양방향 함의(Entailment) 판단은 모델의 자기평가 능력에 의존하며, 모델별로 불일치 판단이 빈번함

2

비용 폭발

생성 출력 수에 대해 O(|x|²)의 LLM 호출이 필요하여, 다단계 에이전트 워크플로우에서 실용성 없음

3

구조적 한계

구조 엔트로피만으로는 기능적으로 동등한 코드를 다른 클러스터로 분류하는 오분류 문제 발생

→ 더 정확하고, 효율적이며, 모델에 의존하지 않는 방법이 필요하다

FASE: 핵심 아이디어

Fast Adaptive Semantic Entropy

코드 임베딩 모델로 생성 코드 간 의미적 거리를 측정하고, 최소 신장 트리(MST) 기반 적응적 클러스터링으로 기능적 동등성을 모델링하여, LLM 추론 없이 의미 엔트로피를 근사하는 지표

Fast

LLM 대신 임베딩 모델 사용 → 기존 방법 대비 0.3% 수준의 비용

Adaptive

고정 임계값 없이 각 태스크의 해 공간 구조에 맞게 동적 클러스터링

FASE 워크플로우

1
코드 생성
LLM이 동일 태스크에 대해 N개의 코드 샘플 생성
2
임베딩 추출
Encoder-only 모델로 각 코드를 벡터 공간에 매핑
3
거리 행렬 계산
Cosine distance 기반 O(N²) 쌍대 의미 거리 행렬 구성
4
MST 기반 클러스터링
최소 신장 트리에서 적응적 임계값으로 엣지 제거
5
엔트로피 계산
클러스터 확률 분포로 의미 엔트로피 도출

전체 과정에서 LLM 추론 호출 0회

의미 엔트로피의 수학적 정의

예측 엔트로피 (Predictive Entropy)

PE(X) = H(𝒞|x) = −Σc [ Σs∈c P(s|x) ] · log [ Σs∈c P(s|x) ]

𝒞: 동등성 클래스 집합, c: 개별 클래스, s: 생성된 코드 샘플

텍스트 유사성 기반

토큰 수준 비교 → 환각 탐지 불가부적합

구조 유사성 기반

AST 기반 비교 → 다양한 구조의 동등 코드 오분류한계

LLM 의미 엔트로피

양방향 함의 → 정확하나 비용 폭발비효율

FASE (제안)

임베딩 + MST → 정확 + 저비용최적

핵심 기술: 임베딩 기반 거리 & MST 클러스터링

쌍대 의미 거리 행렬

  • Encoder-only 임베딩 모델 사용 (예: Qwen3-Embedding-8B)
  • 코사인 거리로 di,j ∈ [0, 1] 계산
  • LLM 호출 없이 O(|x|²) 병렬 연산 가능

최소 신장 트리(MST) 기반 클러스터링

  • 완전 그래프에서 MST 구성
  • 에지 가중치 분포에서 적응적 임계값 도출
  • 임계값 초과 에지 제거 → 자연스러운 클러스터 분리

고정 임계값 vs 적응적 임계값

기존 방법: 모든 태스크에 동일한 θ 적용 → 단순한 태스크에서 과분류, 복잡한 태스크에서 과소분류

FASE: 각 태스크의 MST 에지 분포에서 θ를 자동 산출 → 해 공간의 복잡도에 맞는 클러스터링

실험 설정

벤치마크

  • HumanEval — 164개 파이썬 함수 생성 태스크
  • BigCodeBench — 더 복잡하고 실용적인 코딩 문제

코드 생성 모델

DeepSeek-Coder-V2, Qwen2.5-Coder-32B 등 다양한 모델에서 N=20개 샘플 생성

상세 분석: 임베딩 표현 & 적응적 클러스터링

RQ1: 임베딩이 기능적 정확도를 포착하는가?

  • Qwen3-Embedding-8B가 코드 기능적 동등성을 효과적으로 인코딩
  • 텍스트/구조 유사성보다 의미적 거리가 Pass@1과 더 높은 상관관계 보임
  • 다양한 임베딩 모델 간 비교에서 코드 특화 임베딩이 우수

RQ2: 적응적 클러스터링의 효과는?

  • 고정 임계값 클러스터링 대비 일관된 성능 우위
  • 단순한 태스크(해 공간 좁음)와 복잡한 태스크(해 공간 넓음) 모두에서 안정적
  • MST 기반 방식이 k-means, DBSCAN 등 대안보다 우수

계산 비용 & 확장성 분석

실행 시간 비교

방법 상대 비용
LLM 의미 엔트로피 100%
LLM 자기평가 ~80%
FASE ~0.3%

확장성 함의

  • 임베딩은 일회 추론 후 캐싱 가능 → 반 평가 시 비용 추가 없음
  • GPU 배치 처리로 거리 행렬 계산 병렬화 용이
  • 다중 에이전트 실시간 모니터링에 실질적으로 배포 가능
  • 샘플 수 N 증가 시에도 선형적 비용 증가

기여 요약 & 시사점

1

경량화된 의미 엔트로피 지표 제안

임베딩 + MST 기반으로 Ground-truth 없이 코드 기능적 정확도 추정

2

종합적인 실험 평가

LLM 의미 엔트로피, 자기평가, 구조 엔트로피 등 다양한 베이스라인과 비교 분석

3

비용-정확도 트레이드오프 정량화

0.3% 비용으로 최고 수준의 정확도 달성 → 실제 다중 에이전트 시스템 배포 가능성 입증

핵심 시사점: FASE는 불확실성 정량화를 "사후 검증"에서 "실시간 관측"으로 전환하여, 다중 에이전트 코드 생성 워크플로우의 신뢰성을 실용적으로 향상시킬 수 있다.

한계점 & 향후 방향

연구 한계

  • 파이썬 중심 평가 → 타 언어(Java, C++ 등)로의 일반화 미검증
  • 임베딩 모델 자체의 한계가 FASE 성능에 직접 영향
  • 매우 긴 코드(리포지토리 수준)에서 임베딩 품질 저하 가능성
  • MST 클러스터링의 이론적 보장에 대한 추가 분석 필요

향후 연구 방향

  • 다언어 임베딩 모델 통합 및 교차 언어 평가
  • 실제 다중 에이전트 프레임워크(MetaGPT 등)에 통합 검증
  • FASE 점수를 피드백으로 활용하는 적응적 재생성 루프 설계
  • 동적 샘플링 — 엔트로피가 높을 때만 추가 샘플 생성
