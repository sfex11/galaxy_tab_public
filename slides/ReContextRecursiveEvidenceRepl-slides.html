ReContext: Recursive Evidence Replay
as LLM Harness for Long-Context Reasoning

Yanjun Zhao, Ruizhong Qiu, Tianxin Wei, Yuanchen Bei,
Zhining Liu, Lingjie Chen, Ismini Lourentzou, Hanghang Tong, Jingrui He

University of Illinois Urbana-Champaign

문제 정의: 컨텍스트 접근 vs 활용의 간극

 컨텍스트 접근 (Context Access)

최신 LLM은 128K 이상의 긴 컨텍스트 창을 지원합니다. 전체 문서, 다중 문서, 대화 기록을 단일 프롬프트에 담을 수 있습니다.

 컨텍스트 활용 (Context Harnessing)

하지만 정답에 필요한 증거가 이미 입력에 존재함에도, 모델이 이를 일관되게 활용하지 못하는 실패가 반복적으로 관찰됩니다.

"병목은 컨텍스트 접근에만 있는 것이 아니라, 컨텍스트 하니싱에 있다." 긴 컨텍스트 내에서 현재 추론 단계와 가장 관련 있는 정보를 동적으로 식별·조직·갱신하는 메커니즘이 필요합니다.

핵심 관찰: 희소한 관련성 신호

128K 토큰 컨텍스트에서 상위 0.1% 토큰(약 128개)이 전체 질문-조건부 관련성 점수의 약 50%~80%를 차지합니다.

이 희소한 신호를 명시적인 증거 풀로 변환하는 것이 ReContext의 핵심 아이디어입니다. 세 가지 LLM 백본(Qwen3-4B, Qwen3-8B, Llama3-8B)에서 일관되게 관찰됩니다.

  • 학습 불필요 (Training-free): 별도의 학습이나 파인튜닝 없이 추론 시에만 적용
  • 전체 컨텍스트 보존: 원본 컨텍스트는 그대로 유지, 증거 풀은 강조 용도
  • 외부 메모리 불필요: 검색 시스템이나 외부 저장소 없이 모델 내부 신호만 사용

증거 선택 (Evidence Selection)

어텐션 기반 관련성 추출

질문 토큰이 컨텍스트 토큰에 부여하는 어텐션 점수를 질문-조건부 관련성 신호로 활용합니다.

프롬프트 분할

컨텍스트를 세그먼트로 분할하여 어텐션 누적 계산

어텐션 리드아웃

질문 토큰→컨텍스트 토큰 어텐션을 집계하여 점수화

상위 토큰 선택

누적 관련성 점수 기준 상위 토큰 추출

핵심 설계 원칙

  • 질문-조건부: 일반적인 중요도가 아닌 현재 질문과의 관련성으로 선택
  • 모델 내부 신호만 사용: 외부 임베딩 모델이나 검색기 불필요
  • 토큰→문장 복원: 선택된 토큰을 포함하는 완전한 문장으로 재구성

증거 구체화 및 재생

 증거 구체화 (Materialization)

선택된 개별 토큰을 원본 컨텍스트에서 해당 토큰을 포함하는 완전한 문장(span)으로 복원합니다. 이 과정에서 문맥이 보존된 일관된 텍스트 단위가 생성됩니다.

 증거 재생 (Replay)

구체화된 증거 문장들을 원본 컨텍스트 앞에 삽입하여 새로운 프롬프트를 구성합니다. 원본 컨텍스트는 그대로 유지되며, 증거 풀은 모델의 주의를 안내하는 역할을 합니다.

프롬프트 구조 변화

기본 (Vanilla)

[전체 컨텍스트 128K] + [질문] → [답변]

ReContext 적용

[증거 풀] + [전체 컨텍스트 128K] + [질문] → [답변]

재귀적 증거 선택

단일 라운드로 끝나지 않고, 이전 라운드의 증거 풀을 조건으로 새로운 증거를 반복적으로 발견합니다.

R=1
1차 선택
원본 컨텍스트 + 질문으로 어텐션 계산
R=2
2차 선택
원본 + 질문 + 1차 증거 풀로 재계산
R=3
3차 선택
원본 + 질문 + 누적 증거 풀로 재계산
최종 생성
완성된 증거 풀과 함께 답변 생성

"재귀"의 의미: 개방형 추론 루프가 아니라, 각 증거 제안이 이전 라운드가 생성한 증거 풀에 의존하는 제한된 추론 시점의 재귀입니다. 재생된 스캐폴드가 모델 상태를 변경하여, 이후 라운드에서 이전에 선택된 스팬과 관련된 새로운 증거가 표면화됩니다.

 단조적 개선 정리 (Theorem 1)

재귀적 증거 재생은 은닉 표현을 답변 임베딩 방향으로 단조적으로 이동시킨다는 것을 수학적으로 증명합니다.

ReContext는 학습된 검색기나 컨텍스트 pruning이 아닌, 내부 관련성 신호를 통한 질문-증거 재결합 (Query-Evidence Rebinding)으로 정식화됩니다.

핵심 발견

  • Qwen3-4B, Qwen3-8B, Llama3-8B 세 백본 모두에서 최고 평균 순위 달성
  • 학습 기반 방법들보다 학습 없이도 우수한 성능, 특히 다중 홉(multi-hop) 추론에서 두드러짐
  • 컨텍스트 압축 방법과 달리 미세 세부사항 손실 없이 증거 활용도 개선
  • 어텐션 개입 방법과 달리 모델 포워드 로직 변경 없이 적용 가능

절제 연구 및 효율성

절제 연구 (Ablation)

  • 재귀 라운드 수: 라운드를 증가시킬수록 성능이 단조적 향상 — 이론적 분석과 일치
  • 증거 풀 없이: 단일 라운드 선택만으로도 개선되나, 재귀가 추가 이득 제공
  • 원본 컨텍스트 제거: 증거 풀만 사용하면 성능 하락 — 전체 컨텍스트 보존의 중요성 확인
  • 토큰 vs 문장: 문장 수준 구체화가 토큰 단위보다 우수

시간 및 공간 효율성

추가 오버헤드: 어텐션 리드아웃과 토큰 선택은 경량 연산이며, 재귀 라운드당 한 번의 추가 포워드 패스만 필요

메모리: 원본 컨텍스트 KV 캐시를 재사용하므로 추가 메모리 최소화

Trade-off: 소수의 추가 포워드 패스로 24.6% 정확도 향상 — 효율성 대비 성능 이득이 뛰어남

결론 및 핵심 기여

방법론 기여

학습 불필요한 추론 시점 방법으로, 모델 내부 관련성 신호를 명시적 재귀 증거 풀로 변환하면서 전체 컨텍스트를 보존

이론적 기여

연합 기억 관점에서 ReContext를 정식화하고, 재귀적 증거 재생의 단조적 개선을 수학적으로 증명

실증적 기여

8개 128K 데이터셋에서 3개 백본 모두 최고 평균 순위 달성, Vanilla 대비 24.6% 상대적 정확도 향상

ReContext는 컨텍스트를 줄이거나 모델을 수정하는 대신, LLM이 이미 가진 능력을 하니싱하는 방법을 제시합니다. 더 긴 컨텍스트가 아니라, 더 나은 컨텍스트 활용을 위한 실용적인 접근입니다.
