배경

VLM 기반 구현 플래너의 부상

시각-언어 모델(VLM)이 로봇의 고수준 플래너로 널리 활용되고 있습니다. 추상적인 지시를 실행 가능한 하위 스킬로 분해하여 저수준 정책과 결합하는 계층적 구조가 표준으로 자리 잡고 있습니다.

추세: 테스트타임 연산 확장

추론 시 연산량을 늘려 성능을 끌어올리는 전략이 각광받고 있음

현실: 비용만 급증

지연 시간, 토큰 사용량, FLOPs가 증가하면서 실제 로봇 배포가 제한됨

무조건적인 연산 확장은 낭비다

축 1

사고연쇄 깊이: 깊이 vs. 지연

VLABench에서 Qwen3-VL 8B의 Instruct vs. Thinking 비교

관찰

  • Thinking이 평균 성능은 향상
  • 하지만 44%의 작업에서 Instruct가 동등 또는 우수
  • 해당 작업에서 지연은 2% 미만

시사점

많은 장면에서 추론 모델이 불필요한 비용을 발생시키며, 실시간 로봇 운영에서는 특히 문제가 됨

44% 의 작업에서 저비용 모델이 고비용 모델과 동등 이상의 성능 달성

축 2

모델 크기: 더 많은 스킬 명령

Qwen3-VL Instruct 2B → 235B 파라미터 스케일링 (VLABench)

비단조적 스케일링

  • 점수와 지연이 모두 비단조적
  • 32B가 235B보다 느린 경우도 존재
  • 파라미터 수 ≠ 토큰 생성량

스킬 폭의 확장

모델 크기의 진정한 역할은 스킬 커버리지 확장임. 대형 모델의 우위는 더 넓은 스킬 세트를 정확히 명령할 때만 발현됨

축 3

메모리 아키텍처: 문맥 오버헤드

RoboMME에서 5가지 메모리 압축 방식 비교

방식 접근 강점
FrameSamp 시각 토큰 감소 쉬운 작업에서 최소 FLOPs
TokenDrop 시각 토큰 드롭 가벼운 오버헤드
SimpleSG 언어 하위목표 요약 중간 난이도에 균형
GroundSG 기반 하위목표 요약 어려운 작업에서 강력
MemER 이전 키프레임 회상 긴 이력 작업에서 최고

단일 아키텍처가 모든 난이도에서 지배적이지 않음 → 난이도에 따른 적응적 선택 필요

핵심 발견

조사로부터 얻은 세 가지 트렌드

큰 능력 격차 존재

저비용과 고비용 설정 사이에 상당한 성능 차이가 있음

격차는 비균일

스킬 명령 요구, 회상 범위 등 작업 특정 수요에 좌우됨

경량 모델로 충분

많은 작업에서 고비용 설정이 불필요함

→ 정적 모델 선택은 단일 지연-능력 타협점만 고정함
→ 장면 맥락에서 작업 수요를 추론하고 연산을 동적 할당하는 프레임워크 필요

1

DIRECT의 핵심 구성 요소

데이터 수집 및 합성

  • 성공률-비용 파레토 프론티어 최적화

  • 세 축 각각에 독립적 라우터 구성 가능
  • 기존 플래닝 스택에 플러그인 방식 적용

결론

핵심 메시지

테스트타임 연산의 무조건적 확장은 낭비다.
DIRECT는 프론티어 수준의 구현 플래닝을
단편의 비용으로 제공할 수 있다.

  • 테스트타임 연산은 균일한 레버가 아님 — 각 축이 질적으로 다른 이득 제공
  • 다중모달 장면 맥락이 라우팅에 필수적 — 텍스트만으로는 부족
  • 경량 라우터가 고비용 모델의 성공률을 최대 65% 낮은 지연으로 재현
  • 기존 계층적 플래닝 스택에 플러그인 방식으로 적용 가능

한계: 후보 풀에 없는 모델로는 라우팅 불가, 라우터 학습 데이터 의존성, 실시간 장면 변화에 대한 적응 속도
