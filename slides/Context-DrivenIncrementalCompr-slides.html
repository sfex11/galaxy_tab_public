Context-Driven Incremental Compression for Multi-Turn Dialogue Generation

Yeongseo Jung, Jaehyeok Kim, Eunseo Jung, Jiachuan Wang, Yongqi Zhang, Ka Chun Cheung, Simon See, Lei Chen

문제 정의: 멀티턴 대화의 비용 폭발

계산 비용

전체 대화 기록을 매 턴마다 인코딩하면 self-attention 비용이 O(T³L²)로 증가합니다. 100턴 대화에서는 비용이 기하급수적으로 폭발합니다.

맥락 침식

대화가 길어지면 모델이 초기 맥락을 잊어버리고, 주제가 바뀌거나 관련 없는 응답을 생성하는 시맨틱 드리프트가 발생합니다.

"196턴 사이에 흩어진 저녁 파티 관련 언급을 회수해야 하는 상황에서, 기존 방법들은 모두 실패한다." — Figure 1 예시

핵심 관찰: 정적 압축의 취약성

연속 압축 붕괴

ICAE 기반 정적 압축기는 3~4회 연속 압축 후 Perplexity가 급격히 상승합니다. 압축 오차가 누적되어 정보가 소실됩니다.

갱신 메커니즘 부재

정적 압축은 한 번 생성된 잠재 표현을 수정할 수 없습니다. 주제가 재방문되거나 정보가 업데이트되어도 대응 불가합니다.

근본 원인

정적 압축기는 교차 턴 메모리 공유(cross-turn memory sharing)와 수정(revision) 메커니즘을 갖추지 못합니다. 대화의 동적 특성을 처리하려면 압축 상태를 지속적이고 수정 가능한 형태로 유지해야 합니다.

C-DIC: 개요

대화를 교차 주제 스레드(interleaved contextual threads)로 간주하고, 각 스레드별 압축 상태를 단일 컴팩트 대화 메모리에 저장합니다.

1
Query
현재 질문과 유사도 점수로 관련 스레드 상태 검색
2
Compress
검색된 상태 + 현재 턴을 새 스레드 상태로 요약
3
Write-back
기존 상태 갱신 또는 새 스레드 삽입 (기울기 없이)
  • 전체 기록 재인코딩 없이 점진적 압축
  • 토픽 전환 감지 후 자동 스레드 분기
  • 추론 시 기울기 불필요한 메모리 갱신

스레드 인식 메모리 검색

유사도 기반 검색

메모리 M에 저장된 각 스레드 상태와 현재 질문 qt 간의 코사인 유사도를 계산하여 최적 매칭 스레드를 찾습니다.

주제 전환 처리

최대 유사도 δt가 임계값 τ 미만이면 주제 전환으로 판단하고, 기존 스레드를 분리(detach)하여 새 스레드를 시작합니다.

메모리 M은 고정 크기의 잠재 벡터 집합으로, 초기 시스템 지시어를 첫 스레드 상태로 압축하여 초기화됩니다. 이후 매 턴마다 검색-압축-갱신 루프를 통해 진화합니다.

점진적 압축과 메모리 갱신 규칙

압축 과정

학습 가능한 압축기 fcomp가 검색된 스레드 상태 Rt, 현재 질문 qt, 응답 rt, 그리고 시스템 지시어 C를 입력받아 새로운 스레드 상태로 요약합니다. 이를 통해 미래 턴이 전체 기록을 재인코딩 없이 재사용 가능합니다.

주제 전환 → 삽입 (Insert)

δt < τ 일 때, 새 스레드 상태를 메모리에 새로운 슬롯으로 삽입합니다. 기존 스레드는 보존됩니다.

동일 주제 → 수정 (Revise)

δt ≥ τ 일 때, 최적 매칭 스레드 상태를 새로운 압축 결과로 대체하여 갱신합니다. 기울기 없이 수행됩니다.

학습: 검색 인식 TBPTT

전체 기록에 대한 역전파는 O(T) 메모리와 오차 누적을 초래합니다. C-DIC은 검색 인식 절단 역전파(ra-TBPTT)로 이를 해결합니다.

1-hop 기울기 전파

턴 t에서 기울기는 arg-max 사용 엣지를 따라 정확히 1홉만 역전파된 후 절단됩니다. 전체 히스토리 역전파가 불필요합니다.

오차 누적 완화

정적 압축기의 연쇄적 오차 누적 문제를 회피합니다. 각 턴의 손실이 독립적으로 최소화되며, 검색 경로를 통한 교차 턴 의존성을 학습합니다.

응답 생성기 fgen은 고정(frozen)되며, 검색된 상태 Rt만 조건부로 사용하여 응답 r̂t를 생성합니다. 압축기 fcomp만 학습됩니다.

장거리 진단과 지연시간 안정성

수백 턴 안정성

C-DIC은 수백 턴에 걸쳐 Perplexity가 안정적으로 유지됩니다. 반면, 전체 기록 방식은 턴이 길어질수록 지연시간이 선형적으로 증가하고, 정적 압축은 Perplexity가 폭발합니다.

일정한 추론 지연

메모리 크기가 고정되어 있어, 턴 수가 증가해도 추론 지연시간이 거의 일정하게 유지됩니다. 실제 서비스 배치에 적합한 확장성을 보여줍니다.

제한사항과 향후 방향

인식된 제한

  • 메모리 슬롯 수가 고정되어, 매우 많은 동시 주제 처리에 한계
  • 기울기 없는 메모리 갱신이 복잡한 주제 전환 패턴에 비최적적일 수 있음
  • 7B 백본에서만 검증; 더 큰 모델에서의 스케일링 미확인
  • 멀티모달 대화(이미지, 오디오)로의 확장 미탐구

가능한 확장

  • 동적 메모리 풀 크기 조절 메커니즘
  • 학습 가능한 메모리 갱신 규칙 도입
  • 13B~70B 모델로 스케일업 검증
  • 멀티모달 스레드 압축으로 확장

결론

정적 잠재 압축기가 대화 동역학 하에서 취약하다는 것을 실증적으로 보였습니다. 이를 극복하기 위해 Context-Driven Incremental Compression (C-DIC)을 제안합니다.

최초의 턴 수준 점진적 압축 프레임워크 — 단일 컴팩트 대화 메모리 내에서 수행

Retrieve → Revise → Write-back 루프로 교차 턴 공유 및 수정 달성

검색 인식 TBPTT로 전체 기록 역전파 없이 교차 턴 의존성 학습

수백 턴에서 안정적인 지연시간과 Perplexity를 보이며 고품질 대화 모델링의 확장 가능한 경로 제시
