01 · 문제 정의

왜 기존 추측적 디코딩은

추측적 디코딩 (Speculative Decoding)

경량 드래프트 모델이 출력을 제안하고, 강력한 타겟 모델이 이를 검증하여 추론 속도를 높이는 기법

문제 1: 토큰 중심 검증

개별 토큰이 그럴듯해도 전체 추론 단계가 틀릴 수 있음 — 오류가 연쇄적으로 전파됨

문제 2: 외부 보상 모델

기존 해결책(RSD)은 외부 프로세스 보상 모델(PRM)에 의존 — 추가 지연 및 일반화 한계

SpecGuard의 해답

외부 검증기를 완전히 제거하고, 모델의 내부 신호 두 가지(어텐션 기반 근거 + 로그 확률 신뢰도)를 앙상블하여 단계 수준 검증 수행

03 · 핵심 기법

두 가지 내부 검증 신호

1. 어텐션 기반 근거 검증 (ABGV)

출력 토큰이 입력 컨텍스트에 충분히 기반하는지 검사

R = A(L) · A(L-1) · ... · A(1)
Gstep = mint G(yi,t)   ← 가장 약한 토큰 기준
  • 마지막 3개 레이어만 사용 → 최적 정확도-지연 균형
  • 헤드 희소화: 0.01 미만 어텐션 값 제거 → 정확도↑ 속도↑

2. 로그 확률 기반 검증 (LPBV)

모델의 자체 신뢰도를 활용한 품질 측정

Lstep = mint log p(yi,t | x, yi,<t)

단계 내 최소 신뢰도 토큰 기준으로 전체 단계를 판단 — 가장 엄격한 기준 적용

앙상블 결합

두 점수를 [0,1]로 정규화 후 가중 결합. β=0.3으로 근거 검증(ABGV)에 더 높은 가중치 부여 → 최적 성능

04 · 자기 일관성 선택기

대표 추론 단계 선별

k개 후보 중 가장 대표적인 단계를 외부 감독 없이 선택하는 알고리즘

단계

  • k개 후보를 문장 트랜스포머로 임베딩
  • L2 정규화 후 쌍별 코사인 유사도 행렬 S 계산
  • 행별 소프트맥스 적용 → S̃
  • 대각 점수 dj = S̃jj 추출
  • dj가 최소인 후보 = 전체 분포의 대표

핵심 직관

자기 유사도가 가장 낮은 후보 = 다른 후보들과 고르게 유사한 후보

→ 특이한 극단값이 아닌, 분포의 중심에 위치한 답변을 선택

j* = argminj dj

복잡도: O(k² · d) per step

05 · 이론적 보장

형식적 보장

보조정리 1: 건전성 (Soundness)

각 신호가 독립적으로 ≥ 1−ε 확률로 정확한 단계를 수락하면:

Pr[수락 단계가 정확] ≥ 1 − (εl + εg)

보조정리 2: 효율성 (Efficiency)

기대 타겟 모델 호출 횟수는 수락 확률로 제한:

E[CT] = Σ(1−πi) ≤ T·(1−πmin)

정리 1: 정확도-효율성 트레이드오프

길이 T인 시퀀스에 대해:

Pr[모든 수락 단계 정확] ≥ (1−εl−εg)T · (1−δ)CT

→ 정확도와 효율성 사이의 수학적 트레이드오프를 형식적으로 보장

12 · 결론

결론 및 향후 과제

핵심 기여

  • 외부 프로세스 보상 모델을 두 가지 경량 내부 신호(어텐션 근거 + 로그 확률 신뢰도)로 대체
  • 단계 수준 검증으로 토큰 수준 오류 전파 문제 해결
  • 자기 일관성 선택기로 외부 감독 없이 대표 추론 경로 선별
  • 정확도-효율성 트레이드오프에 대한 형식적 이론 보장 제공

한계점

  • 구조화된 단계별 추론(수학)에만 평가 — 개방형 생성·창작에는 미검증
  • 단일 인스턴스 분석만 수행 — 대규모 배치 환경 미연구
  • 장기 대화 도메인에 대한 평가 부재

향후 연구 방향

개방형 생성 태스크 확장 · 대규모 배치 추론 최적화 · 다양한 도메인(코딩, 과학) 적용
