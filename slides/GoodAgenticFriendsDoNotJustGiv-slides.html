UCF · Westlake University · Snap Inc. · UT-Austin · Tencent

문제 제기

다중 에이전트 시스템의 소통 병목

다중 에이전트 LLM 시스템은 자연어 메시지로 협력하지만, 이는 신경망 간 소통에 비효율적인 인터페이스입니다.

텍스트 기반 소통의 문제

  • 중간 계산을 토큰으로 직렬화해야 함
  • 수신자가 토큰을 다시 인코딩하는 비용
  • Prefill 오버헤드 급증
  • KV-cache 메모리 팽창

핵심 질문

에이전트의 내부 상태를 수신자의 가중치 변화로 컴파일할 수는 없을까?

토큰 생성이 아닌, 계산 경로 수정으로 소통하는 새로운 패러다임

핵심 아이디어

가중치 공간 소통 (Weight-Space Communication)

송신자의 메시지를 텍스트로 전달하는 대신, 송신자의 은닉 상태를 수신자의 일시적 저랭크 가중치 변화로 변환합니다.

기존 방식

텍스트 → 컨텍스트 추가 → 재인코딩

잠재 공간 방식

히든 스테이트 → 활성화/캐시에 주입

TFlow (제안)

히든 스테이트 → LoRA 변화 → 계산 경로 수정

소통 객체는 문장도 아니고, 캐시에 주입되는 히든 스테이트도 아닙니다. 수신자 파라미터 공간에서의 쿼리별 업데이트입니다.
비교 분석

TextMAS vs Weight-Collaboration MAS

측면 TextMAS TFlow (제안)
전달 매체 자연어 토큰 LoRA 가중치 변화
수신자 컨텍스트 메시지로 확장됨 변경 없음
Prefill 비용 메시지 길이에 비례 증가 추가 비용 없음
KV-cache 메모리 팽창 최소화
호환성 범용 (아키텍처 무관) 수신자 아키텍처 고정 필요
해석 가능성 높음 (인간이 읽을 수 있음) 낮음 (가중치 수준)
방법론

TFlow: Thought Flow 프레임워크

고정된 수신자 아키텍처를 전제로, 송신자 은닉 상태를 수신자 맞춤형 LoRA 변화로 변환하는 실행 가능한 소통 채널입니다.

1
송신자 처리
프리징된 에이전트들이 역할 프롬프트와 함께 입력을 처리
2
은닉 상태 추출
각 송신자의 내부 활성화를 조건 신호로 수집
3
파라미터 생성
학습된 생성기가 은닉 상태를 LoRA 인자로 매핑
4
일시적 주입
수신자 생성 시에만 변화를 적용, 이후 복원

모든 LLM 백본은 프리징된 채 학습되는 것은 수신자 전용 파라미터 생성기뿐입니다.

방법론

송신자 조건화 (Sender Conditioning)

송신자의 다층 은닉 상태를 효율적으로 집계하여 파라미터 생성기에 전달합니다.

학습 가능한 층 집계

  • 송신자의 L개 층에서 은닉 상태 수집
  • 학습 가능한 가중치로 층별 가중합
  • 각 송신자별 독립적인 집계 파라미터
  • 다중 송신자 조건을 결합하여 생성기 입력 구성

조건화 신호의 형태

송신자 S개의 집계된 은닉 상태가 결합되어 토큰 그리드(Token Grid) 형태의 초기화 입력으로 변환됩니다. 이는 이후 파라미터 생성기의 트랜스포머 트렁크에 입력됩니다.

방법론

파라미터 생성기: 3단계 구조

1

초기화 (Initialization)

송신자 조건 신호를 토큰 그리드 형태로 재구성. 다중 축(층, 모듈, 송신자)을 따라 구조화된 텐서 생성

2

신경망 트랜스포머 (Neural Network Transformer)

다중 축 트랜스포머 트렁크가 토큰 그리드를 처리. 층-모듈 간 의존성을 포착하여 목표 LoRA 인자에 대응하는 표현 학습

3

비토큰화 (Detokenization)

트랜스포머 출력을 수신자 타겟 레이어에 맞는 LoRA 인자 쌍 (A, B 행렬)로 변환. 저랭크 제약(r << d) 적용

방법론

일시적 주입 (Transient Injection)

생성된 LoRA 변화는 수신자의 생성 단계에서만 일시적으로 적용됩니다.

동작 원리

  • 수신자의 타겟 선형 레이어 순방향 패스를 일시적으로 패치
  • 다중 송신자 변화를 결합하여 단일 LoRA로 융합
  • 수신자는 송신자가 작성한 메시지를 텍스트 컨텍스트에서 전혀 보지 않음
  • 생성 완료 후 프리징된 기본 모델로 복원

보장되는 특성

  • 기본 모델 파라미터 영구 변경 없음
  • 인스턴스별 적응 (쿼리마다 다른 변화)
  • 수신자 컨텍스트 길이 증가 없음
  • 재사용 가능: 다른 쿼리에 다른 변화 적용
방법론

학습 목표

엔드투엔드 최적화

전체 파이프라인(송신자 → 조건화 → 파라미터 생성 → 일시적 주입 → 수신자 생성)을 통과하는 그래디언트 흐름으로 학습합니다.

LLM 백본은 모두 프리징되며, 오직 파라미터 생성기와 층 집계 가중치만 업데이트됩니다.

다양성 정규화기

생성된 가중치 변화가 모든 인스턴스에서 동일한 방향으로 수렴하는 것을 방지합니다.

캐시 기반 정규화로 이전 배치의 변화와 현재 변화 간 유사성을 페널티하여, 인스턴스별로 구별되는 변화를 유도합니다.

실험

실험 설정

백본 및 구성

  • 백본: Qwen3-4B (프리징)
  • 3 에이전트 설정 (2 송신자 + 1 수신자)
  • 송신자: 역할 프롬프트 부여
  • LoRA 랭크: 8, 타겟: QKV/FFN 레이어

벤치마크 (5개)

  • GSM8K — 수학 추론
  • Arc-Challenge — 과학 지식
  • MMLU-Pro — 다영역 지식
  • BBH — 하드 추론
  • HumanEval+ — 코드 생성

분석

인스턴스 수준 분석

히든 스테이트 증거

  • 학습된 층 가중치가 의미 있는 패턴을 보임
  • 쉬운/어려운 인스턴스 간 쌍별 유사성이 체제별로 구별됨
  • 생성기가 입력 난이도에 따라 다른 변화를 생성하는 것을 확인

LoRA 텐서 증거

  • 인스턴스마다 구별되는 LoRA가 생성됨
  • 동일 도메인 내에서는 유사한 기하 구조를 공유
  • 도메인 간에는 다른 방향성을 가짐
TFlow는 단순히 평균적인 변화를 적용하는 것이 아니라, 각 쿼리의 특성에 맞춤화된 가중치 변화를 생성합니다.
절제 연구

인스턴스 특이성의 중요성

가중치 변화가 쿼리별로 다르게 생성되어야 한다는 것을 절제 연구로 확인했습니다.

설정 설명 성능
TFlow (정상) 인스턴스별 LoRA 생성 최고 성능
Static LoRA 모든 인스턴스에 동일한 LoRA 적용 성능 저하
Mismatched Injection 다른 인스턴스의 LoRA를 잘못 주입 더 큰 성능 저하

Static LoRA보다 Mismatched Injection이 더 나쁜 성능을 보이는 것은, 잘못된 변화가 없는 변화보다 해롭다는 것을 의미합니다. 인스턴스 특이성이 TFlow의 핵심입니다.

결론

요약 및 시사점

새로운 소통 패러다임

자연어가 아닌 일시적 저랭크 가중치 변화를 에이전트 간 소통 매체로 사용하는 가중치 공간 소통을 제안

TFlow 프레임워크

송신자 히든 스테이트를 수신자 맞춤형 LoRA로 변환하여, 텍스트 컨텍스트 확장 없이 인스턴스별 적응 달성

실질적 효율 개선

토큰 최대 83% 절감, 속도 최대 4.6배 향상하면서도 경쟁력 있는 정확도 유지

한계: 수신자 아키텍처 고정 필요, 해석 가능성 부족, HumanEval+에서의 성능 격차. 향후 동적 아키텍처 지원과 혼합 소통(텍스트+가중치)이 연구 방향으로 제시됩니다.
