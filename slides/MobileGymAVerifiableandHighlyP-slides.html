  • 재현 가능한 평가
  • 프로그램적 검증 가능
  • 시스템 유틸/오픈소스 앱에 한정

실기기 기반

  • 일상 앱 커버리지
  • 리얼리즘 우수
  • 백엔드 상태 불가독/불가쓰
  • 앱 버전 드리프트 문제
  • 되돌릴 수 없는 위험 연산

일상 앱의 상태는 읽을 수 없고, 쓸 수 없고, 포크할 수 없다 — 이것이 구조적 한계다.

핵심 아이디어: 상호작용 충실도

GUI 에이전트는 스크린샷만 관찰하고 이산 액션만 수행한다. 백엔드를 복제할 필요 없이, 상호작용 충실도만 보장하면 된다.

01

상태 프로그래밍 가능성

전체 환경 상태가 구조화된 JSON으로 표현되어 4가지 핵심 속성을 동시에 달성한다.

Readable — 읽기 가능

내부 상태(잔액, 주문 등)를 JSON으로 직접 조회하여 VLM 없이 결정론적 판정

Writable — 쓰기 가능

JSON 수정만으로 원하는 초기 상태 구성 및 에피소드 리셋

Forkable — 포크 가능

동일 초기 상태에서 여러 롤아웃 포크 — GRPO 등 그룹 기반 RL에 필수

Sandboxed — 샌드박스

실제 메시지/송금/계정 변경 없이 고위험 연산 안전하게 시뮬레이션

AnswerSheet 프로토콜

쿼리 태스크에서 자유 텍스트 매칭의 취약성을 해결하기 위한 구조화된 답변 제출 방식

기존 방식의 문제

  • VLM 판사: 스크린샷만으로 부분 증거
  • 자유 텍스트 매칭: "12,345원" vs "12345"
  • 10.2% 오판정률 (VLM judge audit 결과)

AnswerSheet 방식

  • 타입 필드: 텍스트, 숫자, 선택지, 날짜 등
  • GUI를 통해 정형화된 답변 제출
  • Matcher 기반 결정론적 정답 검증

단일 프로그램적 판정 메커니즘이 벤치마크 평가 verdict와 RL dense reward를 동시에 제공

한계점 및 논의

시각적 충실도

실제 앱의 픽셀 수준 렌더링을 완벽히 재현하지는 않음. 상호작용 충실도에 초점

백엔드 모델링

상용 백엔드 로직을 완전히 복제하지 않으며, 동적 콘텐츠 생성에 한계

법적 고려

상용 앱의 시뮬레이션이 저작권법적으로 허용 가능한지에 대한 논의 필요

고위험 연산 평가

안전한 시뮬레이션으로 고위험 능력을 평가하는 것의 양면성 인지 필요

확장 활용 가능성

커스텀 모바일 환경 구축

모듈형 아키텍처로 연구자가 자체 앱과 벤치마크를 쉽게 추가 가능

통제된 강건성/안전성 평가

의도적으로 악의적 UI나 엣지 케이스를 주입하여 에이전트 견고성 테스트

저비용 온라인 RL 연구

상태 포크 + 병렬 롤아웃으로 GRPO, PPO 등 온라인 RL 알고리즘 실험 용이

통제된 학습 데이터 합성

다양한 초기 상태에서 고품질 트레이젝토리를 대규모로 생성

요약

MobileGym은 상호작용 충실도라는 원칙으로, 일상 앱의 상태를 읽고 쓰고 포크할 수 있는 경량 브라우저 환경을 제공한다.

  • 구조화된 JSON 상태로 결정론적 판정과 RL 보상 동시 제공
  • 인스턴스당 ~400MB, ~3초 콜드 스타트로 수백 개 병렬 롤아웃
  • 416개 파라미터화 태스크, 28개 앱, L1–L4 난이도 계층
  • GRPO 학습으로 +12.8pt, 실기기 전이율 95.1% 검증
  • VLM 판사 10.2% 오판정을 상태 기반 판정으로 완전 제거
