왜 ClawBench가 필요한가?

AI 에이전트가 이메일을 자동화할 수 있다면, 일상의 다른 온라인 작업도 자동화할 수 있을까?

🏗️
기존 벤치마크의 한계
오프라인 샌드박스 환경의 정적 페이지만 테스트 — 실제 웹의 복잡성을 반영하지 못함
🌐
실제 환경 테스트 부재
프로덕션 웹사이트에서의 쓰기 작업(폼 작성, 결제 등)을 평가하는 벤치마크가 없음
📄
문서 정보 활용
사용자 문서에서 정보를 추출하여 다단계 워크플로우를 수행하는 능력 평가 필요
📝
복잡한 양식 작성
다양한 플랫폼에서 상세한 양식을 정확하게 작성하는 쓰기 중심 작업 평가 필요

핵심 발견

주요 결론

🔴 실제 환경 역량 격차
기존 벤치마크에서의 강한 성능이 ClawBench로 전이되지 않음 — 통제된 환경과 실제 웹 사이에 상당한 격차가 존재
🔵 카테고리별 불균일한 성능
모델들이 모든 영역에서 균일한 역량을 보이지 않음 — 특정 카테고리에서만 강점을 보이며, 최고 성과도 포화와는 거리가 멂
🟡 상용 vs 오픈소스 모두 한계
독점 모델과 오픈소스 모델 모두 제한된 역량을 보여줌 — 범용 AI 비서로서의 실제 웹 작업 자동화는 아직 미해결 과제
🟣 추적 가능한 실패 진단
에이전트 평가자를 통해 어떤 필드나 단계가 잘못되었는지 정확히 추적 가능 — 미래 에이전트 개선의 구체적 방향 제시
