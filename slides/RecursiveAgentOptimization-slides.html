재귀적 에이전트 최적화

Recursive Agent Optimization (RAO)

강화학습을 활용하여 자신의 복제본을 생성하고 하위 작업을 위임하는
재귀적 에이전트를 훈련하는 새로운 접근법

문제 제기

기존 에이전트의 한계

단일 에이전트 구조

하나의 에이전트가 모든 작업을 처리해야 하므로, 복잡한 다단계 작업에서 성능이 저하됩니다.

확장성 부족

추론 시점에 계산량을 유연하게 늘리는 inference-time scaling이 자연스럽게 지원되지 않습니다.

고정된 위임 구조

기존 다중 에이전트 시스템은 위임 구조가 사전에 고정되어 있어 유연성이 떨어집니다.

훈련의 어려움

재귀적 행동을 학습하기 위한 체계적인 훈련 방법론이 부재합니다.

핵심 개념

재귀적 에이전트란?

"에이전트가 자신의 새로운 인스턴스를 생성(spawn)하고, 하위 작업을 재귀적으로 위임할 수 있는 에이전트"

1

생성 (Spawn)

자신과 동일한 구조의 새 에이전트 인스턴스를 동적으로 생성

2

위임 (Delegate)

복잡한 작업을 더 작은 하위 작업으로 분해하여 위임

3

재귀 (Recurse)

하위 에이전트도 동일하게 재귀적으로 작업을 분해 가능

제안 방법

RAO: 재귀적 에이전트 최적화

RAO는 강화학습(RL)을 기반으로 재귀적 에이전트를 end-to-end로 훈련하는 프레임워크입니다. 에이전트는 언제 하위 에이전트를 생성하고, 어떤 작업을 위임할지 스스로 학습합니다.

1
작업 수신
상위 에이전트로부터 작업 입력
2
분해 결정
RL 정책에 따라 하위 작업 분해 여부 판단
3
재귀 위임
새 인스턴스 생성 후 하위 작업 위임
4
결과 통합
하위 에이전트 결과를 취합하여 응답 생성
핵심 메커니즘

추론 시점 확장 (Inference-Time Scaling)

재귀적 에이전트는 추론 시점에 계산량을 자연스럽게 조절할 수 있는 구조적 장점을 가집니다.

적은 계산량

  • 재귀 깊이를 낮게 설정
  • 에이전트가 직접 작업 수행
  • 빠른 응답 시간

많은 계산량

  • 재귀 깊이를 높게 설정
  • 다단계 작업 분해 수행
  • 더 정확한 결과 도출

추가 훈련 없이도 재귀 깊이만 조절하여 성능-비용 트레이드오프를 제어할 수 있습니다.

훈련 방법론

RAO 훈련 과정

초기화

기본 LLM을 에이전트 정책으로 초기화. 재귀적 행동을 위한 액션 공간(spawn, delegate, solve) 정의.

환경 상호작용

에이전트가 작업을 수행하며 재귀적으로 하위 에이전트를 생성하고 위임하는 trajectory를 생성.

보상 설계

최종 작업 성공 여부뿐만 아니라, 효율적인 재귀 분해와 자원 사용에 대한 보상 신호를 제공.

정책 업데이트

강화학습 알고리즘(GRPO 등)을 통해 재귀적 행동 정책을 반복적으로 개선.

설계 의사결정

재귀 분해 전략

RAO에서 에이전트는 어떻게 작업을 분해할지를 학습해야 합니다. 이는 사전 정의된 규칙이 아닌 RL을 통해 발견되는 전략입니다.

분해 시점 결정

에이전트가 직접 해결하기 어려운 작업을 인식하고, 적절한 시점에 재귀 분해를 트리거하는 능력을 학습합니다.

하위 작업 정의

상위 작업을 독립적이고 해결 가능한 하위 작업으로 자연스럽게 분해하는 프롬프트를 생성하는 방법을 학습합니다.

종료 조건 학습

불필요한 재귀를 방지하고, 작업이 충분히 단순해지면 직접 해결하도록 적절한 종료 조건을 학습합니다.

결과 통합

하위 에이전트들의 결과를 효과적으로 취합하여 상위 작업의 최종 답변을 구성하는 전략을 학습합니다.

핵심 장점

RAO의 주요 이점

자연스러운 추론 시점 확장

재귀 깊이를 조절하는 것만으로 더 많은 계산을 투입하여 성능을 향상시킬 수 있습니다.

학습된 분해 전략

인간이 설계한 규칙이 아닌, RL을 통해 발견된 최적의 작업 분해 방식을 사용합니다.

단일 정책의 재사용

모든 재귀 단계에서 동일한 정책을 사용하므로, 하나의 모델로 전체 계층 구조를 처리합니다.

효율성과 정확도의 균형

보상 설계를 통해 불필요한 재귀를 억제하고, 실제로 필요한 경우에만 분해하도록 유도합니다.

응용 분야

RAO의 활용 시나리오

복잡한 정보 검색

다단계 조사가 필요한 질문을 하위 검색 작업으로 분해하여 정확한 정보 수집

소프트웨어 개발

큰 프로젝트를 모듈 단위로 분해하고 각 모듈을 독립적인 에이전트가 구현

수학적 추론

복잡한 수학 문제를 보조 정리와 하위 증명으로 재귀적으로 분해하여 해결

계획 수립

장기 목표를 단기 목표로 계층적으로 분해하여 실행 가능한 계획 생성

다중 문서 요약

대규모 문서 집합을 개별 문서 요약 후 계층적 통합으로 효율적으로 요약

자율 시스템

로봇 제어 등 복잡한 환경에서 센서-액추에이터 계층을 재귀적으로 관리

구조적 이해

재귀 호출 트리 예시

복잡한 작업이 재귀적으로 분해되는 과정을 시각화한 구조입니다.

Root Agent ── "시장 분석 보고서 작성" ├── Sub-Agent 1 ── "경쟁사 분석" │ ├── Sub-Agent 1.1 ── "A사 재무제표 분석" │ └── Sub-Agent 1.2 ── "B사 제품 포트폴리오 분석" ├── Sub-Agent 2 ── "시장 트렌드 분석" │ ├── Sub-Agent 2.1 ── "최근 3개월 뉴스 요약" │ └── Sub-Agent 2.2 ── "소비자 설문 데이터 분석" └── Sub-Agent 3 ── "결론 및 제언 도출"

각 노드는 동일한 정책을 공유하지만, 독립적인 컨텍스트와 상태를 가진 에이전트 인스턴스입니다.

한계점 및 향후 방향

현재 한계

  • 재귀 깊이 증가에 따른 지연 시간 누적
  • 하위 에이전트 간 정보 중복 및 불일치 가능성
  • 에러가 재귀 트리 전체로 전파될 위험
  • 보상 설계의 복잡성 및 신호 희석 문제

향후 연구 방향

  • 병렬 재귀 실행을 통한 지연 시간 최적화
  • 하위 에이전트 간 통신 메커니즘 설계
  • 계층형 보상 shaping 기법 개발
  • 더 다양한 도메인에 대한 일반화 성능 검증
요약

핵심 Takeaway

재귀적 에이전트는 새로운 패러다임입니다

에이전트가 자신의 복제본을 생성하고 작업을 위임하는 구조는 기존 고정 아키텍처의 한계를 극복합니다.

RL이 재귀적 행동을 학습합니다

RAO는 강화학습을 통해 언제, 어떻게 작업을 분해할지를 데이터 기반이 아닌 경험 기반으로 학습합니다.

Inference-time scaling이 구조적으로 내재됩니다

재귀 깊이 조절이라는 직관적이고 자연스러운 방식으로 추론 시 계산량을 확장할 수 있습니다.

단일 정책으로 계층적 문제 해결

하나의 훈련된 정책이 재귀 트리의 모든 레벨에서 재사용되어 효율적이고 일관된 문제 해결을 달성합니다.
