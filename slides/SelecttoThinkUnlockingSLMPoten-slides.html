Select to Think

Local Sufficiency로 SLM의 잠재력 해제하기

Wenxuan Ye · Yangyang Zhang · Xueli An · Georg Carle · Yunpu Ma

문제 정의

SLM의 추론 한계

SLM의 장점

  • 높은 연산 효율성
  • 대규모 배포 가능
  • 낮은 지연 시간

SLM의 한계

  • 다단계 추론 성능 부족
  • LLM 대비 추론력 격차
  • 복잡한 작업에서 오류 누적

SLM은 효율적이지만, 복잡한 추론에서 LLM의 성능을 따라가지 못합니다. 이 격차를 어떻게 줄일 것인가?

기존 접근의 딜레마

두 가지 경로, 두 가지 한계

경로 A: 협력적 추론

발산 지점에서 LLM을 호출하여 토큰 생성

!
문제: 동기식 LLM 호출로 인한 상당한 지연과 비용

경로 B: 표준 증류

LLM의 출력 분포를 SLM이 모방하도록 학습

!
문제: 용량 격차로 인한 취약한 분포 매칭

협력적 추론은 비용 문제를, 표준 증류는 용량 문제를 겪습니다. 둘 다 근본적인 한계에 부딪힙니다.

핵심 발견

Local Sufficiency

발산 지점에서 LLM이 선호하는 토큰은 SLM의 Top-K 후보 안에 이미 존재한다

95%

1.5B SLM의 Top-8 후보가 32B LLM의 선택을 포함하는 비율

SLM이 Top-1로 선택하지 못했더라도, 올바른 토큰이 후보군 내에 존재합니다.
즉, 새로운 토큰을 생성할 필요 없이, 후보 중에서 선택만 하면 됩니다.

프레임워크

Select to Think (S2T) 구조

1

발산 감지

트리거 헤드가 SLM-LLM 불일치를 예측하여 개입 시점 결정

2

후보 생성

SLM이 Top-K 후보 토큰 집합을 생성

3

선택 개입

LLM이 SLM 후보 중 최적 토큰을 선택 (개방형 생성이 아닌 선택)

핵심 차이점: LLM은 토큰을 "생성"하지 않고 SLM이 제안한 후보 중에서 "선택"합니다.

핵심 메서드

S2T-Local: 교사 없는 내면화

선택 논리를 SLM 내부로 증류하여 추론 시 LLM 호출 없이 자율 재정렬

ZIP 기반 설계

  • 예약 토큰(Reserved Tokens)의 logit을 활용
  • 생성 분포는 그대로 유지
  • 선택 점수만 별도로 인코딩

생성-평가 분리

  • 표준 어휘 통계에 영향 없음
  • 학습된 "내면 비평가"로 자가 평가
  • 추론 시 외부 오버헤드 제로

핵심 아이디어: 확률 직접 정렬은 원래 분포를 왜곡하므로, 선택 신호를 생성과 완전히 분리합니다.

메커니즘 상세

예약 토큰을 활용한 선택 인코딩

1
SLM 순방향 패스
전체 어휘 logit 계산 (예약 토큰 포함)
2
예약 토큰 추출
예약 토큰 ℛ의 logit만 분리
3
선택 점수 계산
ψ: ℝ^|ℛ| → ℝ 매핑으로 스칼라 점수 산출
4
재정렬
선택 점수로 Top-K 후보 순위 재조정

예약 토큰은 생성 시 마스킹되어 텍스트 출력에 영향을 주지 않지만, 그 logit 값은 여전히 계산됩니다. S2T-Local은 이 "숨겨진" 신호를 활용하여 추가 연산 없이(Zero-overhead) 선택 기능을 구현합니다.

검증

Local Sufficiency 실증 분석

모델 스케일별 Hit Rate

SLM 크기 Top-4 Top-8 Top-16
0.5B 87% 93% 97%
1.5B 89% 95% 98%
3B 92% 97% 99%

주요 관찰

  • 0.5B에서도 Top-8 Hit Rate 93%
  • 모델 스케일·아키텍처 무관하게 일관된 현상
  • 증류된 선택기는 held-out에서 LLM과 67% 일치율 달성

SLM이 "무엇을 생성할지 모르는 것"이 아니라 "무엇을 먼저 선택할지 모르는 것"입니다.
