Xiao Ye, Jacob Dineen, Evan Zhu, Shijie Lu, Kevin Song, Ben Zhou

Arizona State University

LLM 예측 평가의 근본적 문제

확률적 예측(calibrated forecasting)은 LLM의 불확실성 하 추론 능력을 검증하는 표준 방법으로 자리 잡고 있습니다. 하지만 현재 백테스팅 방식에는 치명적인 누출 경로가 두 가지 존재합니다.

검색 경로 누출

검색 증강 모델이 이벤트 이후에 작성된 보고서를 검색 결과로 가져옵니다. 예측이 아닌 조회(lookup)로 변질됩니다.

파라미터 누출

새로운 모델일수록 이벤트에 더 가까운 학습 데이터로 훈련됩니다. 작년엔 미래였던 질문이 올해는 학습 데이터 안에 들어 있습니다.

"이 평가는 회상(recall)을 측정하면서 선견(foresight)을 측정한다고 주장한다."

직관적 예시: 2022 월드컵

"아르헨티나가 2022 월드컵을 우승할까?" — 오늘 묻면 거의 확정에 가깝게 대답합니다. 이 확신은 토너먼트 이후 기사에서 옵니다.

핵심 질문: 우리가 진짜 측정하고 싶은 것은 과거 시점 t₀에서 모델이 할당했을 반사실적 확률입니다. 그것만이 당시 행동 가능했던 유일한 예측입니다.

Hindcast: 핵심 아이디어

과거 특정 시점 t₀로 돌아가, 그 시점에서 접근 가능했던 정보만으로 예측을 평가합니다. 두 가지 누출 경로를 동시에 차단합니다.

불변 아카이브
  • 공개 Reddit 월별 스냅샷 사용
  • 캡처된 달은 영구 보존 (불변)
  • t₀ 이전 게시물만 접근 허용
시장 가격 기준선
  • 해결된 Polymarket 시장 활용
  • t₀ 시점의 시장 암시 확률 q_{t₀}
  • 동일 정보 환경의 인간 예측 비교

t₀는 자유 변수입니다. 새 모델이 나오면 새로 해결된 시장에 대해 재실행 가능하므로 벤치마크가 낡지 않습니다.

평가 축 1: 결과 회복

Brier Score, 정확도 — 실제 해결 결과 y에 대해 예측 확률 p̂_{t₀} 채점

평가 축 2: 시장 대비 우위

|p̂_{t₀} − q_{t₀}| — 동일 시점 시장 가격 대비 모델 예측의 편차 측정

검색 백엔드: Reddit 아카이브

46M

누적 게시물 수

12K+

서브레딧 커버리지

월별

불변 스냅샷 단위

왜 Reddit인가?

불변성

웹페이지나 위키백과와 달리, 캡처된 월은 영구적으로 보존되어 사후 수정 불가

조기 신호

뉴스 스냅샷보다 이벤트를 더 일찍, 더 다양한 관점에서 논의

Cutoff 강제 방식

모든 검색 쿼리에 t₀ 필터를 주입. 시스템 프롬프트와 검색 API 호출 양쪽에서 이중으로 시간 제한 적용

평가 세팅

평가셋 구성
  • Polymarket 해결된 이진 시장
  • 커버리지 테스트: Reddit에서 관련 문서 존재 여부 분류
  • 결과(YES/NO) 및 주제 균형 맞춤
  • Lookback fraction: 시장 수명의 비율로 t₀ 설정
지표 체계
  • Brier Score: 확률 예측의 정확도 (↓ 좋음)
  • Accuracy: 0.5 기준 이진 분류 정확도
  • |p̂ − q|: 시장 암시 확률 대비 편차
  • Cutoff: t₀를 시장 종료 1개월 전으로 고정

커버리지 테스트는 각 시장에 대해 Reddit 아카이브가 t₀ 이전에 실제로 관련 정보를 포함하는지를 사전에 분류하여, 검색의 조건부 효과를 분리할 수 있게 합니다.

두 가지 예측 파이프라인

Zero-shot Baseline

검색 없이 모델 파라미터만 사용

  • 시장 질문 + 해결 기준만 입력
  • 확률 + 짧은 근거 요구
  • 파라미터 메모리 고립 측정
Retrieval Agent

t₀ 제한 Reddit 아카이브 검색 추가

  • 연구 단계: 관련 문서 검색 후 요약
  • 예측 단계: 요약 기반 확률 산출
  • 증거 기반 추론 기여도 측정

두 파이프라인의 차이 = 검색이 제공하는 증거의 순수 기여도
(누출이 통제된 상태에서만 의미 있는 비교)

핵심 실험 결과

9개 오픈 가중치 모델에 대해 누출 통제 하 평가를 수행한 결과:

8/9

모델에서 검색이 Brier 개선

조건부

개선은 커버리지에 의존

역전

추측만 있는 곳에서는 악화

이전 연구들이 오염 가능성 아래 보고한 검색 기반 개선이 완전히 사라지지는 않았지만, 그 이득의 실체가 달라집니다.

조건 검색 효과 원인
커버리지 있음 Brier 개선 t₀ 전 실제 관련 정보 활용
커버리지 없음 Brier 악화 추측성 잡담에 과신

검색이 도움이 되는 곳과 해가 되는 곳

검색이 도움되는 경우
  • Reddit가 사건을 사전에 구체적으로 논의한 시장
  • 전문 서브레딧의 기술적/법적 분석
  • 현장 증언, 실시간 논의 등 1차 정보
  • 사건 결정적 요인이 t₀ 전에 알려진 경우
검색이 해가 되는 경우
  • 아카이브에 추측성 잡담만 있는 시장
  • 소문, 감정적 반응, 근거 없는 예측
  • 에이전트가 잡음에 과신(over-read)하는 현상
  • 사건 결정 요인이 t₀ 후에 발생한 경우

"검색은 돕지만, t₀ 이전 코퍼스가 사건을 결정한 내용을 이미 논의한 경우에만 그렇다. 추측만 있는 곳에서는 역효과가 난다."

모델별 효과와 정확도 vs 보정

9개 오픈 가중치 모델 전반에서 일관된 패턴이 확인되며, 정확도(accuracy)와 보정(calibration) 사이에 트레이드오프가 관찰됩니다.

모델 간 공통 패턴
  • 대부분 모델에서 검색이 Brier 개선
  • 개선 폭은 모델별로 차이 존재
  • 더 큰 모델이 항상 더 나은 것은 아님
정확도 vs 보정
  • 높은 정확도 ≠ 좋은 보정
  • 과신(confidence) 높은 모델은 Brier 악화
  • 보정이 정확도보다 예측 품질 지표로 적합

예측자의 한계는 모델 자체가 아니라 증거 환경(evidence environment)에 있다

관련 연구와의 차별성

접근법 누출 통제 시간 축 비교 기준
기존 백테스팅 없음 현재 시점 결과만
학습 데이터 재훈련 파라미터만 컷오프 고정 결과만
뉴스 스냅샷 고정 검색만 단일 스냅샷 결과만
FutureSim 부분 공유 윈도우 순방향 적응 측정
Hindcast 검색 + 파라미터 시장별 t₀ 결과 + 시장 가격

Hindcast는 재훈련 방식(파라미터 통제)과 스냅샷 방식(검색 통제)을 모두 포괄하며, 두 채널을 동시에 차단하고 인간 예측 기준선까지 추가합니다.

기여 요약 및 핵심 시사점

1
Hindcasting 프로토콜

과거 t₀에서 검색 가능 정보를 고정한 채 모델 예측을 평가. 새 모델에 대해 재실행 가능한 영구 벤치마크.

2
불변 Reddit 아카이브

월별 불변 스냅샷으로 임의 과거 시점에 대한 누출 없는 검색 환경 제공.

3
두 가지 참조 예측기

Zero-shot baseline과 Retrieval agent로 검색의 순수 기여도 분리 측정.

4
실증적 발견

검색은 커버리지가 있는 곳에서 도움되고, 추측만 있는 곳에서 역효과. 예측자의 한계는 모델이 아닌 증거 환경에 있다.

"회상을 선견이라 부르지 말자. Hindcast가 그 구별을 가능하게 한다."
