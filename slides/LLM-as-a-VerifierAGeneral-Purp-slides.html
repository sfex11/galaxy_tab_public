사후 학습 스케일링

추론 시 스케일링

테스트 시 더 많은 컴퓨트를 활용한 Chain-of-Thought 등

생성은 스케일링의 혜택을 크게 받았지만, 검증은 동등한 수준의 스케일링을 경험하지 못했습니다.

문제 정의

기존 검증 방법의 한계

표준 LM 판사

  • 이산적 점수만 산출 → 동점 발생 빈번
  • 복잡한 솔루션 간 분별력 부족
  • 평가 불확실성을 무시

학습된 보상 모델

  • 학습 데이터에 제한됨
  • 도메인 간 일반화 어려움
  • 추가 학습 비용 발생

검증은 별개의 스케일링 축으로서 아직 충분히 탐구되지 않았습니다.

제안 방법

LLM-as-a-Verifier 핵심 아이디어

이산적 점수 대신, 스코어링 토큰 로짓의 전체 분포에 대한 기댓값을 계산하여 연속적 점수를 생성하는 확률적 공식화를 도입합니다.

기존 방식 (이산)

"1~5점 중 하나를 선택" → 동점률 높음, 정보 손실

제안 방식 (연속)

토큰 로짓 분포의 기댓값 → 동점률 감소, 미세한 차이 구별

핵심 장점: 추가 학습 없이, 어떤 모달리티에도 적용 가능한 범용 프레임워크

검증 스케일링

세 가지 스케일링 차원

점수 세분화

더 많은 토큰 로짓 추출 → 신호 대 잡음비 향상 → 양성/음성 분리 개선

반복 평가

다회 평가 수행 → 평가 분산 감소 → 검증 정확도 향상

기준 분해

평가 기준을 세분화 → 프롬프트 편향 감소 → 복잡도 낮춤

세 차원 모두 일관되게 검증 정확도 개선 — 통제된 예산 하에서도 이산적 LM 판사 대비 우수

알고리즘

확률적 피벗 토너먼트

검증 스케일링을 실용적으로 만들기 위해, 검증자의 연속 점수로부터 도출된 선호 확률을 활용한 비용 효율적 순위 알고리즘을 제안합니다.

1
후보 생성
다수 솔루션 트라젝토리 수집
2
연속 점수 산출
로짓 분포 기댓값 계산
3
선호 확률 추정
쌍별 비교 확률 도출
4
피벗 토너먼트
최적 솔루션 선택

전체 쌍을 비교하는 대신 피벗 기반 토너먼트로 비용을 크게 절감하면서도 높은 정확도 유지

응용 1

과제 진행률 추정 프록시

LLM-as-a-Verifier의 미세한 점수 신호는 에이전트의 과제 진행 상황과 강한 상관관계를 보입니다.

코드 생성

시간 순서대로 진행 단계와 검증 점수 간 뚜렷한 상관관계 관찰

로봇공학

Value-Order Correlation (VOC) 평균 0.966 달성 — 기존 보상 모델 초과

개발자 도구 확장

Claude Code 및 Codex용 확장 제공 — 에이전트 시스템의 동작 모니터링 및 개선 가능

응용 2

강화학습을 위한 밀집 보상

LLM-as-a-Verifier의 미세 신호를 RL의 밀집 보상(dense reward)으로 활용하여 샘플 효율성 개선

Off-policy RL

DSRL-SAC + π₀ 정책 (LIBERO)

  • 희소 보상 대비 ≈1.8× 샘플 효율
  • 더 높은 최종 성공률 달성

On-policy RL

GRPO + Qwen3-8B (MATH)

  • 희소 보상 대비 ≈1.1× 샘플 효율
  • 수학적 추론 벤치마크에서 유효성 입증
