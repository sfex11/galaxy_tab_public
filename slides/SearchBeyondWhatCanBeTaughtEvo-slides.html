시각 생성기의 근본적 한계

렌더링은 훌륭하지만, 모르는 것은 자신 있게 지어냅니다

렌더링 능력

  • 정밀한 조명과 구조
  • 일관된 장면 구성
  • 기존 벤치마크 포화

지식 결핍 문제

  • 2025 오사카 엑스포 마스코트 → 허구 생성
  • 스파르타 팔랑크스 → 시대착오적 갑옷
  • "모른다"고 말할 메커니즘 없음

시각 생성기는 항상 이미지를 생성하도록 훈련되므로, 자신이 무엇을 모르는지 인식하거나 지식을 획득할 프로토콜이 없습니다.

순진한 검색(Naive Search)의 실패

검색 보강이 오히려 품질을 저하시키는 역설

문제 무차별 검색

모든 프롬프트에 대해 검색을 트리거하는 에이전트는 상당 부분의 프롬프트 품질을 저하시킵니다.

  • 이미 잘 처리하는 프롬프트에도 노이즈 주입
  • 관련 없는 시각적 디테일 혼입
  • 스타일 오염 및 허위 구조 유입

원인 두 가지 근본 문제

1
언제 검색할 것인가?

내재화 가능한 지식 vs 외부 컨텍스트가 필요한 지식의 구분 불가

2
검색이 반환하는 것은?

모든 검색 결과가 노이즈를 포함하며, 생성기는 유용한 참조와 노이즈를 구분 못 함

오픈 가중치 생성기는 모든 조건 신호를 권위적인 것으로 취급하여, 유용한 참조 콘텐츠와 노이즈를 구별할 능력이 없습니다.

지식 경계(Knowledge Boundary)

생성기마다 다르고, 진화하는 구조적 분계선

내재화 가능한 지식

안정적이고 저차원적인 지식

  • 캐릭터의 정형 외모
  • 국기의 고정된 기하학
  • 증강 감독을 통해 파라미터로 이주
  • 학습 후 검색 불필요

컨텍스트 의존적 지식

본질적으로 외부 컨텍스트가 필요한 지식

  • 재훈련 주기보다 빠르게 진화
  • 롱테일 깊숙이 위치
  • 인스턴스별 구성 추론 필요
  • 파라미터화 저항

핵심 통찰: 이 경계는 생성기별로 다르고 생성기가 개선됨에 따라 계속 이동합니다. 약한 생성기를 돕는 검색 정책이 강한 생성기에는 해가 될 수 있습니다.

노이즈 저항적 에이전트 리즈너

생성기에 무엇을 렌더링할지 구축하는 3단계 파이프라인

1
Gate (게이트)
검색 여부 결정
SFT Task A
"이 프롬프트에 검색이 필요한가?"
2
Filter (필터)
유용한 검색 결과만 유지
SFT Task B
"어떤 검색 결과를 보관할까?"
3
Integrate (통합)
외부 정보를 생성에 통합
SFT Task C
"검색 지식을 프롬프트에 어떻게 녹일까?"

설계 철학: 노이즈는 생성기에 도달하기 전에 억제

리즈너가 노이즈 저항성을 내장하여, 생성기는 불완전한 검색 보강 입력에 대한 저항력을 갖추게 됩니다. 8B 파라미터의 VLM 기반 리즈너로 구현됩니다.

공동 훈련 프레임워크: 가르치고, 그런 다음 검색하라

Teach, Then Search — 리즈너와 생성기의 상호 보완적 학습

Phase 1 Teach (가르치기)

생성기: 온라인 반복 DPO

  • 내재화 가능한 세계 지식을 파라미터에 흡수
  • 불완전한 검색 보강 입력에 대한 노이즈 저항력 구축
  • Flow-matching 동역학 기반 선호도 학습

Phase 2 Search (검색)

리즈너: 거부 샘플링 미세조정

  • 개선된 생성기가 여전히 못하는 것만 검색하도록 보정
  • 지식 경계에 맞춰 검색 범위 조정
  • 리즈너-생성기 간 조정 문제 해결

순환적 자기 개선 플라이휠: 생성기가 렌더링 지식을 점진적으로 확장하는 동시에, 리즈너는 검색 범위를 점진적으로 조정합니다.

단조적 개선의 검증

No Search → Blind Search → Generator-Adaptive Search로의 일관된 성장

방법 SearchGen-Bench 특징
No Search (순수 생성) 21–28 지식 병목으로 심각한 저하
Blind Search (무차별 검색) 일부 개선 / 일부 저하 검색 노이즈로 인한 개념 오염
Generator-Adaptive Search 최고 점수 지식 경계를 존중하는 선택적 검색

핵심 기여 요약

이 논문의 네 가지 주요 기여

1
대규모 벤치마크 구축

SearchGen-20K (20,839 프롬프트, 12 실패 유형, 22 도메인) + SearchGen-Corpus-1M 멀티모달 검색 말뭉치를 공개하여 재현 가능한 연구 기반 제공

2
평가 갭 발견

최첨단 오픈 생성기가 SearchGen-Bench에서 최대 40점 하락하며, 이 갭은 생성기 개선에 따라 변화하고 기존 벤치마크에서는 전혀 보이지 않음을 증명

3
순진한 검색의 한계 규명

검색 노이즈가 개념 오염과 복사 효과를 유발하며, 근본 원인이 생성기별 지식 경계에 의해 지배되는 검색자-생성자 조정 문제임을 식별

4
공동 훈련 프레임워크 제안

노이즈 저항적 에이전트 검색 프로토콜과 공동 훈련으로 8B 리즈너가 순수 생성 및 무차별 검색을 모두 능가하며, 순환적 자기 개선의 기반 확립
