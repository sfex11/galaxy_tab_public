Articraft

확장 가능한 관절형 3D 에셋 생성을 위한 에이전트 시스템

Matt Zhou# · Ruining Li · Xiaoyang Lyu · Zhaomou Song · Zhening Huang · Chuanxia Zheng · Christian Rupprecht · Andrea Vedaldi · Shangzhe Wu#

University of Cambridge · University of Oxford · Nanyang Technological University

문제 정의: 관절형 3D 에셋의 데이터 병목

관절형 3D 객체(서랍장, 펜치, 데스크 램프, 유모차 등)는 실세계에无处不在하며, 3D 콘텐츠 제작, 게임, 로봇공학 등에 필수적입니다. 하지만 기하학만으로는 부족하며, 파트 계층 구조, 조인트, 운동 범위까지 포착해야 합니다.

기존 데이터셋의 한계

  • 규모가 작고 카테고리 coverage가 좁음
  • 품질이 불균일함
  • 실용적 카테고리가 과소대표됨
  • 학습 기반 모델이 과적합 및 일반화 실패

해결 방향

  • 관절형 3D 에셋을 프로그램으로 환원
  • LLM의 코딩 능력과 일상 객체에 대한 사전 지식 활용
  • 재훈련 없이 오프더셸프 LLM으로 대규모 생성

핵심 아이디어

관절형 3D 에셋 생성 문제를 해당 에셋을 구축하는 프로그램을 작성하는 문제로 축소한다. 관절형 객체는 프로그램과 같은 재귀적·구성적 구조를 공유한다.

설계 원칙

자동화

객체 설명만 입력하면 수동 개입 없이 완성된 에셋 생성

경량화

Blender 등 무거운 외부 소프트웨어 없이, 이미지 피드백 없이 저비용 추론

표현력

복잡한 메커니즘을 포함한 광범위한 관절형 카테고리 커버

Articraft 시스템 개요

자연어 설명 x (및 선택적 참조 이미지)가 주어지면, Articraft는 Python 프로그램 y를 작성하여 실행 시 관절형 3D 에셋 a를 출력합니다.

핵심 구성 요소: 도메인 특화 SDK(LLM이 이해하기 쉬운 프로그래밍 인터페이스) + 에이전트 하니스(제한된 작업 공간, 검증, 구조화된 피드백 제공)

프로그래밍 인터페이스: SDK 설계

기하학 및 파트 정의

  • 저수준 프리미티브: 원기둥, 박스 등 추가
  • 고수준 추상화: 힌지, 슬라이더 등 한 번에 추가
  • 프로그램을 간결하고 읽기 쉽게 유지

관절 정의

  • 조인트 유형, 축, 운동 범위 지정
  • 파트 간 계층 구조 표현

자체 검증

  • LLM이 객체별 검증 코드 직접 작성 가능
  • 구조적 무결성 테스트 (충돌, 자가 침투 등)
  • 실행 시간에 에셋 품질 보장

LLM 친화적 설계: 기존 CadQuery 등의 API를 확장하되, 정적 CAD가 아닌 시뮬레이션 준비된 관절형 에셋을 생성하도록 설계

에이전트 하니스

하니스는 LLM을 단일 패스 생성기가 아닌 반복적 에이전트로 변환합니다. SWE-agent와 같은 원칙을 따르되, 3D 에셋 생성에 특화됩니다.

제한된 작업 공간

단일 model.py 파일만 편집 가능. SDK 문서와 선별된 예시만 읽기 전용으로 제공. LLM이 불필요한 세부사항(URDF 작성, 복잡한 환경 관리)에 주의를 빼앗기지 않음.

제한된 행동 공간

프로그램 편집, 실행, 프로브(probe) 요청 등 소수의 행동만 허용. 에이전트가 탐색 공간을 좁히고 효율적으로 수렴하도록 유도.

실행 피드백

컴파일 에러, 런타임 예외, 검증 실패를 구조화된 형식으로 반환. LLM이 다음 수정에서 구체적으로 무엇을 고쳐야 할지 알 수 있음.

추적 및 메타데이터 로깅

에이전트의 전체 추적(trace)과 생성 메타데이터를 기록. 디버깅, 분석, 후속 연구에 활용 가능.

이미지 조건부 생성

텍스트 설명뿐 아니라 참조 이미지로부터도 관절형 에셋을 생성할 수 있습니다. 두 가지 수준의 재구성을 지원합니다.

객체 수준 재구성

단일 객체 이미지가 주어지면, VLM이 이미지를 분석하여 객체의 파트 구조, 조인트 위치, 기하학적 특성을 추론하고 이를 바탕으로 SDK 프로그램을 생성합니다.

장면 수준 재구성

실내 장면 이미지에서 여러 관절형 객체를 식별하고, 각각에 대해 독립적으로 에셋을 생성한 후 원래 장면 구성에 맞게 배치합니다.

차별점: 기존 방법(Articulate-Anything 등)은 기존 3D 에셋 라이브러리에서 파트 메시를 검색해야 하므로 카테고리 다양성이 제한됨. Articraft는 완전히 코드 기반으로 생성하여 이 제약을 극복.

실험 결과: 에이전트 품질 평가

주요 결과

  • 기존 관절형 에셋 생성기보다 더 높은 품질의 에셋 생산
  • 범용 코딩 에이전트(Codex, Claude Code)보다 우수한 성능
  • Blender 등 무거운 외부 소프트웨어 없이 경량적으로 동작
  • LLM 백엔드 교체 가능 — 특정 모델에 종속되지 않음

LLM 소거 실험: 다양한 LLM(Claude, GPT 등)에 대해 reasoning effort 수준을 조절하며 성능을 분석. 더 높은 reasoning effort가 품질 향상에 기여함을 확인.

실제 장면 재구성

실내 장면 이미지를 입력으로 받아, 장면 내의 관절형 객체(서랍장, 냉장고 등)를 인식하고 각각에 대한 관절형 3D 에셋을 성공적으로 재구성.

실패 사례 분석

복잡한 기어 메커니즘이나 비표준 관절 구조에서는 여전히 어려움. 검증-품질 간 트레이드오프 존재.

Articraft-10K 활용: 다운스트림 응용

관절 구조 추정 모델 학습

Particulate 모델을 Articraft-10K로 재훈련하여 성능이 크게 향상됨. 3D 객체의 관절 구조를 추정하는 피드포워드 모델 학습에 직접 활용.

로봇 시뮬레이션

생성된 에셋을 로봇 조작 시뮬레이션 환경에 직접 통합. 다양한 카테고리의 관절형 객체에 대해 로봇이 상호작용 학습 가능.

가상 현실 (VR)

시뮬레이션 준비된 관절형 에셋을 VR 환경에 배치하여 상호작용 가능한 가상 공간 구성에 활용.

공개 예정: Articraft-10K 데이터셋, 모든 에셋의 코드 표현, 에이전트 추적(trace), 에이전트 환경 자체를 공개하여 다양한 LLM 백엔드로의 확장을 지원.

핵심 요약 및 시사점

기여

  • 관절형 3D 에셋 생성을 프로그램 작성으로 환원하는 새로운 패러다임 제안
  • 도메인 특화 SDK + 하니스를 통한 효과적인 에이전트 설계
  • Articraft-10K: 245카테고리 10K+ 에셋의 대규모 데이터셋 구축
  • 기존 방법 및 범용 코딩 에이전트 대비 우수한 품질 달성
  • 다운스트림 응용(모델 학습, 로봇 시뮬레이션, VR)에서 실용성 입증

한계 및 향후 방향

  • 기어 등 복잡한 메커니즘 생성에 어려움
  • 시각적 사실성은 기하학적 프리미티브 기반이라 한계 존재
  • 검증 엄격성과 생성 다양성 간 트레이드오프
  • 텍스처, 재질 등 시각적 세부사항은 향후 확장 필요

핵심 메시지: 관절형 객체의 구성적 특성을 프로그램으로 포착하고, LLM의 코딩 능력을 활용하는 것이 대규모 고품질 데이터 생성의 효과적인 경로임을 입증.
