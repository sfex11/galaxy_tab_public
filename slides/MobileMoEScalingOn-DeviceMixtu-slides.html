온디바이스 Mixture of Experts의 스케일링

왜 온디바이스 MoE인가?

온디바이스 LLM의 현주소

  • MobileLLM, Gemma 등은 전부 Dense 아키텍처
  • 품질 향상 = 활성 파라미터 증가 → 계산량 증가
  • 엣지 디바이스의 엄격한 메모리·연산 제약

새로운 가능성

  • 스마트폰 DRAM 성장: 4GB → 12~16GB
  • MoE로 총 파라미터 확장하면서 활성 파라미터는 소수 유지
  • Sub-billion 활성 파라미터 스케일에서 MoE는 미탐구 영역

MoE의 세 가지 효율성

파라미터 효율성

다수 전문가 네트워크로 총 용량 확장, 토큰당 희소 부분만 활성화하여 더 큰 Dense 모델과 동등한 성능 달성

런타임 효율성

희소 활성화로 추론 FLOPs 감소, 모바일 배터리 수명 보존 및 레이턴시 저감

학습 효율성

전문가 네트워크가 지식, 코드, 수학 등 영역별로 특화, 하나의 모델에 다중 작업 능력을 패킹

세 가지 핵심 기여

1

온디바이스 MoE 스케일링 법칙 기반 모델 family

메모리·연산 제약 하에서 최적의 MoE 아키텍처 도출 — 적정 희소성, 세분화된 전문가, 공유 전문가의 스윗 스팟 발견

2

4단계 학습 레시피

사전학습 → 중간학습 → SFT → INT4 QAT. ~6T 토큰으로 Dense 베이스라인(9~11T)보다 적은 데이터로 우수한 성능 달성

3

상용 스마트폰에서의 최초 MoE 추론

Samsung Galaxy S25, iPhone 16 Pro 배포. MobileMoE-S가 Dense 베이스라인 대비 prefill 1.8–3.8×, decode 2.2–3.4× 빠른 속도

온디바이스 MoE 스케일링 법칙

기존 스케일링 법칙의 한계

  • 서버 스케일 LLM에 최적화 (수십~수백억 파라미터)
  • 추론 비용과 메모리 공간을 분리하여 고려
  • 모바일 제약(메모리 + 연산 동시 고려) 불가

새로운 접근

  • 활성 파라미터(N_act) → 추론 연산량 결정
  • 총 파라미터(N_total) → 메모리 공간 결정
  • 두 변수를 동시 최적화하는 파라미트릭 피팅
  • Sub-billion 활성 파라미터 영역에 특화

기존 법칙은 "클수록 좋다"를 말하지만, 온디바이스에서는 "메모리와 연산의 균형점"을 찾는 것이 핵심

최적 아키텍처: 스윗 스팟 발견

스케일링 법칙이 가리키는 세 가지 설계 원칙

적정

희소성 (Sparsity)

너무 낮으면 효율 없고, 너무 높으면 메모리 과부하. 적정 수준의 희소성이 메모리·연산 동시 최적

세분화

전문가 입도 (Granularity)

소수의 큰 전문가보다 다수의 작은 전문가가 더 효율적 라우팅과 특화 학습 가능

공유

공유 전문가 (Shared Expert)

모든 토큰이 공통으로 사용하는 공유 전문가 추가로 지식 보존 및 안정성 향상

4단계 학습 레시피

1
사전학습
~6T 토큰, 오픈소스 데이터 기반. MoE 특화 안정성 기법 적용
2
중간학습
고품질 데이터로 정제, 도메인 균형 재조정
3
SFT
지시 미세조정으로 대화·추론 능력 부여
4
INT4 QAT
양자화 인식 학습으로 온디바이스 배포 준비

MoE 특유의 라우팅 불안정성과 전문가 붕괴 문제를 각 단계에서 명시적으로 해결하는 기법 포함

학습 효율성: 적은 토큰으로 더 높은 성능

사전학습 토큰 비교

MobileMoE ~6T
Llama 3.2 1B 9T
SmolLM2 1.7B 11T

핵심 인사이트

  • Dense 대비 1.5~2× 적은 토큰으로 동등 이상 성능
  • 전문가 특화로 토큰당 학습 효율 극대화
  • MoE의 학습 효율성이 sub-billion 스케일에서도 유효함을 최초 입증

벤치마크 결과: 새로운 파레토 프론티어

14개 벤치마크 평균 정확도 (상식·지식·과학·이해·추론)

추론 연산량 vs 정확도

  • MobileMoE-S/M: 동등 성능에서 2~4× 적은 FLOPs
  • MobileMoE-L: sub-billion 활성 스케일 최고 정확도
  • 활성 파라미터 기준 기존 Dense 모델 완벽히 지배

총 파라미터 vs 정확도

  • 동등 INT4 메모리에서 더 높은 정확도
  • OLMoE-1B-7B 대비 ~60% 적은 파라미터로 동등 성능
  • MobileMoE-L: OLMoE 대비 30% 적은 활성 + 23% 작은 메모리로 더 높은 정확도

온디바이스 배포: 기술적 도전과 해결

도전

  • 기존 모바일 추론 스택에 MoE 네이티브 지원 부재
  • 전문가 라우팅의 메모리 접근 비용 문제
  • CPU/GPU 백엔드 간 이기종 최적화 필요

해결

  • ExecuTorch 기반 커스텀 퓨즈드 MoE 커널 개발
  • 라우팅 + 전문가 연산을 단일 커널로 퓨징
  • CPU + GPU 양쪽 백엔드에서 체계적 프로파일링

상용 스마트폰에서 MoE 추론을 최초로 실현하고, 전체 런타임 메트릭을 정량적으로 분석

CPU 및 GPU 백엔드 모두에서

  • MobileMoE-L: 현저히 높은 정확도에 적정 런타임 비용

결론

MoE의 이점이 sub-billion 활성 파라미터 스케일에서도 유효함을 최초로 체계적으로 입증

메모리·연산 제약을 동시에 고려하는 온디바이스 MoE 스케일링 법칙 제안

4단계 학습 레시피로 ~6T 토큰으로 파레토 최적 달성

상용 스마트폰에서 최초 MoE 추론 구현, Dense 대비 최대 3.8× 빠른 런타임

MoE는 효율적 온디바이스 LLM을 위한 실용적 경로임을 확인
