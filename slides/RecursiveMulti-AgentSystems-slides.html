핵심 아이디어

재귀적 언어 모델을 MAS로 확장

재귀적 언어 모델(RLM)은 동일한 Transformer 층을 반복적으로 재사용하여 잠재 공간에서 추론을 심화하는 새로운 확장 축이다.

RecursiveMAS는 이 원리를 단일 모델 → 다중 에이전트 시스템으로 확장한다.

각 에이전트가 RLM의 하나의 층처럼 동작하며, 잠재 표현을 다음 에이전트에게 반복적으로 전달하여 루프 형태의 상호작용을 형성한다.

배경 지식

잠재 공간 생성과 재귀적 계산

잠재 공간 자가회귀 생성

표준 디코딩은 마지막 은닉 상태를 어휘 공간으로 투영하지만, 잠재 생성은 이를 직접 다음 forward pass에 입력으로 사용한다.

ht+1 = fθ([E≤t ; ht])

ht+1을 모델의 진행 중인 잠재 사고(latent thought)라고 부른다.

재귀적 계산

동일한 L층 Transformer를 n회 반복하여 추론 깊이를 증가시킨다.

H(0) = E
H(r) = fθ(H(r-1)), r = 1,...,n

마지막 라운드의 H(n)이 최종 예측에 사용된다.

  • 2층 잔차 투영 모듈

  • 2층 잔차 투영 모듈

협업 패턴

4가지 대표적 MAS 협업 구조

Sequential

Planner → Critic → Solver 순차 파이프라인으로 문제를 점진적으로 분해, 판단, 정제, 해결한다.

Mixture

Math · Code · Science 도메인 전문 에이전트가 병렬 추론 후 Summarizer가 결과를 집계한다.

Distillation

큰 Expert 에이전트가 지식을 작고 빠른 Learner 에이전트로 증류하면서 생성 효율을 유지한다.

Deliberation

Reflector와 Tool-Caller가 반복적으로 교환·비판·정제하며 합의 도달 후 최종 답변 생성한다.

RecursiveMAS는 구조에 구애받지 않고(structure-agnostic) 모든 협업 패턴에 일관되게 적용 가능하다.

학습 방법

Inner-Outer Loop 학습 패러다임

Inner Loop — 모델 수준 웜스타트

1
각 에이전트의 Inner RecursiveLink만 학습
2
잠재 사고 생성 과정에 더 잘 정렬되도록 최적화
3
에이전트 개별의 기본적 잠재 표현 능력 확보

Outer Loop — 시스템 수준 공동 최적화

1
에이전트 간 Outer RecursiveLink 학습
2
기울기를 재귀 라운드 전체에 걸쳐 역전파
3
이전 라운드의 자기/타인 피드백을 활용한 반복적 협업 정제
공유 기울기 기반 크레딧 할당(credit assignment)을 통해 재귀 라운드 전체에 걸쳐 시스템이 통합적으로 최적화된다.
이론적 분석

왜 잠재 공간에서 재귀해야 하는가?

실행 시간 복잡도

텍스트 기반 MAS는 중간 에이전트마다 반복적인 디코딩(토큰 생성)이 필요하다.

RecursiveMAS는 잠재 공간에서 직접 변환하므로 불필요한 디코딩을 회피하고 더 효율적인 복잡도를 달성한다.

학습 역학

텍스트 기반 상호작용은 이산적 토큰 생성으로 인해 기울기 소실(gradient vanishing)을 유발한다.

RecursiveMAS의 잠재 공간 연결은 재귀 라운드 간 안정적인 기울기 전파 흐름을 유지한다.
