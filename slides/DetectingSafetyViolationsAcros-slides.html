University of Pennsylvania
01 — 문제 정의

왜 기존 모니터링으로는 부족한가?

AI 에이전트의 안전성 위반은 단일 트레이스가 아닌 여러 트레이스에 걸쳐 분산되어 나타납니다.

🔍
분산된 증거
위반의 증거가 여러 트레이스에 분산되어 있어 단일 트레이스 분석으로는 포착 불가
🌊
희소한 위반
위반 트레이스는 전체의 극히 일부에 불과하며 대부분은 정상 행동
🎭
적대적 위장
악의적 트레이스가 정상 활동과 의도적으로 유사하게 위장되어 탐지 회피
⚠️
예시: 랜섬웨어 분해 공격 — 악의적 캠페인이 여러 개의 무해해 보이는 개별 요청으로 분해되어, 각 트레이스는 정상으로 보이지만 전체를 합치면 완전한 공격을 구성합니다.
02 — 이론적 기반

하이퍼프로퍼티 검증 프레임워크

핵심 정의
트레이스 저장소 R = {t₁, t₂, ..., tₙ}과 자연어 안전 속성 φ가 주어졌을 때, 시스템은 위반 증인 집합(violating witness set) W ⊆ R을 찾아야 합니다.
// 형식적 모델
let Σ* = 트레이스 공간
let R ⊆ Σ* = 트레이스 저장소
let φ = 자연어 안전 속성
let W ⊆ R = 위반 증인 집합

// 기존 모니터링과의 차이
기존: |W| = 1  ← 단일 트레이스 판정
본 논문: |W| ≥ 1  ← 다중 트레이스 교차 분석
제약: |∪W| ≪ |R|  ← 위반은 극히 희소

  • →

  • →

03 — 핵심 기여

Meerkat 시스템 아키텍처

클러스터링 + 에이전틱 검색을 결합한 안전성 위반 탐지 파이프라인

→
STEP 3

안전 속성 + 저장소 구조 인코딩

STEP 4

적응적 추론으로 증인 집합 탐색

STEP 5

통과 / 실패 / 불확실
🎯 클러스터링의 역할
수천~수만 개의 트레이스를 구조화하여 검색 부담을 감소시키고, 유사 패턴의 트레이스를 그룹화하여 이상 패턴 발견을 용이하게 함
🤖 에이전틱 검색의 역할
고정된 규칙이 아닌 자연어 안전 속성에 따른 적응적 추론 수행. 시드 시나리오나 고정 워크플로우 불필요

07 — 추론 시간 확장

더 많은 계산 = 더 나은 탐지

테스트 시간 컴퓨팅 스케일링

🔍 흥미로운 발견: 이중 모드(Bimodal) 행동
실행 결과가 크게 성공하거나 완전히 실패하는 이중 모드 분포를 보임. 중간 수준의 부분 성공은 드물게 나타남 — 이는 에이전틱 검색의 본질적 특성을 반영합니다.
💡
시사점: 안전성 감사에서 추론 시간 컴퓨팅 예산을 늘리면 어려운 케이스에서도 탐지율을 크게 향상시킬 수 있습니다. pass@k 전략이 특히 유효합니다.
