01 / 문제 정의

기존 모바일 GUI 에이전트의 핵심 한계

⚠

비효율적 학습

실패 궤적(trajectory)에서 유용한 지식을 추출하지 못하고 단순히 폐기하는 기존 방식의 한계

🎯

신용 할당 모호성

장기 GUI 작업에서 희소 보상(sparse reward)으로 인해 어떤 행동이 실패를 초래했는지 파악 불가

💰

수동 주석 비용

기존 학습 데이터 구축에 고비용 인간 라벨링이 필수적이라는 확장성 병목

문제 정식화: POMDP

모바일 GUI 작업을 부분 관측 마르코프 결정 과정 (S, O, A, T)으로 모델링. 관측은 스크린샷 + 언어 명령, 행동은 click, swipe, input_text 등 12종의 사전정의 프리미티브, 보상은 궤적 수준 이진 신호(성공=1, 실패=0).

⚙ RFT의 핵심 역할

시드 작업 생성기로 새로운 작업을 합성하고, 규칙 기반 검증기로 성공 궤적만 선별하여 SFT 코퍼스 구축. 매 라운드마다 이전 모델이 새 데이터를 생성하는 자율 루프.

🔬 GRSD의 핵심 역할

그룹 롤아웃에서 성공/실패 궤적 간 포크 포인트(분기점)를 탐지하고, 실패 맥락에 올바른 행동을 주입하여 단계별 교정 감독 신호 생성.

03 / Stage 1 상세

거부 미세조정 (Rejection Fine-Tuning)

1. 시드 작업 생성
원본 작업 템플릿에서 시간 제약, 수량, 파일 엔티티 등 핵심 파라미터를 변형하여 새로운 작업 합성 — 7,000+ 다양한 작업 생성
2. 궤적 생성
현재 모델(초기: Qwen3-VL-4B)이 에이전트로서 AndroidWorld 환경에서 자동으로 행동 궤적 생성
3. 거부 샘플링
ADB 기반 규칙 검증기로 성공 궤적만 필터링 — 구조적 무결성과 개별 행동 단계의 정확성 검증
4. 반복 학습
선별된 고품질 데이터로 SFT 수행. 다음 라운드에서 개선된 모델이 다시 새 궤적 생성 → 공진화 순환

3라운드 반복 결과: 37% ➡ 64% ➡ 73.2% Pass@1

04 / Stage 2 상세

그룹 상대 자기증류 (GRSD)

1

포크 포인트 탐지

SSIM 기반 스크린샷 매칭으로 성공/실패 궤적이 동일 상태에서 다른 행동을 선택한 분기점 식별

2

단계별 자기증류

실패 궤적의 맥락(context)에 성공 궤적의 올바른 행동을 주입하여 교정 학습 샘플 구축

3

혼합 데이터 학습

원본 성공 궤적 + 교정 샘플을 결합하여 SFT 수행 → 정책 개선

포크 포인트 탐지 수식

Same(oa, ob) = 1[SSIM(φ(oa), φ(ob)) ≥ θ]
φ: 상태바 제거 → 저해상도 축소 → 그레이스케일 전처리
해시 유사도 < 0.80인 쌍은 사전 필터링으로 빠르게 제거

GRSD 학습 목표

LGRSD = -1/|D| Σx∈D 1/Tx Σt=1Tx log πθ(yt | s1,...,sP, y<t)
응답 토큰에 대해서만 자기회귀 다음 토큰 예측 손실 계산

09 / 기여 & 결론

핵심 기여 및 향후 과제

✅ 핵심 기여

  • 2단계 자기진화 GUI 에이전트 아키텍처 제안
  • 거부 미세조정(RFT)으로 자율 데이터-모델 공진화 실현
  • GRSD 기법으로 외부 모델 없이 밀집 단계별 감독 구성
  • 4B 모델로 인간 수준(80%) 초과 달성 (81.0%)
  • 수동 데이터 주석 없는 완전 자율 학습 파이프라인

⚠ 인지된 한계

  • 실시간 실행: 애니메이션, 키보드 전환, 토스트 알림 등 일시적 시각 변동이 SSIM 매칭 정확도 저하 가능
  • 행동 공간 제약: 사전정의 행동이 제스처 지속시간, 타이밍 등 저수준 터치 역학을 추상화

결론: UI-Voyager는 고비용 수동 주석 없이
실패 경험으로부터 효율적으로 학습하는
자기진화형 모바일 GUI 자동화의 새로운 패러다임을 제시합니다.
