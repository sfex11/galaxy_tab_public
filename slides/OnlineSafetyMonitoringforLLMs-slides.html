LLM을 위한 온라인 안전 모니터링

배포 시점에서도 LLM은 불안전한 출력을 생성할 수 있으며,
실시간으로 안전성을 감시하고 경보를Raise하는 것이 필수적이다.

Mona Schirmer, Metod Jazbec, Alexander Timans, Christian Naesseth, Maja Waldron, Eric Nalisnick

arXiv:2607.02510v1

왜 온라인 모니터링이 필요한가?

기존 안전 조치의 한계

  • 정렬 훈련(RLHF 등)으로도 완전히 제어 불가
  • 사전 평가는 모든 프롬프트 시나리오를 커버하지 못함
  • 사후 검출은 이미 유해 출력이 완료된 후에야 작동

온라인 모니터링의 필요성

  • 토큰이 생성되는 동안 실시간 감시
  • 유해 출력이 완료되기 전에 중단 가능
  • 안전 라벨이 없는 환경에서도 작동해야 함

모니터의 핵심 과제: 불완전한 안전 신호를 이진 결정(경보 vs 침묵)으로 변환하는 것

문제 설정

1
프롬프트 입력
x ~ Px
2
순차적 출력 생성
o1:T 자가회귀
3
안전 변수
y ∈ {0, 1}
4
실시간 모니터링
o1:t 검사

y = 1 안전 상태

사실적으로 정확하거나 무해한 출력

y = 0 불안전 상태

환각, 사실 오류, 유해한 출력

목표: 불안전한 시퀀스를 가능한 한 조기에 식별하는 것

안전 신호 (Safety Signal)

안전 변수 y에 대한 정보를 담는 불완전한 신호 st를 활용

외부 검증자 신호

pψ(y | x, o1:t) — 외부 모델의 예측 확률

  • 수학 추론: 프로세스 보상 모델(PRM)
  • 콘텐츠 검열: LLM 세이프가드 분류기
  • 효과적이지만 배포 비용이 높음

내부 신호

pθ(ot | x, o1:t-1) — 생성 모델 자체의 토큰 로그확률

  • 추가 모델 배포 불필요
  • 비용 효율적
  • 신호 품질은 외부 신호보다 낮음

안전 모니터: 단일 임계값 규칙

복잡한 순차 검정 대신, 단일 시불변 임계값 λ를 적용하는 매우 단순한 규칙

Φt := 𝟏{ ∃k, 1 ≤ k ≤ t : sk < λ }

시간 t까지 신호 sk가 임계값 λ보다 처음 낮아지는 순간 경보 발생

경보 발생 시 후속 조치

생성 중단

강한 검증자 에스컬레이션

인간 검토 요청

모니터의 두 가지 위험

I

위양성 위험 (False Alarm)

실제로는 안전한 출력에 불필요하게 경보를Raise

ℛI(λ) = ℙ(∃t ≥ 1 : st < λ | y = 1)

λ가 클수록 위양성 위험 증가 (단조 증가)

사용자 경험 저해, LLM 기능성 방해

II

위음성 위험 (Missed Detection)

불안전한 출력을 놓쳐서 경보를Raise하지 않음

ℛII(λ) = ℙ(∀t ≥ 1 : st ≥ λ | y = 0)

λ가 작을수록 위음성 위험 증가

거짓 안도감 부여, 모니터 무용화

본 논문은 주로 위양성 위험(ℛI)의 통제에 집중

위험 통제: 기댓값 제어

순익 위험 통제 (Conformal Risk Control, Angelopoulos et al. 2022)

𝔼𝒟cal[ℛ(λ̂)] ≤ ε

캘리브레이션 셋 추출에 대해 평균적으로 위험이 ε 이하 보장

캘리브레이션 데이터셋

𝒟cal = {(x(i), o1:T(i), y(i))}i=1n — 배포와 동일 분포에서 추출

임계값 선택 규칙

보정된 경험적 위험이 ε 이하인 가장 큰 λ를 선택 — 모니터를 최대한 관대하게 만듦

위험 통제: 고확률 제어

Bates et al. (2021) 기반 — 더 강한 통계적 보장

ℙ𝒟cal(ℛ(λ̂) ≤ ε) ≥ 1 − δ

캘리브레이션 셋 추출의 최대 δ 비율을 제외하고 위험이 ε 이하 보장

1
UCB 계산
경험적 위험의 (1−δ) 상한 신뢰계산 U(λ, n, δ)
2
임계값 선택
UCB가 ε 이하인 가장 큰 λ 선택
3
보장 달성
고확률 위험 통제 완료

기댓값 제어보다 더 보수적인 임계값을 산출하지만, 실제 배포에서 더 강력한 안전 보장 제공

실험 1: 사실성(Factuality) 모니터링

설정

  • 수학 문제 해결 데이터셋 (GSM8K, MATH)
  • 안전 신호: 프로세스 보상 모델(PRM) 점수
  • 비교 대상: 순차 가설 검정 기반 모니터 (E-valuator)
  • 목표 위험 수준: ε = 0.1

주요 결과

  • 위양성 위험이 목표 수준 이하로 통제됨을 경험적으로 확인
  • 단순 임계값 모니터가 순차 검정 방법과 경쟁력 있는 성능 달성
  • 생성 과정에서 더 조기에 경보를Raise

복잡한 순차 가설 검정 없이도 단순한 임계값 설계로 충분한 성능을 얻을 수 있음

실험 2: 무해성(Harmlessness) 모니터링

설정

  • 레드팀 대화 데이터셋 활용
  • 안전 신호: LLM 세이프가드 분류기
  • 독성, 악의적 사용 탐지

핵심 발견

  • 더 나은 신호 → 더 나은 모니터
  • 신호 품질이 모니터 성능의 결정적 요인
  • 프레임워크는 신호에 구애받지 않고 범용 적용 가능

"모니터 설계의 복잡도를 줄이고, 신호 품질 향상에 자원을 집중하는 것이 더 효과적"

신호 비교: PRM vs 토큰 로그확률

모니터 효율성과 성능 간의 트레이드오프 분석

특성 외부 PRM 신호 내부 토큰 로그확률
추가 비용 높음 (별도 모델 필요) 없음
신호 품질 높음 상대적으로 낮음
위험 통제 품질 우수 보통
조기 탐지 능력 빠름 상대적으로 느림
배포 복잡도 복잡 단순

토큰 로그확률은 비용 효율적이지만, 안전 모니터링 목적으로는 PRM 신호가 확실히 우위

핵심 요약

단순함의 힘

단일 임계값 규칙이 복잡한 순차 검정과 경쟁력 있음

통계적 보장

기댓값 또는 고확률로 위양성 위험을 엄격히 통제

범용성

사실성, 독성, 악의적 사용 등 다양한 안전 위험에 적용 가능

조기 탐지

생성 중간에 불안전 출력을 식별하여 피해 최소화

신호가 핵심

모니터 설계보다 신호 품질 향상에 투자하는 것이 효과적

트레이드오프 명시

비용 vs 신호 품질 간 명확한 비교 프레임워크 제공

한계점 및 향후 방향

한계점

  • 캘리브레이션 셋이 배포 분포와 동일하다고 가정 — 분포 시프트 시 성능 저하 가능
  • 주로 위양성 위험에 집중, 위음성 위험의 상세 분석은 부록에서만 다룸
  • 단일 임계값의 단순성이 복잡한 패턴 탐지에는 한계가 될 수 있음

향후 방향

  • 분포 시프트에 강건한 적응적 캘리브레이션 방법
  • 위양성·위음성 위험의 동시 통제
  • 더 다양한 안전 위험 유형으로 확장 (편향, 프라이버시 등)
  • 다중 모달 LLM (비전·오디오)로의 일반화
