문제 제기

기존 벤치마크의 세 가지 한계

01

환경 분리

코드 샌드박스에만 국한되어 스프레드시트 네이티브 객체 모델, 데이터-차트 바인딩, GUI 제약을 무시

02

생성 편향

원샷 차트 구성만 평가하고, 새로운 데이터와 요구사항에 맞춰 시각화를 수정하는 진화적 작업을 소홀히 함

03

완벽 의도 가정

완전히 명시된 프롬프트만 사용하여, 능동적 확인과 대화 기반 정렬의 필요성을 간과

실제 환경의 데이터 시각화는 네이티브 환경 정착, 크로스 플랫폼 진화, 능동적 의도 정렬을 요구한다.

DV-World 개요

실제 시나리오 전체 수명주기 평가

DV-Sheet

스프레드시트 네이티브 조작

  • DVSheet-Create: 네이티브 차트 생성
  • DVSheet-Fix: 진단 및 수정
  • DVSheet-Dash: 대시보드 구성

DV-Evol

크로스 패러다임 논리 진화

  • 참조 이미지→실행 가능 코드 변환
  • 5가지 패러다임 간 이식
  • Python, Vega-Lite, D3.js 등

DV-Inter

능동적 의도 정렬

  • 모호한 요구사항 식별
  • 목표 지향적 질문 생성
  • 사용자 시뮬레이터 기반 평가
총 260개 태스크
DV-Sheet 상세

스프레드시트 네이티브 시각화 조작

DVSheet-Create

다양한 데이터에서 직접 셀 참조를 사용하여 네이티브 차트 객체 생성

핵심: 네이티브 객체 모델 숙달, 데이터-차트 바인딩 이해

DVSheet-Fix

손상된 시각화 바인딩을 식별하고 진단적 추론으로 수정

핵심: 파손 원인 진단, 최소 변경 원칙 준수

DVSheet-Dash

여러 차트와 테이블을 종합하여 전문적 분석 뷰 구성

핵심: 공간 계획, 다중 시각화 조화, 비즈니스 맥락 반영

코드 샌드박스가 아닌 실제 스프레드시트 환경에서의 네이티브 객체 조작 능력을 평가합니다.

DV-Evol 상세

크로스 패러다임 시각화 진화

정적 재현이 아닌, 요구사항 변화에 따른 논리 이식과 구조적 재구성 평가

1
원자 서브플롯 분리
참조 시각화를 개별 차트로 분해
2
적대적 데이터 변환
새 데이터로 스케일·값 변환
3
분류 기반 과제 설계
5가지 패러다임별 진화 태스크 생성
4
실행 가능 검증
생성 코드의 실제 실행 결과 확인

평가 차원

  • 데이터 일관성 (축 스케일, 값)
  • 시각적 의미 보존
  • 프레임워크 특성 준수
DV-Inter 상세

능동적 상호작용과 의도 정렬

모호한 사용자 요구사항 아래에서 에이전트의 능동적 확인 및 대화 기반 정렬 평가

사용자 시뮬레이터

게이트키퍼
보안 지향 필터링, 적절한 응답 라우팅
사용자 생성기
실제 사용자 행동 기반 grounded 상호작용 생성

모호성 주입 설계

  • 이중 상태 의도 모델링 (표면/심층 의도)
  • 모호성 분류법 기반 복잡도 스케일링
  • 3차원 루브릭: 전략적 능동성·상호작용 효율성·최종 품질

단순히 명령을 실행하는 것이 아니라, 불충분한 정보를 식별하고 목적 지향적 질문을 통해 사용자의 진짜 분석 목표에 도달해야 합니다.

오류 분석

도메인별 실패 패턴

DV-Sheet

  • 데이터 정착 및 수치 추론 체계적 실패
  • 시각적 규칙의 의미론적 오용으로 가독성 저하
  • 수정 과정에서 파괴적 편집 및 원본 무결성 상실
  • 대시보드에서 비즈니스 맥락 분석 부족

DV-Evol

  • 크로스 백엔드 시각적 매핑 실패
  • 변환 및 스케일 불일치가 핵심 병목
  • 백엔드별 렌더링 누락 및 차트 유형 퇴화
  • 모델 의존적 오류 경향 (5개 백엔드 간 편차)

DV-Inter

  • 가정 우선 실행으로 상호작용 회피
  • 질문 결핍 및 시각화 전략 부정렬
  • 데이터 파싱의 취약성에서 오는 인지-실행 간극
  • 복잡한 의미 요구 아래 시각적 설계 불일치
핵심 인사이트

DV-World가 제시하는 방향

원샷 생성 → 수명주기 관리로 전환

단일 차트 생성에서 벗어나 생성-수정-진화-상호작용의 전체 라이프사이클을 다루어야 합니다.

세 가지 역량의 통합 필요

환경 숙달 + 의미론적 이식성 + 능동적 의도 정렬의 결합이 필수적입니다.

엔터프라이즈 워크플로우 현실 반영

실제 기업 환경에서 요구되는 다기능 시각화 전문성을 목표로 개발 방향을 설정해야 합니다.

평가 방법론의 진화

단순 실행 검증을 넘어 수치 정합성 + 루브릭 기반 MLLM 판단의 하이브리드 평가가 필요합니다.
