01 — 문제 정의

기존 시스템의 한계

멀티에이전트 워크플로우는 여러 LLM 호출을 순차적으로 실행하며, 각 단계의 출력이 다음 단계의 컨텍스트가 됩니다.

⚠

동질적 클러스터 가정

기존 서빙 시스템(vLLM, SGLang)은 동일한 모델 복제본만을 가정하여 이기종 배포의 최적화 기회를 놓침

⚔

지연시간-성능 트레이드오프

더 능력있는 모델로 라우팅하면 성능은 올라가지만 GPU 자원 경합으로 지연시간이 급증

🔗

워크플로우 의존성

단계 간 의존성으로 인해 큐잉 상호작용이 복잡해지고, 요청 단위 스케줄링으로는 최적화 불가

🔄

모델 선택과 혼잡의 결합

모델 선택이 큐 혼잡에 직접 영향 — 라우팅과 로드 밸런싱을 통합적으로 고려해야 함

03 — 시스템 아키텍처

Chimera 시스템 구조

에이전트 앱과 추론 엔진 사이의 미들웨어 계층으로 4개의 핵심 모듈 통합

🧭

시맨틱 라우터

Semantic Router

ModernBERT-large 기반 인코더로 모델별 신뢰도 점수(0~1) 추정. 이진 교차 엔트로피로 학습.

📏

길이 예측기

Length Predictor

분위 랜덤 포레스트(QRF)로 워크플로우의 총 남은 출력 토큰 수를 CPU에서 경량 예측.

📊

활동 모니터

Activity Monitor

요청별 메타데이터 추적. 엔진별 비행 중 예측 토큰 볼륨을 실시간 집계하여 부하 추정.

⚖

로드 밸런서

Load Balancer

모델 선택 + 요청 우선순위 결정. 지연시간 슬랙(τ) 내에서 최고 신뢰도 모델 선택.

04 — 핵심 알고리즘

라우팅 및 스케줄링 정책

🎯 모델 선택 전략

  • 각 모델 m의 신뢰도 q[m] 계산
  • 비행 중 토큰 볼륨으로 TTLT 추정
  • 가장 빠른 모델 m_fast 식별
  • 지연 슬랙 τ 내에서 최고 신뢰도 모델 선택
  • 캐시 지역성을 위해 워크플로우 내 할당 재사용
L[m] ≤ (1 + τ) · Lfast
AND
q[m] ≥ q* + Δs

⏱ STJF 우선순위

워크플로우의 총 남은 작업 길이를 우선순위로 사용

Yr = ∑j=i..Np OutTokens(p, j)

SJF 대비 15~34% 추가 큐 시간 단축

🛡 기아 방지 메커니즘

  • 에이징 기반 승격 (S: 기아 임계값)
  • starvation_count ≥ S일 때 레벨 강등
  • Q 반복 후 원래 레벨로 복귀
  • 키: (starvation_level, priority, arrival)
05 — 핵심 수식

주요 수학적 모델

비행 중 토큰 볼륨

Pm = ∑t ∈ I[m] t

모델 m에 할당된 모든 진행 중 요청의 예측 토큰 합계

TTLT 추정

L[m] = Pm · decodems/tok / batchmax

오프라인 프로파일링된 디코딩 속도와 배치 크기로 지연시간 추정

라우터 출력

q[m] = σ(ℓr,m)

시그모이드 활성화로 각 모델의 신뢰도 점수 산출

예측 중앙값

ŷ ≈ median(Yr | φ(r), m*)

분위 랜덤 포레스트로 워크플로우 총 출력 길이의 중앙값 예측

🔧 길이 예측기 특성

토큰 카운트 워크플로우 식별자 현재 단계 모델 이름 TF-IDF + SVD 텍스트 스케치

08 — 어블레이션 연구

각 컴포넌트의 기여도

🧭 라우터 기여

  • 라우터 제거 시 파레토 프론티어가 현저히 열화
  • 고정 휴리스틱으로는 동적 부하 변화에 대응 불가
  • 쿼리별 라우팅과 예측 스케줄링의 결합이 핵심

🔮 오라클 비교

  • 오라클 예측기: 타이트한 슬랙에서만 의미있는 개선
  • 오라클 라우터: Qwen1.5+14B에서 상당한 프론티어 향상
  • 여러 설정에서 Chimera가 근최적

09 — 비교 분석

기존 시스템 대비 차별점

vs vLLM

  • 이기종 모델별 라우팅 결정 추가
  • 워크플로우 인식 스케줄링
  • 부하 모니터링과 신뢰도 점수 결합
  • → 최대 3.4배 속도 향상

vs MLFQ

  • 성능과 지연시간을 동시 개선 (트레이드오프 아님)
  • 워크플로우 인식 우선순위 vs 균일 승격/강등
  • 예측 기반 vs 반응형
  • → 최대 +19.6% 성능

vs LTR

  • 상대 순서 + 절대 신뢰도 점수 통합
  • 로드 밸런싱과 라우팅 통합
  • 워크플로우 레벨 예측 vs 요청 레벨
  • → 최대 +16.0% 성능

🎯 핵심 차별화 포인트

시맨틱 라우팅 + 워크플로우 레벨 예측 + 실시간 부하 모니터링을
하나의 통합된 스케줄링 프레임워크로 결합한 최초의 시스템
