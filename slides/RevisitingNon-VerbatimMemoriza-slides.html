배경과 동기

LLM의 사실 기억

LLM은 파라미터에 광범위한 사실 지식을 저장하며, 외부 검색 없이도 지식 집약적 질문에 답변 가능

환각의 위험

필요한 지식이 없거나 접근 불가능할 때, 환각 또는 오답을 생성할 수 있음

비축어적 기억을 분석하는 일반적 방법은 개체 기반 QA이지만, 일반적으로 각 개체를 단일 정형 표면 형태로만 질의함

→ 사실 기억 여부와 특정 이름으로의 접근 가능성을 분리하기 어려움

핵심 관찰

동일한 개체·사실 triples에 대해 표면 형태만 바꿨을 때 예측 결과가 달라지는 경우

사례 표면 형태 유형 질문 Pythia-12B
1 정형 David Guetta의 직업은? ✓ DJ
별명/약어 Jack Back의 직업은? ✗ actor
2 정형 José García Castro의 종목은? ✗ baseball
별명/약어 Pepillo II의 종목은? ✓ soccer
3 정형 J. M. Coetzee의 출생지는? ✓ Cape Town
철자 변이 J M Coetzee의 출생지는? ✓ Cape Town
4 정형 Stan Coveleski의 직업은? ✓ baseball player
오타 Stan Covaleski의 직업은? ✗ actor

정답은 고정되어 있으나, 표면 형태에 따라 정답/오답이 뒤집힘

표면 형태 범주 분류

33개의 리다이렉트 범주를 수동 선정하여 3가지 대유형으로 분류

대체 명칭 · 약어
  • 출생명 (from birth names)
  • 이니셜/약어 (from initialisms)
  • 별명 (from nicknames)
  • 이전 명칭 (from former names)

예: "Stevland Hardaway Judkins" → "Stevie Wonder"

철자 변이
  • 분음부호 (from diacritics)
  • 구두점 변화
  • 공백·하이픈 차이
  • 대소문자 변이

예: "Nicolas Sarközy" → "Nicolas Sarkozy"

일반적 오류
  • 잘못된 이름 (from incorrect names)
  • 흔한 오타 (from misspellings)
  • 혼동 명칭

예: "Christian Ronaldo" → "Cristiano Ronaldo"

실험 설정

평가 모델 13개
  • Pythia 계열 (70M ~ 12B)
  • GPT-2 계열 (124M ~ 1.5B)
  • OPT 계열 (125M ~ 66B)
  • Llama 계열 (7B, 13B, 70B)
  • 기타 (Gemma, Mistral 등)
추론 및 평가
  • 개방 도메인 QA 설정 (외부 증거 없음)
  • 모델의 첫 토큰 예측을 정답으로 평가
  • 정형–리다이렉트 쌍의 예측 일관성 분석
  • 범주별 일관성 비교
일관성 지표: 동일 triple의 정형 질문과 리다이렉트 질문에 대해 모델이 동일하게 정답/오답을 예측하는 비율

주요 결과: 표면 형태에 따른 예측 불일치

13개 LLM에서 표면 형태만 변경해도 예측 결과가 자주 달라짐

철자 변이에 대한 강건성
높음

분음부호, 구두점, 공백 등
사소한 정자법 변이에는 비교적 안정

대체 명칭·약어에 대한 취약성
낮음

별명, 대체 명칭, 약어 등
큰 어휘적 변이에는 불일치 많음

비축어적 기억은 개체와 사실이 동일하더라도 완전히 표면 불변(surface-invariant)이 아님

범주별 일관성 분석

리다이렉트 범주 유형에 따른 예측 일관성의 차이

범주 대유형 변이 정도 일관성 해석
철자 변이 사소함 높음 정자법 수준의 미세 차이는 모델이 잘 처리
일반적 오류 보통 보통 흔한 오타는 부분적으로 처리하지만 완전하지 않음
대체 명칭·약어 큼 낮음 어휘적 변이가 클수록 사실 접근 불일치 증가
불일치는 범주 의존적(category-dependent)이며, 변이의 정도에 따라 체계적으로 달라짐

빈도 분석: 개체 vs 표면 형태

집계 개체 빈도를 표면 수준 빈도로 분해하여 각각의 기여 분석

표면 형태 빈도

특정 표면 형태가 말뭉치에 등장하는 빈도와 정답률이 양의 상관

→ 해당 표면 형태로 직접 노출된 경험이 기억에 기여

개체 빈도

개체의 전체 빈도가 표면 빈도 이상으로 추가적인 기여를 제공

→ 표면 형태 간에 교차 결합(cross-surface coupling) 존재

편상관 분석 결과: 표면 빈도를 통제한 후에도 개체 빈도가 정답률과 유의미한 상관을 유지 → 각 표면 형태가 독립적으로 기억되는 것은 아님

종합 논의

비축어적 기억의 스펙트럼
완전 표면 특이적
각 표면 형태가 독립적으로 기억됨
중간적 그림 ← 본 연구
표면 빈도 + 개체 빈도 모두 기여, 교차 결합 존재
완전 표면 불변
표면 형태와 무관하게 동일하게 접근
  • 사실 기억은 순수 표면 특이적이지 않음 — 개체 수준에서 결합 존재
  • 사실 기억은 완전 표면 불변도 아님 — 표면 형태에 따라 접근성 변화
  • 범주에 따라 강건성이 달라지는 체계적 패턴 확인

결론 및 시사점

정형 개체명만으로 비축어적 기억을 평가하면
표면 조건부 실패를 놓칠 수 있다

기여
  • RedirectQA: 표면 형태 다양성을 갖춘 통제된 QA 벤치마크
  • 범주별 불일치 패턴의 체계적 발견
  • 개체·표면 빈도의 분해 분석 프레임워크
향후 방향
  • 표면 형태 간 교차 결합 메커니즘 심층 분석
  • 다국어 환경으로의 확장
  • 표면 다양성을 고려한 기억 평가 방법론 발전
핵심 메시지: LLM의 사실 기억을 평가할 때 표면 형태 다양성을 필수적으로 고려해야 함
