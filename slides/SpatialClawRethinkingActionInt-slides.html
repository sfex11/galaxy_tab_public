공간 추론이란 무엇인가

3D 공간에서 객체의 위치, 관계, 이동을 파악하는 능력

"자동차가 카메라 쪽으로
이동하고 있는가?"

"테이블에 가장
가까운 객체는?"

"사람이 왼쪽으로
돌았는가, 오른쪽으로?"

인간에게는 자연스럽지만, 최첨단 VLM조차도 여전히 어려워하는 근본적 과제

왜 VLM만으로는 부족한가

공간 추론은 다양한 증거의 조합이 필요한 구조화된 분석 과정

필요한 증거 유형

  • 깊이 추정 (Depth)
  • 카메라 포즈 (Camera Pose)
  • 시간적 대응 (Temporal Correspondence)
  • 객체 분할 (Segmentation)
  • 기하학적 계산 (Geometry)

VLM의 한계

  • 픽셀만으로 신뢰할 수 있는 깊이 추정 불가
  • 다단계 기하학적 연산 수행 불가
  • 프레임 간 대응 관계 추적 불안정
  • 수치적 정밀도 부족
  • 중간 결과 검증 불가

→ 전문 인지 모듈로 VLM을 보강하는 Tool-augmented Agent가 자연스러운 해결책

기존 액션 인터페이스의 한계

방식 A

단일 패스 코드 실행

완전한 파이썬 프로그램을 작성 후 한 번에 실행

  • 유연한 도구 호출 가능
  • 중간 결과를 보기 전에 완전한 전략을 확정해야 함
  • 중간 시각적 증거가 추론 루프에 참여 불가
방식 B

구조화된 툴 호출

JSON/XML 형태로 미리 정의된 도구 선택

  • 깔끔한 인터페이스
  • numpy, scipy 등 과학 라이브러리 조합 불가
  • 테스트 시점에서만 등장하는 계산에 대응 불가

두 방식 모두 복잡한 3D/4D 공간 추론이 요구하는 개방형 구성적 추론을 지원하지 못함

핵심 아이디어: 코드를 액션 인터페이스로

"코드는 일회성 프로그램이나 도구 디스패치가 아니라,
인지 도구를 순서화, 조합, 진단하는 오케스트레이션 공간이어야 한다"

순서화

단계별로 인지 도구 실행 순서 결정

조합

인지 출력과 수치 프리미티브 자유롭게 결합

진단

중간 결과 확인 후 분석 전략 수정

새로운 공간 분석 = 새로운 API 항목이 아니라, 인지 도구와 수치 프리미티브의 새로운 조합

SpatialClaw 개요

Training-free 프레임워크 · 코드 기반 액션 인터페이스 · 상태 유지 파이썬 커널

설계 원칙

  • 인지 도구는 Python callable
  • 마스크, 깊이맵, 포즈 등은 일반 Python 변수
  • 커널이 상태를 턴 간에 독립적으로 보존
  • 한 단계에서 생성된 객체는 이후 모든 단계에서 활용 가능
  • 벤치마크/모델 특화 엔지니어링 없음

제공 도구 세트

  • Reconstruct — 3D 재구성 및 BEV 렌더링
  • SAM3 — 이미지/비디오 분할
  • Geometry — 기하학적 연산
  • Mask / Time / Graph / Draw
  • numpy, scipy 등 과학 라이브러리

지속적 커널 워크스페이스

입력 프레임, 인지 도구, 과학 라이브러리가 사전 로드된 상태 유지 Python 커널

커널에 사전 로드되는 것

  • 입력 프레임 (이미지/비디오)
  • 인지 도구 (Python callable)
  • 과학 라이브러리 (numpy, scipy)
  • 좌표계 변환 유틸리티

상태 유지의 핵심

  • 매 단계 하나의 셀 실행
  • 이전 출력 기반 조건부 코드 생성
  • 객체 생성→검사→수정의 자유로운 반복
  • 보안 샌드박스 내 실행

단일 패스와의 결정적 차이: 중간 결과를 관찰한 후 다음 단계를 결정할 수 있다

공간 추론 루프: 5단계 파이프라인

1
계획
Planner가 작업 분해 및 단계별 계획 수립
2
코드 생성
VLM이 이전 출력 기반으로 하나의 셀 작성
3
실행
커널에서 코드 실행, 상태 유지
4
피드백 조립
텍스트 출력 + 시각적 렌더링을 피드백으로 구성
5
답변 제출
충분한 증거 확보 시 최종 답변 도출

단계 2→4는 중간 증거가 충분해질 때까지 반복되며, 매 반복마다 이전 모든 출력을 조건으로 활용

액션 인터페이스 비교

특성 단일 패스 코드 구조화된 툴 호출 SpatialClaw
중간 결과 관찰 불가 부분 가능 모든 단계에서 가능
과학 라이브러리 조합 가능 불가 자유로운 조합
전략 수정 (Revision) 불가 제한적 단계별 적응
상태 유지 불가 불가 커널 수준 유지
테스트 시점 계산 적응 제한적 불가 완전 지원

제어된 비교 실험에서 SpatialClaw의 인터페이스가 가장 강력한 일반화를 달성

핵심 인사이트

인터페이스 설계의 중요성

어떤 도구가 있는지보다 그 도구를 어떻게 호출하고 결과를 어떻게 관찰할 수 있는지가 에이전트 능력을 결정함

조합성이 핵심

사전 정의된 유틸리티 래퍼를 모두 제거해도 성능 향상이 지속됨 — 인지 프리미티브의 유연한 조합 자체가 핵심

중간 관찰의 가치

단일 패스와의 격차는 중간 시각적 증거를 추론 루프에 참여시키는 능력에서 기인

학습 불필요

Training-free로 동작하며, 프롬프트나 도구 세트 수정 없이 다양한 백본과 벤치마크에 전이 가능

결론

  • 공간 추론 에이전트에서 액션 인터페이스 설계가 성능의 결정적 요인임을 입증
  • 지속적 다중 턴 Python 커널을 액션 인터페이스로 채택한 SpatialClaw 제안
  • 20개 벤치마크에서 평균 59.9% 정확도 달성 (+11.2p 향상)
  • 6개 VLM 백본 (27B~397B)에서 일관된 전이성 확인
  • 동적 4D 비디오 및 다중뷰 추론에서 가장 큰 향상 — 연쇄적 기하 계산에 강력

"코드는 도구를 호출하는 수단이 아니라, 인지 도구를 오케스트레이션하는 공간이다"
