Deep Interaction

문제 배경: CoT 추론의 한계

Chain-of-Thought의 발전

DeepSeek-R1, GPT-o1, Qwen 2.5 등 CoT 기반 추론 모델이 복잡한 다단계 작업에서 놀라운 성능을 보여줌

  • 단계적 추론 능력 향상
  • 복잡한 벤치마크에서 강력한 성능

여전히 존재하는 문제들

중학교 수준의 문제에서도 명백한 오류를 발생시키며, 긴 추론 사슬에서 취약함

  • 깊은 이해 부족
  • 훈련 데이터 외 지식 한계

3단계 이상의 추론이 필요한 작업에서 모델은 여전히 명확한 오류나 완전히 엇빗나간 추론을 생성한다.

기존 상호작용 방식의 실패

1

재생성 접근

오류 발생 시 전체 응답을 재생성하지만, 같은 실수를 반복할 가능성이 높음

2

대화 기반 피드백

사용자가 오류를 지적해도 "맞습니다, 제가 실수했어요"라고 인정한 뒤 동일한 오류 반복

3

자가 교정의 한계

자신의 추론 실수를 스스로 찾는 것은 매우 어려우며, 감지 가능한 오류에만 한정됨

사용자는 오류 식별과 교정에 상당한 노력을 투자하지만, 유사한 실수를 되받음으로써 LLM에 대한 신뢰가 심각히 훼손됨

식별된 핵심 이슈 5가지

이슈 1

대화형 명령 수신 실패

후속 상호작용에서 새로운 지식이나 수정 지시를 따르지 못하고, 잘못된 기반 위에서 추론을 계속함

이슈 2

훈련 질문 암기

훈련셋의 질문을 암기하여 재생산하며, 입력 값이 변경되어도 동일한 답변을 반복함

이슈 3

수행적 추론 (Performative Reasoning)

진정한 이해 없이 훈련 데이터의 "해결 템플릿"을 이어 붙이며, 과도한 추론 단계로 오히려 방해됨

이슈 4

끈적한 사고 (Sticky Thinking)

초기 토큰이 모델을 고정된 추론 경로에 잠가버려, 대안 탐색 능력 제한

이슈 5

KV 캐시 효과

질문의 수치나 의미가 크게 변경되어도 이전 응답을 그대로 재생산하는 현상

Deep Interaction: 핵심 아이디어

사용자가 문서 수정 모드처럼 추론 단계를 직접 편집할 수 있도록 하여, 정확한 단계는 보존하고 오류만 정밀 교정

  • 대화로 오류 위치를 설명할 필요 없음
  • 긴 대화 기록이 모델의 주의를 분산시키지 않음
  • 정확한 추론 단계를 그대로 유지
  • 편집된 CoT를 증류된 프롬프트로 변환

수정 모드 기반 응답 (§3.1)

핵심 원칙

인간이 직접 답변을 주는 것이 아니라 추론을 안내한다. 명백한 오류가 있거나 기대와 다른 추론 단계를 교정하며, 후속 추론 단계에 대한 지침도 제공 가능

r번째 상호작용

복잡한 쿼리 𝒬ᵣ에 대해 모델 응답은 CoT 추론 시퀀스와 최종 답변으로 구성. 사용자는 추론 단계를 직접 편집하여 수정된 응답 𝒮'ᵣ을 생성

유지된 내용

변경되지 않은 정확한 추론 단계

각 세그먼트는 원본 쿼리와 문맥적으로 결합되어 처리된 후 재조합됨. 이를 통해 모델이 어떤 부분이 수정되었는지 명확히 인지하고 교정된 경로를 따름

의미적 동등 변환 (§3.3)

CoT Reprompter

편집된 응답을 구조화된 "Feedback-CoT"로 재구성하는 선택적 컴포넌트

  • 사용자의 자연어 편집을 의미적으로 동등한 프롬프트로 변환
  • 원본 쿼리와 결합하여 모델 가이드
  • 미세조정된 전용 모델 사용

Reprompter는 교차 패밀리 전이 능력을 보여주며, 한 모델 패밀리에서 학습된 데이터로 다른 패밀리에서도 효과적으로 동작

모델 스케일 일반화 (§4.6)

다양한 크기의 모델에서 일관된 성능 향상을 확인. 작은 모델에서도 효과적으로 동작하며, 모델 스케일에 따른 성능 저하 없음

절제 연구 (§4.8)

각 컴포넌트(변경 추적, CoT Reprompter, 의미적 변환)의 개별 기여도 분석을 통해 프레임워크 설계의 타당성 검증

사례 연구 및 추가 실험

Deep Interaction vs RCoT 사례

동일한 오류가 포함된 CoT 응답에서:

  • RCoT: 추론 체인을 역방향으로 검증하나, 오류 위치를 정확히 파악하지 못해 유사한 실수 반복
  • Deep Interaction: 사용자가 오류 단계를 직접 수정하고 정확한 단계는 그대로 유지하여 올바른 최종 답변 도달

초기 사용자 연구 (G.1)

실제 사용자를 대상으로 Deep Interaction의 사용성과 효과성 검증. 대화 기반 방식보다 만족도 유의미하게 높음

교차 패밀리 전이 (G.2)

CoT Reprompter가 한 모델 패밀리에서 학습되더라도 다른 패밀리의 모델에서도 충실도와 견고성을 유지하며 효과적으로 동작

본 방법은 SafeWork-R1 플랫폼에 Human-CoT 애플리케이션 모드로 실제 배포되어 운영 중

결론 및 시사점

주요 기여

  • CoT 추론 오류에 대한 효율적인 인간 개입 메커니즘 제안
  • 문서 수정 모드 방식의 직접 편집 패러다임 도입
  • 변경 추적 및 의미적 정제를 통한 정밀 교정 달성
  • STEM 과제에서 25%+ 성공률 향상, 40% 토큰 절감
  • 실제 플랫폼(SafeWork-R1)에 배포 및 검증 완료

핵심 통찰

대화 기반 피드백의 근본적 한계는 모델이 "무엇이 잘못되었는지"를 정확히 이해하지 못하는 데 있음. Deep Interaction은 편집 자체가 피드백이 되어 이 문제를 우회

향후 방향

자동 오류 탐지와 결합한 하이브리드 방식, 멀티모달 추론으로의 확장, 더 다양한 도메인에서의 검증이 필요
