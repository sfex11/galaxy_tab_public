Influcoder: 디코더의 기울기 영향력 순위를
인코더로 증류하는 데이터 귀속 방법

Distilling Decoders' Gradient Influence Rankings
into an Encoder for Data Attribution

Dimitri Kachler, Damien Sileo, Pascal Denis · Inria, Université de Lille

문제 정의: 데이터 귀속(Data Attribution)

핵심 질문

훈련 데이터 내 개별 샘플이 모델의 특정 출력을 어떻게 조건화하는지 추정하는 것

예: 어떤 훈련 샘플이 모델의 유독한 응답의 원인인가?

두 가지 활용 분야

  • 데이터 선택: 벤치마크 성능 최적화를 위한 큐레이션
  • 필터링: 원치 않는 행동 유발 샘플 탐지 및 제거

LLM의 능력이 성장함에 따라, 고품질 데이터셋을 구축하기 위해 훈련 데이터를 필터링하는 수요가 급증하고 있다.

기존 방법론의 한계

표현 기반

TF-IDF / RDS+ / Semantic

훈련에 구애받지 않는 특징 공간 활용 → 빠르지만 영향력 신호가 약함

투영 기반 영향 함수

TRAK / LoGRA / LESS / DataInf

기울기 투영으로 메모리 절약 → 여전히 샘플별 기울기 연산 필요

증류 기반

IProX / AirRep

작은 모델로 효율화 → AirRep은 훈련 손실 증류, 기울기 영향력 증류는 없음

공통 문제: 영향 함수 계열은 효과적이나, 대규모 데이터셋에 적용하기엔 연산 속도와 저장 공간이 부족함

Influcoder: 제안 방법 개요

디코더의 기울기 영향력 순위를 인코더로 증류하여, 대규모 데이터 귀속을 빠르고 저렴하게 수행

1
Ground Truth 수집
LoRA 기울기 계산 → CountSketch 투영 → 코사인 유사도 영향력 행렬 구축
2
인코더 증류 학습
풀/쿼리 샘플을 인코더에 통과 → 영향력 임베딩 생성 → 순위 손실로 최적화
3
추론 시 활용
인코더 임베딩 간 코사인 유사도만으로 영향력 순위 예측

핵심 장점

인코더의 높은 처리 속도를 활용하여 대규모 데이터 규모에서도 실용적인 데이터 귀속 가능

설계 철학

샘플을 독립적으로 처리하며, SFT 최적화가 아닌 영향력 순위 보존에 집중

코사인 LoRA 기울기 영향력

왜 1차 근사인가?

전통적 영향 함수는 역헤시안을 사용하지만, 이는 근사 오류로 인해 불안정성을 초래할 수 있음. 증류 과정에서 이 불안정성이 더욱 악화될 수 있으므로 1차 공식을 채택.

1차 영향력 (SGD 기반)

InfSGD(p, q) = η ⟨∇ℓ(θ, p), ∇ℓ(θ, q)⟩

LoRA + 코사인 유사도

  • 기울기를 LoRA 어댑터 파라미터로 제한
  • 내적을 코사인 유사도로 대체
  • 기울기 방향 측정 → 고기울기 노름 샘플 과대가중 방지
  • 응답 전용 기울기 사용 (프롬프트 마스킹) → 더 정보성 높음

최종 영향력 공식

Inf'(p, q) = cos(∇ℓ(θL, p), ∇ℓ(θL, q))

순위 기반 영향력 증류

사전에 전체 풀-쿼리 간 Ground Truth 영향력 행렬 IGT를 계산 후, 인코더가 이 순위를 재현하도록 학습

손실 1 Pearson 상관계수 손실

Spearman은 미분 불가능하므로, 미분 가능한 Pearson 상관계수를 대체 지표로 사용

배치 전체의 순위를 한 번에 정렬하는 효과 → 하지만 Top-k 최적화에는 약함

손실 2 Softmax KL 발산 손실

각 쿼리에 대해 풀 후보들의 영향력 점수에 tempered softmax 적용 후 KL 발산 계산

쿼리별로 Top-k 순위에 집중하는 보완 효과

ℒTotal = α · ℒPearson + (1 − α) · ℒKL,   α ∈ [0, 1]

두 손실을 선형 결합하여 전체 순위 정렬과 Top-k 집중을 동시에 달성

Ground Truth 수집 과정

1
LoRA 기울기 계산

풀 샘플 𝒫과 쿼리 샘플 𝒬를 타겟 디코더에 통과시킨 후, LoRA 파라미터에 대해서만 응답 부분의 기울기 계산

2
CountSketch 투영

저장 공간 절약을 위해 기울기를 CountSketch로 낮은 차원으로 무작위 투영 (Johnson-Lindenstrauss 보존)

3
노이즈 제외

LM 헤드와 어휘 임베딩은 노이즈가 많아 학습을 방해하므로 제외

4
영향력 행렬 구축

IGT ∈ ℝNP × NQ — 모든 풀-쿼리 쌍의 코사인 유사도를 사전 계산하여 저장

학습 중에는 IGT에서 해당 인덱스의 부분행렬을 직접 조회하므로, 추가 기울기 계산 없이 학습 가능

실험 설정: 영향력 순위 추정

실험 구성

  • 타겟 모델: SmolLM2-1.7B (명령어 미세조정 없음)
  • 인코더: ettin-encoder (150M, 68M)
  • 풀 셋: Dolly (200 샘플)
  • 쿼리 셋: BBH (200 샘플)
  • 3회 반복 평균

평가 지표

  • Mean-Aggregated Spearman 상관계수
  • Per-Anchor Spearman 상관계수
  • Raw FLOPs (연산량)
  • GPU 벽시계 시간 (실제 추론 시간)

비교 대상 (Baselines)

저장 공간

영향 함수 방법은 투영된 기울기를 저장해야 하므로 데이터셋 크기에 비례하여 저장 요구량 증가

  • Influcoder: 고정 크기 모델 파라미터만 저장
  • 데이터셋 크기와 무관한 저장 공간
  • 추론 시 기울기 계산 불필요

실험: 유독성 필터링 응용

설정

  • 태스크: XSTest-response (Jiao et al., 2025)
  • 변형: Heterogeneous / Homogeneous
  • 타겟 모델: Pythia-1B
  • 평가 지표: AUPRC (Precision-Recall 곡선 아래 면적)
  • 평가 데이터의 일부만 인코더 학습에 사용
  • 훈련/평가 간 데이터 누수 없음 보장

목적

단일 체크포인트 영향력 추정이 실제 유독성 필터링에 얼마나 유용한지 검증

Jiao et al. (2025)에 따르면 영향 함수 기반 방법은 필터링 태스크에서 일관되게 랜덤 베이스라인을 능가함

Influcoder가 이 장점을 유지하면서도 속도 이점을 제공하는지 확인

결과: 유독성 필터링 성능

XSTest-response (Pythia-1B)에서의 AUPRC 성능

Heterogeneous 설정

Influcoder가 영향 함수 기반 방법과 유사하게 랜덤 베이스라인을 일관되게 상회하며, 표현 기반 방법보다 우수한 필터링 성능을 달성

Homogeneous 설정

동일한 조건에서도 Influcoder의 증류된 영향력 신호가 실제 필터링 유틸리티로 변환됨을 확인

핵심 발견

순위 정확도에서 약간의 손실이 있더라도, 필터링 태스크에서는 실질적인 성능 저하가 미미함 → Top-k 순위가 중요한 응용에서 Influcoder의 KL 손실이 효과적으로 작동

결론 및 핵심 기여

기여 요약

  • 기울기 영향력 순위를 인코더로 증류하는 최초의 방법 제안
  • Pearson + KL 이중 손실로 전체 순위와 Top-k 동시 최적화
  • 영향 함수 수준의 정확도를 인코더 속도로 달성
  • 유독성 필터링 실제 응용에서 유효성 입증

한계점

  • Ground Truth 수집 시 1회 디코더 기울기 계산 필요
  • IProX와 직접 비교 불가 (전체 기울기 vs LoRA 기울기 공간 차이)
  • 단일 체크포인트 기반으로 훈련 과정 전체 영향력은 미포함

향후 방향: 다중 체크포인트 지원, 더 다양한 태스크에서의 검증, 대규모 데이터셋 스케일 테스트
