PagedWeight

동적 품질 인식 가중치 양자화를 통한
효율적인 MoE LLM 서빙

Yuchen Yang, Yifan Zhao, Anisha Dasgupta, Sasa Misailovic

University of Illinois Urbana-Champaign

문제 제기

MoE 서빙에서의 메모리 긴장

GPU 메모리 병목

LLM 서빙 시 GPU 메모리에는 모델 가중치와 KV 캐시가 동시에 상주해야 함

  • MoE 모델 가중치가 GPU 메모리의 60% 이상 점유
  • 긴 컨텍스트에서 KV 캐시 급증
  • 두 요소 간 메모리 경쟁 심화

기존 방법의 한계

PagedAttention은 KV 캐시만 관리하고, 정적 양자화는 서빙 중 적응 불가

  • KV 캐시 압축은 가중치 메모리 해소 불가
  • 정적 양자화는 고정된 비트폭만 사용
  • 런타임 메모리 재할분 기회를 놓침
핵심 질문: KV 캐시가 커질 때, 덜 중요한 전문가 가중치에서 메모리를 동적으로 회수할 수 있는가?

라우팅 질량 (Routing Mass)

전문가의 중요도를 정량화하는 지표

mℓ,e = 해당 전문가의 라우팅 가중치 합 / 전체 라우팅 가중치 합

라우팅 질량이 높은 전문가는 자주 활성화되므로, 낮은 전문가보다 양자화에 더 민감

→ 불균형 활성화 패턴은 차등적 양자화의 기회를 제공

PagedWeight는 양자화 수준 변경을 GPU 메모리 재할분 메커니즘과 결합하는 최초의 시스템

핵심 아이디어

PagedWeight: 페이징된 가중치 관리

전문가 가중치를 KV 캐시의 페이지 블록처럼 관리하자 — PagedAttention이 KV 캐시를 페이징하듯, PagedWeight는 가중치를 페이징한다.

동적 KV/가중치 트레이드오프

KV 캐시가 커지면 가중치의 비트폭을 낮춰 메모리 확보

품질 인식 전문가 선택

비트폭 변경이 출력 품질에 미치는 영향이 최소인 블록 우선 선택

비동기 실행

페이지 이동을 서빙과 겹쳐 실행하여 추가 지연 최소화

기반 기술: Any-Precision LLM (APL) — 비트플레인 오버레이 형식으로 다중 비트폭 표현 가능

비동기 오프로드/리로드

  • 플래너가 di를 결정하면 비동기적으로 페이지 이동 시작
  • 모델 서빙과 겹쳐서 실행 (overlap)
  • 안전 경계에서만 qi를 di에 커밋
  • 추론 중단 없이 메모리 전송 지연 숨김

혼합 정밀도 MoE 커널

  • AP 비트플레인과 LUT를 직접 읽는 융합 커널
  • 전문가별로 다른 비트폭 지원
  • 중간 비트폭 변환 없이 원시 AP 형식 처리
  • 추가 복사 오버헤드 제거
안전 경계: 해당 전문가가 현재 배치에서 사용되지 않는 시점에만 커밋 수행

긴 컨텍스트에서의 품질

KV 캐시 수요가 큰 시나리오에서 PagedWeight의 동적 적응이 특히 효과적

  • 컨텍스트 길이 증가 시 KV 캐시 메모리 확보를 위해 자동으로 가중치 비트폭 하향
  • 정적 양자화는 메모리 부족으로 서빙 불가한 경우에도 동작 가능
  • LongBench v2에서 베이스라인 대비 일관된 품질 우위

서빙 처리량

처리량 손실은 최대 4.1%에 불과

  • 비동기 페이지 이동으로 대부분의 전송 지연 숨김
  • 메모리 절감분을 더 큰 배치 사이즈에 활용 → 순 처리량 증가
  • 융합 혼합 정밀도 커널로 추가 오버헤드 최소화

제거 연구

PagedWeight의 각 구성 요소가 품질에 미치는 기여도 분석

구성 요소 제거품질 영향분석
오프라인 민감도 제거 중간 저하 Hessian 기반 점수 없이는 양자화 대상 선택의 정확도 하락
라우팅 인식 그룹화 제거 중간 저하 자주 사용되는 전문가가 보호 없이 양자화되어 품질 손실
프롬프트 잔차 제거 약간 저하 입력별 보정이 없어 일부 프롬프트에서 예측 오차 증가
비동기 이동 → 동기 처리량 저하 페이지 전송 지연이 추론 경로에 직접 포함
세 가지 품질 신호(오프라인 민감도 + 라우팅 그룹화 + 프롬프트 잔차)가 모두 결합될 때 최적 성능 달성
요약

결론 및 기여

핵심 기여

  • 페이징된 가중치 시스템: 온라인 Any-Precision MoE 가중치의 커밋/희망 비트폭, 페이지 상태, 메모리 관리
  • 품질 인식 런타임 플래너: 오프라인 민감도 + 라우팅 통계 + 프롬프트 잔차 결합
  • 비동기 페이지 이동: 최소 처리량 손실(최대 4.1%)로 메모리 전송 지연 숨김
  • 세 가지 인기 MoE 모델에서 모든 베이스라인 일관된 우위 입증

의의

MoE 서빙에서 모델 가중치와 KV 캐시 간의 동적 메모리 트레이드오프라는 새로운 설계 공간을 개척

정적 양자화의 "고정된 비트폭" 패러다임에서, 서빙 조건에 맞춰 메모리를 동적으로 재할분하는 패러다임으로 전환

PagedAttention이 KV 캐시를 페이징한 것처럼, PagedWeight는 가중치를 페이징하여 LLM 서빙 메모리 관리의 새로운 축을 제공
