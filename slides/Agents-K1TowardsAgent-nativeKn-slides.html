체계적 연구 부재

기존 한계

과학 지식 인프라의 세 가지 한계

1

축소된 표현 — 초록만, 멀티모달 증거 누락

LightRAG, HippoRAG 등은 텍스트 전용 트리플만 생성. 그림·표·수식은 캡션으로 축소되어 핵심 증거가 묻힘

2

평탄한 인용 그래프 — cites 엣지만 존재

논문 간 "인용" 관계만 표시. 메서드 확장, 주장 도전, 베이스라인 인용 등 의도 구분 불가

3

런타임 중복 추출 — 추적 불가

에이전트가 매 쿼리마다 원본 PDF를 다시 읽고 추출. 답변을 정확한 증거로 소급 추적 불가

설계 목표

에이전트 네이티브 지식 오케스트레이션

연구 에이전트가 신뢰할 수 있는 지식을 사용하려면 다음 세 가지 요구사항을 충족해야 합니다.

01

전문 보장

텍스트·그림·표·수식을 분리된 산물이 아닌 연결된 증거로 취급

02

유형화된 지식

엔티티·주장·기전·메서드 링크·인용 의도·관계를 타입화

03

감사 가능한 검색

모든 답변·결정을 안정적인 그래프 ID와 정확한 증거로 추적

프레임워크

Agents-K1 전체 아키텍처

원본 문서를 에이전트 네이티브 멀티모달 지식 그래프로 변환하는 엔드투엔드 파이프라인

동일 파이프라인은 과학 논문뿐 아니라 임의의 문서 코퍼스에도 재적용 가능

KG Layer

멀티모달 파서와 5모듈 스키마

MinerU 기반 오프라인 파서가 전체 논문을 수집하고, 5개 모듈 스키마로 구조화합니다.

  • A. 메타/사실 엔티티 — 논문 메타데이터, 핵심 사실
  • B. 텍스트 언급 엔티티 — 본문에 명시적으로 등장하는 개념
  • C. 암시적/추상화 엔티티 — 텍스트에 직접 언급되지 않으나 추론 가능한 개념
  • D. 인용 관계 — 단순 cites가 아닌 의도 분류(확장·도전·베이스라인 등)
  • E. 엔티티 간 관계 — 세분화된 타입의 상호 관계

핵심 차별점

그림·표·수식을 캡션이 아닌 1급 증거(first-class evidence)로 취급하여 텍스트 엔티티와 동등하게 연결

데이터셋 구성

이론적 기반

식별자 보존 조인, 교차 뷰 도달성, 후보 커버리지에 대한 정리 증명으로 설계 정당화

LLM Layer

GRPO 학습 정보 추출 백본

학습 방법

GRPO(Group Relative Policy Optimization)로 4B 파라미터 모델 학습

  • 형식 준수(Form compliance)
  • JSON 유효성(JSON validity)
  • 태스크 조건부 F1 (NER·관계 추출·장문 구조화 추출)

성능 요약

8B 오픈소스 기준 모델 대비 10개 벤치마크에서 우수
Qwen3-32B과의 스케일 갭(약 8배)을 거의 해소
NER 태스크에서는 Qwen3-32B을 초과
관계 추출에서 약간의 갭 잔존

새로운 도메인에 저비용으로 재학습 가능한 경량 모델 설계

CLI Layer

그래프 오퍼레이터와 멀티 에이전트

6가지 그래프 오퍼레이터

  • O1 시드 해결 (Seed Resolution)
  • O2 인용 계보 재구성 (Citation Lineage)
  • O3 비교 베이스라인 검색
  • O4 멀티모달 앵커 검색
  • O5 갭 탐지 (Gap Detection)
  • O6 아이디어 그라운딩 및 신규성 판정

아이디어→실험 파이프라인

반복적 아이디어 정제 → 메서드 명세 작성 → 코드 자동 합성의 폐루프 워크플로

접근 방식

CLI / MCP / API를 통한 에이전트 접근. Swarm 런타임에서 역할 분할 및 툴 서피스 제공
