동기: 에이전트 사후학습의 두 축

강화학습 (RL)
  • 환경/검증기 피드백 기반
  • 궤적 수준의 조밀하지 않은 보상
  • 긴 호라이즌 상호작용에 대한 감독이 거침
온폴리시 자기증류 (OPSD)
  • 토큰 수준의 조밀한 안내 신호
  • 특권 컨텍스트가 증강된 교사 분기
  • 멀티턴 환경으로의 전이가 문제적
핵심 질문: OPSD를 멀티턴 에이전트에 안정적으로 적용하려면 어떻게 해야 하는가?

관찰 1: 멀티턴 OPSD 불안정성

학생 에이전트가 교사 지원 궤적에서 inevitably 이탈하면, 한때 유용했던 토큰 수준 감독이 점점 더 신뢰할 수 없게 된다.

누적 오류 메커니즘
  • 턴이 진행될수록 교사-학생 간극 확대
  • 턴별 KL 발산 급증
  • 작업 성능의 치명적 저하 유발
기존 접근법의 한계
  • TCOD: 경직된 시간 스케줄 의존
  • RLSD: 자기발산으로 토큰 RL 이점 재가중 → 초기 불안정성 악화

관찰 2: 특권 안내에 대한 비대칭적 신뢰

OPSD의 교사는 독립적으로 더 강한 모델이 아닌, 특권 컨텍스트로 증강된 동일 정책이다.

양의 갭 (긍정적 지지)

교사가 학생보다 높은 확률을 할당 → 검색된 스킬이 온폴리시 행동을 보증
증류에 매우 적합

음의 갭 (부정적 거부)

교사가 학생보다 낮은 확률을 할당 → 스킬 품질/활용 문제일 수 있음
보수적으로 해석 필요

음의 갭의 세 가지 근원
스킬 품질
관련 없거나 불완전한 스킬
스킬 활용
관련 스킬을 토큰 선호도로 정박 실패
멀티턴 드리프트
턴 간 교사-학생 갭 확대

Qwen2.5-3B 기준, 음의 갭 토큰이 전체 토큰의 50% 이상을 차지

SDAR: 핵심 아이디어

RL을 주 최적화 백본으로 유지하고, OPSD를 게이팅된 보조 목표로 격하한다.
1
RL 백본 보존
GRPO 손실을 그대로 유지하여 검증기 기반 이점의 의미론과 무편향성 엄격 보존
2
비대칭 게이팅
시그모이드 게이트로 양의 갭 토큰은 강화, 음의 갭 토큰은 부드럽게 감쇠
3
토큰 수준 자가 커리큘럼
각 토큰이 자신의 감독 강도를 결정하는 가장 세밀한 자기 주도 학습
RLSD와의 차이

RLSD는 자기발산으로 RL 이점을 재가중하지만, SDAR는 OPSD 손실을 직접 보조 목표로 처리하여 RL 정책 손실을 변형하지 않음

Skill-SD와의 차이

Skill-SD는 경직된 수동 스케줄링을 사용하지만, SDAR는 적응적이고 부드러운 토큰 수준 게이팅을 사용

문제 설정 및 스킬 검색

멀티턴 에이전트 설정

초기 프롬프트 x에서 시작하여, 턴 k마다 관측 ok을 받고 응답 ak을 생성

전체 궤적의 유효 토큰을 단일 시퀀스 y = (y₁, ..., yT)로 평면화
학생 컨텍스트: st = (x, y<t)
교사 컨텍스트: st+ = (x, c+, y<t)
스킬 검색 전략

멀티암드 밴딧 문제로 공식화. 스킬 라이브러리 ε = {e₁, ..., eM}에서 태스크 관련 스킬 검색

UCB 탐색-활용 균형 기반 선택
KM 키워드 매칭 기반 직접 검색
Full 전체 스킬 제공
Random 무작위 검색 (견고성 평가용)

최적화 목표

ℒ(θ) = ℒGRPO(θ) + λSDAR · ℒSDAR(θ)

GRPO 손실에 게이팅된 OPSD 보조 목표를 추가

ℒGRPO: RL 최적화
  • G개 샘플 그룹에서 환경 보상 기반 시퀀스 수준 이점 A(i) 계산
  • 클리핑된 중요도 샘플링 비율 적용
  • 참조 정책 πref에 대한 KL 정규화 (β 제어)
ℒSDAR: 게이팅된 OPSD
  • 토큰 위치 t에서 교사-학생 역 KL 발산 계산
  • 시그모이드 게이트 gt로 토큰별 증류 강도 제어
  • detach된 신호 기반 → RL 그래디언트 흐름에 영향 없음

토큰 수준 게이팅 메커니즘

핵심 철학: 각 토큰이 자신의 감독 강도를 결정하게 한다.

gt
시그모이드 게이트

detach된 토큰 수준 신호를 시그모이드에 매핑하여 [0,1] 범위의 게이트 값 생성

양의 갭 → 1에 가까움
긍정적 지지 강화

교사가 학생보다 높은 확률을 할당한 토큰에 대해 증류를 강하게 적용

음의 갭 → 0에 가까움
부정적 거부 감쇠

교사가 낮은 확률을 할당한 토큰에 대해 증류를 부드럽게 약화

검색 품질에 대한 견고성

SDAR의 게이팅 설계는 저품질 스킬의 노이즈를 자동으로 필터링하고 유익한 신호만 증류한다.

절제 연구

게이팅 전략
  • 게이팅 없음: GRPO+OPSD와 동일한 불안정성
  • 하드 임계값: 성능 변동 큼
  • 시그모이드 게이팅: 가장 안정적이고 우수
하이퍼파라미터 민감도
  • λSDAR: 넓은 범위에서 안정적 성능 (0.1~1.0)
  • β (KL 날카로움): 적절한 범위 내 견고
  • 순방향 KL vs 역방향 KL: 역방향이 더 효과적
증류 목표 비교
  • 역방향 KL (SDAR 기본): 우수
  • 순방향 KL: 상대적으로 낮은 성능
  • 교차 엔트로피: 중간 수준
훈련 역학
  • GRPO+OPSD: 훈련 중 급격한 성능 하락
  • RLSD: 초기 업데이트 과대 증폭
  • SDAR: 안정적인 수렴 곡선
