원시 경험에서 스킬 소비까지

모델 생성 에이전트 스킬에 대한 체계적 연구

Zisu Huang, Jingwen Xu, Yifan Yang 외  · 复旦大学, Microsoft Research, 上海交通大学

배경과 동기

에이전트 스킬: 경험의 재사용

언어 에이전트는 과거 궤적에서 증류된 스킬—짧고 구조화된 절차적 산물—을 재사용하며 지속적으로 개선됩니다.

  • 도메인 수준 스킬: 도메인 내 반복 절차를 하나의 산물로 패키징
  • 모델 생성 스킬: 수작업 없이 자동으로 대규모 생산 가능
  • 추론 시 로드 가능하여 재학습 불필요

상용 에이전트 플랫폼에서 스킬은 이미 표준 컴포넌트로 자리 잡았으나, 수작업으로는 에이전트 능력 확장 속도를 따라갈 수 없습니다.

핵심 질문: 자동 생성된 스킬은 실제로 작동하는가? 언제 작동하며, 무엇이 성공과 실패를 결정하는가?

연구 공백

기존 연구의 한계

기존 벤치마크 다루는 단계 주요 한계
SkillsBench 소비만 인간 작성 스킬만 사용, 추출 단계 미포함
SWE-Skills-Bench 소비만 공개 스킬 저장소 기반, 추출 외부
Skills-in-the-Wild 소비만 야생 스킬 발견에 집중
SkillCraft 추출+소비 실행 가능한 함수 조합으로 제한, 일반화 불명확

공통적 공백: 경험 생성 → 스킬 추출 → 스킬 소비의 전체 라이프사이클을 아우르는 체계적 연구가 부재합니다.

연구 질문

세 가지 핵심 질문

RQ1

모델 생성 도메인 수준 스킬이 타겟, 추출기, 도메인 전반에서 안정적으로 이익을 주는가?

RQ2

경험 생성, 스킬 추출, 스킬 소비의 각 라이프사이클 단계에서 무엇이 스킬의 하류 유용성을 실제로驱动하는가?

RQ3

실증적 발견을 구체적인 개선 방법으로 변환할 수 있는가?

평가 프레임워크

스킬 라이프사이클의 세 단계

1
경험 생성
타겟 모델 M이 학습 분할의 태스크를 실행하여 성공/실패 궤적 풀 𝒯 생성
2
스킬 추출
추출기 E가 궤적 풀을 증류하여 단일 도메인 수준 스킬 𝒮 생성
3
스킬 소비
생성된 스킬을 타겟 모델에 주입하고 테스트 분할에서 성능 변화 측정

설계 원칙: 추출 프레임워크는 최소한의 스캐폴딩만 사용하여 추출기 자체의 능력을 반영하도록 설계했습니다.

추출 프레임워크

궤적에서 스킬로의 증류 과정

궤적별 분석

각 궤적에서 재사용 가능한 절차적 패턴을 개별적으로 식별

계층적 통합

궤적별 패턴을 중복 제거 및 병합하여 일관된 스킬 구조로 정리

스킬 표현

최종적으로 단일 도메인 수준 스킬 산물로 합성

이 3단계 파이프라인은 Trace2Skill 등 기존 방법과 동일한 핵심 구조를 따르되, 특정 스캐폴딩 트릭 없이 추출기 모델의 순수 능력을 평가할 수 있도록 최소화했습니다.

평가 지표

추출 효능과 타겟 진화 가능성

EE — Extraction Efficacy

추출 효능

고정된 추출기 E가 다양한 타겟 모델에 대해 얼마나 일관되게 유용한 스킬을 생산하는가?

추출기 관점: "내가 뽑은 스킬이 남에게도 통하는가?"

TE — Target Evolvability

타겟 진화 가능성

고정된 타겟 M이 다양한 추출기가 자신의 경험에서 뽑은 스킬로부터 얼마나 이익을 얻는가?

타겟 관점: "나는 남이 뽑은 스킬을 잘 활용하는가?"

EE와 TE는 스킬 유용성을 추출기 역할과 소비자 역할로 분리하여 독립적으로 평가합니다.

실험 설정

5개 도메인, 다양한 모델 조합

 5개 에이전트 태스크 도메인

  • 구현된 계획 (Embodied Planning)
  • 생산성 소프트웨어 (Productivity Software)
  • 소프트웨어 엔지니어링 (Software Engineering)
  • 웹 검색 (Web Search)
  • 도구 호출 (Tool Calling)

 실험 설계

  • 추출기와 타겟을 독립적으로 교차 조합
  • 학습/테스트 분할 엄격히 분리
  • 스킬 없는 베이스라인 대비 성능 변화를 유용성의 프록시로 사용
  • 복수 회 반복 실행 및 집계

주요 결과

스킬은 평균적으로 유익하지만, 보장되지는 않는다

긍정적 발견

  • 모델 생성 스킬은 전반적으로 하류 성능 향상에 기여
  • 수작업 스킬 없이도 자동화된 파이프라인으로 실질적 이익 달성 가능

부정적 전이

  • 비무시할 수 있는 수준의 네거티브 트랜스퍼 관찰
  • 스킬 장착이 오히려 성능을 저하시키는 사례 존재

핵심 인사이트: 강한 실행기가 반드시 강한 추출기가 아니며, 스킬 유용성은 모델 규모나 베이스라인 태스크 성능과 독립적입니다.

주요 결과

추출기와 소비자의 비일관적 행동

역할 분리

강한 추출기가 약한 소비자일 수 있고, 그 역도 성립

타겟 의존성

동일 스킬이라도 소비자에 따라 유용성이 크게 달라짐

규모 무관

EE와 TE는 모델 파라미터 규모와 단순 비례하지 않음

이 결과는 스킬 생태계에서 추출 능력과 소비 능력이 별개의 축으로 측정되어야 함을 시사합니다.

심층 분석

라이프사이클 각 단계의 동인 분석

STAGE 1

경험 생성

성공 궤적만이 좋은 스킬을 만드는 것은 아님. 실패 경험의 구성이 스킬 품질에 미묘한 영향을 미침

STAGE 2

스킬 추출

텍스트적 그럴싸함이 유용성을 예측하지 못함. 구체적 해결책이 일반적 조언보다 유용

STAGE 3

스킬 소비

크로스 모델 전이에서 스킬 효과가 비일관적. 소비자의 의사결정·탐색·도구 사용 행동이 변화

심층 분석

좋은 스킬의 특성: 형식이 아닌 내용

낮은 유용성  일반적 조언

"항상 주의 깊게 생각하고, 가능한 옵션을 고려하여 신중하게 결정하세요."

텍스트적으로 그럴싸하지만, 실제 실행에 구체적 지침을 제공하지 못함

높은 유용성  구체적 해결책

"파일 삭제 전 반드시 백업을 생성하고, 삭제 후 파일 목록을 재확인하여 의도치 않은 삭제를 방지하세요."

특정 행동과 조건을 명시하여 에이전트의 실행을 직접적으로 가이드

대조 분석 결과: 스킬 품질은 표면적 형식으로 환원 불가하며, 구체성(concreteness)과 행동 가이딩 가능성이 핵심 차이점입니다.

해결책

메타-스킬 유도 추출

진단에서 개입으로

분석을 통해 발견한 "좋은 스킬의 특성"을 메타-스킬이라는 형태로 추출 프롬프트에 주입합니다.

1
루브릭 생성
대조 분석으로 유용/무용 스킬 구분 루브릭 도출
2
메타-스킬 주입
루브릭을 추출 프롬프트에 가이드로 통합

효과

  • 모든 도메인에서 스킬 품질 일관적 향상
  • 네거티브 트랜스퍼 대폭 감소
  • 추출기 변경 없이 드롭인 방식으로 적용 가능
  • 추가 학습이나 최적화 불필요

결론

휴리스틱에서 원칙 기반으로

주요 기여

  • 스킬 라이프사이클 전체를 아우르는 최초의 체계적 연구
  • EE와 TE 지표로 추출기·소비자 역할 분리 평가
  • 각 단계별 동인에 대한 심층 분석
  • 메타-스킬 유도 추출이라는 구체적 해결책 제시

핵심 메시지

스킬 라이프사이클을 직관 주도의 휴리스틱에서 유용성 기반의 원칙적 규율로 전환해야 합니다.

이종 모델과 도메인에 걸쳐 스킬 라이브러리가 확산됨에 따라, 실제 배치에서 안정적이고 효과적인 스킬을 얻는 것이 점점 더 중요해집니다.
