AI 에이전트는 토큰을 어디에 소비하는가?

2

어떤 모델이 더 토큰 효율적인가?

동일한 작업에서 모델 간 토큰 소비량과 작업 성능 간의 트레이드오프 비교

3

토큰 소비를 사전에 예측할 수 있는가?

연구 범위: 에이전트 코딩 작업

소프트웨어 개발 워크플로우에서 에이전트가 수행하는 전형적인 코딩 작업들

작업 유형

  • 기능 구현 (Feature Implementation)
  • 버그 수정 (Bug Fixing)
  • 리팩토링 (Refactoring)
  • 테스트 작성 (Test Generation)
  • 문서화 (Documentation)

에이전트 특성

  • 다단계 추론 및 계획 수립
  • 코드 생성 → 실행 → 피드백 루프
  • 파일 시스템 탐색 및 수정
  • 오류 시 자가 교정 반복
  • 컨텍스트 축적에 따른 토큰 증가

연구 방법론

토큰 소비를 체계적으로 분석하고 예측하기 위한 파이프라인

1
데이터 수집

2
소비 분해

3
모델 비교

4
예측 모델링
작업 특성 기반 토큰 소비 예측 모델 구축 및 검증

RQ1: 토큰은 어디에 소비되는가?

에이전트 루프 내 구성 요소별 토큰 소비 비중 분석 결과

에이전트 단계 소비 비중 특징
컨텍스트 구성 가장 높음 누적된 대화 및 파일 내용
코드 생성 높음 출력 토큰 중심 소비
계획 및 추론 중간 입력 토큰 중심
검증 및 테스트 중간 실행 결과 파싱
오류 수정 루프 가변적 실패 시 급증

핵심 발견

  • 입력 토큰이 출력 토큰보다 압도적으로 많음
  • 컨텍스트 누적이 비용 증가의 주요 원인
  • 오류 수정 루프가 예측 불가능한 비용 폭증 유발
  • 작업 복잡도보다 루프 반복 횟수가 비용을 더 좌우

RQ2: 어떤 모델이 더 효율적인가?

모델 간 토큰 소비량 대비 작업 성능 비교

성능-비용 트레이드오프

강력한 모델

더 적은 반복으로 성공 → 총 토큰 감소 가능

약한 모델

더 많은 오류 수정 루프 → 총 토큰 급증

단순한 단일 호출 비용이 아닌 에이전트 루프 완료까지의 총 비용으로 비교해야 함

주요 비교 지표

  • 토큰 당 성공률 (Success per Token)
  • 작업 완료 총 토큰 (Total Tokens to Completion)
  • 루프 반복 횟수 (Iteration Count)
  • 입력/출력 토큰 비율 (I/O Ratio)

핵심 높은 단가 모델이 오히려 총 비용은更低한 역설적 결과 관찰

RQ3: 토큰 소비 예측

작업 시작 전 토큰 소비량을 예측할 수 있는가?

예측 특성 (Features)

  • 작업 설명 길이 및 복잡도
  • 대상 코드베이스 크기
  • 관련 파일 수
  • 작업 유형 (버그 수정 vs 기능 구현)
  • 사용 모델 종류

예측 결과

  • 작업 수준 특성만으로도 유의미한 예측 가능
  • 코드베이스 크기가 가장 강한 예측 인자
  • 오류 수정 루프 횟수는 예측 난이도가 높음
  • 비용 사전 추정 → 예산 할당에 활용 가능
예측 모델은 에이전트 배포 전 비용 예산 수립과 모델 선택 최적화에 실질적으로 활용될 수 있다.

에이전트 토큰 소비의 구조적 이해

단일 LLM 호출과 에이전트 루프의 토큰 소비 구조 차이

단일 LLM 호출

1

프롬프트 입력 → 고정된 토큰

2

모델 응답 → 예측 가능한 토큰

비용 = f(입력 길이, 출력 길이) — 비교적 단순

에이전트 루프

1

계획 → 코드 생성 → 실행

2

결과 평가 → 컨텍스트 축적

3

오류 시 수정 루프 반복...

비용 = f(루프 횟수, 누적 컨텍스트, ...) — 고도로 복잡

실무적 시사점

연구 결과가 에이전트 배포 실무에 주는 함의

비용 예산 수립

예측 모델을 활용해 작업별 비용을 사전에 추정하고 예산 할당

지능적 모델 선택

작업 복잡도에 따라 비용-성능 최적 모델을 동적으로 선택

컨텍스트 최적화

컨텍스트 누적을 제어하는 설계로 입력 토큰 폭증 방지

단가가 낮은 모델을 선택하는 것이 항상 비용 절감이 아니다. 오류 수정 루프 증가로 인해 총 비용이 더 높아질 수 있으므로, 에이전트 완료 비용(End-to-End Cost) 기준으로 평가해야 한다.

결론

분해 — 토큰 소비의 구조적 규명

에이전트 루프 내에서 컨텍스트 구성이 가장 큰 비용 요소이며, 입력 토큰이 출력 토큰을 압도함

비교 — 모델 효율성의 재정의

단일 호출 비용이 아닌 에이전트 완료 총비용으로 효율성을 평가해야 하며, 강한 모델이 오히려 더 저렴할 수 있음

예측 — 비용 예측의 실용화

작업 수준 특성만으로 토큰 소비를 유의미하게 예측할 수 있어, 예산 수립 및 모델 선택에 활용 가능

AI 에이전트의 토큰 소비는 단순한 API 호출 비용이 아닌,
에이전트 시스템 설계에 의해 결정되는 구조적 비용이다.
