문제 정의

왜 AMIGO가 필요한가?

기존 평가의 한계

현재 비전-언어 모델 평가는 단일 이미지, 단일 턴 정확도에 집중되어 있어, 에이전틱 시스템의 핵심 역량을 측정하지 못합니다.

🚫

기존 방식

  • 단일 이미지 입력
  • 1회 질의-응답
  • 전략/기억력 측정 불가
✅

AMIGO

  • 다중 이미지 갤러리
  • 장기 다중 턴 상호작용
  • 전략 + 상태 추적 평가

💡 핵심 동기: 모델이 정보를 수집하고, 상태를 유지하며, 불확실성 하에서 적응하는 능력 — 에이전틱 시스템의 근본적 역량을 평가해야 합니다.

핵심 도전

세 가지 핵심 역량

1

불확실성 하의 질문 선택

모호성을 효율적으로 줄이는 질문을 선택하여 후보군을 빠르게 좁히는 능력

2

일관된 제약 조건 추적

여러 턴에 걸쳐 수집된 증거를 드리프트 없이 유지하고 적용하는 능력

3

정밀 판별

증거가 누적됨에 따라 시각적으로 유사한 이미지 간 미세한 차이를 구분하는 능력

🎯 이 세 역량은 단일 턴 평가에서는 관찰할 수 없는
에이전틱 고유의 도전 과제입니다.

프로토콜

상호작용 프로토콜

AMIGO는 엄격한 10가지 규칙 기반 프로토콜로 모델을 평가합니다.

Q

턴당 1개 질문

관찰 가능한 속성에 대한 Yes/No/Unsure 질문만 허용

S

Skip 페널티 시스템

프로토콜 위반 시 정보 없이 Skip 응답 반환 — 턴 낭비

금지 속성 (고의 제외)

소매 길이 옷 길이 색상 프린트/패턴 연령대 사이즈 배경

→ 미묘한 구조적 디테일(네크라인, 봉제선, 드레이프, 장식)에 의존하도록 유도

🏁 최종 추측: 후보가 정확히 1개일 때만 — "My guess of your favorite dress: #N"

프레임워크

4개 통합 모듈

🤖

1. 벤치마크 모델

질문 생성 및 최종 추측 수행

🛡

2. 질문 위반 탐지기

LLM 기반 프로토콜 강제 — 위반 질문에 Skip 부여

👤

3. 사용자/오라클 시뮬레이터

VLM 에이전트 + 이미지 줌인 도구로 Yes/No/Unsure 응답

✅

4. 검증 모듈

후보 집합 관리, 궤적 감사, 모순 탐지

🔄 모델 질문 → 위반 검사 → 오라클 응답 → 검증 & 후보 축소 → 반복

데이터셋 태스크

"내가 좋아하는 드레스 맞추기"

4,880
고유 드레스 이미지

Target 온라인 패션 카탈로그 출처

5
유사도 임계값 (τ)

0.3, 0.4, 0.5, 0.6, 0.8

난이도 제어 메커니즘

τ=0.3 (쉬움, 큰 갤러리) 1,000 에피소드
τ=0.5 (중간) 1,000 에피소드
τ=0.8 (어려움, 작은 갤러리) 587 에피소드

비대칭 유사도 점수

Sim(A,B) = |Attr(A) ∩ Attr(B)| / |Attr(A)|

타겟과 속성 중복도가 τ 이상인 이미지를 교란자로 선택

파이프라인

속성 라벨링 파이프라인

3단계 반자동 프로세스로 고품질 속성 레이블 생성

1

속성 발견 및 정규화

카탈로그 주석 수집 → 다중 LLM으로 동의어 정규화 → 종합 목록 구성

2

이진 질문 템플릿 구성

각 속성 값에 대해 다양한 Yes/No 질문 템플릿 생성

3

VLM 기반 라벨링

3개 오픈소스 VLM의 가중 다수결 투표

품질 보증 체계

  • 앙상블 가중치 적용
  • 패러프레이즈 템플릿
  • 다중 해상도 증강
  • 수동 레이블 감사

사용 VLM: Qwen3-VL-235B-FP8 (높은 가중치) + Intern-S1 + GLM-4.5V

평가 지표

4축 평가 체계

🎯 식별 정확도

  • 전체 (비검증) 정확도 — 최종 추측이 타겟과 일치하는 비율
  • 검증된 정확도 — 후보 집합이 정확히 1개(타겟)인 경우만 인정
  • 랜덤 추측 정확도 — 후보 >1인 상태에서 우연히 맞춘 비율

⏱ 상호작용 효율성

최종 추측까지의 총 턴 수 (Skip 포함) — 적을수록 효율적

📋 프로토콜 준수도

  • Skip 비율 — 규칙 위반 빈도
  • 조기 출력 비율 — 업로드 완료 전 질문 시도

🛡 노이즈 강건성

Yes↔No 응답 뒤집기 또는 Unsure 교란 주입 후 정확도 측정
→ 모델의 재확인 및 복구 능력 진단

실패 분석

주요 실패 모드

🔁

반복 규칙 위반

동일 속성 유형을 여러 턴에 걸쳐 열거하거나 (규칙 5) 금지 속성을 질문 (규칙 4)

예: Qwen3-VL, τ=0.8, 6개 후보 — 9턴 중 4턴이 Skip

🔄

재열거 루프

이미 확인된 속성의 조합을 반복 순회 — 새로운 특징 탐색 실패

예: Qwen3.5, τ=0.5, 14개 후보 — 20턴 중 12턴이 Skip, 9턴 재열거 루프

⏰

조기 추측 & 예산 소진

충분한 증거 없이 일찍 추측하거나, 20턴을 모두 사용하고도 추측하지 못하는 양극단

🧩

불충분한 제약 조건 통합

유효한 증거를 축적하지만 제약을 통합하여 고유 식별에 실패

핵심 인사이트

모델별 행동 특성

Qwen3-VL-235B

공격적 낮은 프로토콜 준수
  • 큰 갤러리에서 상대적으로 우수
  • 충분한 증거 없이 조기 추측 경향
  • 높은 전체 정확도 ≠ 높은 검증 정확도

Qwen3.5-397B

보수적 최고 검증 정확도
  • τ=0.3~0.6에서 검증 정확도 최고
  • 가장 높은 Skip율 — 프로토콜 준수 어려움
  • 실패 시에도 끝까지 질문 지속

Step3-VL-10B 🌟

10B 소형 τ=0.8 최강
  • 10B 파라미터임에도 가장 어려운 조건에서 최고 성능
  • 시각적 혼동이 큰 후보군에서 뛰어난 판별력
  • 갤러리 크기 증가에는 민감
기여 결론

핵심 기여 및 결론

① 새로운 평가 패러다임

장기 지평 계획과 정밀 교차 이미지 그라운딩을 강조하는 대화형 다중 이미지 숨김 타겟 식별 벤치마크 제안

② 제약 기반 프로토콜

무효 행동 피드백(Skip)이 포함된 제약된 Yes/No 프로토콜로 지속적 지시 따르기 및 에이전틱 실패 모드 진단

③ 오라클 불완전성

제어된 오라클 결함 주입을 통한 강건성 테스트 — 일관되지 않은 피드백 하의 검증 행동 탐침

④ 다차원 평가 체계

결과뿐 아니라 상호작용 품질까지 측정하는 포괄적 메트릭 스위트

🚀 향후 가능성

잘 정의된 상태-행동-관찰 구조로 오프라인 강화학습 기반 멀티모달 정책 훈련을 자연스럽게 지원합니다.

AMIGO 요약

🔍 무엇을 — 다중 이미지 갤러리에서 숨겨진 타겟을 찾는 장기 대화형 벤치마크

❓ 왜 — 기존 단일 턴 평가로는 에이전틱 역량을 측정할 수 없기 때문

⚙ 어떻게 — 10개 규칙 기반 Yes/No/Skip 프로토콜 + 4개 모듈 프레임워크

📊 결과 — 모델 크기 ≠ 성능; 검증된 추론이 핵심; 실패 모드 다양
