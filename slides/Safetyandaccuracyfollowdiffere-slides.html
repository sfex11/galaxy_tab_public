Safety and accuracy follow different scaling laws in clinical large language models

Sebastian Wind, Tri-Thien Nguyen, Jeta Sopa 등 · FAU Erlangen-Nürnberg, RWTH Aachen

arXiv:2605.04039v1 · 2026

문제 제기

정확도가 높으면 안전할 것이라는 암묵적 가정

임상 LLM 배포 시 모델 크기, 문맥 길이, 검색 복잡도, 추론 시 연산량을 늘리는 스케일링이 일반적입니다. 이때 평균 정확도 향상이 곧 안전한 임상 행동으로 이어질 것이라는 암묵적 기대가 있습니다.

일반적인 기대

  • 모델이 커지면 정확도 상승
  • 정확도가 높으면 안전도 높음
  • 추론 연산을 늘리면 답 안정화

의학에서의 현실

  • 소수의 고위험 오류가 치명적
  • 잘못된 답에 높은 자신감 부여 가능
  • 증거와 모순되는 답변 발생
핵심 통찰

동일한 오답이라도 임상적 위험도는 다르다

기존 벤치마크는 모든 오답을 동등하게 취급하지만, 실제 임상에서는 오답의 종류, 위험도, 자신감 수준이 배포 위험을 결정합니다.

A

안전한 오답

낮은 위험도 · 적절한 불확실성

B

고위험 오류

치료 계획 변경 가능 · 임상적 중대함

C

위험한 과신

고위험 오류 + 높은 자신감

"안전성은 스케일링의 수동적 결과가 아니라, 증거 품질, 검색 설계, 문맥 구성, 집단적 실패 행동에 의해 형성되는 배포 속성이다."

프레임워크

SaFE-Scale: 안전성 중심 스케일링 평가 프레임워크

임상 LLM의 안전성이 모델 스케일, 증거 품질, 검색 전략, 문맥 노출, 추론 시 연산량에 따라 어떻게 변하는지 측정하는 체계적 프레임워크입니다.

1
평가 축
정확도 · 고위험 오류 · 불안전 답변 · 위험한 과신
2
배포 조건
6가지 조건 팩토리얼 교차 실험
3
벤치마크
RadSaFE-200: 옵션 수준 안전 라벨
4

34개 LLM, 7개 모델 패밀리

총 34 모델 × 6 조건 × 200 문항 = 40,800개 모델-조건-문항 평가

벤치마크

RadSaFE-200: 방사선 안전성 중심 평가 벤치마크

구성

  • 200개 다지선다형 방사선 문항
  • 기존 문항집 + 새로 큐레이션된 사례 통합
  • 문항당 4~5개 선택지
  • 임상의가 작성한 정제 증거 제공
  • 임상의가 작성한 갈등 증거 제공

옵션 수준 라벨

각 선택지마다 임상의가 정의한 안전 라벨 부여:

라벨의미
고위험 오류치료 계획 변경 가능
불안전 답변환자 안전에 부정적 영향
증거 모순제공된 증거와 배치
정상안전한 범위 내 오답

기존 리더보드가 아닌, 통제된 배포 조건 하에서 안전성 변화를 측정하는 도구로 설계

실험 설계

6가지 배포 조건

Closed-book

외부 증거 없이 제로샷 프롬프팅만 수행

Clean Evidence

임상의가 작성한 정확한 정제 증거 제공

Conflict Evidence

의도적으로 오도하는 갈등 증거 제공

Standard RAG

Radiopaedia 기반 표준 검색 증강 생성

Agentic RAG

RaR 프레임워크 기반 에이전트 검색 + 다단계 추론

Max-context

최대 문맥 길이로 증거 노출 극대화

보조 실험: 8개 대표 모델에서 5샘플 자기일관성(self-consistency) · 4개 고정 3모델 앙상블 다수결 투표

주요 결과

정제 증거가 안전성을 극적으로 향상시킨다

지표 Closed-book Clean Evidence 변화량
정확도 73.5% 94.1% +20.6%p
고위험 오류 12.0% 2.6% −9.4%p
증거 모순 12.7% 2.3% −10.4%p
위험한 과신 8.0% 1.6% −6.4%p
평균 자신감 92.9% 97.6% +4.7%p

정제 증거는 정확도뿐 아니라 고위험 오류, 모순, 과신까지 동시에 감소시키는 유일한 조건이었습니다.

핵심 발견

RAG는 정제 증거의 안전성 프로필을 재현하지 못한다

Standard RAG

지표값
정확도76.0%
고위험 오류여전히 높음
증거 모순11.7%
위험한 과신감소하지 않음

Agentic RAG

지표값
정확도78.1% (+2.1%p)
고위험 오류여전히 높음
증거 모순9.0% (개선)
위험한 과신감소하지 않음

에이전트 RAG는 정확도와 모순률을 개선했지만, 고위험 오류와 위험한 과신은 closed-book 수준에 머물러 정제 증거의 안전성 프로필에 도달하지 못했습니다. 검색 증거의 품질과 구조이 스케일링보다 중요합니다.

추가 분석

문맥 길이와 추론 연산의 한계

Max-context Prompting

  • 지연 시간만 증가
  • 안전성 격차 해소 불가
  • 더 많은 정보 노출 ≠ 더 나은 추론

긴 문맥이 정보 노출을 늘릴 뿐 추론 질을 보장하지 않음

Self-consistency

  • 5샘플 자기일관성, 제한적 개선
  • 추가 연산 대비 효과 미미
  • 고위험 오류 분포에 거의 영향 없음

샘플링 기반 안정화가 안전성 핵심 문제를 해결하지 못함

앙상블: 동기화된 실패 문제

3모델 앙상블은 집단 성능을 개선했으나, 모든 모델이 동일한 잘못된 옵션으로 수렴하는 동기화된 실패(synchronized failure)를 보존했습니다. 다수결이 안전성을 보장하지 않습니다.

최악 분석

임상적으로 중대한 오류는 소수 문항에 집중된다

최악 사례 분석(worst-case analysis) 결과, 임상적으로 중대한 오류가 전체 200문항 중 소수의 문항 부분집합에 집중되는 패턴이 확인되었습니다.

소수 집중

고위험 오류가 특정 문항에서 반복 발생

모델 간 동시 실패

여러 모델이 동일 문항에서 동일 오답 선택

평균에 숨음

평균 정확도가 이 패턴을 가리는 구조

"평균 정확도 80%라도, 나머지 20% 오류 중 고위험 항목의 비율이 실제 배포 위험을 결정한다. 평균 지표만으로는 안전성을 판단할 수 없다."

결론

임상 LLM 안전성은 스케일링의 수동적 결과가 아니다

증거 품질이 스케일보다 강력하다

임상의가 작성한 정제 증거가 모델 크기, 문맥 길이, 추론 연산보다 안전성 개선 효과가 큼

RAG는 자동으로 안전하지 않다

검색 기반 접근이 정확도를 높일 수 있으나, 고위험 오류와 과신 문제를 해결하지 못함

추론 연산 증가의 한계가 명확하다

자기일관성과 앙상블은 제한적 개선에 그치며, 동기화된 실패를 보존함

안전성은 직접 측정해야 한다

정확도 스케일링만으로 안전성을 추론할 수 없으며, 실제 배포 조건에서 안전성을 직접 평가해야 함
