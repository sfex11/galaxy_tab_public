SOL-ExecBench

실제 GPU 커널을 하드웨어 한계에 대비하여 평가하는
광속(Speed-of-Light) 벤치마크
NVIDIA Blackwell B200 GPU 대상
Edward Lin, Sahil Modi, S.K.S. Hari 외 30+ 저자 · 2026년 3월
문제 정의

왜 기존 벤치마크로는 부족한가?

"솔루션이 PyTorch 대비 10배 빠를 수 있지만, 여전히 하드웨어 SOL로부터 10배 이상 멀리 떨어져 있을 수 있다."

기존 접근법의 한계

  • 소프트웨어 베이스라인 대비 상대적 속도 향상만 측정
  • 베이스라인이 개선되면 점수가 변동됨
  • 최적화 여유 공간(headroom)을 파악 불가
  • 리워드 해킹에 취약

SOL-ExecBench의 접근법

  • 하드웨어 물리적 한계를 고정 목표로 설정
  • 하드웨어가 변하지 않으면 점수도 안정적
  • SOL까지의 절대적 거리를 정량화
  • 14가지 게이밍 방지 전략 적용

STAGE 3

SOL 분석기

루프라인 기반 성능 한계 계산. 온칩 버퍼 용량 고려한 정밀 한계 도출

SOL 시간 한계 (루프라인 모델)
TSOL = max( FLOPs총량 / 연산처리량 , 퓨즈드바이트총량 / 메모리대역폭 )
피크 연산 처리량과 메모리 대역폭 사용 · Orojenesis로 온칩 버퍼 제약 반영
평가 지표

SOL 점수: 하드웨어 한계까지의 거리

SOL 점수 (S ∈ [0, 1])
S(Tk) = (Tb − TSOL) / [ (Tk − TSOL) + (Tb − TSOL) ]
Tb = 베이스라인 실행시간  ·  TSOL = 하드웨어 한계 시간  ·  Tk = 후보 커널 실행시간
S → 0

Tk → ∞
성능 극히 저조

S = 0.5

Tk = Tb
베이스라인과 동등

S = 1.0

Tk = TSOL
하드웨어 한계 도달

핵심 장점

  • 속도향상과의 상관계수: SOL 점수 r = 0.981 vs 단순 Speedup r = 0.81 — 최적화 여유 공간을 훨씬 정확하게 반영
  • 안정적 목표: 베이스라인이 변해도 하드웨어 한계는 고정됨

SOL-ExecBench만의 차별점

  • 하드웨어 앵커: 소프트웨어가 아닌 물리적 한계 기준
  • 최신 정밀도: FP8, NVFP4 등 차세대 형식 지원
  • 학습 워크로드: 순전파 + 역전파 모두 포함
  • 실제 모델: 합성이 아닌 프로덕션 모델 기반

강건한 평가 체계

알려진 한계점

  • 텐서 값이 아닌 형상 기반 분석 (압축/상수 전파 미반영)
  • 전력 제한/열 쓰로틀링으로 SOL 한계가 느슨할 수 있음
  • CUDA 스트림 비활성화가 일부 커널에 불리
