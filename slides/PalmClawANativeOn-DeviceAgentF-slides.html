PalmClaw

모바일 폰을 위한 네이티브 온디바이스 에이전트 프레임워크

Hongru Cai, Yongqi Li, Ran Wei, Wenjie Li

홍콩 폴리테크닉 대학교 · 항저우 디아겐스 바이오테크놀로지

배경: LLM 에이전트의 발전

최근 LLM 에이전트는 단순한 응답 생성을 넘어, 도구를 호출하고 결과를 관찰하며 다음 행동을 반복적으로 결정하는 다단계 작업 수행으로 진화했습니다.

데스크톱/서버 에이전트

  • AutoGen, AgentScope, OpenClaw
  • 풍부한 도구 사용 및 자동화
  • 안정적인 실행 환경

모바일 기기의 잠재력

  • 전 세계 10세 이상 5명 중 4명 보유
  • 개인 데이터, 센서, 일상 앱
  • 사용자와 가장 가까운 환경

기존 모바일 에이전트의 한계

기존 모바일 에이전트는 주로 GUI 액션(탭, 스와이프, 타이핑)으로 스마트폰을 조작합니다.

1

긴 액션 시퀀스

고수준 작업이 레이아웃 변화에 취약한 긴 인터페이스 종속 시퀀스로 변환됨

2

디바이스 접근 불가

로컬 파일, 센서 데이터 등 디바이스 리소스와 기능에 직접 접근할 수 없음

3

실행 경계 없음

GUI 제어는 광범위한 접근을 부여하며, 작업별 접근 제한이나 실행 경계가 불명확

터미널 네이티브 에이전트(Codex CLI, Claude Code, Gemini CLI)의 등장은 명시적 도구 기반 상호작용으로의 전환을 보여줍니다.

PalmClaw: 접근 방식

모바일 기기에 에이전트 프레임워크 자체를 네이티브로 호스팅하고, 디바이스 기능을 명시적인 도구로 노출합니다.

기존: 외부 호스팅 모바일 에이전트

  • 데스크톱/클라우드에서 에이전트 실행
  • GUI 상태와 액션을 주고받음
  • 외부 장치에 의존적

PalmClaw: 네이티브 온디바이스

  • 모바일 기기에서 에이전트 루프 실행
  • 디바이스 도구로 기능 직접 호출
  • 명시적 인자, 구조화된 결과, 실행 경계

프레임워크 아키텍처

PalmClaw는 세션, 메모리, 스킬, 도구를 모바일 기기에서 직접 관리합니다.

1
입력
로컬 채팅, 원격 채널, 예약 웨이크업
2
세션 라우팅
기존/신규 세션에 입력 할당
3
컨텍스트 조립
메모리, 스킬, 도구 사양 결합
4
에이전트 루프
LLM 호출 → 도구 실행 → 관찰 반복
5
디바이스 도구 실행
모바일 서비스 호출 및 결과 반환

LLM 추론만 원격 API를 통해 제공되며, 컨텍스트 관리·도구 실행·세션 상태는 모바일 기기에 유지됩니다.

핵심 컴포넌트

세션 (Sessions)

3가지 입력 소스(로컬 채팅, 원격 채널, 예약 웨이크업)의 상호작용을 조직화. 대화 기록, 도구 추적, 첨부파일, 전용 워크스페이스를 저장하며 동시 작업을 분리 관리합니다.

메모리 (Memory)

두 계층 구조: ① 세션 간 공유되는 장기 메모리 ② 세션별 요약. 설정 가능한 윈도우 도달 시 별도 LLM 호출로 이전 메시지를 요약하고 양쪽 계층을 갱신합니다.

스킬 (Skills)

SKILL.md 파일에 저장된 재사용 가능한 작업 지침. 항상 활성 스킬은 전체 로드되고, 나머지는 사용자 메시지와 매칭하여 선택적 로드 — 컨텍스트 과부하를 방지합니다.

디바이스 도구 (Tools)

모델 대면 사양(이름, 설명, JSON 인자 스키마)과 디바이스 대면 함수(모바일 서비스 호출)를 연결. 구조화된 결과 또는 에러를 반환합니다.

디바이스 도구 카테고리

PalmClaw의 내장 도구 그룹이 모바일 기기의 다양한 기능을 체계적으로 노출합니다.

디바이스

상태 확인, 권한 관리, 블루투스

미디어

카메라, 마이크, 사진/동영상 접근

캘린더

일정 조회, 생성, 수정, 삭제

연락처

연락처 조회 및 관리

워크스페이스

파일 읽기/쓰기, 문서 관리

웹

웹 검색, 콘텐츠 가져오기

통신

메시지 전송, 알림 관리

자동화

예약 작업, 스케줄링

메모리

장기 메모리 읽기/쓰기, 세션 검색

디바이스 도구의 핵심 설계 원칙

GUI 액션의 한계를 극복하기 위해 세 가지 핵심 원칙을 따릅니다.

명시적 인자 (Explicit Arguments)

JSON 스키마로 정의된 구조화된 인자 — 모호한 탭/스와이프 위치가 아닌 의미 있는 파라미터로 디바이스 기능 호출

구조화된 결과 (Structured Results)

스크린샷 해석이 아닌 프로그램적 반환값 — 에이전트가 결과를 정확히 파싱하고 후속 행동을 결정할 수 있음

도구별 실행 경계 (Execution Boundaries)

각 도구가 정의하는 접근 범위와 권한 — GUI 제어의 광범위한 접근과 달리 작업에 필요한 최소 권한만 부여

실행 경계의 예시

실행 경계는 실제 시나리오에서 어떻게 적용될까요?

사례 1 캘린더 권한

에이전트가 캘린더 도구를 호출하면, 해당 도구의 실행 경계 내에서만 캘린더 데이터에 접근합니다. 연락처나 메시지 등 다른 데이터에는 자동으로 접근할 수 없으며, 사용자 동의 없이 권한이 확장되지 않습니다.

사례 2 워크스페이스 경계

각 세션은 전용 워크스페이스를 가집니다. 파일 도구는 해당 세션의 워크스페이스 내에서만 파일을 읽고 쓰며, 다른 세션의 파일이나 시스템 전체 파일에 접근할 수 없습니다.

GUI 제어는 도달 가능한 모든 인터페이스에 광범위한 접근을 부여하지만, 디바이스 도구는 각 호출이 명시적이고 제어된 범위 내에서만 작동합니다.

평가 설정 (RQ1–RQ3)

데이터셋

  • MobileTask — 모바일 작업 벤치마크
  • AssistantBench — 어시스턴트 벤치마크

베이스라인

  • GUI 기반 모바일 에이전트 시스템들
  • 가장 강력한 베이스라인과 비교

RQ1

작업 완료 성능은 어떤가?

RQ2

배포 및 운영 부담은 어떤가?

RQ3

실행 경계는 어떻게 적용되는가?

주요 결과 (RQ1 & RQ2)

+11.5%

작업 성공률 향상

가장 강력한 베이스라인 대비 상대적 성공률 11.5% 개선. 디바이스 도구의 명시적 인자와 구조화된 결과가 정확한 작업 수행에 기여합니다.

−94.9%

완료 시간 감소

평균 작업 완료 시간 94.9% 단축. GUI 액션의 긴 시퀀스를 단일 도구 호출로 대체하여 극적인 효율성 향상을 달성합니다.

배포 및 운영 (RQ2)

외부 데스크톱/서버 설정 없이 모바일 앱 설치만으로 배포 가능. 컨텍스트 조립, 도구 실행, 세션 관리가 기기 내에서 완료되어 설정 부담이 크게 낮아집니다.

결론 및 기여

PalmClaw는 모바일 에이전트의 새로운 패러다임을 제시합니다.

1

모바일 네이티브 에이전트 프레임워크

에이전트 루프, 메모리, 스킬, 도구, 세션 상태를 모바일 기기에서 직접 호스팅하는 오픈소스 프레임워크

2

디바이스 도구 설계

명시적 인자, 구조화된 결과, 도구별 실행 경계를 통해 모바일 기능을 안전하고 효율적으로 노출

3

실증적 성능 향상

성공률 11.5% 향상, 완료 시간 94.9% 단축, 낮은 설정 부담, 실행 경계 트레이스 분석 제공
