문제 정의

RAG 시스템의 근본적 한계

밀도 검색의 문제

  • 의미적 유사도 ≠ 생성 유틸리티
  • 높은 주제적 겹침이 오히려 의미적 방해물 유발
  • 롱컨텍스트에서 디코딩 불확실성 증가

LLM 유틸리티의 문제

  • 테스트 시 LLM 추론 → 계산 비용 폭발
  • Perplexity 신호의 극심한 노이즈
  • 토큰 수준 변동에 과민하게 반응
"의미적으로 유사한 문서가 실제로는 답변에 필요한 정보를 제공하지 못하고, 생성기를 오도할 수 있다."
핵심 통찰

생성 유틸리티를 임베딩 공간에 직접 증류하자

생성 유틸리티는 바이인코더의 임베딩 공간 내로 증류되어야 한다. 비싼 테스트 시 LLM 추론을 우회하면서 생성기의 실제 선호를 포착한다.
정렬을 분포 매칭 문제로 공식화

노이즈 많은 유틸리티 점수를 회귀 목표로 사용하지 않고, 유틸리티 유도 분포를 모방하도록 훈련

추론 시 LLM 없이 ANN 검색 가능

표준 근사 최근접 이웃(ANN) 검색과 완전 호환되는 실용적 솔루션

UAE 프레임워크

Utility-Aligned Embeddings 전체 구조

A
보상 모델 학습
노이즈 많은 유틸리티 신호를 파라미터화된 보상 모델 R_ψ로 증류
B
분포 매칭 정렬
보상 모델이 정의한 목표 유틸리티 분포와 바이인코더 분포의 KL 발산 최소화
C
추론 (ANN)
훈련된 바이인코더로 표준 ANN 검색 — LLM·보상 모델 불필요

핵심 목표: Utility-Modulated InfoNCE

표준 InfoNCE가 모든 부정 예제를 동등하게 무관하게 취급하는 것과 달리, UAE의 목표 함수는 유틸리티 공간의 순서적(ordinal) 구조를 보존하여 검색기가 관련성의 정도를 구별할 수 있도록 한다.

Stage A

파라미터화된 유틸리티 근사

유틸리티 정의 (Perplexity 기반)

U(q, d, a) = exp( (1/|a|) Σ log p_θ(x_t | x_<t, q, d) )
문서 d가 질문 q에 대한 정답 a를 더 예측 가능하게 만들수록 유틸리티가 높다.

원시 유틸리티의 불안정성 원인

1

정답의 언어적
표현 다양성

2

행렬 병렬화 및
부동소수점 오차

3

답변 길이 |a|에 대한
과민성

Stage A

보상 모델 학습: 순서 보존 접근

왜 회귀가 아닌 순서 학습인가?

원시 유틸리티의 분포 불안정성(heavy-tailed, high variance)으로 인해 MSE/Huber 같은 회귀 손실은 학습 불안정과 과적합을 유발한다. 대신 후보 간 상대적 순서만 보존하는 것이 훨씬 강건하다.

Margin-based Ranking Loss

ℒ_Reward = max(0, δ − (R_ψ(q, d_i) − R_ψ(q, d_j)))

U(q, d_i, 𝒜_q) > U(q, d_j, 𝒜_q)일 때
R_ψ(q, d_i) > R_ψ(q, d_j) + δ를 강제

  • R_ψ는 Transformer 기반 (BERT) 크로스인코더
  • 정확한 유틸리티 값이 아닌 생성기의 서수적 선호만 보존
  • "의미적 하드 네거티브"를 효과적으로 식별
Stage B

보상 기반 임베딩 훈련

크로스인코더인 R_ψ는 대규모 추론에 비용이 높으므로, 이를 오프라인 교사로 사용하여 바이인코더에 유틸리티 선호를 증류한다.

교사 분포 π*

π*(d_i|q) = exp(λ·R_ψ(q,d_i)) / Σ exp(λ·R_ψ(q,d_j))

λ: 유틸리티 분포의 날카로움 제어 온도

학생 분포 π_φ

π_φ(d_i|q) = exp(⟨f_φ(q), f_φ(d_i)⟩/τ) / Σ exp(⟨f_φ(q), f_φ(d_j)⟩/τ)

τ: 검색기 스케일링 온도

UAE 목표 함수 (KL 발산 최소화)

ℒ_UAE(φ) = 𝔼_q [ D_KL(π* ∥ π_φ) ] ∝ −Σ Σ π*(d_i|q) log π_φ(d_i|q)

Stage B

유틸리티 인식 하드 네거티브 마이닝

왜 표준 하드 네거티브가 부족한가?

랜덤 네거티브는 trivial gradient를, 표준 의미적 하드 네거티브는 생성기가 실제로 유용하다고 판단한 문서를 포함할 수 있다.

Utility-Gated Mining: 이중 조건 필터

1
의미적 유사도 조건

기본 유사도 함수 S_sem으로 top-k 내에 랭킹

2
유틸리티 마진 조건

R_ψ(q, d⁺) − R_ψ(q, d⁻) > δ 보장

"보수적 마진 δ는 검색기가 의미적 방해물(semantic distractors) 해결에 집중하도록 한다 — 임베딩 공간에서 근접하지만 유틸리티가 무시할 수준인 문서들."
왜 RL이 아닌 모방 학습인가?

설계 선택의 당위성

강화 학습 (RL)의 한계

  • 거대한 이산 행동 공간 (전체 코퍼스)
  • 높은 보상 분산으로 학습 불안정
  • 탐색의 극단적 계산 비용

모방 학습 (Imitation)의 장점

  • 정규화된 보상 분포를 전문가 정책으로 사용
  • 지도 그래디언트로 분산 발산 최소화
  • 탐색 없이 안정적이고 효율적인 학습

UAE는 검색기 π_φ를 훈련하여 전문가 유틸리티 분포 π*와의 발산을 최소화함으로써, RL의 탐색 도전을 우회하면서도 생성기의 선호 구조를 임베딩 공간에 직접 주입한다.
