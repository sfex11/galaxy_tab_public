Data Journalist Agent

원시 데이터를 검증 가능한 멀티모달 스토리로 변환하는 에이전트 프레임워크

Kevin Qinghong Lin, Batu EI, Yuhong Shi, Pan Lu, Philip Torr, James Zou

University of Oxford · Stanford University

데이터 저널리즘의 문제

시간 비용

고품질 뉴스 기사 하나를 완성하는 데 수주가 소요됩니다. 맥락 수집, 통계 분석, 각도 선정, 시각 설계 등 다학제적 역량이 동시에 필요합니다.

검증 부재

기존 AI 저널리즘 시스템은 숫자의 출처, 차트의 정확성, 주장의 근거를 독자가 확인할 방법이 없습니다. 언어 모델의 환각 문제가 이를 더욱 악화시킵니다.

"에이전트가 원시 데이터를 받아 독자가 끝까지 읽고 신뢰할 수 있는 스토리를 만들 수 있는가?"

선행 연구와의 비교

시스템 외부 검색 서사 각도 멀티모달 근거 추적
Deep Search Agents 부분
데이터 시각화 에이전트 이미지만 코드만
AI Scientist 부분
인간 저널리스트 비구조화
Data2Story 전체

Data2Story는 검색, 서사, 멀티모달 생성, 근거 추적을 모두 갖춘 유일한 시스템

Data2Story 개요

원시 데이터(CSV 등)를 입력받아 검증 가능한 멀티모달 기사(웹사이트)를 자동 생성하는 다중 에이전트 프레임워크

근거 기반 주장

기사 내 숫자, 인용, 시각 자산을 실행 가능한 코드, 데이터 소스, 외부 URL로 추적 가능하게 만들어 기사 전체가 감사 가능하도록 설계

멀티모달 생성

단순 텍스트·정적 차트가 아닌, 인터랙티브 맵, 오디오 클립, 비디오, 플레이어블 데모 등 주제와 독자에 맞는 형태를 추론하여 생성

예: 카드 게임 데이터 → 인터랙티브 카드 뽑기 데모 포함 기사 생성 (Figure 1)

가상 뉴스룸의 7개 역할

1

탐정 Detective

맥락 수집, 배경 조사, 웹 검색

2

분석가 Analyst

통계 실행, 데이터 프로파일링, 코드 작성

3

편집자 Editor

서사 프레이밍, 각도 선정, 기사 구조화

4

디자이너 Designer

시각 자산 기획, 멀티모달 형태 결정

5

프로그래머 Programmer

웹사이트 구현, 인터랙티브 요소 코딩

6

감사관 Auditor

프로그래머 출력 검토, 수정 제안

★ 검사관 Inspector

최종 기사의 모든 요소를 상류 근거(코드, 데이터, URL)에 연결 — Data2Story만의 핵심 혁신

Inspector: 검증 가능성의 보장

기사의 각 주장이 "어디서 왔는지"를 명시적으로 추적하는 전담 에이전트

1
주장 추출
기사 내 모든 숫자, 인용, 시각 자산 식별
2
근거 연결
각 요소를 코드 줄, 데이터 소스, 또는 외부 URL에 매핑
3
검증 실행
코드 재실행 및 출처 확인으로 주장 독립 검증

근거의 3가지 유형

  • 실행 가능한 코드 (재현 가능)
  • 데이터 소스 (출처 명시)
  • 외부 URL (참조 문헌)

핵심 차별점

기존 시스템은 사실성(factuality)만 목표로 하나, Data2Story는 감사 가능성(auditability)을 목표로 — 독자와 편집자가 직접 근거를 검증할 수 있는 구조

멀티모달 생성 전략

Designer가 주제와 독자를 고려하여 최적의 미디어 형태를 추론

지리 기사

줌 가능한 인터랙티브 맵

예: 2026 월드컵 경기장 지리 + 날씨 위험도 시각화

음악 기사

오디오 클립 임베딩

예: 세대별 팝 가수 발성 변화 분석에 음원 샘플 첨부

게임 기사

플레이어블 데모 직접 체험

예: 카드 선택 데이터 기사에 카드 뽑기 인터랙션 포함

분석 기사

인터랙티브 차트 + 스크롤 애니메이션

예: 추세 분석에 호버/필터 기능이 있는 차트

미탐구 데이터에서의 발견

아직 인간이 기사로 작성하지 않은 최신 데이터셋에서 Data2Story가 독자적 발견을 수행

2026 월드컵 일정 데이터

  • 경기장 지리 위치와 기상 데이터를 교차 분석
  • 인터랙티브 맵으로 고온 위험 경기 시각적 하이라이트
  • 기존 보도에서 다루지 않은 관점 자동 발견

Data2Story는 뉴스룸이 대역폭 부족으로 다루지 못하는 데이터 밀집형 스토리를 발굴하는 데 특히 유용합니다.

평가 설계

3개 출처의 18개 데이터셋, 각각 전문가 작성 기사와 페어링

인간-에이전트 각도 커버리지

양측 기사에서 사실적 주장을 추출하고 유사도 매칭으로 각도 중복·보완 분석

인간 평가 (53명)

시각 디자인, 서사 속도, 데이터 투명성, 주장-데이터 정합성, 인사이트 가치 — 블라인드 테스트

컴퓨터 사용 에이전트 판정

클릭, 스크롤 등 실제 인터페이스 상호작용으로 저비용 자동 평가 대체

검증 가능성 분석

교차 패밀리 코딩 에이전트가 코드 재실행 및 참조 검색으로 각 주장 독립 검증

인간 평가 결과

53명 참가자가 블라인드로 에이전트/인간 기사를 5개 차원에서 평가

Data2Story가 우세한 영역

  • 데이터 투명성 — Inspector의 근거 추적이 결정적 차이
  • 주장-데이터 정합성 — 코드 기반이 신뢰도 향상

인간이 우세한 영역

  • 편집 각도 — 창의적 시각과 독창적 프레이밍
  • 시각 디자인 — 예술적 감각과 정보 전달의 균형
  • 서사 속도 — 독자를 끌어당기는 리듬감

분석 장르에서는 에이전트 우위가 확대되지만, 에디토리얼 스크롤텔링에서는 인간 우위가 좁혀지지 않음

컴퓨터 사용 에이전트 판정

실제 사용자의 탐색 행위를 시뮬레이션하는 저비용 자동 평가 방법

작동 방식

에이전트가 렌더링된 웹페이지를 클릭, 스크롤, 읽기 등의 행위로 탐색하며 품질을 평가. 인간 패널의 순위를 보존하면서 비용은 분의 일 수준

투명성 차이가 가장 큼

투명성(Transparency) 차원에서 Data2Story의 점수 상승이 다음 차원의 약 2.5배 — Inspector의 근거 추적이 압도적 차별점으로 작용

컴퓨터 사용 에이전트 판정은 인간 평가의 비용 효율적 대체재로서 유망하며, 특히 대규모 배포 시나리오에서 실용적 가치가 높습니다.

검증 가능성 분석: 사실성이 아닌 감사 가능성

전통적 접근: 사실성 (Factuality)

주장이 "맞는지 틀린지" 이진 판정. 근거를 보여주지 않으면 독자가 직접 확인 불가. 인간 기사도 사실성 보장이 어려움.

Data2Story: 감사 가능성 (Auditability)

각 주장에 대해 "어떤 코드로 도출되었는지", "어떤 데이터에서 왔는지"를 명시적으로 제공. 독자와 편집자가 직접 재검증 가능.

검증 프로세스

1
주장 추출
기사 내 모든 수치적 주장 식별
2
코드 재실행
교차 패밀리 에이전트가 코드 독립 실행
3
근거 확인
참조 URL 검색 및 데이터 소스 교차 검증

인간이 더 나은 영역

정성 평가를 통해 확인된 인간 저널리스트의 강점

편집 각도

데이터에서 의미 있는 각도를 찾아내는 직관적 감각. 에이전트는 통계적 발견에는 강하지만, 독자에게 왜 중요한지를 설득하는 각도 선정에서 부족

창의적 디자인

예술적 감각과 정보 디자인의 균형. 에이전트는 기능적 시각화는 잘하지만, 감성적 호소와 미적 완성도에서 열위

정보 전달

복잡한 맥락을 일반 독자가 이해하기 쉽게 전달하는 능력. 서사 리듬과 호흡 조절이 인간의 핵심 역량

"Data2Story는 저널리스트를 대체하는 것이 아닙니다. 인간이 관점과 편집적 판단을 설정하고, 에이전트가 노동 집약적 계산과 그래픽 설계를 담당하는 협업 모델입니다."

핵심 요약

기여

  • 7개 전문 역할을 오케스트레이션하는 가상 뉴스룸 아키텍처
  • Inspector를 통한 주장 수준 근거 추적 — 감사 가능성 보장
  • 주제·독자에 맞는 멀티모달 형태 추론 및 자동 생성
  • 4축 평가 프레임워크 (각도 커버리지, 인간 평가, 에이전트 판정, 검증 가능성)
  • 미탐구 데이터에서의 자발적 발견 능력 입증

핵심 발견

  • 투명성·감사 가능성에서 압도적 우위 (Inspector 효과 ≈ 2.5×)
  • 분석 장르에서 경쟁력 입증
  • 편집 각도·창의적 디자인에서는 인간 우위 지속
  • 컴퓨터 사용 에이전트가 인간 순위를 저비용으로 보존
