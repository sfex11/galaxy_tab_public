AutoMat: 설계 개요

도메인 전문가(SME)가 큐레이션한 85개의 계산 재료과학 클레임을 기반으로, LLM 에이전트가 종단 간(end-to-end) 재현 능력을 평가받는 벤치마크

1
클레임 수집
실제 논문에서 도메인 전문가가 주요 클레임 선정 및 주석 달기
2
태스크 패키징
클레임 + 논문 + 메타데이터 + (선택) 아티팩트를 실행 가능한 태스크로 구성
3
에이전트 실행
코딩 에이전트가 HPC 환경에서 워크플로우 복구·실행
4
평가
별도 평가자 에이전트가 산출물 검사 후 재현 가능성 판정

RQ3

주요 실패 모드 분석

절차적 불완전성 (Procedural Incompleteness)

필요한 계산 단계를 누락하거나 불충분하게 구현. 논문에 암시된 절차를 완전히 복구하지 못함.

방법론적 편차 (Methodological Deviation)

올바른 방법론 대신 근사치나 대체 방식을 사용하여 과학적으로 부정확한 결과 생성.

자원·실행 실패 (Resource & Execution Failure)

환경 설정 오류, 의존성 문제, 메모리 부족 등으로 워크플로우 실행 자체가 실패.

과신 판단 (Overconfident Assessment)

잘못된 결과를 올바르게 재현되었다고 자기 평가하는 현상.

실패 모드 심층

추가 실패 모드와 순환 추론 문제

순환 추론 (Circular Reasoning)

"Begging the Question" — 에이전트가 클레임을 전제로 삼아 그 클레임을 지지하는 것처럼 보이는 결과를 생성. 논리적 순환에 빠짐.

과소 훈련/단순화 (Under-training)

복잡한 도메인 절차를 과도하게 단순화하여 구현. 전문 도구의 세부 설정이나 미묘한 차이를 무시.

특히 과신 판단과 순환 추론은 에이전트가 "성공했다"고 보고하지만 실제로는 과학적으로 무의미한 결과를 낳는 가장 위험한 실패 모드다.

결론

결론 및 향후 방향

현재 LLM 기반 코딩 에이전트는 계산 재료과학의 연구 결과를 자율적으로 재현하는 데 아직 부족하다. 소프트웨어 공학에서의 능력이 과학적 워크플로우로 자연스럽게 전이되지 않으며, 절차 복구와 도메인 특화 실행이 주요 병목이다.

향후 과제

  • 도메인 특화 사전 훈련 강화
  • 과신 판단 완화 메커니즘
  • 순환 추론 탐지 기법
  • 더 많은 과학 도메인으로 확장

AutoMat의 역할

  • 계산 과학 재현성 벤치마크
  • 에이전트 시스템 한계 진단 도구
  • AI-for-Science 연구 방향 지표
