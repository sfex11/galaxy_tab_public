태스크 개요

문제 정의

소셜 미디어 텍스트가 극단화(polarized)인지 아닌지를 이진 분류하는 태스크

  • 고정관념, 비인간화, 불관용 포함
  • 평가 지표: Macro F1-score
주요 도전 과제
  • 22개 언어의 다양한 타입학적 특성
  • 언어별 데이터 크기 편차 심함
  • 문화별 극단화 표현 방식 차이
  • 심각한 클래스 불균형 문제
대상 언어: 암하라어, 아랍어, 벵골어, 버마어, 중국어, 영어, 독일어, 하우사어, 힌디어, 이탈리아어, 크메르어, 네팔어, 오리야어, 페르시아어, 폴란드어, 펀자브어, 러시아어, 스페인어, 스와힐리어, 텔루구어, 터키어, 우르두어

임계값 튜닝

핵심 아이디어

기본 임계값 t=0.5 대신, 언어별로 개발셋 macro-F1을 최대화하는 최적 임계값 탐색

탐색 범위: t ∈ {0.30, 0.35, 0.40, …, 0.70}
과소신뢰 사례
러시아어: 평균 예측 확률 0.246
→ 낮은 임계값 필요
과신뢰 사례
크메르어: 평균 예측 확률 0.919
→ 높은 임계값 필요
최적 임계값 범위: 네팔어 0.30 ~ 크메르어·암하라어·벵골어 0.70 → 모델이 기본 임계값에서 잘 보정되지 않음

앙상블 전략

언어별로 개발셋에서 최고 성능의 전략을 자동 선택

핵심 시사점

성공 요인
  • 균형잡힌 다국어 사전학습 분포 (Gemma 3)
  • 언어별 임계값 튜닝만으로 +2~4% F1
  • 합성 데이터 증강 시 철저한 품질 필터링
  • 재훈련 없는 사후 최적화 (임계값 + 앙상블)
남은 과제
  • 모델 확률 보정(miscalibration) 문제
  • 특정 언어에서 합성 데이터 과도 의존 시 성능 저하
  • 이탈리아어 등 일부 언어에서 여전히 낮은 F1
  • Dev-Test 간격으로 인한 전이 불안정성
"개발셋 성능 극대화가 아닌 견고한 일반화(robust generalization)이 성공의 핵심 요인"

결론

1
Gemma 3 (12B + 27B)의 LoRA 미세조정이 22개 언어 극단화 탐지에서 강력한 일반화 성능 입증
2
3중 합성 데이터 전략(직접 생성 + 패러프레이징 + 대조 쌍)과 다단계 품질 필터링의 효과 확인
3
언어별 임계값 튜닝이 재훈련 없이 +2~4% F1 향상, 앙상블로 14/22 언어에서 추가 개선
4
최종 시스템: Mean Macro-F1 0.811, 60팀 중 2위, 3개 언어 1위
주요 교훈

사전학습 분포의 균형성과 일반화 능력이 다국어 태스크에서 순수 개발셋 성능보다 중요하며, 합성 데이터 증강은 저자원 언어에 집중적으로 활용해야 함
