메타기억: 인지과학에서의 영감

인간의 메타기억 (Metamemory)

  • 무엇을 기억할지 결정
  • 언제 검색할지 판단
  • 지식을 어떻게 조직화할지 설계
  • 연습을 통해 발달하는 학습된 기술
"외부 보조수단(메모, 색인, 파일)의 숙련된 사용은 작업기억을 넘어 인지를 확장하는 핵심 방법이다." — Clark & Chalmers, 1998

LLM의 유사한 병목

  • 컨텍스트 윈도우 = 작업기억
  • 장기 호라이즌 작업에서 용량 초과
  • 기존 접근: 메모리를 고정된 아키텍처 모듈로 처리
  • 우리의 관점: 메모리 관리는 능동적·학습 가능한 기술

핵심 아이디어: 파일 시스템을 메모리로

파일 시스템 연산을 1급 메모리 액션으로 승격시켜, 작업 액션과 동일한 액션 공간에 배치

메모리 액션

  • CREATE 새 파일 생성
  • WRITE 파일 내용 덮어쓰기
  • APPEND 파일 끝에 추가
  • SEARCH 키워드로 검색
  • READ 파일 내용 읽기

설계 원칙

  • 모델이 무엇을 저장할지 스스로 결정
  • 모델이 언제 검색할지 스스로 결정
  • 모델이 어떻게 조직할지 스스로 결정
  • 모든 메모리 결정이 궤적에서 추적 가능

단일 forward pass에서 작업 액션과 메모리 액션을 동시에 선택

AutoMem 프레임워크 개요

메모리 기술은 두 축에서 향상 — 구조(Structure)와 숙련도(Proficiency)

1
Inner-loop 에이전트
파일 시스템 메모리를 장착한 에이전트가 에피소드 실행
2
Outer-loop #1
메타-LLM이 궤적 검토 후 스캐폴드(코드·프롬프트·스키마) 반복 개정
3
Outer-loop #2
좋은 메모리 결정을 학습 신호로 활용해 메모리 전문 모델 파인튜닝
공통 원리: 장기 호라이즌 작업 개선은 궤적 수준 리뷰 → 대상修订으로 분해 가능하며, 이를 메타-LLM이 자율적으로 실행

Inner-loop 에이전트: LOG와 PLAN

매 스텝마다 두 루틴을 순차적으로 실행하여 메모리 관리의 양면을 다룸

LOG 루틴

"방금 일어난 일 중 무엇을 기록할까?"

  • 환경 응답을 기록할지 여부 판단
  • 기존 파일에 APPEND 또는 새 파일 CREATE
  • 필요시 기존 엔트리 재작성(WRITE)

PLAN 루틴

"지금 행동하기 위해 무엇을 떠올려야 할까?"

  • 파일 간 SEARCH로 관련 정보 탐색
  • 특정 파일 또는 파일 끝부분 READ
  • 다음 세계 액션(commit) 실행

통합 액션 공간: LOG 액션 + PLAN 액션 + 세계 액션이 하나의 forward pass에서 결정됨

Outer-loop #1: 메모리 스캐폴드 최적화

메타-LLM이 코드修订을 통해 에이전트의 메모리 구조를 점진적으로 개선

1
에피소드 수집 — 현재 스캐폴드로 다수 에피소드 실행, 완전 궤적 기록
2
실패 패턴 진단 — 메타-LLM이 전체 궤적을 읽고 메모리 사용의 결함 식별
3
스캐폴드 수정 — 프롬프트, 파일 스키마, 검증 로직, 액션 어휘를 코드 단위로 개정
4
반복 — Crafter v0→v5, MiniHack v0→v4, NetHack v0→v2까지 반복 진화
모델 가중치는 변경하지 않음 — 구조적 개선만으로도 대규모 성능 향상 달성

Outer-loop #2: 메모리 숙련도 학습

스캐폴드의 한계를 넘어, 모델 자체의 메모리 판단 능력을 직접 향상

데이터 큐레이션

  • 다수 에피소드에서 메모리 결정 추출
  • 메타-LLM이 강화할 가치가 있는 결정 선별
  • 지도 학습용 훈련 데이터 구성

메모리 전문가 학습

  • 메모리 전문 모델만 파인튜닝
  • 작업 모델은 동결 — 기존 작업 능력 보존
  • 메타-LLM이 학습 설정(하이퍼파라미터 등)도 자동 결정

핵심: 메모리를 분리 가능한 기술로 취급하여, 작업 능력에 위험 없이 메모리 숙련도만 집중 훈련

실험 설정: 3개의 장기 호라이즌 게임

Crafter

오픈월드 생존 게임

  • 제작·전투·자원 관리
  • 스캐폴드 v0 → v5

MiniHack

퍼즐·탐색·전투

  • NetHack 엔진 기반 집중 과제
  • 스캐폴드 v0 → v4

NetHack

가장 복잡한 로그라이크

  • 10⁴–10⁵ 턴, 인간 수년 수준
  • 스캐폴드 v0 → v2

기본 모델: Qwen2.5-32B-Instruct · 메타-LLM: Claude Opus 4.5 · 절차적 생성으로 사전학습 지식 전이 최소화

주요 결과: 메모리 최적화만으로 ~2×–4× 향상

핵심 발견들

메모리는 독립적인 고레버리지 축
작업 액션 행동을 수정하지 않고 메모리만 최적화해도 대규모 향상
두 축의 상호보완성
스캐폴드 최적화의 천장 위로 숙련도 학습이 추가 향상을 제공
공유 액션 공간에서의 동시 개선
작업 결정과 메모리 연산이 통합 공간에서 함께 향상
"쓰기 전에 확인" 메모리 규율의 내재화
학습된 모델이 기록 전 관련 정보를 자발적으로 검색하는 패턴 형성

스캐폴드 진화의 예시

Crafter 환경에서 메타-LLM이 반복적으로 발견하고 수정한 패턴

문제 진단된 결함

  • 단일 파일에 모든 정보 덤프
  • 중복 기록으로 파일 팽창
  • 검색 키워드 불일치
  • 중요 정보 누락

해결 진화된 스캐폴드

  • 목적별 파일 분리 (지도, 인벤토리, 전략)
  • 중복 검증 로직 추가
  • 표준화된 검색 키워드 가이드
  • 필수 기록 항목 명시

v0 → v5까지 5번의 반복修订을 거치며 점진적으로 구조가 정교해짐
