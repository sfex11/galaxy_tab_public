SkillCenter: 대규모 소스 기반
자율 AI 에이전트용 스킬 라이브러리

A Large-Scale Source-Grounded Skill Library for Autonomous AI Agents

Tianming Sha (Stony Brook) · Yue Zhao (USC) · Lichao Sun (Lehigh) · Yushun Dong (FSU)

바이브 코딩과 자율성 스펙트럼

"개발자가 자연어로 의도를 전달하고, LLM이 코드를 작성하며, 개발자가 줄별 검토 없이 결과를 수용한다."

— Andrej Karpathy, "Vibe Coding" (2025)

자율성의 4단계

1
인간 전용
모든 코드와 실험을 직접 수행
2
LLM 보조
제안만 제공, 인간이 최종 승인
3
에이전트 실행
최소 개입으로 다단계 출력 생성
4
에이전트 자율
목표 설정만, 예외 처리만 인간

핵심 tension

자율성이 높아질수록 에이전트는 기반 지식의 품질에 더 의존하지만, 정작 그 자율성이 인간을 지식이 적용되던 자리에서 밀어냄

실행 성공 ≠ 실질적 정확성

코드가 "돌아간다"는 것은 품질의 약한 신호에 불과합니다.

숨겨진 결함

메모리 누수, 폐기된 API 사용, 엣지 케이스 무시

부적절한 방법

유효하지만 비최적 통계 검정, 다중 비교 미보정

보안 취약점

기존 테스트를 통과하지만 보안 허점 내재

기존 대응: Human-in-the-loop (확장 불가) vs RAG (원천 노이즈, 해석 부담 여전)

스킬이 필요한 이유

스킬 = 구조화된, 검색 가능한, 소스 기반의 운영 지식 단위

특성 RAG 청크 스킬 (Skill)
구조 임의의 부분 문자열 정의된 스키마 (제목, 설명, 적용조건, 증거, 점수)
품질 게이트 원천 품질 직접 상속 LLM 기반 SkillGate가 명확성·정확성·실행가능성 독립 평가
실행가능성 에이전트가 직접 해석·운용 필요 에이전트가 작업 중 직접 적용 가능
점수화 관련성 순위 품질 점수로 신뢰도 우선순위화

기존 스킬 라이브러리: 수동 작성 · 수백~수천 규모 · 단일 도메인 한정 → 확장 불가

SkillGate 파이프라인 생성

학술 원천

  • ArXiv 프리프린트
  • paper.experiment / experiment_design 분류

기술·커뮤니티 원천

  • 24,000+ 기술 문서/웹 소스
  • GitHub 리포지토리 (102,373 스킬)
  • ClawHub 마켓플레이스 통합

모든 스킬은 오프라인 검색 가능한 SQLite FTS5 번들로 배포

수집 프레임워크: 엔드-투-엔드 파이프라인

1
다원천 획득
저널, ArXiv, GitHub, 기술 문서에서 자동 수집
2
SkillGate 품질 게이트
LLM이 명확성·정확성·실행가능성 평가 (1~5점)
3
템플릿 기반 생성
정의된 스키마에 따라 구조화된 스킬 마크다운 생성
4
소스 기반 검증
각 주장이 원문의 정확한 인용과 매핑되는지 반복 확인
5
품질 통제 배포
중복 제거, 라이선스·안전 감사 후 SQLite FTS5 번들로 출판

Case A 통과 → 점수 5 스킬로 직접 생성

Case B 실패 위험 → 점수 2로 낮춰 보존 (에이전트가 판단)

SkillGate: 품질 게이트 상세

평가 기준 (3차원)

  • 명확성 — 모호성 없이 에이전트가 이해 가능한가
  • 정확성 — 원천과 일치하며 사실적으로 올바른가
  • 실행가능성 — 에이전트가 작업 중 직접 적용할 수 있는가

점수 분포 해석

점수의미
5높은 품질, 즉시 적용 가능
3~4보통, 일부 맥락에서 유용
1~2낮음, 참고용 또는 폐기 후보

SkillGate 동작 원리

입력: 원천 텍스트 추출물

처리: LLM이 3차원 루브릭에 따라 1~5점 평가

출력: 점수 + 통과/보존/폐기 결정

내부 QA 신호 — 외부 검증이 아닌 프레임워크 내 자체 품질 지표

소스 기반성(Source Grounding)

소스 기반성은 추적성 보장이다: 유지된 각 주장은 원천의 정확한 인용문에 매핑된다.

반복 검증 루프

1
스킬 초안 생성
2
각 주장을 원천 인용문과 대조
3
불일치 발견 시 수정 후 재검증
4
모든 주장이 매핑되면 출판 게이트 통과

스킬 스키마 구조

title — 스킬 제목

description — 설명 및 적용 조건

evidence — 원천 인용문 (추적성)

score — 품질 점수 (1~5)

domain / bundle — 분류 메타데이터

하류 평가: 스킬이 언제 도움이 되는가?

솔버 능력 내 과제

에이전트가 이미 아는 영역의 작업

→ 스킬 보유 여부의 효과가 제한적

솔버 지식 초과 과제

에이전트의 사전 지식으로 해결 불가한 작업

→ 실제 코퍼스 스킬이 지식 격차를 실질적으로 좁힘

핵심 발견

스킬의 가치는 에이전트가 모르는 것을 채울 때 극대화된다. 능력 내 과제에서는 RAG와 유사하지만, 지식 초과 과제에서는 실제 코퍼스 스킬이 합성 스킬이나 무 스킬 조건보다 유의미하게 우수한 성능을 보임.

코퍼스 품질 및 관리

중복 제거

동일 도메인 내 의미적 중복 스킬을 식별하고 제거. 부록 A에 실제 중복 예시 수록

라이선스 감사

각 번들의 출처 라이선스를 감사하여 CC BY 4.0 호환성 확인 (부록 F)

안전 스캔

유해 콘텐츠, 보안 취약성 유도 스킬에 대한 자동 스캔 수행

검색 시스템 및 에이전트 통합

  • 모든 스킬이 SQLite FTS5 전문 검색 인덱스로 번들화
  • 오프라인 환경에서도 즉시 검색 가능
  • OpenClaw 에이전트 프레임워크의 스킬 레지스트리와 직접 통합 가능
  • 도메인별 번들 선택적 로딩으로 메모리 효율적 운용

한계 및 윤리적 고려

주요 한계

  • SkillGate는 내부 QA 신호이며 외부 검증 아님
  • 소스 기반성이 Hallucination을 완전히 제거하지는 못함
  • 커뮤니티 스킬(GitHub/ClawHub)의 품질 편차 존재
  • 평가가 제한된 도메인·솔버 조합에서 수행됨

윤리적 고려

  • 라이선스 감사로 CC BY 4.0 호환성 보장
  • 안전 스캔으로 유해 스킬 사전 차단
  • 에이전트 자율성 증대에 따른 책임 소재 문제 인식
  • Datasheet (부록 H)로 투명한 데이터셋 문서화

비전 및 향후 방향

코퍼스 확장
더 많은 도메인, 언어, 실시간 업데이트
품질 향상
외부 검증 루프, 사용자 피드백 반영
안전 가드레일
도메인별 보안 정책, 실행 시 검증
지속 가능한 성장
자동화된 파이프라인으로 지속적 갱신
에코시스템 비전
에이전트 프레임워크 간 상호운용 스킬 표준

궁극적 목표: 에이전트가 인간의 운영적 판단을 구조화된 스킬로 대체할 수 있는 인프라 구축
