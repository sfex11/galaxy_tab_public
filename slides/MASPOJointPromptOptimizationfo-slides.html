왜 다중 에이전트 프롬프트 최적화가 어려운가?

조합적 탐색 공간

N개 에이전트의 프롬프트를 동시에 최적화해야 하며, 하나의 프롬프트 변경이 다른 모든 에이전트에 영향을 미칩니다. 검색 공간은 𝒮N으로 기하급수적으로 확장됩니다.

국소-전역 부정합

중간 에이전트가 자신의 역할을 완벽히 수행하더라도, 그 출력이 하류 에이전트를 오도해 시스템 전체가 실패하는 치명적 문제가 발생합니다.

신뢰 할당 문제

최종 정답만 존재하는 상황에서, 시스템 실패의 원인이 어느 에이전트의 프롬프트인지 특정하기 어렵습니다. 기존 방법들은 이를 해결하지 못합니다.

비정상성 최적화

상류 에이전트의 프롬프트가 변경되면 하류 에이전트의 입력 분포가 변하여, 이전에 최적이던 프롬프트가 더 이상 유효하지 않게 됩니다.

핵심 문제: Local-Global Misalignment

"중간 에이전트가 국소적 지시를 완벽히 충족하면서도, 하류 동료 에이전트를 오도하는 출력을 생성하여 시스템 전체의 실패를 초래하는 현상"

1
추론 에이전트
자신의 역할은 완벽히 수행 국소 성공
2
요약 에이전트
잘못된 형식의 입력 수신 전파 오류
3
답변 에이전트
오도된 정보로 잘못된 최종 출력 전체 실패

기존 단일 에이전트 프롬프트 최적화는 각 에이전트를 독립적으로 평가하므로 이 문제를 탐지할 수 없습니다.

다중 에이전트 시스템의 형식화

유향 통신 그래프

MAS를 유향 그래프 𝒢 = (𝒱, ℰ)로 모델링합니다.

  • 𝒱 = {vi}N — N개의 에이전트 집합
  • ℰ ⊆ 𝒱 × 𝒱 — 통신 토폴로지 (방향 간선)
  • 𝒫 = {pi}N — 최적화 대상: 역할별 프롬프트 집합

방향 간선 (vj, vi) ∈ ℰ은 vj의 출력이 vi의 입력으로 사용됨을 의미합니다.

최적화 목표

𝒫* = argmax𝒫∈𝒮N 𝔼(q,o*)∼𝒟 [ R(Φ(𝒢,𝒫,q), o*) ]

세 가지 근본적 난관:

  • 탐색 공간 𝒮N이 조합적·고차원
  • 토큰의 이산성으로 미분 불가능
  • 에이전트 간 기능적 결합으로 비정상성

토폴로지 스케줄링과 추적 기반 제안

좌표 상승 기반 스케줄링

모든 에이전트를 동시에 최적화하는 것은 비현실적이므로, 토폴로지 순서를 따라 한 번에 하나의 에이전트를 선택합니다.

  • 상류 에이전트를 먼저 최적화
  • 하류 에이전트는 업데이트된 상류 컨텍스트 반영
  • 비정상성을 체계적으로 관리

추적 기반 프롬프트 생성

프롬프트 최적화기가 실제 실행 흔적(컨텍스트 𝒞와 출력 o)을 분석하여 후보 프롬프트를 생성합니다.

  • 현재 반복 배치 ℬiter에서 추적 수집
  • 부정합 사례 배치 ℬmis에서 실패 패턴 주입
  • 고정된 후보 풀이 아닌 개방형 생성

다중 입자도 통합 평가 메커니즘

신뢰 할당 문제를 해결하기 위해 세 가지 차원을 융합한 통합 보상 모델을 설계

DIMENSION 1

국소 유효성 (LV)

에이전트가 자신의 역할 지시를 얼마나 잘 준수하는지 평가합니다. 기존 단일 에이전트 평가와 유사한 기본 척도입니다.

DIMENSION 2

전망 잠재력 (LP)

현재 에이전트의 출력이 직계 후속 에이전트의 성공에 기여하는 잠재력을 평가합니다. 인과 사슬의 한 단계를 내다봅니다.

DIMENSION 3

전역 정합 (GA)

해당 에이전트의 기여가 최종 시스템 출력의 품질로 어떻게 이어지는지 종단 간 평가합니다.

핵심 아이디어: 에이전트를 고립된 출력이 아닌 협업 인과 사슬 내의 기여도로 평가하여 국소-전역 부정합을 극복합니다.

부정합 인식 샘플링

부정합 사례(Misalignment Cases)란?

에이전트가 국소적 역할은 충족했으나 시스템 전체 실패를 초래한 과거 실행 흔적을 명시적으로 발굴합니다.

역사적 트레이스에서 LV가 높지만 GA가 낮은 사례 탐색
발굴된 사례를 최적화기의 입력 배치 ℬmis에 주입
최적화기가 조율 실패의 원인을 진단하고 수정 방향 탐색

왜 효과적인가?

일반적인 샘플링은 성공/실패 사례를 무차별적으로 수집하지만, 부정합 인식 샘플링은 가장 교육적인 신호를 집중적으로 타겟팅합니다.

일반 실패 사례와 달리, 부정합 사례는 "왜 국소적으로는 맞았는데 전체적으로 틀렸는가"라는 질문에 대한 직접적인 단서를 제공합니다.

적응적 진화 빔 탐색

추적 기반 빔 탐색

고차원 프롬프트 공간을 탐색하기 위해 Top-K 후보를 유지하는 빔 탐색을 사용합니다.

  • 각 빔 노드는 프롬프트 후보를 나타냄
  • 경로를 따라 누적 통합 보상 점수 계산
  • 실행 흔적을 기반으로 후보 반복적 진화
  • 기존 TPE 기반 베이지안 탐색과 달리 개방형 생성 가능

빔 리프레시 메커니즘

다중 에이전트 환경의 비정상성을 해결하는 핵심 장치입니다.

1
상류 에이전트 프롬프트 업데이트 완료
2
하류 에이전트의 입력 분포 변화 감지
3
기존 빔 트리를 폐기하고 새로운 컨텍스트로 재초기화
4
변경된 동료 행동에 적응된 탐색 재시작

강건성 검증

  • 초기화 무감성: 무작위·수동·템플릿 초기화 모두에서 안정적 수렴
  • 이전 가능성: 한 토폴로지에서 최적화된 프롬프트를 다른 토폴로지로 전이 시 유의미한 성능 유지
  • 계산 예산: 최적화 라운드 증가에 따른 점진적 성능 향상
  • 평가자 능력: 더 강한 평가자 LLM이 더 나은 최적화 결과 산출
