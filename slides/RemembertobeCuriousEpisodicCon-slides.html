3D 탐색

Lily Goli, Justin Kerr, Daniele Reda, Alec Jacobson, Andrea Tagliasacchi, Angjoo Kanazawa

University of Toronto · UC Berkeley · Wayve · Vector Institute · Simon Fraser University

왜 호기심 기반 탐색인가?

"보상이 없어도 쥐는 미로의 구조를 학습한다"
— Edward Tolman, 잠재 학습 (1948)

현실의 문제

  • 시각-운동 내비게이션은 본질적으로 희소 보상
  • 목표 조건부 방법도 결국 성공 신호에 의존
  • 명시적 목표 없이도 환경을 이해해야 함

호기심의 접근

  • 세계 모델의 예측 오차을 내재적 보상으로 사용
  • 외부 보상 없이 자가 학습 가능
  • 탐색 → 밀도 높은 경험 → 하위 작업 전이

"건망증" 탐색의 두 가지 실패 모드

기존 호기심 기반 RL은 복잡한 3D 환경에서 반복 행동에 빠진다. 그 원인은 두 가지 결핍에 있다.

실패 1: 에피소드 맥락 부재

과거 이력 없이 현재 관찰만으로 행동 결정 → 같은 위치 반복 방문 → 루프에 갇힘

실패 2: 지속적 세계 모델 부재

통계적 사전만 학습하는 모델 → 이전에 본 상태를 잊어버림 → 재방문 시 허위 신규 보상 발생

두 문제가 동시에 발생하면: 이미 본 곳을 잊은 모델이 반복 방문에 신선한 보상을 주고, 맥락 없는 에이전트는 계속 그곳을 향한다.

핵심 통찰

1

지속적 세계 모델

환경의 영구적 기록을 유지하고, 지속적으로 업데이트되는 전방 모델 → 재방문 시 허위 보상 방지

2

에피소드 맥락 유지

과거 RGB 관찰 시퀀스를 조건화 → 에이전트가 내부 지도를 형성하고 되돌아가기(backtracking) 학습

  핵심 차별점: 기존 방법들은 명시적 지도(GPS, 깊이)를 사용하여 탐색을 단순화하지만, 본 방법은 순수 RGB 스트림만으로 end-to-end 학습하며 배포 시 지도가 불필요하다.

에이전트 (배포 시 전체)

RGB 스트림만으로 행동 생성

3DGS 전방 모델 (학습 시만)

깊이+포즈로 3D 재구성 후 렌더링 비교

지속적 3D 전방 모델 (3D Gaussian Splatting)

온라인 3DGS를 전방 모델로 사용하여 공간적 지속성을 확보한다.

관찰 누적: 각 RGB-D 프레임마다 픽셀당 가우시안 추가 → 컬러, 깊이, 포즈로 초기화
지속적 최적화: 정기적으로 과거 프레임 무작위 샘플링하여 재구성 품질 개선, 가지치기(prune) 및 밀도화(densify)
예측 렌더링: 새 포즈 pt+1에서 3DGS 렌더러로 예측 이미지 생성
호기심 보상: rcurt = d(Ît+1, It+1) — 렌더링과 실제 관찰 간 시각적 불일치

학습 시에만 RGB-D + 포즈 필요. 배포 시 에이전트는 순수 RGB로 동작.

에이전트 아키텍처 (Transformer 시퀀스 모델)

1
프레임 인코딩: 각 RGB 프레임을 패치 임베딩 + DINOv2 특징을 학습 가능한 쿼리로 융합하여 토큰 생성
2
인과적 시간 자기주의: Causal temporal self-attention으로 과거 프레임 간 의존성 모델링
3
선형 주의 + 전역 은닉 상태: 긴 맥락을 효율적으로 압축하여 유지
4
Actor-Critic 헤드: 다음 행동(at)과 가치 추정(Vt) 출력

설계 철학

  • 깊이 센서 불필요
  • 위치 추적(GPS) 불필요
  • 명시적 지도 구축 불필요
  • RGB 시퀀스만으로 내부 표현 형성
  • 의미적 다양성에 유연한 일반화

학습 및 정규화 어닐링

장기 수평 탐색의 보상 공백 문제

이미 본 영역을 지나 새로운 영역으로 향하는 경로에서는 호기심 보상이 거의 0에 가깝다. 에이전트가 이 "보상 없는 구간"을 건너지 못하고 정체된다.

해결책: 예정된 무작위 행동 주입

학습 초기에 ε-greedy 스타일로 무작위 행동을 일시적으로 주입한 뒤 점진적으로 감소시킨다.

결과적 새로운 행동

  • 이미 본 영역을 횡단하여 새로운 "가지"를 찾는 전략이 자발적으로 등장
  • 모방 학습 부트스트래핑 불필요
  • 계층적 목표 선택 불필요
  • 순수 호기심만으로 장기 탐색 해결

실험: 실내 장면 탐색 (HM3D)

HM3D에서 순수 호기심으로 학습한 후, 학습에 사용되지 않은 장면에서 평가

방법 탐색 면적 (%) 특징
Random Walk 낮음 기준선
ICM (Pathak et al.) 낮음 지속적 메모리 없음
Active Mapping (RL) 중간 명시적 지도 사용
ReCuriosity (Ours) 최고 지속적 3DGS + 에피소드 맥락

  명시적 지도를 사용하는 Active Mapping baseline보다 우수한 성능

메모리 소거 실험 (Ablation)

지속적 메모리의 중요성을 검증하기 위해 3DGS의 메모리를 인위적으로 제한

단기 메모리 (최근 N프레임만)

3DGS가 오직 최근 프레임만 유지하면 → 이전에 재구성한 영역을 잊어버림 → 재방문 시 허위 호기심 보상 발생 → 탐색 성능 급감

전체 지속적 메모리

모든 과거 관찰을 유지하며 지속적으로 업데이트 → 안정적인 보상 신호 → 에이전트가 진정으로 새로운 영역만 탐색

"지속성(persistence)은 안정적인 호기심 보상을 위한 필수 조건이다. 메모리가 단기로 제한되면 탐색 능력이 현저히 저하된다."

하위 작업 미세조정 (Fine-tuning)

호기심으로 사전학습된 정책을 희소 보상 하위 작업에 단 몇 에피소드만으로 적응

사과 수확

희소 보상: 사과에 도달 시만 보상

이미지 목표 내비게이션

목표 이미지와 일치하는 위치로 이동

결과

처음부터 학습한 baseline 우회

  End-to-end 설계의 이점: 사전학습된 시각-운동 표현이 하위 작업으로 원활하게 전이된다. 모듈식 지도 기반 방법에서는 어려운 의미적 적응이 가능하다.

제로샷 일반화 (Zero-shot Transfer)

HM3D에서만 학습하고, 완전히 새로운 환경에 추가 학습 없이 적용

Gibson

다른 실내 스캔 데이터셋. HM3D와 다른 건물 구조, 조명, 해상도

제로샷 전이 성공

AI 생성 3D 월드

학습 중 한 번도 본 적 없는 완전히 AI가 생성한 3D 환경

OOD 일반화 성공
명시적 지도에 의존하지 않는 RGB-only 정책이기에, 기하학적으로 정의 가능한 작업에만 국한되지 않고 의미적으로 다양한 환경에 일반화할 수 있다.

결론 및 핵심 기여

문제 진단: 호기심 탐색의 실패는 "건망증"에서 비롯됨 — 에피소드 맥락과 지속적 메모리의 동시 결핍
지속적 세계 모델: 온라인 3DGS로 공간적 지속성 확보, 허위 신규 보상 방지
에피소드 맥락 에이전트: RGB 시퀀스 Transformer로 내부 지도 형성, 되돌아가기 자발적 학습
실용적 결과: Active Mapping 우회, OOD 제로샷 일반화, 효율적 하위 작업 전이

향후 방향

  • 동적 환경으로 확장 (현재 정적 가정)
  • 3DGS → action-conditioned video model (LingBot-World 등의 공간 지속성 개선 시)
  • 더 복잡한 로봇 작업으로의 전이
  • 다중 에이전트 협동 탐색
