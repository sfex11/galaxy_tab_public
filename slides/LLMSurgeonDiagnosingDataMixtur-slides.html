LLMSurgeon

대형 언어 모델의 사전학습 데이터 혼합 비율 진단

Yaxin Luo, Jiacheng Cui, Xiaohan Zhao, Xinyi Shang, Jiacheng Liu, Xinyue Bi, Zhaoyi Li, Zhiqiang Shen

VILA Lab, MBZUAI / UCL

디지털 DNA의 비밀

"이 LLM은 무엇으로 학습되었는가?"

현대 LLM의 사전학습 데이터 구성은 가장 잘 보호된 비밀 중 하나입니다. 데이터 혼합 비율은 모델의 행동, 능력, 실패 모드를 결정하는 "디지털 DNA"이지만, 대부분 공개되지 않습니다.

안전 감사

편향 데이터 포함 여부 확인 불가

저작권 리스크

특정 데이터셋 사용 여부 불투명

성능 격차

도메인별 성능 편차 원인 규명 불가

기존 접근법의 한계: 멤버십 추론 공격

MIA의 강점

  • 특정 문서의 학습 여부 판별
  • 개인정보 유출 감지에 효과적
  • 미시적(microscopic) 분석 가능

MIA의 한계

  • 단일 샘플 단위 — 거시적 분석 불가
  • 짧은 토큰 윈도우에 의존
  • 수백만 샘플 집계 시 치명적 오류 누적

모래알 한 알은 감지할 수 있어도, 해변의 풍경은 묘사할 수 없다.

데이터 혼합 수술(DMS): 문제 정의

대상 LLM이 생성한 텍스트만을 입력으로 받아, 사전학습 코퍼스의 도메인 수준 분포를 추정하는 과제입니다.

학습 코퍼스의 혼합 분포:

pα(x) = Σ αi · pi(x)

생성 분포 (복구 대상):

qπ(x) = Σ πi · pi(x)

입력

LLM 생성 텍스트 Xgen

출력

도메인 혼합 벡터 π

제약

가중치/학습 데이터 접근 불가

라벨 시프트 가설

DMS의 핵심 가정: 도메인의 비율은 변하더라도, 각 도메인의 언어적 특성은 통계적으로 불변한다.

변하는 것: 도메인 비율

학습 시 코드 비율 αcode ≠ 생성 시 코드 비율 πcode

최적화 동역학, 언더피팅, 샘플링 온도 등으로 인해 발생

불변하는 것: 조건부 분포

q(x|y=i) ≈ p(x|y=i)

모델이 "코드"를 생성할 때, 학습에 사용된 코드의 통계적 패턴을 유지

핵심 통찰: 이 안정성 덕분에 π 복구를 분포 매칭 문제로 공식화할 수 있다

LLMSurgeon 프레임워크 개요

1
편향 특성화
외부 분류기 학습 후 교정된 소프트 혼동 행렬 계산
2
관측 획득
대상 LLM의 중립 샘플링 → 분류기로 편향된 관측 벡터 획득
3
역수술(Inverse Surgery)
혼동 행렬로 관측 "디블러링" → 진짜 π 복구

핵심 차별점

단순 집계가 아닌, 수학적 역문제로 체계적 편향을 보정하여 충실도 높은 비율 복구

중립 샘플링

프롬프트/정렬로 인한 스타일 왜곡을 최소화하여 사전학습 prior를 충실히 반영

단계 1: 교정된 소프트 혼동 행렬

외부 분류기는 유사 도메인 간에 체계적 오분류 편향을 가집니다. 이를 정량화하여 보정합니다.

하드 혼동 행렬

전통적 방식: 단일 예측 라벨만 기록

→ 정보 손실 큼, 유사 도메인 구별 불가

소프트 혼동 행렬

본 연구: 확률 분포 전체를 활용

→ 미세한 도메인 간 편향까지 포착

참조 데이터셋에서 학습된 분류기의 K×K 혼동 행렬 C를 사전 계산하여 모든 도메인 쌍 간의 체계적 편향을 인코딩

단계 2–3: 관측과 역수술

2

편향된 관측 벡터 획득

대상 LLM에서 중립 프롬프트로 텍스트 생성 → 고정 분류기로 분류 → 관측 분포 μ 획득 (혼동 행렬 C로 인해 편향됨)

3

제약 기반 역문제 해결

μ = C · π 관계를 활용, π ≥ 0, Σπi = 1 제약 하에서 최적화하여 진짜 혼합 비율 π 복구

왜 직접 집계는 실패하는가?

  • 토큰 한계: MIA는 짧은 시퀀스에 설계되어 문서 단위 확장 불가
  • 오류 누적: 수백만 포인트별 예측 오차가 합산되어 분산 폭발
  • 알고리즘 편향: 도메인별 MIA 성능 차이(예: 코드 vs 일반 텍스트)가 체계적 왜곡 유발

LLMScan 벤치마크

DMS 평가를 위한 최초의 레시피 검증 가능한 벤치마크입니다. 합성 데이터가 아닌, 사전학습 레시피가 투명한 오픈소스 LLM으로 구성됩니다.

모델 유형 대상 모델 사전학습 코퍼스 세분화 수준 도메인 수 K
범용 모델 LLaMA-1 (7B, 65B) Public Mix 조대분류 6
범용 모델 OLMo (1B) Dolma 조대분류 6
범용 모델 Amber (13B) LLM360 Mix 조대분류 6
Pile 기반 Pythia (2.8B, 12B) The Pile 중분류 17
Pile 기반 GPT-Neo (2.7B) The Pile 중분류 17
도메인 특화 StarCoder (15.5B) The Stack 세분류 87

실험 결과

LLMSurgeon은 LLMScan 벤치마크 전반에서 집계 기반 베이스라인을 큰 폭으로 능가합니다.

조대분류 (K=6)

높은 충실도

LLaMA, OLMo, Amber에서 도메인 혼합 비율을 정확히 복구. 직접 집계 방식 대비 체계적 편향 대폭 감소.

중분류 (K=17)

안정적 성능

Pythia, GPT-Neo에서도 혼동 행렬 보정 효과가 지속적으로 나타남. 도메인 수 증가에도 견고.

세분류 (K=87) — StarCoder

87개 프로그래밍 언어/도메인 세부 범주에서도 LLMSurgeon의 역문제 접근법이 유효함을 확인. 소프트 혼동 행렬이 미세한 도메인 간 편향 포착에 필수적.

제어 실험 및 일반화

절제 연구

  • 소프트 vs 하드 혼동 행렬: 소프트가 유의미하게 우수
  • 샘플 수에 따른 수렴성 분석
  • 분류기 아키텍처 선택의 영향

Held-out 일반화

  • 학습에 사용하지 않은 모델에 대한 일반화 능력
  • 제어된 혼합 비율 실험에서의 정확도
  • 미지의 모델 아키텍처에 대한 견고성

안전 감사 응용: 독성 주입 트리아지

LLMSurgeon을 활용하여 모델의 사전학습 데이터에 유해 도메인이 포함되었는지 사후 감사(triage)하는 응용 시나리오를 제시. 독성 데이터 주입 수준을 정량적으로 추정 가능.

핵심 기여 요약

1

문제 공식화

멤버십 추론에서 분포 복구로 초점을 전환한 Data Mixture Surgery (DMS) 문제를 최초로 공식화

2

방법론 제안

라벨 시프트 가설 하에서 교정된 소프트 혼동 행렬과 제약 기반 역문제 해결을 결합한 LLMSurgeon 프레임워크

3

벤치마크 구축

투명한 사전학습 레시피를 갖춘 오픈소스 LLM 기반 최초의 검증 가능한 DMS 평가 제품 LLMScan

가중치나 학습 데이터에 접근하지 않고도 파운데이션 모델의 디지털 DNA를 사후 감사하는 실용적 접근법을 제시합니다.

한계점 및 향후 과제

현재 한계

  • 사전 정의된 분류 체계(Taxonomy)에 의존
  • 라벨 시프트 가설이 완벽히 성립하지 않을 수 있음 (정렬, 프롬프트 영향)
  • 학습 비율 α과 생성 비율 π 간의 간극을 완전히 제어하지 못함
  • 분류기 참조 데이터의 품질에 민감

향후 방향

  • 개방형 도메인 발견 (사전 정의 없이 자동 분류)
  • 정렬 단계의 영향을 명시적으로 모델링
  • 다국어 모델로의 확장
  • 규제·거버넌스 실제 적용 시나리오 연구
