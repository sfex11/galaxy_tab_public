NeuralUCB 접근법

비용-품질 균형을 위한 문맥적 밴딧 기반 대규모 언어 모델 라우팅

01 / 동기

왜 LLM 라우팅이 필요한가?

💰

비용 격차

프리미엄 모델(GPT-4 등)은 높은 품질을 제공하지만 추론 비용이 크게 높음. 경제적 모델은 복잡한 추론/프로그래밍에서 성능 저하

🎯

핵심 문제

모든 쿼리에 최고급 모델을 사용하면 비용 폭증, 저렴한 모델만 사용하면 품질 저하 — 최적 균형점이 필요

⚠

기존 방법의 한계

  • 지도학습 라우팅: 대량의 레이블 데이터 필요
  • 분포 변화 시 재학습 필수
  • 온라인 적응성 부족
💡

제안 접근법

NeuralUCB: 문맥적 밴딧 프레임워크로 비선형 정책을 효율적으로 학습하면서 비용-품질 트레이드오프를 명시적으로 모델링

03 / 관련 연구

기존 접근법 비교

📊

지도학습 라우팅 (Full-Information)

RouterDC GraphRouter
  • 모든 후보의 출력과 점수로 분류기/랭커 학습
  • 높은 데이터 수집 비용
  • 분포 변화 시 재학습 필요
  • 높은 초기 정확도, 낮은 적응성
🔮

부분 피드백 방법 (Bandit)

LinUCB NeuralUCB
  • 선택된 행동의 보상만 관측
  • LinUCB: 선형 가정 → 제한적
  • 커널 방법: 고차원에서 계산 비효율
  • NeuralUCB: 비선형 + 비용 모델링
NeuralUCB의 장점
비선형 쿼리-모델 매핑을 효율적으로 학습하면서도 불확실성 추정을 통한 탐색-활용 균형을 유지. 비용-품질 트레이드오프를 통합 보상 프레임워크로 직접 모델링

05 / 핵심 알고리즘

NeuralUCB 라우팅 정책

특성 표현 (마지막 은닉층)
g(x,a) = [h(x,a); 1]
UCB 점수
s(x,a) = μ(x,a) + β √(gT A-1 g)
안전 폴백
asafe = argmaxa μ(x,a)
최종 행동 선택
a* = argmax s(x,a)  if p(x)≥τg
     asafe           otherwise

구성 요소 해석

  • μ(x,a): 예측 유틸리티 (활용)
  • β√(...): 불확실성 보너스 (탐색)
  • A-1: 공유 역공분산 행렬
  • p(x): 게이팅 확률
  • τg: 게이팅 임계값

하이퍼파라미터

  • 학습률: 10-3
  • UCB 계수 β: 1
  • 릿지 정규화 λ: 1
  • 학습 에폭: 5 (슬라이스당)

09 / 기여 및 결론

핵심 기여와 향후 과제

✔

주요 기여

  • 1 NeuralUCB를 비용 인식 LLM 라우팅에 최초 적용
  • 2 통합 유틸리티 보상 함수로 비용-품질 동시 최적화
  • 3 공유 공분산 행렬로 행동별 통계 유지 불필요
  • 4 게이팅 메커니즘으로 문맥 의존적 탐색-활용 균형
🚧

한계 및 향후 과제

  • 1 비용 제약 하 행동 식별력 향상 필요
  • 2 최적-품질과 보상-최적 결정의 높은 중복
  • 3 추가 학습 데이터 확보 필요
🔍

핵심 결론

NeuralUCB는 ~67% 비용 절감과 경쟁력 있는 품질을 동시에 달성하며, 지도학습 및 단순 베이스라인을 일관되게 초과

한 줄 요약
NeuralUCB 기반 라우팅은 온라인 환경에서 비용 효율적이고 적응적인 LLM 선택을 가능하게 하는 유망한 프레임워크이다
