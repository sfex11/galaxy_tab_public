LLM이 절차를 따르지 않을 때:
언어 모델의 절차적 실행에 대한 진단 연구

When LLMs Stop Following Steps: A Diagnostic Study of Procedural Execution in Language Models

Sailesh Panda, Pritam Kadasi, Abhishek Upperwal, Mayank Singh

Indian Institute of Technology Gandhinagar  ·  Soket AI

왜 이 연구가 필요한가?

겉보기 성공 ≠ 진정한 추론

LLM은 산술 추론, 기호 조작, 코드 생성 등에서 강한 성능을 보이지만, 최종 정답의 정확도만으로 모델이 프롬프트에 지정된 절차를 충실히 수행했는지 알 수 없습니다.

숨겨진 실패 모드

  • 단축키 및 패턴 매칭에 의존
  • 불안정한 중간 계산
  • 필요한 단계 생략 또는 추가 단계 헛지시
  • 이전 오류 후 우연히 정답 도달

"명시적 알고리즘이 프롬프트에 제공되어도 LLM은 여전히 이를 실행하는 데 실패한다" — Shojaee et al.

연구 질문

RQ1

알고리즘 길이와 의존성이 증가할 때, LLM은 명시적 다단계 절차를 얼마나 안정적으로 실행하는가?

절차의 길이(5~95단계)와 look-back 의존성 깊이(1~7)를 변화시키며 정확도 추이 분석

RQ2

모델이 실패할 때 어떤 생성 수준의 실패 모드가 나타나며, 이는 최종 정답 정확도와 어떤 관계가 있는가?

누락 답변, 조기 답변, 자기 교정, 미실행, 헛지시 단계 등 세부 분석

핵심 개념: 절차적 실행(Procedural Execution) — LLM이 명시적으로 지정된 절차를 그대로 수행하는 능력

과제 정의

자연어로 작성된 단계별 산술 알고리즘과 두 개의 숫자 입력을 주고, 최종 계산값만 반환하도록 요청

예시: 5단계 절차 (x=12, y=3)

Let S₁ = 12,   S₂ = 3
Step 1: S₃ = S₁ + S₂   → 15
Step 2: S₄ = S₃ − S₂   → 12
Step 3: S₅ = S₄ / S₂   → 4
Step 4: S₆ = S₅ × S₃   → 60
Step 5: S₇ = S₆ − S₄   → 48
Final Step: Return S₇
r₀ = x

초기 실행 값

rₜ = f(rₜ₋₁, y; sₜ)

재귀적 연산 정의

{+, −, ×, ÷}

입력 범위, 데이터 유형, 작업 유형만으로는 성능 저하를 완전히 설명할 수 없음

입력 범위

데이터 유형

미실행이 지배적

모델이 지정된 절차를 끝까지 따르지 못하고 중간에 멈추는 현상이 가장 흔한 실패 원인

고립된 산술 오류가 아님

개별 연산 실수가 아니라 절차를 잃어버리는 것이 핵심 문제

실패 모드 분류

모델이 보이는 5가지 주요 실패 패턴

누락 답변 (Missing Answer)

생성 전체에 유효한 최종 답변이 없음

조기 답변 (Premature Answer)

모든 단계를 마치기 전에 답변 출력

자기 교정 (Self-correction)

초기 오류 후 스스로 수정 시도

미실행 (Under-executed Trace)

지정된 단계보다 적게 수행 후 종료

헛지시 단계 (Hallucinated Steps)

존재하지 않는 추가 단계를 생성

이러한 실패는 최종 정답 정확도만으로는 진단 불가

Correct@any
vs First

Correct@any이 First-answer보다 높음 → 자기 교정이 종종 발생하지만 안정적이지 않음

모델은 "언제 멈춰야 하는지"를 학습하지 못하며, 긴 절차에서는 종료 조건 인식이 특히 취약함

종합 발견

  • LLM의 추론 능력처럼 보이는 성과가 실제 지시 수행 능력을 가리는 경우가 많다
  • 단순 산술이 아닌 절차적 상태 추적이 병목; 길이와 비국소 의존성에 취약
  • 가장 흔한 실패는 연산 오류가 아니라 미실행(절차 중단)
  • 입력 범위·데이터 유형 변화는 절차적 복잡도 효과에 비해 부수적
  • 최종 정답 정확도 단일 지표로는 충실한 실행 여부를 판단할 수 없음

시사점: 추론 벤치마크 해석에 주의가 필요하며, 절차적 충실도를 직접 측정하는 진단 도구가 필수

결론 및 향후 과제

결론

본 벤치마크는 LLM이 지시를 진정으로 실행하는지 평가하는 유용한 렌즈를 제공합니다. 최종 정답만으로는 판별할 수 없는 숨겨진 약점을 체계적으로 드러냅니다.

한계 & 향후 과제

  • 산술에 국한된 설정 → 다른 연산 영역으로 확장 필요
  • 텍스트 기반 절차(규칙 기반 의사결정, 정책 실행 등)로 일반화
  • 실패 원인의 미시적 분석 (어텐션 패턴, 중간 표현 등)
  • 절차적 충실도를 개선하는 학습·추론 전략 탐색
