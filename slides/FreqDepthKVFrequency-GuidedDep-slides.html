FreqDepthKV

주파수 기반 깊이 공유를 통한
장문맥 LLM 추론에서의 견고한 KV 캐시 압축

Anna Córdoba, Adam Puente Tercero, Nerea Angulo Hijo,
Mar Linares Tercero, Julia Barrientos, Ainhoa Miranda, Jesús Olivera

Instituto de Investigación en Visión Artificial

문제 정의

KV 캐시 병목

자가회귀 디코딩 시 모든 생성 토큰이 이전 전체 토큰의 키-값 캐시를 참조하여 메모리 트래픽과 캐시 크기가 컨텍스트 길이에 비례하여 증가

공격적 압축의 한계

검색 및 다단계 추론에 필요한 레이어 특화 증거가 제거되어 정답이 바뀌는 새로운 실패 모드 발생

기존 방법들은 중복성을 토큰 수준이나 정밀도 수준에서만 다루며, 인접 레이어 간 깊이 차원의 중복성은 충분히 탐구되지 않음

핵심 통찰

저주파 깊이 성분 — 인접 레이어가 공유

대부분의 레이어는 유사한 표현을 저장하므로 저주파 성분을 블록 단위로 1회만 저장

고주파 잔차 — 희소하게 보존

바늘 문장 판별, 코드 의존성 유지, 추론 사슬 보존에 결정적인 레이어 특화 정보

어텐션 로짓 기반 온라인 라우팅

각 헤드마다 어느 모드를 사용할지 프리필 시점에서 경량 프로브로 결정

관련 연구 포지셔닝

토큰 선택

H2O, StreamingLLM, SnapKV, PyramidKV — 시퀀스 차원 축소

후속 레이어 합성 후에야 결정적인 증거를 버릴 위험

양자화

KVQuant, KIVI — 수치 정밀도 축소

FreqDepthKV와 직교, 병행 적용 가능

깊이 공유

MiniCache 등 — 균일한 레이어 병합

고주파 잔차 손실로 검색 정확도 저하

FreqDepthKV는 깊이 차원을 직접 공략하면서, 균일 공유를 주파수 인식 잔차 보존으로 대체

방법론 개요

1
깊이 스택 구성
B개 인접 레이어의 KV 캐시를 깊이 축으로 적층
2
DCT 주파수 분해
직교 깊이 변환 F_B로 저주파/고주파 분리
3
온라인 프로브
프리필 시 어텐션 로짓 비교로 헤드별 모드 할당
4
적응형 캐시 배치
모드별로 다른 압축率 적용 후 디코딩

DCT 기저는 고정되어 있어 프리필 오버헤드가 무시 가능하며, 모델 재학습이 불필요함

깊이-주파수 분해

레이어 블록 b의 헤드 h에 대해 KV 캐시를 깊이 축으로 적층:

Xb,hK = [Kℓ,h]ℓ∈b ∈ ℝB × T × dh

고정 DCT 기저 FB를 적용하여 주파수 분해:

Zb,hK = FB Xb,hK,    Xb,hK = FB⊤ Zb,hK

저주파 성분 (첫 번째 계수)

레이어 블록당 1회만 저장 → 깊이 차원 B배 압축

고주파 잔차 (나머지 계수)

선택된 토큰-헤드 쌍에 대해서만 희소 저장

세 가지 캐시 모드

Shared-Depth

저주파 계수만 유지. 인접 레이어와 거의 동일한 어텐션 패턴을 보이는 헤드에 할당

최대 압축 효과

Residual-Depth

공유 성분 + 희소 잔차 세트 유지. 검색에 민감한 헤드에 할당

정확도-압축 균형

Exact

원본 KV 캐시 그대로 유지. 압축 시 어텐션 로짓 변화가 큰 헤드에 할당

안전 보장

프로브 위치 P는 최근 토큰, 문서 경계, 고엔트로피 어텐션 행에서 샘플링

재구성 인식 라우팅 손실

프리필 시 캐시된 어텐션 로짓에서 계산되는 라우팅 손실:

  • 원본 캐시 vs 후보 재구성 캐시의 어텐션 로짓 차이를 정량화
  • 로짓 변화가 작은 헤드 → shared-depth 모드 우선 할당
  • 로짓 변화가 큰 헤드 → residual-depth 또는 exact 모드 할당
  • 컨텍스트 길이와 프롬프트 구조에 따라 압축 비율 자동 적응

모델 재학습 없이 추론 시점(inference-time)에서만 동작하며, 표준 자가회귀 디코딩과 완전 호환

실험 설정

벤치마크

  • 장문맥 질의응답 (Long-context QA)
  • 바늘 검색 (Needle Retrieval)
  • 요약 (Summarization)
  • 코드 생성 (Code Generation)

설정

  • 프리필 윈도우: 32K 토큰
  • 레이어 블록 크기 B = 2 또는 4
  • 고정 DCT 기저 (재학습 불필요)
  • Full KV 캐시 및 기존 압축 방법과 비교
