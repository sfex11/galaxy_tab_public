01 · 문제 정의

에이전틱 워크플로우의 서빙 과제

핵심 문제

에이전틱 워크플로우는 여러 LLM과 도구를 조합하여 복잡한 작업을 수행합니다. 이를 목표 처리량과 낮은 지연시간으로 서빙하는 것이 매우 어렵습니다.

예측 불가능한 실행

  • 분기(Branching) — 데이터에 따라 다른 경로
  • 팬아웃(Fan-out) — 다수 후보 동시 생성
  • 재귀(Recursion) — 반복적 추론 루프

GPU 자원 부족

  • LLM 수가 GPU 수를 초과
  • 임의의 프레임워크 지원 필요
  • 정적 분석으로는 한계

02 · 핵심 통찰

안정적인 상대 비율의 발견

핵심 관찰

엔드투엔드 지연시간은 크게 변동하지만, 각 LLM의 총 실행 시간 비율은 실행 간에 비교적 안정적입니다.

절대 실행 시간
9~264초
29배 변동 — 예측 불가
상대 실행 비율
4배 안정
절대 시간 대비 — 예측 가능!

이것이 의미하는 바

개별 LLM의 실행 시간은 예측할 수 없지만, 전체 워크플로우에서 각 LLM이 차지하는 비중은 예측 가능합니다. 이 비율을 활용하면 GPU 자원을 효율적으로 배분할 수 있습니다.

이 통찰이 Scepsy의 핵심 기반

지연시간 수식

Lw(λw) = Σm Lm(λw · nm) · (nm / pm)

Lm = LLM m의 지연시간  |  nm = 호출 횟수  |  pm = 요청 병렬성

처리량 수식

Tw = minm (Tm / nm)

07 · 실험 워크플로우

평가 대상 에이전틱 워크플로우

Beam Search (추론 시간 스케일링)

  • 생성기: Llama-3.2-1B
  • 검증기: Llama-3.1-8B-PRM
  • 반복적 빔 확장 + 점수 매김
  • 호출: 24~844회/요청
  • 실행: 9~264초
  • 팬아웃 + 순차 의존성

  • 생성기: Llama-3-8B
  • 문서 검색 + 생성 + 재순위
  • 검색 + LLM 조합
  • 비-LLM + LLM 혼합
  • 비교적 예측 가능한 실행

12 · 한계 및 향후 연구

현재 한계와 미래 방향

LLM 간 병렬성

파이프라인이 직렬 LLM을 가정합니다. 서로 다른 LLM으로의 팬아웃 시 부정확한 모델링이 발생할 수 있습니다.

해결: 임계 경로 기여도를 별도 추적

이중 모달 부하 분포

하나의 LLM이 여러 워크플로우나 역할에 재사용되면 이중 모달 분포가 발생합니다.

해결: 사용 패턴별 트레이스 분류, 패턴별 파이프라인 구축

비-무시 도구 실행

도구 실행이 수 밀리초 수준이라 가정하지만, 긴 모니터링 작업 등이 포함되면 지연시간을 지배할 수 있습니다.

해결: 고정 지연시간 바이어스의 비-최적화 도구 단계 추가
