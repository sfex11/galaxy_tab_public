🎯 문제 정의

다중 사족 로봇 시스템의 안전한 궤적 계획은 왜 어려운가?

❌ 중앙 집중식의 한계

  • 로봇 수 증가 → 계산 비용 기하급수적 증가
  • 단일 장애점(Single Point of Failure)
  • 통신 대역폭 병목
  • 실시간 제어 불가능

⚠️ 안전성 결합 문제

  • CBF 제약조건 → 에이전트 간 명시적 결합 발생
  • 충돌 회피를 위한 상호 의존성
  • 기존 분산 방식으로는 직접 분해 불가
  • 안전 보장과 분산화의 동시 달성 난제

핵심 질문

안전성(CBF)을 보장하면서도 완전 분산화된 MPC를 어떻게 구현할 수 있을까?

💡 핵심 기여

01  노드-에지 분할 기법

ADMM 기반의 새로운 노드-에지 분할(Node-Edge Splitting) 공식화로 CBF 결합 문제를 해결. 전역 문제를 독립적인 노드-로컬 및 에지-로컬 QP로 분해

02  완전 분산 MPC 프레임워크

이웃 간 통신만으로 독립적 병렬 QP 풀이 가능. 대칭적 계산 분배로 확장성 확보

03  3단계 계층적 아키텍처

분산 궤적 계획(상위) + 비선형 MPC(중위) + 전신 제어(하위)의 3단계 계층 구조 설계

04  실제 하드웨어 검증

Unitree Go2 사족 로봇 2대에서 실시간 분산 제어 구현 및 검증. 시뮬레이션에서 최대 4대 로봇으로 확장

🛡️ MPC + CBF 공식화

모델 예측 제어 (MPC)

min Σₖ [ ‖xₖ - x*ₖ‖²_Q + ‖uₖ‖²_R ] s.t. xₖ₊₁ = Aₖxₖ + Bₖuₖ (선형화된 동역학) xₖ ∈ 𝒳, uₖ ∈ 𝒰 (상태/입력 제약)

각 로봇의 무게중심(CoM) 궤적을 예측 구간에 걸쳐 최적화

제어 장벽 함수 (CBF) 제약

h(xᵢ, xⱼ) = ‖pᵢ - pⱼ‖² - d²_safe ≥ 0 Δh(xᵢ, xⱼ) + γ · h(xᵢ, xⱼ) ≥ 0, γ ∈ (0, 1]

로봇 i와 j 사이의 안전 거리 d_safe를 보장하는 명시적 안전 제약

⚡ 핵심 문제

CBF 제약이 xᵢ와 xⱼ를 동시에 포함 → 에이전트 간 결합 발생
→ 이것이 바로 분산화를 어렵게 만드는 원인!
